Общая информация
================

Этот документ содержит полезную информацию, которую следует знать при работе
с поддержкой Rust в ядре.


``no_std``
----------

Поддержка Rust в ядре может линковать только `core <https://doc.rust-lang.org/core/>`_,
но не `std <https://doc.rust-lang.org/std/>`_. Крейты для использования в ядре
должны явно включать это поведение с помощью атрибута ``#![no_std]``.


.. _rust_code_documentation:

Документация кода
-----------------

Код ядра на Rust документируется с помощью ``rustdoc`` — встроенного генератора
документации.

Сгенерированная HTML-документация включает интегрированный поиск, связанные элементы
(например, типы, функции, константы), исходный код и т. п. Её можно прочитать по адресу:

	https://rust.docs.kernel.org

Для linux-next см.:

	https://rust.docs.kernel.org/next/

Существуют также теги для каждого основного релиза, например:

	https://rust.docs.kernel.org/6.10/

Документацию также можно легко сгенерировать и прочитать локально. Это довольно
быстро (того же порядка, что и компиляция самого кода), и для этого не нужны
особые инструменты или окружение. Дополнительное преимущество в том, что она будет
адаптирована под конкретную используемую конфигурацию ядра. Чтобы сгенерировать её,
используйте цель ``rustdoc`` с тем же вызовом, что и при компиляции, например::

	make LLVM=1 rustdoc

Чтобы прочитать документацию локально в веб-браузере, выполните, например::

	xdg-open Documentation/output/rust/rustdoc/kernel/index.html

О том, как писать документацию, см. coding-guidelines.rst.


Дополнительные линты
--------------------

Хотя ``rustc`` — это весьма полезный компилятор, некоторые дополнительные линты
и виды анализа доступны через ``clippy`` — линтер для Rust. Чтобы включить его,
передайте ``CLIPPY=1`` тому же вызову, что и при компиляции, например::

	make LLVM=1 CLIPPY=1

Обратите внимание, что Clippy может изменить генерацию кода, поэтому его не следует
включать при сборке production-ядра.


Абстракции и привязки (bindings)
--------------------------------

Абстракции — это код на Rust, оборачивающий функциональность ядра со стороны C.

Чтобы использовать функции и типы со стороны C, создаются привязки (bindings).
Привязки — это объявления для Rust тех функций и типов, что находятся на стороне C.

Например, можно написать абстракцию ``Mutex`` на Rust, которая оборачивает
``struct mutex`` со стороны C и вызывает её функции через привязки.

Абстракции доступны не для всех внутренних API и концепций ядра, но предполагается,
что их покрытие со временем будет расширяться. «Листовые» модули (например, драйверы)
не должны использовать C-привязки напрямую. Вместо этого подсистемы должны
предоставлять по мере необходимости настолько безопасные, насколько это возможно,
абстракции.

.. code-block::

	                                                rust/bindings/
	                                               (rust/helpers/)

	                                                   include/ -----+ <-+
	                                                                 |   |
	  drivers/              rust/kernel/              +----------+ <-+   |
	    fs/                                           | bindgen  |       |
	   .../            +-------------------+          +----------+ --+   |
	                   |    Abstractions   |                         |   |
	+---------+        | +------+ +------+ |          +----------+   |   |
	| my_foo  | -----> | | foo  | | bar  | | -------> | Bindings | <-+   |
	| driver  |  Safe  | | sub- | | sub- | |  Unsafe  |          |       |
	+---------+        | |system| |system| |          | bindings | <-----+
	     |             | +------+ +------+ |          |  crate   |       |
	     |             |   kernel crate    |          +----------+       |
	     |             +-------------------+                             |
	     |                                                               |
	     +------------------# FORBIDDEN #--------------------------------+

Основная идея состоит в том, чтобы инкапсулировать всё прямое взаимодействие с
C-API ядра в тщательно проверенные и задокументированные абстракции. Тогда
пользователи этих абстракций не смогут привнести неопределённое поведение
(undefined behavior, UB) до тех пор, пока:

#. Абстракции корректны («sound»).
#. Любые блоки ``unsafe`` соблюдают контракт безопасности, необходимый для вызова
   операций внутри блока. Аналогично, любые ``unsafe impl``\ ы соблюдают контракт
   безопасности, необходимый для реализации трейта.

Привязки (bindings)
~~~~~~~~~~~~~~~~~~~~

Включив C-заголовок из ``include/`` в ``rust/bindings/bindings_helper.h``,
вы заставите инструмент ``bindgen`` автоматически сгенерировать привязки для
включённой подсистемы. После сборки см. выходные файлы ``*_generated.rs``
в каталоге ``rust/bindings/``.

Для тех частей C-заголовка, которые ``bindgen`` не генерирует автоматически,
например C-функций ``inline`` или нетривиальных макросов, допустимо добавить
небольшую функцию-обёртку в ``rust/helpers/``, чтобы сделать их доступными также
и для стороны Rust.

Абстракции
~~~~~~~~~~

Абстракции — это слой между привязками и внутриядерными пользователями. Они
расположены в ``rust/kernel/``, и их роль состоит в том, чтобы инкапсулировать
небезопасный доступ к привязкам в настолько безопасный, насколько это возможно,
API, который они предоставляют своим пользователям. К пользователям абстракций
относятся такие вещи, как драйверы или файловые системы, написанные на Rust.

Помимо аспекта безопасности, абстракции должны быть «эргономичными» в том смысле,
что они превращают C-интерфейсы в «идиоматический» код на Rust. Базовые примеры —
превращение захвата и освобождения ресурсов в C в конструкторы и деструкторы Rust
или целочисленных кодов ошибок C в ``Result``\ ы Rust.


Условная компиляция
-------------------

Код на Rust имеет доступ к условной компиляции на основе конфигурации ядра:

.. code-block:: rust

	#[cfg(CONFIG_X)]       // Enabled               (`y` or `m`)
	#[cfg(CONFIG_X="y")]   // Enabled as a built-in (`y`)
	#[cfg(CONFIG_X="m")]   // Enabled as a module   (`m`)
	#[cfg(not(CONFIG_X))]  // Disabled

Для других предикатов, которые ``cfg`` в Rust не поддерживает, например выражений
с числовыми сравнениями, можно определить новый символ Kconfig:

.. code-block:: kconfig

	config RUSTC_VERSION_MIN_107900
		def_bool y if RUSTC_VERSION >= 107900
