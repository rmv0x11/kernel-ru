Ядро Linux для процессоров ARC
******************************

Другие источники информации
###########################

Ниже приведены некоторые ресурсы, где можно найти дополнительную информацию о
процессорах ARC и соответствующих проектах с открытым исходным кодом.

- `<https://embarc.org>`_ — портал сообщества для открытого исходного кода на ARC.
  Хорошая отправная точка для поиска подходящих FOSS-проектов, выпусков тулчейна,
  новостей и многого другого.

- `<https://github.com/foss-for-synopsys-dwc-arc-processors>`_ —
  площадка для всей разработки, связанной с проектами с открытым исходным кодом для
  процессоров ARC. Некоторые из проектов являются форками различных апстрим-проектов,
  где «работа в процессе» размещается до отправки в апстрим-проекты.
  Другие проекты разрабатываются Synopsys и предоставляются сообществу
  как открытый исходный код для использования на процессорах ARC.

- `Official Synopsys ARC Processors website
  <https://www.synopsys.com/designware-ip/processor-solutions.html>`_ —
  ресурс с доступом к некоторой документации по IP (`Programmer's Reference
  Manual, AKA PRM for ARC HS processors
  <https://www.synopsys.com/dw/doc.php/ds/cc/programmers-reference-manual-ARC-HS.pdf>`_)
  и бесплатным версиям некоторых коммерческих инструментов (`Free nSIM
  <https://www.synopsys.com/cgi-bin/dwarcnsim/req1.cgi>`_ и
  `MetaWare Light Edition <https://www.synopsys.com/cgi-bin/arcmwtk_lite/reg1.cgi>`_).
  Обратите, однако, внимание, что для доступа как к документации, так и к
  инструментам требуется регистрация.

Важное замечание о конфигурируемости процессоров ARC
####################################################

Процессоры ARC обладают высокой степенью конфигурируемости, и в Linux
поддерживается несколько конфигурируемых параметров. Некоторые параметры прозрачны для
программного обеспечения (например, геометрия кэшей), некоторые могут быть обнаружены во
время выполнения и соответственно настроены и использованы, тогда как некоторые требуют
явного выбора или настройки в утилите конфигурации ядра (AKA «make menuconfig»).

Однако не все конфигурируемые параметры поддерживаются, когда процессор ARC
должен работать под управлением Linux. Командам, занимающимся проектированием SoC, следует
обратиться к «Appendix E: Configuration for ARC Linux» в ARC HS Databook за рекомендациями
по конфигурируемости.

Следование этим рекомендациям и предварительный выбор корректных параметров
конфигурации критически важны для предотвращения любых нежелательных проблем при
bringup'е SoC и разработке программного обеспечения в целом.

Сборка ядра Linux для процессоров ARC
#####################################

Процесс сборки ядра для процессоров ARC такой же, как и для любой другой
архитектуры, и может быть выполнен двумя способами:

- Кросс-компиляция: процесс компиляции для целевых платформ ARC на хосте
  разработки с другой архитектурой процессора (как правило, x86_64/amd64).
- Нативная компиляция: процесс компиляции для ARC на платформе ARC
  (аппаратной плате или симуляторе, таком как QEMU) с полностью установленной на платформе
  средой разработки (GNU toolchain, dtc, make и т. д.).

В обоих случаях для хоста необходим актуальный GNU toolchain для ARC.
Synopsys предлагает готовые сборки тулчейна, которые можно использовать для этой цели,
доступные по адресам:

- Выпуски GNU toolchain от Synopsys:
  `<https://github.com/foss-for-synopsys-dwc-arc-processors/toolchain/releases>`_

- Коллекция компиляторов ядра Linux:
  `<https://mirrors.edge.kernel.org/pub/tools/crosstool>`_

- Коллекция тулчейнов Bootlin: `<https://toolchains.bootlin.com>`_

После установки тулчейна в систему убедитесь, что его папка «bin»
добавлена в переменную окружения ``PATH``. Затем задайте ``ARCH=arc`` и
``CROSS_COMPILE=arc-linux`` (или то, что соответствует префиксу установленного тулчейна ARC)
и далее, как обычно, ``make defconfig && make``.

Это создаст файл «vmlinux» в корне дерева исходного кода ядра,
пригодный для загрузки на целевую систему через JTAG.
Если вам нужно получить образ, пригодный для использования с загрузчиком U-Boot,
наберите ``make uImage``, и ``uImage`` будет создан в папке ``arch/arc/boot``.
