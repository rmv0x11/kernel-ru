=================================================
Фиктивный модуль трассировки Coresight (Dummy)
=================================================

    :Author:   Hao Zhang <quic_hazha@quicinc.com>
    :Date:     June 2023

Введение
--------

Фиктивный (dummy) модуль трассировки Coresight предназначен для конкретных
устройств, к которым у ядра нет прав на доступ или настройку, например,
CoreSight TPDM на платформах Qualcomm. Для таких устройств необходим фиктивный
драйвер, регистрирующий их как устройства Coresight. Модуль также может
использоваться для описания компонентов, которые могут не иметь каких-либо
программных интерфейсов, чтобы в драйвере можно было создавать пути. Он
предоставляет API Coresight для операций над фиктивными устройствами, таких как
их включение и выключение. Он также предоставляет фиктивные пути приёмник/источник
(sink/source) Coresight для отладки.

Детали конфигурации
-------------------

Существует два типа узлов: фиктивный приёмник (dummy sink) и фиктивный источник
(dummy source). Эти узлы доступны по адресу ``/sys/bus/coresight/devices``.

Пример вывода::

    $ ls -l /sys/bus/coresight/devices | grep dummy
    dummy_sink0 -> ../../../devices/platform/soc@0/soc@0:sink/dummy_sink0
    dummy_source0 -> ../../../devices/platform/soc@0/soc@0:source/dummy_source0
