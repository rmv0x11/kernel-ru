==========================================================
Менеджер системной конфигурации CoreSight
==========================================================

    :Author:   Mike Leach <mike.leach@linaro.org>
    :Date:     October 2020

Введение
========

Менеджер системной конфигурации CoreSight — это API, позволяющий
программировать систему CoreSight предопределёнными конфигурациями, которые
затем можно легко включать из sysfs или perf.

Многие компоненты CoreSight можно программировать сложными способами — особенно
ETM. Кроме того, компоненты могут взаимодействовать в рамках системы CoreSight,
зачастую через компоненты перекрёстного срабатывания (cross trigger), такие как
CTI и CTM. Эти системные настройки можно определять и включать как именованные
конфигурации.


Базовые понятия
===============

В этом разделе вводятся базовые понятия системной конфигурации CoreSight.


Возможности (features)
----------------------

Возможность (feature) — это именованный набор настроек программирования для
устройства CoreSight. Эти настройки зависят от устройства и могут задаваться в
терминах абсолютных значений регистров, использования ресурсов и значений
параметров.

Возможность определяется с помощью дескриптора. Этот дескриптор используется для
загрузки на подходящее устройство — либо при загрузке возможности в систему,
либо при регистрации устройства CoreSight в менеджере конфигурации.

Процесс загрузки заключается в интерпретации дескриптора в набор обращений к
регистрам в драйвере — описания использования ресурсов и параметров транслируются
в соответствующие обращения к регистрам. Такая интерпретация делает программирование
возможности на устройство при необходимости простым и эффективным.

Возможность не будет активной на устройстве до тех пор, пока сама возможность не
будет включена и не будет включено само устройство. При включении устройства
включённые возможности будут запрограммированы в аппаратуру устройства.

Возможность включается как часть включения конфигурации в системе.


Значение параметра
~~~~~~~~~~~~~~~~~~~

Значение параметра — это именованное значение, которое может быть задано
пользователем до включения возможности и которое позволяет скорректировать
поведение операции, запрограммированной этой возможностью.

Например, это может быть значение счётчика в запрограммированной операции,
которая повторяется с заданной частотой. При включении возможности текущее
значение параметра используется при программировании устройства.

Дескриптор возможности определяет значение параметра по умолчанию, которое
используется, если пользователь не предоставит новое значение.

Пользователи могут обновлять значения параметров с помощью configfs API системы
CoreSight, который описан ниже.

Текущее значение параметра загружается в устройство при включении возможности на
этом устройстве.


Конфигурации
------------

Конфигурация определяет набор возможностей, которые должны использоваться в
сеансе трассировки, где выбрана данная конфигурация. Для любого сеанса
трассировки может быть выбрана только одна конфигурация.

Определённые возможности могут относиться к устройству любого типа, которое
зарегистрировано для поддержки системной конфигурации. Конфигурация может выбирать
возможности для включения на классе устройств — т. е. на любом ETMv4 — или на
конкретных устройствах, например на конкретном CTI в системе.

Как и для возможности, для определения конфигурации используется дескриптор.
Он определяет возможности, которые должны быть включены как часть конфигурации, а
также любые предустановленные значения, которые можно использовать для
переопределения значений параметров по умолчанию.


Предустановленные значения (preset)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Предустановленные значения (preset) — это легко выбираемые наборы значений
параметров для возможностей, используемых конфигурацией. Количество значений в
одном предустановленном наборе равно сумме значений параметров в возможностях,
используемых конфигурацией.

Например, конфигурация состоит из 3 возможностей: у одной 2 параметра, у одной
один параметр, а у ещё одной параметров нет. Таким образом, единичный
предустановленный набор будет содержать 3 значения.

Предустановки определяются конфигурацией необязательно, всего можно определить до
15. Если предустановка не выбрана, то значения параметров, определённые в
возможности, используются как обычно.


Работа
~~~~~~

При работе конфигурации происходят следующие шаги.

1) В этом примере конфигурация — 'autofdo', с которой связана возможность
   'strobing', работающая на устройствах ETMv4 CoreSight.

2) Конфигурация включается. Например, 'perf' может выбрать конфигурацию как часть
   своей командной строки::

    perf record -e cs_etm/autofdo/ myapp

   что включит конфигурацию 'autofdo'.

3) perf начинает трассировку в системе. По мере того как включается каждый ETMv4,
   который perf использует для трассировки, менеджер конфигурации будет проверять,
   есть ли у этого ETMv4 возможность, относящаяся к текущей активной конфигурации.
   В данном случае 'strobing' включается и программируется в ETMv4.

4) При отключении ETMv4 любые регистры, помеченные как требующие сохранения, будут
   считаны обратно.

5) В конце сеанса perf конфигурация будет отключена.


Просмотр конфигураций и возможностей
====================================

Набор конфигураций и возможностей, которые в данный момент загружены в систему,
можно просмотреть с помощью configfs API.

Смонтируйте configfs как обычно, и появится подсистема 'cs-syscfg'::

    $ ls /config
    cs-syscfg  stp-policy

В ней есть два подкаталога::

    $ cd cs-syscfg/
    $ ls
    configurations  features

В систему встроена конфигурация 'autofdo'. Её можно изучить следующим образом::

    $ cd configurations/
    $ ls
    autofdo
    $ cd autofdo/
    $ ls
    description  feature_refs  preset1  preset3  preset5  preset7  preset9
    enable       preset        preset2  preset4  preset6  preset8
    $ cat description
    Setup ETMs with strobing for autofdo
    $ cat feature_refs
    strobing

Для каждой объявленной предустановки объявляется подкаталог 'preset<n>'. Значения
предустановки можно изучить::

    $ cat preset1/values
    strobing.window = 0x1388 strobing.period = 0x2
    $ cat preset2/values
    strobing.window = 0x1388 strobing.period = 0x4

Файлы 'enable' и 'preset' позволяют управлять конфигурацией при использовании
CoreSight с sysfs.

Возможности, на которые ссылается конфигурация, можно изучить в каталоге
features::

    $ cd ../../features/strobing/
    $ ls
    description  matches  nr_params  params
    $ cat description
    Generate periodic trace capture windows.
    parameter 'window': a number of CPU cycles (W)
    parameter 'period': trace enabled for W cycles every period x W cycles
    $ cat matches
    SRC_ETMV4
    $ cat nr_params
    2

Перейдите в каталог params, чтобы изучить и скорректировать параметры::

    cd params
    $ ls
    period  window
    $ cd period
    $ ls
    value
    $ cat value
    0x2710
    # echo 15000 > value
    # cat value
    0x3a98

Параметры, скорректированные таким образом, отражаются во всех экземплярах
устройств, которые загрузили данную возможность.


Использование конфигураций в perf
=================================

Конфигурации, загруженные в систему управления конфигурацией CoreSight, также
объявляются в инфраструктуре событий perf 'cs_etm', чтобы их можно было выбирать
при запуске трассировки под perf::

    $ ls /sys/devices/cs_etm
    cpu0  cpu2  events  nr_addr_filters		power  subsystem  uevent
    cpu1  cpu3  format  perf_event_mux_interval_ms	sinks  type

Ключевым каталогом здесь является 'events' — общий каталог perf, позволяющий
выполнять выбор в командной строке perf. Как и записи sinks, он предоставляет хеш
имени конфигурации.

Запись в каталоге 'events' использует встроенный в perf генератор синтаксиса для
подстановки синтаксиса вместо имени при вычислении команды::

    $ ls events/
    autofdo
    $ cat events/autofdo
    configid=0xa7c3dddd

Конфигурацию 'autofdo' можно выбрать в командной строке perf::

    $ perf record -e cs_etm/autofdo/u --per-thread <application>

Также можно выбрать предустановку для переопределения текущих значений
параметров::

    $ perf record -e cs_etm/autofdo,preset=1/u --per-thread <application>

При выборе конфигураций таким образом приёмник трассировки (trace sink)
выбирается автоматически.

Использование конфигураций в sysfs
==================================

Coresight можно управлять с помощью sysfs. При использовании этого способа
конфигурацию можно сделать активной для устройств, которые используются в сеансе
sysfs.

В конфигурации есть файлы 'enable' и 'preset'.

Чтобы включить конфигурацию для использования с sysfs::

    $ cd configurations/autofdo
    $ echo 1 > enable

После этого будут использоваться значения параметров возможностей по умолчанию,
которые можно скорректировать так, как описано выше.

Чтобы использовать набор значений параметров preset<n>::

    $ echo 3 > preset

Это выберет preset3 для конфигурации.
Допустимые значения preset: 0 — для отмены выбора предустановок, и любое значение
<n>, для которого присутствует подкаталог preset<n>.

Обратите внимание, что активная конфигурация sysfs является глобальным
параметром, поэтому в каждый момент времени для sysfs может быть активна только
одна конфигурация. Попытка включить вторую конфигурацию приведёт к ошибке.
Кроме того, попытка отключить конфигурацию во время её использования также
приведёт к ошибке.

Использование активной конфигурации в sysfs независимо от конфигурации,
используемой в perf.


Создание и загрузка пользовательских конфигураций
=================================================

Пользовательские конфигурации и (или) возможности могут динамически загружаться
в систему с помощью загружаемого модуля.

Пример пользовательской конфигурации находится в ./samples/coresight.

Он создаёт новую конфигурацию, которая использует существующую встроенную
возможность strobing, но предоставляет другой набор предустановок.

При загрузке модуля конфигурация появляется в файловой системе configfs и
становится выбираемой так же, как встроенная конфигурация, описанная выше.

Конфигурации могут использовать ранее загруженные возможности. Система обеспечит
невозможность выгрузки возможности, которая в данный момент используется,
обеспечивая строго обратный порядок выгрузки относительно порядка загрузки.
