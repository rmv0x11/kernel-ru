======
Kbuild
======


Выходные файлы
==============

modules.order
-------------
В этом файле фиксируется порядок, в котором модули появляются в Makefile'ах.
Он используется modprobe для детерминированного разрешения псевдонимов,
соответствующих нескольким модулям.

modules.builtin
---------------
В этом файле перечислены все модули, встроенные в ядро. Он используется
modprobe, чтобы не завершаться ошибкой при попытке загрузить что-то встроенное.

modules.builtin.modinfo
-----------------------
Этот файл содержит modinfo из всех модулей, встроенных в ядро.
В отличие от modinfo отдельного модуля, все поля снабжены префиксом с именем модуля.

modules.builtin.ranges
----------------------
Этот файл содержит диапазоны смещений адресов (по секциям ELF) для всех модулей,
встроенных в ядро. Вместе с System.map он может использоваться
для сопоставления имён модулей с символами.

Переменные окружения
====================

KCPPFLAGS
---------
Дополнительные параметры, передаваемые при препроцессировании. Параметры
препроцессирования будут использоваться во всех случаях, когда kbuild выполняет
препроцессирование, включая сборку файлов на C и файлов ассемблера.

KAFLAGS
-------
Дополнительные параметры для ассемблера (для встроенного кода и модулей).

AFLAGS_MODULE
-------------
Дополнительные параметры ассемблера для модулей.

AFLAGS_KERNEL
-------------
Дополнительные параметры ассемблера для встроенного кода.

KCFLAGS
-------
Дополнительные параметры для компилятора C (для встроенного кода и модулей).

KRUSTFLAGS
----------
Дополнительные параметры для компилятора Rust (для встроенного кода и модулей).

CFLAGS_KERNEL
-------------
Дополнительные параметры для $(CC) при компиляции
кода, который компилируется как встроенный.

CFLAGS_MODULE
-------------
Дополнительные специфичные для модулей параметры для $(CC).

RUSTFLAGS_KERNEL
----------------
Дополнительные параметры для $(RUSTC) при компиляции
кода, который компилируется как встроенный.

RUSTFLAGS_MODULE
----------------
Дополнительные специфичные для модулей параметры для $(RUSTC).

LDFLAGS_MODULE
--------------
Дополнительные параметры, используемые для $(LD) при компоновке модулей.

HOSTCFLAGS
----------
Дополнительные флаги, передаваемые в $(HOSTCC) при сборке хост-программ.

HOSTCXXFLAGS
------------
Дополнительные флаги, передаваемые в $(HOSTCXX) при сборке хост-программ.

HOSTRUSTFLAGS
-------------
Дополнительные флаги, передаваемые в $(HOSTRUSTC) при сборке хост-программ.

PROCMACROLDFLAGS
----------------
Флаги, передаваемые при компоновке процедурных макросов (proc macros) Rust.
Поскольку процедурные макросы загружаются rustc во время сборки, они должны
компоноваться способом, совместимым с используемым тулчейном rustc.

Например, это может быть полезно, когда rustc использует другую библиотеку C,
нежели та, которую пользователь хочет применять для хост-программ.

Если значение не задано, по умолчанию используются флаги, передаваемые
при компоновке хост-программ.

HOSTLDFLAGS
-----------
Дополнительные флаги, передаваемые при компоновке хост-программ.

HOSTLDLIBS
----------
Дополнительные библиотеки, с которыми выполняется компоновка при сборке хост-программ.

.. _userkbuildflags:

USERCFLAGS
----------
Дополнительные параметры, используемые для $(CC) при компиляции userprogs.

USERLDFLAGS
-----------
Дополнительные параметры, используемые для $(LD) при компоновке userprogs. userprogs
компонуются с помощью CC, поэтому $(USERLDFLAGS) должен при необходимости включать
префикс "-Wl,".

KBUILD_KCONFIG
--------------
Устанавливает Kconfig-файл верхнего уровня равным значению этой переменной
окружения. Имя по умолчанию — "Kconfig".

KBUILD_VERBOSE
--------------
Устанавливает уровень подробности вывода kbuild. Может принимать те же значения,
что и "V=...".

Полный список см. в make help.

Установка "V=..." имеет приоритет над KBUILD_VERBOSE.

KBUILD_EXTMOD
-------------
Устанавливает каталог для поиска исходного кода ядра при сборке внешних
модулей.

Установка "M=..." имеет приоритет над KBUILD_EXTMOD.

KBUILD_OUTPUT
-------------
Указывает выходной каталог при сборке ядра.

Эту переменную можно также использовать, чтобы указать выходной каталог ядра при
сборке внешних модулей относительно заранее собранного ядра в отдельном каталоге
сборки. Обратите внимание, что это НЕ задаёт выходной каталог для самих
внешних модулей. (Для этой цели используйте KBUILD_EXTMOD_OUTPUT.)

Выходной каталог также можно указать с помощью "O=...".

Установка "O=..." имеет приоритет над KBUILD_OUTPUT.

KBUILD_EXTMOD_OUTPUT
--------------------
Указывает выходной каталог для внешних модулей.

Установка "MO=..." имеет приоритет над KBUILD_EXTMOD_OUTPUT.

KBUILD_EXTRA_WARN
-----------------
Указывает дополнительные проверки сборки. То же значение можно задать, передав
W=... из командной строки.

Список поддерживаемых значений см. в `make help`.

Установка "W=..." имеет приоритет над KBUILD_EXTRA_WARN.

KBUILD_DEBARCH
--------------
Для цели deb-pkg позволяет переопределить обычные эвристики, применяемые
deb-pkg. Обычно deb-pkg пытается угадать правильную архитектуру на основе
переменной UTS_MACHINE, а на некоторых архитектурах — также и конфигурации ядра.
Значение KBUILD_DEBARCH предполагается (но не проверяется) корректной
архитектурой Debian.

KDOCFLAGS
---------
Указывает дополнительные флаги (предупреждений/ошибок) для проверок kernel-doc
во время сборки; перечень поддерживаемых флагов см. в tools/docs/kernel-doc.
Обратите внимание, что (на данный момент) это не относится к сборкам документации.

ARCH
----
Установите ARCH равным архитектуре, для которой выполняется сборка.

В большинстве случаев имя архитектуры совпадает с
именем каталога, находящегося в каталоге arch/.

Но некоторые архитектуры, такие как x86 и sparc, имеют псевдонимы.

- x86: i386 для 32-битной, x86_64 для 64-битной
- parisc: parisc64 для 64-битной
- sparc: sparc32 для 32-битной, sparc64 для 64-битной

CROSS_COMPILE
-------------
Указывает необязательную фиксированную часть имени файла binutils.
CROSS_COMPILE может быть частью имени файла или полным путём.

CROSS_COMPILE также используется для ccache в некоторых конфигурациях.

CF
--
Дополнительные параметры для sparse.

CF часто используется в командной строке следующим образом::

    make CF=-Wbitwise C=2

INSTALL_PATH
------------
INSTALL_PATH указывает, куда помещать обновлённые образы ядра и system map.
По умолчанию это /boot, но вы можете задать и другие значения.

INSTALLKERNEL
-------------
Скрипт установки, вызываемый при использовании "make install".
Имя по умолчанию — "installkernel".

Скрипт будет вызван со следующими аргументами:

   - $1 — версия ядра
   - $2 — файл образа ядра
   - $3 — файл map ядра
   - $4 — путь установки по умолчанию (если пусто, используется корневой каталог)

Реализация "make install" зависит от архитектуры
и может отличаться от описанной выше.

INSTALLKERNEL предоставляется, чтобы дать возможность
указать собственный установщик при кросс-компиляции ядра.

MODLIB
------
Указывает, куда устанавливать модули.
Значение по умолчанию::

     $(INSTALL_MOD_PATH)/lib/modules/$(KERNELRELEASE)

Это значение можно переопределить, и в этом случае значение по умолчанию игнорируется.

INSTALL_MOD_PATH
----------------
INSTALL_MOD_PATH задаёт префикс к MODLIB для перемещений каталога модулей,
необходимых при использовании build root'ов. Она не определена в
makefile, но при необходимости этот аргумент можно передать make.

INSTALL_MOD_STRIP
-----------------
INSTALL_MOD_STRIP, если определена, приводит к тому, что у модулей
после установки будет удалена отладочная информация (strip). Если INSTALL_MOD_STRIP равна '1', то
будет использован параметр по умолчанию --strip-debug. В противном случае
значение INSTALL_MOD_STRIP будет использовано как параметры команды strip.

INSTALL_HDR_PATH
----------------
INSTALL_HDR_PATH указывает, куда устанавливать заголовочные файлы пространства
пользователя при выполнении "make headers_*".

Значение по умолчанию::

    $(objtree)/usr

$(objtree) — это каталог, в котором сохраняются выходные файлы.
Выходной каталог часто задаётся с помощью "O=..." в командной строке.

Это значение можно переопределить, и в этом случае значение по умолчанию игнорируется.

INSTALL_DTBS_PATH
-----------------
INSTALL_DTBS_PATH указывает, куда устанавливать device tree blob'ы для
перемещений, необходимых при использовании build root'ов. Она не определена в
makefile, но при необходимости этот аргумент можно передать make.

KBUILD_ABS_SRCTREE
--------------------------------------------------
Kbuild по возможности использует относительный путь для указания на дерево. Например,
при сборке в дереве исходного кода путь к дереву исходного кода равен '.'

Установка этого флага требует, чтобы Kbuild использовал абсолютный путь к дереву
исходного кода. Для этого есть несколько полезных случаев, например при генерации
файлов тегов с записями, содержащими абсолютные пути, и т. п.

KBUILD_SIGN_PIN
---------------
Эта переменная позволяет передать парольную фразу или PIN утилите sign-file
при подписывании модулей ядра, если этого требует закрытый ключ.

KBUILD_MODPOST_WARN
-------------------
KBUILD_MODPOST_WARN можно установить, чтобы избежать ошибок в случае неопределённых
символов на финальной стадии компоновки модуля. Она превращает такие ошибки
в предупреждения.

KBUILD_MODPOST_NOFINAL
----------------------
KBUILD_MODPOST_NOFINAL можно установить, чтобы пропустить финальную компоновку модулей.
Это полезно исключительно для ускорения тестовых компиляций.

KBUILD_EXTRA_SYMBOLS
--------------------
Для модулей, использующих символы из других модулей.
Подробнее см. в modules.rst.

ALLSOURCE_ARCHS
---------------
Для целей tags/TAGS/cscope вы можете указать более одной архитектуры
для включения в базы данных, разделяя их пробелами. Например::

    $ make ALLSOURCE_ARCHS="x86 mips arm" tags

Чтобы получить все доступные архитектуры, можно также указать all. Например::

    $ make ALLSOURCE_ARCHS=all tags

IGNORE_DIRS
-----------
Для целей tags/TAGS/cscope вы можете выбрать, какие каталоги не будут
включены в базы данных, разделяя их пробелами. Например::

    $ make IGNORE_DIRS="drivers/gpu/drm/radeon tools" cscope

KBUILD_BUILD_TIMESTAMP
----------------------
Установка этой переменной в строку с датой переопределяет временную метку,
используемую в определении UTS_VERSION (uname -v в работающем ядре). Значение
должно быть строкой, которую можно передать команде date -d. Например::

    $ KBUILD_BUILD_TIMESTAMP="Mon Oct 13 00:00:00 UTC 2025" make

Значение по умолчанию — вывод команды date в некоторый момент во время
сборки. Если она задана, эта временная метка также будет использована для полей mtime
в любом архиве initramfs. Метки mtime в initramfs 32-битные, поэтому даты до
эпохи Unix (1970) или после 2106-02-07 06:28:15 UTC приведут к ошибке.

KBUILD_BUILD_USER, KBUILD_BUILD_HOST
------------------------------------
Эти две переменные позволяют переопределить строку user@host, отображаемую во время
загрузки и в /proc/version. Значение по умолчанию — вывод команд
whoami и host соответственно.

LLVM
----
Если эта переменная установлена в 1, Kbuild будет использовать утилиты Clang и LLVM вместо
GCC и GNU binutils для сборки ядра.
