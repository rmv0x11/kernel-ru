=====================================================================
Экспорт заголовочных файлов ядра для использования из userspace
=====================================================================

Команда «make headers_install» экспортирует заголовочные файлы ядра в
форме, пригодной для использования программами из пространства пользователя.

Экспортируемые заголовочные файлы ядра Linux описывают API для программ
пространства пользователя, которые пытаются обращаться к службам ядра.  Эти
заголовочные файлы ядра используются системной библиотекой C (такой как glibc
или uClibc) для определения доступных системных вызовов, а также констант и
структур, используемых с этими системными вызовами.  Заголовочные файлы
библиотеки C включают заголовочные файлы ядра из подкаталога «linux».
Заголовочные файлы системной libc обычно устанавливаются в расположение по
умолчанию /usr/include, а заголовочные файлы ядра — в подкаталоги внутри него
(в первую очередь /usr/include/linux и /usr/include/asm).

Заголовочные файлы ядра обратно совместимы, но не совместимы вперёд.  Это
означает, что программа, собранная с использованием библиотеки C, которая
применяет более старые заголовочные файлы ядра, должна работать на более новом
ядре (хотя у неё может не быть доступа к новым возможностям), но программа,
собранная с использованием более новых заголовочных файлов ядра, может не
работать на более старом ядре.

Команду «make headers_install» можно запустить в каталоге верхнего уровня
исходного кода ядра (или с использованием стандартной сборки вне дерева
исходников).  Она принимает два необязательных аргумента::

  make headers_install ARCH=i386 INSTALL_HDR_PATH=/usr

ARCH указывает, для какой архитектуры создавать заголовочные файлы, и по
умолчанию принимает значение текущей архитектуры.  Каталог linux/asm
экспортируемых заголовочных файлов ядра специфичен для платформы; чтобы
увидеть полный список поддерживаемых архитектур, используйте команду::

  ls -d include/asm-* | sed 's/.*-//'

INSTALL_HDR_PATH указывает, куда устанавливать заголовочные файлы. По
умолчанию он принимает значение «./usr».

Каталог «include» автоматически создаётся внутри INSTALL_HDR_PATH, и
заголовочные файлы устанавливаются в «INSTALL_HDR_PATH/include».
