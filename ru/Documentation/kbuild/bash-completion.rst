==================================
Bash-дополнение для Kbuild
==================================

Система сборки ядра написана с использованием Makefile'ов, а Bash-дополнение
для команды `make` доступно через проект `bash-completion`_.

Однако Makefile'ы для сборки ядра сложны. Обобщённые правила дополнения для
команды `make` не дают осмысленных подсказок для системы сборки ядра, за
исключением опций самой команды `make`.

Чтобы улучшить дополнение для различных переменных и целей, в исходном коде
ядра имеется собственный скрипт дополнения по пути `scripts/bash-completion/make`.

Этот скрипт предоставляет дополнительные дополнения при работе внутри дерева
исходного кода ядра. Вне дерева исходного кода ядра он переходит к обобщённым
правилам дополнения для команды `make`.

Предварительные требования
==========================

Скрипт опирается на вспомогательные функции, предоставляемые проектом
`bash-completion`_. Убедитесь, что он установлен в вашей системе. В большинстве
дистрибутивов можно установить пакет `bash-completion` через стандартный
менеджер пакетов.

Как использовать
================

Вы можете подключить скрипт напрямую::

  $ source scripts/bash-completion/make

Или можно скопировать его в путь поиска скриптов дополнения Bash.
Например::

  $ mkdir -p ~/.local/share/bash-completion/completions
  $ cp scripts/bash-completion/make ~/.local/share/bash-completion/completions/

Подробности
===========

Дополнительное дополнение для Kbuild включается в следующих случаях:

 - Вы находитесь в корневом каталоге исходного кода ядра.
 - Вы находитесь в каталоге сборки верхнего уровня, созданном опцией O=
   (проверяется через символическую ссылку `source`, указывающую на исходный код ядра).
 - Опция -C команды make указывает на каталог исходного кода или сборки ядра.
 - Опция -f команды make указывает на файл в каталоге исходного кода или сборки ядра.

Если ни одно из вышеперечисленных условий не выполнено, происходит переход к
обобщённым правилам дополнения.

Дополнение поддерживает:

  - Часто используемые цели, такие как `all`, `menuconfig`, `dtbs` и т.д.
  - Переменные make (или окружения), такие как `ARCH`, `LLVM` и т.д.
  - Сборки одной цели (`foo/bar/baz.o`)
  - Файлы конфигурации (`*_defconfig` и `*.config`)

Некоторые переменные обеспечивают интеллектуальное поведение. Например, `CROSS_COMPILE=`
с последующим нажатием TAB отображает установленные тулчейны. Список показываемых
файлов defconfig зависит от значения переменной `ARCH=`.
