=========================================
Инфраструктура GCC-плагинов
=========================================


Введение
========

GCC-плагины — это загружаемые модули, которые предоставляют компилятору
дополнительные возможности [1]_. Они полезны для инструментирования во время
выполнения и для статического анализа. Мы можем анализировать, изменять и
добавлять дополнительный код во время компиляции через коллбэки (callbacks) [2]_,
а также через проходы GIMPLE [3]_, IPA [4]_ и RTL [5]_.

Инфраструктура GCC-плагинов ядра поддерживает сборку модулей вне дерева
исходников (out-of-tree), кросс-компиляцию и сборку в отдельном каталоге.
Исходные файлы плагинов должны быть компилируемы компилятором C++.

В настоящее время инфраструктура GCC-плагинов поддерживает лишь некоторые
архитектуры. Выполните grep по "select HAVE_GCC_PLUGINS", чтобы узнать, какие
архитектуры поддерживают GCC-плагины.

Эта инфраструктура была портирована из grsecurity [6]_ и PaX [7]_.

Назначение
==========

GCC-плагины предназначены для того, чтобы дать место для экспериментов с
потенциальными возможностями компилятора, которых нет ни в апстриме GCC, ни в
апстриме Clang. После того как их полезность доказана, цель состоит в том, чтобы
включить возможность в апстрим GCC (а также Clang), а затем окончательно удалить
её из ядра, как только эта возможность станет доступна во всех поддерживаемых
версиях GCC.

В частности, новые плагины должны реализовывать только те возможности, для
которых нет поддержки в апстриме компиляторов (ни в GCC, ни в Clang).

Когда возможность существует в Clang, но отсутствует в GCC, следует приложить
усилия, чтобы включить эту возможность в апстрим GCC (а не оставлять её лишь в
виде GCC-плагина, специфичного для ядра), чтобы вся экосистема могла извлечь из
неё пользу.

Аналогично, даже если возможность, предоставляемая GCC-плагином, *не*
существует в Clang, но доказана её полезность, следует приложить усилия, чтобы
включить эту возможность в апстрим GCC (а также Clang).

После того как возможность становится доступна в апстриме GCC, плагин делается
несобираемым для соответствующей версии GCC (и более поздних). Как только все
поддерживаемые ядром версии GCC предоставляют эту возможность, плагин удаляется
из ядра.


Файлы
=====

**$(src)/scripts/gcc-plugins**

	Это каталог GCC-плагинов.

**$(src)/scripts/gcc-plugins/gcc-common.h**

	Это заголовочный файл совместимости для GCC-плагинов.
	Его следует всегда включать вместо отдельных заголовков gcc.

**$(src)/scripts/gcc-plugins/gcc-generate-gimple-pass.h,
$(src)/scripts/gcc-plugins/gcc-generate-ipa-pass.h,
$(src)/scripts/gcc-plugins/gcc-generate-simple_ipa-pass.h,
$(src)/scripts/gcc-plugins/gcc-generate-rtl-pass.h**

	Эти заголовочные файлы автоматически генерируют структуры регистрации для
	проходов GIMPLE, SIMPLE_IPA, IPA и RTL.
	Им следует отдавать предпочтение перед созданием структур вручную.


Использование
=============

Вы должны установить заголовочные файлы GCC-плагинов для вашей версии gcc,
например, в Ubuntu для gcc-10::

	apt-get install gcc-10-plugin-dev

Или в Fedora::

	dnf install gcc-plugin-devel libmpc-devel

Или в Fedora при использовании кросс-компиляторов, включающих плагины::

	dnf install libmpc-devel

Включите инфраструктуру GCC-плагинов и нужные вам плагины в конфигурации ядра::

	CONFIG_GCC_PLUGINS=y
	CONFIG_GCC_PLUGIN_LATENT_ENTROPY=y
	...

Запустите gcc (нативный или кросс-компилятор), чтобы убедиться, что заголовочные
файлы плагинов обнаруживаются::

	gcc -print-file-name=plugin
	CROSS_COMPILE=arm-linux-gnu- ${CROSS_COMPILE}gcc -print-file-name=plugin

Слово "plugin" означает, что они не обнаружены::

	plugin

Полный путь означает, что они обнаружены::

       /usr/lib/gcc/x86_64-redhat-linux/12/plugin

Чтобы скомпилировать минимальный набор инструментов, включая плагины::

	make scripts

или просто запустите сборку ядра командой make и скомпилируйте всё ядро с
GCC-плагином цикломатической сложности.


4. Как добавить новый GCC-плагин
================================

GCC-плагины находятся в scripts/gcc-plugins/. Вам нужно поместить исходные файлы
плагина прямо в scripts/gcc-plugins/. Создание подкаталогов не поддерживается.
Плагин должен быть добавлен в scripts/gcc-plugins/Makefile, scripts/Makefile.gcc-plugins
и в соответствующий файл Kconfig.
