=============================================
Цели конфигурации и редакторы конфигурации
=============================================

Этот файл содержит некоторые сведения, полезные при использовании ``make *config``.

Используйте ``make help``, чтобы вывести список всех возможных целей конфигурации.

Программы xconfig ('qconf'), menuconfig ('mconf') и nconfig ('nconf')
также имеют встроенный текст справки.  Обязательно ознакомьтесь с ним, чтобы узнать
о навигации, поиске и других общих сведениях.

Программа gconfig ('gconf') имеет ограниченный текст справки.


Общие сведения
==============

Новые выпуски ядра часто вводят новые символы конфигурации.  Зачастую более
важно то, что новые выпуски ядра могут переименовывать символы конфигурации.  Когда
это происходит, использование ранее работавшего файла .config и запуск
"make oldconfig" не обязательно дадут вам работающее новое ядро,
поэтому вам может понадобиться узнать, какие НОВЫЕ символы ядра
были введены.

Чтобы увидеть список новых символов конфигурации, используйте::

    cp user/some/old.config .config
    make listnewconfig

и программа конфигурации выведет все новые символы, по одному на строку.

Кроме того, можно воспользоваться методом «грубой силы»::

    make oldconfig
    scripts/diffconfig .config.old .config | less


Переменные окружения
====================

Переменные окружения для ``*config``:

``KCONFIG_CONFIG``
    Эту переменную окружения можно использовать, чтобы задать имя файла конфигурации
    ядра по умолчанию, переопределяющее имя по умолчанию ".config".

``KCONFIG_DEFCONFIG_LIST``
    Эта переменная окружения задаёт список файлов конфигурации, которые могут быть
    использованы в качестве базовой конфигурации в случае, если .config ещё не существует.
    Записи в списке отделяются друг от друга пробелами, и
    используется первая из существующих.

``KCONFIG_OVERWRITECONFIG``
    Если задать KCONFIG_OVERWRITECONFIG в окружении, Kconfig не будет
    нарушать символические ссылки, когда .config является символической ссылкой на другое место.

``KCONFIG_WARN_UNKNOWN_SYMBOLS``
    Эта переменная окружения заставляет Kconfig выдавать предупреждения обо всех нераспознанных
    символах во входных данных конфигурации.

``KCONFIG_WERROR``
    Если задана, Kconfig трактует предупреждения как ошибки.

``CONFIG_``
    Если задать ``CONFIG_`` в окружении, Kconfig будет добавлять ко всем символам
    префикс с его значением при сохранении конфигурации, вместо использования
    значения по умолчанию ``CONFIG_``.

Переменные окружения для ``{allyes/allmod/allno/alldef/rand}config``:

``KCONFIG_ALLCONFIG``
    Варианты allyesconfig/allmodconfig/alldefconfig/allnoconfig/randconfig
    также могут использовать переменную окружения KCONFIG_ALLCONFIG как флаг или как
    имя файла, содержащего символы конфигурации, которым пользователь требует задать
    конкретное значение.  Если KCONFIG_ALLCONFIG используется без имени файла, когда
    KCONFIG_ALLCONFIG == "" или KCONFIG_ALLCONFIG == "1", ``make *config``
    ищет файл с именем "all{yes/mod/no/def/random}.config"
    (соответствующий использованной команде ``*config``) для получения значений символов,
    которые должны быть принудительно заданы.  Если этот файл не найден, он ищет
    файл с именем "all.config" в качестве источника принудительных значений.

    Это позволяет вам создавать «миниатюрные» файлы конфигурации (miniconfig) или собственные
    файлы конфигурации, содержащие только те символы конфигурации, которые вас
    интересуют.  После этого система конфигурации ядра генерирует полный файл .config,
    включающий символы из вашего файла miniconfig.

    Этот файл ``KCONFIG_ALLCONFIG`` представляет собой файл конфигурации, содержащий
    (обычно подмножество всех) предустановленные символы конфигурации.  Эти заданные
    значения переменных по-прежнему подвергаются обычным проверкам зависимостей.

    Примеры::

        KCONFIG_ALLCONFIG=custom-notebook.config make allnoconfig

    или::

        KCONFIG_ALLCONFIG=mini.config make allnoconfig

    или::

        make KCONFIG_ALLCONFIG=mini.config allnoconfig

    Эти примеры отключат большинство параметров (allnoconfig), но включат или
    отключат те параметры, которые явно перечислены в указанных
    файлах mini-config.

Переменные окружения для ``randconfig``:

``KCONFIG_SEED``
    Вы можете задать здесь целочисленное значение, используемое для инициализации ГСЧ, если хотите
    каким-либо образом отладить поведение парсера/фронтендов kconfig.
    Если значение не задано, будет использовано текущее время.

``KCONFIG_PROBABILITY``
    Эту переменную можно использовать для смещения вероятностей. Эта переменная может
    быть не задана или пуста, либо задана в одном из трёх различных форматов:

    =======================     ==================  =====================
    KCONFIG_PROBABILITY         y:n split           y:m:n split
    =======================     ==================  =====================
    unset or empty              50  : 50            33  : 33  : 34
    N                            N  : 100-N         N/2 : N/2 : 100-N
    [1] N:M                     N+M : 100-(N+M)      N  :  M  : 100-(N+M)
    [2] N:M:L                    N  : 100-N          M  :  L  : 100-(M+L)
    =======================     ==================  =====================

где N, M и L — целые числа (в десятичной системе) в диапазоне [0,100], причём так,
чтобы:

    [1] N+M находится в диапазоне [0,100]

    [2] M+L находится в диапазоне [0,100]

Примеры::

    KCONFIG_PROBABILITY=10
        10% of booleans will be set to 'y', 90% to 'n'
        5% of tristates will be set to 'y', 5% to 'm', 90% to 'n'
    KCONFIG_PROBABILITY=15:25
        40% of booleans will be set to 'y', 60% to 'n'
        15% of tristates will be set to 'y', 25% to 'm', 60% to 'n'
    KCONFIG_PROBABILITY=10:15:15
        10% of booleans will be set to 'y', 90% to 'n'
        15% of tristates will be set to 'y', 15% to 'm', 70% to 'n'

Переменные окружения для ``syncconfig``:

``KCONFIG_NOSILENTUPDATE``
    Если эта переменная имеет непустое значение, она предотвращает молчаливые
    обновления конфигурации ядра (требуются явные обновления).

``KCONFIG_AUTOCONFIG``
    Эту переменную окружения можно задать, чтобы указать путь и имя
    файла "auto.conf".  Её значение по умолчанию — "include/config/auto.conf".

``KCONFIG_AUTOHEADER``
    Эту переменную окружения можно задать, чтобы указать путь и имя
    файла "autoconf.h" (заголовочного).
    Её значение по умолчанию — "include/generated/autoconf.h".


menuconfig
==========

Поиск в menuconfig:

    Функция поиска ищет имена символов конфигурации ядра,
    поэтому вам нужно знать что-то близкое к тому, что вы
    ищете.

    Пример::

        /hotplug
        This lists all config symbols that contain "hotplug",
        e.g., HOTPLUG_CPU, MEMORY_HOTPLUG.

    Чтобы получить справку по поиску, введите / с последующим TAB-TAB (чтобы подсветить
    <Help>) и Enter.  Это сообщит вам, что в строке поиска можно также использовать
    регулярные выражения (regex), поэтому, если вы
    не заинтересованы в MEMORY_HOTPLUG, можно попробовать::

        /^hotplug

    При поиске символы сортируются следующим образом:

    - сначала точные совпадения, отсортированные по алфавиту (точное совпадение —
      это когда поиск совпадает с полным именем символа);
    - затем прочие совпадения, отсортированные по алфавиту.

    Например, ^ATH.K совпадает с:

        ATH5K ATH9K ATH5K_AHB ATH5K_DEBUG [...] ATH6KL ATH6KL_DEBUG
        [...] ATH9K_AHB ATH9K_BTCOEX_SUPPORT ATH9K_COMMON [...]

    из которых только ATH5K и ATH9K совпадают точно и потому сортируются
    первыми (и в алфавитном порядке), затем идут все остальные символы,
    отсортированные в алфавитном порядке.

    В этом меню нажатие клавиши, указанной в префиксе (#), приведёт вас
    непосредственно к этому месту. Вы будете возвращены к текущим
    результатам поиска после выхода из этого нового меню.

Параметры пользовательского интерфейса для 'menuconfig':

``MENUCONFIG_COLOR``
    С помощью переменной MENUCONFIG_COLOR можно выбрать различные цветовые
    темы.  Чтобы выбрать тему, используйте::

        make MENUCONFIG_COLOR=<theme> menuconfig

    Доступные темы::

      - mono       => selects colors suitable for monochrome displays
      - blackbg    => selects a color scheme with black background
      - classic    => theme with blue background. The classic look
      - bluetitle  => a LCD friendly version of classic. (default)

``MENUCONFIG_MODE``
    Этот режим показывает все подменю в виде одного большого дерева.

    Пример::

        make MENUCONFIG_MODE=single_menu menuconfig


nconfig
=======

nconfig — это альтернативный текстовый конфигуратор.  Он выводит функциональные
клавиши вдоль нижней части терминала (окна), которые выполняют команды.
Вы также можете просто использовать соответствующую цифровую клавишу для выполнения
команд, если только вы не находитесь в окне ввода данных.  Например, вместо F6
для сохранения (Save) можно просто нажать 6.

Используйте F1 для глобальной справки (Global help) или F3 для меню краткой справки (Short help).

Поиск в nconfig:

    Вы можете искать либо в строках-подсказках "prompt" пунктов меню,
    либо в символах конфигурации.

    Используйте /, чтобы начать поиск по пунктам меню.  Это
    не поддерживает регулярные выражения.  Используйте <Down> или <Up> для
    перехода к следующему и предыдущему совпадению соответственно.  Используйте <Esc>, чтобы
    завершить режим поиска.

    F8 (SymSearch) выполняет поиск заданной строки или регулярного выражения (regex)
    среди символов конфигурации.

    В режиме SymSearch нажатие клавиши, указанной в префиксе (#), приведёт вас
    непосредственно к этому месту. Вы будете возвращены к
    текущим результатам поиска после выхода из этого нового меню.

Переменные окружения:

``NCONFIG_MODE``
    Этот режим показывает все подменю в виде одного большого дерева.

    Пример::

        make NCONFIG_MODE=single_menu nconfig


xconfig
=======

Поиск в xconfig:

    Функция поиска ищет имена символов конфигурации ядра,
    поэтому вам нужно знать что-то близкое к тому, что вы
    ищете.

    Пример::

        Ctrl-F hotplug

    или::

        Menu: File, Search, hotplug

    выводит все записи символов конфигурации, которые содержат "hotplug" в
    имени символа.  В этом диалоге поиска вы можете изменить
    значение конфигурации для любой из записей, которые не отображаются серым цветом.
    Вы также можете ввести другую строку поиска, не возвращаясь
    в главное меню.


gconfig
=======

Поиск в gconfig:

    В gconfig нет команды поиска.  Однако в gconfig
    есть несколько различных вариантов отображения, режимов и параметров.
