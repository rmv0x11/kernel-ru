Консоль Брайля в Linux
======================

Чтобы получать ранние сообщения загрузки на устройстве Брайля (до того, как
смогут запуститься программы чтения с экрана в пространстве пользователя),
сначала нужно скомпилировать поддержку обычной последовательной консоли
(см. :ref:`Documentation/admin-guide/serial-console.rst <serial_console>`), а
также поддержку устройства Брайля
(в :menuselection:`Device Drivers --> Accessibility support --> Console on braille device`).

Затем нужно указать параметр ``console=brl`` в командной строке ядра, формат
следующий::

	console=brl,serial_options...

где ``serial_options...`` те же, что описаны в
:ref:`Documentation/admin-guide/serial-console.rst <serial_console>`.

Так, например, можно использовать ``console=brl,ttyS0``, если устройство Брайля
подключено к первому последовательному порту, и ``console=brl,ttyS0,115200``, чтобы
переопределить скорость передачи на 115200, и т. д.

По умолчанию устройство Брайля просто показывает последнее сообщение ядра (режим
консоли).  Чтобы просмотреть предыдущие сообщения, нажмите клавишу Insert для
переключения в режим просмотра VT.  В режиме просмотра клавиши со стрелками
позволяют перемещаться по содержимому VT, клавиши `PAGE-UP`/`PAGE-DOWN`
переходят в начало/конец экрана, а клавиша `HOME` возвращает
к курсору, обеспечивая тем самым самые базовые средства просмотра экрана.

Звуковую обратную связь можно получить, добавив параметр ядра
``braille_console.sound=1``.

Для простоты можно включить только одну консоль Брайля, остальные использования
``console=brl,...`` будут отброшены.  Также обратите внимание, что это не мешает
работе механизма выбора консоли, описанного в
:ref:`Documentation/admin-guide/serial-console.rst <serial_console>`.

На данный момент поддерживается только устройство VisioBraille.
