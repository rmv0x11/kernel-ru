===================================================
Журналируемая файловая система IBM (JFS) для Linux
===================================================

Домашняя страница JFS:  http://jfs.sourceforge.net/

Поддерживаются следующие параметры монтирования:

(*) == по умолчанию

iocharset=name
                Набор символов, используемый для преобразования из Unicode
		в ASCII.  По умолчанию преобразование не выполняется.  Используйте
		iocharset=utf8 для преобразований в UTF-8.  Для этого требуется,
		чтобы в файле .config ядра был установлен CONFIG_NLS_UTF8.
		iocharset=none задаёт поведение по умолчанию явным образом.

resize=value
                Изменить размер тома до <value> блоков.  JFS поддерживает только
		увеличение тома, но не уменьшение.  Этот параметр действителен только
		во время перемонтирования, когда том смонтирован
		на чтение-запись.  Ключевое слово resize без значения увеличит
		том до полного размера раздела.

nointegrity
                Не выполнять запись в журнал.  Основное назначение этого параметра —
		обеспечить более высокую производительность при восстановлении тома
		с резервного носителя.  Целостность тома не
		гарантируется при аварийном завершении работы системы.

integrity(*)
                Фиксировать изменения метаданных в журнале.  Используйте этот параметр для
		перемонтирования тома, для которого ранее был указан параметр nointegrity,
		чтобы восстановить нормальное поведение.

errors=continue
                        Продолжать работу при ошибке файловой системы.
errors=remount-ro(*)
                        Перемонтировать файловую систему только для чтения при ошибке.
errors=panic
                        Вызвать панику и остановить машину, если возникнет ошибка.

uid=value
                Переопределить хранящийся на диске uid указанным значением
gid=value
                Переопределить хранящийся на диске gid указанным значением
umask=value
                Переопределить хранящийся на диске umask указанным восьмеричным значением. Для
                каталогов бит выполнения будет установлен, если установлен соответствующий
		бит чтения.

discard=minlen, discard/nodiscard(*)
                Это включает/отключает использование команд discard/TRIM.
		Команды discard/TRIM отправляются нижележащему
                блочному устройству при освобождении блоков. Это полезно для SSD-
                устройств и разреженных/тонко выделяемых (thinly-provisioned) LUN.  Команда ioctl
		FITRIM также доступна совместно с параметром nodiscard.
		Значение minlen задаёт минимальное число блоков, при котором
		команда TRIM к блочному устройству считается полезной.
		Если параметру discard не задано значение, по умолчанию используется
		64 блока, что в JFS означает 256 КиБ.
		Значение minlen параметра discard переопределяет значение minlen, заданное
		в ioctl() FITRIM.

"Mail list Subscribe" на нашей веб-странице http://jfs.sourceforge.net/
