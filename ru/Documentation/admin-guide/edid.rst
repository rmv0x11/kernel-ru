====
EDID
====

В старые добрые времена, когда параметры графики настраивались явно в файле
с именем xorg.conf, можно было справиться даже с неисправным оборудованием.

Сегодня, с появлением Kernel Mode Setting, графическая плата либо работает
корректно, потому что все компоненты следуют стандартам, — либо компьютер
непригоден к использованию, потому что после загрузки экран остаётся тёмным
или отображает не ту область. Это происходит в следующих случаях:

- Графическая плата не распознаёт монитор.
- Графическая плата не способна обнаружить никаких данных EDID.
- Графическая плата некорректно передаёт данные EDID драйверу.
- Монитор не отправляет данные EDID или отправляет недостоверные данные.
- KVM отправляет собственные данные EDID вместо опроса подключённого монитора.

Добавление параметра ядра «nomodeset» помогает в большинстве случаев, но
влечёт за собой ограничения в дальнейшем.

В качестве средства для таких ситуаций был введён пункт конфигурации ядра
CONFIG_DRM_LOAD_EDID_FIRMWARE. Он позволяет предоставить индивидуально
подготовленный или исправленный набор данных EDID в каталоге /lib/firmware,
откуда он загружается через интерфейс firmware.
