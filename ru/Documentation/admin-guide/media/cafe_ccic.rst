Драйвер cafe_ccic
=================

Автор: Jonathan Corbet <corbet@lwn.net>

Введение
--------

«cafe_ccic» — это драйвер для контроллера CMOS-камеры Marvell 88ALP01 «cafe».
Этот контроллер встречается в системах OLPC первого поколения, и данный драйвер
был написан при поддержке проекта OLPC.

Текущее состояние: основной драйвер работает. Он может формировать данные в
форматах YUV422, RGB565 и RGB444. (Просматривая код, можно также увидеть RGB32,
но это средство отладки, которое вскоре будет удалено). Режимы VGA и QVGA
работают; CIF присутствует, но цвета остаются искажёнными. На данный момент с
этим контроллером заведомо работает только сенсор OV7670.

Чтобы опробовать его: подойдёт любая из этих команд:

.. code-block:: none

     $ mplayer tv:// -tv driver=v4l2:width=640:height=480 -nosound
     $ mplayer tv:// -tv driver=v4l2:width=640:height=480:outfmt=bgr16 -nosound

Утилита «xawtv» также работает; gqcam не работает по неизвестным причинам.

Параметры времени загрузки
--------------------------

Существует несколько параметров времени загрузки, большинство из которых можно
изменить и после загрузки через sysfs:

 - alloc_bufs_at_load: Обычно драйвер не выделяет никаких DMA-буферов до тех
   пор, пока не наступит время передачи данных. Если этот параметр установлен,
   то буферы максимально возможного размера выделяются во время загрузки модуля.
   Этот параметр закрепляет память на всё время жизни модуля, но, возможно,
   уменьшает вероятность сбоя выделения памяти в дальнейшем.

 - dma_buf_size: Размер выделяемых DMA-буферов. Обратите внимание, что этот
   параметр учитывается только при выделении во время загрузки; когда буферы
   выделяются во время работы, их размер будет выбран соответственно текущим
   настройкам камеры.

 - n_dma_bufs: Контроллер может циклически переключаться между двумя или тремя
   DMA-буферами. Обычно драйвер пытается использовать три буфера; однако на
   более быстрых системах он будет хорошо работать и всего с двумя.

 - min_buffers: Минимальное число буферов потокового ввода-вывода, с которым
   драйвер согласится работать. По умолчанию равно единице, но на более
   медленных системах более качественного поведения с mplayer можно добиться,
   установив более высокое значение (например, шесть).

 - max_buffers: Максимальное число буферов потокового ввода-вывода; по умолчанию
   равно десяти. Это число было тщательно взято с потолка, и не следует полагать,
   что оно действительно что-либо означает.

 - flip: Если этот логический параметр установлен, сенсору будет дана команда
   инвертировать видеоизображение. Имеет ли это смысл, определяется тем, как
   именно установлена ваша конкретная камера.
