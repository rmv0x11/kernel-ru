=========================================
Интерфейс камеры Rockchip (CIF)
=========================================

Введение
========

Интерфейс камеры Rockchip (Camera Interface, CIF) присутствует во многих SoC
Rockchip в различных вариантах.
Различные варианты представляют собой комбинации общих строительных блоков, таких как

* INTERFACE-блоки разных типов, а именно

  * Digital Video Port (DVP, параллельный интерфейс данных)
  * INTERFACE-блок для приёмника MIPI CSI-2

* блоки CROP

* приёмник MIPI CSI-2 (доступен не во всех вариантах): этот блок называется
  MIPI CSI HOST в документации Rockchip.
  Технически это отдельный аппаратный блок, но он тесно связан с CIF
  и поэтому рассматривается здесь.

* блоки MUX (доступны не во всех вариантах), которые передают видеоданные
  процессору обработки изображений (image signal processor, ISP)

* блоки SCALE (доступны не во всех вариантах)

* движки DMA, которые переносят видеоданные в системную память с использованием
  механизма двойной буферизации, называемого режимом ping-pong

* поддержка четырёх потоков на каждый INTERFACE-блок (доступна не во всех
  вариантах), например, для виртуальных каналов (Virtual Channels, VC) MIPI CSI-2

В этом документе описаны различные варианты CIF, их аппаратная компоновка,
а также их представление в ориентированном на media controller драйвере
устройства rkcif, который расположен в drivers/media/platform/rockchip/rkcif.

Варианты
========

Rockchip PX30 Video Input Processor (VIP)
-----------------------------------------

Video Input Processor (VIP) у PX30 содержит digital video port, который принимает
параллельные видеоданные или BT.656.
Поскольку эти протоколы не предусматривают нескольких потоков, VIP имеет один
движок DMA, который переносит входные видеоданные в системную память.

Драйвер rkcif представляет этот аппаратный вариант, предоставляя одно V4L2-субустройство
(блок DVP INTERFACE/CROP) и одно V4L2-устройство (движок DMA для DVP).

Rockchip RK3568 Video Capture (VICAP)
-------------------------------------

Блок Video Capture (VICAP) у RK3568 содержит digital video port и приёмник MIPI
CSI-2, которые могут принимать видеоданные независимо.
DVP принимает параллельные видеоданные, BT.656 и BT.1120.
Поскольку протокол BT.1120 может предусматривать более одного потока, DVP в составе
RK3568 VICAP содержит четыре движка DMA, которые могут захватывать разные потоки.
Аналогично, приёмник MIPI CSI-2 у RK3568 VICAP содержит четыре движка DMA для
обработки разных виртуальных каналов (Virtual Channels, VC).

Драйвер rkcif представляет этот аппаратный вариант, предоставляя следующие
V4L2-субустройства:

* rkcif-dvp0: блок INTERFACE/CROP для DVP

и следующие видеоустройства:

* rkcif-dvp0-id0: поддержка нескольких потоков на DVP пока не реализована,
  так как трудно найти тестовое оборудование. Поэтому данное видеоустройство
  представляет первый движок DMA у RK3568 DVP.
