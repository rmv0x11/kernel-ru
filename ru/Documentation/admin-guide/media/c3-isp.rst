==========================================================
Драйвер Amlogic C3 Image Signal Processing (C3ISP)
==========================================================

Введение
========

Этот файл документирует драйвер Amlogic C3ISP, расположенный в
drivers/media/platform/amlogic/c3/isp.

Текущая версия драйвера поддерживает C3ISP, имеющийся на
процессоре Amlogic C308L.

Драйвер реализует интерфейсы V4L2, Media controller и V4L2 subdev.
Поддерживается датчик камеры (camera sensor), использующий интерфейс V4L2 subdev в ядре.

Драйвер протестирован на платформе AW419-C308L-Socket.

Amlogic C3 ISP
==============

Аппаратное обеспечение камеры, имеющееся на процессорах C308L и поддерживаемое
драйвером, состоит из:

- 1 модуль MIPI-CSI-2: обслуживает физический уровень приёмника MIPI CSI-2 и
  принимает данные от подключённого датчика камеры.
- 1 модуль MIPI-ADAPTER: упорядочивает данные MIPI в соответствии с требованиями
  ко входу ISP и отправляет данные MIPI в ISP.
- 1 модуль ISP (Image Signal Processing): содержит конвейер аппаратных блоков
  обработки изображения. Конвейер ISP содержит на выходе три ресайзера (resizer),
  каждый из которых подключён к интерфейсу DMA, записывающему выходные данные в память.

Высокоуровневое функциональное представление C3 ISP приведено ниже.::

                                                                   +----------+    +-------+
                                                                   | Resizer  |--->| WRMIF |
  +---------+    +------------+    +--------------+    +-------+   |----------+    +-------+
  | Sensor  |--->| MIPI CSI-2 |--->| MIPI ADAPTER |--->|  ISP  |---|----------+    +-------+
  +---------+    +------------+    +--------------+    +-------+   | Resizer  |--->| WRMIF |
                                                                   +----------+    +-------+
                                                                   |----------+    +-------+
                                                                   | Resizer  |--->| WRMIF |
                                                                   +----------+    +-------+

Архитектура и устройство драйвера
=================================

С целью смоделировать аппаратные связи между модулями и предоставить чистый,
логичный и удобный интерфейс драйвер регистрирует следующие
V4L2 sub-device:

- 1 sub-device `c3-mipi-csi2` — приёмник MIPI CSI-2
- 1 sub-device `c3-mipi-adapter` — адаптер MIPI
- 1 sub-device `c3-isp-core` — ядро ISP
- 3 sub-device `c3-isp-resizer` — ресайзеры ISP

Sub-device `c3-isp-core` связан с 2 узлами видеоустройств для захвата
статистики и программирования параметров:

- узел видеоустройства захвата `c3-isp-stats` для захвата статистики
- выходное видеоустройство `c3-isp-params` для программирования параметров

Каждый sub-device `c3-isp-resizer` связан с узлом видеоустройства захвата, из
которого захватываются кадры:

- `c3-isp-resizer0` связан с видеоустройством захвата `c3-isp-cap0`
- `c3-isp-resizer1` связан с видеоустройством захвата `c3-isp-cap1`
- `c3-isp-resizer2` связан с видеоустройством захвата `c3-isp-cap2`

Реализация
==========

Конфигурация аппаратного обеспечения ISP во время работы выполняется на узле
видеоустройства `c3-isp-params` с использованием :ref:`V4L2_META_FMT_C3ISP_PARAMS
<v4l2-meta-fmt-c3isp-params>` в качестве формата данных. Структура буфера определяется
:c:type:`c3_isp_params_cfg`.

Статистика захватывается из узла видеоустройства `c3-isp-stats` с использованием
формата данных :ref:`V4L2_META_FMT_C3ISP_STATS <v4l2-meta-fmt-c3isp-stats>`.

Итоговый размер и формат изображения настраиваются с помощью интерфейса захвата
видео V4L2 на узлах видеоустройств `c3-isp-cap[0, 2]`.

Amlogic C3 ISP поддерживается `libcamera <https://libcamera.org>`_ с
выделенным обработчиком конвейера и алгоритмами, которые выполняют коррекцию и
улучшение изображения во время работы.
