Интерфейс условного доступа цифрового ТВ
========================================


.. note::

   Эта документация устарела.

Данный документ описывает использование высокоуровневого CI API в
соответствии с Linux DVB API. Это не документация по существующему
низкоуровневому CI API.

.. note::

   Для устройств Twinhan и их клонов работу с аппаратной частью CI
   выполняет модуль dst_ca. Этот модуль загружается автоматически, если
   обнаружен CI (Common Interface, содержащий CAM — Conditional Access
   Module).

ca_zap
~~~~~~

Для обработки зашифрованных потоков MPEG-TS требуется приложение из
пространства пользователя, например ``ca_zap``.

Приложение из пространства пользователя ``ca_zap`` отвечает за отправку
информации, относящейся к дескремблированию, в модуль условного доступа
(CAM).

На данный момент для корректной работы этому приложению требуется
следующее.

a) Настройка на корректный канал с помощью szap.

  напр.: $ szap -c channels.conf -r "TMC" -x

b) channels.conf, содержащий корректный PMT PID

  напр.: TMC:11996:h:0:27500:278:512:650:321

  здесь 278 — это корректный PMT PID. Остальные значения те же,
  что использует szap.

c) после запуска szap необходимо запустить ca_zap, чтобы дескремблер
   заработал,

  напр.: $ ca_zap channels.conf "TMC"

d) Будем надеяться, вы насладитесь любимым каналом по подписке так же,
   как с FTA-картой.

.. note::

  В настоящее время и ca_zap, и dst_test предназначены только для
  демонстрационных целей; при необходимости они могут стать полноценными
  приложениями.


Карты, относящиеся к этой категории
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

В настоящее время к этой категории относятся карты Twinhan и их клоны;
эти карты доступны как VVMER, Tomato, Hercules, Orange и так далее.

Поддерживаемые модули CI
~~~~~~~~~~~~~~~~~~~~~~~~~~

Поддержка модулей CI в значительной степени зависит от прошивки на
картах. Некоторые карты поддерживают почти все доступные модули CI.
Сделать что-либо для того, чтобы с этими картами работали дополнительные
модули CI, практически невозможно.

На данный момент этим драйвером протестированы следующие модули

(1) Irdeto 1 и 2 от SCM
(2) Viaccess от SCM
(3) Dragoncam
