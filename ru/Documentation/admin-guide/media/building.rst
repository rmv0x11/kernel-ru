==========================================
Сборка поддержки для медиаустройства
==========================================

Первый шаг — загрузить исходный код ядра, либо через специфичный для
дистрибутива файл с исходниками, либо через основное git-дерево ядра\ [1]_.

Обратите, однако, внимание, что если:

- вы смельчак и хотите поэкспериментировать с чем-то новым;
- вы хотите сообщить об ошибке;
- вы разрабатываете новые патчи,

то вам следует использовать ветку ``master`` основного дерева разработки медиа:

    https://git.linuxtv.org/media.git/

В этом случае вы можете найти полезную информацию на
`страницах wiki LinuxTv <https://linuxtv.org/wiki>`_:

    https://linuxtv.org/wiki/index.php/How_to_Obtain,_Build_and_Install_V4L-DVB_Device_Drivers

.. [1] Апстрим-дерево разработки ядра Linux расположено по адресу

       https://git.kernel.org/pub/scm/li  nux/kernel/git/torvalds/linux.git/

Конфигурирование ядра Linux
===========================

Вы можете получить доступ к меню параметров сборки ядра с помощью::

    $ make menuconfig

Затем выберите все нужные параметры и выйдите, сохранив конфигурацию.

Изменённая конфигурация будет находиться в файле ``.config``. Она будет
выглядеть примерно так::

    ...
    # CONFIG_RC_CORE is not set
    # CONFIG_CEC_CORE is not set
    CONFIG_MEDIA_SUPPORT=m
    CONFIG_MEDIA_SUPPORT_FILTER=y
    ...

Медиаподсистема управляется следующими параметрами конфигурационного меню::

    Device Drivers --->
	<M> Remote Controller support  --->
	[ ] HDMI CEC RC integration
	[ ] Enable CEC error injection support
	[*] HDMI CEC drivers  --->
	<*> Multimedia support  --->

Параметр ``Remote Controller support`` включает базовую поддержку
пультов дистанционного управления\ [2]_.

Параметр ``HDMI CEC RC integration`` включает интеграцию HDMI CEC
с Linux, позволяя получать данные через HDMI CEC так, как если бы они
поступали от пульта дистанционного управления, напрямую подключённого к машине.

Параметр ``HDMI CEC drivers`` позволяет выбрать платформенные и USB-драйверы,
которые принимают и/или передают коды CEC через интерфейсы HDMI\ [3]_.

Последний параметр (``Multimedia support``) включает поддержку камер,
устройств захвата аудио/видео и ТВ.

Поддержка медиаподсистемы может быть собрана как вместе с основным
ядром, так и в виде модуля. Для большинства сценариев использования
предпочтительнее собирать её в виде модулей.

.. note::

   Вместо использования меню ядро предоставляет скрипт, позволяющий
   включать параметры конфигурации напрямую. Чтобы включить поддержку медиа
   и поддержку пультов дистанционного управления с использованием модулей ядра,
   вы можете использовать::

	$ scripts/config -m RC_CORE
	$ scripts/config -m MEDIA_SUPPORT

.. [2] ``Remote Controller support`` также следует включить, если вы
       хотите использовать некоторые драйверы ТВ-карт, которые могут
       зависеть от базовой поддержки пультов дистанционного управления.

.. [3] Обратите внимание, что подсистема DRM также имеет драйверы для GPU,
       которые используют поддержку HDMI CEC медиа.

       Эти специфичные для GPU драйверы выбираются через меню ``Graphics support``
       в разделе ``Device Drivers``.

       Когда драйвер GPU поддерживает HDMI CEC, он автоматически
       включает базовую поддержку CEC в медиаподсистеме.

Зависимости медиа
-----------------

Следует отметить, что включения вышеперечисленного из чистой конфигурации
обычно недостаточно. Медиаподсистема зависит от ряда других компонентов
базовой поддержки Linux, чтобы работать.

Например, большинство медиаустройств используют последовательную шину связи,
чтобы взаимодействовать с некоторыми периферийными устройствами. Такая шина
называется I²C (Inter-Integrated Circuit). Чтобы иметь возможность собрать
поддержку для такого оборудования, следует включить поддержку шины I²C,
либо через меню, либо с помощью::

    ./scripts/config -m I2C

Другой пример: ядро пультов дистанционного управления требует поддержки
устройств ввода, которую можно включить с помощью::

    ./scripts/config -m INPUT

Может также потребоваться другая базовая функциональность (например, поддержка
PCI и/или USB) в зависимости от конкретного драйвера (или драйверов),
который вы хотите включить.

Включение поддержки пультов дистанционного управления
-----------------------------------------------------

Меню пультов дистанционного управления позволяет выбрать драйверы для
конкретных устройств. Его меню выглядит так::

         --- Remote Controller support
         <M>   Compile Remote Controller keymap modules
         [*]   LIRC user interface
         [*]     Support for eBPF programs attached to lirc devices
         [*]   Remote controller decoders  --->
         [*]   Remote Controller devices  --->

Параметр ``Compile Remote Controller keymap modules`` создаёт карты клавиш для
нескольких популярных пультов дистанционного управления.

Параметр ``LIRC user interface`` добавляет расширенную функциональность при
использовании программы ``lirc``, включая API, который позволяет пространству
пользователя получать необработанные данные от пультов дистанционного управления.

Параметр ``Support for eBPF programs attached to lirc devices`` позволяет
использовать специальные программы (называемые eBPF), которые дают приложениям
возможность добавлять в ядро Linux дополнительную функциональность декодирования
сигналов пультов дистанционного управления.

Параметр ``Remote controller decoders`` позволяет выбрать
протоколы, которые будут распознаваться ядром Linux. За исключением случаев, когда
вы хотите отключить какой-либо конкретный декодер, рекомендуется оставить все
подпараметры включёнными.

Параметр ``Remote Controller devices`` позволяет выбрать драйверы,
которые потребуются для поддержки вашего устройства.

Ту же конфигурацию можно также задать через скрипт ``script/config``.
Так, например, чтобы обеспечить поддержку драйвера пульта дистанционного
управления ITE (встречается на Intel NUC и на некоторых настольных
ПК ASUS x86), вы можете выполнить::

	$ scripts/config -e INPUT
	$ scripts/config -e ACPI
	$ scripts/config -e MODULES
	$ scripts/config -m RC_CORE
	$ scripts/config -e RC_DEVICES
	$ scripts/config -e RC_DECODERS
	$ scripts/config -m IR_RC5_DECODER
	$ scripts/config -m IR_ITE_CIR

Включение поддержки HDMI CEC
----------------------------

Поддержка HDMI CEC устанавливается автоматически, когда она требуется драйверу.
Поэтому всё, что вам нужно сделать, — это включить поддержку либо для
графической карты, которой она нужна, либо для одного из существующих
HDMI-драйверов.

Специфичные для HDMI драйверы доступны в меню ``HDMI CEC drivers``\ [4]_::

	--- HDMI CEC drivers
	< >   ChromeOS EC CEC driver
	< >   Amlogic Meson AO CEC driver
	< >   Amlogic Meson G12A AO CEC driver
	< >   Generic GPIO-based CEC driver
	< >   Samsung S5P CEC driver
	< >   STMicroelectronics STiH4xx HDMI CEC driver
	< >   STMicroelectronics STM32 HDMI CEC driver
	< >   Tegra HDMI CEC driver
	< >   SECO Boards HDMI CEC driver
	[ ]     SECO Boards IR RC5 support
	< >   Pulse Eight HDMI CEC
	< >   RainShadow Tech HDMI CEC

.. [4] Приведённое выше содержимое — лишь пример. Фактические параметры для
       HDMI-устройств зависят от архитектуры системы и могут различаться
       в новых ядрах.

Включение поддержки медиа
-------------------------

Меню медиа имеет гораздо больше параметров, чем меню пультов дистанционного
управления. После выбора вы должны увидеть следующие параметры::

	--- Media support
	[ ] Filter media drivers
	[*] Autoselect ancillary drivers
	    Media device types --->
	    Media core support --->
	    Video4Linux options --->
	    Media controller options --->
	    Digital TV options --->
	    HDMI CEC options --->
	    Media drivers --->
	    Media ancillary drivers --->

За исключением случаев, когда вы точно знаете, что делаете, или когда хотите
собрать драйвер для платформы SoC, настоятельно рекомендуется оставить
параметр ``Autoselect ancillary drivers`` включённым, так как он автоматически
выберет необходимые вспомогательные драйверы I²C.

Теперь существует два способа выбора драйверов медиаустройств, как описано
ниже.

Меню ``Filter media drivers``
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Это меню предназначено для упрощения настройки оборудования ПК и ноутбуков.
Оно работает, позволяя пользователю указать, какие виды медиадрайверов
желательны, с помощью следующих параметров::

	[ ] Cameras and video grabbers
	[ ] Analog TV
	[ ] Digital TV
	[ ] AM/FM radio receivers/transmitters
	[ ] Software defined radio
	[ ] Platform-specific devices
	[ ] Test drivers

Так, если вы хотите добавить поддержку только камеры или устройства захвата
видео, выберите лишь первый параметр. Допускается выбор нескольких параметров.

После того как параметры в этом меню выбраны, система сборки автоматически
выберет необходимые базовые драйверы для поддержки выбранной
функциональности.

.. note::

   Большинство ТВ-карт гибридные: они поддерживают как аналоговое, так и
   цифровое ТВ.

   Если у вас гибридная карта, вам может потребоваться включить в меню
   как ``Analog TV``, так и ``Digital TV``.

При использовании этого параметра значения по умолчанию для базовой
функциональности поддержки медиа обычно достаточно хороши, чтобы обеспечить
базовую функциональность драйвера. Тем не менее вы можете вручную включить
некоторую желаемую дополнительную (необязательную) функциональность с помощью
настроек в каждом из следующих подменю ``Media support``::

	    Media core support --->
	    Video4Linux options --->
	    Media controller options --->
	    Digital TV options --->
	    HDMI CEC options --->

После того как вы выберете нужные фильтры, драйверы, соответствующие критериям
фильтрации, станут доступны в подменю ``Media support->Media drivers``.

Меню ``Media Core Support`` без фильтрации
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Если вы отключите меню ``Filter media drivers``, в меню ``Media drivers``
должны отображаться все доступные для вашей системы драйверы, зависимости
которых удовлетворены.

Обратите, однако, внимание, что сначала вам следует убедиться, что меню
``Media Core Support`` содержит всю базовую функциональность, которая
потребуется вашим драйверам, иначе соответствующие драйверы устройств
не будут показаны.

Пример
------

Чтобы включить модульную поддержку для одной из плат, перечисленных в
:doc:`этой таблице <cx231xx-cardlist>`, с модульными базовыми модулями медиа,
файл ``.config`` должен содержать следующие строки::

    CONFIG_MODULES=y
    CONFIG_USB=y
    CONFIG_I2C=y
    CONFIG_INPUT=y
    CONFIG_RC_CORE=m
    CONFIG_MEDIA_SUPPORT=m
    CONFIG_MEDIA_SUPPORT_FILTER=y
    CONFIG_MEDIA_ANALOG_TV_SUPPORT=y
    CONFIG_MEDIA_DIGITAL_TV_SUPPORT=y
    CONFIG_MEDIA_USB_SUPPORT=y
    CONFIG_VIDEO_CX231XX=y
    CONFIG_VIDEO_CX231XX_DVB=y

Сборка и установка нового ядра
==============================

После того как файл ``.config`` содержит всё необходимое, для сборки
достаточно выполнить команду ``make``::

    $ make

А затем установить новое ядро и его модули::

    $ sudo make modules_install
    $ sudo make install

Сборка только новых медиадрайверов и ядра медиа
===============================================

Запуск нового ядра для разработки из дерева разработки обычно рискован,
поскольку оно может содержать экспериментальные изменения, в которых могут быть
ошибки. Поэтому существует несколько способов собрать только новые драйверы,
используя альтернативные деревья.

Существует `проект бэкпортов ядра Linux
<https://backports.wiki.kernel.org/index.php/Main_Page>`_, который содержит
более новые драйверы, предназначенные для компиляции под стабильные ядра.

Разработчики LinuxTV, которые отвечают за сопровождение медиаподсистемы,
также сопровождают дерево бэкпортов, содержащее только медиадрайверы,
ежедневно обновляемые из новейшего ядра. Это дерево доступно по адресу:

https://git.linuxtv.org/media_build.git/

Следует отметить, что, хотя использовать дерево ``media_build`` для целей
тестирования относительно безопасно, нет гарантий, что оно будет работать
(или хотя бы собираться) на произвольном ядре. Это дерево сопровождается
по принципу «насколько хватает сил», по мере того как время позволяет нам
исправлять проблемы в нём.

Если вы заметите в нём что-то неправильное, не стесняйтесь присылать патчи
в список рассылки медиаподсистемы Linux: media@vger.kernel.org. Пожалуйста,
добавляйте ``[PATCH media-build]`` в тему письма, если вы отправляете новый
патч для media-build.

Перед использованием вам следует выполнить::

    $ ./build

.. note::

    1) вам может потребоваться запустить это дважды, если дерево
       ``media-build`` обновилось;
    2) вам может потребоваться выполнить ``make distclean``, если вы
       собирали его ранее для версии ядра, отличной от той,
       которую вы используете сейчас;
    3) по умолчанию для медиа будут использоваться те же параметры
       конфигурации, что и заданные в ядре, которое вы используете.

Чтобы выбрать другие драйверы или другие параметры конфигурации,
используйте::

    $ make menuconfig

Затем вы можете собрать и установить новые драйверы::

    $ make && sudo make install

Это переопределит предыдущие медиадрайверы, которые использовало ваше ядро.
