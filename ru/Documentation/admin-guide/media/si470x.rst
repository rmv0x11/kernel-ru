Информация от Silicon Labs
--------------------------

Silicon Laboratories — производитель радио-микросхем, которые в наши дни являются
наиболее часто используемыми радиоприёмниками в мобильных телефонах. Обычно они
подключаются по I2C. Но SiLabs также предоставляет эталонную схему (reference
design), которая объединяет эту микросхему с небольшим микроконтроллером
C8051F321, образуя USB-радио. Частью этой эталонной схемы является и радио-
приложение в бинарном и исходном коде. Программное обеспечение также содержит
средство автоматического обновления прошивки до самой свежей версии. Информацию
об этом можно загрузить здесь:
http://www.silabs.com/usbradio


Поддерживаемые микросхемы
-------------------------

Следующие микросхемы имеют очень похожий набор регистров, поэтому они уже
поддерживаются или будут когда-нибудь поддерживаться драйвером:

- Si4700: FM-радиоприёмник
- Si4701: FM-радиоприёмник, поддержка RDS
- Si4702: FM-радиоприёмник
- Si4703: FM-радиоприёмник, поддержка RDS
- Si4704: FM-радиоприёмник, внешняя антенна не требуется
- Si4705: FM-радиоприёмник, внешняя антенна не требуется, поддержка RDS, Dig I/O
- Si4706: Улучшенный FM RDS/TMC радиоприёмник, внешняя антенна не требуется,
	  поддержка RDS
- Si4707: Специализированный радиоприёмник метеорологического диапазона с
  декодером SAME, поддержка RDS
- Si4708: Самые маленькие FM-приёмники
- Si4709: Самые маленькие FM-приёмники, поддержка RDS

Дополнительную информацию об этом можно загрузить здесь:
http://www.silabs.com/products/mcu/Pages/USBFMRadioRD.aspx

Программное обеспечение
-----------------------

Тестирование обычно выполняется с большинством приложений под Debian/testing:

- fmtools - Утилита для управления картами FM-тюнеров
- gnomeradio - FM-радио тюнер для рабочего стола GNOME
- gradio - GTK FM-радио тюнер
- kradio - Удобное радиоприложение для KDE
- radio - радиоприложение на основе ncurses
- mplayer - The Ultimate Movie Player For Linux
- v4l2-ctl - Набор утилит командной строки video4linux

Например, можно использовать:

.. code-block:: none

	v4l2-ctl -d /dev/radio0 --set-ctrl=volume=10,mute=0 --set-freq=95.21 --all

Существует также библиотека libv4l, которую можно использовать. В ней появится
функция для поиска частоты — либо за счёт использования аппаратной
функциональности, как в radio-si470x, либо за счёт реализации функции, какая
сейчас есть в каждой из упомянутых программ. Когда-нибудь радиопрограммы должны
будут использовать libv4l.

Для обработки информации RDS существует продолжающийся проект по адресу:
http://rdsd.berlios.de/

В настоящее время нет проекта для приведения предложений TMC в человекочитаемый
вид.


Прослушивание звука
-------------------

USB Audio обеспечивается модулем ALSA snd_usb_audio. Рекомендуется также выбрать
SND_USB_AUDIO, так как это необходимо для получения звука с радио. Для
прослушивания нужно перенаправить звук, например, с помощью одной из следующих
команд. Пожалуйста, скорректируйте аудиоустройства под свои нужды (/dev/dsp* и
hw:x,x).

Если вы просто хотите проверить звук (очень низкого качества):

.. code-block:: none

	cat /dev/dsp1 > /dev/dsp

Если вы используете sox + OSS, попробуйте:

.. code-block:: none

	sox -2 --endian little -r 96000 -t oss /dev/dsp1 -t oss /dev/dsp

или с использованием sox + alsa:

.. code-block:: none

	sox --endian little -c 2 -S -r 96000 -t alsa hw:1 -t alsa -r 96000 hw:0

Если вы используете arts, попробуйте:

.. code-block:: none

	arecord -D hw:1,0 -r96000 -c2 -f S16_LE | artsdsp aplay -B -

Если вы используете mplayer, попробуйте:

.. code-block:: none

	mplayer -radio adevice=hw=1.0:arate=96000 \
		-rawaudio rate=96000 \
		radio://<frequency>/capture

Параметры модуля
----------------

После загрузки модуля у вас по-прежнему есть доступ к некоторым из них в точке
монтирования sysfs по пути /sys/module/radio_si470x/parameters. Содержимое
файлов только для чтения (0444) не обновляется, даже если space, band и de
изменяются с помощью приватных видеоэлементов управления. Остальные изменяемы во
время выполнения.


Ошибки
------

Увеличьте tune_timeout, если вы часто получаете ошибки -EIO.

При истечении времени ожидания или достижении границы диапазона hw_freq_seek
возвращает -EAGAIN.

Если вы получаете какие-либо ошибки от snd_usb_audio, пожалуйста, сообщайте о них
разработчикам ALSA.


Открытые вопросы
----------------

Выделение младшего номера устройства V4L и установка параметров не идеальны.
Решение в настоящее время обсуждается.

Существует USB-интерфейс для загрузки/выгрузки новых образов прошивки. Поддержку
для него можно реализовать с помощью интерфейса request_firmware.

Существует режим прерываний RDS. Драйвер уже использует тот же интерфейс для
опроса информации RDS, но в настоящее время не использует режим прерываний.

Существует интерфейс светодиодов (LED), который можно использовать для
переопределения управления светодиодами, запрограммированного в прошивке. Это
можно сделать доступным с помощью функций поддержки светодиодов в ядре.
