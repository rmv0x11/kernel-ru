Драйвер mgb4
============

Copyright |copy| 2023 - 2025 Digiteq Automotive
    author: Martin Tůma <martin.tuma@digiteqautomotive.com>

Это драйвер устройства v4l2 для Digiteq Automotive FrameGrabber 4 — карты PCIe,
способной захватывать и генерировать видеопотоки FPD-Link III и GMSL2/3,
используемые в автомобильной промышленности.

Интерфейс sysfs
---------------

Драйвер mgb4 предоставляет интерфейс sysfs, который используется для настройки
параметров, связанных с видеопотоком (некоторые из них должны быть установлены
надлежащим образом до того, как можно будет открыть устройство v4l2), и для
получения статуса видеоустройства/видеопотока.

Существует два типа параметров — глобальные / относящиеся к PCI-карте,
находящиеся в ``/sys/class/video4linux/videoX/device``, и специфичные для модуля,
находящиеся в ``/sys/class/video4linux/videoX``.

Глобальные параметры (PCI-карты)
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

**module_type** (R):
    Тип модуля.

    | 0 - Модуль отсутствует
    | 1 - FPDL3
    | 2 - GMSL3 (один сериализатор, два десериализатора, соединённых гирляндой)
    | 3 - GMSL3 (один сериализатор, два десериализатора)
    | 4 - GMSL3 (два десериализатора с двумя гирляндными выходами)
    | 6 - GMSL1
    | 8 - GMSL3 coax

**module_version** (R):
    Номер версии модуля. Ноль в случае отсутствия модуля.

**fw_type** (R):
    Тип прошивки.

    | 1 - FPDL3
    | 2 - GMSL3
    | 3 - GMSL1

**fw_version** (R):
    Номер версии прошивки.

**serial_number** (R):
    Серийный номер карты. Формат::

        PRODUCT-REVISION-SERIES-SERIAL

    где каждый компонент является 8-битным числом.

Общие входные параметры FPDL3/GMSL
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

**input_id** (R):
    Числовой идентификатор входа, начинающийся с нуля.

**oldi_lane_width** (RW):
    Число выходных линий десериализатора.

    | 0 - одиночная
    | 1 - двойная (по умолчанию)

**color_mapping** (RW):
    Отображение входящих бит сигнала на биты цвета пикселей.

    | 0 - OLDI/JEIDA
    | 1 - SPWG/VESA (по умолчанию)

**link_status** (R):
    Статус видеоканала. Если канал заблокирован, чипы правильно соединены и
    взаимодействуют на одной скорости и по одному протоколу. Канал может быть
    заблокирован без активного видеопотока.

    Значение 0 эквивалентно флагу V4L2_IN_ST_NO_SYNC битов статуса
    VIDIOC_ENUMINPUT в V4L2.

    | 0 - не заблокирован
    | 1 - заблокирован

**stream_status** (R):
    Статус видеопотока. Поток считается обнаруженным, если канал заблокирован,
    входной пиксельный тактовый сигнал работает и сигнал DE меняется.

    Значение 0 эквивалентно флагу V4L2_IN_ST_NO_SIGNAL битов статуса
    VIDIOC_ENUMINPUT в V4L2.

    | 0 - не обнаружен
    | 1 - обнаружен

**video_width** (R):
    Ширина видеопотока. Это фактическая ширина, определённая аппаратурой.

    Значение идентично тому, что VIDIOC_QUERY_DV_TIMINGS возвращает в поле width
    структуры v4l2_bt_timings.

**video_height** (R):
    Высота видеопотока. Это фактическая высота, определённая аппаратурой.

    Значение идентично тому, что VIDIOC_QUERY_DV_TIMINGS возвращает в поле height
    структуры v4l2_bt_timings.

**vsync_status** (R):
    Тип импульсов VSYNC, определённый детектором видеоформата.

    Значение эквивалентно флагам, возвращаемым VIDIOC_QUERY_DV_TIMINGS в поле
    polarities структуры v4l2_bt_timings.

    | 0 - активный низкий
    | 1 - активный высокий
    | 2 - недоступно

**hsync_status** (R):
    Тип импульсов HSYNC, определённый детектором видеоформата.

    Значение эквивалентно флагам, возвращаемым VIDIOC_QUERY_DV_TIMINGS в поле
    polarities структуры v4l2_bt_timings.

    | 0 - активный низкий
    | 1 - активный высокий
    | 2 - недоступно

**vsync_gap_length** (RW):
    Если входящий видеосигнал не содержит синхронизирующих импульсов VSYNC и
    HSYNC, они должны генерироваться внутренне в FPGA для достижения правильного
    порядка кадров. Это значение указывает, сколько «пустых» пикселей (пикселей
    со снятым сигналом Data Enable) необходимо для генерации внутреннего импульса
    VSYNC.

**hsync_gap_length** (RW):
    Если входящий видеосигнал не содержит синхронизирующих импульсов VSYNC и
    HSYNC, они должны генерироваться внутренне в FPGA для достижения правильного
    порядка кадров. Это значение указывает, сколько «пустых» пикселей (пикселей
    со снятым сигналом Data Enable) необходимо для генерации внутреннего импульса
    HSYNC. Значение должно быть больше 1 и меньше vsync_gap_length.

**pclk_frequency** (R):
    Частота входного пиксельного тактового сигнала в кГц.

    Значение идентично тому, что VIDIOC_QUERY_DV_TIMINGS возвращает в поле
    pixelclock структуры v4l2_bt_timings.

    *Примечание: параметр frequency_range должен быть установлен надлежащим
    образом, чтобы получить здесь корректную частоту.*

**hsync_width** (R):
    Ширина сигнала HSYNC в тактах PCLK.

    Значение идентично тому, что VIDIOC_QUERY_DV_TIMINGS возвращает в поле
    hsync структуры v4l2_bt_timings.

**vsync_width** (R):
    Ширина сигнала VSYNC в тактах PCLK.

    Значение идентично тому, что VIDIOC_QUERY_DV_TIMINGS возвращает в поле
    vsync структуры v4l2_bt_timings.

**hback_porch** (R):
    Число импульсов PCLK между снятием сигнала HSYNC и первым корректным пикселем
    в видеостроке (отмеченным DE=1).

    Значение идентично тому, что VIDIOC_QUERY_DV_TIMINGS возвращает в поле
    hbackporch структуры v4l2_bt_timings.

**hfront_porch** (R):
    Число импульсов PCLK между концом последнего корректного пикселя в
    видеостроке (отмеченного DE=1) и установкой сигнала HSYNC.

    Значение идентично тому, что VIDIOC_QUERY_DV_TIMINGS возвращает в поле
    hfrontporch структуры v4l2_bt_timings.

**vback_porch** (R):
    Число видеострок между снятием сигнала VSYNC и видеостроки с первым
    корректным пикселем (отмеченным DE=1).

    Значение идентично тому, что VIDIOC_QUERY_DV_TIMINGS возвращает в поле
    vbackporch структуры v4l2_bt_timings.

**vfront_porch** (R):
    Число видеострок между концом последней корректной строки пикселей
    (отмеченной DE=1) и установкой сигнала VSYNC.

    Значение идентично тому, что VIDIOC_QUERY_DV_TIMINGS возвращает в поле
    vfrontporch структуры v4l2_bt_timings.

**frequency_range** (RW)
    Диапазон частот PLL генератора входного тактового сигнала OLDI. Частота PLL
    производится из частоты пиксельного тактового сигнала (PCLK) и равна PCLK,
    если oldi_lane_width установлено в «single», и PCLK/2, если oldi_lane_width
    установлено в «dual».

    | 0 - PLL < 50MHz (по умолчанию)
    | 1 - PLL >= 50MHz

    *Примечание: этот параметр нельзя изменить, пока входное устройство v4l2
    открыто.*

Общие выходные параметры FPDL3/GMSL
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

**output_id** (R):
    Числовой идентификатор выхода, начинающийся с нуля.

**video_source** (RW):
    Источник выходного видео. Если установлено в 0 или 1, источником является
    соответствующий вход карты, а выходные устройства v4l2 отключены. Если
    установлено в 2 или 3, источником является соответствующее выходное
    видеоустройство v4l2. По умолчанию — соответствующий выход v4l2, то есть
    2 для OUT1 и 3 для OUT2.

    | 0 - вход 0
    | 1 - вход 1
    | 2 - выход v4l2 0
    | 3 - выход v4l2 1

    *Примечание: этот параметр нельзя изменить, пока открыто ЛЮБОЕ из входных/
    выходных устройств v4l2.*

**display_width** (RW):
    Ширина дисплея. Автоопределение подключённого дисплея отсутствует, поэтому
    надлежащее значение должно быть установлено перед началом потоковой передачи.
    Ширина по умолчанию — 1280.

    *Примечание: этот параметр нельзя изменить, пока выходное устройство v4l2
    открыто.*

**display_height** (RW):
    Высота дисплея. Автоопределение подключённого дисплея отсутствует, поэтому
    надлежащее значение должно быть установлено перед началом потоковой передачи.
    Высота по умолчанию — 640.

    *Примечание: этот параметр нельзя изменить, пока выходное устройство v4l2
    открыто.*

**frame_rate** (RW):
    Ограничение частоты кадров выходного видеосигнала в кадрах в секунду. Из-за
    ограниченных шагов выходного пиксельного тактового сигнала карта не всегда
    может сгенерировать частоту кадров, в точности соответствующую значению,
    требуемому подключённым дисплеем. С помощью этого параметра можно ограничить
    частоту кадров путём «искажения» сигнала так, чтобы строки не были одинаковы
    (porch'и последней строки отличаются), но сигнал для подключённого дисплея
    выглядел как имеющий точную частоту кадров. Ограничение частоты кадров по
    умолчанию — 60 Гц.

**hsync_polarity** (RW):
    Полярность сигнала HSYNC.

    | 0 - активный низкий (по умолчанию)
    | 1 - активный высокий

**vsync_polarity** (RW):
    Полярность сигнала VSYNC.

    | 0 - активный низкий (по умолчанию)
    | 1 - активный высокий

**de_polarity** (RW):
    Полярность сигнала DE.

    | 0 - активный низкий
    | 1 - активный высокий (по умолчанию)

**pclk_frequency** (RW):
    Частота выходного пиксельного тактового сигнала. Допустимые значения находятся
    в диапазоне 25000-190000(кГц), причём между двумя последовательными
    допустимыми частотами имеется нелинейный шаг. Драйвер находит ближайшую
    допустимую частоту к заданному значению и устанавливает её. При чтении этого
    свойства вы получаете точную частоту, установленную драйвером. Частота по
    умолчанию — 61150 кГц.

    *Примечание: этот параметр нельзя изменить, пока выходное устройство v4l2
    открыто.*

**hsync_width** (RW):
    Ширина сигнала HSYNC в пикселях. Значение по умолчанию — 40.

**vsync_width** (RW):
    Ширина сигнала VSYNC в видеостроках. Значение по умолчанию — 20.

**hback_porch** (RW):
    Число импульсов PCLK между снятием сигнала HSYNC и первым корректным пикселем
    в видеостроке (отмеченным DE=1). Значение по умолчанию — 50.

**hfront_porch** (RW):
    Число импульсов PCLK между концом последнего корректного пикселя в
    видеостроке (отмеченного DE=1) и установкой сигнала HSYNC. Значение по
    умолчанию — 50.

**vback_porch** (RW):
    Число видеострок между снятием сигнала VSYNC и видеостроки с первым корректным
    пикселем (отмеченным DE=1). Значение по умолчанию — 31.

**vfront_porch** (RW):
    Число видеострок между концом последней корректной строки пикселей
    (отмеченной DE=1) и установкой сигнала VSYNC. Значение по умолчанию — 30.

Специфичные для FPDL3 входные параметры
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

**fpdl3_input_width** (RW):
    Число входных линий десериализатора.

    | 0 - авто (по умолчанию)
    | 1 - одиночная
    | 2 - двойная

Специфичные для FPDL3 выходные параметры
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

**fpdl3_output_width** (RW):
    Число выходных линий сериализатора.

    | 0 - авто (по умолчанию)
    | 1 - одиночная
    | 2 - двойная

Специфичные для GMSL входные параметры
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

**gmsl_mode** (RW):
    Скоростной режим GMSL.

    | 0 - 12Gb/s (по умолчанию)
    | 1 - 6Gb/s
    | 2 - 3Gb/s
    | 3 - 1.5Gb/s

**gmsl_stream_id** (RW):
    Мультипоток GMSL содержит до четырёх видеопотоков. Этот параметр выбирает,
    какой поток захватывается видеовходом. Значение — это индекс потока,
    начинающийся с нуля. Идентификатор потока по умолчанию — 0.

    *Примечание: этот параметр нельзя изменить, пока входное устройство v4l2
    открыто.*

**gmsl_fec** (RW):
    Прямая коррекция ошибок GMSL (Forward Error Correction, FEC).

    | 0 - отключена
    | 1 - включена (по умолчанию)

Разделы MTD
-----------

Драйвер mgb4 создаёт устройство MTD с двумя разделами:
 - mgb4-fw.X - прошивка FPGA.
 - mgb4-data.X - заводские настройки, например серийный номер карты.

Раздел *mgb4-fw* доступен для записи и используется для обновлений прошивки,
*mgb4-data* доступен только для чтения. Символ *X*, присоединённый к имени
раздела, обозначает номер карты. В зависимости от конфигурации ядра
CONFIG_MTD_PARTITIONED_MASTER в системе может также присутствовать третий раздел
с именем *mgb4-flash*. Этот раздел представляет всю, неразбитую на разделы,
FLASH-память карты, и с ним лучше не возиться...

IIO (триггеры)
--------------

Драйвер mgb4 создаёт устройство Industrial I/O (IIO), которое предоставляет
возможность работы с триггерами и статусом уровня сигнала. Доступны следующие
элементы сканирования:

**activity**:
	Уровни триггеров и статус ожидания.

	| bit 1 - ожидание триггера 1
	| bit 2 - ожидание триггера 2
	| bit 5 - уровень триггера 1
	| bit 6 - уровень триггера 2

**timestamp**:
	Временная метка события триггера.

Устройство iio может работать либо в «сыром» (raw) режиме, в котором можно
получать уровни сигналов (биты activity 5 и 6) через доступ sysfs, либо в режиме
триггерного буфера. В режиме триггерного буфера можно отслеживать изменения
уровня сигнала (биты activity 1 и 2), используя устройство iio в /dev. Если
включить временные метки, вы также будете получать точное время события триггера,
которое можно сопоставить с видеокадром (каждый видеокадр mgb4 имеет временную
метку с тем же источником тактирования).

*Примечание: хотя выборка activity всегда содержит все биты статуса, не имеет
смысла получать биты ожидания в сыром режиме или биты уровня в режиме триггерного
буфера — значения не представляют корректных данных в таком случае.*
