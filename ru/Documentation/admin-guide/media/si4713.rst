Сведения об устройстве
----------------------

Эта микросхема — продукт компании Silicon Labs. Это устройство I2C, в настоящее время по адресу 0x63.
По сути, она обладает функциями передачи и измерения уровня шума сигнала.

Si4713 объединяет функции передачи для FM-вещания в стереорежиме.
Микросхема также позволяет выполнять интегрированное сканирование мощности приёма для
выявления FM-каналов со слабым уровнем сигнала.

Микросхема программируется с помощью команд и ответов. Существует также несколько
свойств, которые могут изменять поведение этой микросхемы.

Пользователи обязаны соблюдать местные нормы, регулирующие радиочастотную (RF) передачу.

Описание драйвера устройства
----------------------------

Для работы с этим устройством существует два модуля. Один — это драйвер устройства I2C,
а другой — платформенный драйвер.

Драйвер устройства I2C экспортирует в ядро интерфейс v4l2-subdev.
Ко всем свойствам также можно обращаться через интерфейс расширенных элементов управления v4l2,
используя вызовы v4l2-subdev (g_ext_ctrls, s_ext_ctrls).

Платформенный драйвер устройства экспортирует в пространство пользователя интерфейс radio-устройства v4l2.
Поэтому он использует драйвер устройства I2C как подустройство, чтобы передавать команды
пользователя реальному устройству. По сути, это обёртка над драйвером устройства I2C.

Приложения могут использовать v4l2 radio API для задания частоты работы, состояния mute
и т. д. Но большинство свойств устройства представлено в расширенных элементах управления.

Когда свойство mute v4l2 устанавливается в 1 (true), драйвер выключает микросхему.

Ниже приведена их сводка:

* Pilot — это слышимый тон, посылаемый устройством.

- pilot_frequency - Задаёт частоту стереофонического пилот-тона.
- pilot_deviation - Задаёт уровень частотной девиации пилот-тона.
- pilot_enabled - Включает или отключает функцию пилот-тона.

* Устройство si4713 способно применять аудиокомпрессию к
  передаваемому сигналу.

- acomp_enabled - Включает или отключает функцию управления динамическим диапазоном аудио.
- acomp_gain - Задаёт усиление для управления динамическим диапазоном аудио.
- acomp_threshold - Задаёт пороговый уровень для управления динамическим диапазоном аудио.
- acomp_attack_time - Задаёт время атаки для управления динамическим диапазоном аудио.
- acomp_release_time - Задаёт время восстановления для управления динамическим диапазоном аудио.

* Limiter настраивает функцию ограничителя девиации аудио. После возникновения
  чрезмерной девиации появляется возможность регулировать усиление входного каскада
  аудиовхода и всегда предотвращать чрезмерную девиацию.

- limiter_enabled - Включает или отключает функцию ограничителя.
- limiter_deviation - Задаёт уровень частотной девиации аудио.
- limiter_release_time - Задаёт время восстановления ограничителя.

* Мощность настройки

- power_level - Задаёт уровень выходной мощности для передачи сигнала.
  antenna_capacitor - Выбирает значение конденсатора настройки антенны
  вручную или автоматически, если задано нулевое значение.

* Относящиеся к RDS

- rds_ps_name - Задаёт поле RDS ps name для передачи.
- rds_radio_text - Задаёт RDS radio text для передачи.
- rds_pi - Задаёт поле RDS PI для передачи.
- rds_pty - Задаёт поле RDS PTY для передачи.

* Относящиеся к региону

- preemphasis - задаёт предыскажение, применяемое при передаче.

RNL
---

Это устройство также имеет интерфейс для измерения уровня принимаемого шума. Чтобы сделать
это, следует выполнить ioctl на узле устройства. Ниже приведён пример кода:

.. code-block:: none

	int main (int argc, char *argv[])
	{
		struct si4713_rnl rnl;
		int fd = open("/dev/radio0", O_RDWR);
		int rval;

		if (argc < 2)
			return -EINVAL;

		if (fd < 0)
			return fd;

		sscanf(argv[1], "%d", &rnl.frequency);

		rval = ioctl(fd, SI4713_IOC_MEASURE_RNL, &rnl);
		if (rval < 0)
			return rval;

		printf("received noise level: %d\n", rnl.rnl);

		close(fd);
	}

Структура si4713_rnl и SI4713_IOC_MEASURE_RNL определены в
include/linux/platform_data/media/si4713.h.

Стерео/моно и подканалы RDS
---------------------------

Устройство также можно настроить, используя доступные подканалы для
передачи. Чтобы сделать это, используйте ioctl S/G_MODULATOR и правильно настройте txsubchans.
Обратитесь к спецификации V4L2 API для корректного использования этого ioctl.

Тестирование
------------
Тестирование обычно выполняется с помощью утилиты v4l2-ctl для управления картами FM-тюнеров.
Этот инструмент можно найти в репозитории v4l-dvb в каталоге v4l2-apps/util.

Пример задания rds ps name:

.. code-block:: none

	# v4l2-ctl -d /dev/radio0 --set-ctrl=rds_ps_name="Dummy"
