Что предстоит сделать в драйвере
--------------------------------

- Согласно техническому описанию (datasheet) от SiLabs, прошивку
  радиочипа можно обновлять во время работы, тем самым доводя её до
  самой последней версии. К сожалению, мне не удалось найти никаких
  упоминаний об указанном обновлении прошивки для старых чипов, на
  которых я тестировал драйвер, поэтому для таких чипов драйвер
  предоставляет только старую функциональность.


Параметры, доступные через debugfs
-----------------------------------
SI476x позволяет пользователю получать множество характеристик, которые
могут быть очень полезны для тестирования EoL/оценки производительности
РЧ-тракта, — параметры, имеющие очень мало общего с подсистемой V4L2.
Такие параметры предоставляются через debugfs и доступны через обычные
файловые операции ввода-вывода.

Драйвер предоставляет следующие файлы:

* /sys/kernel/debug/<device-name>/acf
  Этот файл содержит информацию о состоянии ACF (Automatically
  Controlled Features). Содержимое файла представляет собой двоичные
  данные следующего формата:

  .. tabularcolumns:: |p{7ex}|p{12ex}|L|

  =============  ==============   ====================================
  Offset	 Name		  Description
  =============  ==============   ====================================
  0x00		 blend_int	  Флаг, устанавливается, когда стерео-
				  разделение опустилось ниже порога blend
  0x01		 hblend_int	  Флаг, устанавливается, когда частота
				  среза HiBlend ниже порога
  0x02		 hicut_int	  Флаг, устанавливается, когда частота
				  среза HiCut ниже порога
  0x03		 chbw_int	  Флаг, устанавливается, когда полоса
				  пропускания канального фильтра меньше порога
  0x04		 softmute_int	  Флаг, указывающий, что ослабление
				  softmute превысило порог softmute
  0x05		 smute		  0 - звук не приглушён softmute
				  1 - звук приглушён softmute
  0x06		 smattn		  Уровень ослабления softmute в дБ
  0x07		 chbw		  Полоса пропускания канального фильтра в кГц
  0x08		 hicut		  Частота среза HiCut в единицах по
				  100 Гц
  0x09		 hiblend	  Частота среза HiBlend в единицах по
				  100 Гц
  0x10		 pilot		  0 - стереопилот отсутствует
				  1 - стереопилот присутствует
  0x11		 stblend	  Стереосмешивание (blend) в %
  =============  ==============   ====================================


* /sys/kernel/debug/<device-name>/rds_blckcnt
  Этот файл содержит статистику о приёмах RDS. Его двоичные данные имеют
  следующий формат:

  .. tabularcolumns:: |p{7ex}|p{12ex}|L|

  =============  ==============   ====================================
  Offset	 Name		  Description
  =============  ==============   ====================================
  0x00		 expected	  Количество ожидаемых блоков RDS
  0x02		 received	  Количество принятых блоков RDS
  0x04		 uncorrectable	  Количество неисправимых блоков RDS
  =============  ==============   ====================================

* /sys/kernel/debug/<device-name>/agc
  Этот файл содержит информацию о параметрах, относящихся к AGC
  (Automatic Gain Control)

  Формат:

  .. tabularcolumns:: |p{7ex}|p{12ex}|L|

  =============  ==============   ====================================
  Offset	 Name		  Description
  =============  ==============   ====================================
  0x00		 mxhi		  0 - верхний порог PD микшера FM не
				  превышен
				  1 - верхний порог PD микшера FM
				  превышен
  0x01		 mxlo		  то же для нижнего PD микшера FM
  0x02		 lnahi		  то же для верхнего PD LNA FM
  0x03		 lnalo		  то же для нижнего PD LNA FM
  0x04		 fmagc1		  Сопротивление аттенюатора FMAGC1
				  (подробнее см. datasheet)
  0x05		 fmagc2		  то же для FMAGC2
  0x06		 pgagain	  Усиление PGA в дБ
  0x07		 fmwblang	  Усиление LNA FM/WB в дБ
  =============  ==============   ====================================

* /sys/kernel/debug/<device-name>/rsq
  Этот файл содержит информацию о параметрах, относящихся к RSQ
  (Received Signal Quality)

  Формат:

  .. tabularcolumns:: |p{7ex}|p{12ex}|p{60ex}|

  =============  ==============   ====================================
  Offset	 Name		  Description
  =============  ==============   ====================================
  0x00		 multhint	  0 - значение многолучёвости не
				  пересекло верхний порог Multipath
				  1 - значение многолучёвости
				  пересекло верхний порог Multipath
  0x01		 multlint	  то же для нижнего порога Multipath
  0x02		 snrhint	  0 - SNR принятого сигнала не
				  пересёк верхний порог
				  1 - SNR принятого сигнала
				  пересёк верхний порог
  0x03		 snrlint	  то же для нижнего порога
  0x04		 rssihint	  то же для верхнего порога RSSI
  0x05		 rssilint	  то же для нижнего порога RSSI
  0x06		 bltf		  Флаг, указывающий, достигла ли
				  команда поиска границы диапазона/перешла за неё
  0x07		 snr_ready	  Указывает, что метрики SNR готовы
  0x08		 rssiready	  то же для метрик RSSI
  0x09		 injside	  0 - используется нижняя инжекция
				  1 - используется верхняя инжекция
  0x10		 afcrl		  Флаг, указывающий, в насыщении ли AFC
  0x11		 valid		  Флаг, указывающий, действителен ли канал
  0x12		 readfreq	  Текущая настроенная частота
  0x14		 freqoff	  Знаковое смещение частоты в единицах
				  по 2 ppm
  0x15		 rssi		  Знаковое значение RSSI в dBuV
  0x16		 snr		  Знаковое РЧ SNR в дБ
  0x17		 issi		  Знаковый индикатор силы зеркального
				  сигнала (Image Strength Signal)
  0x18		 lassi		  Знаковый индикатор силы нижнего
				  соседнего канала
  0x19		 hassi		  то же для верхней стороны
  0x20		 mult		  Индикатор многолучёвости
  0x21		 dev		  Девиация частоты
  0x24		 assi		  SSI соседнего канала
  0x25		 usn		  Индикатор ультразвукового шума
  0x26		 pilotdev	  Девиация пилота в единицах по 100 Гц
  0x27		 rdsdev		  то же для RDS
  0x28		 assidev	  то же для ASSI
  0x29		 strongdev	  Девиация частоты
  0x30		 rdspi		  Код RDS PI
  =============  ==============   ====================================

* /sys/kernel/debug/<device-name>/rsq_primary
  Этот файл содержит информацию о параметрах, относящихся к RSQ
  (Received Signal Quality), только для первичного тюнера. Формат такой
  же, как выше.
