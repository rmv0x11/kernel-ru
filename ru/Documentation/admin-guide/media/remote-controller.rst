================================================================================
Поддержка инфракрасных пультов дистанционного управления в драйверах video4linux
================================================================================

Авторы: Gerd Hoffmann, Mauro Carvalho Chehab

Основы
======

Большинство аналоговых и цифровых ТВ-плат поддерживают пульты дистанционного
управления. Некоторые из них имеют микропроцессор, который принимает ИК-несущие,
преобразует их в последовательности импульс/пауза, а затем в скан-коды, возвращая
такие коды в пространство пользователя (userspace) («режим скан-кодов»). Другие
платы возвращают только последовательности импульс/пауза («сырой режим»).

Поддержка пультов дистанционного управления в режиме скан-кодов обеспечивается
стандартным слоем ввода Linux. Поддержка сырого режима обеспечивается через LIRC.

Чтобы проверить поддержку и протестировать её, рекомендуется загрузить
`v4l-utils <https://git.linuxtv.org/v4l-utils.git/>`_. Этот набор предоставляет
два инструмента для работы с пультами дистанционного управления:

- ir-keytable: предоставляет способ опросить пульт дистанционного управления,
  вывести список поддерживаемых им протоколов, включить встроенную в ядро
  поддержку ИК-декодера или переключить протокол, а также протестировать приём
  скан-кодов;

- ir-ctl: предоставляет инструменты для работы с пультами дистанционного
  управления, поддерживающими сырой режим через интерфейс LIRC.

Обычно модуль пульта дистанционного управления загружается автоматически при
обнаружении ТВ-карты. Однако для некоторых устройств необходимо вручную загрузить
модуль ir-kbd-i2c.

Как это работает
================

Модули регистрируют пульт как клавиатуру в слое ввода Linux, то есть вы увидите
клавиши пульта как обычные нажатия клавиш (если включён CONFIG_INPUT_KEYBOARD).

Используя event-устройства (CONFIG_INPUT_EVDEV), приложения могут обращаться к
пульту через устройства /dev/input/event<n>. udev/systemd автоматически создаст
эти устройства. Если вы установите
`v4l-utils <https://git.linuxtv.org/v4l-utils.git/>`_, может также автоматически
загрузиться иная keytable, отличная от используемой по умолчанию. Подробности см.
на man-странице ir-keytable.1 из
`v4l-utils <https://git.linuxtv.org/v4l-utils.git/>`_.

Инструмент ir-keytable удобен для устранения неполадок, то есть для проверки того,
действительно ли присутствует устройство ввода, каким из устройств оно является,
проверки того, действительно ли нажатие клавиш на пульте генерирует события, и
тому подобного.  Вы также можете использовать любую другую утилиту ввода, которая
изменяет раскладки клавиш, например утилиту input kbd.


Использование с lircd
---------------------

Последние версии демона lircd поддерживают чтение событий из слоя ввода Linux
(через event-устройство). Они также поддерживают приём ИК-кодов в режиме lirc.


Использование без lircd
-----------------------

Xorg распознаёт несколько ИК-кодов клавиш, числовое значение которых меньше 247.
С появлением Wayland драйвер ввода также был обновлён и теперь должен принимать
все коды клавиш. Тем не менее, вы можете захотеть просто переназначить коды клавиш
на что-то, что нравится вашему любимому медиаприложению.

Это можно сделать, настроив
`v4l-utils <https://git.linuxtv.org/v4l-utils.git/>`_ на загрузку вашей
собственной keytable во время выполнения. Подробности читайте на man-странице
ir-keytable.1.
