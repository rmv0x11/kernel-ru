Драйвер ivtv
============

Автор: Hans Verkuil <hverkuil@kernel.org>

Это драйвер устройства v4l2 для MPEG-кодера/декодера Conexant cx23415/6.
Микросхема cx23415 может выполнять как кодирование, так и декодирование, а
cx23416 — только кодирование MPEG. В настоящее время единственной картой с
полной поддержкой декодирования является Hauppauge PVR-350.

.. note::

   #) Этому драйверу требуется новейшая прошивка кодера (версия 2.06.039, размер
      376836 байт). Получить прошивку можно здесь:

      https://linuxtv.org/downloads/firmware/#conexant

   #) «обычные» ТВ-приложения не работают с этим драйвером; вам нужно
      приложение, способное обрабатывать MPEG на входе, например mplayer, xine,
      MythTV и т.п.

Основная цель проекта IVTV — предоставить реализацию драйвера Linux с открытым
исходным кодом, разработанную «с нуля» (clean room), для карт видеозахвата на
основе MPEG-кодека iCompression iTVC15 или Conexant CX23415/CX23416.

Возможности
-----------

 * Аппаратный захват mpeg2 эфирного видео (и звука) через тюнер либо
   S-Video/Composite и линейный аудиовход.
 * Аппаратный захват mpeg2 FM-радио там, где есть аппаратная поддержка.
 * Поддержка NTSC, PAL, SECAM со стереозвуком.
 * Поддержка SAP и двуязычных передач.
 * Поддержка необработанного VBI (субтитры closed captions и телетекст).
 * Поддержка сегментированного (sliced) VBI (субтитры closed captions и
   телетекст) и возможность вставки этих данных в захватываемый MPEG-поток.
 * Поддержка необработанного YUV и PCM на входе.

Дополнительные возможности для PVR-350 (на основе CX23415)
----------------------------------------------------------

 * Обеспечивает аппаратное воспроизведение mpeg2.
 * Обеспечивает полнофункциональный OSD (On Screen Display, то есть графику,
   накладываемую поверх видеосигнала).
 * Предоставляет фреймбуфер (позволяющий приложениям X выводиться на
   видеоустройство).
 * Поддержка необработанного YUV на выходе.

Устройства
----------

В настоящий момент допускается не более 12 плат ivtv.

Карты, не имеющие возможности видеовывода (т.е. карты, отличные от PVR350),
не имеют устройств vbi8, vbi16, video16 и video48. Они также не
поддерживают фреймбуфер-устройство /dev/fbx для OSD.

Устройство radio0 может присутствовать или отсутствовать в зависимости от того,
есть ли у карты радиотюнер.

Вот список базовых устройств v4l:

.. code-block:: none

	crw-rw----    1 root     video     81,   0 Jun 19 22:22 /dev/video0
	crw-rw----    1 root     video     81,  16 Jun 19 22:22 /dev/video16
	crw-rw----    1 root     video     81,  24 Jun 19 22:22 /dev/video24
	crw-rw----    1 root     video     81,  32 Jun 19 22:22 /dev/video32
	crw-rw----    1 root     video     81,  48 Jun 19 22:22 /dev/video48
	crw-rw----    1 root     video     81,  64 Jun 19 22:22 /dev/radio0
	crw-rw----    1 root     video     81, 224 Jun 19 22:22 /dev/vbi0
	crw-rw----    1 root     video     81, 228 Jun 19 22:22 /dev/vbi8
	crw-rw----    1 root     video     81, 232 Jun 19 22:22 /dev/vbi16

Базовые устройства
------------------

Для каждой дополнительной имеющейся у вас карты номера увеличиваются на единицу.
Например, /dev/video0 указано как «базовое» устройство захвата с кодированием,
поэтому имеем:

- /dev/video0  — устройство захвата с кодированием для первой карты (карта 0)
- /dev/video1  — устройство захвата с кодированием для второй карты (карта 1)
- /dev/video2  — устройство захвата с кодированием для третьей карты (карта 2)

Обратите внимание: если у первой карты нет какой-либо возможности (например, нет
декодера, а значит нет video16), вторая карта всё равно будет использовать
video17. Простое правило таково: «прибавь номер карты к номеру базового
устройства». Если у вас есть другие карты захвата (например, WinTV PCI),
которые обнаруживаются первыми, то нужно сообщить об этом модулю ivtv, чтобы он
начал отсчёт с 1 (или с 2, или с какого угодно числа). Иначе номера устройств
могут запутать. Для этого можно использовать опцию модуля ivtv
'ivtv_first_minor'.


- /dev/video0

  Устройство(а) захвата с кодированием.

  Только для чтения.

  Чтение из этого устройства даёт программный поток MPEG1/2.
  Пример:

  .. code-block:: none

	cat /dev/video0 > my.mpg (для выхода нужно нажать ctrl-c)


- /dev/video16

  Устройство(а) вывода декодера.

  Только для записи. Присутствует, только если существует MPEG-декодер (т.е.
  CX23415).

  Поток mpeg2, отправленный на это устройство, будет отображён на выбранном
  видеодисплее, а звук появится на линейном/аудиовыходе. Доступно только для
  карт с поддержкой видеовыхода. Пример:

  .. code-block:: none

	cat my.mpg >/dev/video16


- /dev/video24

  Устройство(а) захвата необработанного звука.

  Только для чтения.

  Необработанный стереопоток PCM-звука с выбранного в данный момент
  тюнера или линейного аудиовхода. Чтение из этого устройства приводит к
  необработанному захвату (signed 16 bit Little Endian, 48000 Hz, stereo pcm).
  Это устройство захватывает только звук. В будущем оно должно быть заменено
  устройством ALSA.
  Обратите внимание: соответствующего устройства вывода необработанного звука
  нет, это не поддерживается прошивкой декодера.


- /dev/video32

  Устройство(а) захвата необработанного видео.

  Только для чтения.

  Необработанный вывод видео YUV с текущего видеовхода. Формат YUV —
  16x16 линейно-тайловый формат NV12 (V4L2_PIX_FMT_NV12_16L16).

  Обратите внимание: потоки YUV и PCM не синхронизированы, поэтому их
  применение ограничено.


- /dev/video48

  Устройство(а) отображения необработанного видео.

  Только для записи. Присутствует, только если существует MPEG-декодер (т.е.
  CX23415).

  Записывает поток YUV в декодер карты.


- /dev/radio0

  Устройство(а) радиотюнера.

  Не может быть прочитано или записано.

  Используется для включения радиотюнера и настройки на частоту. С помощью
  этого устройства нельзя читать или записывать аудиопотоки. После того как вы
  настроите радио с помощью этого устройства, используйте /dev/video24 для
  чтения необработанного потока pcm или /dev/video0 для получения потока mpeg2
  с чёрным видео.


- /dev/vbi0

  Устройство(а) захвата «интервала вертикального гашения» (Teletext, CC, WSS
  и т.п.).

  Только для чтения.

  Захватывает необработанные (или сегментированные) видеоданные, отправляемые
  в течение интервала вертикального гашения (Vertical Blank Interval). Эти
  данные используются для кодирования телетекста, субтитров closed captions,
  VPS, широкоэкранной сигнализации, информации электронной программы передач и
  других сервисов.


- /dev/vbi8

  Устройство(а) обратной связи обработанного vbi.

  Только для чтения. Присутствует, только если существует MPEG-декодер (т.е.
  CX23415).

  Сегментированные данные VBI, встроенные в MPEG-поток, воспроизводятся на этом
  устройстве. Поэтому при воспроизведении записи на /dev/video16 вы можете
  читать встроенные данные VBI из /dev/vbi8.


- /dev/vbi16

  Устройство(а) «отображения» vbi.

  Только для записи. Присутствует, только если существует MPEG-декодер (т.е.
  CX23415).

  Может использоваться для отправки сегментированных данных VBI на разъём
  видеовыхода.
