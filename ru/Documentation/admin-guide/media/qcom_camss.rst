Драйвер Qualcomm Camera Subsystem
=================================

Введение
--------

Этот файл документирует драйвер Qualcomm Camera Subsystem, расположенный в
drivers/media/platform/qcom/camss.

Текущая версия драйвера поддерживает подсистему Camera Subsystem, имеющуюся в
процессорах Qualcomm MSM8916/APQ8016 и MSM8996/APQ8096.

Драйвер реализует интерфейсы V4L2, Media controller и V4L2 subdev.
Поддерживается датчик камеры, использующий интерфейс V4L2 subdev в ядре.

Драйвер реализован с использованием в качестве образца драйвера Qualcomm Camera
Subsystem для Android, который можно найти в Code Linaro [#f1]_ [#f2]_.


Оборудование Qualcomm Camera Subsystem
--------------------------------------

Оборудование Camera Subsystem, имеющееся в процессорах 8x16 / 8x96 и
поддерживаемое драйвером, состоит из:

- 2 / 3 модулей CSIPHY. Они обрабатывают физический уровень приёмников CSI2.
  К каждому из модулей CSIPHY может быть подключён отдельный датчик камеры;
- 2 / 4 модулей CSID (CSI Decoder). Они обрабатывают уровень протокола и уровень
  приложения приёмников CSI2. CSID может декодировать поток данных от любого из
  CSIPHY. Каждый CSID также содержит блок TG (Test Generator), который может
  генерировать искусственные входные данные в целях тестирования;
- модуля ISPIF (ISP Interface). Обрабатывает маршрутизацию потоков данных от
  CSID на входы VFE;
- 1 / 2 модуля(ей) VFE (Video Front End). Содержат конвейер аппаратных блоков
  обработки изображения. У VFE есть различные входные интерфейсы. Входной
  интерфейс PIX (Pixel) подаёт входные данные в конвейер обработки изображения.
  Конвейер обработки изображения содержит также модуль масштабирования и
  обрезки в конце. Три входных интерфейса RDI (Raw Dump Interface) обходят
  конвейер обработки изображения. VFE также содержит интерфейс шины AXI, который
  записывает выходные данные в память.


Поддерживаемая функциональность
-------------------------------

Текущая версия драйвера поддерживает:

- Ввод от датчика камеры через CSIPHY;
- Генерацию тестовых входных данных блоком TG в CSID;
- Интерфейс RDI у VFE

  - Сырой дамп входных данных в память.

    Поддерживаемые форматы:

    - YUYV/UYVY/YVYU/VYUY (упакованный YUV 4:2:2 - V4L2_PIX_FMT_YUYV /
      V4L2_PIX_FMT_UYVY / V4L2_PIX_FMT_YVYU / V4L2_PIX_FMT_VYUY);
    - MIPI RAW8 (8-битный Bayer RAW - V4L2_PIX_FMT_SRGGB8 /
      V4L2_PIX_FMT_SGRBG8 / V4L2_PIX_FMT_SGBRG8 / V4L2_PIX_FMT_SBGGR8);
    - MIPI RAW10 (10-битный упакованный Bayer RAW - V4L2_PIX_FMT_SBGGR10P /
      V4L2_PIX_FMT_SGBRG10P / V4L2_PIX_FMT_SGRBG10P / V4L2_PIX_FMT_SRGGB10P /
      V4L2_PIX_FMT_Y10P);
    - MIPI RAW12 (12-битный упакованный Bayer RAW - V4L2_PIX_FMT_SRGGB12P /
      V4L2_PIX_FMT_SGBRG12P / V4L2_PIX_FMT_SGRBG12P / V4L2_PIX_FMT_SRGGB12P).
    - (только 8x96) MIPI RAW14 (14-битный упакованный Bayer RAW - V4L2_PIX_FMT_SRGGB14P /
      V4L2_PIX_FMT_SGBRG14P / V4L2_PIX_FMT_SGRBG14P / V4L2_PIX_FMT_SRGGB14P).

  - (только 8x96) Преобразование формата входных данных.

    Поддерживаемые входные форматы:

    - MIPI RAW10 (10-битный упакованный Bayer RAW - V4L2_PIX_FMT_SBGGR10P / V4L2_PIX_FMT_Y10P).

    Поддерживаемые выходные форматы:

    - Plain16 RAW10 (10-битный распакованный Bayer RAW - V4L2_PIX_FMT_SBGGR10 / V4L2_PIX_FMT_Y10).

- Интерфейс PIX у VFE

  - Преобразование формата входных данных.

    Поддерживаемые входные форматы:

    - YUYV/UYVY/YVYU/VYUY (упакованный YUV 4:2:2 - V4L2_PIX_FMT_YUYV /
      V4L2_PIX_FMT_UYVY / V4L2_PIX_FMT_YVYU / V4L2_PIX_FMT_VYUY).

    Поддерживаемые выходные форматы:

    - NV12/NV21 (двухплоскостной YUV 4:2:0 - V4L2_PIX_FMT_NV12 / V4L2_PIX_FMT_NV21);
    - NV16/NV61 (двухплоскостной YUV 4:2:2 - V4L2_PIX_FMT_NV16 / V4L2_PIX_FMT_NV61).
    - (только 8x96) YUYV/UYVY/YVYU/VYUY (упакованный YUV 4:2:2 - V4L2_PIX_FMT_YUYV /
      V4L2_PIX_FMT_UYVY / V4L2_PIX_FMT_YVYU / V4L2_PIX_FMT_VYUY).

  - Поддержка масштабирования. Конфигурация модуля VFE Encoder Scale
    для уменьшения масштаба с коэффициентом до 16x.

  - Поддержка обрезки. Конфигурация модуля VFE Encoder Crop.

- Одновременное и независимое использование двух (8x96: трёх) входов данных -
  это могут быть датчики камеры и/или TG.


Архитектура и устройство драйвера
---------------------------------

Драйвер реализует интерфейс V4L2 subdev. С целью смоделировать аппаратные связи
между модулями и предоставить чистый, логичный и удобный интерфейс драйвер
разбит на устройства V4L2 sub-devices следующим образом (8x16 / 8x96):

- 2 / 3 sub-device CSIPHY - каждый CSIPHY представлен одним sub-device;
- 2 / 4 sub-device CSID - каждый CSID представлен одним sub-device;
- 2 / 4 sub-device ISPIF - ISPIF представлен числом sub-devices, равным
  числу sub-devices CSID;
- 4 / 8 sub-device VFE - VFE представлен числом sub-devices, равным числу
  входных интерфейсов (3 RDI и 1 PIX на каждый VFE).

Соображения, по которым драйвер разбит именно таким образом, следующие:

- представление модулей CSIPHY и CSID отдельным sub-device для каждого модуля
  позволяет смоделировать аппаратные связи между этими модулями;
- представление VFE отдельными sub-devices для каждого входного интерфейса
  позволяет использовать входные интерфейсы одновременно и независимо, как это
  поддерживается оборудованием;
- представление ISPIF числом sub-devices, равным числу sub-devices CSID,
  позволяет создавать линейные конвейеры media controller при одновременном
  использовании двух камер. Это позволяет избежать ответвлений в конвейерах,
  которые в противном случае потребовали бы от a) userspace и b) media-фреймворка
  (например, операций включения/выключения питания) делать предположения о
  потоке данных от sink pad к source pad на одной media-сущности.

Каждое sub-device VFE связано с отдельным узлом видеоустройства.

Реализация
----------

Конфигурация оборудования во время выполнения (обновление настроек во время
потоковой передачи) не требуется для реализации поддерживаемой в настоящее
время функциональности. Полная конфигурация каждого аппаратного модуля
применяется по ioctl STREAMON на основе текущих активных media-связей, форматов
и установленных элементов управления (controls).

Выходной размер модуля масштабирования (scaler) в VFE настраивается фактическим
прямоугольником выбора композиции (compose selection) на sink pad сущности
'msm_vfe0_pix'.

Область вывода обрезки модуля обрезки (crop) в VFE настраивается фактическим
прямоугольником выбора обрезки (crop selection) на source pad сущности
'msm_vfe0_pix'.
