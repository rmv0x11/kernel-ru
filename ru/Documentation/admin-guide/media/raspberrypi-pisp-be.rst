=========================================================
Raspberry Pi PiSP Back End Memory-to-Memory ISP (pisp-be)
=========================================================

PiSP Back End
=============

PiSP Back End — это memory-to-memory сигнальный процессор изображений (Image
Signal Processor, ISP), который читает данные изображения из памяти DRAM и
выполняет их обработку в соответствии с указаниями приложения, заданными через
параметры в буфере конфигурации, после чего записывает данные пикселей обратно в
память через два отдельных выходных канала.

Регистры ISP и его программная модель описаны в документе `Raspberry Pi
Image Signal Processor (PiSP) Specification document`_

PiSP Back End ISP обрабатывает изображения тайлами (tiles). Обработка
тесселяции изображения и вычисление низкоуровневых параметров конфигурации
реализуются свободно распространяемой библиотекой `libpisp
<https://github.com/raspberrypi/libpisp>`_.

Полный конвейер обработки изображений, который включает захват необработанных
данных RAW Bayer с датчика изображения через совместимый с MIPI CSI-2 интерфейс
захвата, их сохранение в памяти DRAM и обработку в PiSP Back End для получения
изображений, пригодных для использования приложением, реализован в `libcamera
<https://libcamera.org>`_ как часть поддержки платформы Raspberry Pi.

Драйвер pisp-be
===============

Драйвер Raspberry Pi PiSP Back End (pisp-be) расположен в каталоге
drivers/media/platform/raspberrypi/pisp-be. Он использует `V4L2 API` для
регистрации ряда устройств захвата и вывода видео, `V4L2 subdev API` для
регистрации поддустройства для ISP, которое соединяет видеоустройства в единый
медиаграф, реализованный с помощью `Media Controller (MC) API`.

Медиаграф регистрирует следующие узлы видеоустройств:

- pispbe-input: устройство вывода для изображений, передаваемых ISP на
  обработку.
- pispbe-tdn_input: устройство вывода для временного шумоподавления (temporal
  denoise).
- pispbe-stitch_input: устройство вывода для сшивки изображений (HDR).
- pispbe-output0: первое устройство захвата для обработанных изображений.
- pispbe-output1: второе устройство захвата для обработанных изображений.
- pispbe-tdn_output: устройство захвата для временного шумоподавления.
- pispbe-stitch_output: устройство захвата для сшивки изображений (HDR).
- pispbe-config: устройство вывода для параметров конфигурации ISP.

pispbe-input
------------

Изображения, подлежащие обработке ISP, помещаются в очередь узла устройства
вывода `pispbe-input`. Список форматов изображений, поддерживаемых на входе ISP,
приведён в документе `Raspberry Pi Image Signal Processor (PiSP) Specification
document`_.

pispbe-tdn_input, pispbe-tdn_output
-----------------------------------

Видеоустройство вывода `pispbe-tdn_input` принимает изображения, подлежащие
обработке блоком временного шумоподавления, которые захватываются с
видеоустройства захвата `pispbe-tdn_output`. Пространство пользователя
(userspace) отвечает за поддержание очередей на обоих устройствах и за
обеспечение того, чтобы буферы, завершённые на выходе, помещались в очередь на
входе.

pispbe-stitch_input, pispbe-stitch_output
-----------------------------------------

Для реализации обработки изображений HDR (high dynamic range, расширенный
динамический диапазон) используются блоки сшивки изображений и тональной
компрессии. `pispbe-stitch_output` записывает изображения в память, а
`pispbe-stitch_input` принимает ранее записанный кадр, чтобы обработать его
вместе с текущим входным изображением. Пространство пользователя (userspace)
отвечает за поддержание очередей на обоих устройствах и за обеспечение того,
чтобы буферы, завершённые на выходе, помещались в очередь на входе.

pispbe-output0, pispbe-output1
------------------------------

Два устройства захвата записывают в память данные пикселей в том виде, в котором
они обработаны ISP.

pispbe-config
-------------

Устройства вывода видео `pispbe-config` принимают буфер параметров конфигурации,
которые определяют желаемую обработку изображения, выполняемую ISP.

Формат параметра конфигурации ISP определяется C-структурой
:c:type:`pisp_be_tiles_config`, а смысл каждого параметра описан в документе
`Raspberry Pi Image Signal Processor (PiSP) Specification
document`_.

Конфигурация ISP
================

Конфигурация ISP описывается исключительно содержимым буфера параметров.
Единственный параметр, который пространству пользователя (userspace) необходимо
сконфигурировать с помощью V4L2 API, — это формат изображения на устройствах
вывода и захвата видео, используемый для проверки содержимого буфера параметров.

.. _Raspberry Pi Image Signal Processor (PiSP) Specification document: https://datasheets.raspberrypi.com/camera/raspberry-pi-image-signal-processor-specification.pdf
