Драйвер Renesas R-Car Fine Display Processor (FDP1)
===================================================

Драйвер R-Car FDP1 реализует специфичные для драйвера элементы управления, как описано ниже.

``V4L2_CID_DEINTERLACING_MODE (menu)``
    Режим деинтерлейсинга видео (такой как Bob, Weave, ...). Драйвер R-Car FDP1
    реализует следующие режимы.

.. flat-table::
    :header-rows:  0
    :stub-columns: 0
    :widths:       1 4

    * - ``"Progressive" (0)``
      - Входной видеопоток является прогрессивным (без чередования строк).
        Деинтерлейсинг не выполняется. За исключением (необязательного)
        преобразования формата и кодирования выходные кадры идентичны входным.
    * - ``"Adaptive 2D/3D" (1)``
      - Адаптивная по движению версия 2D- и 3D-деинтерлейсинга. Использует
        3D-деинтерлейсинг при наличии быстрого движения и 2D-деинтерлейсинг с
        диагональной интерполяцией в остальных случаях.
    * - ``"Fixed 2D" (2)``
      - Текущее поле масштабируется по вертикали путём усреднения соседних
        строк для восстановления недостающих строк. Этот метод также известен
        как смешивание (blending) или усреднение строк (Line Averaging, LAV).
    * - ``"Fixed 3D" (3)``
      - Предыдущее и следующее поля усредняются для восстановления строк,
        отсутствующих в текущем поле. Этот метод также известен как усреднение
        полей (Field Averaging, FAV).
    * - ``"Previous field" (4)``
      - Текущее поле сплетается с предыдущим полем, то есть предыдущее поле
        используется для заполнения недостающих строк текущего поля. Этот метод
        также известен как деинтерлейсинг weave.
    * - ``"Next field" (5)``
      - Текущее поле сплетается со следующим полем, то есть следующее поле
        используется для заполнения недостающих строк текущего поля. Этот метод
        также известен как деинтерлейсинг weave.
