==============================
Контроллер учёта CPU (cpuacct)
==============================

Контроллер учёта CPU используется для группировки задач с помощью cgroups и
учёта использования CPU этими группами задач.

Контроллер учёта CPU поддерживает группы с несколькими иерархиями. Группа
учёта накапливает использование CPU всеми своими дочерними группами и
задачами, непосредственно находящимися в этой группе.

Группы учёта можно создать, сначала смонтировав файловую систему cgroup::

  # mount -t cgroup -ocpuacct none /sys/fs/cgroup

После описанного выше шага начальная, или родительская, группа учёта
становится видимой в /sys/fs/cgroup. При загрузке системы эта группа включает
все задачи в системе. /sys/fs/cgroup/tasks перечисляет задачи в этой cgroup.
/sys/fs/cgroup/cpuacct.usage выдаёт время CPU (в наносекундах), полученное
этой группой, что по сути равно времени CPU, полученному всеми задачами
в системе.

Новые группы учёта можно создать под родительской группой /sys/fs/cgroup::

  # cd /sys/fs/cgroup
  # mkdir g1
  # echo $$ > g1/tasks

Описанные выше шаги создают новую группу g1 и перемещают в неё текущий процесс
оболочки (bash). Время CPU, потреблённое этим bash и его потомками,
можно получить из g1/cpuacct.usage, и оно же накапливается в
/sys/fs/cgroup/cpuacct.usage.

Файл cpuacct.stat перечисляет несколько статистик, которые дополнительно
делят время CPU, полученное cgroup, на пользовательское и системное время.
В настоящее время поддерживаются следующие статистики:

user: время, проведённое задачами cgroup в пользовательском режиме.
system: время, проведённое задачами cgroup в режиме ядра.

user и system измеряются в единицах USER_HZ.

Контроллер cpuacct использует интерфейс percpu_counter для сбора
пользовательского и системного времени. У этого есть два побочных эффекта:

- Теоретически возможно увидеть неверные значения пользовательского и
  системного времени. Это связано с тем, что percpu_counter_read() на
  32-битных системах небезопасен относительно параллельных записей.
- Возможно увидеть слегка устаревшие значения пользовательского и системного
  времени из-за пакетной природы обработки percpu_counter.
