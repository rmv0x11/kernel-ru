=================================
Cgroup сетевого приоритета
=================================

Cgroup сетевого приоритета (Network priority cgroup) предоставляет интерфейс,
позволяющий администратору динамически задавать приоритет сетевого трафика,
порождаемого различными приложениями.

Обычно приложение задаёт приоритет своего трафика через параметр сокета
SO_PRIORITY.  Однако это не всегда возможно, потому что:

1) Приложение может быть не запрограммировано на установку этого значения.
2) Приоритет трафика приложения зачастую является административным решением,
   специфичным для конкретной площадки, а не определяется приложением.

Этот cgroup позволяет администратору назначить процесс группе, которая определяет
приоритет исходящего трафика на заданном интерфейсе. Группы сетевого приоритета
можно создать, сначала смонтировав файловую систему cgroup::

	# mount -t cgroup -onet_prio none /sys/fs/cgroup/net_prio

После выполнения этого шага начальная группа, выступающая в роли родительской
учётной группы, становится видимой по пути '/sys/fs/cgroup/net_prio'.  Эта группа
включает все задачи в системе. '/sys/fs/cgroup/net_prio/tasks' перечисляет задачи
в этом cgroup.

Каждый cgroup net_prio содержит два файла, специфичных для подсистемы.

net_prio.prioidx
  Этот файл доступен только для чтения и носит чисто информативный характер.  Он
  содержит уникальное целочисленное значение, которое ядро использует как
  внутреннее представление данного cgroup.

net_prio.ifpriomap
  Этот файл содержит отображение приоритетов, назначенных трафику, исходящему
  от процессов в этой группе и покидающему систему через различные интерфейсы.
  Он содержит список кортежей в форме <ifname priority>.  Содержимое этого
  файла можно изменить, выведя строку в файл с использованием того же формата
  кортежа. Например::

	echo "eth0 5" > /sys/fs/cgroups/net_prio/iscsi/net_prio.ifpriomap

Эта команда заставит любой трафик, исходящий от процессов, принадлежащих cgroup
net_prio iscsi и покидающий систему через интерфейс eth0, получить приоритет
указанного трафика, равный значению 5. Родительская учётная группа также имеет
доступный для записи файл 'net_prio.ifpriomap', который можно использовать для
задания системного приоритета по умолчанию.

Приоритеты устанавливаются непосредственно перед постановкой кадра в очередь к
дисциплине очередей устройства (qdisc), поэтому приоритеты будут назначены до того,
как будет произведён выбор аппаратной очереди.

Один из вариантов использования cgroup net_prio — совместно с qdisc mqprio,
позволяющим направлять трафик приложений к аппаратным/драйверным классам трафика.
Этими отображениями затем могут управлять администраторы или другие сетевые
протоколы, такие как DCBX.

Новый cgroup net_prio наследует конфигурацию родителя.
