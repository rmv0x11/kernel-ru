Драйвер для тестирования GPIO
=============================

.. note::

   Этот модуль признан устаревшим в пользу более гибкого gpio-sim.rst.
   Новые разработки должны использовать тот API, а существующие разработки
   рекомендуется как можно скорее перевести на него.
   Этот модуль продолжит сопровождаться, но новая функциональность в него
   добавляться не будет.

Драйвер для тестирования GPIO (gpio-mockup) предоставляет способ создавать
имитированные микросхемы GPIO для целей тестирования. К линиям, экспортируемым
этими микросхемами, можно обращаться как через стандартный символьный интерфейс
устройства GPIO, так и манипулировать ими через выделенную структуру каталогов
debugfs.

Создание имитированных микросхем с помощью параметров модуля
------------------------------------------------------------

При загрузке драйвера gpio-mockup модулю можно передать ряд параметров.

    gpio_mockup_ranges

        Этот параметр принимает аргумент в виде массива пар целых чисел. Каждая
        пара задаёт базовый номер GPIO (неотрицательное целое число) и первое
        число после последнего у данной микросхемы. Если базовый GPIO равен -1,
        gpiolib назначит его автоматически. При этом следующий параметр — это
        количество линий, экспортируемых микросхемой.

        Пример: gpio_mockup_ranges=-1,8,-1,16,405,409

        Приведённая выше строка создаёт три микросхемы. Первая экспортирует
        8 линий, вторая — 16, а третья — 4. Базовый GPIO для третьей микросхемы
        задан равным 405, тогда как для двух первых микросхем он будет назначен
        автоматически.

    gpio_mockup_named_lines

        Этот параметр не принимает никаких аргументов. Он сообщает драйверу,
        что экспортируемые им линии GPIO должны быть именованными.

        Формат имени: gpio-mockup-X-Y, где X — идентификатор имитированной
        микросхемы, а Y — смещение линии.

Манипулирование имитированными линиями
--------------------------------------

Каждая имитированная микросхема создаёт собственный подкаталог в
/sys/kernel/debug/gpio-mockup/. Каталог называется по метке микросхемы. Также
создаётся символическая ссылка, названная по имени микросхемы, которая указывает
на каталог метки.

Внутри каждого подкаталога находится отдельный атрибут для каждой линии GPIO.
Имя атрибута представляет смещение линии в микросхеме.

Чтение из атрибута линии возвращает текущее значение. Запись в него (0 или 1)
изменяет конфигурацию имитированного подтягивающего вверх/вниз резистора
(1 — подтяжка вверх, 0 — подтяжка вниз).
