Симулятор GPIO на основе configfs
=================================

Симулятор GPIO на основе configfs (gpio-sim) предоставляет способ создавать
симулированные микросхемы GPIO для целей тестирования. К линиям, экспонируемым
этими микросхемами, можно обращаться через стандартный интерфейс символьного
устройства GPIO, а также управлять ими с помощью атрибутов sysfs.

Создание симулированных микросхем
---------------------------------

Модуль gpio-sim регистрирует подсистему configfs с именем ``'gpio-sim'``.
Подробности о файловой системе configfs см. в документации по configfs.

Пользователь может создавать иерархию групп и элементов configfs, а также
изменять значения экспонируемых атрибутов. После создания экземпляра микросхемы
эта иерархия будет преобразована в соответствующие свойства устройства. Общая
структура такова:

**Группа:** ``/config/gpio-sim``

Это верхний каталог дерева configfs модуля gpio-sim.

**Группа:** ``/config/gpio-sim/gpio-device``

**Атрибут:** ``/config/gpio-sim/gpio-device/dev_name``

**Атрибут:** ``/config/gpio-sim/gpio-device/live``

Это каталог, представляющий платформенное устройство GPIO. Атрибут
``'dev_name'`` доступен только для чтения и позволяет пространству пользователя
прочитать имя платформенного устройства (например, ``'gpio-sim.0'``). Атрибут
``'live'`` позволяет инициировать фактическое создание устройства после того,
как оно полностью сконфигурировано. Допустимые значения: ``'1'`` для включения
симулированного устройства и ``'0'`` для его отключения и демонтажа.

**Группа:** ``/config/gpio-sim/gpio-device/gpio-bankX``

**Атрибут:** ``/config/gpio-sim/gpio-device/gpio-bankX/chip_name``

**Атрибут:** ``/config/gpio-sim/gpio-device/gpio-bankX/num_lines``

Эта группа представляет банк GPIO под верхним платформенным устройством. Атрибут
``'chip_name'`` доступен только для чтения и позволяет пространству пользователя
прочитать имя устройства данного банка. Атрибут ``'num_lines'`` позволяет задать
число линий, экспонируемых этим банком.

**Группа:** ``/config/gpio-sim/gpio-device/gpio-bankX/lineY``

**Атрибут:** ``/config/gpio-sim/gpio-device/gpio-bankX/lineY/name``

**Атрибут:** ``/config/gpio-sim/gpio-device/gpio-bankX/lineY/valid``

Эта группа представляет одну линию по смещению Y. Атрибут ``valid`` указывает,
может ли линия использоваться как GPIO. Атрибут ``name`` позволяет задать имя
линии, как оно представлено в свойстве 'gpio-line-names'.

**Элемент:** ``/config/gpio-sim/gpio-device/gpio-bankX/lineY/hog``

**Атрибут:** ``/config/gpio-sim/gpio-device/gpio-bankX/lineY/hog/name``

**Атрибут:** ``/config/gpio-sim/gpio-device/gpio-bankX/lineY/hog/direction``

Этот элемент заставляет модуль gpio-sim захватить (hog) связанную линию. Атрибут
``'name'`` задаёт имя внутриядерного потребителя, которое следует использовать.
Атрибут ``'direction'`` задаёт направление захвата и должен быть одним из:
``'input'``, ``'output-high'`` и ``'output-low'``.

Внутри каталога каждого банка находится набор атрибутов, которые можно
использовать для настройки новой микросхемы. Кроме того, пользователь может
создавать с помощью ``mkdir()`` подкаталоги внутри каталога микросхемы, что
позволяет передавать дополнительную конфигурацию для конкретных линий. Имена
таких подкаталогов должны иметь вид: ``'line<offset>'`` (например, ``'line0'``,
``'line20'`` и т. д.), так как имя будет использоваться модулем для назначения
конфигурации конкретной линии по заданному смещению.

После завершения конфигурации атрибуту ``'live'`` необходимо установить значение
1, чтобы создать экземпляр микросхемы. Его можно вернуть в 0, чтобы уничтожить
симулированную микросхему. Модуль будет синхронно ожидать успешного зондирования
(probe) нового симулированного устройства, и если этого не происходит, запись в
``'live'`` приведёт к ошибке.

Симулированные микросхемы GPIO также могут быть определены в дереве устройств
(device-tree). Строка совместимости (compatible) должна быть: ``"gpio-simulator"``.
Поддерживаемые свойства:

  ``"gpio-sim,label"`` - метка микросхемы

Другие стандартные свойства GPIO (такие как ``"gpio-line-names"``, ``"ngpios"``
или ``"gpio-hog"``) также поддерживаются. Подробности см. в документации по GPIO.

Пример кода дерева устройств, определяющего симулятор GPIO:

.. code-block :: none

    gpio-sim {
        compatible = "gpio-simulator";

        bank0 {
            gpio-controller;
            #gpio-cells = <2>;
            ngpios = <16>;
            gpio-sim,label = "dt-bank0";
            gpio-line-names = "", "sim-foo", "", "sim-bar";
        };

        bank1 {
            gpio-controller;
            #gpio-cells = <2>;
            ngpios = <8>;
            gpio-sim,label = "dt-bank1";

            line3 {
                gpio-hog;
                gpios = <3 0>;
                output-high;
                line-name = "sim-hog-from-dt";
            };
        };
    };

Управление симулированными линиями
----------------------------------

Каждая симулированная микросхема GPIO создаёт отдельную группу sysfs в каталоге
своего устройства для каждой экспонируемой линии
(например, ``/sys/devices/platform/gpio-sim.X/gpiochipY/``). Имя каждой группы
имеет вид: ``'sim_gpioX'``, где X — смещение линии. Внутри каждой группы есть два
атрибута:

    ``pull`` - позволяет читать и устанавливать текущую симулированную настройку
               подтяжки (pull) для каждой линии; при записи значение должно быть
               одним из: ``'pull-up'``, ``'pull-down'``

    ``value`` - позволяет прочитать текущее значение линии, которое может
                отличаться от подтяжки, если линия управляется из пространства
                пользователя
