=======
SELinux
=======

Информацию о подсистеме ядра SELinux можно найти по следующим
ссылкам:

	https://git.kernel.org/pub/scm/linux/kernel/git/pcmoore/selinux.git/tree/README.md

	https://github.com/selinuxproject/selinux-kernel/wiki

Информацию о пространстве пользователя SELinux можно найти по адресу:

	https://github.com/SELinuxProject/selinux/wiki

Если вы хотите использовать SELinux, скорее всего, вам понадобятся
политики, предоставляемые дистрибутивом, либо установка
последнего релиза эталонной политики (reference policy) с

	https://github.com/SELinuxProject/refpolicy

Однако, если вы хотите установить фиктивную (dummy) политику для
тестирования, вы можете сделать это с помощью ``mdp``, предоставляемого в
scripts/selinux.  Обратите внимание, что для этого требуется установленное
пространство пользователя selinux — в частности, вам
понадобится checkpolicy для компиляции ядра, а также setfiles и
fixfiles для разметки файловой системы.

	1. Скомпилируйте ядро с включённым selinux.
	2. Введите ``make``, чтобы скомпилировать ``mdp``.
	3. Убедитесь, что система не работает с
	   включённым SELinux и реальной политикой.  Если
	   это так, перезагрузитесь с отключённым selinux,
	   прежде чем продолжить.
	4. Запустите install_policy.sh::

		cd scripts/selinux
		sh install_policy.sh

Шаг 4 создаст новую фиктивную политику, действительную для вашего
ядра, с единственным пользователем, ролью и типом selinux.
Он скомпилирует политику, установит ``SELINUXTYPE`` в значение
``dummy`` в ``/etc/selinux/config``, установит скомпилированную политику
как ``dummy`` и переразметит вашу файловую систему.
