=======
LoadPin
=======

LoadPin — это модуль безопасности Linux (Linux Security Module), который
гарантирует, что все загружаемые ядром файлы (модули, прошивки и т.п.)
происходят из одной и той же файловой системы, в расчёте на то, что эта
файловая система размещена на устройстве только для чтения, таком как
dm-verity или CDROM. Это позволяет системам с проверенной и/или неизменяемой
файловой системой применять ограничения на загрузку модулей и прошивок без
необходимости подписывать файлы по отдельности.

Этот LSM выбирается на этапе сборки с помощью ``CONFIG_SECURITY_LOADPIN`` и
может управляться на этапе загрузки через параметр командной строки ядра
"``loadpin.enforce``". По умолчанию он включён, но может быть отключён при
загрузке ("``loadpin.enforce=0``").

LoadPin начинает закрепление (pinning), когда видит первый загруженный файл.
Если блочное устройство, на котором размещена файловая система, не является
устройством только для чтения, создаётся sysctl для переключения закрепления:
``/proc/sys/kernel/loadpin/enabled``. (Изменяемая файловая система означает,
что и закрепление становится изменяемым, но наличие sysctl упрощает
тестирование на системах с изменяемой файловой системой.)

Также возможно исключить определённые типы файлов из LoadPin с помощью
параметра командной строки ядра "``loadpin.exclude``". По умолчанию включены
все файлы, но их можно исключить с помощью параметра командной строки ядра,
например "``loadpin.exclude=kernel-module,kexec-image``". Это позволяет
использовать различные механизмы, такие как ``CONFIG_MODULE_SIG`` и
``CONFIG_KEXEC_VERIFY_SIG``, для проверки модуля ядра и образа ядра, продолжая
при этом использовать LoadPin для защиты целостности других файлов, загружаемых
ядром. Полный список допустимых типов файлов можно найти в
``kernel_read_file_str``, определённом в ``include/linux/kernel_read_file.h``.
