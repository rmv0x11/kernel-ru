===============================================
Использование Linux Security Module (LSM)
===============================================

Фреймворк Linux Security Module (LSM) предоставляет механизм, позволяющий
новым расширениям ядра встраивать (hook) различные проверки безопасности.
Название «module» здесь несколько вводит в заблуждение, поскольку эти
расширения на самом деле не являются загружаемыми модулями ядра. Вместо
этого они выбираются на этапе сборки через CONFIG_DEFAULT_SECURITY и могут
быть переопределены во время загрузки через аргумент командной строки ядра
``"security=..."`` — в случае, когда в данное ядро было встроено несколько
LSM.

Основные потребители интерфейса LSM — это расширения мандатного управления
доступом (Mandatory Access Control, MAC), которые предоставляют комплексную
политику безопасности. В качестве примеров можно назвать SELinux, Smack,
Tomoyo и AppArmor. Помимо крупных MAC-расширений, на основе LSM могут быть
построены и другие расширения, обеспечивающие отдельные изменения в работе
системы в тех случаях, когда такие правки недоступны в базовой
функциональности самого Linux.

Модули capabilities Linux включаются всегда. За ними может следовать любое
число «второстепенных» (minor) модулей и не более одного «основного» (major)
модуля. Более подробную информацию о capabilities см. в ``capabilities(7)`` в
проекте Linux man-pages.

Список активных модулей безопасности можно получить, прочитав
``/sys/kernel/security/lsm``. Это список, разделённый запятыми, и он всегда
включает модуль capability. Список отражает порядок, в котором выполняются
проверки. Модуль capability всегда стоит первым, за ним следуют любые
«второстепенные» модули (например, Yama), а затем единственный «основной»
модуль (например, SELinux), если он сконфигурирован.

Доступ к атрибутам процесса, связанным с «основными» модулями безопасности,
и их сопровождение должны осуществляться с помощью специальных файлов в
``/proc/.../attr``. Модуль безопасности может поддерживать там специфичный
для модуля подкаталог, названный по имени этого модуля. ``/proc/.../attr/smack``
предоставляется модулем безопасности Smack и содержит все его специальные
файлы. Файлы, расположенные непосредственно в ``/proc/.../attr``, остаются в
качестве унаследованных интерфейсов для модулей, предоставляющих подкаталоги.
