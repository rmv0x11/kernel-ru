============================================================
Отчёты о событиях термического троттлинга процессоров Intel
============================================================

:Author: Srinivas Pandruvada <srinivas.pandruvada@linux.intel.com>

Введение
--------

Процессоры Intel имеют встроенные механизмы автоматического и адаптивного
термического мониторинга, которые заставляют процессор снижать своё
энергопотребление, чтобы работать в пределах заранее определённых температурных
ограничений.

Подробности см. в разделе «THERMAL MONITORING AND PROTECTION» в документе
«Intel® 64 and IA-32 Architectures Software Developer’s Manual Volume 3 (3A, 3B,
3C, & 3D): System Programming Guide».

В общем случае существуют два механизма управления температурой ядра процессора.
Они называются «Thermal Monitor 1 (TM1) и Thermal Monitor 2 (TM2)».

Состояние датчика температуры, который запускает термический монитор (TM1/TM2),
отражается флагами «thermal status flag» и «thermal status log flag» в
MSR_IA32_THERM_STATUS на уровне ядра и MSR_IA32_PACKAGE_THERM_STATUS на уровне
пакета.

Thermal Status flag, бит 0 — когда установлен, указывает, что температура ядра
процессора в данный момент находится на пороговой температуре термического
монитора и что энергопотребление процессора снижается с помощью либо TM1, либо
TM2, в зависимости от того, какой из них включён. Когда сброшен, флаг указывает,
что температура ядра ниже пороговой температуры термического монитора. Этот флаг
доступен только для чтения.

Thermal Status Log flag, бит 1 — когда установлен, указывает, что термический
датчик срабатывал с момента последнего включения питания или сброса, либо с
момента, когда программное обеспечение в последний раз сбрасывало этот флаг. Этот
флаг является «залипающим» (sticky) битом; будучи установленным, он остаётся
установленным до тех пор, пока не будет сброшен программным обеспечением или пока
не произойдёт включение питания или сброс процессора. Состояние по умолчанию —
сброшен.

Возможна ситуация, когда при чтении пользователем MSR_IA32_THERM_STATUS или
MSR_IA32_PACKAGE_THERM_STATUS TM1/TM2 не активны. В этом случае «Thermal Status
flag» будет прочитан как «0», а «Thermal Status Log flag» будет установлен, чтобы
показать любую предшествующую активацию «TM1/TM2». Но поскольку он должен быть
сброшен программным обеспечением, он не может показать количество случаев
активации «TM1/TM2».

Поэтому Linux предоставляет счётчики того, сколько раз был установлен «Thermal
Status flag». Также отображается, как долго «Thermal Status flag» был активен, в
миллисекундах. Используя эти счётчики, пользователи могут проверить, была ли
производительность ограничена из-за термических событий. Рекомендуется читать из
sysfs вместо непосредственного чтения регистров MSR, так как «Thermal Status Log
flag» сбрасывается драйвером для реализации ограничения частоты обновления.

Интерфейс Sysfs
---------------

События термического троттлинга представлены для каждого CPU в
«/sys/devices/system/cpu/cpuX/thermal_throttle/», где «X» — номер CPU.

Все эти счётчики доступны только для чтения. Их нельзя сбросить в 0. Поэтому они
потенциально могут переполниться после достижения максимального 64-битного
беззнакового целого.

``core_throttle_count``
	Показывает, сколько раз «Thermal Status flag» изменялся с 0 на 1 для данного
	CPU с момента загрузки ОС и инициализации термического вектора. Это 64-битный
	счётчик.

``package_throttle_count``
	Показывает, сколько раз «Thermal Status flag» изменялся с 0 на 1 для пакета,
	содержащего данный CPU, с момента загрузки ОС и инициализации термического
	вектора. Состояние пакета транслируется всем CPU; все CPU в пакете увеличивают
	этот счётчик. Это 64-битный счётчик.

``core_throttle_max_time_ms``
	Показывает максимальное время, в течение которого «Thermal Status flag» был
	установлен в 1 для данного CPU на уровне ядра с момента загрузки ОС и
	инициализации термического вектора.

``package_throttle_max_time_ms``
	Показывает максимальное время, в течение которого «Thermal Status flag» был
	установлен в 1 для пакета, содержащего данный CPU, с момента загрузки ОС и
	инициализации термического вектора.

``core_throttle_total_time_ms``
	Показывает суммарное время, в течение которого «Thermal Status flag» был
	установлен в 1 для данного CPU на уровне ядра с момента загрузки ОС и
	инициализации термического вектора.

``package_throttle_total_time_ms``
	Показывает суммарное время, в течение которого «Thermal Status flag» был
	установлен в 1 для пакета, содержащего данный CPU, с момента загрузки ОС и
	инициализации термического вектора.
