==========================================
Структура данных ядра для DRBD-9
==========================================

Здесь описывается внутриядерная структура данных для DRBD-9. Начиная с
Linux v3.14 мы реорганизуем DRBD для использования этой структуры данных.

Базовая структура данных
=========================

Узел имеет некоторое число ресурсов DRBD. Каждый такой ресурс имеет некоторое
число устройств (так называемых томов) и соединений с другими узлами («одноранговые
узлы»). Каждое устройство DRBD локально представлено блочным устройством.

Объекты DRBD взаимосвязаны и образуют матрицу, изображённую ниже; объект
drbd_peer_device располагается на каждом пересечении между drbd_device и
drbd_connection::

  /--------------+---------------+.....+---------------\
  |   resource   |    device     |     |    device     |
  +--------------+---------------+.....+---------------+
  |  connection  |  peer_device  |     |  peer_device  |
  +--------------+---------------+.....+---------------+
  :              :               :     :               :
  :              :               :     :               :
  +--------------+---------------+.....+---------------+
  |  connection  |  peer_device  |     |  peer_device  |
  \--------------+---------------+.....+---------------/

В этой таблице по горизонтали к устройствам можно обращаться из ресурсов по их
номеру тома. Аналогично, к peer_devices можно обращаться из соединений по их
номеру тома. Объекты в вертикальном направлении связаны двусвязными списками.
Существуют обратные указатели от peer_devices к их соединениям и устройствам, а
также от соединений и устройств к их ресурсу.

Все ресурсы находятся в двусвязном списке drbd_resources. Кроме того, ко всем
устройствам можно обращаться по их младшему номеру устройства через idr
drbd_devices.

Объекты drbd_resource, drbd_connection и drbd_device имеют счётчик ссылок.
Объекты peer_device служат лишь для установления связей между устройствами и
соединениями; их время жизни определяется временем жизни устройства и соединения,
на которые они ссылаются.
