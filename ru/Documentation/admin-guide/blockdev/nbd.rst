===========================================
Сетевое блочное устройство (версия для TCP)
===========================================

1) Обзор
--------

Что это такое: при включении этой функциональности в ядро (или сборке в виде
модуля) Linux может использовать удалённый сервер как одно из своих блочных
устройств. Поэтому каждый раз, когда клиентский компьютер хочет прочитать,
например, /dev/nb0, он посылает запрос по TCP серверу, который в ответ
присылает прочитанные данные. Это можно использовать для станций с малым
объёмом дискового пространства (или вообще бездисковых), чтобы заимствовать
дисковое пространство у другого компьютера.
В отличие от NFS, на нём можно разместить любую файловую систему и т.д.

Дополнительную информацию, а также инструменты nbd-client и nbd-server для
загрузки можно найти по адресу https://github.com/NetworkBlockDevice/nbd.

Модуль ядра nbd нужно устанавливать только на клиентской системе, поскольку
nbd-server полностью работает в пространстве пользователя (userspace). Более
того, nbd-server был успешно портирован на другие операционные системы,
включая Windows.

A) Параметры NBD
----------------

max_part
	Число разделов на устройство (по умолчанию: 0).

nbds_max
	Число блочных устройств, которые должны быть инициализированы
	(по умолчанию: 16).
