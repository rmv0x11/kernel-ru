1. Введение
===========

Благодаря простоте и почти повсеместной распространённости интерфейса
параллельного порта в персональных компьютерах, многие внешние устройства,
такие как портативные жёсткие диски, приводы CD-ROM, LS-120 и ленточные
накопители, используют параллельный порт для подключения к компьютеру-хосту.
Хотя некоторые устройства (в частности, сканеры) применяют специальные методы
для передачи команд и данных через интерфейс параллельного порта, большинство
внешних устройств фактически идентичны внутренней модели, но с добавленным
чипом-адаптером параллельного порта.  Некоторые из исходных адаптеров
параллельного порта представляли собой немногим больше, чем механизмы для
мультиплексирования шины SCSI.
(Адаптер Iomega PPA-3, используемый в накопителях ZIP, является примером
такого подхода).  Однако большинство современных конструкций используют иной
подход.  Чип-адаптер воспроизводит небольшую шину ISA или IDE во внешнем
устройстве, а коммуникационный протокол предоставляет операции для чтения и
записи регистров устройства, а также функции передачи блоков данных.  Иногда
устройство, к которому происходит обращение через параллельный кабель,
представляет собой стандартный контроллер SCSI, например NCR 5380.  Семейство
внешних ленточных накопителей «ditto» использует репликатор ISA для подключения
контроллера флоппи-диска, который затем соединяется с механизмом floppy-tape.
Однако подавляющее большинство внешних устройств с параллельным портом теперь
основано на стандартных устройствах типа IDE, которые не требуют промежуточного
контроллера.  Если бы кто-то, например, вскрыл привод CD-ROM с параллельным
портом, он обнаружил бы стандартный привод ATAPI CD-ROM, блок питания и один
адаптер, соединяющий стандартный кабель параллельного порта PC и стандартный
кабель IDE.  Обычно можно заменить устройство CD-ROM любым другим устройством,
использующим интерфейс IDE.

Этот документ описывает поддержку в Linux устройств IDE на параллельном порту.
Он не охватывает устройства SCSI на параллельном порту, ленточные накопители
«ditto» и сканеры.  Подсистемой IDE параллельного порта поддерживается множество
различных устройств, включая:

	- MicroSolutions backpack CD-ROM
	- MicroSolutions backpack PD/CD
	- MicroSolutions backpack hard-drives
	- MicroSolutions backpack 8000t tape drive
	- SyQuest EZ-135, EZ-230 & SparQ drives
	- Avatar Shark
	- Imation Superdisk LS-120
	- Maxell Superdisk LS-120
	- FreeCom Power CD
	- Hewlett-Packard 5GB and 8GB tape drives
	- Hewlett-Packard 7100 and 7200 CD-RW drives

а также большинство клонов и безымянных продуктов, представленных на рынке.

Для поддержки столь широкого спектра устройств pata_parport фактически разделён
на две части. Существует базовый модуль pata_parport, который предоставляет
интерфейс к подсистеме libata ядра, реестр и некоторые общие методы для доступа
к параллельным портам.

Второй компонент — это набор низкоуровневых протокольных драйверов для каждого
из чипов-адаптеров IDE параллельного порта.  Благодаря интересу и поддержке
пользователей Linux из многих уголков мира, поддержка доступна почти для всех
известных протоколов адаптеров:

	====    ====================================== ====
        aten    ATEN EH-100                            (HK)
        bpck    Microsolutions backpack                (US)
        comm    DataStor (old-type) "commuter" adapter (TW)
        dstr    DataStor EP-2000                       (TW)
        epat    Shuttle EPAT                           (UK)
        epia    Shuttle EPIA                           (UK)
	fit2    FIT TD-2000			       (US)
	fit3    FIT TD-3000			       (US)
	friq    Freecom IQ cable                       (DE)
        frpw    Freecom Power                          (DE)
        kbic    KingByte KBIC-951A and KBIC-971A       (TW)
	ktti    KT Technology PHd adapter              (SG)
        on20    OnSpec 90c20                           (US)
        on26    OnSpec 90c26                           (US)
	====    ====================================== ====


2. Использование подсистемы pata_parport
=========================================

При конфигурировании ядра Linux вы можете выбрать либо встраивание драйверов
pata_parport в ядро, либо их сборку в виде модулей.

В любом случае вам потребуется выбрать "Parallel port IDE device support"
и как минимум один из протоколов связи параллельного порта.
Если вы не знаете, какой тип адаптера параллельного порта используется в вашем
накопителе, можно начать с проверки имён файлов и любых текстовых файлов на
установочной дискете DOS.  В качестве альтернативы можно посмотреть на маркировку
самого чипа-адаптера.  Этого обычно достаточно для определения нужного
устройства.

Фактически вы можете выбрать все протокольные модули и позволить подсистеме
pata_parport перебрать их все за вас.

Для перечисленных выше «брендовых» продуктов вот протокольные и высокоуровневые
драйверы, которые вам следует использовать:

	================	============	========
	Manufacturer		Model		Protocol
	================	============	========
	MicroSolutions		CD-ROM		bpck
	MicroSolutions		PD drive	bpck
	MicroSolutions		hard-drive	bpck
	MicroSolutions          8000t tape      bpck
	SyQuest			EZ, SparQ	epat
	Imation			Superdisk	epat
	Maxell                  Superdisk       friq
	Avatar			Shark		epat
	FreeCom			CD-ROM		frpw
	Hewlett-Packard		5GB Tape	epat
	Hewlett-Packard		7200e (CD)	epat
	Hewlett-Packard		7200e (CD-R)	epat
	================	============	========

Все parport'ы и все протокольные драйверы зондируются автоматически, если не
используется параметр probe=0. Поэтому для работы накопителя Imation SuperDisk
достаточно просто "modprobe epat".

Создание устройства вручную::

	# echo "port protocol mode unit delay" >/sys/bus/pata_parport/new_device

где:

	======== ================================================
	port	 имя parport (или "auto" для всех parport'ов)
	protocol имя протокола (или "auto" для всех протоколов)
	mode	 номер режима (зависит от протокола) или -1 для зондирования
	unit	 номер юнита (только для backpack, см. ниже)
	delay	 задержка ввода-вывода (см. раздел об устранении неполадок ниже)
	======== ================================================

Если вы используете устройство MicroSolutions backpack, вам также потребуется
знать номер unit ID для каждого накопителя.  Обычно это последние две цифры
серийного номера накопителя (но прочитайте об этом в документации MicroSolutions).

Если вы опускаете параметры с конца, будут использованы значения по умолчанию,
например:

Зондировать все parport'ы всеми протоколами::

	# echo auto >/sys/bus/pata_parport/new_device

Зондировать parport0 с использованием протокола epat и режима 4 (EPP-16)::

	# echo "parport0 epat 4" >/sys/bus/pata_parport/new_device

Зондировать parport0 с использованием всех протоколов::

	# echo "parport0 auto" >/sys/bus/pata_parport/new_device

Зондировать все parport'ы с использованием протокола epat::

	# echo "auto epat" >/sys/bus/pata_parport/new_device

Удаление устройств::

	# echo pata_parport.0 >/sys/bus/pata_parport/delete_device


3. Устранение неполадок
=======================

3.1  По возможности используйте режим EPP
-----------------------------------------

Наиболее распространённые проблемы, о которых сообщают пользователи драйверов
pata_parport, связаны с настройками CMOS параллельного порта.  В настоящее время
ни один из протокольных модулей не поддерживает режим ECP или какие-либо
комбинированные режимы ECP.
Если у вас есть такая возможность, пожалуйста, переведите параллельный порт в
режим EPP с помощью процедуры настройки CMOS.

3.2  Проверьте задержку порта
-----------------------------

Некоторые параллельные порты не могут надёжно передавать данные на полной
скорости.  Чтобы компенсировать ошибки, протокольные модули вводят «задержку
порта» между каждым обращением к портам ввода-вывода.  Каждый протокол
устанавливает для этой задержки значение по умолчанию.  В большинстве случаев
пользователь может переопределить значение по умолчанию и установить его равным
0, что приводит к несколько более высоким скоростям передачи.  В некоторых редких
случаях (особенно со старыми системами 486) задержек по умолчанию недостаточно.
Если вы сталкиваетесь с повреждёнными передачами данных или неожиданными сбоями,
возможно, вам стоит увеличить задержку порта.

3.3  Некоторым накопителям требуется сброс принтера
---------------------------------------------------

По-видимому, на рынке имеется ряд безымянных внешних накопителей, которые не
всегда корректно включаются.  Мы замечали это у некоторых накопителей на основе
адаптеров OnSpec и более старых адаптеров Freecom.  В этих редких случаях адаптер
часто можно переинициализировать, выполнив «сброс принтера» на параллельном
порту.  Поскольку операция сброса потенциально нарушает работу в средах с
несколькими устройствами, драйверы pata_parport не выполняют её автоматически.
Однако вы можете принудительно выполнить сброс принтера, сделав следующее::

	insmod lp reset=1
	rmmod lp

Если у вас один из этих пограничных случаев, вам, вероятно, следует собрать
драйверы pata_parport в виде модулей и организовать выполнение сброса принтера
перед загрузкой драйверов pata_parport.
