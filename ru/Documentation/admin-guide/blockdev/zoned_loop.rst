========================================
Зонированное петлевое блочное устройство
========================================

.. Contents:

	1) Обзор

	3) Удаление зонированного устройства
	4) Пример


1) Обзор
--------

Драйвер зонированного петлевого блочного устройства (zloop) позволяет
пользователю создать зонированное блочное устройство, используя для каждой
зоны один обычный файл в качестве хранилища данных. Этот драйвер не управляет
оборудованием напрямую и использует операции чтения, записи и усечения над
обычными файлами файловой системы для эмуляции зонированного блочного
устройства.

С помощью zloop можно создавать зонированные блочные устройства с настраиваемой
ёмкостью, размером зоны и числом обычных (conventional) зон. Хранилище для
каждой зоны устройства реализуется с помощью обычного файла, максимальный
размер которого равен размеру зоны. Размер файла, обслуживающего обычную зону,
всегда равен размеру зоны. Размер файла, обслуживающего последовательную зону,
указывает объём данных, последовательно записанных в файл, то есть размер
файла напрямую указывает положение указателя записи зоны.

При сбросе последовательной зоны размер её файла усекается до нуля. И наоборот,
при операции завершения зоны (zone finish) файл усекается до размера зоны.
Благодаря этому максимальная ёмкость создаваемого зонированного блочного
устройства zloop может быть настроена так, чтобы превышать пространство
хранения, доступное в базовой файловой системе. Разумеется, при такой
конфигурации запись большего объёма данных, чем доступно в пространстве
хранения базовой файловой системы, приведёт к ошибкам записи.

Драйвер зонированного петлевого блочного устройства реализует полный конечный
автомат переходов состояний зоны. То есть зоны могут быть пустыми, неявно
открытыми, явно открытыми, закрытыми или заполненными. Текущая реализация не
поддерживает каких-либо ограничений на максимальное число открытых и активных
зон.

Для создания и удаления устройств zloop не требуется никаких пользовательских
инструментов.

2) Создание зонированного устройства
------------------------------------

После загрузки модуля zloop (или если zloop встроен в ядро) символьный файл
устройства /dev/zloop-control можно использовать для добавления устройства
zloop. Это делается записью команды "add" непосредственно в устройство
/dev/zloop-control::

	$ modprobe zloop
        $ ls -l /dev/zloop*
        crw-------. 1 root root 10, 123 Jan  6 19:18 /dev/zloop-control

        $ mkdir -p <base directory/<device ID>
        $ echo "add [options]" > /dev/zloop-control

Список доступных параметров команды add можно получить, прочитав устройство
/dev/zloop-control::

	$ cat /dev/zloop-control
        add id=%d,capacity_mb=%u,zone_size_mb=%u,zone_capacity_mb=%u,conv_zones=%u,base_dir=%s,nr_queues=%u,queue_depth=%u,buffered_io
        remove id=%d

Подробнее параметры, которые можно использовать с командой "add",
перечислены ниже.

===================   =========================================================
id                    Номер устройства (значение X в /dev/zloopX).
                      По умолчанию: назначается автоматически.
capacity_mb           Общая ёмкость устройства в МиБ. Всегда округляется вверх
                      до ближайшего большего кратного размеру зоны.
                      По умолчанию: 16384 МиБ (16 ГиБ).
zone_size_mb          Размер зоны устройства в МиБ. По умолчанию: 256 МиБ.
zone_capacity_mb      Ёмкость зоны устройства (всегда должна быть равна размеру
                      зоны или меньше его. По умолчанию: размер зоны.
conv_zones            Общее число обычных (conventional) зон, начиная с
                      сектора 0
                      По умолчанию: 8
base_dir              Путь к базовому каталогу, в котором создаётся каталог,
                      содержащий файлы зон устройства.
                      По умолчанию=/var/local/zloop.
                      Каталог устройства, содержащий файлы зон, всегда
                      именуется по идентификатору устройства. Например, каталог
                      файлов зон по умолчанию для /dev/zloop0 — это
                      /var/local/zloop/0.
nr_queues             Число очередей ввода-вывода зонированного блочного
                      устройства. Это значение всегда ограничивается числом
                      доступных CPU
                      По умолчанию: 1
queue_depth           Максимальная глубина очереди ввода-вывода на одну очередь
                      ввода-вывода.
                      По умолчанию: 64
buffered_io           Выполнять буферизованный ввод-вывод вместо прямого
                      (по умолчанию: false)
zone_append           Включить или отключить нативную поддержку zone append в
                      устройстве zloop.
                      По умолчанию: 1 (включено).
                      Если нативная поддержка zone append отключена, блочный
                      уровень будет эмулировать эту операцию с помощью обычных
                      операций записи.
ordered_zone_append   Включить смягчение переупорядочения zone append в zloop.
                      По умолчанию: отключено.
                      Это полезно для тестирования отображения данных файла
                      (extents) в файловых системах, так как при включении это
                      может значительно сократить число экстентов данных,
                      необходимых для отображения данных файла.
===================   =========================================================

3) Удаление зонированного устройства
------------------------------------

Удаление неиспользуемого зонированного петлевого блочного устройства
выполняется выдачей команды "remove" устройству /dev/zloop-control с указанием
идентификатора удаляемого устройства::

        $ echo "remove id=X" > /dev/zloop-control

Команда remove не имеет никаких параметров.

Удалённое зонированное устройство можно снова добавить без каких-либо изменений
состояния зон устройства: зоны устройства восстанавливаются в их последнее
состояние перед удалением устройства. Повторное добавление зонированного
устройства после его удаления всегда должно выполняться с той же конфигурацией,
что и при первом добавлении устройства. Если обнаружено изменение конфигурации
зон, будет возвращена ошибка и зонированное устройство не будет создано.

Чтобы полностью удалить зонированное устройство, после выполнения операции
remove необходимо удалить базовый каталог устройства, содержащий файлы
хранилища зон устройства.

4) Пример
---------

Следующая последовательность команд создаёт зонированное устройство объёмом
2 ГБ с зонами по 64 МБ и ёмкостью зоны 63 МБ::

        $ modprobe zloop
        $ mkdir -p /var/local/zloop/0
        $ echo "add capacity_mb=2048,zone_size_mb=64,zone_capacity_mb=63" > /dev/zloop-control

Для созданного устройства (/dev/zloop0) файлы хранилища зон создаются в базовом
каталоге по умолчанию (/var/local/zloop)::

        $ ls -l /var/local/zloop/0
        total 0
        -rw-------. 1 root root 67108864 Jan  6 22:23 cnv-000000
        -rw-------. 1 root root 67108864 Jan  6 22:23 cnv-000001
        -rw-------. 1 root root 67108864 Jan  6 22:23 cnv-000002
        -rw-------. 1 root root 67108864 Jan  6 22:23 cnv-000003
        -rw-------. 1 root root 67108864 Jan  6 22:23 cnv-000004
        -rw-------. 1 root root 67108864 Jan  6 22:23 cnv-000005
        -rw-------. 1 root root 67108864 Jan  6 22:23 cnv-000006
        -rw-------. 1 root root 67108864 Jan  6 22:23 cnv-000007
        -rw-------. 1 root root        0 Jan  6 22:23 seq-000008
        -rw-------. 1 root root        0 Jan  6 22:23 seq-000009
        ...

Затем созданное зонированное устройство (/dev/zloop0) можно использовать в
обычном режиме::

        $ lsblk -z
        NAME   ZONED        ZONE-SZ ZONE-NR ZONE-AMAX ZONE-OMAX ZONE-APP ZONE-WGRAN
        zloop0 host-managed     64M      32         0         0       1M         4K
        $ blkzone report /dev/zloop0

          start: 0x000020000, len 0x020000, cap 0x020000, wptr 0x000000 reset:0 non-seq:0, zcond: 0(nw) [type: 1(CONVENTIONAL)]
          start: 0x000040000, len 0x020000, cap 0x020000, wptr 0x000000 reset:0 non-seq:0, zcond: 0(nw) [type: 1(CONVENTIONAL)]
          start: 0x000060000, len 0x020000, cap 0x020000, wptr 0x000000 reset:0 non-seq:0, zcond: 0(nw) [type: 1(CONVENTIONAL)]
          start: 0x000080000, len 0x020000, cap 0x020000, wptr 0x000000 reset:0 non-seq:0, zcond: 0(nw) [type: 1(CONVENTIONAL)]
          start: 0x0000a0000, len 0x020000, cap 0x020000, wptr 0x000000 reset:0 non-seq:0, zcond: 0(nw) [type: 1(CONVENTIONAL)]

Удаление этого устройства выполняется командой::

        $ echo "remove id=0" > /dev/zloop-control

Удалённое устройство можно снова добавить той же командой "add", что и при
первом создании устройства. Чтобы полностью удалить зонированное устройство,
после выполнения команды remove следует также удалить его файлы хранилища::

        $ rm -r /var/local/zloop/0
