==================================================
Список совместимости пространств имён (namespaces)
==================================================

Этот документ содержит информацию о проблемах, с которыми может
столкнуться пользователь при создании задач, существующих в разных
пространствах имён (namespaces).

Ниже приведена сводка. Эта матрица показывает известные проблемы,
возникающие, когда задачи разделяют некоторое пространство имён
(столбцы), но при этом существуют в разных других пространствах
имён (строки):

====	===	===	===	===	====	===
-	UTS	IPC	VFS	PID	User	Net
====	===	===	===	===	====	===
UTS	 X
IPC		 X	 1
VFS			 X
PID		 1	 1	 X
User		 2	 2		 X
Net						 X
====	===	===	===	===	====	===

1. И пространство имён IPC, и пространство имён PID предоставляют
   идентификаторы (ID) для адресации объектов внутри ядра. Например,
   семафор по IPCID или группа процессов по pid.

   В обоих случаях задачи не должны пытаться раскрывать этот ID
   какой-либо другой задаче, существующей в другом пространстве имён,
   через разделяемую файловую систему или через IPC shmem/message.
   Дело в том, что этот ID действителен только внутри того пространства
   имён, в котором он был получен, и в другом пространстве имён может
   ссылаться на какой-то другой объект.

2. Сознательно сделано так, чтобы два равных идентификатора пользователя
   (user ID) в разных пространствах имён пользователей не были равны
   с точки зрения VFS. Иными словами, пользователь 10 в одном
   пространстве имён пользователей не должен иметь те же права доступа
   к файлам, принадлежащим пользователю 10 в другом пространстве имён.

   То же самое верно и для разделяемых пространств имён IPC — два
   пользователя из разных пространств имён пользователей не должны
   получать доступ к одним и тем же объектам IPC, даже имея равные UID.

   Но в настоящее время это не так.
