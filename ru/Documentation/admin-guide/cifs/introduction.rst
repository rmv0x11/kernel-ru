==========
Введение
==========

  Это клиентский VFS-модуль для протокола SMB3 (для сетевых хранилищ NAS),
  а также для более старых диалектов, таких как протокол Common Internet
  File System (CIFS), который был преемником протокола Server Message Block
  (SMB) — нативного механизма совместного доступа к файлам для большинства
  ранних операционных систем для ПК. Новые и улучшенные версии CIFS теперь
  называются SMB2 и SMB3. Использование SMB3 (и более поздних версий, включая
  SMB3.1.1 — самый актуальный диалект) настоятельно предпочтительнее
  использования более старых диалектов, таких как CIFS, по соображениям
  безопасности. Все современные диалекты, включая самый последний, SMB3.1.1,
  поддерживаются VFS-модулем CIFS. Протокол SMB3 реализован и поддерживается
  всеми основными файловыми серверами, такими как Windows (включая Windows
  2019 Server), а также Samba (которая предоставляет отличную серверную
  поддержку CIFS/SMB2/SMB3 и инструменты для Linux и многих других
  операционных систем). Системы Apple также хорошо поддерживают SMB3, как и
  большинство поставщиков сетевых хранилищ (Network Attached Storage), поэтому
  этот клиент сетевой файловой системы может монтироваться к самым разным
  системам. Он также поддерживает монтирование к облаку (например, Microsoft
  Azure), включая необходимые функции безопасности.

  Назначение этого модуля — предоставить наиболее продвинутую функциональность
  сетевой файловой системы для серверов, совместимых с SMB3, включая
  расширенные функции безопасности, отличный распараллеленный
  высокопроизводительный ввод-вывод, лучшую совместимость с POSIX, безопасное
  установление сессий для каждого пользователя, шифрование,
  высокопроизводительное безопасное распределённое кэширование
  (leases/oplocks), необязательную подпись пакетов, большие файлы, поддержку
  Unicode и другие улучшения интернационализации. Поскольку и сервер Samba, и
  этот клиент файловой системы поддерживают расширения CIFS Unix, а
  Linux-клиент также поддерживает расширения SMB3 POSIX, такая комбинация
  может стать разумной альтернативой другим сетевым и кластерным файловым
  системам для файлового обслуживания в некоторых средах Linux—Linux, а не
  только в средах Linux—Windows (или Linux—Mac).

  У этой файловой системы есть утилита монтирования (mount.cifs) и различные
  инструменты пространства пользователя (включая smbinfo и setcifsacl), которые
  можно получить по адресу

      https://git.samba.org/?p=cifs-utils.git

  или

      git://git.samba.org/cifs-utils.git

  mount.cifs следует установить в каталог с остальными вспомогательными
  утилитами монтирования.

  Дополнительную информацию о модуле смотрите на вики-странице проекта по адресу

      https://wiki.samba.org/index.php/LinuxCIFS

  и

      https://wiki.samba.org/index.php/LinuxCIFS_utils
