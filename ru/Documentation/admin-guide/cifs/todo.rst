Частичный список отсутствующей функциональности
================================================

Вклад приветствуется.  Существует множество возможностей внести заметный и важный
вклад в этот модуль.  Ниже приведён частичный список известных проблем и
отсутствующей функциональности:

a) Отсутствующая необязательная функциональность SMB3 (и SMB3.1.1):
   оптимизации производительности multichannel, алгоритмический выбор каналов,
   оптимизации directory leases,
   поддержка более быстрого подписания пакетов (GMAC),
   поддержка сжатия при передаче по сети,
   T10 copy offload, то есть «ODX» (copy chunk и ioctl «Duplicate Extents» —
   в настоящее время единственные два поддерживаемых механизма копирования на стороне сервера)

b) Лучше оптимизированные compounding и обработка ошибок для поддержки разреженных файлов,
   возможно, добавление новых необязательных fsctl-ов SMB3.1.1, чтобы сделать collapse range
   и insert range более атомарными

c) Поддержка SMB3.1.1 поверх QUIC (и, возможно, других протоколов на основе сокетов,
   таких как SCTP)

d) Поддержка квот (требуется небольшое изменение ядра, поскольку иначе вызовы квот
   не доходят до сетевых файловых систем или файловых систем без устройств).

e) Дополнительные сценарии использования могут быть оптимизированы за счёт применения
   «compounding» (например, open/query/close и open/setinfo/close) для сокращения числа
   обращений к серверу и повышения производительности. Различные случаи
   (stat, statfs, create, unlink, mkdir, xattrs) уже были улучшены за счёт
   использования compounding, но можно сделать больше. Кроме того, можно
   значительно сократить избыточные открытия за счёт использования отложенного закрытия (с
   leases для кэширования дескрипторов) и более эффективного использования счётчиков ссылок
   на дескрипторы файлов.

f) Завершить поддержку inotify, чтобы окна списка файлов в kde и gnome
   автоматически обновлялись (частично выполнено Asser). Требуется небольшое изменение
   vfs ядра для поддержки снятия D_NOTIFY с файла.

g) Добавить GUI-инструмент для настройки параметров /proc/fs/cifs и для отображения
   статистики CIFS (начато)

h) Реализовать поддержку категорий security и trusted для xattrs
   (требуется небольшое расширение протокола), чтобы обеспечить лучшую поддержку SELINUX

i) Добавить поддержку tree connect contexts (см. MS-SMB2) — новой возможности протокола SMB3.1.1
   (может быть особенно полезной для виртуализации).

j) Создать механизм отображения UID, чтобы UID сервера могли отображаться на UID
   клиента или на nobody, если отображение отсутствует, в расчёте на отдельное
   монтирование или на отдельный сервер. Также лучшая интеграция с winbind для
   разрешения владельцев SID

k) Добавить инструменты, использующие больше специфичных для smb3 ioctl-ов и возможностей
   (passthrough ioctl/fsctl теперь реализован в cifs.ko, что позволяет
   отправлять различные fsctl-ы SMB3, а также вызовы query info и set info
   напрямую из пространства пользователя). Добавить инструменты, упрощающие установку различных
   не-POSIX-атрибутов метаданных с помощью инструментов (например, расширив то, что было сделано
   в инструменте smb-info).

l) Поддержка зашифрованных файлов (в настоящее время атрибут, показывающий, что файл
   зашифрован на сервере, сообщается, но изменение этого атрибута не
   поддерживается).

m) Улучшенные инструменты сбора статистики (возможно, интеграция с nfsometer?),
   чтобы расширить и упростить использование того, что в настоящее время находится в /proc/fs/cifs/Stats

n) Добавить поддержку ACL на основе claims («DAC»)

o) GUI-помощник монтирования (для упрощения различных параметров конфигурации при монтировании)

p) Расширить поддержку протокола witness, чтобы обеспечить уведомление о перемещении общего
   ресурса и об изменениях сетевого адаптера сервера. В настоящее время клиент Linux поддерживает
   только уведомления протокола witness о перемещении сервера.

q) Позволить mount.cifs более подробно сообщать об ошибках, связанных с диалектом
   или неподдерживаемой функциональностью. Сейчас это было бы проще благодаря
   реализации нового API монтирования.

r) Обновление документации cifs и руководства пользователя.

s) Устранение ошибок, обнаруженных при запуске более широкого набора xfstests в стандартном
   наборе xfstest для файловых систем.

t) Разделить поддержку cifs и smb3 на отдельные модули, чтобы устаревший (и менее
   безопасный) диалект CIFS можно было отключить в средах, которым он не нужен,
   и упростить код.

v) Дополнительное тестирование POSIX Extensions для SMB3.1.1

w) Поддержка расширений Mac SMB3.1.1 для улучшения взаимодействия с серверами Apple

x) Поддержка дополнительных вариантов аутентификации (например, IAKERB, одноранговый
   Kerberos, SCRAM и других, поддерживаемых существующими серверами)

y) Улучшенная трассировка, больше точек трассировки eBPF, лучшие скрипты для анализа
   производительности

Известные ошибки
================

См. https://bugzilla.samba.org — поиск по продукту «CifsVFS» для
получения текущего списка ошибок.  Также проверьте http://bugzilla.kernel.org (Product = File System, Component = CIFS)
и результаты xfstest, например https://wiki.samba.org/index.php/Xfstest-results-smb3

Различное тестирование, которое нужно выполнить
===============================================
1) проверить максимальные длины путей и максимальное число компонентов пути для различных типов
   серверов. Попробовать вложенные символические ссылки (глубиной 8). Возвращать максимальную длину пути в информации stat -f

2) Улучшить поддержку cifs/smb3 в xfstest и при необходимости адаптировать xfstests, чтобы лучше тестировать
   cifs/smb3

3) Дополнительное тестирование и оптимизация производительности с помощью iozone и подобных инструментов —
   есть несколько простых изменений, которые можно сделать для распараллеливания последовательных записей,
   а при отключённом подписании — запрашивать большие размеры чтения (больше
   согласованного размера) и отправлять большие размеры записи современным серверам.

4) Более тщательное тестирование на менее распространённых серверах

5) Продолжить развитие smb3 «buildbot», который в настоящее время выполняет автоматизированное
   xfstest-тестирование на Windows, Samba и Azure — добавить дополнительные тесты и
   позволить buildbot выполнять тесты быстрее. URL
   buildbot: http://smb3-test-rhel-75.southcentralus.cloudapp.azure.com

6) Устранить различные предупреждения coverity (большинство из них не являются ошибками как таковыми, но
   чем больше предупреждений устранено, тем легче обнаруживать реальные
   проблемы, на которые статические анализаторы будут указывать в будущем).
