Parport
+++++++

Код ``parport`` обеспечивает поддержку параллельного порта в Linux. Сюда входит
возможность совместного использования одного порта несколькими драйверами
устройств.

Вы можете передать коду ``parport`` параметры, чтобы переопределить его
автоматическое определение вашего оборудования. Это особенно полезно, если вы
хотите использовать IRQ, поскольку в общем случае их не удаётся успешно
автоматически определить. По умолчанию IRQ не используются, даже если их
**можно** определить. Это связано с тем, что многие люди используют один и тот
же IRQ для своего параллельного порта и звуковой или сетевой карты.

Код ``parport`` разделён на две части: общую (которая занимается совместным
использованием порта) и зависящую от архитектуры (которая занимается
непосредственным использованием порта).


Parport как модули
==================

Если вы загружаете код ``parport`` как модуль, выполните::

	# insmod parport

чтобы загрузить общий код ``parport``. Затем вы должны загрузить
зависящий от архитектуры код, например::

	# insmod parport_pc io=0x3bc,0x378,0x278 irq=none,7,auto

чтобы сообщить коду ``parport``, что вам нужны три порта PC-типа: один по адресу
0x3bc без IRQ, один по адресу 0x378 с использованием IRQ 7 и один по адресу
0x278 с автоматически определяемым IRQ. В настоящее время поддерживается
оборудование PC-типа (``parport_pc``), Sun ``bpp``, Amiga, Atari и MFC3.

Поддержка карт параллельного ввода-вывода PCI обеспечивается кодом
``parport_pc``. Базовые адреса ввода-вывода не следует указывать для
поддерживаемых карт PCI, поскольку они определяются автоматически.


modprobe
--------

Если вы используете modprobe, вам будет полезно добавить строки, подобные
приведённым ниже, в файл конфигурации в каталоге /etc/modprobe.d/::

	alias parport_lowlevel parport_pc
	options parport_pc io=0x378,0x278 irq=7,auto

modprobe загрузит ``parport_pc`` (с опциями ``io=0x378,0x278 irq=7,auto``)
всякий раз, когда загружается драйвер устройства параллельного порта (такой как
``lp``).

Учтите, что это лишь примеры строк! В общем случае вам не нужно указывать
какие-либо опции для ``parport_pc``, чтобы иметь возможность использовать
параллельный порт.


Parport probe [необязательно]
-----------------------------

В ядрах 2.2 был модуль под названием ``parport_probe``, который использовался
для сбора информации об идентификаторе устройства IEEE 1284. Теперь он усовершенствован
и стал частью поддержки IEEE 1284. Когда обнаруживается параллельный порт,
анализируются подключённые к нему устройства, и информация записывается в
журнал примерно так::

	parport0: Printer, BJC-210 (Canon)

Информация автоопределения доступна из файлов в ``/proc/sys/dev/parport/``.


Parport, статически встроенный в ядро
=====================================

Если вы компилируете код ``parport`` в состав ядра, то можете использовать
параметры загрузки ядра для достижения того же эффекта. Добавьте что-то вроде
следующего в командную строку LILO::

	parport=0x3bc parport=0x378,7 parport=0x278,auto,nofifo

Вы можете указать множество выражений ``parport=...``, по одному для каждого
порта, который хотите добавить. Добавление ``parport=0`` в командную строку ядра
полностью отключит поддержку parport. Добавление ``parport=auto`` в командную
строку ядра заставит ``parport`` использовать любые линии IRQ или каналы DMA,
которые он автоматически определит.


Файлы в /proc
=============

Если вы включили файловую систему ``/proc`` в свою конфигурацию ядра, то увидите
новую запись каталога: ``/proc/sys/dev/parport``. В нём будет запись каталога для
каждого параллельного порта, для которого настроен parport. В каждом из этих
каталогов находится набор файлов, описывающих этот параллельный порт.

Дерево каталога ``/proc/sys/dev/parport`` выглядит так::

	parport
	|-- default
	|   |-- spintime
	|   `-- timeslice
	|-- parport0
	|   |-- autoprobe
	|   |-- autoprobe0
	|   |-- autoprobe1
	|   |-- autoprobe2
	|   |-- autoprobe3
	|   |-- devices
	|   |   |-- active
	|   |   `-- lp
	|   |       `-- timeslice
	|   |-- base-addr
	|   |-- irq
	|   |-- dma
	|   |-- modes
	|   `-- spintime
	`-- parport1
	|-- autoprobe
	|-- autoprobe0
	|-- autoprobe1
	|-- autoprobe2
	|-- autoprobe3
	|-- devices
	|   |-- active
	|   `-- ppa
	|       `-- timeslice
	|-- base-addr
	|-- irq
	|-- dma
	|-- modes
	`-- spintime

.. tabularcolumns:: |p{4.0cm}|p{13.5cm}|

=======================	=======================================================
Файл			Содержимое
=======================	=======================================================
``devices/active``	Список драйверов устройств, использующих этот порт. Рядом
			с именем устройства, которое в данный момент использует
			порт, появляется знак «+» (он может не появляться ни у
			одного из них). Строка "none" означает, что этот порт не
			используется ни одним драйвером устройства.

``base-addr``		Базовый адрес параллельного порта или адреса, если у порта
			их несколько, и в этом случае они разделяются
			табуляциями. Для некоторых портов эти значения могут не
			иметь какого-либо разумного смысла.

``irq``			IRQ параллельного порта или -1, если ни один не используется.

``dma``			Канал DMA параллельного порта или -1, если ни один не
			используется.

``modes``		Аппаратные режимы параллельного порта, разделённые
			запятыми, означающие:

			- PCSPP
				Доступны регистры SPP PC-типа.

			- TRISTATE
				Порт является двунаправленным.

			- COMPAT
				Аппаратное ускорение для принтеров
				доступно и будет использоваться.

			- EPP
				Аппаратное ускорение для протокола EPP
				доступно и будет использоваться.

			- ECP
				Аппаратное ускорение для протокола ECP
				доступно и будет использоваться.

			- DMA
				DMA доступен и будет использоваться.

			Учтите, что текущая реализация будет использовать
			преимущества режимов COMPAT и ECP только при наличии
			линии IRQ.

``autoprobe``		Любая информация об идентификаторе устройства IEEE-1284,
			которая была получена от устройства (не относящегося к
			IEEE 1284.3).

``autoprobe[0-3]``	Информация об идентификаторе устройства IEEE 1284,
			полученная от устройств в цепочке (daisy-chain),
			соответствующих IEEE 1284.3.

``spintime``		Количество микросекунд для активного ожидания в цикле
			(busy-loop) отклика периферийного устройства. Возможно,
			вы обнаружите, что регулировка этого значения улучшает
			производительность в зависимости от ваших периферийных
			устройств. Это настройка уровня порта, т. е. она
			применяется ко всем устройствам конкретного порта.

``timeslice``		Количество миллисекунд, в течение которых драйверу
			устройства разрешено удерживать порт за собой. Это
			рекомендательное значение, и драйвер может его
			проигнорировать, если это необходимо.

``default/*``		Значения по умолчанию для spintime и timeslice. Когда
			регистрируется новый порт, он принимает spintime по
			умолчанию. Когда регистрируется новое устройство, оно
			принимает timeslice по умолчанию.
=======================	=======================================================

Драйверы устройств
==================

После инициализации кода parport вы можете привязать драйверы устройств к
конкретным портам. Обычно это происходит автоматически; если загружен драйвер
lp, он создаст одно устройство lp для каждого найденного порта. Однако вы можете
переопределить это с помощью параметров либо при загрузке драйвера lp::

	# insmod lp parport=0,2

либо в командной строке LILO::

	lp=parport0 lp=parport2

Оба приведённых выше примера сообщат lp, что вы хотите, чтобы ``/dev/lp0`` был
первым параллельным портом, а /dev/lp1 — **третьим** параллельным портом, при
этом со вторым портом (parport1) не связано ни одно устройство lp. Учтите, что
это отличается от того, как работали более старые ядра; раньше существовала
статическая связь между адресом порта ввода-вывода и именем устройства, поэтому
``/dev/lp0`` всегда был портом по адресу 0x3bc. Это больше не так — если у вас
только один порт, он по умолчанию станет ``/dev/lp0`` независимо от базового
адреса.

Кроме того:

 * Если вы выбрали поддержку IEEE 1284 на этапе компиляции, вы можете указать
   ``lp=auto`` в командной строке ядра, и lp создаст устройства только для тех
   портов, к которым, по-видимому, подключены принтеры.

 * Если вы передаёте PLIP параметр ``timid`` — либо с помощью ``plip=timid`` в
   командной строке, либо с помощью ``insmod plip timid=1`` при использовании
   модулей — он будет избегать любых портов, которые, по-видимому, используются
   другими устройствами.

 * Автоопределение IRQ на данный момент работает только для нескольких типов
   портов.

Сообщение о проблемах с принтером в parport
===========================================

Если у вас возникли проблемы с печатью, пожалуйста, выполните эти шаги, чтобы
попытаться сузить область, в которой находится проблема.

При сообщении о проблемах с parport вам действительно нужно привести все
сообщения, которые выдаёт ``parport_pc`` при инициализации. Существует несколько
путей выполнения кода:

- опрос (polling)
- управление по прерываниям, протокол в программном обеспечении
- управление по прерываниям, протокол в аппаратном обеспечении с использованием PIO
- управление по прерываниям, протокол в аппаратном обеспечении с использованием DMA

Сообщения ядра, которые записывает в журнал ``parport_pc``, указывают на то,
какой путь выполнения кода используется. (На самом деле они могли бы быть
гораздо лучше..)

Для обычного протокола принтера то, включены режимы IEEE 1284 или нет, не должно
иметь значения.

Чтобы отключить пути выполнения кода с «протоколом в аппаратном обеспечении»,
отключите ``CONFIG_PARPORT_PC_FIFO``. Учтите, что когда они включены, они не
обязательно **используются**; это зависит от того, доступно ли оборудование,
включено ли оно в BIOS и обнаружено ли драйвером.

Итак, для начала отключите ``CONFIG_PARPORT_PC_FIFO`` и загрузите ``parport_pc``
с ``irq=none``. Посмотрите, работает ли тогда печать. Она действительно должна
работать, потому что это самый простой путь выполнения кода.

Если это работает нормально, попробуйте с ``io=0x378 irq=7`` (скорректируйте под
своё оборудование), чтобы заставить его использовать управляемый прерываниями
протокол в программном обеспечении.

Если **это** работает нормально, значит, один из аппаратных режимов работает
неправильно. Включите ``CONFIG_FIFO`` (нет, это не опция модуля, и да, это
должно так быть), установите в BIOS режим ECP для порта, запомните канал DMA и
попробуйте с::

    io=0x378 irq=7 dma=none (для PIO)
    io=0x378 irq=7 dma=3 (для DMA)
