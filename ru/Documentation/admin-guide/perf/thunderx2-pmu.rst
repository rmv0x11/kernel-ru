==========================================================================
Блок мониторинга производительности (PMU UNCORE) SoC Cavium ThunderX2
==========================================================================

PMU SoC ThunderX2 состоит из независимых, общесистемных PMU для каждого
сокета, таких как кэш третьего уровня (L3C), контроллер памяти DDR4 (DMC) и
Cavium Coherent Processor Interconnect (CCPI2).

DMC имеет 8 чередующихся (interleaved) каналов, а L3C — 16 чередующихся
плиток (tiles). События подсчитываются для канала по умолчанию (т. е. канала 0)
и пропорционально пересчитываются на общее число каналов/плиток.

DMC и L3C поддерживают до 4 счётчиков, а CCPI2 — до 8 счётчиков. Счётчики
независимо программируются на разные события и могут запускаться и
останавливаться по отдельности. Ни один из счётчиков не поддерживает прерывание
по переполнению. Счётчики DMC и L3C 32-битные и считываются каждые 2 секунды.
Счётчики CCPI2 64-битные и предполагается, что при нормальной работе они не
переполняются.

Драйвер PMU UNCORE (perf):

Драйвер thunderx2_pmu регистрирует perf PMU для каждого сокета для устройств
DMC и L3C. Каждый PMU может использоваться для одновременного подсчёта до 4
(DMC/L3C) или до 8 (CCPI2) событий. PMU предоставляют описание доступных
событий и параметров конфигурации в sysfs, см.
/sys/bus/event_source/devices/uncore_<l3c_S/dmc_S/ccpi2_S/>; S — это идентификатор сокета.

Драйвер не поддерживает выборку (sampling), поэтому "perf record" не будет
работать. Сессии perf для отдельных задач также не поддерживаются.

Примеры::

  # perf stat -a -e uncore_dmc_0/cnt_cycles/ sleep 1

  # perf stat -a -e \
  uncore_dmc_0/cnt_cycles/,\
  uncore_dmc_0/data_transfers/,\
  uncore_dmc_0/read_txns/,\
  uncore_dmc_0/write_txns/ sleep 1

  # perf stat -a -e \
  uncore_l3c_0/read_request/,\
  uncore_l3c_0/read_hit/,\
  uncore_l3c_0/inv_request/,\
  uncore_l3c_0/inv_hit/ sleep 1
