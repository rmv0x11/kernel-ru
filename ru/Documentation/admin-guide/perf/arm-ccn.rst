==========================
ARM Cache Coherent Network
==========================

CCN-504 — это кольцевая шина-интерконнект, состоящая из 11 перекрёстков
(crosspoints, XP), причём каждый перекрёсток поддерживает до двух портов
устройств, поэтому узлы (устройства) 0 и 1 подключены к перекрёстку 0,
узлы 2 и 3 — к перекрёстку 1 и т.д.

Драйвер PMU (perf)
------------------

Драйвер CCN регистрирует драйвер perf PMU, который предоставляет
описание доступных событий и параметров конфигурации в sysfs, см.
/sys/bus/event_source/devices/ccn*.

Каталог "format" описывает формат полей config, config1 и config2
структуры perf_event_attr. Каталог "events" предоставляет шаблоны
конфигурации для всех документированных событий, которые можно
использовать с утилитой perf. Например, "xp_valid_flit" эквивалентно
"type=0x8,event=0x4". Остальные параметры должны быть указаны явно.

Для событий, исходящих от устройства, "node" задаёт его индекс.

События PMU перекрёстка (crosspoint) требуют "xp" (индекс), "bus" (номер
шины) и "vc" (идентификатор виртуального канала).

События перекрёстка на основе точек наблюдения (watchpoint) (специальное
значение "event" 0xfe) требуют "xp" и "vc" как указано выше, плюс "port"
(индекс порта устройства), "dir" (направление передачи/приёма), значения
компаратора ("cmp_l" и "cmp_h") и "mask", являющийся индексом маски
компаратора.

Маски задаются отдельно от описания события (из-за ограниченного числа
значений config) в каталоге "cmp_mask", причём первые 8 настраиваются
пользователем, а ещё 4 жёстко заданы для наиболее частых случаев
использования.

Счётчик циклов описывается значением "type" 0xff и не требует никаких
других настроек.

Драйвер также предоставляет атрибут sysfs "cpumask", который содержит
единственный идентификатор CPU — процессора, который будет использоваться
для обработки всех событий CCN PMU. Рекомендуется, чтобы инструменты
пространства пользователя запрашивали события на этом процессоре (в
противном случае значение perf_event->cpu всё равно будет перезаписано).
В случае перевода этого процессора в offline события мигрируют на другой,
а атрибут обновляется.

Пример использования утилиты perf::

  / # perf list | grep ccn
    ccn/cycles/                                        [Kernel PMU event]
  <...>
    ccn/xp_valid_flit,xp=?,port=?,vc=?,dir=?/          [Kernel PMU event]
  <...>

  / # perf stat -a -e ccn/cycles/,ccn/xp_valid_flit,xp=1,port=0,vc=1,dir=1/ \
                                                                         sleep 1

Драйвер не поддерживает сэмплирование, поэтому "perf record" работать не
будет. Сеансы perf для отдельной задачи (без "-a") не поддерживаются.
