===============================================================
Модуль мониторинга производительности (PMU) DDR Freescale i.MX8
===============================================================

Внутри контроллера DRAM нет счётчиков производительности, поэтому сигналы
производительности выводятся на край контроллера, где реализован набор из
4 x 32-битных счётчиков. Это управляется режимами CSV, запрограммированными в
регистре управления счётчиками, что приводит к генерации большого числа сигналов
PERF.

Выбор значения для каждого счётчика выполняется через регистры config. Для каждого
счётчика есть один регистр. Счётчик 0 является особенным в том смысле, что он всегда
считает «время», и при его переполнении происходит блокировка его самого и остальных
счётчиков, а также генерируется прерывание. Если переполняется любой другой счётчик,
он продолжает счёт, и прерывание не генерируется.

Каталог "format" описывает формат полей config (идентификатор события) и config1/2
(настройка AXI-фильтра) структуры perf_event_attr, см. /sys/bus/event_source/
devices/imx8_ddr0/format/. Каталог "events" описывает типы событий, поддерживаемые
аппаратно, которые можно использовать с инструментом perf, см. /sys/bus/event_source/
devices/imx8_ddr0/events/. Каталог "caps" описывает возможности фильтрации, реализованные
в DDR PMU, см. /sys/bus/events_source/devices/imx8_ddr0/caps/.

    .. code-block:: bash

        perf stat -a -e imx8_ddr0/cycles/ cmd
        perf stat -a -e imx8_ddr0/read/,imx8_ddr0/write/ cmd

AXI-фильтрация используется только режимами CSV 0x41 (axid-read) и 0x42 (axid-write)
для подсчёта операций чтения или записи, совпадающих с настройкой фильтра. Настройка
фильтра различается в разных реализациях контроллера DRAM, что распознаётся по quirk'ам
в драйвере. Также можно вывести информацию из userspace: каталог "caps" показывает
тип AXI-фильтра (filter, enhanced_filter и super_filter). Значение 0 означает
неподдерживаемый, а значение 1 — поддерживаемый.

* С quirk DDR_CAP_AXI_ID_FILTER (filter: 1, enhanced_filter: 0, super_filter: 0).
  Фильтр определяется двумя частями конфигурации:
  --AXI_ID задаёт значение для сопоставления AxID.
  --AXI_MASKING задаёт, какие биты AxID значимы для сопоставления.

      - 0: соответствующий бит маскируется.
      - 1: соответствующий бит не маскируется, т. е. используется для сопоставления.

  AXI_ID и AXI_MASKING отображаются на регистр DPCR1 в счётчике производительности.
  Когда немаскированные биты совпадают с соответствующими битами AXI_ID, счётчик
  инкрементируется. Счётчик perf инкрементируется, если::

        AxID && AXI_MASKING == AXI_ID && AXI_MASKING

  Этот фильтр не поддерживает фильтрацию по разным AXI ID для событий axid-read и
  axid-write одновременно, поскольку фильтр является общим для счётчиков.

  .. code-block:: bash

      perf stat -a -e imx8_ddr0/axid-read,axi_mask=0xMMMM,axi_id=0xDDDD/ cmd
      perf stat -a -e imx8_ddr0/axid-write,axi_mask=0xMMMM,axi_id=0xDDDD/ cmd

  .. note::

      axi_mask инвертируется в userspace (т. е. установленные биты — это биты для
      маскирования), и он будет автоматически реверсирован в драйвере. Таким образом,
      пользователь может просто указать axi_id для мониторинга конкретного id вместо
      того, чтобы указывать axi_mask.

  .. code-block:: bash

        perf stat -a -e imx8_ddr0/axid-read,axi_id=0x12/ cmd, при этом будет вестись мониторинг ARID=0x12

* С quirk DDR_CAP_AXI_ID_FILTER_ENHANCED (filter: 1, enhanced_filter: 1, super_filter: 0).
  Это расширение quirk'а DDR_CAP_AXI_ID_FILTER, которое позволяет подсчитывать
  число байтов (в отличие от числа пакетов передачи) от операций чтения и записи
  DDR одновременно с другим набором счётчиков данных.

* С quirk DDR_CAP_AXI_ID_PORT_CHANNEL_FILTER (filter: 0, enhanced_filter: 0, super_filter: 1).
  В предыдущем AXI-фильтре есть ограничение: он не может фильтровать разные ID
  одновременно, поскольку фильтр является общим для счётчиков. Этот quirk является
  расширением AXI ID-фильтра. Одно из улучшений состоит в том, что счётчики 1-3 имеют
  собственный фильтр, что означает поддержку одновременной фильтрации различных ID.
  Другое улучшение состоит в том, что счётчики 1-3 поддерживают выбор AXI PORT и
  CHANNEL. Поддерживается выбор адресного канала или канала данных.

  Фильтр определяется 2 регистрами конфигурации на каждый счётчик 1-3.
  --Регистр Counter N MASK COMP — включает AXI_ID и AXI_MASKING.
  --Регистр Counter N MUX CNTL — включает AXI CHANNEL и AXI PORT.

      - 0: адресный канал
      - 1: канал данных

  В подсистеме DDR в PMU существует только один порт port0, поэтому axi_port
  зарезервирован и должен быть равен 0.

  .. code-block:: bash

      perf stat -a -e imx8_ddr0/axid-read,axi_mask=0xMMMM,axi_id=0xDDDD,axi_channel=0xH/ cmd
      perf stat -a -e imx8_ddr0/axid-write,axi_mask=0xMMMM,axi_id=0xDDDD,axi_channel=0xH/ cmd

  .. note::

      axi_channel инвертируется в userspace и будет автоматически реверсирован в
      драйвере. Таким образом, пользователям не нужно указывать axi_channel, если они
      хотят вести мониторинг канала данных от операций передачи DDR, поскольку канал
      данных более значим.
