======================================
HNS3 Performance Monitoring Unit (PMU)
======================================

HNS3 (HiSilicon network system 3) Performance Monitoring Unit (PMU) — это
оконечное устройство (End Point), собирающее статистику производительности
сетевой карты (NIC) в SoC HiSilicon. На Hip09 каждый SICL (Super I/O cluster)
имеет одно устройство PMU.

HNS3 PMU поддерживает сбор статистики производительности, такой как пропускная
способность, задержка (latency), частота пакетов и частота прерываний.

Каждый HNS3 PMU поддерживает 8 аппаратных событий.

Драйвер HNS3 PMU
================

Драйвер HNS3 PMU регистрирует perf PMU с именем, соответствующим его sicl id.::

  /sys/bus/event_source/devices/hns3_pmu_sicl_<sicl_id>

Драйвер PMU предоставляет в sysfs описание доступных событий, режимов фильтрации,
формата, идентификатора и cpumask.

Каталог "events" описывает код события для всех поддерживаемых событий,
отображаемых в perf list.

Каталог "filtermode" описывает поддерживаемые режимы фильтрации для каждого
события.

Каталог "format" описывает все форматы полей config (события) и config1 (опции
фильтрации) структуры perf_event_attr.

Файл "identifier" показывает версию аппаратного устройства PMU.

Файлы "bdf_min" и "bdf_max" показывают поддерживаемый диапазон bdf для каждого
устройства pmu.

Файл "hw_clk_freq" показывает частоту аппаратного тактового сигнала каждого
устройства pmu.

Пример использования для проверки кода события и кода подсобытия::

  $# cat /sys/bus/event_source/devices/hns3_pmu_sicl_0/events/dly_tx_normal_to_mac_time
  config=0x00204
  $# cat /sys/bus/event_source/devices/hns3_pmu_sicl_0/events/dly_tx_normal_to_mac_packet_num
  config=0x10204

Каждая статистика производительности имеет пару событий для получения двух
значений, чтобы вычислить реальные данные производительности в пространстве
пользователя (userspace).

Биты 0~15 поля config (здесь 0x0204) — это истинный код аппаратного события.
Если у двух событий совпадают значения битов 0~15 поля config, это означает,
что они образуют пару событий. А бит 16 поля config указывает, что считывается
counter 0 или counter 1 аппаратного события.

После получения двух значений пары событий в пространстве пользователя формула
вычисления реальных данных производительности такова:::

  counter 0 / counter 1

Пример использования для проверки поддерживаемого режима фильтрации::

  $# cat /sys/bus/event_source/devices/hns3_pmu_sicl_0/filtermode/bw_ssu_rpu_byte_num
  filter mode supported: global/port/port-tc/func/func-queue/

Пример использования perf::

  $# perf list
  hns3_pmu_sicl_0/bw_ssu_rpu_byte_num/ [kernel PMU event]
  hns3_pmu_sicl_0/bw_ssu_rpu_time/     [kernel PMU event]
  ------------------------------------------

  $# perf stat -g -e hns3_pmu_sicl_0/bw_ssu_rpu_byte_num,global=1/ -e hns3_pmu_sicl_0/bw_ssu_rpu_time,global=1/ -I 1000
  or
  $# perf stat -g -e hns3_pmu_sicl_0/config=0x00002,global=1/ -e hns3_pmu_sicl_0/config=0x10002,global=1/ -I 1000


Режимы фильтрации
-----------------

1. режим global
PMU собирает статистику производительности для всех функций HNS3 PCIe в IO DIE.
Установка опции фильтрации "global" в 1 включит этот режим.
Пример использования perf::

  $# perf stat -a -e hns3_pmu_sicl_0/config=0x1020F,global=1/ -I 1000

2. режим port
PMU собирает статистику производительности одного физического порта целиком.
Идентификатор порта совпадает с mac id. В этом режиме опция фильтрации "tc"
должна быть установлена в 0xF, здесь tc обозначает класс трафика (traffic class).

Пример использования perf::

  $# perf stat -a -e hns3_pmu_sicl_0/config=0x1020F,port=0,tc=0xF/ -I 1000

3. режим port-tc
PMU собирает статистику производительности одного tc физического порта.
Идентификатор порта совпадает с mac id. В этом режиме опция фильтрации "tc"
должна быть установлена в 0 ~ 7.
Пример использования perf::

  $# perf stat -a -e hns3_pmu_sicl_0/config=0x1020F,port=0,tc=0/ -I 1000

4. режим func
PMU собирает статистику производительности одного PF/VF. Идентификатор функции —
это BDF PF/VF, формула его преобразования::

  func = (bus << 8) + (device << 3) + (function)

например:
  BDF         func
  35:00.0    0x3500
  35:00.1    0x3501
  35:01.0    0x3508

В этом режиме опция фильтрации "queue" должна быть установлена в 0xFFFF.
Пример использования perf::

  $# perf stat -a -e hns3_pmu_sicl_0/config=0x1020F,bdf=0x3500,queue=0xFFFF/ -I 1000

5. режим func-queue
PMU собирает статистику производительности одной очереди PF/VF. Идентификатор
функции — это BDF PF/VF, опция фильтрации "queue" должна быть установлена в
точный идентификатор очереди функции.
Пример использования perf::

  $# perf stat -a -e hns3_pmu_sicl_0/config=0x1020F,bdf=0x3500,queue=0/ -I 1000

6. режим func-intr
PMU собирает статистику производительности одного прерывания PF/VF. Идентификатор
функции — это BDF PF/VF, опция фильтрации "intr" должна быть установлена в точный
идентификатор прерывания функции.
Пример использования perf::

  $# perf stat -a -e hns3_pmu_sicl_0/config=0x00301,bdf=0x3500,intr=0/ -I 1000
