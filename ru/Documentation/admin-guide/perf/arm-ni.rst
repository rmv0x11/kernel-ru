==================================================
PMU интерконнекта Arm Network-on Chip Interconnect
==================================================

NI-700 и сходные устройства реализуют отдельный PMU для каждого тактового домена
(clock domain) внутри интерконнекта. Соответственно, драйвер предоставляет
несколько устройств PMU с именами arm_ni_<x>_cd_<y>, где <x> — это (произвольный)
идентификатор экземпляра, а <y> — идентификатор тактового домена в рамках
данного экземпляра. Если в системе присутствует несколько экземпляров NI,
устройства PMU можно сопоставить с соответствующим аппаратным экземпляром через
иерархию родства в sysfs.

Каждый PMU предоставляет базовые псевдонимы (alias) событий для типов
интерфейсов, присутствующих в его тактовом домене. Их необходимо уточнять
параметрами «eventid» и «nodeid», чтобы задать код подсчитываемого события и
интерфейс, на котором его следует подсчитывать (в соответствии с настроенным
аппаратным ID, отражённым в регистре xxNI_NODE_INFO). Исключением является
псевдоним «cycles» для счётчика тактов PMU, который кодируется типом узла PMU и
не требует дальнейшего уточнения.
