==========================================================
Модуль мониторинга производительности Uncore (PMU) Fujitsu
==========================================================

Этот драйвер поддерживает Uncore MAC PMU и Uncore PCI PMU, имеющиеся
в чипах Fujitsu.
Каждый MAC PMU на этих чипах представлен как uncore perf PMU с именем устройства
mac_iod<iod>_mac<mac>_ch<ch>.
А каждый PCI PMU на этих чипах представлен как uncore perf PMU с именем устройства
pci_iod<iod>_pci<pci>.

Драйвер предоставляет описание доступных событий и параметров конфигурации
в sysfs, см. /sys/bus/event_sources/devices/mac_iod<iod>_mac<mac>_ch<ch>/
и /sys/bus/event_sources/devices/pci_iod<iod>_pci<pci>/.
Этот драйвер экспортирует:

- форматы, используемые пространством пользователя perf и другими инструментами для конфигурации событий
- события, используемые пространством пользователя perf и другими инструментами для символьного
  создания событий, например::

    perf stat -a -e mac_iod0_mac0_ch0/event=0x21/ ls
    perf stat -a -e pci_iod0_pci0/event=0x24/ ls

- cpumask, используемую пространством пользователя perf и другими инструментами, чтобы знать, на каких CPU
  открывать события

Этот драйвер поддерживает следующие события для MAC:

- cycles
  Это событие подсчитывает циклы MAC на частоте MAC.
- read-count
  Это событие подсчитывает число запросов на чтение к MAC.
- read-count-request
  Это событие подсчитывает число запросов на чтение, включая повторные, к MAC.
- read-count-return
  Это событие подсчитывает число ответов на запросы на чтение к MAC.
- read-count-request-pftgt
  Это событие подсчитывает число запросов на чтение, включая повторные, с флагом
  PFTGT.
- read-count-request-normal
  Это событие подсчитывает число запросов на чтение, включая повторные, без флага
  PFTGT.
- read-count-return-pftgt-hit
  Это событие подсчитывает число ответов на запросы на чтение, попавшие в
  буфер PFTGT.
- read-count-return-pftgt-miss
  Это событие подсчитывает число ответов на запросы на чтение, не попавшие в
  буфер PFTGT.
- read-wait
  Это событие подсчитывает невыполненные запросы на чтение, выданные контроллером памяти DDR
  за цикл.
- write-count
  Это событие подсчитывает число запросов на запись к MAC (включая нулевую запись,
  полную запись, частичную запись, отмену записи).
- write-count-write
  Это событие подсчитывает число запросов на полную запись к MAC (не включая
  нулевую запись).
- write-count-pwrite
  Это событие подсчитывает число запросов на частичную запись к MAC.
- memory-read-count
  Это событие подсчитывает число запросов на чтение от MAC к памяти.
- memory-write-count
  Это событие подсчитывает число запросов на полную запись от MAC к памяти.
- memory-pwrite-count
  Это событие подсчитывает число запросов на частичную запись от MAC к памяти.
- ea-mac
  Это событие подсчитывает энергопотребление MAC.
- ea-memory
  Это событие подсчитывает энергопотребление памяти.
- ea-memory-mac-write
  Это событие подсчитывает число запросов на запись от MAC к памяти.
- ea-ha
  Это событие подсчитывает энергопотребление HA.

  'ea' — это сокращение от 'Energy Analyzer'.

Примеры использования с perf::

  perf stat -e mac_iod0_mac0_ch0/ea-mac/ ls

А также этот драйвер поддерживает следующие события для PCI:

- pci-port0-cycles
  Это событие подсчитывает циклы PCI на частоте PCI в port0.
- pci-port0-read-count
  Это событие подсчитывает транзакции чтения для передачи данных в port0.
- pci-port0-read-count-bus
  Это событие подсчитывает транзакции чтения для использования шины в port0.
- pci-port0-write-count
  Это событие подсчитывает транзакции записи для передачи данных в port0.
- pci-port0-write-count-bus
  Это событие подсчитывает транзакции записи для использования шины в port0.
- pci-port1-cycles
  Это событие подсчитывает циклы PCI на частоте PCI в port1.
- pci-port1-read-count
  Это событие подсчитывает транзакции чтения для передачи данных в port1.
- pci-port1-read-count-bus
  Это событие подсчитывает транзакции чтения для использования шины в port1.
- pci-port1-write-count
  Это событие подсчитывает транзакции записи для передачи данных в port1.
- pci-port1-write-count-bus
  Это событие подсчитывает транзакции записи для использования шины в port1.
- ea-pci
  Это событие подсчитывает энергопотребление PCI.

  'ea' — это сокращение от 'Energy Analyzer'.

Примеры использования с perf::

  perf stat -e pci_iod0_pci0/ea-pci/ ls

Поскольку это uncore PMU, драйвер не поддерживает сэмплирование, поэтому
"perf record" работать не будет. Сеансы perf для отдельных задач не поддерживаются.
