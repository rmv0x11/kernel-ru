=============================================================================
Uncore-блок мониторинга производительности (PMU) SoC T-Head компании Alibaba
=============================================================================

Yitian 710, разработанный по заказу подразделением Alibaba Group по созданию
чипов T-Head, реализует uncore PMU для отладки производительности и
функциональной отладки с целью облегчить обслуживание системы.

Драйвер PMU для DDR Sub-System Driveway (DRW)
=============================================

Yitian 710 использует восемь каналов DDR5/4, по четыре на каждом кристалле.
Каждый канал DDR5 независим от остальных при обслуживании запросов к системной
памяти. При этом один канал DDR5 разделён на два независимых подканала. DDR
Sub-System Driveway реализует отдельные PMU для каждого подканала, чтобы
отслеживать различные метрики производительности.

В perf устройства Driveway PMU именуются как ali_drw_<sys_base_addr>.
Например, ali_drw_21000 и ali_drw_21080 — это два устройства PMU для двух
подканалов одного и того же канала на кристалле 0. А устройство PMU кристалла 1
имеет префикс ali_drw_400XXXXX, например ali_drw_40021000.

Каждый подканал в общей сложности имеет 36 счётчиков PMU, которые
классифицируются на четыре группы:

- Group 0: PMU Cycle Counter. В этой группе есть одна пара счётчиков
  pmu_cycle_cnt_low и pmu_cycle_cnt_high, которая используется как счётчик
  тактов на основе тактовой частоты ядра DDRC.

- Group 1: PMU Bandwidth Counters. В этой группе 8 счётчиков, которые
  используются для подсчёта общего числа обращений либо к восьми группам банков
  в выбранном ранге, либо к четырём рангам по отдельности в первых 4 счётчиках.
  Базовая единица передачи — 64 байта.

- Group 2: PMU Retry Counters. В этой группе 10 счётчиков, предназначенных для
  подсчёта общего числа повторов для каждого типа неисправимой ошибки.

- Group 3: PMU Common Counters. В этой группе 16 счётчиков, которые
  используются для подсчёта общих событий.

На данный момент драйвер Driveway PMU использует только счётчики групп 0 и 3.

DDR Controller (DDRCTL) и DDR PHY вместе образуют законченное решение для
подключения прикладной шины SoC к устройствам памяти DDR. DDRCTL принимает
транзакции через интерфейс Host Interface (HIF), который определён компанией
Synopsys. Эти транзакции ставятся во внутреннюю очередь и планируются для
доступа с соблюдением требований к таймингам протокола SDRAM, приоритетов
транзакций и зависимостей между транзакциями. В свою очередь, DDRCTL выдаёт
команды по интерфейсу DDR PHY Interface (DFI) модулю PHY, который отправляет и
захватывает данные в SDRAM и из неё. PMU driveway имеют аппаратную логику для
сбора статистики и сигналов журналирования производительности на HIF, DFI и т. д.

Подсчитывая команды READ, WRITE и RMW, отправленные в DDRC через интерфейс HIF,
мы можем вычислить пропускную способность. Пример использования для подсчёта
пропускной способности данных памяти::

  perf stat \
    -e ali_drw_21000/hif_wr/ \
    -e ali_drw_21000/hif_rd/ \
    -e ali_drw_21000/hif_rmw/ \
    -e ali_drw_21000/cycle/ \
    -e ali_drw_21080/hif_wr/ \
    -e ali_drw_21080/hif_rd/ \
    -e ali_drw_21080/hif_rmw/ \
    -e ali_drw_21080/cycle/ \
    -e ali_drw_23000/hif_wr/ \
    -e ali_drw_23000/hif_rd/ \
    -e ali_drw_23000/hif_rmw/ \
    -e ali_drw_23000/cycle/ \
    -e ali_drw_23080/hif_wr/ \
    -e ali_drw_23080/hif_rd/ \
    -e ali_drw_23080/hif_rmw/ \
    -e ali_drw_23080/cycle/ \
    -e ali_drw_25000/hif_wr/ \
    -e ali_drw_25000/hif_rd/ \
    -e ali_drw_25000/hif_rmw/ \
    -e ali_drw_25000/cycle/ \
    -e ali_drw_25080/hif_wr/ \
    -e ali_drw_25080/hif_rd/ \
    -e ali_drw_25080/hif_rmw/ \
    -e ali_drw_25080/cycle/ \
    -e ali_drw_27000/hif_wr/ \
    -e ali_drw_27000/hif_rd/ \
    -e ali_drw_27000/hif_rmw/ \
    -e ali_drw_27000/cycle/ \
    -e ali_drw_27080/hif_wr/ \
    -e ali_drw_27080/hif_rd/ \
    -e ali_drw_27080/hif_rmw/ \
    -e ali_drw_27080/cycle/ -- sleep 10

Пример использования для подсчёта суммарной пропускной способности чтения/записи
памяти через метрику::

  perf stat -M ddr_read_bandwidth.all -- sleep 10
  perf stat -M ddr_write_bandwidth.all -- sleep 10

Среднюю пропускную способность DRAM можно вычислить следующим образом:

- Read Bandwidth =  perf_hif_rd * DDRC_WIDTH * DDRC_Freq / DDRC_Cycle
- Write Bandwidth = (perf_hif_wr + perf_hif_rmw) * DDRC_WIDTH * DDRC_Freq / DDRC_Cycle

Здесь DDRC_WIDTH = 64 байта.

Текущий драйвер не поддерживает сэмплирование. Поэтому "perf record" не
поддерживается.  Также не поддерживается привязка к задаче, так как все события
являются uncore.
