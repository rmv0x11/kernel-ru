======================================================================
Synopsys DesignWare Cores (DWC) PCIe Performance Monitoring Unit (PMU)
======================================================================

DesignWare Cores (DWC) PCIe PMU
===============================

PMU представляет собой блок регистров конфигурационного пространства PCIe,
предоставляемый каждым корневым портом (Root Port) PCIe в составе расширенной
возможности Vendor-Specific Extended Capability с именем RAS D.E.S (Debug, Error
injection, and Statistics).

Как следует из названия, возможность RAS DES поддерживает отладку на уровне
системы, инъекцию ошибок AER и сбор статистики. Для облегчения сбора статистики
контроллер PCIe Synopsys DesignWare Cores предоставляет следующие две функции:

- один 64-битный счётчик для анализа на основе времени (Time Based Analysis)
  (пропускная способность данных RX/TX и время, проведённое в каждом
  низкопотребляющем состоянии LTSSM), и
- один 32-битный счётчик на каждое событие для подсчёта событий (Event Counting)
  (ошибочные и неошибочные события для заданной полосы)

Примечание: прерывание для переполнения счётчика отсутствует.

Time Based Analysis
-------------------

Используя эту функцию, вы можете получить информацию о пропускной способности
данных RX/TX и времени, проведённом контроллером в каждом низкопотребляющем
состоянии LTSSM. PMU измеряет данные в двух категориях:

- Group#0: процент времени, которое контроллер находится в состояниях LTSSM.
- Group#1: объём обработанных данных (в единицах по 16 байт).

Lane Event counters
-------------------

Используя эту функцию, вы можете получить информацию об ошибочных и неошибочных
событиях контроллера в конкретной полосе. Событие PMU выбирается совокупностью:

- Group i
- Event j внутри Group i
- Lane k

Некоторые из событий существуют только для определённых конфигураций.

DesignWare Cores (DWC) PCIe PMU Driver
=======================================

Этот драйвер добавляет устройства PMU для каждого корневого порта PCIe, имя
которых формируется на основе SBDF корневого порта. Например,

    0001:30:03.0 PCI bridge: Device 1ded:8000 (rev 01)

имя устройства PMU для этого корневого порта — dwc_rootport_13018.

Драйвер DWC PCIe PMU регистрирует драйвер perf PMU, который предоставляет
описание доступных событий и параметров конфигурации в sysfs, см.
/sys/bus/event_source/devices/dwc_rootport_{sbdf}.

Каталог "format" описывает формат полей config структуры perf_event_attr.
Каталог "events" предоставляет шаблоны конфигурации для всех документированных
событий. Например, "rx_pcie_tlp_data_payload" является эквивалентом
"eventid=0x21,type=0x0".

Команда "perf list" должна вывести список доступных событий из sysfs, например::

    $# perf list | grep dwc_rootport
    <...>
    dwc_rootport_13018/Rx_PCIe_TLP_Data_Payload/        [Kernel PMU event]
    <...>
    dwc_rootport_13018/rx_memory_read,lane=?/               [Kernel PMU event]

Time Based Analysis Event Usage
-------------------------------

Пример подсчёта полезной нагрузки данных PCIe RX TLP (в единицах байт)::

    $# perf stat -a -e dwc_rootport_13018/Rx_PCIe_TLP_Data_Payload/

Среднюю пропускную способность RX/TX можно вычислить по следующей формуле:

    PCIe RX Bandwidth = rx_pcie_tlp_data_payload / Measure_Time_Window
    PCIe TX Bandwidth = tx_pcie_tlp_data_payload / Measure_Time_Window

Lane Event Usage
-------------------------------

Каждая полоса имеет одинаковый набор событий, и чтобы избежать генерации списка
из сотен событий, пользователю необходимо явно указать идентификатор полосы
(lane ID), например::

    $# perf stat -a -e dwc_rootport_13018/rx_memory_read,lane=4/

Драйвер не поддерживает выборку (sampling), поэтому "perf record" работать не
будет. Сессии perf для отдельной задачи (без "-a") не поддерживаются.
