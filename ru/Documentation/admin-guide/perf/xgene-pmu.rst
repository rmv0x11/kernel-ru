=================================================================
Блок мониторинга производительности (PMU) для APM X-Gene SoC
=================================================================

PMU в X-Gene SoC состоит из нескольких независимых системных PMU-устройств,
таких как кэш(и) L3, мост(ы) ввода-вывода, мост(ы) контроллера памяти и
контроллер(ы) памяти. Эти PMU-устройства слабо архитектурно построены по той
же модели, что и PMU для ядер ARM. PMU совместно используют одно и то же
прерывание верхнего уровня и одну и ту же область CSR со статусом.

Драйвер PMU (perf)
------------------

Драйвер xgene-pmu регистрирует несколько драйверов perf PMU. Каждый из драйверов
perf предоставляет описание доступных ему событий и параметров конфигурации
в sysfs, см. /sys/bus/event_source/devices/<l3cX/iobX/mcbX/mcX>/.

Каталог «format» описывает формат полей config (event ID) и config1 (agent ID)
структуры perf_event_attr. Каталог «events» предоставляет шаблоны конфигурации
для всех поддерживаемых типов событий, которые можно использовать с инструментом
perf. Например, «l3c0/bank-fifo-full/» эквивалентно «l3c0/config=0x0b/».

Большинство PMU в SoC имеют определённый список agent ID, используемых для
мониторинга производительности конкретного тракта данных. Например, агентами
кэша L3 могут быть конкретный CPU или мост ввода-вывода. Каждый PMU имеет набор
из 2 регистров, способных маскировать агентов, от которых поступает запрос. Если
установлен бит с номером, соответствующим агенту, событие учитывается только в
том случае, если оно вызвано запросом от этого агента. Каждый бит agent ID
инверсно отображается на соответствующий бит в поле «config1». По умолчанию
событие будет учитываться для запросов всех агентов (config1 = 0x0). Информацию
обо всех поддерживаемых агентах каждого PMU см. в руководстве пользователя
APM X-Gene User Manual.

Каждый драйвер perf также предоставляет sysfs-атрибут «cpumask», который содержит
единственный CPU ID процессора, который будет использоваться для обработки всех
событий PMU.

Пример использования инструмента perf::

 / # perf list | grep -e l3c -e iob -e mcb -e mc
   l3c0/ackq-full/                                    [Kernel PMU event]
 <...>
   mcb1/mcb-csw-stall/                                [Kernel PMU event]

 / # perf stat -a -e l3c0/read-miss/,mcb1/csw-write-request/ sleep 1

 / # perf stat -a -e l3c0/read-miss,config1=0xfffffffffffffffe/ sleep 1

Драйвер не поддерживает сэмплирование, поэтому «perf record» работать не будет.
Сессии perf для отдельной задачи (без «-a») не поддерживаются.
