==========================
Настройка NFS/RDMA
==========================

:Author:
  NetApp and Open Grid Computing (May 29, 2008)

.. warning::
  Этот документ, вероятно, устарел.

Обзор
=====

В этом документе описывается, как установить и настроить клиентское и серверное
программное обеспечение Linux NFS/RDMA.

Клиент NFS/RDMA впервые был включён в Linux 2.6.24. Сервер NFS/RDMA впервые
был включён в следующем выпуске, Linux 2.6.25.

В ходе наших тестов мы получили превосходные результаты по производительности
(полная полоса пропускания канала 10 Гбит при минимальной нагрузке на CPU клиента)
при многих видах рабочих нагрузок. Код проходит полный набор тестов Connectathon
и работает как с адаптерами Infiniband, так и с адаптерами iWARP RDMA.

Где получить помощь
===================

Если вы столкнулись с затруднениями, вы можете задать вопросы в списке рассылки
nfs-rdma-devel@lists.sourceforge.net.

Установка
=========

Эти инструкции представляют собой пошаговое руководство по подготовке машины
для использования с NFS/RDMA.

- Установите устройство RDMA

  Подходит любое устройство, поддерживаемое драйверами из drivers/infiniband/hw.

  Тестирование проводилось с использованием нескольких IB-карт на базе Mellanox,
  адаптера Ammasso AMS1100 iWARP и адаптера Chelsio cxgb3 iWARP.

- Установите дистрибутив Linux и инструменты

  Первым выпуском ядра, содержащим как клиент, так и сервер NFS/RDMA, был
  Linux 2.6.25  Поэтому следует установить дистрибутив, совместимый с этим и
  последующими выпусками ядра Linux.

  Процедуры, описанные в этом документе, были протестированы с дистрибутивами
  проекта Red Hat's Fedora Project (http://fedora.redhat.com/).

- Установите nfs-utils-1.1.2 или новее на клиенте

  Точку монтирования NFS/RDMA можно получить с помощью команды mount.nfs из
  nfs-utils-1.1.2 или новее (nfs-utils-1.1.1 был первой версией nfs-utils с
  поддержкой монтирования NFS/RDMA, но по различным причинам мы рекомендуем
  использовать nfs-utils-1.1.2 или новее). Чтобы узнать, какую версию
  mount.nfs вы используете, введите:

  .. code-block:: sh

    $ /sbin/mount.nfs -V

  Если версия ниже 1.1.2 или команда не существует, вам следует установить
  последнюю версию nfs-utils.

  Скачайте последний пакет с: https://www.kernel.org/pub/linux/utils/nfs

  Распакуйте пакет и следуйте инструкциям по установке.

  Если вам не понадобятся исполняемые файлы idmapper и gssd (они не нужны для
  создания команды монтирования с поддержкой NFS/RDMA), процесс установки можно
  упростить, отключив эти возможности при запуске configure:

  .. code-block:: sh

    $ ./configure --disable-gss --disable-nfsv4

  Для сборки nfs-utils вам потребуется установленный пакет tcp_wrappers.
  Дополнительную информацию об этом см. в файлах README и INSTALL пакета.

  После сборки пакета nfs-utils в каталоге utils/mount появится бинарный файл
  mount.nfs. Этот бинарный файл можно использовать для инициирования
  монтирований NFS v2, v3 или v4. Чтобы инициировать монтирование v4, бинарный
  файл должен называться mount.nfs4.  Стандартный приём — создать символьную
  ссылку mount.nfs4 на mount.nfs.

  Этот бинарный файл mount.nfs следует установить в /sbin/mount.nfs следующим
  образом:

  .. code-block:: sh

    $ sudo cp utils/mount/mount.nfs /sbin/mount.nfs

  В этом расположении mount.nfs будет автоматически вызываться для монтирований
  NFS системной командой mount.

    .. note::
      mount.nfs и, следовательно, nfs-utils-1.1.2 или новее нужны только на
      клиентской машине NFS. На сервере эта конкретная версия nfs-utils не
      нужна. Более того, на клиенте нужна только команда mount.nfs из
      nfs-utils-1.1.2.

- Установите ядро Linux с поддержкой NFS/RDMA

  Клиент и сервер NFS/RDMA оба включены в mainline-ядро Linux версии 2.6.25 и
  более поздних. Это и другие версии ядра Linux можно найти по адресу:
  https://www.kernel.org/pub/linux/kernel/

  Скачайте исходные коды и поместите их в подходящее место.

- Настройте стек RDMA

  Убедитесь, что в конфигурации вашего ядра включена поддержка RDMA. В разделе
  Device Drivers -> InfiniBand support измените конфигурацию ядра, чтобы включить
  поддержку InfiniBand [ПРИМЕЧАНИЕ: название параметра вводит в заблуждение.
  Включение поддержки InfiniBand требуется для всех устройств RDMA (IB, iWARP
  и т. д.)].

  Включите поддержку соответствующего IB HCA (mlx4, mthca, ehca, ipath и т. д.)
  или поддержку адаптера iWARP (amso, cxgb3 и т. д.).

  Если вы используете InfiniBand, обязательно включите поддержку IP-over-InfiniBand.

- Настройте клиент и сервер NFS

  В конфигурации вашего ядра также должна быть включена поддержка файловой
  системы NFS и/или поддержка сервера NFS. Эти и другие параметры конфигурации,
  связанные с NFS, можно найти в разделе File Systems -> Network File Systems.

- Соберите, установите, перезагрузите

  Код NFS/RDMA будет включён автоматически, если включены NFS и RDMA. Клиент и
  сервер NFS/RDMA настраиваются через скрытый параметр конфигурации
  SUNRPC_XPRT_RDMA, который зависит от SUNRPC и INFINIBAND. Значение
  SUNRPC_XPRT_RDMA будет:

    #. N, если SUNRPC или INFINIBAND равны N; в этом случае клиент и сервер
       NFS/RDMA собираться не будут

    #. M, если оба, SUNRPC и INFINIBAND, включены (M или Y) и хотя бы один из
       них равен M; в этом случае клиент и сервер NFS/RDMA будут собраны как
       модули

    #. Y, если оба, SUNRPC и INFINIBAND, равны Y; в этом случае клиент и сервер
       NFS/RDMA будут встроены в ядро

  Поэтому, если вы выполнили описанные выше шаги и включили NFS и RDMA, клиент и
  сервер NFS/RDMA будут собраны.

  Соберите новое ядро, установите его, загрузите его.

Проверка настройки RDMA и NFS
=============================

Прежде чем настраивать программное обеспечение NFS/RDMA, рекомендуется
протестировать ваше новое ядро, чтобы убедиться, что ядро работает правильно.
В частности, рекомендуется проверить, что стек RDMA функционирует так, как
ожидается, и что стандартный NFS поверх TCP/IP и/или UDP/IP работает корректно.

- Проверьте настройку RDMA

  Если вы собрали компоненты RDMA как модули, загрузите их в этот момент.
  Например, если вы используете карту Mellanox Tavor/Sinai/Arbel:

  .. code-block:: sh

    $ modprobe ib_mthca
    $ modprobe ib_ipoib

  Если вы используете InfiniBand, убедитесь, что в сети работает менеджер
  подсети (Subnet Manager, SM). Если ваш IB-коммутатор имеет встроенный SM, вы
  можете использовать его. В противном случае вам потребуется запустить SM,
  например OpenSM, на одном из ваших оконечных узлов.

  Если в вашей сети работает SM, вы должны увидеть следующее:

  .. code-block:: sh

    $ cat /sys/class/infiniband/driverX/ports/1/state
    4: ACTIVE

  где driverX — это mthca0, ipath5, ehca3 и т. д.

  Чтобы дополнительно протестировать программный стек InfiniBand, используйте
  IPoIB (предполагается, что у вас есть два IB-хоста с именами host1 и host2):

  .. code-block:: sh

    host1$ ip link set dev ib0 up
    host1$ ip address add dev ib0 a.b.c.x
    host2$ ip link set dev ib0 up
    host2$ ip address add dev ib0 a.b.c.y
    host1$ ping a.b.c.y
    host2$ ping a.b.c.x

  Для других типов устройств следуйте соответствующим процедурам.

- Проверьте настройку NFS

  Для включённых выше компонентов NFS (клиента и/или сервера) протестируйте их
  работу поверх стандартного Ethernet с использованием TCP/IP или UDP/IP.

Конфигурация NFS/RDMA
=====================

Мы рекомендуем использовать две машины: одну в роли клиента и одну в роли
сервера.

Однократная настройка:
----------------------

- На серверной системе настройте файл /etc/exports и запустите сервер NFS/RDMA.

  Были протестированы записи exports следующих форматов::

  /vol0   192.168.0.47(fsid=0,rw,async,insecure,no_root_squash)
  /vol0   192.168.0.0/255.255.255.0(fsid=0,rw,async,insecure,no_root_squash)

  IP-адрес(а) — это IPoIB-адрес клиента для InfiniBand HCA или iWARP-адрес(а)
  клиента для RNIC.

  .. note::
    Параметр "insecure" должен использоваться, поскольку клиент NFS/RDMA не
    использует зарезервированный порт.

При каждой загрузке машины:
---------------------------

- Загрузите и настройте драйверы RDMA

  Для InfiniBand с использованием адаптера Mellanox:

  .. code-block:: sh

    $ modprobe ib_mthca
    $ modprobe ib_ipoib
    $ ip li set dev ib0 up
    $ ip addr add dev ib0 a.b.c.d

  .. note::
    Пожалуйста, используйте уникальные адреса для клиента и сервера!

- Запустите сервер NFS

  Если сервер NFS/RDMA был собран как модуль (CONFIG_SUNRPC_XPRT_RDMA=m в
  конфигурации ядра), загрузите транспортный модуль RDMA:

  .. code-block:: sh

    $ modprobe svcrdma

  Независимо от того, как был собран сервер (модуль или встроенный), запустите
  сервер:

  .. code-block:: sh

    $ /etc/init.d/nfs start

  или

  .. code-block:: sh

    $ service nfs start

  Укажите серверу слушать на транспорте RDMA:

  .. code-block:: sh

    $ echo rdma 20049 > /proc/fs/nfsd/portlist

- На клиентской системе

  Если клиент NFS/RDMA был собран как модуль (CONFIG_SUNRPC_XPRT_RDMA=m в
  конфигурации ядра), загрузите клиентский модуль RDMA:

  .. code-block:: sh

    $ modprobe xprtrdma.ko

  Независимо от того, как был собран клиент (модуль или встроенный), используйте
  эту команду для монтирования сервера NFS/RDMA:

  .. code-block:: sh

    $ mount -o rdma,port=20049 <IPoIB-server-name-or-address>:/<export> /mnt

  Чтобы убедиться, что монтирование использует RDMA, выполните "cat /proc/mounts"
  и проверьте поле "proto" для данного монтирования.

  Поздравляем! Вы используете NFS/RDMA!
