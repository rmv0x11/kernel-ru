============================================================
Руководство пользователя сервера pNFS с блочной разметкой
============================================================

Сервер Linux NFS теперь поддерживает расширение pNFS с блочной разметкой (block
layout). В этом случае сервер NFS выступает в роли сервера метаданных (Metadata
Server, MDS) для pNFS, который, помимо обработки всего доступа к метаданным
экспорта NFS, также выдаёт клиентам разметки (layouts) для прямого доступа к
нижележащим блочным устройствам, разделяемым с клиентом.

Чтобы использовать pNFS с блочной разметкой на сервере Linux NFS, экспортируемая
файловая система должна поддерживать блочные разметки pNFS (на данный момент это
только XFS), и файловая система должна располагаться на разделяемом хранилище
(как правило, iSCSI), которое доступно клиентам в дополнение к MDS. На данный
момент файловая система должна располагаться непосредственно на экспортируемом
томе; чередование (striping) или конкатенация томов на MDS и клиентах пока не
поддерживаются.

На сервере поддержка блочных томов pNFS включается автоматически, если файловая
система её поддерживает. На клиенте убедитесь, что в ядре включён параметр
CONFIG_PNFS_BLOCK, запущен демон blkmapd из nfs-utils, а файловая система
смонтирована с использованием версии протокола NFSv4.1 (mount -o vers=4.1).

Если серверу nfsd нужно изолировать (fence) не отвечающего клиента, он вызывает
/sbin/nfsd-recall-failed, передавая в первом аргументе IP-адрес клиента, а во
втором аргументе — узел устройства без префикса /dev для файловой системы,
которую нужно изолировать. Ниже приведён пример файла, показывающий, как
преобразовать устройство в серийный номер из SCSI EVPD 0x80::

	cat > /sbin/nfsd-recall-failed << EOF

.. code-block:: sh

	#!/bin/sh

	CLIENT="$1"
	DEV="/dev/$2"
	EVPD=`sg_inq --page=0x80 ${DEV} | \
		grep "Unit serial number:" | \
		awk -F ': ' '{print $2}'`

	echo "fencing client ${CLIENT} serial ${EVPD}" >> /var/log/pnfsd-fence.log
	EOF
