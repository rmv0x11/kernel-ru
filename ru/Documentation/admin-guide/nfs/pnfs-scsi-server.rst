
========================================================
Руководство пользователя по серверу с pNFS SCSI layout
========================================================

Этот документ описывает поддержку pNFS SCSI layout в NFS-сервере Linux.
При использовании pNFS SCSI layout NFS-сервер выступает в роли сервера
метаданных (Metadata Server, MDS) для pNFS, который, помимо обработки
всего доступа к метаданным экспортируемого ресурса NFS, также выдаёт
клиентам layout, чтобы те могли напрямую обращаться к базовым SCSI LUN,
общим для них.

Чтобы использовать pNFS SCSI layout с NFS-сервером Linux, экспортируемая
файловая система должна поддерживать pNFS SCSI layout (в настоящее время
только XFS), и файловая система должна располагаться на SCSI LUN, доступном
клиентам в дополнение к MDS.  На данный момент файловая система должна
располагаться непосредственно на экспортируемом LUN; чередование (striping)
или объединение (concatenation) LUN на MDS и клиентах пока не поддерживается.

На сервере, собранном с CONFIG_NFSD_SCSI, поддержка томов pNFS SCSI
включается автоматически, если файловая система экспортирована с параметром
«pnfs» и базовое SCSI-устройство поддерживает постоянные резервирования
(persistent reservations).  На клиенте убедитесь, что в ядре включён параметр
CONFIG_PNFS_BLOCK, и файловая система смонтирована с использованием версии
протокола NFSv4.1 (mount -o vers=4.1).
