============================================
Интерфейсы администрирования для nfsd
============================================

Обратите внимание, что обычно эти интерфейсы используются только утилитами из
nfs-utils.

nfsd управляется главным образом через псевдофайлы файловой системы "nfsd",
которая обычно монтируется в /proc/fs/nfsd/.

Сервер всегда запускается первой записью ненулевого значения в
nfsd/threads.

Перед этим NFSD можно указать, какие сокеты слушать, выполнив запись в
nfsd/portlist; этой записью может быть:

	-  файловый дескриптор в ascii-кодировке, который должен ссылаться на
	   привязанный (и слушающий, для tcp) сокет, либо
	-  "transportname port", где transportname в настоящее время — это
	   "udp", "tcp" или "rdma".

Если nfsd запускается без выполнения какой-либо из этих операций, то он создаст один
udp- и один tcp-слушатель на порту 2049 (см. nfsd_init_socks).

При запуске начинаются grace-периоды nfsd и lockd. nfsd завершается записью
0 в nfsd/threads.  В этот момент все блокировки и состояние отбрасываются.

Между запуском и завершением число потоков может быть увеличено или
уменьшено дополнительными записями в nfsd/threads или записями в
nfsd/pool_threads.

Подробнее о файлах в nfsd/ и о том, чем они управляют, см.
fs/nfsd/nfsctl.c; большинство из них снабжены подробными комментариями.

Замечания по реализации
=======================

Обратите внимание, что rpc-сервер требует от вызывающей стороны сериализации добавления и
удаления слушающих сокетов, а также запуска и завершения сервера.
Для nfsd это делается с помощью nfsd_mutex.
