==========================================
Аппаратные генераторы случайных чисел
==========================================

Введение
========

Фреймворк hw_random — это программное обеспечение, которое использует
специальную аппаратную возможность вашего CPU или материнской платы —
генератор случайных чисел (Random Number Generator, RNG). Это программное
обеспечение состоит из двух частей: ядро, предоставляющее символьное
устройство /dev/hwrng и его поддержку в sysfs, и аппаратно-зависимый
драйвер, который подключается к этому ядру.

Чтобы наиболее эффективно использовать эти механизмы, вам следует также
загрузить вспомогательное программное обеспечение. Загрузите последнюю
версию пакета «rng-tools» по адресу:

	https://github.com/nhorman/rng-tools

Эти инструменты используют /dev/hwrng для заполнения пула энтропии ядра,
который применяется внутри ядра и экспортируется через специальные файлы
/dev/urandom и /dev/random.

Принцип работы
==============

СИМВОЛЬНОЕ УСТРОЙСТВО. С помощью стандартных системных вызовов open()
и read() вы можете считывать случайные данные с аппаратного устройства RNG.
Эти данные НЕ ПРОВЕРЯЮТСЯ какими-либо тестами на пригодность и потенциально
могут быть недостоверными (если оборудование неисправно или было
подвергнуто вмешательству). Данные выдаются только при установленном
аппаратном флаге «has-data», тем не менее человек, заботящийся о
безопасности, прогнал бы тесты на пригодность этих данных, прежде чем
считать их действительно случайными.

Пакет rng-tools использует такие тесты в «rngd» и позволяет запускать их
вручную с помощью утилиты «rngtest».

/dev/hwrng — это символьное устройство с major-номером 10 и minor-номером 183.

УСТРОЙСТВО КЛАССА. Существует узел /sys/class/misc/hw_random с двумя
уникальными атрибутами, «rng_available» и «rng_current». Атрибут
«rng_available» перечисляет доступные аппаратно-зависимые драйверы,
тогда как «rng_current» указывает тот из них, который в настоящее время
подключён к /dev/hwrng. Если в вашей системе доступно более одного RNG,
вы можете сменить используемый, записав имя из списка в «rng_available»
в «rng_current».

Об оборудовании Intel RNG, по данным из datasheet на firmware hub
=================================================================

Firmware Hub содержит встроенный генератор случайных чисел (RNG),
использующий тепловой шум, порождаемый изначально случайными
квантово-механическими свойствами кремния. Когда новые случайные биты
не генерируются, схема RNG переходит в состояние низкого энергопотребления.
Intel предоставит двоичный программный драйвер, дающий стороннему
программному обеспечению доступ к нашему RNG для использования в качестве
средства безопасности. На данный момент RNG предназначен для использования
только в системе с присутствующей ОС.

.. note::

	request_mem_region был удалён по трём причинам:

	1) этим драйвером поддерживается только один RNG;
	2) расположение, используемое RNG, является фиксированным
	   расположением в памяти, адресуемой через MMIO;
	3) у пользователей с корректно работающей обработкой BIOS e820
	   область, в которой расположен RNG, всегда будет зарезервирована,
	   поэтому вызовы request_mem_region для корректных конфигураций
	   всегда завершаются неудачей. Однако у тех, кто использует mem=XX,
	   информация BIOS e820 **отсутствует** в /proc/iomem, и вызов
	   request_mem_region(RNG_ADDR) может завершиться успешно.
