=============================================
Состояния производительности вентилятора ACPI
=============================================

Когда необязательный объект _FPS присутствует под ACPI-устройством,
представляющим вентилятор (например, PNP0C0B или INT3404), драйвер
вентилятора ACPI создаёт дополнительные атрибуты «state*» в sysfs-каталоге
данного ACPI-устройства. Эти атрибуты перечисляют свойства состояний
производительности вентилятора.

Дополнительную информацию о _FPS см. в спецификации ACPI по адресу:

http://uefi.org/specifications

Например, содержимое sysfs-каталога ACPI-устройства INT3404
может выглядеть следующим образом::

 $ ls -l /sys/bus/acpi/devices/INT3404:00/
 total 0
 ...
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state0
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state1
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state10
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state11
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state2
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state3
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state4
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state5
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state6
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state7
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state8
 -r--r--r-- 1 root root 4096 Dec 13 20:38 state9
 -r--r--r-- 1 root root 4096 Dec 13 01:00 status
 ...

где каждый из файлов «state*» представляет одно состояние производительности
вентилятора и содержит разделённый двоеточиями список из 5 целочисленных
значений (полей) со следующей интерпретацией::

  control_percent:trip_point_index:speed_rpm:noise_level_mdb:power_mw

* ``control_percent``: Процентное значение, используемое для установки скорости
  вентилятора на определённый уровень с помощью объекта _FSL (0-100).

* ``trip_point_index``: Номер точки срабатывания активного охлаждения, которая
  соответствует данному состоянию производительности (0-9).

* ``speed_rpm``: Скорость вентилятора в оборотах в минуту.

* ``noise_level_mdb``: Слышимый шум, издаваемый вентилятором в этом состоянии, в
  миллидецибелах.

* ``power_mw``: Потребляемая вентилятором мощность в этом состоянии в милливаттах.

Например::

 $cat /sys/bus/acpi/devices/INT3404:00/state1
 25:0:3200:12500:1250

Когда заданное поле не заполнено или его значение, предоставленное прошивкой
платформы (platform firmware), недопустимо, вместо значения отображается строка
«not-defined».

Точное управление вентилятором ACPI (Fine Grain Control)
========================================================

Когда объект _FIF указывает на поддержку точного управления, скорость
вентилятора может быть задана от 0 до 100% с рекомендуемым минимальным «шагом»
через объект _FSL. Пользователь может регулировать скорость вентилятора с
помощью устройства охлаждения thermal в sysfs.

Здесь пользователь может посмотреть состояния производительности вентилятора для
получения эталонной скорости (speed_rpm) и задать её, изменяя cur_state
устройства охлаждения. Если поддерживается точное управление, то пользователь
также может настроить некоторые другие скорости, которые не определены в
состояниях производительности.

Поддержка точного управления отражается через sysfs-атрибут
«fine_grain_control». Если точное управление присутствует, этот атрибут
покажет «1», иначе «0».

Этот sysfs-атрибут располагается в том же каталоге, что и состояния
производительности.

Обратная связь о производительности вентилятора ACPI
====================================================

Необязательный объект _FST предоставляет информацию о состоянии устройства
вентилятора. Сюда входит поле, предоставляющее текущую скорость вентилятора в
оборотах в минуту, с которой вращается вентилятор.

Эта скорость отражается в sysfs с помощью атрибута «fan_speed_rpm», в том же
каталоге, что и состояния производительности.
