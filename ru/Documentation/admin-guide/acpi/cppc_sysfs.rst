==================================================
Collaborative Processor Performance Control (CPPC)
==================================================

.. _cppc_sysfs:

CPPC
====

CPPC, определённый в спецификации ACPI, описывает механизм, позволяющий ОС
управлять производительностью логического процессора в рамках непрерывной и
абстрактной шкалы производительности. CPPC предоставляет набор регистров для
описания абстрактной шкалы производительности, запроса уровней
производительности и измерения фактически обеспеченной производительности по
каждому процессору.

Более подробные сведения о CPPC см. в спецификации ACPI по адресу:

http://uefi.org/specifications

Некоторые из регистров CPPC доступны через sysfs по пути::

  /sys/devices/system/cpu/cpuX/acpi_cppc/

для каждого процессора X::

  $ ls -lR  /sys/devices/system/cpu/cpu0/acpi_cppc/
  /sys/devices/system/cpu/cpu0/acpi_cppc/:
  total 0
  -r--r--r-- 1 root root 65536 Mar  5 19:38 feedback_ctrs
  -r--r--r-- 1 root root 65536 Mar  5 19:38 highest_perf
  -r--r--r-- 1 root root 65536 Mar  5 19:38 lowest_freq
  -r--r--r-- 1 root root 65536 Mar  5 19:38 lowest_nonlinear_perf
  -r--r--r-- 1 root root 65536 Mar  5 19:38 lowest_perf
  -r--r--r-- 1 root root 65536 Mar  5 19:38 nominal_freq
  -r--r--r-- 1 root root 65536 Mar  5 19:38 nominal_perf
  -r--r--r-- 1 root root 65536 Mar  5 19:38 reference_perf
  -r--r--r-- 1 root root 65536 Mar  5 19:38 wraparound_time

* highest_perf : Наивысшая производительность данного процессора (абстрактная шкала).
* nominal_perf : Наивысшая устойчивая производительность данного процессора
  (абстрактная шкала).
* lowest_nonlinear_perf : Наименьшая производительность данного процессора с
  нелинейным энергосбережением (абстрактная шкала).
* lowest_perf : Наименьшая производительность данного процессора (абстрактная шкала).

* lowest_freq : Частота CPU, соответствующая lowest_perf (в МГц).
* nominal_freq : Частота CPU, соответствующая nominal_perf (в МГц).
  Указанные выше частоты следует использовать только для представления
  производительности процессора в виде частоты вместо абстрактной шкалы. Эти
  значения не должны использоваться для каких-либо функциональных решений.

* feedback_ctrs : Включает как опорный счётчик производительности, так и счётчик
  фактически обеспеченной производительности.
  Опорный счётчик увеличивается пропорционально опорной производительности процессора.
  Счётчик обеспеченной производительности увеличивается пропорционально фактически
  обеспеченной производительности процессора.
* wraparound_time: Минимальное время до переполнения счётчиков обратной связи
  (в секундах).
* reference_perf : Уровень производительности, на котором накапливается опорный
  счётчик производительности (абстрактная шкала).


Вычисление средней обеспеченной производительности
==================================================

Ниже описаны шаги для вычисления средней обеспеченной производительности путём
снятия двух разных снимков счётчиков обратной связи в моменты времени T1 и T2.

  T1: Прочитать feedback_ctrs как fbc_t1
      Подождать или выполнить какую-либо нагрузку

  T2: Прочитать feedback_ctrs как fbc_t2

::

  delivered_counter_delta = fbc_t2[del] - fbc_t1[del]
  reference_counter_delta = fbc_t2[ref] - fbc_t1[ref]

  delivered_perf = (reference_perf x delivered_counter_delta) / reference_counter_delta
