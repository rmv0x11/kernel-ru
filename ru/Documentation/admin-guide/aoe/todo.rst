Существует потенциальная возможность взаимоблокировки (deadlock) при выделении
struct sk_buff для данных, которые требуется записать в хранилище aoe. Если
данные записываются из «грязной» страницы с целью освободить эту страницу и при
этом нет других доступных страниц, то взаимоблокировка может возникнуть, когда
свободная страница понадобится для выделения sk_buff. Такая ситуация не
наблюдалась, но было бы желательно устранить любую потенциальную возможность
взаимоблокировки в условиях нехватки памяти.

Поскольку ATA over Ethernet не фрагментируется IP-кодом ядра, член destructor
структуры struct sk_buff доступен драйверу aoe. Используя mempool для выделения
всех sk_buff, кроме первых нескольких, и регистрируя destructor, мы должны
получить возможность эффективно выделять sk_buff, не создавая никакой
потенциальной возможности взаимоблокировки.
