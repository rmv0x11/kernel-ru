Введение
========

ATA over Ethernet — это сетевой протокол, обеспечивающий простой доступ к
блочному хранилищу в локальной сети (LAN).

  http://support.coraid.com/documents/AoEr11.txt

Руководство EtherDrive (R) HOWTO для ядер 2.6 и 3.x находится по адресу ...

  http://support.coraid.com/support/linux/EtherDrive-2.6-HOWTO.html

В нём много советов и подсказок!  Обратите особое внимание на рекомендуемые
настройки виртуальной памяти:

  http://support.coraid.com/support/linux/EtherDrive-2.6-HOWTO-5.html#ss5.19

aoetools — это программы пространства пользователя, предназначенные для работы с
этим драйвером.  aoetools размещены на sourceforge.

  http://aoetools.sourceforge.net/

Скрипты в этом каталоге Documentation/admin-guide/aoe предназначены для того,
чтобы документировать использование драйвера, и не нужны, если вы устанавливаете
aoetools.


Создание узлов устройств
========================

  Пользователи udev должны обнаружить, что узлы блочных устройств создаются
  автоматически, но для создания всех необходимых узлов устройств используйте
  правила конфигурации udev, предоставленные в udev.txt (в этом каталоге).

  Существует скрипт udev-install.sh, который показывает, как установить эти
  правила в вашей системе.

  Также существует скрипт autoload, который показывает, как отредактировать
  /etc/modprobe.d/aoe.conf, чтобы гарантировать загрузку модуля aoe при
  необходимости.  Однако предварительная загрузка модуля aoe предпочтительнее
  автозагрузки, поскольку обнаружение AoE занимает несколько секунд.  Может
  возникнуть путаница, когда устройство AoE отсутствует при первом запуске
  команды, но появляется секундой позже.

Использование узлов устройств
=============================

  "cat /dev/etherd/err" блокируется, ожидая вывода диагностики ошибок,
  например о любых повторно переданных пакетах.

  "echo eth2 eth4 > /dev/etherd/interfaces" указывает драйверу aoe
  ограничить трафик ATA over Ethernet интерфейсами eth2 и eth4.  Трафик AoE из
  недоверенных сетей следует игнорировать из соображений безопасности.  См.
  также описанную ниже опцию драйвера aoe_iflist.

  "echo > /dev/etherd/discover" указывает драйверу выяснить, какие устройства
  AoE доступны.

  В будущем эти символьные устройства могут исчезнуть и быть заменены
  аналогами в sysfs.  Использование команд из aoetools ограждает
  пользователей от этих деталей реализации.

  Блочные устройства именуются следующим образом::

	e{shelf}.{slot}
	e{shelf}.{slot}p{part}

  ... так что "e0.2" — это третий blade слева (слот 2) в
  первой полке (адрес полки ноль).  Это диск целиком.  Первый
  раздел на этом диске будет называться "e0.2p1".

Использование sysfs
===================

  Каждое блочное устройство aoe в /sys/block имеет дополнительные атрибуты
  state, mac и netif.  Атрибут state равен "up", когда устройство
  готово к вводу-выводу (I/O), и "down", если оно обнаружено, но непригодно к
  использованию.  Состояние "down,closewait" показывает, что устройство всё
  ещё открыто и не может снова перейти в рабочее состояние, пока не будет закрыто.

  Атрибут mac — это ethernet-адрес удалённого устройства AoE.
  Атрибут netif — это сетевой интерфейс на локальном узле,
  через который происходит обмен данными с удалённым устройством AoE.

  В этом каталоге есть скрипт, который форматирует эту информацию
  удобным образом.  Пользователям aoetools следует использовать команду
  aoe-stat::

    root@makki root# sh Documentation/admin-guide/aoe/status.sh
       e10.0            eth3              up
       e10.1            eth3              up
       e10.2            eth3              up
       e10.3            eth3              up
       e10.4            eth3              up
       e10.5            eth3              up
       e10.6            eth3              up
       e10.7            eth3              up
       e10.8            eth3              up
       e10.9            eth3              up
        e4.0            eth1              up
        e4.1            eth1              up
        e4.2            eth1              up
        e4.3            eth1              up
        e4.4            eth1              up
        e4.5            eth1              up
        e4.6            eth1              up
        e4.7            eth1              up
        e4.8            eth1              up
        e4.9            eth1              up

  Используйте /sys/module/aoe/parameters/aoe_iflist (или, лучше, опцию
  драйвера, обсуждаемую ниже) вместо /dev/etherd/interfaces, чтобы ограничить
  трафик AoE сетевыми интерфейсами из заданного списка, разделённого
  пробелами.  В отличие от старого символьного устройства, запись
  sysfs можно как читать, так и записывать.

  Полезно инициировать обнаружение после установки списка разрешённых
  интерфейсов.  Пакет aoetools предоставляет для этой цели скрипт
  aoe-discover.  Вы также можете напрямую использовать
  специальный файл /dev/etherd/discover, описанный выше.

Опции драйвера
==============

  Существует опция загрузки для встроенного драйвера aoe и
  соответствующий параметр модуля aoe_iflist.  Без этой опции
  для ATA over Ethernet могут использоваться все сетевые интерфейсы.  Вот
  пример использования параметра модуля::

    modprobe aoe_iflist="eth1 eth3"

  Параметр модуля aoe_deadsecs определяет максимальное число
  секунд, в течение которых драйвер будет ожидать от устройства AoE
  ответа на команду AoE.  По истечении aoe_deadsecs секунд
  устройство AoE будет помечено как "down".  Значение ноль
  поддерживается в целях тестирования и заставляет драйвер aoe
  пытаться выполнять команды AoE бесконечно.

  Параметр модуля aoe_maxout имеет значение по умолчанию 128.  Это
  максимальное число пакетов без ответа, которые будут отправлены цели AoE
  одновременно.

  Параметр модуля aoe_dyndevs по умолчанию равен 1, что означает, что
  драйвер будет назначать обнаруженной цели AoE младший номер блочного
  устройства на основе порядка её обнаружения.  При использовании динамических
  младших номеров устройств может поддерживаться более широкий диапазон адресов
  полок (shelf) и слотов (slot) AoE.  Пользователям udev никогда не придётся
  задумываться о младших номерах.  Использование aoe_dyndevs=0 позволяет
  предварительно создавать узлы устройств с применением схемы статических младших
  номеров с помощью скрипта aoe-mkshelf из aoetools.
