=========
dm-flakey
=========

Эта цель аналогична линейной (linear) цели, за исключением того, что она
периодически демонстрирует ненадёжное поведение. Она оказалась полезной для
имитации сбойных устройств в целях тестирования.

Начиная с момента загрузки таблицы, устройство доступно в течение <up interval>
секунд, затем демонстрирует ненадёжное поведение в течение <down interval>
секунд, после чего этот цикл повторяется.

Также рассмотрите возможность использования этого совместно с целью dm-delay,
которая может задерживать чтения и записи и/или направлять их на различные
нижележащие устройства.

Параметры таблицы
-----------------

::

  <dev path> <offset> <up interval> <down interval> \
    [<num_features> [<feature arguments>]]

Обязательные параметры:

    <dev path>:
        Полный путь к нижележащему блочному устройству или номер
        устройства в виде "major:minor".
    <offset>:
        Начальный сектор внутри устройства.
    <up interval>:
        Число секунд, в течение которых устройство доступно.
    <down interval>:
        Число секунд, в течение которых устройство возвращает ошибки.

Необязательные параметры возможностей:

  Если параметры возможностей не заданы, в периоды ненадёжности весь
  ввод-вывод (I/O) возвращает ошибки.

  error_reads:
	Весь ввод-вывод чтения завершается с сигнализированием об ошибке.
	Ввод-вывод записи обрабатывается корректно.

  drop_writes:
	Весь ввод-вывод записи молча игнорируется.
	Ввод-вывод чтения обрабатывается корректно.

  error_writes:
	Весь ввод-вывод записи завершается с сигнализированием об ошибке.
	Ввод-вывод чтения обрабатывается корректно.

  corrupt_bio_byte <Nth_byte> <direction> <value> <flags>:
	В течение <down interval> заменять <Nth_byte> данных каждого
	подходящего bio на <value>.

    <Nth_byte>:
	Смещение заменяемого байта.
	Отсчёт начинается с 1, чтобы заменить первый байт.
    <direction>:
	Либо 'r' для повреждения чтений, либо 'w' для повреждения записей.
	'w' несовместимо с drop_writes.
    <value>:
	Значение (от 0 до 255) для записи.
    <flags>:
	Выполнять замену только в том случае, если в bio->bi_opf
	установлены все выбранные флаги.

  random_read_corrupt <probability>
	В течение <down interval> заменять случайный байт в bio чтения
	на случайное значение. probability — целое число от
	0 до 1000000000, означающее вероятность повреждения от 0% до 100%.

  random_write_corrupt <probability>
	В течение <down interval> заменять случайный байт в bio записи
	на случайное значение. probability — целое число от
	0 до 1000000000, означающее вероятность повреждения от 0% до 100%.

Примеры:

Заменяет 32-й байт bio чтения (READ) значением 1::

  corrupt_bio_byte 32 r 1 0

Заменяет 224-й байт bio типа REQ_META (=32) значением 0::

  corrupt_bio_byte 224 w 0 32
