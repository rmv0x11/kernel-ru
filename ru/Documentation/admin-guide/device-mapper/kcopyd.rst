======
kcopyd
======

Kcopyd предоставляет возможность копировать диапазон секторов с одного
блочного устройства на одно или несколько других блочных устройств с
асинхронным уведомлением о завершении. Он используется dm-snapshot и dm-mirror.

Пользователи kcopyd сначала должны создать клиента и указать, сколько страниц
памяти выделить для своих заданий копирования. Это делается вызовом
kcopyd_client_create()::

   int kcopyd_client_create(unsigned int num_pages,
                            struct kcopyd_client **result);

Чтобы запустить задание копирования, пользователь должен подготовить структуры
io_region, описывающие источник и приёмники копирования. Каждая io_region
указывает блочное устройство вместе с начальным сектором и размером области.
Источник копирования задаётся одной структурой io_region, а приёмники
копирования задаются массивом структур io_region::

   struct io_region {
      struct block_device *bdev;
      sector_t sector;
      sector_t count;
   };

Чтобы запустить копирование, пользователь вызывает kcopyd_copy(), передавая
указатель на клиента, указатели на io_region источника и приёмников, имя
процедуры обратного вызова о завершении и указатель на некоторые контекстные
данные для копирования::

   int kcopyd_copy(struct kcopyd_client *kc, struct io_region *from,
                   unsigned int num_dests, struct io_region *dests,
                   unsigned int flags, kcopyd_notify_fn fn, void *context);

   typedef void (*kcopyd_notify_fn)(int read_err, unsigned int write_err,
				    void *context);

Когда копирование завершается, kcopyd вызовет пользовательскую процедуру
завершения, передавая обратно пользовательский указатель на контекст. Он также
сообщит, произошла ли ошибка чтения или записи во время копирования.

Когда пользователь завершил все свои задания копирования, ему следует вызвать
kcopyd_client_destroy(), чтобы удалить клиента kcopyd, что освободит связанные
с ним страницы памяти::

   void kcopyd_client_destroy(struct kcopyd_client *kc);
