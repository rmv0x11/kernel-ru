==============
Статистика DM
==============

Device Mapper поддерживает сбор статистики ввода-вывода для определяемых
пользователем областей DM-устройства.	 Если ни одна область не определена,
статистика не собирается, поэтому никакого влияния на производительность нет.
В настоящее время поддерживаются только bio-based DM-устройства.

Каждая определяемая пользователем область задаёт начальный сектор, длину и шаг.
Для каждой области размером в шаг внутри заданного диапазона будет собираться
отдельная статистика.

Счётчики статистики ввода-вывода для каждой области размером в шаг имеют тот же
формат, что и `/sys/block/*/stat` или `/proc/diskstats` (см.:
Documentation/admin-guide/iostats.rst).  Но предоставляются два дополнительных
счётчика (12 и 13): общее время, затраченное на чтение и запись.  При
использовании аргумента histogram сообщается 14-й параметр, который представляет
гистограмму задержек.  Доступ ко всем этим счётчикам можно получить, отправив
сообщение @stats_print соответствующему DM-устройству через dmsetup.

Сообщаемые значения времени указываются в миллисекундах, а гранулярность
зависит от тиков ядра.  При использовании параметра precise_timestamps
сообщаемые значения времени указываются в наносекундах.

Каждая область имеет соответствующий уникальный идентификатор, который мы
называем region_id и который назначается при создании области.	 region_id
должен указываться при запросе статистики об области, удалении области и т. д.
Уникальные region_id позволяют нескольким программам пространства пользователя
запрашивать и обрабатывать статистику для одного и того же DM-устройства, не
мешая данным друг друга.

Создание статистики DM выделяет память через kmalloc либо, в качестве запасного
варианта, использует пространство vmalloc.  Статистикой DM может быть выделено
не более 1/4 всей системной памяти.  Администратор может узнать, сколько памяти
используется, прочитав:

	/sys/module/dm_mod/parameters/stats_current_allocated_bytes

Сообщения
=========

    @stats_create <range> <step> [<number_of_optional_arguments> <optional_arguments>...] [<program_id> [<aux_data>]]
	Создать новую область и вернуть region_id.

	<range>
	  "-"
		всё устройство
	  "<start_sector>+<length>"
		диапазон из <length> 512-байтовых секторов,
		начинающийся с <start_sector>.

	<step>
	  "<area_size>"
		диапазон делится на области, каждая из которых содержит
		<area_size> секторов.
	  "/<number_of_areas>"
		диапазон делится на указанное
		число областей.

	<number_of_optional_arguments>
	  Число необязательных аргументов

	<optional_arguments>
	  Поддерживаются следующие необязательные аргументы:

	  precise_timestamps
		использовать точный таймер с наносекундным разрешением
		вместо переменной "jiffies".  При использовании этого
		аргумента итоговые значения времени указываются в наносекундах
		вместо миллисекунд.  Точные временные метки получать
		немного медленнее, чем временные метки на основе jiffies.
	  histogram:n1,n2,n3,n4,...
		собирать гистограмму задержек.  Числа
		n1, n2 и т. д. — это значения времени, представляющие границы
		гистограммы.  Если precise_timestamps не используется, значения
		времени указываются в миллисекундах, иначе — в
		наносекундах.  Для каждого диапазона ядро будет сообщать
		число запросов, завершившихся в пределах этого диапазона. Например,
		если мы используем "histogram:10,20,30", ядро сообщит
		четыре числа a:b:c:d. a — это число запросов,
		на завершение которых потребовалось 0-10 мс, b — число запросов,
		на завершение которых потребовалось 10-20 мс, c — число запросов,
		на завершение которых потребовалось 20-30 мс, и d — число
		запросов, на завершение которых потребовалось более 30 мс.

	<program_id>
	  Необязательный параметр.  Имя, однозначно идентифицирующее
	  владельца диапазона в пространстве пользователя.  Оно группирует диапазоны вместе,
	  так что программы пространства пользователя могут идентифицировать созданные ими
	  диапазоны и игнорировать созданные другими.
	  Ядро возвращает эту строку в выводе
	  сообщения @stats_list, но больше ни для чего её не использует.
	  Если мы опускаем число необязательных аргументов, program id не должен
	  быть числом, иначе он будет интерпретирован как число
	  необязательных аргументов.

	<aux_data>
	  Необязательный параметр.  Слово, предоставляющее вспомогательные данные,
	  полезные клиентской программе, создавшей диапазон.
	  Ядро возвращает эту строку в выводе
	  сообщения @stats_list, но никак не использует это значение.

    @stats_delete <region_id>
	Удалить область с указанным id.

	<region_id>
	  region_id, возвращённый из @stats_create

    @stats_clear <region_id>
	Сбросить все счётчики, кроме счётчиков незавершённого ввода-вывода (in-flight).

	<region_id>
	  region_id, возвращённый из @stats_create

    @stats_list [<program_id>]
	Перечислить все области, зарегистрированные с помощью @stats_create.

	<program_id>
	  Необязательный параметр.
	  Если этот параметр указан, возвращаются только совпадающие
	  области.
	  Если он не указан, возвращаются все области.

	Формат вывода:
	  <region_id>: <start_sector>+<length> <step> <program_id> <aux_data>
	        precise_timestamps histogram:n1,n2,n3,...

	Строки "precise_timestamps" и "histogram" выводятся только в том случае,
	если они были указаны при создании области.

    @stats_print <region_id> [<starting_line> <number_of_lines>]
	Вывести счётчики для каждой области размером в шаг в составе области.

	<region_id>
	  region_id, возвращённый из @stats_create

	<starting_line>
	  Индекс начальной строки в выводе.
	  Если опущен, возвращаются все строки.

	<number_of_lines>
	  Число строк, включаемых в вывод.
	  Если опущено, возвращаются все строки.

	Формат вывода для каждой области размером в шаг в составе области:

	  <start_sector>+<length>
		counters

	  Первые 11 счётчиков имеют то же значение, что и
	  `/sys/block/*/stat or /proc/diskstats`.

	  Подробности см. в Documentation/admin-guide/iostats.rst.

	  1. число завершённых операций чтения
	  2. число объединённых операций чтения
	  3. число прочитанных секторов
	  4. число миллисекунд, затраченных на чтение
	  5. число завершённых операций записи
	  6. число объединённых операций записи
	  7. число записанных секторов
	  8. число миллисекунд, затраченных на запись
	  9. число операций ввода-вывода, выполняемых в данный момент
	  10. число миллисекунд, затраченных на операции ввода-вывода
	  11. взвешенное число миллисекунд, затраченных на операции ввода-вывода

	  Дополнительные счётчики:

	  12. общее время, затраченное на чтение, в миллисекундах
	  13. общее время, затраченное на запись, в миллисекундах

    @stats_print_clear <region_id> [<starting_line> <number_of_lines>]
	Атомарно вывести, а затем сбросить все счётчики, кроме
	счётчиков незавершённого ввода-вывода (in-flight).	 Полезно, когда клиент, потребляющий
	статистику, не хочет потерять никакую статистику (ту, что была обновлена
	между выводом и сбросом).

	<region_id>
	  region_id, возвращённый из @stats_create

	<starting_line>
	  Индекс начальной строки в выводе.
	  Если опущен, все строки выводятся, а затем сбрасываются.

	<number_of_lines>
	  Число строк для обработки.
	  Если опущено, все строки выводятся, а затем сбрасываются.

    @stats_set_aux <region_id> <aux_data>
	Сохранить вспомогательные данные aux_data для указанной области.

	<region_id>
	  region_id, возвращённый из @stats_create

	<aux_data>
	  Строка, идентифицирующая данные, полезные клиентской
	  программе, создавшей диапазон.  Ядро возвращает эту
	  строку в выводе сообщения @stats_list, но
	  никак не использует это значение.

Примеры
=======

Разделить DM-устройство 'vol' на 100 частей и начать сбор
статистики по ним::

  dmsetup message vol 0 @stats_create - /100

Установить строку вспомогательных данных в "foo bar baz" (экранирование каждого
пробела также должно быть экранировано, иначе оболочка их поглотит)::

  dmsetup message vol 0 @stats_set_aux 0 foo\\ bar\\ baz

Перечислить статистику::

  dmsetup message vol 0 @stats_list

Вывести статистику::

  dmsetup message vol 0 @stats_print 0

Удалить статистику::

  dmsetup message vol 0 @stats_delete 0
