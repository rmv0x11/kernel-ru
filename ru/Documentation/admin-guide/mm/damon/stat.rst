===========================================================
Статистика результатов мониторинга доступа к данным
===========================================================

Data Access Monitoring Results Stat (DAMON_STAT) — это статический модуль
ядра, предназначенный для простого мониторинга шаблонов доступа. Он отслеживает
доступы ко всей физической памяти системы с помощью DAMON и предоставляет
упрощённую статистику результатов мониторинга доступа, а именно процентили
времени простоя и оценочную пропускную способность памяти.

.. _damon_stat_monitoring_accuracy_overhead:

Точность мониторинга и накладные расходы
========================================

DAMON_STAT использует :ref:`автоподстройку <damon_design_monitoring_intervals_autotuning>`
интервалов мониторинга, чтобы обеспечить высокую точность и минимальные
накладные расходы. Он автоматически подстраивает интервалы так, чтобы в каждом
снимке захватывалось 4 % наблюдаемых событий доступа, ограничивая при этом
итоговый интервал выборки минимумом в 5 миллисекунд и максимумом в 10 секунд.
На нескольких рабочих серверных системах это привело к потреблению всего лишь
0.x % времени одного CPU при захвате приемлемого качества шаблонов доступа.
Интервалы, полученные в результате подстройки, можно прочитать через
:ref:`параметр <damon_stat_aggr_interval_us>` ``aggr_interval_us``.

Интерфейс: параметры модуля
===========================

Чтобы использовать эту возможность, сначала следует убедиться, что система
работает на ядре, собранном с ``CONFIG_DAMON_STAT=y``. Эту возможность можно
включить по умолчанию во время сборки, установив ``CONFIG_DAMON_STAT_ENABLED_DEFAULT``
в истинное значение.

Чтобы позволить системным администраторам включать или отключать её при загрузке
и/или во время выполнения, а также читать результаты мониторинга, DAMON_STAT
предоставляет параметры модуля. Следующие разделы содержат описания этих
параметров.

enabled
-------

Включить или отключить DAMON_STAT.

Вы можете включить DAMON_STAT, установив значение этого параметра в ``Y``.
Установка значения ``N`` отключает DAMON_STAT. Значение по умолчанию задаётся
параметром конфигурации сборки ``CONFIG_DAMON_STAT_ENABLED_DEFAULT``.

.. _damon_stat_aggr_interval_us:

aggr_interval_us
----------------

Автоматически подстроенный интервал времени агрегации в микросекундах.

Пользователи могут прочитать интервал агрегации DAMON, используемый экземпляром
DAMON для DAMON_STAT. Он :ref:`автоматически подстраивается <damon_stat_monitoring_accuracy_overhead>`,
и поэтому значение изменяется динамически.

estimated_memory_bandwidth
--------------------------

Оценочное потребление пропускной способности памяти (байт в секунду) системой.

DAMON_STAT читает наблюдаемые события доступа на текущем снимке результатов
DAMON и преобразует их в оценку потребления пропускной способности памяти в
байтах в секунду. Полученная метрика предоставляется пользователю через этот
параметр только для чтения. Поскольку DAMON использует выборку, это лишь оценка
интенсивности доступа, а не точная пропускная способность памяти.

memory_idle_ms_percentiles
--------------------------

Процентили времени простоя (в миллисекундах) для каждого байта системы.

DAMON_STAT вычисляет, как долго каждый байт памяти не подвергался доступу
вплоть до настоящего момента (время простоя), на основе текущего снимка
результатов DAMON. Для областей с частотой доступа (nr_accesses) больше нуля
временем простоя каждого байта области становится длительность сохранения
текущего уровня частоты доступа, умноженная на ``-1``. Если область имеет
нулевую частоту доступа (nr_accesses), временем простоя каждого байта области
становится длительность сохранения нулевой частоты доступа (age). Затем
DAMON_STAT предоставляет процентили значений времени простоя через этот параметр
только для чтения. Чтение параметра возвращает 101 значение времени простоя в
миллисекундах, разделённое запятыми. Каждое значение представляет 0-й, 1-й, 2-й,
3-й, ..., 99-й и 100-й процентиль времени простоя.
