============
Первые шаги
============

Этот документ кратко описывает, как пользоваться DAMON, на примере её
стандартного инструмента для пространства пользователя.  Обратите внимание, что
для краткости здесь описана лишь часть возможностей.  Подробности см. в
документации (`doc
<https://github.com/damonitor/damo/blob/next/USAGE.md>`_) этого инструмента.


Предварительные требования
===========================

Ядро
----

Сначала необходимо убедиться, что система работает на ядре, собранном с
``CONFIG_DAMON_*=y``.


Инструмент пространства пользователя
------------------------------------

Для демонстрации мы будем использовать стандартный инструмент DAMON для
пространства пользователя, называемый DAMON Operator (DAMO).  Он доступен по
адресу https://github.com/damonitor/damo.  В приведённых ниже примерах
предполагается, что ``damo`` находится в вашем ``$PATH``.  Это, впрочем, не
обязательно.

Поскольку DAMO использует интерфейс sysfs (подробности см. в :doc:`usage`)
DAMON, необходимо убедиться, что :doc:`sysfs </filesystems/sysfs>`
смонтирована.


Моментальный снимок шаблонов доступа к данным
=============================================

Приведённые ниже команды показывают шаблон доступа к памяти программы в момент
её выполнения. ::

    $ git clone https://github.com/sjp38/masim; cd masim; make
    $ sudo damo start "./masim ./configs/stairs.cfg --quiet"
    $ sudo damo report access
    heatmap: 641111111000000000000000000000000000000000000000000000[...]33333333333333335557984444[...]7
    # min/max temperatures: -1,840,000,000, 370,010,000, column size: 3.925 MiB
    0   addr 86.182 TiB   size 8.000 KiB   access 0 %   age 14.900 s
    1   addr 86.182 TiB   size 8.000 KiB   access 60 %  age 0 ns
    2   addr 86.182 TiB   size 3.422 MiB   access 0 %   age 4.100 s
    3   addr 86.182 TiB   size 2.004 MiB   access 95 %  age 2.200 s
    4   addr 86.182 TiB   size 29.688 MiB  access 0 %   age 14.100 s
    5   addr 86.182 TiB   size 29.516 MiB  access 0 %   age 16.700 s
    6   addr 86.182 TiB   size 29.633 MiB  access 0 %   age 17.900 s
    7   addr 86.182 TiB   size 117.652 MiB access 0 %   age 18.400 s
    8   addr 126.990 TiB  size 62.332 MiB  access 0 %   age 9.500 s
    9   addr 126.990 TiB  size 13.980 MiB  access 0 %   age 5.200 s
    10  addr 126.990 TiB  size 9.539 MiB   access 100 % age 3.700 s
    11  addr 126.990 TiB  size 16.098 MiB  access 0 %   age 6.400 s
    12  addr 127.987 TiB  size 132.000 KiB access 0 %   age 2.900 s
    total size: 314.008 MiB
    $ sudo damo stop

Первая команда приведённого выше примера загружает и собирает искусственную
программу-генератор обращений к памяти под названием ``masim``.  Вторая команда
просит DAMO запустить программу с помощью заданной команды и заставляет DAMON
отслеживать вновь запущенный процесс.  Третья команда извлекает из DAMON текущий
моментальный снимок отслеживаемого шаблона доступа процесса и отображает шаблон
в удобочитаемом виде.

Первая строка вывода показывает относительную температуру (степень
востребованности) доступа к областям в формате однострочной тепловой карты.
Каждый столбец на тепловой карте представляет области одинакового размера в
отслеживаемом виртуальном адресном пространстве.  Положение столбца в строке и
число в столбце представляют относительное расположение и температуру доступа
области.  ``[...]`` означает не отображённые в память огромные области в
виртуальных адресных пространствах.  Вторая строка показывает дополнительную
информацию для лучшего понимания тепловой карты.

Каждая строка вывода, начиная с третьей, показывает, какой диапазон виртуальных
адресов (``addr XX size XX``) процесса как часто (``access XX %``) и как долго
(``age XX``) был задействован.  Например, одиннадцатая область размером ~9.5 MiB
наиболее часто запрашивается на протяжении последних 3.7 секунд.  Наконец,
четвёртая команда останавливает DAMON.

Обратите внимание, что DAMON может отслеживать не только виртуальные адресные
пространства, но и множество типов адресных пространств, включая физическое
адресное пространство.


Запись шаблонов доступа к данным
================================

Приведённые ниже команды записывают шаблоны доступа к памяти программы и
сохраняют результаты отслеживания в файл. ::

    $ ./masim ./configs/zigzag.cfg &
    $ sudo damo record -o damon.data $(pidof masim)

Эта строка команд снова запускает искусственную программу-генератор обращений к
памяти.  Генератор будет многократно обращаться к двум областям памяти размером
100 MiB по очереди.  Вы можете заменить это на свою реальную нагрузку.
Последняя строка просит ``damo`` записать шаблон доступа в файл ``damon.data``.

Управление памятью с учётом шаблонов доступа к данным
=====================================================

Приведённая ниже команда заставляет каждую область памяти размером >=4K, к
которой не было обращений в течение >=60 секунд в вашей нагрузке, быть
вытесненной в подкачку. ::

    $ sudo damo start --damos_access_rate 0 0 --damos_sz_region 4K max \
                      --damos_age 60s max --damos_action pageout \
                      --target_pid <pid of your workload>
