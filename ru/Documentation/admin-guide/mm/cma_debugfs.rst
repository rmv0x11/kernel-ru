=======================
Интерфейс CMA в debugfs
=======================

Интерфейс CMA в debugfs полезен для получения базовой информации о различных
областях CMA и для тестирования выделения/освобождения в каждой из областей.

Каждой области CMA соответствует каталог в <debugfs>/cma/, представленный
её именем CMA, как показано ниже:

	<debugfs>/cma/<cma_name>

Структура файлов, создаваемых в этом каталоге, такова:

 - [RO] base_pfn: Базовый PFN (Page Frame Number, номер страничного кадра)
        области CMA. Совпадает с ranges/0/base_pfn.
 - [RO] count: Объём памяти в области CMA.
 - [RO] order_per_bit: Порядок страниц, представленных одним битом.
 - [RO] bitmap: Битовая карта выделенных страниц в области.
        Совпадает с ranges/0/base_pfn.
 - [RO] ranges/N/base_pfn: Базовый PFN непрерывного диапазона N
        в области CMA.
 - [RO] ranges/N/bitmap: Битовая карта выделенных страниц в
        диапазоне N в области CMA.
 - [WO] alloc: Выделить N страниц из этой области CMA. Например::

	echo 5 > <debugfs>/cma/<cma_name>/alloc

попытается выделить 5 страниц из области 'cma_name'.

 - [WO] free: Освободить N страниц из этой области CMA, аналогично указанному выше.
