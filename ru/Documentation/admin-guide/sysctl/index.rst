Введение
========

Sysctl — это средство настройки некоторых аспектов ядра во время работы, а
каталог /proc/sys/ существует для того, чтобы вам даже не понадобились
специальные инструменты для этого!
На самом деле для использования этих средств настройки нужны всего четыре вещи:

- работающая система Linux
- доступ root
- здравый смысл (его особенно трудно раздобыть в наши дни)
- понимание того, что означают все эти значения

Как покажет быстрая команда 'ls /proc/sys', этот каталог состоит из нескольких
(зависящих от архитектуры?) подкаталогов. Каждый подкаталог в основном относится
к одной части ядра, поэтому вы можете выполнять настройку по частям или просто
заняться «тематической подкруткой».

Эта документация рассказывает о:

=============== ===============================================================
abi/		домены исполнения и персоналии (personalities)
<$ARCH>		настроечные параметры для различных архитектур CPU (например, csky, s390)
crypto/		<не документировано>
debug/		<не документировано>
dev/		сведения, специфичные для устройств (например, dev/cdrom/info)
fs/		конкретные файловые системы
		настройка filehandle, inode, dentry и квот
		binfmt_misc <Documentation/admin-guide/binfmt-misc.rst>
kernel/		глобальная информация о ядре / настройка
		разное
		некоторые специфичные для архитектуры параметры
		вопросы безопасности (LSM)
net/		вопросы сети, документацию см. в:
		<Documentation/networking/>
proc/		<пусто>
sunrpc/		SUN Remote Procedure Call (NFS)
user/		ограничения на пространство имён каждого пользователя
vm/		настройка управления памятью
		управление буферами и кэшем
xen/		<не документировано>
=============== ===============================================================

Это подкаталоги, которые есть в моей системе или которые были обнаружены при
поиске по исходному коду. В другой конфигурации подкаталогов может быть больше
или они могут быть другими. Если вы увидите другой каталог, я бы очень хотел об
этом узнать :-)
