1. Описание
-----------

Этот драйвер обеспечивает поддержку акселерометра, имеющегося в различных
ноутбуках Toshiba и официально называемого «Toshiba HDD Protection - Shock
Sensor», а также автоматически обнаруживает ноутбуки с этим устройством.
В Windows поставляемое Toshiba программное обеспечение следит за этим
устройством и обеспечивает автоматическую защиту HDD (отвод головок) при
резких перемещениях или сильных вибрациях, однако этот драйвер лишь
предоставляет уведомление через файл sysfs, чтобы инструменты или демоны
пространства пользователя (userspace) могли реагировать соответствующим
образом, а также предоставляет файл sysfs для установки желаемого уровня
защиты или чувствительности датчика.


2. Интерфейс
------------

Это устройство имеет 3 метода:

====	=====================================================================
_STA    Проверяет наличие устройства, возвращая ноль, если устройство
	отсутствует или не поддерживается.
PTLV    Устанавливает желаемый уровень защиты.
RSSS    Отключает интерфейс защиты HDD на несколько секунд,
	затем восстанавливает нормальную работу.
====	=====================================================================

Примечание:
  Наличие твердотельных накопителей (SSD) может приводить к сбою загрузки
  этого драйвера, поскольку у таких накопителей нет движущихся частей, и
  потому им не требуется какая-либо «защита», а также из-за сбоя при
  вычислении метода _STA, имеющегося у этого устройства.


3. Оси акселерометра
--------------------

Это устройство не сообщает о каких-либо осях, однако для запроса положения
датчика предоставляется пара вызовов HCI (Hardware Configuration Interface)
(0x6D и 0xA6) для получения такой информации, которые обрабатываются модулем
ядра toshiba_acpi начиная с версии ядра 3.15.


4. Поддерживаемые устройства
----------------------------

Этот драйвер привязывается к ACPI-устройству TOS620A, и поддерживается любой
ноутбук Toshiba с этим устройством при условии, что в нём присутствует
обычный HDD, а не только SSD, либо сочетание HDD и SSD.


5. Использование
----------------

Под /sys/devices/LNXSYSTM:00/LNXSYBUS:00/TOS620A:00/ имеются следующие файлы
sysfs:

================   ============================================================
protection_level   Файл protection_level доступен для чтения и записи и
		   позволяет пространству пользователя запрашивать текущий
		   уровень защиты, а также устанавливать желаемый уровень
		   защиты; доступные уровни защиты таковы::

		     ============   =======   ==========   ========
		     0 - Disabled   1 - Low   2 - Medium   3 - High
		     ============   =======   ==========   ========

reset_protection   Запись reset_protection доступна только для записи и
		   принимает в качестве параметра единственное значение «1»;
		   она используется для запуска сброса интерфейса защиты.
================   ============================================================
