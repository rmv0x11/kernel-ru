==============================
Дополнения для ноутбуков Asus
==============================

Версия 0.1

6 августа 2009 года

Corentin Chary <corentincj@iksaif.net>
http://acpi4asus.sf.net/

 Этот драйвер обеспечивает поддержку дополнительных возможностей ACPI-совместимых
 ноутбуков ASUS. Он также может поддерживать некоторые ноутбуки MEDION, JVC или
 VICTOR (например, MEDION 9675 или VICTOR XP7210). Он заставляет все дополнительные
 кнопки генерировать события ввода (как клавиатуры).

 На некоторых моделях он добавляет поддержку изменения яркости и вывода
 изображения, включения и выключения подсветки LCD и, что самое важное, позволяет
 мигать теми эффектными светодиодами, которые предназначены для индикации почты и
 состояния беспроводной связи.

Этот драйвер заменяет старый драйвер asus_acpi.

Требования
----------

  Исходный код ядра 2.6.X, настроенный для вашего компьютера, с поддержкой ACPI.
  Также вам понадобятся CONFIG_INPUT и CONFIG_ACPI.

Состояние
---------

 В настоящее время поддерживаются следующие возможности (подробное описание см.
 ниже):

 - Комбинации с клавишей Fn
 - Включение и отключение Bluetooth
 - Включение и отключение Wlan
 - Включение и отключение GPS
 - Переключение видеовывода
 - Включение и выключение датчика внешней освещённости (Ambient Light Sensor)
 - Управление светодиодами (LED)
 - Управление светодиодным дисплеем (LED Display)
 - Управление яркостью LCD
 - Включение и выключение LCD

 Таблица совместимости по моделям и возможностям поддерживается на веб-сайте
 http://acpi4asus.sf.net/.

Использование
-------------

  Попробуйте "modprobe asus-laptop". Проверьте свой dmesg (просто наберите dmesg).
  Вы должны увидеть несколько строк вроде этих :

      Asus Laptop Extras version 0.42
        - L2D model detected.

  Если это не тот вывод, который вы видите на своём ноутбуке, пришлите его (а также
  DSDT ноутбука) мне.

  Вот и всё, теперь все события, генерируемые горячими клавишами вашего ноутбука,
  должны сообщаться через события netlink. Это можно проверить с помощью
  "acpi_genl monitor" (часть проекта acpica).

  Горячие клавиши также сообщаются как клавиши ввода (как клавиатуры); проверить,
  какие клавиши поддерживаются, можно с помощью "xev" под X11.

  Информацию о версии вашей таблицы DSDT можно получить, прочитав запись
  /sys/devices/platform/asus-laptop/infos. Если у вас есть вопрос или вы хотите
  сообщить об ошибке, пожалуйста, включите вывод этой записи.

Светодиоды (LED)
----------------

  Вы можете изменять состояние светодиодов, записывая (echo) значения в
  `/sys/class/leds/asus/*/brightness`::

    echo 1 >  /sys/class/leds/asus::mail/brightness

  включит светодиод почты.

  Также можно узнать, включены они или выключены, прочитав их содержимое, и
  использовать триггеры ядра, такие как disk-activity или heartbeat.

Подсветка (Backlight)
---------------------

  Вы можете управлять питанием и яркостью подсветки lcd с помощью
  /sys/class/backlight/asus-laptop/. Значения яркости находятся в диапазоне от 0
  до 15.

Беспроводные устройства
-----------------------

  Вы можете включать и выключать встроенный адаптер Bluetooth с помощью записи
  bluetooth (только на моделях с Bluetooth). Обычно это управляет связанным с ним
  светодиодом. То же самое касается адаптера Wlan.

Переключение вывода изображения
-------------------------------

  Примечание: код переключения вывода изображения в настоящее время считается
  ЭКСПЕРИМЕНТАЛЬНЫМ.

  Переключение работает для следующих моделей:

    - L3800C
    - A2500H
    - L5800C
    - M5200N
    - W1000N (хотя и с некоторыми сбоями)
    - M6700R
    - A6JC
    - F3J

  Переключение не работает для следующих:

    - M3700N
    - L2X00D (при определённых условиях блокирует ноутбук)

  Чтобы переключить вывод изображения, записывайте (echo) значения от 0 до 15 в
  /sys/devices/platform/asus-laptop/display. Значимость этих величин следующая:

  +-------+-----+-----+-----+-----+-----+
  | Bin   | Val | DVI | TV  | CRT | LCD |
  +-------+-----+-----+-----+-----+-----+
  | 0000  |   0 |     |     |     |     |
  +-------+-----+-----+-----+-----+-----+
  | 0001  |   1 |     |     |     |  X  |
  +-------+-----+-----+-----+-----+-----+
  | 0010  |   2 |     |     |  X  |     |
  +-------+-----+-----+-----+-----+-----+
  | 0011  |   3 |     |     |  X  |  X  |
  +-------+-----+-----+-----+-----+-----+
  | 0100  |   4 |     |  X  |     |     |
  +-------+-----+-----+-----+-----+-----+
  | 0101  |   5 |     |  X  |     | X   |
  +-------+-----+-----+-----+-----+-----+
  | 0110  |   6 |     |  X  |  X  |     |
  +-------+-----+-----+-----+-----+-----+
  | 0111  |   7 |     |  X  |  X  |  X  |
  +-------+-----+-----+-----+-----+-----+
  | 1000  |   8 |  X  |     |     |     |
  +-------+-----+-----+-----+-----+-----+
  | 1001  |   9 |  X  |     |     |  X  |
  +-------+-----+-----+-----+-----+-----+
  | 1010  |  10 |  X  |     |  X  |     |
  +-------+-----+-----+-----+-----+-----+
  | 1011  |  11 |  X  |     |  X  |  X  |
  +-------+-----+-----+-----+-----+-----+
  | 1100  |  12 |  X  |  X  |     |     |
  +-------+-----+-----+-----+-----+-----+
  | 1101  |  13 |  X  |  X  |     |  X  |
  +-------+-----+-----+-----+-----+-----+
  | 1110  |  14 |  X  |  X  |  X  |     |
  +-------+-----+-----+-----+-----+-----+
  | 1111  |  15 |  X  |  X  |  X  |  X  |
  +-------+-----+-----+-----+-----+-----+

  В большинстве случаев для работы приведённых выше комбинаций должны быть
  подключены соответствующие устройства вывода. Возможно, потребуется
  инициализировать TV-Out во время загрузки.

  Отладка:

  1) Проверьте, что клавиша Fn+F8:

     a) не блокирует ноутбук (если блокирует, попробуйте загрузиться с noapic / nolapic)
     b) генерирует события (0x6n, где n — значение, соответствующее приведённой
        выше конфигурации)
     c) действительно работает

     Запишите значение disp для каждой конфигурации.
  2) Записывайте (echo) значения от 0 до 15 в /sys/devices/platform/asus-laptop/display.
     Записывайте полученное значение, отмечайте любые изменения. Если ничего не
     меняется, попробуйте более широкий диапазон, вплоть до 65535.
  3) Отправьте ЛЮБОЙ вывод (нужны как положительные, так и отрицательные отчёты,
     если только ваша машина уже не указана выше) в список рассылки acpi4asus-user.

  Примечание: на некоторых машинах (например, L3C) после загрузки модуля
  генерируются только события 0x6n, а реального переключения не происходит. В таком
  случае обычно помогает строка вроде::

    echo $((10#$arg-60)) > /sys/devices/platform/asus-laptop/display

  ($arg — это событие вида 0000006n, передаваемое в acpid).

  Примечание: в настоящее время нет надёжного способа считать состояние вывода
  изображения на моделях xxN (Centrino).

Светодиодный дисплей (LED display)
----------------------------------

  Некоторые модели, такие как W1N, имеют светодиодный дисплей, который можно
  использовать для отображения нескольких элементов информации.

  Светодиодный дисплей работает для следующих моделей:

    - W1000N
    - W1J

  Чтобы управлять светодиодным дисплеем, используйте следующее::

    echo 0x0T000DDD > /sys/devices/platform/asus-laptop/

  где T управляет дисплеем из 3 букв, а DDD — дисплеем из 3 цифр, согласно таблицам
  ниже::

         DDD (digits)
         000 to 999 = display digits
         AAA        = ---
         BBB to FFF = turn-off

         T  (type)
         0 = off
         1 = dvd
         2 = vcd
         3 = mp3
         4 = cd
         5 = tv
         6 = cpu
         7 = vol

  Например, "echo 0x01000001 >/sys/devices/platform/asus-laptop/ledd"
  отобразит "DVD001".

Параметры драйвера
------------------

 Параметры можно передавать драйверу asus-laptop, используя стандартный синтаксис
 аргументов модуля (<param>=<value> при передаче параметра модулю или
 asus-laptop.<param>=<value> в строке загрузки ядра, когда asus-laptop статически
 скомпонован с ядром).

	     wapf: WAPF определяет поведение клавиши wlan Fn+Fx
		   Значимость значений ещё предстоит выяснить, но в
		   большинстве случаев:

		   - 0x0 не должно ничего делать
		   - 0x1 должно позволять управлять устройством клавишей Fn+Fx.
		   - 0x4 должно отправлять событие ACPI (0x88) при нажатии клавиши Fn+Fx
		   - 0x5 как 0x1 или 0x4

 Значение по умолчанию — 0x1.

Неподдерживаемые модели
-----------------------

 Эти модели никогда не будут поддерживаться этим модулем, поскольку они используют
 совершенно другой механизм для управления светодиодами и дополнительными функциями
 (то есть мы понятия не имеем, как это работает):

 - ASUS A1300 (A1B), A1370D
 - ASUS L7300G
 - ASUS L8400
