SRBDS - Special Register Buffer Data Sampling
=============================================

SRBDS — это аппаратная уязвимость, которая позволяет с помощью методов MDS
Documentation/admin-guide/hw-vuln/mds.rst делать выводы о значениях,
возвращаемых при обращениях к специальным регистрам. Обращения к специальным
регистрам — это обращения к регистрам вне ядра процессора (off core). Согласно
оценке Intel, чтения специальных регистров, для которых ожидается соблюдение
конфиденциальности, — это RDRAND, RDSEED и SGX EGETKEY.

Когда используются инструкции RDRAND, RDSEED и EGETKEY, данные перемещаются
в ядро процессора через механизм специальных регистров, подверженный
атакам MDS.

Затронутые процессоры
---------------------
Модели Core (desktop, mobile, Xeon-E3), реализующие RDRAND и/или RDSEED, могут
быть затронуты.

Процессор затронут уязвимостью SRBDS, если его Family_Model и stepping есть
в приведённом ниже списке, за исключением перечисленных процессоров,
экспортирующих MDS_NO при доступном, но не включённом Intel TSX. Последний
класс процессоров затронут только при включении Intel TSX программно с помощью
TSX_CTRL_MSR; в противном случае они не затронуты.

  =============  ============  ========
  common name    Family_Model  Stepping
  =============  ============  ========
  IvyBridge      06_3AH        All

  Haswell        06_3CH        All
  Haswell_L      06_45H        All
  Haswell_G      06_46H        All

  Broadwell_G    06_47H        All
  Broadwell      06_3DH        All

  Skylake_L      06_4EH        All
  Skylake        06_5EH        All

  Kabylake_L     06_8EH        <= 0xC
  Kabylake       06_9EH        <= 0xD
  =============  ============  ========

Связанные CVE
-------------

Со следующей записью CVE связана данная проблема SRBDS:

    ==============  =====  =====================================
    CVE-2020-0543   SRBDS  Special Register Buffer Data Sampling
    ==============  =====  =====================================

Сценарии атаки
--------------
Непривилегированный пользователь может извлечь значения, возвращаемые
инструкциями RDRAND и RDSEED, выполненными на другом ядре или соседнем потоке,
используя методы MDS.


Механизм смягчения
------------------
Intel выпустит обновления микрокода, которые изменяют инструкции RDRAND, RDSEED
и EGETKEY так, чтобы перезаписывать секретные данные специальных регистров
в общем промежуточном (staging) буфере до того, как к этим секретным данным
сможет обратиться другой логический процессор.

Во время выполнения инструкций RDRAND, RDSEED или EGETKEY обращения вне ядра
(off-core) от других логических процессоров будут задержаны до завершения
чтения специального регистра и перезаписи секретных данных в общем
промежуточном буфере.

Это оказывает три эффекта на производительность:

#. Инструкции RDRAND, RDSEED или EGETKEY имеют более высокую задержку (latency).

#. Одновременное выполнение RDRAND на нескольких логических процессорах будет
   сериализовано, что приведёт к общему снижению максимальной пропускной
   способности RDRAND.

#. Выполнение RDRAND, RDSEED или EGETKEY будет задерживать обращения к памяти
   от других логических процессоров, которые промахиваются мимо своих
   кэшей ядра, с эффектом, аналогичным устаревшим заблокированным обращениям
   с разделением кэш-линии (cache-line-split).

Обновления микрокода предоставляют механизм отказа (opt-out) (RNGDS_MITG_DIS),
позволяющий отключить смягчение для инструкций RDRAND и RDSEED, выполняемых вне
анклавов Intel Software Guard Extensions (Intel SGX). На логических процессорах,
которые отключают смягчение с помощью этого механизма отказа, RDRAND и RDSEED
не выполняются дольше и не влияют на производительность обращений к памяти
соседних логических процессоров. Механизм отказа не влияет на анклавы Intel SGX
(включая выполнение RDRAND или RDSEED внутри анклава, а также выполнение
EGETKEY).

Описание MSR IA32_MCU_OPT_CTRL
------------------------------
Наряду со смягчением этой проблемы Intel добавила новый MSR IA32_MCU_OPT_CTRL
с областью видимости на уровне потока (адрес 0x123). Наличие этого MSR и
RNGDS_MITG_DIS (бит 0) определяется через CPUID.(EAX=07H,ECX=0).EDX[SRBDS_CTRL =
9]==1. Этот MSR вводится через обновление микрокода.

Установка IA32_MCU_OPT_CTRL[0] (RNGDS_MITG_DIS) в 1 для логического процессора
отключает смягчение для RDRAND и RDSEED, выполняемых вне анклава Intel SGX
на этом логическом процессоре. Отказ от смягчения для конкретного логического
процессора не влияет на смягчения RDRAND и RDSEED для других логических
процессоров.

Обратите внимание, что внутри анклава Intel SGX смягчение применяется
независимо от значения RNGDS_MITG_DS.

Управление смягчением через командную строку ядра
-------------------------------------------------
Командная строка ядра позволяет управлять смягчением SRBDS во время загрузки
с помощью параметра "srbds=". Доступный параметр:

  ============= =============================================================
  off           Этот параметр отключает смягчение SRBDS для RDRAND и RDSEED на
                затронутых платформах.
  ============= =============================================================

Системная информация SRBDS
---------------------------
Ядро Linux предоставляет информацию о статусе уязвимости через sysfs. Для
SRBDS она доступна через следующий файл sysfs:
/sys/devices/system/cpu/vulnerabilities/srbds

Возможные значения, содержащиеся в этом файле:

 ============================== =============================================
 Not affected                   Процессор не уязвим
 Vulnerable                     Процессор уязвим, смягчение отключено
 Vulnerable: No microcode       Процессор уязвим, и в микрокоде отсутствует
                                смягчение
 Mitigation: Microcode          Процессор уязвим, и смягчение действует.
 Mitigation: TSX disabled       Процессор уязвим только при включённом TSX,
                                тогда как данная система была загружена с
                                отключённым TSX.
 Unknown: Dependent on
 hypervisor status              Выполнение на процессоре виртуального гостя,
                                который затронут, но без возможности узнать,
                                смягчён или уязвим процессор хоста.
 ============================== =============================================

Смягчение SRBDS по умолчанию
----------------------------
Этот новый микрокод сериализует доступ процессора во время выполнения RDRAND,
RDSEED и обеспечивает перезапись общего буфера до того, как он будет
освобождён для повторного использования. Используйте командную строку ядра
"srbds=off", чтобы отключить смягчение для RDRAND и RDSEED.
