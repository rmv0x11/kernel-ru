==================================
Register File Data Sampling (RFDS)
==================================

Register File Data Sampling (RFDS) — это микроархитектурная уязвимость,
которая затрагивает только процессоры Intel Atom (также продаваемые под
маркой E-cores). RFDS может позволить злоумышленнику получить значения данных,
ранее использовавшихся в регистрах с плавающей точкой, векторных регистрах
или целочисленных регистрах. RFDS не даёт возможности выбирать, какие именно
данные будут получены. Уязвимости RFDS присвоен идентификатор CVE-2023-28746.

Смягчение
=========
Intel выпустила обновление микрокода, которое позволяет программному
обеспечению очищать чувствительную информацию с помощью инструкции VERW.
Как и в случае MDS, для RFDS применяется та же стратегия смягчения: заставить
CPU очистить затронутые буферы до того, как атакующий сможет извлечь секреты.
Это достигается за счёт использования в остальном неиспользуемой и устаревшей
инструкции VERW в сочетании с обновлением микрокода. Микрокод очищает
затронутые буферы CPU при выполнении инструкции VERW.

Точки смягчения
---------------
VERW выполняется ядром перед возвратом в пространство пользователя и средой KVM
перед VMentry. Ни одно из затронутых ядер не поддерживает SMT, поэтому VERW
не требуется при переходах между C-состояниями.

Новые биты в IA32_ARCH_CAPABILITIES
-----------------------------------
Более новые процессоры и обновление микрокода на уже существующих затронутых
процессорах добавили новые биты в MSR IA32_ARCH_CAPABILITIES. Эти биты можно
использовать для перечисления уязвимости и возможностей смягчения:

- Бит 27 — RFDS_NO — если установлен, процессор не подвержен RFDS.
- Бит 28 — RFDS_CLEAR — если установлен, процессор подвержен RFDS и имеет
  микрокод, который очищает затронутые буферы при выполнении VERW.

Управление смягчением через командную строку ядра
-------------------------------------------------
Командная строка ядра позволяет управлять смягчением RFDS во время загрузки
с помощью параметра "reg_file_data_sampling=". Допустимые аргументы:

  ==========  =================================================================
  on          Если CPU уязвим, включить смягчение; очистка буферов CPU при
              выходе в пространство пользователя и перед входом в виртуальную
              машину.
  off         Отключает смягчение.
  ==========  =================================================================

Значение смягчения по умолчанию выбирается параметром CONFIG_MITIGATION_RFDS.

Информация о статусе смягчения
------------------------------
Ядро Linux предоставляет интерфейс sysfs для перечисления текущего статуса
уязвимости системы: уязвима ли система и какие смягчения активны.
Соответствующий файл sysfs:

	/sys/devices/system/cpu/vulnerabilities/reg_file_data_sampling

Возможные значения в этом файле:

  .. list-table::

     * - 'Not affected'
       - Процессор не уязвим
     * - 'Vulnerable'
       - Процессор уязвим, но смягчение не включено
     * - 'Vulnerable: No microcode'
       - Процессор уязвим, но микрокод не обновлён.
     * - 'Mitigation: Clear Register File'
       - Процессор уязвим, и смягчение с очисткой буфера CPU включено.
