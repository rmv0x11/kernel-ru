Параметры командной строки ядра
===============================

Ниже приведён сводный список параметров ядра, реализованных макросами
__setup(), early_param(), core_param() и module_param() и отсортированных
в порядке английского словаря (определяемом как игнорирование всех знаков
препинания и сортировка цифр перед буквами без учёта регистра), с описаниями
там, где они известны.

Ядро разбирает параметры из командной строки ядра вплоть до «``--``»; если
параметр не распознан и не содержит «.», он передаётся init: параметры с «=»
попадают в окружение init, остальные передаются init как аргументы командной
строки. Всё, что идёт после «``--``», передаётся init как аргумент.

Параметры модулей можно задать двумя способами: через командную строку ядра
с префиксом в виде имени модуля либо через modprobe, например::

	(командная строка ядра) usbcore.blinkenlights=1
	(командная строка modprobe) modprobe usbcore blinkenlights=1

Параметры для модулей, встроенных в ядро, должны задаваться в командной строке
ядра.  modprobe просматривает командную строку ядра (/proc/cmdline) и собирает
параметры модуля при его загрузке, поэтому командную строку ядра можно
использовать и для загружаемых модулей.

Этот документ может быть не вполне актуальным и полным. Команда
"modinfo -p ${modulename}" показывает текущий список всех параметров загружаемого
модуля. Загружаемые модули после загрузки в работающее ядро также раскрывают свои
параметры в /sys/module/${modulename}/parameters/. Некоторые из этих параметров
можно изменить во время работы командой
``echo -n ${value} > /sys/module/${modulename}/parameters/${parm}``.

Особая обработка
----------------

Дефисы (тире) и подчёркивания эквивалентны в именах параметров, поэтому::

	log_buf_len=1M print-fatal-signals=1

можно также записать как::

	log-buf-len=1M print_fatal_signals=1

Двойные кавычки можно использовать для защиты пробелов в значениях, например::

	param="spaces in here"

списки CPU
~~~~~~~~~~

Некоторые параметры ядра принимают в качестве значения список CPU, например
isolcpus, nohz_full, irqaffinity, rcu_nocbs.  Формат этого списка таков:

	<cpu number>,...,<cpu number>

или

	<cpu number>-<cpu number>
	(должен быть положительным диапазоном в возрастающем порядке)

или их сочетание

<cpu number>,...,<cpu number>-<cpu number>

Обратите внимание, что в особом случае диапазона его можно разбить на группы
равного размера и для каждой группы использовать некоторое количество CPU
с начала этой группы:

	<cpu number>-<cpu number>:<used size>/<group size>

Например, в командную строку можно добавить следующий параметр:

	isolcpus=1,2,10-20,100-2000:2/25

где последний элемент представляет CPU 100,101,125,126,150,151,...

Значение "N" можно использовать для обозначения численно последнего CPU в
системе, то есть "foo_cpus=16-N" будет эквивалентно "16-31" в системе с 32 ядрами.

Имейте в виду, что "N" динамично, поэтому если изменения в системе приводят
к изменению ширины битовой карты, например меньшее число ядер в списке CPU, то N
и любые диапазоны, использующие N, также изменятся.  Используйте то же самое
в небольшой системе с 4 ядрами, и "16-N" станет "16-3", и теперь те же входные
данные при загрузке будут помечены как недопустимые (начало > конца).

Особое нечувствительное к регистру имя группы "all" означает выбор всех CPU,
так что "nohz_full=all" эквивалентно "nohz_full=0-N".

Семантика "N" и "all" поддерживается на уровне битовых карт и действует для всех
пользователей bitmap_parselist().

Метрические суффиксы
~~~~~~~~~~~~~~~~~~~~~

Суффикс [KMG] обычно указывается после числовых значений параметров ядра.
Допустимы суффиксы 'K', 'M', 'G', 'T', 'P' и 'E'. Эти буквы представляют
_двоичные_ множители 'Kilo', 'Mega', 'Giga', 'Tera', 'Peta' и 'Exa', равные
соответственно 2^10, 2^20, 2^30, 2^40, 2^50 и 2^60 байт. Такие буквенные
суффиксы также можно полностью опускать.

Параметры сборки ядра
---------------------

Перечисленные ниже параметры действительны, только если включены определённые
параметры сборки ядра и присутствует соответствующее оборудование. Этот список
следует держать в алфавитном порядке. Текст в квадратных скобках в начале каждого
описания указывает ограничения, в рамках которых применим параметр.

Параметры, помеченные BOOT, фактически интерпретируются загрузчиком (boot loader)
и не имеют прямого значения для ядра.
Не изменяйте синтаксис параметров загрузчика без крайней необходимости или
согласования с <Documentation/arch/x86/boot.rst>.

Существуют также параметры ядра, специфичные для конкретных архитектур, которые
здесь не задокументированы.

Обратите внимание, что ВСЕ перечисленные ниже параметры ядра ЧУВСТВИТЕЛЬНЫ
К РЕГИСТРУ и что завершающий = в имени любого параметра означает, что параметр
будет введён как переменная окружения, тогда как его отсутствие указывает, что
он появится как аргумент ядра, читаемый через /proc/cmdline программами,
работающими после запуска системы.

Число параметров ядра не ограничено, но длина полной командной строки (параметры
вместе с пробелами и т. п.) ограничена фиксированным числом символов. Этот предел
зависит от архитектуры и составляет от 256 до 4096 символов. Он определён в файле
./include/uapi/asm-generic/setup.h как COMMAND_LINE_SIZE.
