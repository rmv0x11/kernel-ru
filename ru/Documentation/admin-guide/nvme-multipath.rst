==============================
Многопутевость NVMe в Linux
==============================

Этот документ описывает многопутевость (multipath) NVMe и поддерживаемые ею
политики выбора пути в host-драйвере NVMe для Linux.


Введение
========

Функциональность многопутевости NVMe в Linux объединяет пространства имён
(namespaces) с одинаковым идентификатором в единое блочное устройство.
Использование многопутевости повышает надёжность и стабильность доступа по
вводу-выводу, одновременно улучшая показатели пропускной способности. Когда
пользователь отправляет ввод-вывод на это объединённое блочное устройство,
механизм многопутевости выбирает одно из нижележащих блочных устройств (путей)
в соответствии с настроенной политикой. Разные политики приводят к разным
выборам пути.


Политики
========

Все политики следуют механизму ANA (Asymmetric Namespace Access), то есть когда
доступен оптимизированный путь, он будет выбран вместо неоптимизированного.
На текущий момент политики многопутевости NVMe включают numa (по умолчанию),
round-robin и queue-depth.

Чтобы задать нужную политику (например, round-robin), используйте один из
следующих способов:

   1. echo -n "round-robin" > /sys/module/nvme_core/parameters/iopolicy
   2. или добавьте "nvme_core.iopolicy=round-robin" в командную строку (cmdline).


NUMA
----

Политика NUMA выбирает путь, ближайший к NUMA-узлу текущего CPU, для
распределения ввода-вывода. Эта политика поддерживает ближайшие пути к каждому
NUMA-узлу на основе подключений сетевых интерфейсов.

Когда использовать политику NUMA:
  1. Многоядерные системы: оптимизирует доступ к памяти в многоядерных и
     многопроцессорных системах, особенно при архитектуре NUMA.
  2. Нагрузки с высокой аффинностью: привязывает обработку ввода-вывода к CPU,
     чтобы сократить задержки обмена данными между узлами.


Round-Robin
-----------

Политика round-robin равномерно распределяет запросы ввода-вывода по всем путям
для повышения пропускной способности и эффективности использования ресурсов.
Каждая операция ввода-вывода отправляется на следующий по очереди путь.

Когда использовать политику round-robin:
  1. Сбалансированные нагрузки: эффективна для сбалансированных и предсказуемых
     нагрузок со схожими размером и типом ввода-вывода.
  2. Однородная производительность путей: эффективно задействует все пути, когда
     характеристики производительности (например, задержка, пропускная
     способность) схожи.


Queue-Depth
-----------

Политика queue-depth управляет запросами ввода-вывода на основе текущей глубины
очереди каждого пути, выбирая путь с наименьшим числом незавершённых (in-flight)
операций ввода-вывода.

Когда использовать политику queue-depth:
  1. Высокая нагрузка с малыми операциями ввода-вывода: эффективно балансирует
     нагрузку по путям, когда нагрузка высока, а операции ввода-вывода состоят из
     небольших, относительно фиксированного размера запросов.
