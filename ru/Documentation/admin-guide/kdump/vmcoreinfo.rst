==========
VMCOREINFO
==========

Что это такое?
==============

VMCOREINFO — это специальная секция ELF-заметки (ELF note). Она содержит
различную информацию из ядра, такую как размеры структур, размер страницы,
значения символов, смещения полей и т. п. Эти данные упакованы в секцию
ELF-заметки и используются инструментами пространства пользователя, такими как
crash и makedumpfile, для анализа размещения памяти ядра.

Общие переменные
================

init_uts_ns.name.release
------------------------

Версия ядра Linux. Используется для поиска соответствующего исходного кода, из
которого было собрано ядро. Например, crash использует её для поиска
соответствующего vmlinux, чтобы обработать vmcore.

PAGE_SIZE
---------

Размер страницы. Это наименьшая единица данных, используемая средствами
управления памятью. Обычно её размер составляет 4096 байт, и страница
выравнивается по границе 4096 байт. Используется для вычисления адресов страниц.

init_uts_ns
-----------

Пространство имён UTS, которое используется для изоляции двух конкретных
элементов системы, относящихся к системному вызову uname(2). Оно названо по
структуре данных, используемой для хранения информации, возвращаемой системным
вызовом uname(2).

Инструменты пространства пользователя могут получить из него имя ядра, имя хоста,
номер релиза ядра, версию ядра, имя архитектуры и тип ОС.

(uts_namespace, name)
---------------------

Смещение члена name. Crash Utility и Makedumpfile получают из этого начальный
адрес init_uts_ns.name.

node_online_map
---------------

Массив node_states[N_ONLINE], представляющий набор активных узлов в системе, по
одной битовой позиции на номер узла. Используется для отслеживания того, какие
узлы присутствуют в системе и находятся в активном состоянии.

swapper_pg_dir
--------------

Глобальный указатель на каталог страниц ядра. Используется для трансляции
виртуальных адресов в физические.

_stext
------

Определяет начало секции text. В общем случае _stext указывает на начальный адрес
ядра. Используется для преобразования виртуального адреса из прямого отображения
ядра (direct kernel map) в физический адрес.

VMALLOC_START
-------------

Хранит базовый адрес области vmalloc. makedumpfile получает это значение,
поскольку оно необходимо для трансляции vmalloc.

mem_map
-------

Физические адреса транслируются в struct page путём интерпретации их как индекса
в массиве mem_map. Сдвиг физического адреса вправо на PAGE_SHIFT бит преобразует
его в номер фрейма страницы (page frame number), который является индексом в этом
массиве mem_map.

Используется для отображения адреса в соответствующую struct page.

contig_page_data
----------------

Makedumpfile получает из этого символа структуру pglist_data, которая
используется для описания размещения памяти.

Инструменты пространства пользователя используют это для исключения свободных
страниц при дампе памяти.

mem_section|(mem_section, NR_SECTION_ROOTS)|(mem_section, section_mem_map)
--------------------------------------------------------------------------

Адрес массива mem_section, его длина, размер структуры и смещение
section_mem_map.

Существует в модели разреженного отображения памяти (sparse memory mapping) и
также несколько похож на переменную mem_map; обе они используются для трансляции
адреса.

MAX_PHYSMEM_BITS
----------------

Определяет максимальный поддерживаемый объём памяти пространства физических
адресов.

page
----

Размер структуры page. struct page — важная структура данных, и она широко
используется для вычисления непрерывной памяти.

pglist_data
-----------

Размер структуры pglist_data. Это значение используется для проверки того,
является ли структура pglist_data корректной. Оно также используется для проверки
типа памяти.

zone
----

Размер структуры zone. Это значение используется для проверки того, была ли
найдена структура zone. Оно также используется для исключения свободных страниц.

free_area
---------

Размер структуры free_area. Указывает, корректна структура free_area или нет.
Полезно при исключении свободных страниц.

list_head
---------

Размер структуры list_head. Используется при обходе списков в сеансе
посмертного анализа (post-mortem analysis).

nodemask_t
----------

Размер типа nodemask_t. Используется для вычисления числа активных узлов.

(page, flags|_refcount|mapping|lru|_mapcount|private|compound_order|compound_head)
----------------------------------------------------------------------------------

Инструменты пространства пользователя вычисляют свои значения на основе смещения
этих переменных. Переменные используются при исключении ненужных страниц.

(pglist_data, node_zones|nr_zones|node_mem_map|node_start_pfn|node_spanned_pages|node_id)
-----------------------------------------------------------------------------------------

На NUMA-машинах каждый NUMA-узел имеет pg_data_t для описания размещения своей
памяти. На UMA-машинах существует единственная pglist_data, описывающая всю
память.

Эти значения используются для проверки типа памяти и для вычисления виртуального
адреса карты памяти (memory map).

(zone, free_area|vm_stat|spanned_pages)
---------------------------------------

Каждый узел разделён на некоторое число блоков, называемых зонами (zones),
которые представляют диапазоны в пределах памяти. Зона описывается структурой
zone.

Инструменты пространства пользователя вычисляют необходимые значения на основе
смещения этих переменных.

(free_area, free_list)
----------------------

Смещение члена free_list. Это значение используется для вычисления числа
свободных страниц.

Каждая зона имеет массив структур free_area, называемый free_area[NR_PAGE_ORDERS].
free_list представляет связный список блоков свободных страниц.

(list_head, next|prev)
----------------------

Смещения членов list_head. list_head используется для определения кольцевого
связного списка. Инструменты пространства пользователя нуждаются в них, чтобы
обходить списки.

(vmap_area, va_start|list)
--------------------------

Смещения членов vmap_area. Они несут информацию, специфичную для vmalloc.
Makedumpfile получает из этого начальный адрес области vmalloc.

(zone.free_area, NR_PAGE_ORDERS)
--------------------------------

Дескриптор свободных областей. Инструменты пространства пользователя используют
это значение для обхода диапазонов free_area. NR_PAGE_ORDERS используется
buddy-аллокатором зоны.

prb
---

Указатель на кольцевой буфер printk (struct printk_ringbuffer). Он может указывать
на статический загрузочный кольцевой буфер или на динамически выделенный кольцевой
буфер, в зависимости от того, когда произошёл дамп ядра. Используется
инструментами пространства пользователя для чтения активного буфера журнала ядра.

printk_rb_static
----------------

Указатель на статический загрузочный кольцевой буфер printk. Если @prb имеет иное
значение, это полезно для просмотра начальных загрузочных сообщений, которые могли
быть перезаписаны в динамически выделенном кольцевом буфере.

clear_seq
---------

Порядковый номер записи printk() после последней команды очистки. Указывает на
первую запись после последнего SYSLOG_ACTION_CLEAR, как, например, выдаваемого
командой 'dmesg -c'. Используется инструментами пространства пользователя для
дампа подмножества журнала dmesg.

printk_ringbuffer
-----------------

Размер структуры printk_ringbuffer. Эта структура содержит всю информацию,
необходимую для доступа к различным компонентам буфера журнала ядра.

(printk_ringbuffer, desc_ring|text_data_ring|dict_data_ring|fail)
-----------------------------------------------------------------

Смещения для различных компонентов кольцевого буфера printk. Используются
инструментами пространства пользователя для просмотра буфера журнала ядра без
необходимости объявления структуры.

prb_desc_ring
-------------

Размер структуры prb_desc_ring. Эта структура содержит информацию о наборе
дескрипторов записей.

(prb_desc_ring, count_bits|descs|head_id|tail_id)
-------------------------------------------------

Смещения для полей, описывающих набор дескрипторов записей. Используются
инструментами пространства пользователя, чтобы иметь возможность обходить
дескрипторы без необходимости объявления структуры.

prb_desc
--------

Размер структуры prb_desc. Эта структура содержит информацию об одном дескрипторе
записи.

(prb_desc, info|state_var|text_blk_lpos|dict_blk_lpos)
------------------------------------------------------

Смещения для полей, описывающих дескриптор записи. Используются инструментами
пространства пользователя, чтобы иметь возможность читать дескрипторы без
необходимости объявления структуры.

prb_data_blk_lpos
-----------------

Размер структуры prb_data_blk_lpos. Эта структура содержит информацию о том, где
расположены данные текста или словаря (блок данных) в соответствующем кольце
данных.

(prb_data_blk_lpos, begin|next)
-------------------------------

Смещения для полей, описывающих расположение блока данных. Используются
инструментами пространства пользователя, чтобы иметь возможность находить блоки
данных без необходимости объявления структуры.

printk_info
-----------

Размер структуры printk_info. Эта структура содержит все метаданные записи.

(printk_info, seq|ts_nsec|text_len|dict_len|caller_id)
------------------------------------------------------

Смещения для полей, предоставляющих метаданные записи. Используются инструментами
пространства пользователя, чтобы иметь возможность читать информацию без
необходимости объявления структуры.

prb_data_ring
-------------

Размер структуры prb_data_ring. Эта структура содержит информацию о наборе блоков
данных.

(prb_data_ring, size_bits|data|head_lpos|tail_lpos)
---------------------------------------------------

Смещения для полей, описывающих набор блоков данных. Используются инструментами
пространства пользователя, чтобы иметь возможность получать доступ к блокам данных
без необходимости объявления структуры.

atomic_long_t
-------------

Размер структуры atomic_long_t. Используется инструментами пространства
пользователя, чтобы иметь возможность скопировать всю структуру целиком,
независимо от её специфичной для архитектуры реализации.

(atomic_long_t, counter)
------------------------

Смещение для значения типа long переменной atomic_long_t. Используется
инструментами пространства пользователя для доступа к значению long без
необходимости специфичного для архитектуры объявления.

(free_area.free_list, MIGRATE_TYPES)
------------------------------------

Число типов миграции (migrate types) для страниц. free_list описывается этим
массивом. Используется инструментами для вычисления числа свободных страниц.

NR_FREE_PAGES
-------------

В linux-2.6.21 и более поздних версиях число свободных страниц находится в
vm_stat[NR_FREE_PAGES]. Используется для получения числа свободных страниц.

PG_lru|PG_private|PG_swapcache|PG_swapbacked|PG_hwpoison|PG_head_mask
--------------------------------------------------------------------------

Атрибуты страницы. Эти флаги используются для отфильтровывания различных
ненужных для дампа страниц.

PAGE_SLAB_MAPCOUNT_VALUE|PAGE_BUDDY_MAPCOUNT_VALUE|PAGE_OFFLINE_MAPCOUNT_VALUE|PAGE_HUGETLB_MAPCOUNT_VALUE|PAGE_UNACCEPTED_MAPCOUNT_VALUE
------------------------------------------------------------------------------------------------------------------------------------------

Дополнительные атрибуты страницы. Эти флаги используются для отфильтровывания
различных ненужных для дампа страниц.


x86_64
======

phys_base
---------

Используется для преобразования виртуального адреса экспортированного символа
ядра в соответствующий ему физический адрес.

init_top_pgt
------------

Используется для обхода всей таблицы страниц и преобразования виртуальных адресов
в физические. init_top_pgt несколько похож на swapper_pg_dir, но используется
только в x86_64.

pgtable_l5_enabled
------------------

Инструментам пространства пользователя необходимо знать, находилось ли crash-ядро
в режиме 5-уровневой страничной организации памяти.

node_data
---------

Это массив struct pglist_data, в котором хранится информация обо всех NUMA-узлах.
Makedumpfile получает из него структуру pglist_data.

(node_data, MAX_NUMNODES)
-------------------------

Максимальное число узлов в системе.

KERNELOFFSET
------------

Смещение рандомизации ядра. Используется для вычисления смещения страницы. Если
KASLR отключён, это значение равно нулю.

KERNEL_IMAGE_SIZE
-----------------

В настоящее время не используется Makedumpfile. Используется Crash для вычисления
виртуального адреса модуля.

sme_mask
--------

Специфично для AMD с поддержкой SME: указывает маску безопасного шифрования
памяти (secure memory encryption). Инструментам makedumpfile необходимо знать,
было ли crash-ядро зашифровано. Если SME включено в первом ядре, записи таблицы
страниц crash-ядра (pgd/pud/pmd/pte) содержат маску шифрования памяти. Это
используется для удаления SME-маски и получения истинного физического адреса.

В настоящее время sme_mask хранит значение позиции C-бита. При необходимости в эту
переменную может быть помещена дополнительная информация, относящаяся к SME.

Например::

  [ misc	        ][ enc bit  ][ other misc SME info       ]
  0000_0000_0000_0000_1000_0000_0000_0000_0000_0000_..._0000
  63   59   55   51   47   43   39   35   31   27   ... 3

x86_32
======

X86_PAE
-------

Обозначает, включены ли расширения физических адресов (physical address
extensions). Это даёт накладные расходы в виде более затратного поиска по таблице
страниц, а также потребляет больше пространства таблицы страниц на процесс.
Используется для проверки того, было ли PAE включено в crash-ядре при
преобразовании виртуальных адресов в физические.

ARM64
=====

VA_BITS
-------

Максимальное число бит для виртуальных адресов. Используется для вычисления
диапазонов виртуальной памяти.

kimage_voffset
--------------

Смещение между виртуальным и физическим отображениями ядра. Используется для
трансляции виртуальных адресов в физические.

PHYS_OFFSET
-----------

Указывает физический адрес начала памяти. Аналогично kimage_voffset, который
используется для трансляции виртуальных адресов в физические.

KERNELOFFSET
------------

Смещение рандомизации ядра. Используется для вычисления смещения страницы. Если
KASLR отключён, это значение равно нулю.

KERNELPACMASK
-------------

Маска для извлечения кода аутентификации указателя (Pointer Authentication Code)
из виртуального адреса ядра.

TCR_EL1.T1SZ
------------

Указывает смещение размера области памяти, адресуемой через TTBR1_EL1. Размер
области составляет 2^(64-T1SZ) байт.

TTBR1_EL1 — это регистр базового адреса таблицы (table base address register),
определённый архитектурой ARMv8-A, который используется для поиска по таблицам
страниц для виртуальных адресов в верхнем диапазоне VA (за подробностями
обращайтесь к документу ARMv8 ARM).

MODULES_VADDR|MODULES_END|VMALLOC_START|VMALLOC_END|VMEMMAP_START|VMEMMAP_END
-----------------------------------------------------------------------------

Используются для получения корректных диапазонов:
	MODULES_VADDR ~ MODULES_END-1 : пространство модулей ядра.
	VMALLOC_START ~ VMALLOC_END-1 : пространство vmalloc() / ioremap().
	VMEMMAP_START ~ VMEMMAP_END-1 : область vmemmap, используемая для массива struct page.

arm

ARM_LPAE
--------

Указывает, поддерживает ли crash-ядро расширения больших физических адресов
(large physical address extensions). Используется для трансляции виртуальных
адресов в физические.

high_memory
-----------

Используется для получения адреса vmalloc_start из символа high_memory.

(lowcore_ptr, NR_CPUS)

powerpc
=======


node_data|(node_data, MAX_NUMNODES)
-----------------------------------

См. выше.

contig_page_data
----------------

См. выше.

vmemmap_list
------------

vmemmap_list поддерживает всё физическое отображение vmemmap. Используется для
получения числа элементов списка vmemmap и информации о заполненных областях
vmemmap. Если информация о трансляции адресов vmemmap хранится в crash-ядре, она
используется для трансляции виртуальных адресов ядра vmemmap.

mmu_vmemmap_psize
-----------------

Размер страницы. Используется для трансляции виртуальных адресов в физические.

mmu_psize_defs
--------------

Определения размеров страниц, т. е. 4k, 64k или 16M.

Используются для выполнения трансляций vtop.

vmemmap_backing|(vmemmap_backing, list)|(vmemmap_backing, phys)|(vmemmap_backing, virt_addr)
--------------------------------------------------------------------------------------------

Управление пространством виртуальных адресов vmemmap не имеет традиционной таблицы
страниц для отслеживания того, какие виртуальные struct page подкреплены
физическим отображением. Отображения виртуальных адресов в физические
отслеживаются в формате простого связного списка.

Инструментам пространства пользователя необходимо знать смещения list, phys и
virt_addr при вычислении числа областей vmemmap.

mmu_psize_def|(mmu_psize_def, shift)
------------------------------------

Размер struct mmu_psize_def и смещение члена mmu_psize_def.

Используются в трансляциях vtop.

RISCV64
=======

VA_BITS
-------

Максимальное число бит для виртуальных адресов. Используется для вычисления
диапазонов виртуальной памяти.

PAGE_OFFSET
-----------

Указывает виртуальный начальный адрес ядра для области прямо отображаемой
(direct-mapped) оперативной памяти.

phys_ram_base
-------------

Указывает начальный физический адрес оперативной памяти.

MODULES_VADDR|MODULES_END|VMALLOC_START|VMALLOC_END|VMEMMAP_START|VMEMMAP_END|KERNEL_LINK_ADDR
----------------------------------------------------------------------------------------------

Используются для получения корректных диапазонов:

  * MODULES_VADDR ~ MODULES_END : пространство модулей ядра.
  * VMALLOC_START ~ VMALLOC_END : пространство vmalloc() / ioremap().
  * VMEMMAP_START ~ VMEMMAP_END : пространство vmemmap, используемое для массива struct page.
  * KERNEL_LINK_ADDR : начальный адрес компоновки ядра (Kernel link) и BPF

va_kernel_pa_offset
-------------------

Указывает смещение между виртуальным и физическим отображениями ядра. Используется
для трансляции виртуальных адресов в физические.
