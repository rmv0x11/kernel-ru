Обзор
-----

Plug and Play предоставляет средство обнаружения и настройки ресурсов для
устаревших или иным образом ненастраиваемых устройств. Слой Linux Plug and Play
предоставляет эти службы совместимым драйверам.


Пользовательский интерфейс
--------------------------

Пользовательский интерфейс Linux Plug and Play предоставляет средство активации
PnP-устройств для устаревших драйверов и драйверов уровня пользователя, которые
не поддерживают Linux Plug and Play. Пользовательский интерфейс интегрирован в
sysfs.

Помимо стандартного файла sysfs, в каталоге каждого устройства создаются
следующие:
- id - отображает список поддерживаемых EISA ID
- options - отображает возможные конфигурации ресурсов
- resources - отображает выделенные в данный момент ресурсы и позволяет изменять ресурсы

активация устройства
^^^^^^^^^^^^^^^^^^^^^

::

	# echo "auto" > resources

это вызовет систему автоматической настройки ресурсов для активации устройства

ручная активация устройства
^^^^^^^^^^^^^^^^^^^^^^^^^^^^

::

	# echo "manual <depnum> <mode>" > resources

	<depnum> - the configuration number
	<mode> - static or dynamic
		 static = for next boot
		 dynamic = now

отключение устройства
^^^^^^^^^^^^^^^^^^^^^^

::

	# echo "disable" > resources


ПРИМЕР:

Предположим, что вам нужно активировать контроллер дисковода гибких дисков.

1. перейдите в нужный каталог, в моём случае это
   /driver/bus/pnp/devices/00:0f::

	# cd /driver/bus/pnp/devices/00:0f
	# cat name
	PC standard floppy disk controller

2. проверьте, активно ли устройство уже::

	# cat resources
	DISABLED

  - Обратите внимание на строку "DISABLED".  Это означает, что устройство не активно.

3. проверьте возможные конфигурации устройства (необязательно)::

	# cat options
	Dependent: 01 - Priority acceptable
	    port 0x3f0-0x3f0, align 0x7, size 0x6, 16-bit address decoding
	    port 0x3f7-0x3f7, align 0x0, size 0x1, 16-bit address decoding
	    irq 6
	    dma 2 8-bit compatible
	Dependent: 02 - Priority acceptable
	    port 0x370-0x370, align 0x7, size 0x6, 16-bit address decoding
	    port 0x377-0x377, align 0x0, size 0x1, 16-bit address decoding
	    irq 6
	    dma 2 8-bit compatible

4. теперь активируйте устройство::

	# echo "auto" > resources

5. наконец, проверьте, активно ли устройство::

	# cat resources
	io 0x3f0-0x3f5
	io 0x3f7-0x3f7
	irq 6
	dma 2

также имеется ряд параметров ядра::

	pnp_reserve_irq=irq1[,irq2] ....
	pnp_reserve_dma=dma1[,dma2] ....
	pnp_reserve_io=io1,size1[,io2,size2] ....
	pnp_reserve_mem=mem1,size1[,mem2,size2] ....



Единый слой Plug and Play
-------------------------

Все драйверы, протоколы и службы Plug and Play встречаются в центральном месте,
называемом слоем Plug and Play. Этот слой отвечает за обмен информацией между
PnP-драйверами и PnP-протоколами. Таким образом, он автоматически перенаправляет
команды нужному протоколу. Это значительно упрощает написание PnP-драйверов.

Из слоя Plug and Play доступны следующие функции:

pnp_get_protocol
  увеличивает число использований на единицу

pnp_put_protocol
  уменьшает число использований на единицу

pnp_register_protocol
  используйте это для регистрации нового PnP-протокола

pnp_register_driver
  добавляет PnP-драйвер в слой Plug and Play

  это включает интеграцию с моделью драйверов
  возвращает ноль в случае успеха или отрицательный номер ошибки в случае неудачи;
  подсчитывайте вызовы метода .add(), если вам нужно знать, сколько устройств
  привязывается к драйверу

pnp_unregister_driver
  удаляет PnP-драйвер из слоя Plug and Play



Протоколы Plug and Play
-----------------------

Этот раздел содержит информацию для разработчиков PnP-протоколов.

В вычислительном мире в настоящее время доступны следующие протоколы:

- PNPBIOS:
    используется для системных устройств, таких как последовательные и параллельные порты.
- ISAPNP:
    обеспечивает поддержку PnP для шины ISA
- ACPI:
    среди множества его применений ACPI предоставляет информацию об устройствах
    системного уровня.

Он предназначен для замены PNPBIOS. В настоящее время он не поддерживается Linux
Plug and Play, но это планируется в ближайшем будущем.


Требования к протоколу Linux PnP:
1. протокол должен использовать EISA ID
2. протокол должен информировать слой PnP о текущей конфигурации устройства

- возможность задавать ресурсы необязательна, но предпочтительна.

Ниже приведены функции, связанные с PnP-протоколами:

pnp_add_device
  используйте эту функцию для добавления PnP-устройства в слой PnP

  вызывайте эту функцию только тогда, когда все желаемые значения установлены в
  структуре pnp_dev

pnp_init_device
  вызовите это для инициализации структуры PnP

pnp_remove_device
  вызовите это для удаления устройства из слоя Plug and Play.
  это завершится неудачей, если устройство всё ещё используется.
  автоматически освободит память, используемую устройством и связанными структурами

pnp_add_id
  добавляет EISA ID в список поддерживаемых ID для указанного устройства

Для получения дополнительной информации обратитесь к исходному коду протокола,
например /drivers/pnp/pnpbios/core.c.



Драйверы Linux Plug and Play
----------------------------

Этот раздел содержит информацию для разработчиков драйверов Linux PnP.

Новый способ
^^^^^^^^^^^^

1. сначала составьте список поддерживаемых EISA ID

   пример::

	static const struct pnp_id pnp_dev_table[] = {
		/* Standard LPT Printer Port */
		{.id = "PNP0400", .driver_data = 0},
		/* ECP Printer Port */
		{.id = "PNP0401", .driver_data = 0},
		{.id = ""}
	};

   Обратите внимание, что символ 'X' можно использовать как подстановочный знак в
   функциональной части (последние четыре символа).

   пример::

	/* Unknown PnP modems */
	{	"PNPCXXX",		UNKNOWN_DEV	},

   При желании можно определить поддерживаемые ID PnP-карт.
   пример::

	static const struct pnp_id pnp_card_table[] = {
		{	"ANYDEVS",		0	},
		{	"",			0	}
	};

2. При желании определите функции probe и remove. Возможно, имеет смысл не
   определять эти функции, если у драйвера уже есть надёжный метод обнаружения
   ресурсов, как у драйвера parport_pc.

   пример::

	static int
	serial_pnp_probe(struct pnp_dev * dev, const struct pnp_id *card_id, const
			struct pnp_id *dev_id)
	{
	. . .

   пример::

	static void serial_pnp_remove(struct pnp_dev * dev)
	{
	. . .

   обратитесь к /drivers/serial/8250_pnp.c для получения дополнительной информации.

3. создайте структуру драйвера

   пример::

	static struct pnp_driver serial_pnp_driver = {
		.name		= "serial",
		.card_id_table	= pnp_card_table,
		.id_table	= pnp_dev_table,
		.probe		= serial_pnp_probe,
		.remove		= serial_pnp_remove,
	};

   * name и id_table не могут быть NULL.

4. зарегистрируйте драйвер

   пример::

	static int __init serial8250_pnp_init(void)
	{
		return pnp_register_driver(&serial_pnp_driver);
	}
