=================================================================
Драйвер удалённого обновления BIOS на Dell (dell_rbu)
=================================================================

Назначение
==========

Документ, демонстрирующий использование драйвера удалённого обновления BIOS
на Dell для обновления образов BIOS на серверах и настольных компьютерах Dell.

Область применения
==================

Этот документ рассматривает только функциональность драйвера rbu.
Он не охватывает поддержку, необходимую со стороны приложений, чтобы BIOS мог
обновить себя образом, загруженным в память.

Обзор
=====

Этот драйвер работает совместно с Dell OpenManage или Dell Update Packages для
обновления BIOS на серверах Dell (начиная с серверов, продаваемых с 1999 года),
настольных компьютерах и ноутбуках (начиная с тех, что продавались в 2005 году).

Перейдите на  http://support.dell.com, зарегистрируйтесь, и вы сможете найти
информацию об OpenManage и Dell Update packages (DUP).

Для обновления BIOS на системах Dell также можно использовать Libsmbios;
подробности см. на https://linux.dell.com/libsmbios/.

Драйвер Dell_RBU поддерживает обновление BIOS методами монолитного образа и
пакетированного образа. В случае монолитного образа драйвер выделяет непрерывный
блок физических страниц, содержащий образ BIOS. В случае пакетированного образа
приложение, использующее драйвер, разбивает образ на пакеты фиксированного
размера, и драйвер размещает каждый пакет в непрерывной физической памяти.
Драйвер также поддерживает связный список пакетов для их обратного чтения.

При выгрузке драйвера dell_rbu вся выделенная память освобождается.

Драйверу rbu необходимо приложение (как упомянуто выше), которое сообщит BIOS о
включении обновления при следующей перезагрузке системы.

Пользователю не следует выгружать драйвер rbu после загрузки образа BIOS или
после обновления.

При загрузке драйвера создаются следующие каталоги в файловой системе /sys::

	/sys/class/firmware/dell_rbu/loading
	/sys/class/firmware/dell_rbu/data
	/sys/devices/platform/dell_rbu/image_type
	/sys/devices/platform/dell_rbu/data
	/sys/devices/platform/dell_rbu/packet_size

Драйвер поддерживает два типа механизма обновления: монолитный и пакетированный.
Эти механизмы обновления зависят от BIOS, который в данный момент работает в
системе. Большинство систем Dell поддерживают монолитное обновление, при котором
образ BIOS копируется в единый непрерывный блок физической памяти.

В случае пакетного механизма единый блок памяти может быть разбит на меньшие
части непрерывной памяти, и образ BIOS распределяется по этим пакетам.

По умолчанию драйвер использует монолитную память для типа обновления. Это можно
изменить на пакеты во время загрузки драйвера, указав параметр загрузки
image_type=packet.  Это также можно изменить позже, как показано ниже::

	echo packet > /sys/devices/platform/dell_rbu/image_type

В режиме пакетного обновления размер пакета должен быть задан до того, как
можно будет загружать какие-либо пакеты. Это делается следующим образом::

	echo XXXX > /sys/devices/platform/dell_rbu/packet_size

В механизме пакетного обновления пользователю необходимо создать новый файл,
содержащий пакеты данных, расположенные вплотную друг к другу. Это можно сделать
следующим образом:
Пользователь создаёт заголовок пакета, получает фрагмент образа BIOS и
размещает его сразу после заголовка пакета; теперь заголовок пакета + фрагмент
образа BIOS вместе должны соответствовать заданному packet_size. Это образует
один пакет; пользователю необходимо создать ещё такие пакеты из всего файла
образа BIOS, а затем расположить все эти пакеты вплотную друг к другу в один
файл.

Затем этот файл копируется в /sys/class/firmware/dell_rbu/data.
Как только этот файл попадает к драйверу, драйвер извлекает из файла данные
размером packet_size и распределяет их по физической памяти в непрерывных
областях размером в пакет.

Этот метод гарантирует, что все пакеты попадут к драйверу за одну операцию.

При монолитном обновлении пользователь просто получает образ BIOS (файл .hdr) и
копирует его в файл data как есть, без каких-либо изменений самого образа BIOS.

Выполните приведённые ниже шаги для загрузки образа BIOS.

1) echo 1 > /sys/class/firmware/dell_rbu/loading
2) cp bios_image.hdr /sys/class/firmware/dell_rbu/data
3) echo 0 > /sys/class/firmware/dell_rbu/loading

Записи /sys/class/firmware/dell_rbu/ будут оставаться до тех пор, пока не будет
выполнено следующее.

::

	echo -1 > /sys/class/firmware/dell_rbu/loading

Пока этот шаг не завершён, драйвер не может быть выгружен.

Кроме того, запись mono, packet или init в image_type освободит память,
выделенную драйвером.

Если пользователь по ошибке выполнит шаги 1 и 3 выше, не выполнив шаг 2,
записи /sys/class/firmware/dell_rbu/ исчезнут.

Записи можно создать заново, выполнив следующее::

	echo init > /sys/devices/platform/dell_rbu/image_type

.. note:: запись init в image_type не изменяет его исходное значение.

Также драйвер предоставляет файл /sys/devices/platform/dell_rbu/data, доступный
только для чтения, для обратного чтения загруженного образа.

.. note::

   После обновления образа BIOS приложению пользовательского режима необходимо
   выполнить код, который отправляет BIOS запрос на обновление. Тогда при
   следующей перезагрузке BIOS узнаёт о новом загруженном образе и обновляет
   себя. Также не выгружайте драйвер rbu, если образ должен быть обновлён.
