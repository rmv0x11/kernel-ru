Последовательная консоль Linux
==============================

Чтобы использовать последовательный порт в качестве консоли, необходимо
включить соответствующую поддержку при сборке ядра — по умолчанию она не
компилируется. Для последовательных портов в стиле PC это опция конфигурации,
соответствующая пункту меню:

:menuselection:`Character devices --> Serial drivers --> 8250/16550 and compatible serial support --> Console on 8250/16550 and compatible serial port`

Поддержку последовательных портов необходимо встроить в ядро, а не собирать
как модуль.

Можно указать несколько устройств для вывода консоли. Вы можете задать новую
опцию командной строки ядра, чтобы выбрать, какое устройство (или устройства)
использовать для вывода консоли.

Формат этой опции таков::

	console=device,options

	device:		tty0 for the foreground virtual console
			ttyX for any other virtual console
			ttySx for a serial port
			lp0 for the first parallel port
			ttyUSB0 for the first USB serial device

	options:	depend on the driver. For the serial port this
			defines the baudrate/parity/bits/flow control of
			the port, in the format BBBBPNF, where BBBB is the
			speed, P is parity (n/o/e), N is number of bits,
			and F is flow control ('r' for RTS). Default is
			9600n8. The maximum baudrate is 115200.

В командной строке ядра можно указать несколько опций console=.

Поведение чётко определено, когда каждый тип устройства упомянут только один
раз. В этом случае вывод появится на всех запрошенных консолях. А при открытии
``/dev/console`` будет использовано последнее устройство.
Так, например::

	console=ttyS1,9600 console=tty0

определяет, что при открытии ``/dev/console`` вы получите текущую активную
виртуальную консоль, а сообщения ядра будут появляться как на VGA-консоли, так
и на втором последовательном порту (ttyS1 или COM2) на скорости 9600 бод.

Поведение усложняется, когда один и тот же тип устройства определён несколько
раз. В этом случае действуют следующие два правила:

1. Вывод появится только на первом устройстве каждого определённого типа.

2. ``/dev/console`` будет связан с первым зарегистрированным устройством,
   причём порядок регистрации зависит от того, как ядро инициализирует
   различные подсистемы.

   Это правило применяется и тогда, когда последний параметр console= не
   используется по другим причинам. Например, из-за опечатки или потому, что
   оборудование недоступно.

Результат может оказаться неожиданным. Например, следующие две командные
строки дают одинаковый результат::

	console=ttyS1,9600 console=tty0 console=tty1
	console=tty0 console=ttyS1,9600 console=tty1

Сообщения ядра печатаются только на ``tty0`` и ``ttyS1``. А ``/dev/console``
связывается с ``tty0``. Это происходит потому, что ядро пытается
зарегистрировать графические консоли раньше последовательных. Оно делает так
из-за поведения по умолчанию, когда консольное устройство не задано (см. ниже).

Обратите внимание, что последний параметр ``console=tty1`` всё же имеет
значение. Командная строка ядра также используется systemd. Он будет
использовать последний определённый ``tty1`` в качестве консоли входа в
систему.

Если консольное устройство не задано, будет использовано первое найденное
устройство, способное выступать в роли системной консоли. В настоящее время
система сначала ищет VGA-карту, а затем последовательный порт. Поэтому, если в
вашей системе нет VGA-карты, первый последовательный порт автоматически станет
консолью, если только ядро не сконфигурировано с опцией
CONFIG_NULL_TTY_DEFAULT_CONSOLE — тогда по умолчанию будет использоваться
устройство ttynull.

Чтобы использовать ``/dev/console``, потребуется создать новое устройство.
Официально ``/dev/console`` теперь является символьным устройством 5,1.

(В качестве консоли можно также использовать сетевое устройство. Сведения об
этом см. в ``Documentation/networking/netconsole.rst``.)

Вот пример, в котором в качестве консоли используется ``/dev/ttyS1`` (COM2).
Замените образцовые значения по мере необходимости.

1. Создайте ``/dev/console`` (реальная консоль) и ``/dev/tty0`` (главная
   виртуальная консоль)::

     cd /dev
     rm -f console tty0
     mknod -m 622 console c 5 1
     mknod -m 622 tty0 c 4 0

2. LILO также может принимать ввод с последовательного устройства. Это очень
   полезная опция. Чтобы указать LILO использовать последовательный порт:
   в lilo.conf (глобальная секция)::

     serial  = 1,9600n8 (ttyS1, 9600 bd, no parity, 8 bits)

3. Настройте флаги для нового ядра, опять же в lilo.conf (секция ядра)::

     append = "console=ttyS1,9600"

4. Убедитесь, что на последовательном порту запущен getty, чтобы вы могли
   войти в систему после завершения её загрузки. Это делается добавлением
   подобной строки в ``/etc/inittab`` (точный синтаксис зависит от вашего
   getty)::

     S1:23:respawn:/sbin/getty -L ttyS1 9600 vt100

5. Init и ``/etc/ioctl.save``

   Sysvinit запоминает свои настройки stty в файле в ``/etc`` под названием
   ``/etc/ioctl.save``. УДАЛИТЕ ЭТОТ ФАЙЛ перед первым использованием
   последовательной консоли, потому что иначе init, вероятно, установит
   скорость передачи 38400 (скорость виртуальной консоли).

6. ``/dev/console`` и X
   Программы, которые хотят что-либо делать с виртуальной консолью, обычно
   открывают ``/dev/console``. Если вы создали новое устройство
   ``/dev/console``, а ваша консоль НЕ является виртуальной консолью,
   некоторые программы будут давать сбой. Это программы, которые хотят
   обращаться к интерфейсу VT и используют
   ``/dev/console instead of /dev/tty0``. Некоторые из таких программ::

     Xfree86, svgalib, gpm, SVGATextMode

   Впрочем, в современных версиях этих программ это должно быть исправлено.

   Обратите внимание, что если вы загружаетесь без опции ``console=`` (или с
   ``console=/dev/tty0``), то ``/dev/console`` совпадает с ``/dev/tty0``. В
   этом случае всё по-прежнему будет работать.
