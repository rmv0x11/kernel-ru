==============================
Наборы изменений Devicetree
==============================

Набор изменений (changeset) Devicetree — это метод, позволяющий применять
изменения к «живому» дереву таким образом, что будет применён либо весь
набор изменений целиком, либо ни одно из них. Если в процессе применения
набора изменений возникает ошибка, дерево откатывается к предыдущему
состоянию. Набор изменений также может быть удалён после того, как он был
применён.

При применении набора изменений все изменения вносятся в дерево
одновременно, до отправки уведомлений OF_RECONFIG. Это сделано для того,
чтобы получатель видел полное и согласованное состояние дерева в момент
получения уведомления.

Последовательность работы с набором изменений выглядит следующим образом.

1. of_changeset_init() — инициализирует набор изменений

2. Ряд вызовов изменения дерева DT — of_changeset_attach_node(),
   of_changeset_detach_node(), of_changeset_add_property(),
   of_changeset_remove_property, of_changeset_update_property() — для
   подготовки набора изменений. На данном этапе никакие изменения в активное
   дерево не вносятся. Все операции изменения записываются в список 'entries'
   структуры of_changeset.

3. of_changeset_apply() — применяет изменения к дереву. Будет применён либо
   весь набор изменений целиком, либо, если возникнет ошибка, дерево будет
   восстановлено до предыдущего состояния. Ядро обеспечивает надлежащую
   сериализацию посредством блокировок. При необходимости доступна версия без
   блокировки — __of_changeset_apply.

Если успешно применённый набор изменений требуется удалить, это можно сделать
с помощью of_changeset_revert().
