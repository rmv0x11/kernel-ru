=========================================================
Заметки о динамическом резолвере Devicetree
=========================================================

Этот документ описывает реализацию резолвера DeviceTree внутри ядра,
который находится в drivers/of/resolver.c

Как работает резолвер
---------------------

Резолверу на вход подаётся произвольное дерево, скомпилированное с
соответствующей опцией dtc и имеющее тег /plugin/. Это порождает
подходящие узлы __fixups__ и __local_fixups__.

Последовательно резолвер работает по следующим шагам:

1. Получить максимальное значение phandle device tree из «живого» дерева + 1.
2. Скорректировать все локальные phandle разрешаемого дерева на эту величину.
3. Используя информацию из узла __local__fixups__, скорректировать все
   локальные ссылки на ту же величину.
4. Для каждого свойства в узле __fixups__ найти узел, на который оно ссылается,
   в «живом» дереве. Это метка, используемая для тегирования узла.
5. Получить phandle цели данного fixup.
6. Для каждого fixup в свойстве найти местоположение node:property:offset
   и заменить его на значение phandle.
