*****************************
GPIO_V2_LINEINFO_CHANGED_READ
*****************************

Имя
===

GPIO_V2_LINEINFO_CHANGED_READ - чтение событий изменения информации о линии для
отслеживаемых линий с микросхемы.

Синопсис
========

``int read(int chip_fd, void *buf, size_t count)``

Аргументы
=========

``chip_fd``
    Файловый дескриптор символьного устройства GPIO, возвращённый `open()`.

``buf``
    Буфер для хранения :c:type:`событий<gpio_v2_line_info_changed>`.

``count``
    Количество байт, доступных в ``buf``, которое должно быть не меньше размера
    события :c:type:`gpio_v2_line_info_changed`.

Описание
========

Чтение событий изменения информации о линии для отслеживаемых линий с микросхемы.

.. note::
    Отслеживание изменений информации о линии обычно не требуется и, как правило,
    выполняется только компонентом мониторинга системы.

    Эти события относятся к изменениям состояния запроса линии или её
    конфигурации, а не её значения. Используйте gpio-v2-line-event-read.rst для
    получения событий при изменении значения линии.

Чтобы порождались события изменения информации, за линией необходимо установить
наблюдение с помощью gpio-v2-get-lineinfo-watch-ioctl.rst.  Впоследствии запрос,
освобождение или переконфигурация линии будут порождать событие изменения
информации.

Ядро проставляет события временными метками в момент их возникновения и сохраняет
их в буфере, откуда они могут быть прочитаны пространством пользователя в удобный
момент с помощью `read()`.

Размер буфера событий в ядре фиксирован и составляет 32 события на каждый
``chip_fd``.

Буфер может переполниться, если всплески событий происходят быстрее, чем они
читаются пространством пользователя. При переполнении самое недавнее событие
отбрасывается. Переполнение невозможно обнаружить из пространства пользователя.

События, прочитанные из буфера, всегда находятся в том же порядке, в котором они
были обнаружены ядром, в том числе когда через один ``chip_fd`` отслеживается
несколько линий.

Чтобы минимизировать число вызовов, требуемых для копирования событий из ядра в
пространство пользователя, `read()` поддерживает копирование нескольких событий.
Число копируемых событий равно меньшему из числа доступных в буфере ядра и числа,
которое поместится в буфер пространства пользователя (``buf``).

`read()` будет блокироваться, если ни одно событие не доступно и для ``chip_fd`` не
установлен флаг **O_NONBLOCK**.

Наличие события можно проверить, проверив, доступен ли ``chip_fd`` для чтения, с
помощью `poll()` или эквивалентного вызова.

Возвращаемое значение
=====================

В случае успеха — число прочитанных байт, которое будет кратно размеру события
:c:type:`gpio_v2_line_info_changed`.

В случае ошибки -1, и переменная ``errno`` устанавливается соответствующим образом.
Распространённые коды ошибок описаны в error-codes.rst.
