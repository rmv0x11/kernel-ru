*****************************
GPIO_V2_LINE_SET_CONFIG_IOCTL
*****************************

Имя
===

GPIO_V2_LINE_SET_CONFIG_IOCTL — обновление конфигурации ранее запрошенных линий.

Обзор
=====

.. c:macro:: GPIO_V2_LINE_SET_CONFIG_IOCTL

``int ioctl(int req_fd, GPIO_V2_LINE_SET_CONFIG_IOCTL, struct gpio_v2_line_config *config)``

Аргументы
=========

``req_fd``
    Файловый дескриптор символьного устройства GPIO, возвращаемый в
    :c:type:`request.fd<gpio_v2_line_request>` посредством gpio-v2-get-line-ioctl.rst.

``config``
    Новая :c:type:`конфигурация<gpio_v2_line_config>`, применяемая к запрошенным
    линиям.

Описание
========

Обновляет конфигурацию ранее запрошенных линий без освобождения линии и без
возникновения возможных сбоев сигнала (glitches).

Новая конфигурация должна задавать конфигурацию для всех запрошенных линий.

Те же :ref:`gpio-v2-get-line-config-rules` и
:ref:`gpio-v2-get-line-config-support`, которые применяются при запросе линий,
применяются и при обновлении конфигурации линии, с дополнительным ограничением:
для включения переконфигурирования должен быть установлен флаг направления.
Если в конфигурации для данной линии не установлен ни один флаг направления, то
конфигурация для этой линии остаётся без изменений.

Основной сценарий использования этой команды — изменение направления
двунаправленных линий между вводом и выводом, однако она также может
применяться для динамического управления обнаружением фронтов или, в более общем
случае, для бесшовного перевода линий из одного состояния конфигурации в другое.

Чтобы изменить только значение линий вывода, используйте
gpio-v2-line-set-values-ioctl.rst.

Возвращаемое значение
=====================

В случае успеха — 0.

В случае ошибки — -1, а переменная ``errno`` устанавливается соответствующим
образом. Распространённые коды ошибок описаны в error-codes.rst.
