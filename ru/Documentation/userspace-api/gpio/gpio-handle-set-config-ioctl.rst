.. _GPIOHANDLE_SET_CONFIG_IOCTL:

***************************
GPIOHANDLE_SET_CONFIG_IOCTL
***************************

.. warning::
    Этот ioctl является частью chardev_v1.rst и объявлен устаревшим в пользу
    gpio-v2-line-set-config-ioctl.rst.

Имя
===

GPIOHANDLE_SET_CONFIG_IOCTL — обновить конфигурацию ранее запрошенных линий.

Синопсис
========

.. c:macro:: GPIOHANDLE_SET_CONFIG_IOCTL

``int ioctl(int handle_fd, GPIOHANDLE_SET_CONFIG_IOCTL, struct gpiohandle_config *config)``

Аргументы
=========

``handle_fd``
    Файловый дескриптор символьного устройства GPIO, возвращаемый в
    :c:type:`request.fd<gpiohandle_request>` посредством gpio-get-linehandle-ioctl.rst.

``config``
    Новая :c:type:`конфигурация<gpiohandle_config>`, применяемая к запрошенным
    линиям.

Описание
========

Обновляет конфигурацию ранее запрошенных линий без освобождения линии и без
внесения потенциальных сбоев (glitches).

Конфигурация применяется ко всем запрошенным линиям.

Те же :ref:`gpio-get-linehandle-config-rules` и
:ref:`gpio-get-linehandle-config-support`, которые действуют при запросе линий,
действуют и при обновлении конфигурации линии, с дополнительным ограничением:
должен быть установлен флаг направления. Запрос недопустимой конфигурации,
в том числе без установленного флага направления, является ошибкой
(**EINVAL**).

Основной сценарий использования этой команды — изменение направления
двунаправленных линий между вводом и выводом, но в более общем случае она может
использоваться для бесшовного перевода линий из одного состояния конфигурации в
другое.

Чтобы изменить только значение выходных линий, используйте
gpio-handle-set-line-values-ioctl.rst.

Впервые добавлено в 5.5.

Возвращаемое значение
=====================

При успехе 0.

При ошибке -1, и переменная ``errno`` устанавливается соответствующим образом.
Распространённые коды ошибок описаны в error-codes.rst.
