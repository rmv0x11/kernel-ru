*****************************
GPIO_V2_LINE_SET_VALUES_IOCTL
*****************************

Имя
===

GPIO_V2_LINE_SET_VALUES_IOCTL — установка значений запрошенных выходных линий.

Синопсис
========

.. c:macro:: GPIO_V2_LINE_SET_VALUES_IOCTL

``int ioctl(int req_fd, GPIO_V2_LINE_SET_VALUES_IOCTL, struct gpio_v2_line_values *values)``

Аргументы
=========

``req_fd``
    Файловый дескриптор символьного устройства GPIO, возвращаемый в
    :c:type:`request.fd<gpio_v2_line_request>` через gpio-v2-get-line-ioctl.rst.

``values``
    Структура :c:type:`line_values<gpio_v2_line_values>` для установки, где поле
    ``mask`` задаёт подмножество запрошенных линий, значения которых
    устанавливаются, а поле ``bits`` задаёт новое значение.

Описание
========

Установка значений запрошенных выходных линий.

Устанавливаемые значения являются логическими и указывают, должна ли линия быть
активной или неактивной. Флаг ``GPIO_V2_LINE_FLAG_ACTIVE_LOW`` управляет
соответствием между логическими значениями (активно/неактивно) и физическими
значениями (высокий/низкий уровень).
Если флаг ``GPIO_V2_LINE_FLAG_ACTIVE_LOW`` не установлен, то активному состоянию
соответствует высокий уровень, а неактивному — низкий. Если флаг
``GPIO_V2_LINE_FLAG_ACTIVE_LOW`` установлен, то активному состоянию соответствует
низкий уровень, а неактивному — высокий.

Устанавливать можно только значения выходных линий.
Попытка установить значение входной линии является ошибкой (**EPERM**).

Возвращаемое значение
=====================

В случае успеха — 0.

В случае ошибки — -1, и переменной ``errno`` присваивается соответствующее
значение. Распространённые коды ошибок описаны в error-codes.rst.
