.. _GPIO_GET_LINEINFO_WATCH_IOCTL:

*****************************
GPIO_GET_LINEINFO_WATCH_IOCTL
*****************************

.. warning::
    Этот ioctl является частью chardev_v1.rst и объявлен устаревшим в пользу
    gpio-v2-get-lineinfo-watch-ioctl.rst.

Имя
===

GPIO_GET_LINEINFO_WATCH_IOCTL - Включить отслеживание изменений состояния
запроса и информации о конфигурации линии.

Обзор
=====

.. c:macro:: GPIO_GET_LINEINFO_WATCH_IOCTL

``int ioctl(int chip_fd, GPIO_GET_LINEINFO_WATCH_IOCTL, struct gpioline_info *info)``

Аргументы
=========

``chip_fd``
    Файловый дескриптор символьного устройства GPIO, возвращённый `open()`.

``info``
    Структура :c:type:`line_info<gpioline_info>`, подлежащая заполнению, в
    которой поле ``offset`` задаёт линию для отслеживания.

Описание
========

Включить отслеживание изменений состояния запроса и информации о конфигурации
линии. Изменения информации о линии включают запрос линии, её освобождение или
переконфигурирование.

.. note::
    Отслеживание информации о линии обычно не требуется и, как правило,
    используется только компонентом мониторинга системы.

    Информация о линии НЕ включает значение линии.

    Линия должна быть запрошена с помощью gpio-get-linehandle-ioctl.rst или
    gpio-get-lineevent-ioctl.rst для доступа к её значению, а событие линии может
    отслеживать события линии с помощью gpio-lineevent-data-read.rst.

По умолчанию при открытии микросхемы GPIO все линии не отслеживаются.

Несколько линий могут отслеживаться одновременно путём добавления отслеживания
для каждой из них.

После установки отслеживания любые изменения информации о линии будут
генерировать события, которые можно прочитать из ``chip_fd``, как описано в
gpio-lineinfo-changed-read.rst.

Добавление отслеживания к линии, которая уже отслеживается, является ошибкой
(**EBUSY**).

Отслеживания привязаны к конкретному ``chip_fd`` и независимы от отслеживаний на
той же микросхеме GPIO, открытой отдельным вызовом `open()`.

Впервые добавлено в версии 5.7.

Возвращаемое значение
=====================

При успехе возвращается 0, а ``info`` заполняется текущей информацией о линии.

При ошибке возвращается -1, и переменная ``errno`` устанавливается соответствующим
образом. Распространённые коды ошибок описаны в error-codes.rst.
