*****************************
GPIO_V2_LINE_GET_VALUES_IOCTL
*****************************

Имя
===

GPIO_V2_LINE_GET_VALUES_IOCTL - Получить значения запрошенных линий.

Синопсис
========

.. c:macro:: GPIO_V2_LINE_GET_VALUES_IOCTL

``int ioctl(int req_fd, GPIO_V2_LINE_GET_VALUES_IOCTL, struct gpio_v2_line_values *values)``

Аргументы
=========

``req_fd``
    Файловый дескриптор символьного устройства GPIO, возвращаемый в
    :c:type:`request.fd<gpio_v2_line_request>` из gpio-v2-get-line-ioctl.rst.

``values``
    :c:type:`line_values<gpio_v2_line_values>` для получения, где ``mask``
    задаёт подмножество запрошенных линий, значения которых нужно получить.

Описание
========

Получить значения запрошенных линий.

Возвращаемые значения являются логическими и указывают, активна линия или
неактивна. Флаг ``GPIO_V2_LINE_FLAG_ACTIVE_LOW`` управляет соответствием между
физическими значениями (high/low) и логическими значениями (active/inactive).
Если флаг ``GPIO_V2_LINE_FLAG_ACTIVE_LOW`` не установлен, то high соответствует
активному состоянию, а low — неактивному. Если флаг
``GPIO_V2_LINE_FLAG_ACTIVE_LOW`` установлен, то low соответствует активному
состоянию, а high — неактивному.

Можно считывать значения как входных, так и выходных линий.

Для выходных линий возвращаемое значение зависит от драйвера и конфигурации и
может быть либо значением из выходного буфера (последним заданным запрошенным
значением), либо значением из входного буфера (фактическим уровнем линии);
в зависимости от оборудования и конфигурации эти значения могут различаться.

Возвращаемое значение
=====================

При успешном выполнении возвращается 0, и соответствующие
:c:type:`values.bits<gpio_v2_line_values>` содержат считанное значение.

При ошибке возвращается -1, и переменной ``errno`` присваивается
соответствующее значение. Распространённые коды ошибок описаны в
error-codes.rst.
