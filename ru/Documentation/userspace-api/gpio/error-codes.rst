.. _gpio_errors:

***************************
Коды ошибок GPIO
***************************

.. _gpio-errors:

.. tabularcolumns:: |p{2.5cm}|p{15.0cm}|

.. flat-table:: Распространённые коды ошибок GPIO
    :header-rows:  0
    :stub-columns: 0
    :widths: 1 16

    -  -  ``EAGAIN`` (aka ``EWOULDBLOCK``)

       -  Устройство было открыто в неблокирующем режиме, и чтение не может
          быть выполнено, так как нет доступных данных.

    -  -  ``EBADF``

       -  Файловый дескриптор недействителен.

    -  -  ``EBUSY``

       -  ioctl не может быть обработан, так как устройство занято. Обычно
          возвращается, когда ioctl пытается выполнить что-то, что потребовало бы
          использования ресурса, который уже был выделен. ioctl не должен
          повторяться без предварительного выполнения другого действия для
          устранения проблемы.

    -  -  ``EFAULT``

       -  Произошёл сбой при копировании данных из/в пространство пользователя,
	  вероятно вызванный недействительной ссылкой по указателю.

    -  -  ``EINVAL``

       -  Один или несколько параметров ioctl недействительны или выходят за
          допустимый диапазон. Это широко используемый код ошибки.

    -  -  ``ENODEV``

       -  Устройство не найдено или было удалено.

    -  -  ``ENOMEM``

       -  Недостаточно памяти для обработки желаемой операции.

    -  -  ``EPERM``

       -  Доступ запрещён. Обычно возвращается в ответ на попытку
          выполнить действие, несовместимое с текущей конфигурацией
          линии.

    -  -  ``EIO``

       -  Ошибка ввода-вывода (I/O). Обычно возвращается при наличии проблем со связью
          с аппаратным устройством или при запросе возможностей, которые оборудование не
          поддерживает. Это может указывать на неисправное или ненадёжное оборудование.
          Это ошибка типа «Что-то пошло не так, я сдаюсь!».

    -  - ``ENXIO``

       -  Обычно возвращается, когда была запрошена возможность, требующая поддержки
          прерываний, но линия не поддерживает прерывания.

.. note::

  #. Этот список не является исчерпывающим; ioctl-вызовы могут возвращать другие коды ошибок.
     Поскольку ошибки могут иметь побочные эффекты, такие как сброс драйвера,
     приложения должны прерывать работу при непредвиденных ошибках или иным образом
     считать, что устройство находится в неисправном состоянии.

  #. Коды ошибок, специфичные для конкретного запроса, перечислены в описаниях
     отдельных запросов.
