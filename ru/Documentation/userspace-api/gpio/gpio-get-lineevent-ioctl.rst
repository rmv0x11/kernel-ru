.. _GPIO_GET_LINEEVENT_IOCTL:

************************
GPIO_GET_LINEEVENT_IOCTL
************************

.. warning::
    Этот ioctl является частью chardev_v1.rst и признан устаревшим в пользу
    gpio-v2-get-line-ioctl.rst.

Имя
===

GPIO_GET_LINEEVENT_IOCTL - запросить у ядра линию с обнаружением фронтов.

Синтаксис
=========

.. c:macro:: GPIO_GET_LINEEVENT_IOCTL

``int ioctl(int chip_fd, GPIO_GET_LINEEVENT_IOCTL, struct gpioevent_request *request)``

Аргументы
=========

``chip_fd``
    Файловый дескриптор символьного устройства GPIO, возвращённый функцией `open()`.

``request``
    Структура :c:type:`event_request<gpioevent_request>`, задающая запрашиваемую
    линию и её конфигурацию.

Описание
========

Запросить у ядра линию с обнаружением фронтов.

В случае успеха запрашивающий процесс получает эксклюзивный доступ к значению
линии и может получать события при обнаружении фронтов на линии, как
описано в gpio-lineevent-data-read.rst.

Гарантируется, что состояние линии остаётся таким, как было запрошено, пока
возвращённый файловый дескриптор не будет закрыт. После закрытия файлового
дескриптора состояние линии становится неконтролируемым с точки зрения
пространства пользователя (userspace) и может вернуться к своему состоянию
по умолчанию.

Запрос линии, которая уже используется, является ошибкой (**EBUSY**).

Запрос обнаружения фронтов на линии, не поддерживающей прерывания, является
ошибкой (**ENXIO**).

Как и в случае с :ref:`дескриптором линии<gpio-get-linehandle-config-support>`,
конфигурация смещения (bias) выполняется по мере возможности.

Закрытие ``chip_fd`` не влияет на существующие события линий.

Правила конфигурации
--------------------

Применяются следующие правила конфигурации:

Событие линии запрашивается как вход, поэтому флаги, специфичные для линий
вывода, — ``GPIOHANDLE_REQUEST_OUTPUT``, ``GPIOHANDLE_REQUEST_OPEN_DRAIN`` или
``GPIOHANDLE_REQUEST_OPEN_SOURCE`` — устанавливать нельзя.

Может быть установлен только один флаг смещения, ``GPIOHANDLE_REQUEST_BIAS_xxx``.
Если ни один из флагов смещения не установлен, конфигурация смещения не изменяется.

Флаги фронтов, ``GPIOEVENT_REQUEST_RISING_EDGE`` и
``GPIOEVENT_REQUEST_FALLING_EDGE``, можно комбинировать для обнаружения как
нарастающих, так и спадающих фронтов.

Запрос недопустимой конфигурации является ошибкой (**EINVAL**).

Возвращаемое значение
=====================

В случае успеха возвращается 0, а :c:type:`request.fd<gpioevent_request>` содержит
файловый дескриптор для запроса.

В случае ошибки возвращается -1, а переменной ``errno`` присваивается
соответствующее значение.
Распространённые коды ошибок описаны в error-codes.rst.
