.. _GPIO_GET_LINEHANDLE_IOCTL:

*************************
GPIO_GET_LINEHANDLE_IOCTL
*************************

.. warning::
    Этот ioctl является частью chardev_v1.rst и объявлен устаревшим в пользу
    gpio-v2-get-line-ioctl.rst.

Имя
===

GPIO_GET_LINEHANDLE_IOCTL - запрос линии или линий у ядра.

Синтаксис
=========

.. c:macro:: GPIO_GET_LINEHANDLE_IOCTL

``int ioctl(int chip_fd, GPIO_GET_LINEHANDLE_IOCTL, struct gpiohandle_request *request)``

Аргументы
=========

``chip_fd``
    Файловый дескриптор символьного устройства GPIO, возвращённый `open()`.

``request``
    Объект :c:type:`handle_request<gpiohandle_request>`, задающий запрашиваемые
    линии и их конфигурацию.

Описание
========

Запрос линии или линий у ядра.

Хотя может запрашиваться несколько линий, ко всем линиям в запросе применяется
одна и та же конфигурация.

В случае успеха запрашивающему процессу предоставляется монопольный доступ к
значению линии и доступ на запись к конфигурации линии.

Состояние линии, включая значение выходных линий, гарантированно остаётся
запрошенным до тех пор, пока возвращённый файловый дескриптор не будет закрыт.
После закрытия файлового дескриптора состояние линии становится
неконтролируемым с точки зрения пространства пользователя и может вернуться к
своему состоянию по умолчанию.

Запрос линии, которая уже используется, является ошибкой (**EBUSY**).

Закрытие ``chip_fd`` не влияет на существующие дескрипторы линий.

.. _gpio-get-linehandle-config-rules:

Правила конфигурации
--------------------

Применяются следующие правила конфигурации:

Флаги направления, ``GPIOHANDLE_REQUEST_INPUT`` и
``GPIOHANDLE_REQUEST_OUTPUT``, не могут комбинироваться. Если не установлен ни
один из них, то единственным другим флагом, который может быть установлен,
является ``GPIOHANDLE_REQUEST_ACTIVE_LOW``, и линия запрашивается «как есть»
(as-is), чтобы позволить чтение значения линии без изменения её электрической
конфигурации.

Флаги управления выходом (drive), ``GPIOHANDLE_REQUEST_OPEN_xxx``, требуют
установки ``GPIOHANDLE_REQUEST_OUTPUT``.
Может быть установлен только один флаг управления выходом.
Если не установлен ни один из них, предполагается режим push-pull (двухтактный).

Может быть установлен только один флаг смещения (bias),
``GPIOHANDLE_REQUEST_BIAS_xxx``, и он требует, чтобы также был установлен флаг
направления.
Если не установлен ни один флаг смещения, то конфигурация смещения не изменяется.

Запрос недопустимой конфигурации является ошибкой (**EINVAL**).


.. _gpio-get-linehandle-config-support:

Поддержка конфигурации
----------------------

Если запрошенная конфигурация напрямую не поддерживается нижележащим
оборудованием и драйвером, ядро применяет один из следующих подходов:

 - отклонить запрос
 - эмулировать возможность программно
 - трактовать возможность по принципу «насколько возможно» (best effort)

Применяемый подход зависит от того, может ли возможность быть разумно
эмулирована программно, а также от влияния на оборудование и пространство
пользователя в случае, если возможность не поддерживается.
Для каждой возможности применяется следующий подход:

==============   ===========
Возможность      Подход
==============   ===========
Смещение         best effort
Направление      отклонить
Управление       эмулировать
==============   ===========

Смещение трактуется по принципу best effort, чтобы позволить пространству
пользователя применять одну и ту же конфигурацию как для платформ, которые
поддерживают внутреннее смещение, так и для тех, которые требуют внешнего
смещения.
В худшем случае линия остаётся в плавающем состоянии, а не смещается ожидаемым
образом.

Управление выходом эмулируется путём переключения линии на вход, когда линия не
должна управляться.

Во всех случаях конфигурация, сообщаемая gpio-get-lineinfo-ioctl.rst, является
запрошенной конфигурацией, а не результирующей аппаратной конфигурацией.
Пространство пользователя не может определить, поддерживается ли возможность
аппаратно, эмулируется ли она или применяется по принципу best effort.

Возвращаемое значение
=====================

В случае успеха — 0, и :c:type:`request.fd<gpiohandle_request>` содержит
файловый дескриптор для запроса.

В случае ошибки — -1, и переменной ``errno`` присваивается соответствующее
значение.
Распространённые коды ошибок описаны в error-codes.rst.
