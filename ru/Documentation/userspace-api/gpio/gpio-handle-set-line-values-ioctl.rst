*********************************
GPIO_HANDLE_SET_LINE_VALUES_IOCTL
*********************************
.. warning::
    Этот ioctl является частью chardev_v1.rst и считается устаревшим, его
    заменяет gpio-v2-line-set-values-ioctl.rst.

Имя
===

GPIO_HANDLE_SET_LINE_VALUES_IOCTL - Устанавливает значения всех запрошенных выходных линий.

Синопсис
========

.. c:macro:: GPIO_HANDLE_SET_LINE_VALUES_IOCTL

``int ioctl(int handle_fd, GPIO_HANDLE_SET_LINE_VALUES_IOCTL, struct gpiohandle_data *values)``

Аргументы
=========

``handle_fd``
    Файловый дескриптор символьного устройства GPIO, возвращаемый в
    :c:type:`request.fd<gpiohandle_request>` посредством gpio-get-linehandle-ioctl.rst.

``values``
    Устанавливаемые :c:type:`line_values<gpiohandle_data>`.

Описание
========

Устанавливает значения всех запрошенных выходных линий.

Устанавливаемые значения являются логическими и указывают, должна ли линия быть
активной или неактивной. Флаг ``GPIOHANDLE_REQUEST_ACTIVE_LOW`` управляет
соответствием между логическими значениями (активна/неактивна) и физическими
значениями (высокий/низкий уровень).
Если флаг ``GPIOHANDLE_REQUEST_ACTIVE_LOW`` не установлен, то активному состоянию
соответствует высокий уровень, а неактивному — низкий. Если флаг
``GPIOHANDLE_REQUEST_ACTIVE_LOW`` установлен, то активному состоянию соответствует
низкий уровень, а неактивному — высокий.

Устанавливать можно только значения выходных линий.
Попытка установить значение входных линий является ошибкой (**EPERM**).

Возвращаемое значение
=====================

В случае успеха — 0.

В случае ошибки — -1, при этом переменной ``errno`` присваивается соответствующее
значение. Распространённые коды ошибок описаны в error-codes.rst.
