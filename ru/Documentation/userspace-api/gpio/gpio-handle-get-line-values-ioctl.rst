********************************
GPIOHANDLE_GET_LINE_VALUES_IOCTL
********************************
.. warning::
    Этот ioctl является частью chardev_v1.rst и считается устаревшим,
    будучи заменённым на gpio-v2-line-get-values-ioctl.rst.

Имя
===

GPIOHANDLE_GET_LINE_VALUES_IOCTL - получить значения всех запрошенных линий.

Краткое описание
================

.. c:macro:: GPIOHANDLE_GET_LINE_VALUES_IOCTL

``int ioctl(int handle_fd, GPIOHANDLE_GET_LINE_VALUES_IOCTL, struct gpiohandle_data *values)``

Аргументы
=========

``handle_fd``
    Файловый дескриптор символьного устройства GPIO, возвращаемый в
    :c:type:`request.fd<gpiohandle_request>` интерфейсом gpio-get-linehandle-ioctl.rst.

``values``
    Структура :c:type:`line_values<gpiohandle_data>`, которую необходимо заполнить.

Описание
========

Получить значения всех запрошенных линий.

Возвращаемые значения являются логическими и указывают, активна линия или неактивна.
Флаг ``GPIOHANDLE_REQUEST_ACTIVE_LOW`` управляет соответствием между физическими
значениями (high/low) и логическими значениями (активна/неактивна).
Если флаг ``GPIOHANDLE_REQUEST_ACTIVE_LOW`` не установлен, то high соответствует
активному состоянию, а low — неактивному. Если флаг ``GPIOHANDLE_REQUEST_ACTIVE_LOW``
установлен, то low соответствует активному состоянию, а high — неактивному.

Значения могут быть прочитаны как для входных, так и для выходных линий.

Для выходных линий возвращаемое значение зависит от драйвера и конфигурации и
может представлять собой либо выходной буфер (последнее запрошенное установленное
значение), либо входной буфер (фактический уровень линии); в зависимости от
оборудования и конфигурации эти значения могут различаться.

Этот ioctl также может использоваться для чтения значения линии в случае событий
линии, при этом ``event_fd`` подставляется вместо ``handle_fd``.  Поскольку в этом
случае запрашивается только одна линия, в ``values`` возвращается только одно значение.

Возвращаемое значение
=====================

В случае успеха — 0, при этом ``values`` заполнена прочитанными значениями.

В случае ошибки — -1, а переменной ``errno`` присваивается соответствующее значение.
Распространённые коды ошибок описаны в error-codes.rst.
