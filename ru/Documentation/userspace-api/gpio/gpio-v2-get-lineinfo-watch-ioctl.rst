********************************
GPIO_V2_GET_LINEINFO_WATCH_IOCTL
********************************

Имя
===

GPIO_V2_GET_LINEINFO_WATCH_IOCTL - Включить отслеживание линии на предмет
изменений состояния её запроса и информации о конфигурации.

Синопсис
========

.. c:macro:: GPIO_V2_GET_LINEINFO_WATCH_IOCTL

``int ioctl(int chip_fd, GPIO_V2_GET_LINEINFO_WATCH_IOCTL, struct gpio_v2_line_info *info)``

Аргументы
=========

``chip_fd``
    Файловый дескриптор символьного устройства GPIO, возвращённый `open()`.

``info``
    Структура :c:type:`line_info<gpio_v2_line_info>`, которую нужно заполнить, с
    установленным полем ``offset``, указывающим отслеживаемую линию

Описание
========

Включить отслеживание линии на предмет изменений состояния её запроса и информации
о конфигурации. К изменениям информации о линии относятся запрос, освобождение или
переконфигурирование линии.

.. note::
    Отслеживание информации о линии обычно не требуется и, как правило, используется
    только компонентом мониторинга системы.

    Информация о линии НЕ включает значение линии.
    Чтобы получить доступ к её значению, линию необходимо запросить с помощью
    gpio-v2-get-line-ioctl.rst, а запрос линии может отслеживать линию на предмет
    событий с помощью gpio-v2-line-event-read.rst.

По умолчанию при открытии чипа GPIO все линии не отслеживаются.

Несколько линий могут отслеживаться одновременно путём добавления отслеживания для
каждой из них.

После установки отслеживания любые изменения информации о линии будут порождать
события, которые можно прочитать из ``chip_fd``, как описано в
gpio-v2-lineinfo-changed-read.rst.

Добавление отслеживания для линии, которая уже отслеживается, является ошибкой
(**EBUSY**).

Отслеживания привязаны к конкретному ``chip_fd`` и независимы от отслеживаний на
том же чипе GPIO, открытом отдельным вызовом `open()`.

Возвращаемое значение
=====================

При успехе — 0, и ``info`` заполняется текущей информацией о линии.

При ошибке — -1, и переменной ``errno`` присваивается соответствующее значение.
Распространённые коды ошибок описаны в error-codes.rst.
