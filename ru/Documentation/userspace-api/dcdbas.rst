==========================================
Базовый драйвер управления системами Dell
==========================================

Обзор
=====

Базовый драйвер управления системами Dell (Dell Systems Management Base
Driver) предоставляет интерфейс sysfs для программного обеспечения управления
системами, такого как Dell OpenManage, чтобы выполнять прерывания управления
системой (system management interrupts) и действия управления хостом
(перезапуск питания системы или отключение питания после завершения работы ОС)
на некоторых системах Dell.

Dell OpenManage требует этот драйвер на следующих системах Dell PowerEdge:
300, 1300, 1400, 400SC, 500SC, 1500SC, 1550, 600SC, 1600SC, 650, 1655MC,
700 и 750.  Ожидается, что другое программное обеспечение Dell, например
проект с открытым исходным кодом libsmbios, будет использовать этот драйвер,
и оно может включать использование этого драйвера на других системах Dell.

Проект Dell libsmbios нацелен на предоставление доступа к как можно большему
объёму информации BIOS.  Подробнее о проекте libsmbios см.
http://linux.dell.com/libsmbios/main/.


Прерывание управления системой
===============================

На некоторых системах Dell программное обеспечение управления системами должно
обращаться к определённой управляющей информации через прерывание управления
системой (system management interrupt, SMI).  Буфер данных SMI должен
располагаться в 32-битном адресном пространстве, и для SMI требуется
физический адрес этого буфера.  Драйвер поддерживает память, необходимую для
SMI, и предоставляет приложению способ сгенерировать SMI.
Драйвер создаёт следующие записи sysfs, чтобы программное обеспечение
управления системами могло выполнять эти прерывания управления системой::

	/sys/devices/platform/dcdbas/smi_data
	/sys/devices/platform/dcdbas/smi_data_buf_phys_addr
	/sys/devices/platform/dcdbas/smi_data_buf_size
	/sys/devices/platform/dcdbas/smi_request

Программное обеспечение управления системами должно выполнить следующие шаги,
чтобы инициировать SMI с помощью этого драйвера:

1) Заблокировать smi_data.
2) Записать команду управления системой в smi_data.
3) Записать «1» в smi_request, чтобы сгенерировать SMI вызывающего интерфейса
   (calling interface SMI), или «2», чтобы сгенерировать «сырой» SMI (raw SMI).
4) Прочитать ответ на команду управления системой из smi_data.
5) Разблокировать smi_data.


Действие управления хостом
===========================

Dell OpenManage поддерживает функцию управления хостом, которая позволяет
администратору выполнить перезапуск питания или отключение питания системы
после того, как ОС завершит работу.  На некоторых системах Dell эта функция
управления хостом требует, чтобы драйвер выполнил SMI после того, как ОС
завершит работу.

Драйвер создаёт следующие записи sysfs, чтобы программное обеспечение
управления системами могло запланировать выполнение драйвером действия
управления хостом по перезапуску или отключению питания после того, как
система завершит работу:

/sys/devices/platform/dcdbas/host_control_action
/sys/devices/platform/dcdbas/host_control_smi_type
/sys/devices/platform/dcdbas/host_control_on_shutdown

Dell OpenManage выполняет следующие шаги, чтобы инициировать действие
управления хостом по перезапуску или отключению питания с помощью этого
драйвера:

1) Записать выполняемое действие управления хостом в host_control_action.
2) Записать тип SMI, который должен выполнить драйвер, в host_control_smi_type.
3) Записать «1» в host_control_on_shutdown, чтобы включить действие управления
   хостом.
4) Инициировать завершение работы ОС.
   (Драйвер выполнит SMI управления хостом, когда получит уведомление о том,
   что ОС завершила работу.)
