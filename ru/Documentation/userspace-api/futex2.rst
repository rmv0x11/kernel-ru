======
futex2
======

:Author: André Almeida <andrealmeid@collabora.com>

futex, или fast user mutex, — это набор системных вызовов, позволяющих
пространству пользователя создавать производительные механизмы синхронизации,
такие как мьютексы (mutex), семафоры и условные переменные, в пространстве
пользователя. Стандартные библиотеки языка C, например glibc, используют его
как средство для реализации более высокоуровневых интерфейсов, таких как
pthreads.

futex2 — это последующая версия первоначального системного вызова futex,
разработанная для преодоления ограничений исходного интерфейса.

Пользовательский API
====================

``futex_waitv()``
-----------------

Ожидание на массиве futex'ов, пробуждение по любому из них::

  futex_waitv(struct futex_waitv *waiters, unsigned int nr_futexes,
              unsigned int flags, struct timespec *timeout, clockid_t clockid)

  struct futex_waitv {
        __u64 val;
        __u64 uaddr;
        __u32 flags;
        __u32 __reserved;
  };

Пространство пользователя задаёт массив структур struct futex_waitv (не более
128 элементов), используя ``uaddr`` для адреса, на котором нужно ожидать,
``val`` для ожидаемого значения и ``flags`` для указания типа (например,
private) и размера futex'а. Поле ``__reserved`` должно быть равно 0, но может
быть использовано для будущего расширения. Указатель на первый элемент массива
передаётся как ``waiters``. Недопустимый адрес для ``waiters`` или для любого
``uaddr`` приводит к возврату ``-EFAULT``.

Если в пространстве пользователя используются 32-битные указатели, следует
выполнить явное приведение типа, чтобы убедиться, что старшие биты обнулены.
``uintptr_t`` решает эту задачу и работает как для 32-битных, так и для
64-битных указателей.

``nr_futexes`` задаёт размер массива. Числа вне интервала [1, 128] приводят к
возврату системным вызовом ``-EINVAL``.

Аргумент ``flags`` системного вызова должен быть равен 0, но может быть
использован для будущего расширения.

Для каждого элемента массива ``waiters`` текущее значение по адресу ``uaddr``
сравнивается с ``val``. Если оно отличается, системный вызов отменяет всю
проделанную к этому моменту работу и возвращает ``-EAGAIN``. Если все проверки
и сравнения успешны, системный вызов ожидает до тех пор, пока не произойдёт
одно из следующего:

- Истекает таймаут — возвращается ``-ETIMEOUT``.
- Спящей задаче был отправлен сигнал — возвращается ``-ERESTARTSYS``.
- Какой-то futex из списка был разбужен — возвращается индекс разбуженного
  futex'а.

Пример использования интерфейса можно найти в
``tools/testing/selftests/futex/functional/futex_waitv.c``.

Таймаут
-------

Аргумент ``struct timespec *timeout`` является необязательным и указывает на
абсолютный таймаут. Необходимо указать тип используемых часов в аргументе
``clockid``. Поддерживаются ``CLOCK_MONOTONIC`` и ``CLOCK_REALTIME``. Этот
системный вызов принимает только 64-битные структуры timespec.

Типы futex'ов
-------------

Futex может быть либо private, либо shared. Private используется для процессов,
которые разделяют одно и то же пространство памяти, при этом виртуальный адрес
futex'а будет одинаковым для всех процессов. Это позволяет проводить
оптимизации в ядре. Чтобы использовать private futex'ы, необходимо указать флаг
``FUTEX_PRIVATE_FLAG`` во флагах futex'а. Процессы, которые не разделяют одно и
то же пространство памяти и, следовательно, могут иметь разные виртуальные
адреса для одного и того же futex'а (используя, например, разделяемую память на
основе файла), требуют иных внутренних механизмов для корректной постановки в
очередь. Это поведение по умолчанию, и оно работает как с private, так и с
shared futex'ами.

Futex'ы могут быть разных размеров: 8, 16, 32 или 64 бита. В настоящее время
поддерживается только futex размером 32 бита, и его необходимо указывать с
помощью флага ``FUTEX_32``.
