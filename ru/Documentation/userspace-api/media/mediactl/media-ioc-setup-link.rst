**************************
ioctl MEDIA_IOC_SETUP_LINK
**************************

Имя
===

MEDIA_IOC_SETUP_LINK - Изменение свойств связи

Синопсис
========

.. c:macro:: MEDIA_IOC_SETUP_LINK

``int ioctl(int fd, MEDIA_IOC_SETUP_LINK, struct media_link_desc *argp)``

Аргументы
=========

``fd``
    Файловый дескриптор, возвращаемый :c:func:`open()`.

``argp``
    Указатель на struct :c:type:`media_link_desc`.

Описание
========

Чтобы изменить свойства связи, приложения заполняют структуру struct
:c:type:`media_link_desc` информацией, идентифицирующей связь
(пады источника и приёмника) и новыми запрашиваемыми флагами связи. Затем
они вызывают ioctl MEDIA_IOC_SETUP_LINK с указателем на эту
структуру.

Единственное настраиваемое свойство — это флаг связи ``ENABLED``,
включающий/отключающий связь. Связи, помеченные флагом связи ``IMMUTABLE``,
не могут быть включены или отключены.

Настройка связи не имеет побочного эффекта на другие связи. Если включённая
связь на паде приёмника препятствует включению связи, драйвер возвращает
код ошибки ``EBUSY``.

Только связи, помеченные флагом связи ``DYNAMIC``, могут включаться/отключаться
во время потоковой передачи мультимедийных данных. Попытка включить или отключить
потоковую нединамическую связь вернёт код ошибки ``EBUSY``.

Если указанная связь не может быть найдена, драйвер возвращает код ошибки
``EINVAL``.

Возвращаемое значение
=====================

При успехе возвращается 0, при ошибке — -1, и переменная ``errno``
устанавливается соответствующим образом. Общие коды ошибок описаны в
главе :ref:`Общие коды ошибок <gen-errors>`.

EINVAL
    Структура struct :c:type:`media_link_desc` ссылается на
    несуществующую связь, либо связь является неизменяемой, и была сделана
    попытка изменить её конфигурацию.
