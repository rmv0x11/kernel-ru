******************************
ioctl MEDIA_REQUEST_IOC_REINIT
******************************

Имя
===

MEDIA_REQUEST_IOC_REINIT - Повторно инициализировать запрос

Синопсис
========

.. c:macro:: MEDIA_REQUEST_IOC_REINIT

``int ioctl(int request_fd, MEDIA_REQUEST_IOC_REINIT)``

Аргументы
=========

``request_fd``
    Файловый дескриптор, возвращённый :ref:`MEDIA_IOC_REQUEST_ALLOC`.

Описание
========

Если медиаустройство поддерживает :ref:`запросы <media-request-api>`, то
этот ioctl запроса может быть использован для повторной инициализации ранее
выделенного запроса.

Повторная инициализация запроса очищает любые существующие данные из запроса.
Это избавляет от необходимости выполнять :c:func:`close()` для завершённого
запроса и выделять новый запрос. Вместо этого завершённый запрос может быть
просто повторно инициализирован, и он готов к повторному использованию.

Запрос может быть повторно инициализирован только в том случае, если он либо
ещё не был поставлен в очередь, либо был поставлен в очередь и завершён. В
противном случае он установит ``errno`` в значение ``EBUSY``. Никакие другие
коды ошибок возвращены быть не могут.

Возвращаемое значение
=====================

При успешном выполнении возвращается 0, при ошибке -1, и переменная ``errno``
устанавливается соответствующим образом.

EBUSY
    Запрос поставлен в очередь, но ещё не завершён.
