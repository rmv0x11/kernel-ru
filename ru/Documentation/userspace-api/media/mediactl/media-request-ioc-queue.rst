*****************************
ioctl MEDIA_REQUEST_IOC_QUEUE
*****************************

Имя
===

MEDIA_REQUEST_IOC_QUEUE - Поставить запрос в очередь

Объявление
==========

.. c:macro:: MEDIA_REQUEST_IOC_QUEUE

``int ioctl(int request_fd, MEDIA_REQUEST_IOC_QUEUE)``

Аргументы
=========

``request_fd``
    Файловый дескриптор, возвращённый :ref:`MEDIA_IOC_REQUEST_ALLOC`.

Описание
========

Если медиаустройство поддерживает :ref:`запросы <media-request-api>`, то
этот ioctl-запрос может использоваться для постановки в очередь ранее выделенного запроса.

Если запрос был успешно поставлен в очередь, то файловый дескриптор может быть
:ref:`опрошен <request-func-poll>`, чтобы дождаться завершения запроса.

Если запрос уже был поставлен в очередь ранее, то возвращается ``EBUSY``.
Могут быть возвращены и другие ошибки, если содержимое запроса содержало
некорректные или несогласованные данные; список распространённых кодов ошибок
см. в следующем разделе. При ошибке состояние как запроса, так и драйвера остаётся неизменным.

После того как запрос поставлен в очередь, драйвер обязан корректно обрабатывать
ошибки, возникающие при применении запроса к аппаратному обеспечению.
Исключением является ошибка ``EIO``, которая сигнализирует о фатальной ошибке, требующей
от приложения остановить потоковую передачу для сброса состояния аппаратного обеспечения.

Не допускается смешивать постановку в очередь запросов с постановкой в очередь буферов напрямую
(без запроса). ``EBUSY`` будет возвращён, если первый буфер был
поставлен в очередь напрямую, а затем вы попытаетесь поставить в очередь запрос, или наоборот.

Запрос должен содержать как минимум один буфер, иначе этот ioctl
вернёт ошибку ``ENOENT``.

Возвращаемое значение
=====================

При успехе возвращается 0, при ошибке -1, и переменная ``errno`` устанавливается
соответствующим образом. Общие коды ошибок описаны в главе
:ref:`Общие коды ошибок <gen-errors>`.

EBUSY
    Запрос уже был поставлен в очередь, или приложение поставило в очередь первый
    буфер напрямую, но позднее попыталось использовать запрос. Не допускается
    смешивать эти два API.
ENOENT
    Запрос не содержал ни одного буфера. Все запросы обязаны
    иметь как минимум один буфер. Это также может быть возвращено, если в запросе
    отсутствует некоторая необходимая конфигурация.
ENOMEM
    Нехватка памяти при выделении внутренних структур данных для этого
    запроса.
EINVAL
    Запрос содержит некорректные данные.
EIO
    Аппаратное обеспечение находится в плохом состоянии. Для восстановления приложению необходимо
    остановить потоковую передачу для сброса состояния аппаратного обеспечения, а затем попытаться возобновить
    потоковую передачу.
