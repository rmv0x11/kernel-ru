Модель медиаустройства
======================

Обнаружение внутренней топологии устройства и его конфигурирование во время
выполнения — одна из целей media controller API. Для её достижения аппаратные
устройства и интерфейсы ядра Linux моделируются как объекты графа на
ориентированном графе. Типы объектов, составляющих граф, таковы:

-  **Сущность** (entity) — это базовый строительный блок медиа, аппаратный или
   программный. Она может соответствовать самым разнообразным логическим блокам,
   таким как физические аппаратные устройства (например, CMOS-сенсор), логические
   аппаратные устройства (строительный блок конвейера обработки изображений в
   System-on-Chip), каналы DMA или физические разъёмы.

-  **Интерфейс** (interface) — это представление в графе интерфейса userspace API
   ядра Linux, такого как узел устройства или файл sysfs, который управляет одной
   или несколькими сущностями в графе.

-  **Площадка** (pad) — это конечная точка соединения для данных, через которую
   сущность может взаимодействовать с другими сущностями. Данные (не
   ограничивающиеся видео), производимые сущностью, текут от выхода сущности к
   одному или нескольким входам сущностей. Площадки не следует путать с физическими
   выводами на границах микросхемы.

-  **Канал данных** (data link) — это ориентированное соединение «точка-точка»
   между двумя площадками, либо на одной и той же сущности, либо на разных
   сущностях. Данные текут от площадки-источника к площадке-приёмнику.

-  **Канал интерфейса** (interface link) — это двунаправленное управляющее
   соединение «точка-точка» между интерфейсом ядра Linux и сущностью.

- **Вспомогательный канал** (ancillary link) — это соединение «точка-точка»,
  обозначающее, что две сущности образуют единый логический блок. Например, это
  может представлять тот факт, что определённый сенсор камеры и контроллер линз
  образуют единый физический модуль, то есть данный контроллер линз управляет
  линзами этого сенсора камеры.
