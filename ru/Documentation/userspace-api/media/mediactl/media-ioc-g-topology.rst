**************************
ioctl MEDIA_IOC_G_TOPOLOGY
**************************

Имя
===

MEDIA_IOC_G_TOPOLOGY - Перечисление топологии графа и свойств элементов графа

Синопсис
========

.. c:macro:: MEDIA_IOC_G_TOPOLOGY

``int ioctl(int fd, MEDIA_IOC_G_TOPOLOGY, struct media_v2_topology *argp)``

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый :c:func:`open()`.

``argp``
    Указатель на struct :c:type:`media_v2_topology`.

Описание
========

Типичное использование этого ioctl — вызвать его дважды. При первом вызове
структура, определённая как struct
:c:type:`media_v2_topology`, должна быть обнулена. По
возврату, если не произошло ошибок, этот ioctl вернёт
``topology_version`` и общее число сущностей (entities), интерфейсов, площадок (pads)
и связей (links).

Перед вторым вызовом userspace должен выделить массивы для хранения
требуемых элементов графа, поместив указатели на них в
ptr_entities, ptr_interfaces, ptr_links и/или ptr_pads, оставив
остальные значения нетронутыми.

Если ``topology_version`` остался прежним, ioctl должен заполнить
требуемые массивы элементами медиаграфа.

Возвращаемое значение
=====================

При успехе возвращается 0, при ошибке -1, и переменная ``errno`` устанавливается
соответствующим образом. Универсальные коды ошибок описаны в главе
:ref:`Generic Error Codes <gen-errors>`.

ENOSPC
    Возвращается, когда одно или несколько из значений num_entities,
    num_interfaces, num_links или num_pads ненулевые и
    меньше фактического числа элементов внутри графа. Это
    может произойти, если ``topology_version`` изменился по сравнению с
    последним вызовом этого ioctl. Обычно userspace следует освободить
    область для указателей, обнулить элементы структуры и снова вызвать этот ioctl.
