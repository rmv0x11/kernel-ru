***************************
ioctl MEDIA_IOC_DEVICE_INFO
***************************

Имя
===

MEDIA_IOC_DEVICE_INFO - Запрос информации об устройстве

Синопсис
========

.. c:macro:: MEDIA_IOC_DEVICE_INFO

``int ioctl(int fd, MEDIA_IOC_DEVICE_INFO, struct media_device_info *argp)``

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый :c:func:`open()`.

``argp``
    Указатель на struct :c:type:`media_device_info`.

Описание
========

Все медиаустройства должны поддерживать ioctl ``MEDIA_IOC_DEVICE_INFO``.
Чтобы запросить информацию об устройстве, приложения вызывают ioctl с
указателем на struct :c:type:`media_device_info`. Драйвер
заполняет структуру и возвращает информацию приложению. Этот
ioctl никогда не завершается с ошибкой.

.. c:type:: media_device_info

.. tabularcolumns:: |p{4.4cm}|p{4.4cm}|p{8.5cm}|

.. flat-table:: struct media_device_info
    :header-rows:  0
    :stub-columns: 0
    :widths:       1 1 2

    *  -  char
       -  ``driver``\ [16]
       -  Имя драйвера, реализующего media API, в виде ASCII-строки,
	  завершающейся NUL. Версия драйвера хранится в поле
	  ``driver_version``.

	  Приложения, специфичные для конкретного драйвера, могут использовать
	  эту информацию для проверки идентичности драйвера. Она также полезна
	  для обхода известных ошибок или для идентификации драйверов в отчётах
	  об ошибках.

    *  -  char
       -  ``model``\ [32]
       -  Название модели устройства в виде UTF-8-строки, завершающейся NUL.
	  Версия устройства хранится в поле ``device_version`` и не
	  добавляется к названию модели.

    *  -  char
       -  ``serial``\ [40]
       -  Серийный номер в виде ASCII-строки, завершающейся NUL.

    *  -  char
       -  ``bus_info``\ [32]
       -  Расположение устройства в системе в виде ASCII-строки,
	  завершающейся NUL. Сюда входят имя типа шины (PCI, USB, ...) и
	  специфичный для шины идентификатор.

    *  -  __u32
       -  ``media_version``
       -  Версия media API, отформатированная с помощью макроса ``KERNEL_VERSION()``.

    *  -  __u32
       -  ``hw_revision``
       -  Ревизия аппаратного устройства в специфичном для драйвера формате.

    *  -  __u32
       -  ``driver_version``
       -  Версия драйвера медиаустройства, отформатированная с помощью
	  макроса ``KERNEL_VERSION()``. Вместе с полем ``driver`` она
	  идентифицирует конкретный драйвер.

    *  -  __u32
       -  ``reserved``\ [31]
       -  Зарезервировано для будущих расширений. Драйверы и приложения должны
	  установить этот массив в ноль.

Поля ``serial`` и ``bus_info`` могут использоваться для различения
нескольких экземпляров в остальном идентичного оборудования. Серийный
номер имеет приоритет, когда он указан, и может считаться уникальным.
Если серийный номер является пустой строкой, вместо него может
использоваться поле ``bus_info``. Поле ``bus_info`` гарантированно
уникально, но может изменяться при перезагрузках или отключении/повторном подключении устройства.

Возвращаемое значение
=====================

При успехе возвращается 0, при ошибке — -1, и переменная ``errno``
устанавливается соответствующим образом. Общие коды ошибок описаны в
главе :ref:`Общие коды ошибок <gen-errors>`.
