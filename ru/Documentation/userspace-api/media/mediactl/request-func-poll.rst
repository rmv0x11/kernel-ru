**************
request poll()
**************

Имя
===

request-poll - Ожидание события на файловом дескрипторе

Синопсис
========

.. code-block:: c

    #include <sys/poll.h>

.. c:function:: int poll( struct pollfd *ufds, unsigned int nfds, int timeout )

Аргументы
=========

``ufds``
   Список событий файловых дескрипторов, за которыми ведётся наблюдение

``nfds``
   Число событий файловых дескрипторов в массиве \*ufds

``timeout``
   Таймаут ожидания событий

Описание
========

С помощью функции :c:func:`poll()` приложения могут ожидать
завершения запроса.

В случае успеха :c:func:`poll()` возвращает число файловых
дескрипторов, которые были выбраны (то есть файловых дескрипторов, для которых
поле ``revents`` соответствующей структуры :c:type:`pollfd`
ненулевое). Файловый дескриптор запроса устанавливает флаг ``POLLPRI`` в ``revents``
при завершении запроса. По истечении таймаута функция возвращает
значение ноль, в случае ошибки возвращается -1, а переменная ``errno``
устанавливается соответствующим образом.

Попытка опросить запрос, который ещё не поставлен в очередь, приведёт к
установке флага ``POLLERR`` в ``revents``.

Возвращаемое значение
=====================

В случае успеха :c:func:`poll()` возвращает число
структур с ненулевыми полями ``revents`` или ноль, если истёк
таймаут вызова. В случае ошибки возвращается -1, а переменная ``errno``
устанавливается соответствующим образом:

``EBADF``
    Один или несколько элементов ``ufds`` задают недопустимый файловый
    дескриптор.

``EFAULT``
    ``ufds`` ссылается на недоступную область памяти.

``EINTR``
    Вызов был прерван сигналом.

``EINVAL``
    Значение ``nfds`` превышает значение ``RLIMIT_NOFILE``. Используйте
    ``getrlimit()`` для получения этого значения.
