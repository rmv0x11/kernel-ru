*****************************
ioctl MEDIA_IOC_REQUEST_ALLOC
*****************************

Имя
===

MEDIA_IOC_REQUEST_ALLOC - Выделение запроса

Описание прототипа
==================

.. c:macro:: MEDIA_IOC_REQUEST_ALLOC

``int ioctl(int fd, MEDIA_IOC_REQUEST_ALLOC, int *argp)``

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый :c:func:`open()`.

``argp``
    Указатель на целое число.

Описание
========

Если media-устройство поддерживает :ref:`запросы <media-request-api>`, то
этот ioctl может использоваться для выделения запроса. Если поддержки нет, то
``errno`` устанавливается в ``ENOTTY``. Доступ к запросу осуществляется через
файловый дескриптор, который возвращается в ``*argp``.

Если запрос был успешно выделен, то файловый дескриптор запроса
может быть передан в ioctl'ы :ref:`VIDIOC_QBUF <VIDIOC_QBUF>`,
:ref:`VIDIOC_G_EXT_CTRLS <VIDIOC_G_EXT_CTRLS>`,
:ref:`VIDIOC_S_EXT_CTRLS <VIDIOC_G_EXT_CTRLS>` и
:ref:`VIDIOC_TRY_EXT_CTRLS <VIDIOC_G_EXT_CTRLS>`.

Кроме того, запрос может быть поставлен в очередь вызовом
:ref:`MEDIA_REQUEST_IOC_QUEUE` и повторно инициализирован вызовом
:ref:`MEDIA_REQUEST_IOC_REINIT`.

Наконец, файловый дескриптор можно :ref:`опросить <request-func-poll>`, чтобы
дождаться завершения запроса.

Запрос остаётся выделенным до тех пор, пока все связанные с ним
файловые дескрипторы не будут закрыты с помощью :c:func:`close()`, а драйвер
больше не будет использовать запрос внутренне. См. также
:ref:`здесь <media-request-life-time>` для получения дополнительной информации.

Возвращаемое значение
=====================

При успехе возвращается 0, при ошибке -1, а переменная ``errno`` устанавливается
соответствующим образом. Общие коды ошибок описаны в главе
:ref:`Общие коды ошибок <gen-errors>`.

ENOTTY
    Драйвер не поддерживает запросы.
