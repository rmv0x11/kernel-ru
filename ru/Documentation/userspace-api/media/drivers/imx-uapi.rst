==========================================
Драйвер видеозахвата i.MX
==========================================

События
=======

.. _imx_api_ipuX_csiY:

ipuX_csiY
---------

Этот subdev может генерировать следующее событие при включении второго
IDMAC source pad:

- V4L2_EVENT_IMX_FRAME_INTERVAL_ERROR

Пользовательское приложение может подписаться на это событие через узел
subdev ipuX_csiY. Это событие генерируется монитором интервалов кадров
(Frame Interval Monitor) (подробнее о FIM см. ниже).

Элементы управления
===================

.. _imx_api_FIM:

Монитор интервалов кадров в ipuX_csiY
-------------------------------------

Декодеры adv718x иногда могут передавать испорченные поля во время
повторной синхронизации сигнала NTSC/PAL (слишком мало или слишком много
видеолиний). Когда это происходит, IPU запускает механизм восстановления
вертикальной синхронизации, добавляя по 1 фиктивной линии в каждый кадр,
что вызывает эффект «прокрутки» от изображения к изображению и может
длиться долго, прежде чем восстановится стабильное изображение. Либо иногда
этот механизм вообще не срабатывает, вызывая постоянное расщепление
изображения (один кадр содержит линии из двух последовательно захваченных
изображений).

Экспериментально было установлено, что во время прокрутки изображения
интервалы между кадрами (время, прошедшее между двумя EOF) падают ниже
номинального значения для текущего стандарта примерно на время одного
кадра (60 мкс) и остаются на этом значении, пока прокрутка не прекратится.

Хотя причина этого наблюдения неизвестна (механизм фиктивных линий IPU
должен приводить к увеличению интервалов на время 1 линии каждый кадр, а
не к фиксированному значению), мы можем использовать его для обнаружения
испорченных полей с помощью монитора интервалов кадров. Если FIM
обнаруживает некорректный интервал кадров, subdev ipuX_csiY отправит
событие V4L2_EVENT_IMX_FRAME_INTERVAL_ERROR. Пространство пользователя
может зарегистрироваться для получения уведомлений о событиях FIM на узле
устройства subdev ipuX_csiY. При получении этого события пространство
пользователя может выполнить перезапуск потоковой передачи, чтобы
исправить прокрутку/расщепление изображения.

Subdev ipuX_csiY включает пользовательские элементы управления для
подстройки некоторых параметров FIM. Если один из этих элементов
управления изменяется во время потоковой передачи, FIM будет сброшен и
продолжит работу с новыми настройками.

- V4L2_CID_IMX_FIM_ENABLE

Включение/отключение FIM.

- V4L2_CID_IMX_FIM_NUM

Сколько измерений интервала кадров усреднять перед сравнением с
номинальным интервалом кадров, сообщаемым сенсором. Это позволяет снизить
шум, вызванный задержкой обработки прерываний (interrupt latency).

- V4L2_CID_IMX_FIM_TOLERANCE_MIN

Если усреднённые интервалы отклоняются от номинального на эту величину в
микросекундах, отправляется событие V4L2_EVENT_IMX_FRAME_INTERVAL_ERROR.

- V4L2_CID_IMX_FIM_TOLERANCE_MAX

Если какие-либо интервалы превышают это значение, такие отсчёты
отбрасываются и не учитываются в среднем. Это можно использовать для
отбрасывания действительно больших ошибок интервала, которые могут быть
вызваны задержкой обработки прерываний при высокой нагрузке на систему.

- V4L2_CID_IMX_FIM_NUM_SKIP

Сколько кадров пропустить после сброса FIM или перезапуска потока, прежде
чем FIM начнёт усреднять интервалы.

- V4L2_CID_IMX_FIM_ICAP_CHANNEL / V4L2_CID_IMX_FIM_ICAP_EDGE

Эти элементы управления настраивают канал захвата входного сигнала (input
capture) в качестве метода измерения интервалов кадров. Этот метод
превосходит используемый по умолчанию метод измерения интервалов кадров
через прерывание EOF, поскольку он не подвержен ошибкам неопределённости,
вносимым задержкой обработки прерываний.

Захват входного сигнала требует аппаратной поддержки. Сигнал VSYNC должен
быть направлен на один из выводов (pads) канала захвата входного сигнала
i.MX6.

V4L2_CID_IMX_FIM_ICAP_CHANNEL настраивает, какой канал захвата входного
сигнала i.MX6 использовать. Это значение должно быть равно 0 или 1.

V4L2_CID_IMX_FIM_ICAP_EDGE настраивает, какой фронт сигнала будет
запускать события захвата входного сигнала. По умолчанию метод захвата
входного сигнала отключён значением IRQ_TYPE_NONE. Установите этот элемент
управления в IRQ_TYPE_EDGE_RISING, IRQ_TYPE_EDGE_FALLING или
IRQ_TYPE_EDGE_BOTH, чтобы включить захват входного сигнала, запускаемый по
заданному фронту (фронтам) сигнала.

Когда захват входного сигнала отключён, интервалы кадров будут измеряться
через прерывание EOF.
