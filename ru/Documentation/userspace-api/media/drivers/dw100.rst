Драйвер дьюарпинга DW100
========================

IP-ядро Vivante DW100 Dewarp Processor, имеющееся в SoC i.MX8MP, применяет
программируемое геометрическое преобразование к входному изображению для
коррекции искажений, вносимых объективами.

Функция преобразования предоставляется оборудованием в виде карты-сетки с
макроблоками размером 16x16 пикселей, индексируемыми по координатам вершин X, Y.
::

                          Image width
           <--------------------------------------->

      ^    .-------.-------.-------.-------.-------.
      |    | 16x16 |       |       |       |       |
   I  |    | pixel |       |       |       |       |
   m  |    | block |       |       |       |       |
   a  |    .-------.-------.-------.-------.-------.
   g  |    |       |       |       |       |       |
   e  |    |       |       |       |       |       |
      |    |       |       |       |       |       |
   h  |    .-------.-------.-------.-------.-------.
   e  |    |       |       |       |       |       |
   i  |    |       |       |       |       |       |
   g  |    |       |       |       |       |       |
   h  |    .-------.-------.-------.-------.-------.
   t  |    |       |       |       |       |       |
      |    |       |       |       |       |       |
      |    |       |       |       |       |       |
      v    '-------'-------'-------'-------'-------'

            Grid of Image Blocks for Dewarping Map


Каждый регистр координат x, y использует 16 бит для записи адреса координаты в
беззнаковом формате с фиксированной точкой 12.4 (UQ12.4).
::

    .----------------------.--------..----------------------.--------.
    |         31~20        | 19~16  ||         15~4         |  3~0   |
    |       (integer)      | (frac) ||       (integer)      | (frac) |
    '----------------------'--------''----------------------'--------'
    <-------------------------------><------------------------------->
                Y coordinate                     X coordinate

                           Remap Register Layout

Карта дьюарпинга устанавливается приложениями с помощью элемента управления
V4L2_CID_DW100_DEWARPING_16x16_VERTEX_MAP. Этот элемент управления содержит
массив значений u32, хранящих координаты (x, y) точки назначения для каждой
вершины сетки. Координата x хранится в 16 младших битах, а координата y —
в 16 старших битах.

Количество элементов в массиве должно соответствовать размеру изображения:

.. code-block:: C

    elems = (DIV_ROUND_UP(width, 16) + 1) * (DIV_ROUND_UP(height, 16) + 1);

Если элемент управления не был установлен приложением, драйвер использует
тождественное отображение (identity map).

Дополнительные сведения об операциях оборудования DW100 можно найти в
*главе 13.15 DeWarp* справочного руководства IMX8MP_.

m2m-драйвер Vivante DW100 реализует следующий специфичный для драйвера элемент управления:

``V4L2_CID_DW100_DEWARPING_16x16_VERTEX_MAP (__u32 array)``
    Задаёт драйверу DW100 его карту дьюарпинга (она же LUT) в виде blob-данных,
    как описано в *главе 13.15.2.3 Dewarping Remap* справочного руководства
    IMX8MP_, в форме динамического массива U32. Изображение делится на множество
    маленьких блоков 16x16. Если ширина/высота изображения не делится на 16,
    размер крайнего правого/нижнего блока равен остатку. Карта дьюарпинга
    сохраняет только координаты вершин блока. Карта-сетка дьюарпинга состоит из
    координат вершин по x и y. Каждый регистр координат x, y использует 16 бит
    (UQ12.4) для записи адреса координаты, при этом координата Y находится в
    старших битах, а X — в младших. Драйвер изменяет размерности этого элемента
    управления при смене формата приёмника (sink), отражая новое входное
    разрешение.
