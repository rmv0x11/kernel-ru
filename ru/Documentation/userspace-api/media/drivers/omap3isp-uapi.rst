События
-------

Драйвер OMAP 3 ISP поддерживает интерфейс событий V4L2 на subdev'ах CCDC и
статистики (AEWB, AF и гистограмма).

Subdev CCDC формирует событие типа V4L2_EVENT_FRAME_SYNC по прерыванию HS_VS,
которое используется для сигнализации о начале кадра. В более ранних версиях
этого драйвера для данной цели использовалось V4L2_EVENT_OMAP3ISP_HS_VS. Событие
генерируется точно в момент начала приёма первой строки кадра в модуле CCDC.
На событие можно подписаться на subdev'е CCDC.

(При использовании параллельного интерфейса необходимо обращать внимание на
корректную настройку полярности сигнала VS. Она автоматически корректна при
использовании последовательных приёмников.)

Каждый из subdev'ов статистики способен формировать события. Событие
генерируется всякий раз, когда буфер статистики может быть изъят из очереди
приложением пространства пользователя с помощью IOCTL VIDIOC_OMAP3ISP_STAT_REQ.
Доступны следующие события:

- V4L2_EVENT_OMAP3ISP_AEWB
- V4L2_EVENT_OMAP3ISP_AF
- V4L2_EVENT_OMAP3ISP_HIST

Тип данных события для этих ioctl'ов — struct omap3isp_stat_event_status. Если
при вычислении статистики возникает ошибка, событие будет сформировано как
обычно, но связанный с ним буфер статистики отсутствует. В этом случае поле
omap3isp_stat_event_status.buf_err устанавливается в ненулевое значение.


Приватные IOCTL'ы
-----------------

Драйвер OMAP 3 ISP поддерживает стандартные IOCTL'ы и элементы управления V4L2
там, где это возможно и практично. Однако значительная часть функций,
предоставляемых ISP, не подпадает под стандартные IOCTL'ы --- примерами таких
функций служат таблицы гаммы и настройка сбора статистики.

В общем случае для настройки каждого из блоков, содержащих аппаратно-зависимые
функции, имеется приватный ioctl.

Поддерживаются следующие приватные IOCTL'ы:

- VIDIOC_OMAP3ISP_CCDC_CFG
- VIDIOC_OMAP3ISP_PRV_CFG
- VIDIOC_OMAP3ISP_AEWB_CFG
- VIDIOC_OMAP3ISP_HIST_CFG
- VIDIOC_OMAP3ISP_AF_CFG
- VIDIOC_OMAP3ISP_STAT_REQ
- VIDIOC_OMAP3ISP_STAT_EN

Структуры параметров, используемые этими ioctl'ами, описаны в
include/linux/omap3isp.h. Подробные функции самого ISP, относящиеся к
конкретному блоку ISP, описаны в технических справочных руководствах
(Technical Reference Manuals, TRM) --- см. ссылки на них в конце документа.

Хотя использовать драйвер ISP без применения этих приватных IOCTL'ов возможно,
получить оптимальное качество изображения таким образом нельзя. Модули AEWB,
AF и гистограммы невозможно использовать без их настройки с помощью
соответствующих приватных IOCTL'ов.


IOCTL'ы блоков CCDC и предпросмотра
-----------------------------------

IOCTL'ы VIDIOC_OMAP3ISP_CCDC_CFG и VIDIOC_OMAP3ISP_PRV_CFG используются для
настройки, включения и отключения функций в блоках CCDC и предпросмотра
соответственно. Оба IOCTL'а управляют несколькими функциями в тех блоках,
которыми они управляют. IOCTL VIDIOC_OMAP3ISP_CCDC_CFG принимает в качестве
аргумента указатель на struct omap3isp_ccdc_update_config. Аналогично
VIDIOC_OMAP3ISP_PRV_CFG принимает указатель на struct
omap3isp_prev_update_config. Определение обеих структур доступно в [#]_.

Поле update в структурах указывает, нужно ли обновлять конфигурацию для
конкретной функции, а поле flag указывает, включать или отключать функцию.

Битовые маски update и flag принимают следующие значения. Каждая отдельная
функция в блоках CCDC и предпросмотра связана с флагом (либо disable, либо
enable; часть поля flag в структуре) и указателем на данные конфигурации этой
функции.

Допустимые значения для полей update и flag перечислены здесь для
VIDIOC_OMAP3ISP_CCDC_CFG. Значения можно объединять операцией ИЛИ для настройки
более чем одной функции в рамках одного вызова IOCTL.

- OMAP3ISP_CCDC_ALAW
- OMAP3ISP_CCDC_LPF
- OMAP3ISP_CCDC_BLCLAMP
- OMAP3ISP_CCDC_BCOMP
- OMAP3ISP_CCDC_FPC
- OMAP3ISP_CCDC_CULL
- OMAP3ISP_CCDC_CONFIG_LSC
- OMAP3ISP_CCDC_TBL_LSC

Соответствующие значения для VIDIOC_OMAP3ISP_PRV_CFG приведены здесь:

- OMAP3ISP_PREV_LUMAENH
- OMAP3ISP_PREV_INVALAW
- OMAP3ISP_PREV_HRZ_MED
- OMAP3ISP_PREV_CFA
- OMAP3ISP_PREV_CHROMA_SUPP
- OMAP3ISP_PREV_WB
- OMAP3ISP_PREV_BLKADJ
- OMAP3ISP_PREV_RGB2RGB
- OMAP3ISP_PREV_COLOR_CONV
- OMAP3ISP_PREV_YC_LIMIT
- OMAP3ISP_PREV_DEFECT_COR
- OMAP3ISP_PREV_GAMMABYPASS
- OMAP3ISP_PREV_DRK_FRM_CAPTURE
- OMAP3ISP_PREV_DRK_FRM_SUBTRACT
- OMAP3ISP_PREV_LENS_SHADING
- OMAP3ISP_PREV_NF
- OMAP3ISP_PREV_GAMMA

Связанный с функцией указатель конфигурации не может быть NULL при включении
функции. При отключении функции указатель конфигурации игнорируется.


IOCTL'ы блоков статистики
-------------------------

Subdev'ы статистики предлагают больше возможностей динамической настройки, чем
другие subdev'ы. Их можно включать, отключать и перенастраивать, когда конвейер
находится в состоянии потоковой передачи.

Блоки статистики всегда получают входные данные изображения от CCDC (поскольку
чтение из памяти гистограммы не реализовано). Пользователь может изымать
статистику из очереди через узлы subdev'ов статистики с помощью приватных
IOCTL'ов.

Приватные IOCTL'ы, предлагаемые subdev'ами AEWB, AF и гистограммы, во многом
отражают интерфейс уровня регистров, предоставляемый аппаратным обеспечением
ISP. Существуют аспекты, относящиеся исключительно к реализации драйвера, и они
обсуждаются далее.

VIDIOC_OMAP3ISP_STAT_EN
-----------------------

Этот приватный IOCTL включает/отключает модуль статистики. Если этот запрос
выполнен до начала потоковой передачи, он вступит в силу, как только конвейер
начнёт потоковую передачу. Если конвейер уже выполняет потоковую передачу,
запрос вступит в силу, как только CCDC перейдёт в состояние простоя.

VIDIOC_OMAP3ISP_AEWB_CFG, VIDIOC_OMAP3ISP_HIST_CFG и VIDIOC_OMAP3ISP_AF_CFG
---------------------------------------------------------------------------

Эти IOCTL'ы используются для настройки модулей. Они требуют от пользовательских
приложений глубокого знания аппаратного обеспечения. Пояснения по большинству
полей можно найти в TRM'ах OMAP. Два следующих поля, общих для всех
перечисленных выше конфигурационных приватных IOCTL'ов, требуют пояснения для
лучшего понимания, так как они не являются частью TRM.

omap3isp_[h3a_af/h3a_aewb/hist]\_config.buf_size:

Модули управляют своими буферами внутренне. Необходимый размер буфера для вывода
данных модуля зависит от запрошенной конфигурации. Хотя драйвер поддерживает
перенастройку во время потоковой передачи, он не поддерживает перенастройку,
требующую буфера большего размера, чем уже выделенный внутренне, если модуль
включён. В этом случае он вернёт -EBUSY. Чтобы избежать такой ситуации, либо
отключите/перенастройте/включите модуль, либо запросите необходимый размер
буфера во время первой настройки, пока модуль отключён.

Выделение внутреннего размера буфера учитывает минимальный размер буфера для
запрошенной конфигурации и значение, заданное в поле buf_size. Если значение
поля buf_size выходит за пределы диапазона размеров буфера [минимум, максимум],
оно ограничивается так, чтобы поместиться в этот диапазон. Затем драйвер
выбирает наибольшее значение. Скорректированное значение buf_size записывается
обратно для пользовательского приложения.

omap3isp_[h3a_af/h3a_aewb/hist]\_config.config_counter:

Поскольку конфигурация вступает в силу не синхронно с запросом, драйвер должен
предоставлять способ отслеживания этой информации для более точных данных. После
запроса конфигурации значение config_counter, возвращаемое приложению
пространства пользователя, будет уникальным значением, связанным с этим
запросом. Когда пользовательское приложение получает событие о доступности
буфера или когда запрашивается новый буфер, этот config_counter используется
для сопоставления данных буфера и конфигурации.

VIDIOC_OMAP3ISP_STAT_REQ
------------------------

Отправляет в пространство пользователя самые старые доступные данные из
внутренней очереди буферов и затем отбрасывает этот буфер. Поле
omap3isp_stat_data.frame_number соответствует полю field_count видеобуфера.
