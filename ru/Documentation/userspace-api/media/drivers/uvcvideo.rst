Драйвер Linux USB Video Class (UVC)
===================================

Этот файл документирует некоторые специфичные для драйвера аспекты драйвера UVC,
такие как специфичные для драйвера ioctl и заметки по реализации.

Вопросы и замечания можно направлять в список рассылки разработки Linux UVC по
адресу linux-media@vger.kernel.org.


Поддержка Extension Unit (XU)
-----------------------------

Введение
~~~~~~~~

Спецификация UVC допускает расширения от производителя через extension unit
(XU). Драйвер Linux UVC поддерживает элементы управления extension unit
(элементы управления XU) посредством двух отдельных механизмов:

  - через отображения элементов управления XU на элементы управления V4L2
  - через специфичный для драйвера интерфейс ioctl

Первый позволяет обычным приложениям V4L2 использовать элементы управления XU,
отображая определённые элементы управления XU на элементы управления V4L2,
которые затем появляются при обычном перечислении элементов управления.

Второй механизм требует от приложения специфичных для uvcvideo знаний для
доступа к элементам управления XU, но предоставляет пространству пользователя
всю концепцию UVC XU для максимальной гибкости.

Оба механизма дополняют друг друга и более подробно описаны ниже.


Отображения элементов управления
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Драйвер UVC предоставляет API, позволяющий приложениям пространства пользователя
определять так называемые отображения элементов управления во время выполнения.
Они позволяют отображать отдельные элементы управления XU или диапазоны байтов
из них на новые элементы управления V4L2. Такие элементы управления появляются и
функционируют в точности как обычные элементы управления V4L2 (т. е. штатные
элементы управления, такие как яркость, контрастность и т. д.). Однако чтение
или запись такого элемента управления V4L2 вызывает чтение или запись связанного
с ним элемента управления XU.

Ioctl, используемый для создания этих отображений элементов управления,
называется UVCIOC_CTRL_MAP. Предыдущие версии драйвера (до 0.2.0) требовали
предварительного использования другого ioctl (UVCIOC_CTRL_ADD) для передачи
информации об элементе управления XU драйверу UVC. Это больше не нужно,
поскольку новые версии uvcvideo запрашивают эту информацию непосредственно у
устройства.

Подробности об ioctl UVCIOC_CTRL_MAP см. в разделе под названием
"IOCTL reference" ниже.


3. Специфичный для драйвера интерфейс управления XU

Для приложений, которым необходимо обращаться к элементам управления XU
напрямую, например в целях тестирования, загрузки прошивки (firmware) или
доступа к двоичным элементам управления, предоставляется второй механизм доступа
к элементам управления XU в виде специфичного для драйвера ioctl, а именно
UVCIOC_CTRL_QUERY.

Вызов этого ioctl позволяет приложениям отправлять драйверу UVC запросы, которые
напрямую отображаются на низкоуровневые запросы управления UVC.

Для выполнения такого запроса необходимо знать идентификатор UVC unit ID
extension unit данного элемента управления и селектор элемента управления. Эта
информация либо должна быть жёстко задана в приложении, либо запрашивается иными
способами, например путём разбора дескриптора UVC или, если доступно, с помощью
API media controller для перечисления сущностей устройства.

Если размер элемента управления ещё не известен, необходимо сначала выполнить
запрос UVC_GET_LEN, чтобы иметь возможность выделить достаточно большой буфер и
установить размер буфера в правильное значение. Аналогично, чтобы выяснить,
являются ли запросы UVC_GET_CUR или UVC_SET_CUR допустимыми для данного элемента
управления, следует выполнить запрос UVC_GET_INFO. Биты 0 (GET supported) и 1
(SET supported) полученного байта указывают, какие запросы допустимы.

С добавлением ioctl UVCIOC_CTRL_QUERY ioctl UVCIOC_CTRL_GET и UVCIOC_CTRL_SET
стали устаревшими, поскольку их функциональность является подмножеством первого
ioctl. На данный момент они всё ещё поддерживаются, но разработчикам приложений
рекомендуется использовать вместо них UVCIOC_CTRL_QUERY.

Подробности об ioctl UVCIOC_CTRL_QUERY см. в разделе под названием
"IOCTL reference" ниже.


Безопасность
~~~~~~~~~~~~

В настоящее время API не предоставляет средств тонкого управления доступом.
Ioctl UVCIOC_CTRL_ADD и UVCIOC_CTRL_MAP требуют прав суперпользователя.

Предложения о том, как это улучшить, приветствуются.


Отладка
~~~~~~~

Для отладки проблем, связанных с элементами управления XU или элементами
управления в целом, рекомендуется включить бит UVC_TRACE_CONTROL в параметре
модуля 'trace'. Это приводит к записи дополнительного вывода в системный журнал.


IOCTL reference
~~~~~~~~~~~~~~~

UVCIOC_CTRL_MAP - Отображение элемента управления UVC на элемент управления V4L2
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^

Аргумент: struct uvc_xu_control_mapping

**Описание**:

	Этот ioctl создаёт отображение между элементом управления UVC или частью
	элемента управления UVC и элементом управления V4L2. После того как
	отображения определены, приложения пространства пользователя могут
	обращаться к элементам управления UVC, определённым производителем, через
	API элементов управления V4L2.

	Чтобы создать отображение, приложения заполняют структуру
	uvc_xu_control_mapping информацией о существующем элементе управления UVC,
	определённом с помощью UVCIOC_CTRL_ADD, и новом элементе управления V4L2.

	Элемент управления UVC может быть отображён на несколько элементов
	управления V4L2. Например, элемент управления UVC pan/tilt можно
	отобразить на отдельные элементы управления V4L2 pan и tilt. Элемент
	управления UVC делится на непересекающиеся поля с помощью полей 'size' и
	'offset', а затем независимо отображается на элемент управления V4L2.

	Для знаковых целочисленных элементов управления V4L2 поле data_type
	следует установить в UVC_CTRL_DATA_TYPE_SIGNED. Другие значения в
	настоящее время игнорируются.

**Возвращаемое значение**:

	При успехе возвращается 0. При ошибке возвращается -1, а errno
	устанавливается соответствующим образом.

	ENOMEM
		Недостаточно памяти для выполнения операции.
	EPERM
		Недостаточно привилегий (требуются права суперпользователя).
	EINVAL
		Нет такого элемента управления UVC.
	EOVERFLOW
		Запрошенные смещение и размер привели бы к переполнению элемента
		управления UVC.
	EEXIST
		Отображение уже существует.

**Типы данных**:

.. code-block:: none

	* struct uvc_xu_control_mapping

	__u32	id		V4L2 control identifier
	__u8	name[32]	V4L2 control name
	__u8	entity[16]	UVC extension unit GUID
	__u8	selector	UVC control selector
	__u8	size		V4L2 control size (in bits)
	__u8	offset		V4L2 control offset (in bits)
	enum v4l2_ctrl_type
		v4l2_type	V4L2 control type
	enum uvc_control_data_type
		data_type	UVC control data type
	struct uvc_menu_info
		*menu_info	Array of menu entries (for menu controls only)
	__u32	menu_count	Number of menu entries (for menu controls only)

	* struct uvc_menu_info

	__u32	value		Menu entry value used by the device
	__u8	name[32]	Menu entry name


	* enum uvc_control_data_type

	UVC_CTRL_DATA_TYPE_RAW		Raw control (byte array)
	UVC_CTRL_DATA_TYPE_SIGNED	Signed integer
	UVC_CTRL_DATA_TYPE_UNSIGNED	Unsigned integer
	UVC_CTRL_DATA_TYPE_BOOLEAN	Boolean
	UVC_CTRL_DATA_TYPE_ENUM		Enumeration
	UVC_CTRL_DATA_TYPE_BITMASK	Bitmask
	UVC_CTRL_DATA_TYPE_RECT		Rectangular area


UVCIOC_CTRL_QUERY - Запрос элемента управления UVC XU
^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
Аргумент: struct uvc_xu_control_query

**Описание**:

	Этот ioctl запрашивает элемент управления UVC XU, идентифицируемый по его
	extension unit ID и селектору элемента управления.

	Доступен ряд различных запросов, которые тесно соответствуют
	низкоуровневым запросам управления, описанным в спецификации UVC. Эти
	запросы таковы:

	UVC_GET_CUR
		Получить текущее значение элемента управления.
	UVC_GET_MIN
		Получить минимальное значение элемента управления.
	UVC_GET_MAX
		Получить максимальное значение элемента управления.
	UVC_GET_DEF
		Получить значение элемента управления по умолчанию.
	UVC_GET_RES
		Запросить разрешение элемента управления, т. е. шаг допустимых
		значений элемента управления.
	UVC_GET_LEN
		Запросить размер элемента управления в байтах.
	UVC_GET_INFO
		Запросить битовую карту информации элемента управления, которая
		указывает, поддерживаются ли запросы get/set.
	UVC_SET_CUR
		Обновить значение элемента управления.

	Приложения должны установить поле 'size' в правильную длину для элемента
	управления. Исключение составляют запросы UVC_GET_LEN и UVC_GET_INFO, для
	которых размер должен быть установлен в 2 и 1 соответственно. Поле 'data'
	должно указывать на корректный доступный для записи буфер, достаточно
	большой, чтобы вместить указанное число байтов данных.

	Данные копируются непосредственно с устройства без какой-либо обработки
	на стороне драйвера. Приложения отвечают за форматирование буфера данных,
	включая преобразование little-endian/big-endian. Это особенно важно для
	результата запросов UVC_GET_LEN, который всегда возвращается устройством
	как 16-битное целое в формате little-endian.

**Возвращаемое значение**:

	При успехе возвращается 0. При ошибке возвращается -1, а errno
	устанавливается соответствующим образом.

	ENOENT
		Устройство не поддерживает указанный элемент управления, либо
		указанный extension unit не найден.
	ENOBUFS
		Указан неверный размер буфера (слишком большой или слишком
		маленький).
	EINVAL
		Передан недопустимый код запроса.
	EBADRQC
		Указанный запрос не поддерживается данным элементом управления.
	EFAULT
		Указатель данных ссылается на недоступную область памяти.

**Типы данных**:

.. code-block:: none

	* struct uvc_xu_control_query

	__u8	unit		Extension unit ID
	__u8	selector	Control selector
	__u8	query		Request code to send to the device
	__u16	size		Control data size (in bytes)
	__u8	*data		Control value


Специфичные для драйвера элементы управления V4L2
-------------------------------------------------

Драйвер uvcvideo реализует следующие специфичные для UVC элементы управления:

``V4L2_CID_UVC_REGION_OF_INTEREST_RECT (struct)``
	Этот элемент управления определяет область интереса (ROI). ROI — это
	прямоугольная область, представленная структурой :c:type:`v4l2_rect`.
	Прямоугольник задаётся в глобальных координатах сенсора в пикселях. Он не
	зависит от поля зрения и не подвержен влиянию какого-либо кадрирования или
	масштабирования.

	Используйте ``V4L2_CTRL_WHICH_MIN_VAL`` и ``V4L2_CTRL_WHICH_MAX_VAL`` для
	запроса диапазона размеров прямоугольника.

	Задание ROI позволяет камере оптимизировать захват для этой области.
	Значение элемента управления ``V4L2_CID_REGION_OF_INTEREST_AUTO``
	определяет детальное поведение.

	Пример использования этого элемента управления можно найти в:
	`Chrome OS USB camera HAL.
	<https://chromium.googlesource.com/chromiumos/platform2/+/refs/heads/release-R121-15699.B/camera/hal/usb/>`


``V4L2_CID_UVC_REGION_OF_INTEREST_AUTO (bitmask)``
	Этот элемент управления определяет, какие (если таковые имеются)
	встроенные функции должны отслеживать область интереса (Region of
	Interest), заданную текущим значением
	``V4L2_CID_UVD__REGION_OF_INTEREST_RECT``.

	Максимальное значение — это маска, указывающая все поддерживаемые
	элементы управления Auto.

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    * - ``V4L2_UVC_REGION_OF_INTEREST_AUTO_EXPOSURE``
      - Установка этого бита заставляет автоматическую экспозицию отслеживать
	область интереса вместо всего изображения.
    * - ``V4L2_UVC_REGION_OF_INTEREST_AUTO_IRIS``
      - Установка этого бита заставляет автоматическую диафрагму отслеживать
        область интереса вместо всего изображения.
    * - ``V4L2_UVC_REGION_OF_INTEREST_AUTO_WHITE_BALANCE``
      - Установка этого бита заставляет автоматический баланс белого
	отслеживать область интереса вместо всего изображения.
    * - ``V4L2_UVC_REGION_OF_INTEREST_AUTO_FOCUS``
      - Установка этого бита заставляет автоматическую регулировку фокуса
        отслеживать область интереса вместо всего изображения.
    * - ``V4L2_UVC_REGION_OF_INTEREST_AUTO_FACE_DETECT``
      - Установка этого бита заставляет автоматическое распознавание лиц
        отслеживать область интереса вместо всего изображения.
    * - ``V4L2_UVC_REGION_OF_INTEREST_AUTO_DETECT_AND_TRACK``
      - Установка этого бита включает автоматическое распознавание и
	отслеживание лиц. Текущее значение
	``V4L2_CID_REGION_OF_INTEREST_RECT`` может быть обновлено драйвером.
    * - ``V4L2_UVC_REGION_OF_INTEREST_AUTO_IMAGE_STABILIZATION``
      - Установка этого бита включает автоматическую стабилизацию изображения.
	Текущее значение ``V4L2_CID_REGION_OF_INTEREST_RECT`` может быть
	обновлено драйвером.
    * - ``V4L2_UVC_REGION_OF_INTEREST_AUTO_HIGHER_QUALITY``
      - Установка этого бита включает автоматический захват указанной области
        с более высоким качеством, если это возможно.
