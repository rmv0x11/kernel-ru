Драйвер датчика камеры MIPI CCS
===============================

Драйвер датчика камеры MIPI CCS — это универсальный драйвер для датчиков камер,
совместимых с `MIPI CCS
<https://www.mipi.org/specifications/camera-command-set>`_. Он предоставляет три
суб-устройства, представляющие массив пикселей, биннер и масштабатор.

Поскольку возможности отдельных устройств различаются, драйвер предоставляет
интерфейсы на основе тех возможностей, которые имеются в аппаратуре.

См. также :ref:`документацию ядра по драйверу CCS <media-ccs-driver>`.

Суб-устройство массива пикселей
-------------------------------

Суб-устройство массива пикселей представляет матрицу пикселей датчика камеры, а
также функцию аналоговой обрезки (crop), присутствующую во многих совместимых
устройствах. Аналоговая обрезка настраивается с помощью ``V4L2_SEL_TGT_CROP`` на
исходном (source) контакте (pad) (0) сущности. Размер матрицы пикселей можно
получить, запросив цель ``V4L2_SEL_TGT_NATIVE_SIZE``.

Биннер
------

Суб-устройство биннера представляет функцию биннинга на датчике. Для этой цели на
приёмном (sink) контакте (pad) (0) поддерживается цель выбора
``V4L2_SEL_TGT_COMPOSE``.

Кроме того, если устройство не имеет функции масштабатора или цифровой обрезки,
исходный (source) контакт (pad) (1) предоставляет ещё один прямоугольник выбора
цифровой обрезки, который может обрезать только в конце строк и кадров.

Масштабатор
-----------

Суб-устройство масштабатора представляет функцию цифровой обрезки и масштабирования
датчика. Цель выбора V4L2 ``V4L2_SEL_TGT_CROP`` используется для настройки цифровой
обрезки на приёмном (sink) контакте (pad) (0), когда цифровая обрезка
поддерживается. Масштабирование также настраивается с помощью цели выбора
``V4L2_SEL_TGT_COMPOSE`` на приёмном (sink) контакте (pad) (0).

Кроме того, если суб-устройство масштабатора существует, его исходный (source)
контакт (pad) (1) предоставляет ещё один прямоугольник выбора цифровой обрезки,
который может обрезать только в конце строк и кадров.

Цифровая и аналоговая обрезка
-----------------------------

Под функцией цифровой обрезки понимается обрезка, которая фактически работает за
счёт отбрасывания части данных. Аналоговая обрезка, с другой стороны, означает, что
обрезанная информация никогда не извлекается. В случае датчиков камер аналоговые
данные никогда не считываются из тех элементов матрицы пикселей, которые находятся
вне настроенного прямоугольника выбора, обозначающего обрезку. Различие влияет на
тайминги устройства и, вероятно, также на энергопотребление.

Приватные элементы управления
------------------------------

Драйвер MIPI CCS реализует ряд приватных элементов управления в рамках
``V4L2_CID_USER_BASE_CCS`` для управления датчиками камер, совместимыми с MIPI CCS.

Модель аналогового усиления
~~~~~~~~~~~~~~~~~~~~~~~~~~~~

CCS определяет модель аналогового усиления, в которой усиление можно вычислить по
следующей формуле:

	gain = m0 * x + c0 / (m1 * x + c1)

Либо m0, либо c0 будет равно нулю. Константы, специфичные для устройства, можно
получить из следующих элементов управления:

	V4L2_CID_CCS_ANALOGUE_GAIN_M0
	V4L2_CID_CCS_ANALOGUE_GAIN_M1
	V4L2_CID_CCS_ANALOGUE_GAIN_C0
	V4L2_CID_CCS_ANALOGUE_GAIN_C1

Аналоговое усиление (``x`` в формуле) в этом случае управляется через
``V4L2_CID_ANALOGUE_GAIN``.

Альтернативная модель аналогового усиления
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

CCS определяет другую модель аналогового усиления, называемую альтернативным
аналоговым усилением. В этом случае формула для вычисления фактического усиления
состоит из линейной и экспоненциальной частей:

	gain = linear * 2 ^ exponent

Множители ``linear`` и ``exponent`` можно задать с помощью элементов управления
``V4L2_CID_CCS_ANALOGUE_LINEAR_GAIN`` и
``V4L2_CID_CCS_ANALOGUE_EXPONENTIAL_GAIN`` соответственно

Коррекция затенения
~~~~~~~~~~~~~~~~~~~~

Стандарт CCS поддерживает коррекцию затенения объектива (lens shading correction).
Этой возможностью можно управлять с помощью ``V4L2_CID_CCS_SHADING_CORRECTION``.
Кроме того, уровень коррекции яркости можно изменить с помощью
``V4L2_CID_CCS_LUMINANCE_CORRECTION_LEVEL``, где значение 0 означает отсутствие
коррекции, а 128 означает коррекцию яркости в углах до уровня на 10 % меньше, чем в
центре.

Для того чтобы уровень коррекции яркости имел эффект, коррекция затенения должна
быть включена.
