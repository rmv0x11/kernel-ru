****************************************************
ioctls CEC_ADAP_G_PHYS_ADDR and CEC_ADAP_S_PHYS_ADDR
****************************************************

Имя
===

CEC_ADAP_G_PHYS_ADDR, CEC_ADAP_S_PHYS_ADDR - получение или установка физического адреса

Краткое описание
================

.. c:macro:: CEC_ADAP_G_PHYS_ADDR

``int ioctl(int fd, CEC_ADAP_G_PHYS_ADDR, __u16 *argp)``

.. c:macro:: CEC_ADAP_S_PHYS_ADDR

``int ioctl(int fd, CEC_ADAP_S_PHYS_ADDR, __u16 *argp)``

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый :c:func:`open()`.

``argp``
    Указатель на CEC-адрес.

Описание
========

Чтобы запросить текущий физический адрес, приложения вызывают
:ref:`ioctl CEC_ADAP_G_PHYS_ADDR <CEC_ADAP_G_PHYS_ADDR>` с указателем на __u16, в который
драйвер сохраняет физический адрес.

Чтобы установить новый физический адрес, приложения сохраняют физический адрес в
__u16 и вызывают :ref:`ioctl CEC_ADAP_S_PHYS_ADDR <CEC_ADAP_S_PHYS_ADDR>` с указателем на
это целое число. :ref:`ioctl CEC_ADAP_S_PHYS_ADDR <CEC_ADAP_S_PHYS_ADDR>` доступен только в том случае, если
установлен ``CEC_CAP_PHYS_ADDR`` (иначе будет возвращён код ошибки ``ENOTTY``).
:ref:`ioctl CEC_ADAP_S_PHYS_ADDR <CEC_ADAP_S_PHYS_ADDR>` может быть вызван только
файловым дескриптором в режиме инициатора (см. :ref:`CEC_S_MODE`); в противном случае
будет возвращён код ошибки ``EBUSY``.

Чтобы сбросить существующий физический адрес, используйте ``CEC_PHYS_ADDR_INVALID``.
Адаптер перейдёт в ненастроенное состояние.

Если были определены типы логических адресов (см. :ref:`ioctl CEC_ADAP_S_LOG_ADDRS <CEC_ADAP_S_LOG_ADDRS>`),
то этот ioctl будет блокироваться, пока не будут заняты все
запрошенные логические адреса. Если файловый дескриптор находится в неблокирующем режиме,
то он не будет ждать занятия логических адресов, а просто вернёт 0.

При изменении физического адреса отправляется событие
:ref:`CEC_EVENT_STATE_CHANGE <CEC-EVENT-STATE-CHANGE>`.

Физический адрес — это 16-битное число, в котором каждая группа из 4 битов
представляет одну цифру физического адреса a.b.c.d, где старшие 4 бита
представляют 'a'. Корневое CEC-устройство (обычно телевизор)
имеет адрес 0.0.0.0. Каждое устройство, подключённое к входу
телевизора, имеет адрес a.0.0.0 (где 'a' ≥ 1), устройства, подключённые к ним
в свою очередь, имеют адреса a.b.0.0 и т. д. Таким образом, поддерживается топология глубиной
до 5 устройств. Физический адрес, который должно использовать устройство, хранится в
EDID приёмника.

Например, EDID каждого входа HDMI телевизора будет иметь
свой физический адрес вида a.0.0.0, который источники будут
считывать и использовать в качестве своего физического адреса.

Возвращаемое значение
=====================

При успехе возвращается 0, при ошибке -1, а переменная ``errno`` устанавливается
соответствующим образом. Общие коды ошибок описаны в главе
:ref:`Generic Error Codes <gen-errors>`.

:ref:`ioctl CEC_ADAP_S_PHYS_ADDR <CEC_ADAP_S_PHYS_ADDR>` может возвращать следующие
коды ошибок:

ENOTTY
    Возможность ``CEC_CAP_PHYS_ADDR`` не была установлена, поэтому этот ioctl не поддерживается.

EBUSY
    Другой файловый дескриптор находится в эксклюзивном режиме follower или initiator, либо файловый дескриптор
    находится в режиме ``CEC_MODE_NO_INITIATOR``.

EINVAL
    Физический адрес имеет неверный формат.
