**********
cec open()
**********

Имя
===

cec-open - Открытие устройства cec

Синопсис
========

.. code-block:: c

    #include <fcntl.h>

.. c:function:: int open( const char *device_name, int flags )

Аргументы
=========

``device_name``
    Устройство, которое требуется открыть.

``flags``
    Флаги открытия. Режим доступа должен быть ``O_RDWR``.

    Если задан флаг ``O_NONBLOCK``, то ioctl-вызовы
    :ref:`CEC_RECEIVE <CEC_RECEIVE>` и :ref:`CEC_DQEVENT <CEC_DQEVENT>`
    будут возвращать код ошибки ``EAGAIN``, когда нет доступного сообщения или события, а
    ioctl-вызовы :ref:`CEC_TRANSMIT <CEC_TRANSMIT>`,
    :ref:`CEC_ADAP_S_PHYS_ADDR <CEC_ADAP_S_PHYS_ADDR>` и
    :ref:`CEC_ADAP_S_LOG_ADDRS <CEC_ADAP_S_LOG_ADDRS>`
    все возвращают 0.

    Прочие флаги не оказывают влияния.

Описание
========

Чтобы открыть устройство cec, приложения вызывают :c:func:`open()` с
желаемым именем устройства. Функция не имеет побочных эффектов; конфигурация
устройства остаётся неизменной.

Если устройство открыто в режиме только для чтения, попытки изменить его
конфигурацию приведут к ошибке, а ``errno`` будет установлен в
EBADF.

Возвращаемое значение
=====================

В случае успеха :c:func:`open()` возвращает новый файловый дескриптор. В случае ошибки
возвращается -1, а ``errno`` устанавливается соответствующим образом. Возможные коды ошибок
включают:

``EACCES``
    Запрошенный доступ к файлу не разрешён.

``EMFILE``
    У процесса уже открыто максимальное количество файлов.

``ENFILE``
    Достигнут системный лимит на общее количество открытых файлов.

``ENOMEM``
    Недостаточно памяти ядра.

``ENODEV``
    Устройство не найдено или было удалено.
