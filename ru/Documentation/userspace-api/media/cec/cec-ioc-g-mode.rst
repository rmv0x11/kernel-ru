********************************
ioctls CEC_G_MODE and CEC_S_MODE
********************************

CEC_G_MODE, CEC_S_MODE - Получение или установка эксклюзивного использования адаптера CEC

Synopsis
========

.. c:macro:: CEC_G_MODE

``int ioctl(int fd, CEC_G_MODE, __u32 *argp)``

.. c:macro:: CEC_S_MODE

``int ioctl(int fd, CEC_S_MODE, __u32 *argp)``

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый :c:func:`open()`.

``argp``
    Указатель на режим CEC.

Описание
========

По умолчанию любой файловый дескриптор может использовать :ref:`CEC_TRANSMIT`, но,
чтобы приложения не мешали друг другу, должна быть возможность получить
эксклюзивный доступ к адаптеру CEC. Этот ioctl переводит файловый дескриптор
в режим инициатора и/или ведомого (follower), который может быть эксклюзивным
в зависимости от выбранного режима. Инициатор — это файловый дескриптор,
который используется для инициации сообщений, то есть он управляет другими
устройствами CEC. Ведомый — это файловый дескриптор, который принимает
сообщения, отправленные адаптеру CEC, и обрабатывает их. Один и тот же файловый
дескриптор может быть одновременно инициатором и ведомым, либо эти роли могут
быть распределены между двумя разными файловыми дескрипторами.

Когда принимается сообщение CEC, фреймворк CEC решает, как оно будет
обработано. Если сообщение является ответом на ранее переданное сообщение,
то ответ отправляется обратно файловому дескриптору, который его ожидает.
Кроме того, фреймворк CEC обработает его.

Если сообщение не является ответом, то фреймворк CEC сначала обработает его.
Если ведомого нет, то сообщение просто отбрасывается, и инициатору
отправляется feature abort, если фреймворк не смог его обработать. Если ведомый
есть, то сообщение передаётся ведомому, который использует
:ref:`ioctl CEC_RECEIVE <CEC_RECEIVE>` для извлечения нового сообщения из очереди.
Фреймворк ожидает, что ведомый будет принимать правильные решения.

Фреймворк CEC будет обрабатывать основные сообщения, если ведомый не запросит
иного. Ведомый может включить режим passthrough. В этом случае фреймворк CEC
будет передавать большинство основных сообщений без их обработки, и ведомому
придётся реализовывать эти сообщения самостоятельно. Есть некоторые сообщения,
которые ядро всегда будет обрабатывать, независимо от режима passthrough.
Подробности см. в :ref:`cec-core-processing`.

Если инициатора нет, то любой файловый дескриптор CEC может использовать
:ref:`ioctl CEC_TRANSMIT <CEC_TRANSMIT>`. Если есть эксклюзивный инициатор,
то только этот инициатор может вызывать :ref:`CEC_TRANSMIT`. Ведомый, разумеется,
всегда может вызывать :ref:`ioctl CEC_TRANSMIT <CEC_TRANSMIT>`.

Доступные режимы инициатора:

.. tabularcolumns:: |p{5.6cm}|p{0.9cm}|p{10.8cm}|

.. _cec-mode-initiator_e:

.. flat-table:: Режимы инициатора
    :header-rows:  0
    :stub-columns: 0
    :widths:       3 1 16

    * .. _`CEC-MODE-NO-INITIATOR`:

      - ``CEC_MODE_NO_INITIATOR``
      - 0x0
      - Это не инициатор, то есть он не может передавать сообщения CEC или
	вносить какие-либо иные изменения в адаптер CEC.
    * .. _`CEC-MODE-INITIATOR`:

      - ``CEC_MODE_INITIATOR``
      - 0x1
      - Это инициатор (значение по умолчанию при открытии устройства), и
	он может передавать сообщения CEC и вносить изменения в адаптер CEC,
	если только нет эксклюзивного инициатора.
    * .. _`CEC-MODE-EXCL-INITIATOR`:

      - ``CEC_MODE_EXCL_INITIATOR``
      - 0x2
      - Это эксклюзивный инициатор, и данный файловый дескриптор является
	единственным, кто может передавать сообщения CEC и вносить изменения в
	адаптер CEC. Если кто-то другой уже является эксклюзивным инициатором,
	то попытка стать им вернёт код ошибки ``EBUSY``.

Доступные режимы ведомого:

.. tabularcolumns:: |p{6.6cm}|p{0.9cm}|p{9.8cm}|

.. _cec-mode-follower_e:

.. cssclass:: longtable

.. flat-table:: Режимы ведомого
    :header-rows:  0
    :stub-columns: 0
    :widths:       3 1 16

    * .. _`CEC-MODE-NO-FOLLOWER`:

      - ``CEC_MODE_NO_FOLLOWER``
      - 0x00
      - Это не ведомый (значение по умолчанию при открытии устройства).
    * .. _`CEC-MODE-FOLLOWER`:

      - ``CEC_MODE_FOLLOWER``
      - 0x10
      - Это ведомый, и он будет принимать сообщения CEC, если только нет
	эксклюзивного ведомого. Нельзя стать ведомым, если
	:ref:`CEC_CAP_TRANSMIT <CEC-CAP-TRANSMIT>` не установлен или если был указан :ref:`CEC_MODE_NO_INITIATOR <CEC-MODE-NO-INITIATOR>`;
	в этом случае возвращается код ошибки ``EINVAL``.
    * .. _`CEC-MODE-EXCL-FOLLOWER`:

      - ``CEC_MODE_EXCL_FOLLOWER``
      - 0x20
      - Это эксклюзивный ведомый, и только данный файловый дескриптор будет
	принимать сообщения CEC для обработки. Если кто-то другой уже является
	эксклюзивным ведомым, то попытка стать им вернёт
	код ошибки ``EBUSY``. Нельзя стать ведомым, если
	:ref:`CEC_CAP_TRANSMIT <CEC-CAP-TRANSMIT>` не установлен или если был указан :ref:`CEC_MODE_NO_INITIATOR <CEC-MODE-NO-INITIATOR>`;
	в этом случае возвращается код ошибки ``EINVAL``.
    * .. _`CEC-MODE-EXCL-FOLLOWER-PASSTHRU`:

      - ``CEC_MODE_EXCL_FOLLOWER_PASSTHRU``
      - 0x30
      - Это эксклюзивный ведомый, и только данный файловый дескриптор будет
	принимать сообщения CEC для обработки. Кроме того, он переведёт
	устройство CEC в режим passthrough, позволяя эксклюзивному ведомому
	обрабатывать большинство основных сообщений вместо того, чтобы полагаться
	на фреймворк CEC. Если кто-то другой уже является эксклюзивным
	ведомым, то попытка стать им вернёт код ошибки ``EBUSY``. Нельзя стать
	ведомым, если :ref:`CEC_CAP_TRANSMIT <CEC-CAP-TRANSMIT>`
	не установлен или если был указан :ref:`CEC_MODE_NO_INITIATOR <CEC-MODE-NO-INITIATOR>`;
	в этом случае возвращается код ошибки ``EINVAL``.
    * .. _`CEC-MODE-MONITOR-PIN`:

      - ``CEC_MODE_MONITOR_PIN``
      - 0xd0
      - Переводит файловый дескриптор в режим мониторинга вывода (pin). Может
	использоваться только в сочетании с :ref:`CEC_MODE_NO_INITIATOR <CEC-MODE-NO-INITIATOR>`,
	иначе будет возвращён код ошибки ``EINVAL``.
	Этот режим требует, чтобы была установлена возможность :ref:`CEC_CAP_MONITOR_PIN <CEC-CAP-MONITOR-PIN>`,
	иначе возвращается код ошибки ``EINVAL``.
	Находясь в режиме мониторинга вывода, данный файловый дескриптор может принимать
	события ``CEC_EVENT_PIN_CEC_LOW`` и ``CEC_EVENT_PIN_CEC_HIGH``, чтобы видеть
	низкоуровневые переходы вывода CEC. Это очень полезно для отладки.
	Этот режим разрешён только если процесс обладает возможностью ``CAP_NET_ADMIN``.
	Если она не установлена, то возвращается код ошибки ``EPERM``.
    * .. _`CEC-MODE-MONITOR`:

      - ``CEC_MODE_MONITOR``
      - 0xe0
      - Переводит файловый дескриптор в режим мониторинга. Может использоваться только в
	сочетании с :ref:`CEC_MODE_NO_INITIATOR <CEC-MODE-NO-INITIATOR>`,
	иначе будет возвращён код ошибки ``EINVAL``.
	В режиме мониторинга будут сообщаться все сообщения, которые передаёт это
	устройство CEC, и все сообщения, которые оно принимает (как широковещательные
	сообщения, так и направленные сообщения для одного из его логических адресов).
	Это очень полезно для отладки. Это разрешено только
	если процесс обладает возможностью ``CAP_NET_ADMIN``. Если
	она не установлена, то возвращается код ошибки ``EPERM``.
    * .. _`CEC-MODE-MONITOR-ALL`:

      - ``CEC_MODE_MONITOR_ALL``
      - 0xf0
      - Переводит файловый дескриптор в режим 'monitor all'. Может использоваться только
	в сочетании с :ref:`CEC_MODE_NO_INITIATOR <CEC-MODE-NO-INITIATOR>`, иначе
	будет возвращён код ошибки ``EINVAL``. В режиме 'monitor all' будут сообщаться все сообщения,
	которые передаёт это устройство CEC, и все сообщения, которые оно принимает,
	включая направленные сообщения для других устройств CEC. Это
	очень полезно для отладки, но не все устройства это поддерживают. Этот
	режим требует, чтобы была установлена возможность :ref:`CEC_CAP_MONITOR_ALL <CEC-CAP-MONITOR-ALL>`,
	иначе возвращается код ошибки ``EINVAL``. Это разрешено только если
	процесс обладает возможностью ``CAP_NET_ADMIN``. Если она не
	установлена, то возвращается код ошибки ``EPERM``.

Return Value
============

В случае успеха возвращается 0, в случае ошибки -1, и переменная ``errno``
устанавливается соответствующим образом. Общие коды ошибок описаны в главе
:ref:`Generic Error Codes <gen-errors>`.

:ref:`ioctl CEC_S_MODE <CEC_S_MODE>` может вернуть следующие
коды ошибок:

EINVAL
    Запрошенный режим недопустим.

EPERM
    Запрошен режим мониторинга, но процесс не обладает возможностью ``CAP_NET_ADMIN``.

EBUSY
    Кто-то другой уже является эксклюзивным ведомым или инициатором.
