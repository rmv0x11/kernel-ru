**********
cec poll()
**********

Имя
===

cec-poll - Ожидание некоторого события на файловом дескрипторе

Синопсис
========

.. code-block:: c

    #include <sys/poll.h>

.. c:function:: int poll( struct pollfd *ufds, unsigned int nfds, int timeout )

Аргументы
=========

``ufds``
   Список событий FD, за которыми нужно следить

``nfds``
   Количество событий FD в массиве \*ufds

``timeout``
   Тайм-аут ожидания событий

Описание
========

С помощью функции :c:func:`poll()` приложения могут ожидать события
CEC.

В случае успеха :c:func:`poll()` возвращает количество файловых
дескрипторов, которые были выбраны (то есть файловых дескрипторов, для
которых поле ``revents`` соответствующей структуры :c:type:`pollfd`
ненулевое). Устройства CEC устанавливают флаги ``POLLIN`` и ``POLLRDNORM`` в
поле ``revents``, если в очереди приёма есть сообщения. Если в очереди
передачи есть место для новых сообщений, то устанавливаются флаги ``POLLOUT`` и
``POLLWRNORM``. Если в очереди событий есть события,
то устанавливается флаг ``POLLPRI``. Когда время ожидания функции истекает, она возвращает
значение, равное нулю; при ошибке она возвращает -1, а переменной ``errno``
присваивается соответствующее значение.

Подробнее см. справочную страницу :c:func:`poll()`.

Возвращаемое значение
=====================

В случае успеха :c:func:`poll()` возвращает количество структур, имеющих
ненулевые поля ``revents``, или ноль, если время ожидания вызова истекло. При ошибке возвращается -1,
а переменной ``errno`` присваивается соответствующее значение:

``EBADF``
    Один или несколько членов ``ufds`` задают недопустимый файловый
    дескриптор.

``EFAULT``
    ``ufds`` ссылается на недоступную область памяти.

``EINTR``
    Вызов был прерван сигналом.

``EINVAL``
    Значение ``nfds`` превышает значение ``RLIMIT_NOFILE``. Используйте
    ``getrlimit()`` для получения этого значения.
