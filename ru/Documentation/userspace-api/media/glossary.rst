=========
Глоссарий
=========

.. note::

   Цель этого раздела — стандартизировать термины, используемые в документации
   userspace API мультимедиа. Это работа в процессе (Work In Progress).

.. Please keep the glossary entries in alphabetical order

.. glossary::

    Bridge Driver
	:term:`Device Driver`, который реализует основную логику взаимодействия
	с мультимедийным оборудованием.

    CEC API
	**Consumer Electronics Control API**

	API, предназначенный для приёма и передачи данных через интерфейс
	HDMI CEC.

	См. :ref:`cec`.

    Data Unit

	Единица данных, передаваемая по шине. На параллельных шинах единица данных
	состоит из одного или нескольких связанных отсчётов, тогда как на
	последовательных шинах единица данных является логической. Если единица
	данных представляет собой данные изображения, её также можно называть
	пикселем.

    Device Driver
	Часть ядра Linux, которая реализует поддержку аппаратного компонента.

    Device Node
	Символьный узел устройства в файловой системе, используемый для управления
	драйвером ядра и передачи данных в него и из него.

    Digital TV API
	**Previously known as DVB API**

	API, предназначенный для управления подмножеством :term:`Media Hardware`,
	которое реализует цифровое ТВ (например, DVB, ATSC, ISDB и т. д.).

	См. :ref:`dvbapi`.

    DSP
        **Digital Signal Processor**

	Специализированный :term:`Microprocessor`, архитектура которого
	оптимизирована для операционных нужд цифровой обработки сигналов.

    FPGA
	**Field-programmable Gate Array**

	Интегральная схема (:term:`IC`), спроектированная так, чтобы её можно было
	конфигурировать заказчику или разработчику после изготовления.

	См. https://en.wikipedia.org/wiki/Field-programmable_gate_array.

    Hardware Component
	Подмножество :term:`Media Hardware`. Например, устройство :term:`I²C` или
	:term:`SPI`, либо :term:`IP Block` внутри :term:`SoC` или :term:`FPGA`.

    Hardware Peripheral
	Группа аппаратных компонентов (:term:`hardware components <Hardware Component>`),
	которые вместе образуют более крупную функциональную периферию,
	взаимодействующую с пользователем. Например, :term:`IP Block` :term:`ISP`
	в составе :term:`SoC` и внешние датчики камеры вместе образуют аппаратную
	периферию камеры.

	Также известна как :term:`Peripheral`.

    I²C
	**Inter-Integrated Circuit**

	Многомастерная, многоведомая, с коммутацией пакетов, несимметричная,
	последовательная компьютерная шина, используемая для управления некоторыми
	аппаратными компонентами, такими как аппаратные компоненты субустройств.

	См. http://www.nxp.com/docs/en/user-guide/UM10204.pdf.

    IC
	**Integrated circuit**

	Набор электронных схем на одном небольшом плоском кусочке полупроводникового
	материала, обычно кремния.

	Также известна как чип.

    IP Block
	**Intellectual property core**

	В электронном проектировании полупроводниковое ядро интеллектуальной
	собственности — это многократно используемая единица логики, ячейки или
	топологии интегральной схемы, являющаяся интеллектуальной собственностью
	одной стороны. IP-блоки могут лицензироваться другой стороне либо
	принадлежать и использоваться единолично одной стороной.

	См. https://en.wikipedia.org/wiki/Semiconductor_intellectual_property_core).

    ISP
	**Image Signal Processor**

	Специализированный процессор, реализующий набор алгоритмов для обработки
	данных изображения. ISP могут реализовывать алгоритмы коррекции виньетирования
	объектива, дебайеризации, масштабирования и преобразования формата пикселей,
	а также формировать статистику для использования управляющими алгоритмами
	(например, автоматической экспозиции, баланса белого и фокусировки).

    Media API
	Набор userspace API, используемых для управления мультимедийным оборудованием.
	Состоит из:

	  - :term:`CEC API`;
	  - :term:`Digital TV API`;
	  - :term:`MC API`;
	  - :term:`RC API`; и
	  - :term:`V4L2 API`.

	См. Documentation/userspace-api/media/index.rst.

    MC API
	**Media Controller API**

	API, предназначенный для предоставления и управления взаимосвязями между
	мультимедийными устройствами и субустройствами.

	См. :ref:`media_controller`.

    MC-centric
	Драйвер устройства :term:`V4L2 Hardware`, которому требуется :term:`MC API`.

	У таких драйверов установлено поле device_caps ``V4L2_CAP_IO_MC``
	(см. :ref:`VIDIOC_QUERYCAP`).

	Подробнее см. :ref:`v4l2_hardware_control`.

    Media Hardware
	Подмножество оборудования, поддерживаемого Linux Media API.

	Сюда входит оборудование захвата и воспроизведения аудио и видео, цифровое
	и аналоговое ТВ, датчики камер, ISP, пульты дистанционного управления,
	кодеки, HDMI Consumer Electronics Control, захват HDMI и т. д.

    Microprocessor
	Электронная схема, которая выполняет инструкции компьютерной программы,
	производя базовые арифметические, логические, управляющие операции и операции
	ввода-вывода (I/O), задаваемые инструкциями, на единой интегральной схеме.

    Peripheral
	То же, что и :term:`Hardware Peripheral`.

    RC API
	**Remote Controller API**

	API, предназначенный для приёма и передачи данных от пультов дистанционного
	управления.

	См. :ref:`remote_controllers`.

    SMBus
	Подмножество I²C, которое определяет более строгое использование шины.

    SPI
	**Serial Peripheral Interface Bus**

	Спецификация интерфейса синхронной последовательной связи, используемого для
	связи на коротких расстояниях, в первую очередь во встраиваемых системах.

    SoC
	**System on a Chip**

	Интегральная схема, которая объединяет все компоненты компьютера или другой
	электронной системы.

    Stream
	Отдельный поток данных (данных изображения или метаданных) от исходного
	источника до конечного приёмника. Исходным источником может быть, например,
	датчик изображения, а конечным приёмником — например, буфер в памяти.

    V4L2 API
	**V4L2 userspace API**

	Userspace API, определённый в :ref:`v4l2spec`, который используется для
	управления оборудованием V4L2.

    V4L2 Device Node
	:term:`Device Node`, связанный с драйвером V4L.

	Именование узлов устройств V4L2 описано в :ref:`v4l2_device_naming`.

    V4L2 Hardware
	Часть мультимедийного оборудования, поддерживаемая :term:`V4L2 API`.

    V4L2 Sub-device
	Аппаратные компоненты V4L2, которые не управляются :term:`Bridge Driver`.
	См. :ref:`subdev`.

    Video-node-centric
	Драйвер устройства V4L2, которому не требуется media controller для работы.

	У таких драйверов поле device_caps ``V4L2_CAP_IO_MC`` сброшено
	(см. :ref:`VIDIOC_QUERYCAP`).

    V4L2 Sub-device API
	Часть :term:`V4L2 API`, которая управляет субустройствами V4L2
	(:term:`V4L2 sub-devices <V4L2 Sub-device>`), такими как датчики,
	приёмники HDMI, масштабаторы, деинтерлейсеры.

	Подробнее см. :ref:`v4l2_hardware_control`.
