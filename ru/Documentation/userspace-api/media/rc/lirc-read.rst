***********
LIRC read()
***********

Имя
===

lirc-read - Чтение из устройства LIRC

Синопсис
========

.. code-block:: c

    #include <unistd.h>

.. c:function:: ssize_t read( int fd, void *buf, size_t count )

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый ``open()``.

``buf``
   Заполняемый буфер

``count``
   Максимальное число байтов для чтения

Описание
========

:c:func:`read()` пытается прочитать до ``count`` байтов из файлового
дескриптора ``fd`` в буфер, начинающийся с ``buf``. Если ``count`` равно нулю,
:c:func:`read()` возвращает ноль и не имеет других результатов. Если ``count``
больше ``SSIZE_MAX``, результат не определён.

Точный формат данных зависит от того, какой режим (:ref:`lirc_modes`) использует
драйвер. Используйте :ref:`lirc_get_features`, чтобы получить поддерживаемый режим,
и используйте :ref:`lirc_set_rec_mode` для установки текущего активного режима.

Режим :ref:`LIRC_MODE_MODE2 <lirc-mode-mode2>` предназначен для необработанного
(raw) ИК-сигнала, в котором из символьного устройства считываются пакеты,
содержащие значение типа unsigned int, описывающее ИК-сигнал.

В качестве альтернативы может быть доступен :ref:`LIRC_MODE_SCANCODE <lirc-mode-scancode>`;
в этом режиме скан-коды декодируются либо программными декодерами, либо
аппаратными декодерами. Поле :c:type:`rc_proto` устанавливается в значение
:ref:`ИК-протокола <Remote_controllers_Protocols>`,
используемого для передачи, ``scancode`` — в декодированный скан-код,
а ``keycode`` устанавливается в код клавиши или ``KEY_RESERVED``.

Возвращаемое значение
=====================

В случае успеха возвращается число прочитанных байтов. Не является ошибкой,
если это число меньше запрошенного числа байтов или объёма данных, необходимых
для одного кадра. В случае ошибки возвращается -1, а переменной ``errno``
присваивается соответствующее значение.
