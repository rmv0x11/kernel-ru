********************************************************
ioctl-вызовы LIRC_GET_MIN_TIMEOUT и LIRC_GET_MAX_TIMEOUT
********************************************************

Имя
===

LIRC_GET_MIN_TIMEOUT / LIRC_GET_MAX_TIMEOUT - получить возможный диапазон
тайм-аутов для приёма ИК-сигнала.

Синопсис
========

.. c:macro:: LIRC_GET_MIN_TIMEOUT

``int ioctl(int fd, LIRC_GET_MIN_TIMEOUT, __u32 *timeout)``

.. c:macro:: LIRC_GET_MAX_TIMEOUT

``int ioctl(int fd, LIRC_GET_MAX_TIMEOUT, __u32 *timeout)``

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый open().

``timeout``
    Тайм-аут в микросекундах.

Описание
========

Некоторые устройства имеют внутренние таймеры, которые можно использовать
для обнаружения длительного отсутствия ИК-активности. Это может помочь lircd
определить, что ИК-сигнал завершён, и ускорить процесс декодирования.
Возвращает целочисленное значение с минимальным/максимальным тайм-аутом,
который можно установить.

.. note::

   У некоторых устройств тайм-аут фиксирован; в этом случае
   оба ioctl-вызова возвращают одно и то же значение, даже если тайм-аут
   нельзя изменить с помощью :ref:`LIRC_SET_REC_TIMEOUT`.

Возвращаемое значение
=====================

При успехе возвращается 0, при ошибке -1, и переменной ``errno`` присваивается
соответствующее значение. Общие коды ошибок описаны в главе
:ref:`Общие коды ошибок <gen-errors>`.
