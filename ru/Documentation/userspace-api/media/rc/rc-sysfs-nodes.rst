.. _remote_controllers_sysfs_nodes:

********************************************
Узлы sysfs пульта дистанционного управления
********************************************

Как определено в Documentation/ABI/testing/sysfs-class-rc, это
узлы sysfs, которые управляют пультами дистанционного управления (Remote Controllers):


.. _sys_class_rc:

/sys/class/rc/
==============

Подкаталог класса ``/sys/class/rc/`` принадлежит ядру подсистемы
пультов дистанционного управления и предоставляет интерфейс sysfs для настройки
инфракрасных приёмников пультов дистанционного управления.


.. _sys_class_rc_rcN:

/sys/class/rc/rcN/
==================

Каталог ``/sys/class/rc/rcN`` создаётся для каждого устройства-приёмника
пульта дистанционного управления, где N — это номер приёмника.


.. _sys_class_rc_rcN_protocols:

/sys/class/rc/rcN/protocols
===========================

Чтение этого файла возвращает список доступных протоколов, что-то вроде::

	rc5 [rc6] nec jvc [sony]

Включённые протоколы показываются в квадратных скобках [].

Запись "+proto" добавит протокол в список включённых протоколов.

Запись "-proto" удалит протокол из списка включённых
протоколов.

Запись "proto" включит только "proto".

Запись "none" отключит все протоколы.

Запись завершается неудачей с ``EINVAL``, если используется недопустимая комбинация протоколов или неизвестное
имя протокола.


.. _sys_class_rc_rcN_filter:

/sys/class/rc/rcN/filter
========================

Задаёт ожидаемое значение фильтра сканкодов.

Используйте в сочетании с ``/sys/class/rc/rcN/filter_mask`` для задания
ожидаемого значения битов, установленных в маске фильтра. Если оборудование
это поддерживает, то сканкоды, не совпадающие с фильтром, будут
игнорироваться. В противном случае запись завершится неудачей с ошибкой.

Это значение может быть сброшено в 0, если текущий протокол изменён.


.. _sys_class_rc_rcN_filter_mask:

/sys/class/rc/rcN/filter_mask
=============================

Задаёт маску фильтра сканкодов — биты для сравнения. Используйте в сочетании
с ``/sys/class/rc/rcN/filter`` для задания битов сканкода, которые
должны сравниваться с ожидаемым значением. Значение 0 отключает
фильтр, чтобы разрешить обработку всех корректных сканкодов.

Если оборудование это поддерживает, то сканкоды, не совпадающие с фильтром,
будут игнорироваться. В противном случае запись завершится неудачей с ошибкой.

Это значение может быть сброшено в 0, если текущий протокол изменён.


.. _sys_class_rc_rcN_wakeup_protocols:

/sys/class/rc/rcN/wakeup_protocols
==================================

Чтение этого файла возвращает список доступных протоколов для использования в
фильтре пробуждения, что-то вроде::

	rc-5 nec nec-x rc-6-0 rc-6-6a-24 [rc-6-6a-32] rc-6-mce

Обратите внимание, что перечисляются варианты протоколов, поэтому ``nec``, ``sony``, ``rc-5``, ``rc-6``
имеют перечисленные кодировки с различной длиной битов, если они доступны.

Обратите внимание, что перечисляются все варианты протоколов.

Включённый протокол пробуждения показывается в квадратных скобках [].

Одновременно может быть выбран только один протокол.

Запись "proto" приведёт к использованию "proto" для событий пробуждения.

Запись "none" отключит пробуждение.

Запись завершается неудачей с ``EINVAL``, если используется недопустимая комбинация протоколов или неизвестное
имя протокола, либо если пробуждение не поддерживается оборудованием.


.. _sys_class_rc_rcN_wakeup_filter:

/sys/class/rc/rcN/wakeup_filter
===============================

Задаёт ожидаемое значение фильтра пробуждения по сканкодам. Используйте в сочетании с
``/sys/class/rc/rcN/wakeup_filter_mask`` для задания ожидаемого значения
битов, установленных в маске фильтра пробуждения, для инициирования события пробуждения системы.

Если оборудование это поддерживает и wakeup_filter_mask не равна 0, то
сканкоды, совпадающие с фильтром, будут пробуждать систему, например, из режима приостановки
в ОЗУ (suspend to RAM) или из выключенного состояния. В противном случае запись завершится неудачей с ошибкой.

Это значение может быть сброшено в 0, если протокол пробуждения изменён.


.. _sys_class_rc_rcN_wakeup_filter_mask:

/sys/class/rc/rcN/wakeup_filter_mask
====================================

Задаёт маску фильтра пробуждения по сканкодам — биты для сравнения. Используйте в
сочетании с ``/sys/class/rc/rcN/wakeup_filter`` для задания битов
сканкода, которые должны сравниваться с ожидаемым значением для
инициирования события пробуждения системы.

Если оборудование это поддерживает и wakeup_filter_mask не равна 0, то
сканкоды, совпадающие с фильтром, будут пробуждать систему, например, из режима приостановки
в ОЗУ (suspend to RAM) или из выключенного состояния. В противном случае запись завершится неудачей с ошибкой.

Это значение может быть сброшено в 0, если протокол пробуждения изменён.
