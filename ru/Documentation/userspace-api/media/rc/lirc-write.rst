************
LIRC write()
************

Имя
===

lirc-write - Запись в устройство LIRC

Синопсис
========

.. code-block:: c

    #include <unistd.h>

.. c:function:: ssize_t write( int fd, void *buf, size_t count )

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый функцией ``open()``.

``buf``
    Буфер с данными для записи

``count``
    Количество байтов в буфере

Описание
========

:c:func:`write()` записывает до ``count`` байтов в устройство,
на которое ссылается файловый дескриптор ``fd``, из буфера, начинающегося с
``buf``.

Точный формат данных зависит от того, в каком режиме находится драйвер; используйте
:ref:`lirc_get_features` для получения поддерживаемых режимов и
:ref:`lirc_set_send_mode` для установки режима.

В режиме :ref:`LIRC_MODE_PULSE <lirc-mode-PULSE>` данные, записываемые в
символьное устройство, представляют собой последовательность целочисленных значений pulse/space. Импульсы (pulse) и паузы (space)
обозначаются только неявно — по их позиции. Данные должны начинаться и заканчиваться
импульсом, поэтому данные всегда должны содержать нечётное количество
отсчётов. Функция записи блокируется до тех пор, пока данные не будут переданы
аппаратурой. Если предоставлено больше данных, чем аппаратура может отправить,
драйвер возвращает ``EINVAL``.

В режиме :ref:`LIRC_MODE_SCANCODE <lirc-mode-scancode>` за один раз в
символьное устройство должна записываться одна ``struct lirc_scancode``, иначе
возвращается ``EINVAL``. Задайте требуемый scancode в члене ``scancode``,
а :ref:`IR-протокол <Remote_controllers_Protocols>` в члене
:c:type:`rc_proto`:. Все остальные члены должны быть
установлены в 0, иначе возвращается ``EINVAL``. Если для протокола нет кодировщика
или scancode недопустим для указанного протокола,
возвращается ``EINVAL``. Функция записи блокируется до тех пор, пока scancode
не будет передан аппаратурой.

Возвращаемое значение
=====================

В случае успеха возвращается количество записанных байтов. Не является ошибкой, если
это число меньше количества запрошенных байтов или объёма
данных, необходимого для одного кадра.  В случае ошибки возвращается -1, а переменной ``errno``
присваивается соответствующее значение. Общие коды ошибок описаны в
главе :ref:`Общие коды ошибок <gen-errors>`.
