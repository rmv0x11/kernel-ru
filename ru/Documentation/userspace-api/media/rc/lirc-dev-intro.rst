.. _lirc_dev_intro:

********
Введение
********

LIRC расшифровывается как Linux Infrared Remote Control. Интерфейс устройства
LIRC — это двунаправленный интерфейс для передачи данных «сырого» IR и
декодированных сканкодов между пространством пользователя (userspace) и
пространством ядра. По сути, это просто символьное устройство (chardev)
(/dev/lircX, где X = 0, 1, 2, ...), на котором определён ряд стандартных
struct file_operations. Применительно к передаче «сырого» IR и декодированных
сканкодов туда и обратно ключевыми fops являются read, write и ioctl.

Также можно присоединить BPF-программу к устройству LIRC для декодирования
«сырого» IR в сканкоды.

Пример вывода dmesg при регистрации драйвера в LIRC:

.. code-block:: none

    $ dmesg |grep lirc_dev
    rc rc0: lirc_dev: driver mceusb registered at minor = 0, raw IR receiver, raw IR transmitter

Что вы должны увидеть для символьного устройства:

.. code-block:: none

    $ ls -l /dev/lirc*
    crw-rw---- 1 root root 248, 0 Jul 2 22:20 /dev/lirc0

Обратите внимание, что пакет `v4l-utils <https://git.linuxtv.org/v4l-utils.git/>`_
содержит инструменты для работы с устройствами LIRC:

 - ir-ctl: может принимать «сырой» IR и передавать IR, а также запрашивать
   возможности устройства LIRC.

 - ir-keytable: может загружать раскладки клавиш (keymaps); позволяет задавать
   IR-протоколы ядра; загружать BPF IR-декодеры и тестировать декодирование IR.
   Также предоставляются некоторые BPF IR-декодеры.

.. _lirc_modes:

************
Режимы LIRC
************

LIRC поддерживает несколько режимов приёма и отправки IR-кодов, как показано
в следующей таблице.

.. _lirc-mode-scancode:
.. _lirc-scancode-flag-toggle:
.. _lirc-scancode-flag-repeat:

``LIRC_MODE_SCANCODE``

    Этот режим предназначен как для отправки, так и для приёма IR.

    Для передачи (то есть отправки) создайте struct lirc_scancode с нужным
    сканкодом, заданным в поле ``scancode``, с :c:type:`rc_proto`, установленным
    в :ref:`IR-протокол <Remote_controllers_Protocols>`, и со всеми остальными
    полями, установленными в 0. Запишите эту структуру в устройство lirc.

    Для приёма вы читаете struct lirc_scancode из устройства LIRC.
    Поле ``scancode`` устанавливается в принятый сканкод, а
    :ref:`IR-протокол <Remote_controllers_Protocols>` устанавливается в
    :c:type:`rc_proto`. Если сканкод отображается на допустимый код клавиши, то он
    устанавливается в поле ``keycode``, иначе оно устанавливается в ``KEY_RESERVED``.

    Поле ``flags`` может иметь установленный флаг ``LIRC_SCANCODE_FLAG_TOGGLE``,
    если бит переключения (toggle) установлен в протоколах, которые его поддерживают
    (например, rc-5 и rc-6), либо флаг ``LIRC_SCANCODE_FLAG_REPEAT`` для случая,
    когда принимается повтор в протоколах, которые его поддерживают (например, nec).

    В протоколах Sanyo и NEC, если вы удерживаете кнопку на пульте, то вместо
    повторения всего сканкода пульт отправляет более короткое сообщение без
    сканкода, которое означает лишь то, что кнопка удерживается, — «повтор». Когда
    оно принимается, устанавливается ``LIRC_SCANCODE_FLAG_REPEAT``, а сканкод и
    код клавиши повторяются.

    В nec нет способа отличить «удержание кнопки» от «многократного нажатия одной
    и той же кнопки». Протоколы rc-5 и rc-6 имеют бит переключения (toggle). Когда
    кнопка отпускается и нажимается снова, бит переключения инвертируется.
    Если бит переключения установлен, то устанавливается ``LIRC_SCANCODE_FLAG_TOGGLE``.

    Поле ``timestamp`` заполняется временем в наносекундах
    (по ``CLOCK_MONOTONIC``), когда сканкод был декодирован.

.. _lirc-mode-mode2:

``LIRC_MODE_MODE2``

    Драйвер возвращает в пространство пользователя последовательность кодов
    импульсов (pulse) и пауз (space) в виде серии значений u32.

    Этот режим используется только для приёма IR.

    Старшие 8 бит определяют тип пакета, а младшие 24 бита — полезную нагрузку.
    Используйте макрос ``LIRC_VALUE()``, чтобы получить полезную нагрузку, а макрос
    ``LIRC_MODE2()`` даст вам тип, который является одним из:

    ``LIRC_MODE2_PULSE``

        Обозначает наличие IR в микросекундах, также известное как *flash*.

    ``LIRC_MODE2_SPACE``

        Обозначает отсутствие IR в микросекундах, также известное как *gap*.

    ``LIRC_MODE2_FREQUENCY``

        Если измерение несущей частоты было включено с помощью
        :ref:`lirc_set_measure_carrier_mode`, то этот пакет даёт вам несущую
        частоту в герцах.

    ``LIRC_MODE2_TIMEOUT``

        Когда таймаут, заданный с помощью :ref:`lirc_set_rec_timeout`, истекает
        из-за того, что IR не был обнаружен, отправляется этот пакет с числом
        микросекунд без IR.

    ``LIRC_MODE2_OVERFLOW``

        Обозначает, что приёмник IR столкнулся с переполнением и часть IR
        потеряна. Данные IR после этого должны снова стать корректными. Само
        значение не важно, но ядро устанавливает его в 0xffffff для совместимости
        с lircd.

.. _lirc-mode-pulse:

``LIRC_MODE_PULSE``

    В режиме pulse последовательность целочисленных значений импульсов/пауз
    (pulse/space) записывается в устройство lirc с помощью :ref:`lirc-write`.

    Эти значения являются чередующимися длительностями импульсов и пауз в
    микросекундах. Первая и последняя запись должны быть импульсом, поэтому число
    записей должно быть нечётным.

    Этот режим используется только для отправки IR.

************************
IR-декодер на основе BPF
************************

Ядро поддерживает декодирование наиболее распространённых
:ref:`IR-протоколов <Remote_controllers_Protocols>`, но существует много
протоколов, которые не поддерживаются. Чтобы их поддержать, можно загрузить
BPF-программу, которая выполняет декодирование. Это можно сделать только на
устройствах LIRC, которые поддерживают чтение «сырого» IR.

Сначала с помощью системного вызова `bpf(2)`_ с аргументом ``BPF_LOAD_PROG``
должна быть загружена программа типа ``BPF_PROG_TYPE_LIRC_MODE2``. После
присоединения к устройству LIRC эта программа будет вызываться для каждого
события импульса, паузы или таймаута на устройстве LIRC. Контекстом для
BPF-программы является указатель на unsigned int, который является значением
:ref:`LIRC_MODE_MODE2 <lirc-mode-mode2>`. Когда программа декодировала сканкод,
он может быть передан с помощью BPF-функций ``bpf_rc_keydown()`` или
``bpf_rc_repeat()``. О движениях мыши или указателя можно сообщать с помощью
``bpf_rc_pointer_rel()``.

Получив файловый дескриптор для BPF-программы ``BPF_PROG_TYPE_LIRC_MODE2``, её
можно присоединить к устройству LIRC с помощью системного вызова `bpf(2)`_.
Целью должен быть файловый дескриптор устройства LIRC, а тип присоединения
должен быть ``BPF_LIRC_MODE2``. К одному устройству LIRC одновременно может быть
присоединено не более 64 BPF-программ.

.. _bpf(2): http://man7.org/linux/man-pages/man2/bpf.2.html
