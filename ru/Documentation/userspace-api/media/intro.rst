========
Введение
========

Этот документ описывает API между ядром Linux и пространством пользователя
(userspace), используемые устройствами потоковой передачи видео и радио,
включая видеокамеры, карты приёма аналогового и цифрового ТВ, карты приёма
AM/FM, программно определяемое радио (Software Defined Radio, SDR), устройства
потокового захвата и вывода, кодеки и пульты дистанционного управления.

Типичное оборудование медиаустройства показано на :ref:`typical_media_device`.

.. _typical_media_device:

.. kernel-figure:: typical_media_device.svg
    :alt:   typical_media_device.svg
    :align: center

    Типичное медиаустройство

API инфраструктуры media был спроектирован для управления такими устройствами.
Он делится на пять частей.

1. :ref:`Первая часть <v4l2spec>` охватывает радио, захват и вывод видео,
   камеры, устройства аналогового ТВ и кодеки.

2. :ref:`Вторая часть <dvbapi>` охватывает API, используемый для приёма
   цифрового ТВ и Интернета посредством одного из нескольких стандартов
   цифрового ТВ. Хотя он называется DVB API, на самом деле он охватывает
   несколько различных видеостандартов, включая DVB-T/T2, DVB-S/S2, DVB-C,
   ATSC, ISDB-T, ISDB-S, DTMB и т. д. Полный список поддерживаемых стандартов
   можно найти в :c:type:`fe_delivery_system`.

3. :ref:`Третья часть <remote_controllers>` охватывает API пультов
   дистанционного управления (Remote Controller API).

4. :ref:`Четвёртая часть <media_controller>` охватывает API контроллера media
   (Media Controller API).

5. :ref:`Пятая часть <cec>` охватывает API CEC (Consumer Electronics Control).

Следует также отметить, что медиаустройство может иметь и аудиокомпоненты,
такие как микшеры, захват PCM, воспроизведение PCM и т. д., которые управляются
через API ALSA. За дополнительной информацией и последним кодом разработки
обращайтесь к: `https://linuxtv.org <https://linuxtv.org>`__. Для обсуждения
улучшений, сообщений о проблемах, отправки новых драйверов и т. д., пожалуйста,
пишите в: `Linux Media Mailing List (LMML)
<http://vger.kernel.org/vger-lists.html#linux-media>`__.
