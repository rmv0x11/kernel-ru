.. _frontend-stat-properties:

***********************************
Индикаторы статистики фронтенда
***********************************

Значения возвращаются через ``dtv_property.stat``. Если свойство
поддерживается, ``dtv_property.stat.len`` больше нуля.

Для большинства систем доставки ``dtv_property.stat.len`` будет равно 1, если
статистика поддерживается, и свойства будут возвращать одно значение для
каждого параметра.

Однако следует отметить, что новые системы доставки на основе OFDM, такие как ISDB,
могут использовать разные типы модуляции для каждой группы несущих. В таких
стандартах может предоставляться до 3 групп статистики, и
``dtv_property.stat.len`` обновляется так, чтобы отразить «глобальные» метрики,
плюс по одной метрике на каждую группу несущих (называемую «layer» в ISDB).

Поэтому, чтобы сохранить согласованность с другими системами доставки, первое
значение в массиве :c:type:`dtv_property.stat.dtv_stats <dtv_stats>` относится
к глобальной метрике. Остальные элементы массива представляют каждый
слой (layer), начиная со слоя A (индекс 1), слоя B (индекс 2) и так далее.

Число заполненных элементов хранится в ``dtv_property.stat.len``.

Каждый элемент массива ``dtv_property.stat.dtv_stats`` состоит из
двух элементов:

-  ``svalue`` или ``uvalue``, где ``svalue`` предназначено для знаковых значений
   измерения (измерения в дБ), а ``uvalue`` — для беззнаковых значений
   (счётчики, относительная шкала)

-  ``scale`` — шкала для значения. Она может принимать значения:

   -  ``FE_SCALE_NOT_AVAILABLE`` — параметр поддерживается
      фронтендом, но получить его не удалось (это может быть
      временным или постоянным условием)

   -  ``FE_SCALE_DECIBEL`` — параметр является знаковым значением, измеряемым в
      1/1000 дБ

   -  ``FE_SCALE_RELATIVE`` — параметр является беззнаковым значением, где 0
      означает 0%, а 65535 означает 100%.

   -  ``FE_SCALE_COUNTER`` — параметр является беззнаковым значением, которое
      подсчитывает количество появлений события, например ошибки бита, ошибки
      блока или прошедшего времени.


.. _DTV-STAT-SIGNAL-STRENGTH:

DTV_STAT_SIGNAL_STRENGTH
========================

Указывает уровень мощности сигнала на аналоговой части тюнера или
демодулятора.

Возможные шкалы для этой метрики:

-  ``FE_SCALE_NOT_AVAILABLE`` — измерить не удалось или
   измерение ещё не завершено.

-  ``FE_SCALE_DECIBEL`` — мощность сигнала задаётся в единицах 0.001 дБм,
   мощность измеряется в милливаттах. Это значение обычно отрицательное.

-  ``FE_SCALE_RELATIVE`` — фронтенд предоставляет измерение мощности
   в диапазоне от 0% до 100% (фактически от 0 до 65535).


.. _DTV-STAT-CNR:

DTV_STAT_CNR
============

Указывает отношение сигнал/шум для основной несущей.

Возможные шкалы для этой метрики:

-  ``FE_SCALE_NOT_AVAILABLE`` — измерить не удалось или
   измерение ещё не завершено.

-  ``FE_SCALE_DECIBEL`` — отношение сигнал/шум задаётся в единицах 0.001 дБ.

-  ``FE_SCALE_RELATIVE`` — фронтенд предоставляет измерение
   отношения сигнал/шум в диапазоне от 0% до 100% (фактически от 0 до 65535).


.. _DTV-STAT-PRE-ERROR-BIT-COUNT:

DTV_STAT_PRE_ERROR_BIT_COUNT
============================

Измеряет число битовых ошибок до прямой коррекции ошибок
(FEC) во внутреннем блоке кодирования (до Viterbi, LDPC или другого внутреннего
кода).

Это измерение выполняется в течение того же интервала, что и
``DTV_STAT_PRE_TOTAL_BIT_COUNT``.

Чтобы получить измерение BER (Bit Error Rate, частота битовых ошибок), его следует
разделить на
:ref:`DTV_STAT_PRE_TOTAL_BIT_COUNT <DTV-STAT-PRE-TOTAL-BIT-COUNT>`.

Это измерение монотонно увеличивается по мере того, как фронтенд получает больше
измерений счётчика битов. Фронтенд может сбрасывать его при настройке на
канал/транспондер.

Возможные шкалы для этой метрики:

-  ``FE_SCALE_NOT_AVAILABLE`` — измерить не удалось или
   измерение ещё не завершено.

-  ``FE_SCALE_COUNTER`` — число ошибочных битов, подсчитанных до внутреннего
   кодирования.


.. _DTV-STAT-PRE-TOTAL-BIT-COUNT:

DTV_STAT_PRE_TOTAL_BIT_COUNT
============================

Измеряет количество битов, принятых до блока внутреннего кода, в течение
того же периода, что и измерение
:ref:`DTV_STAT_PRE_ERROR_BIT_COUNT <DTV-STAT-PRE-ERROR-BIT-COUNT>`.

Следует отметить, что это измерение может быть меньше общего
количества битов в транспортном потоке, поскольку фронтенду может потребоваться
вручную перезапускать измерение, теряя часть данных между интервалами
измерения.

Это измерение монотонно увеличивается по мере того, как фронтенд получает больше
измерений счётчика битов. Фронтенд может сбрасывать его при настройке на
канал/транспондер.

Возможные шкалы для этой метрики:

-  ``FE_SCALE_NOT_AVAILABLE`` — измерить не удалось или
   измерение ещё не завершено.

-  ``FE_SCALE_COUNTER`` — число битов, подсчитанных при измерении
   :ref:`DTV_STAT_PRE_ERROR_BIT_COUNT <DTV-STAT-PRE-ERROR-BIT-COUNT>`.

.. _DTV-STAT-TOTAL-BLOCK-COUNT:

DTV-STAT_TOTAL_BLOCK_COUNT
==========================

Измеряет общее число блоков, принятых в течение того же периода, что и измерение
:ref:`DTV_STAT_ERROR_BLOCK_COUNT <DTV-STAT-ERROR-BLOCK-COUNT>`.

Может использоваться для вычисления индикатора PER путём деления
:ref:`DTV_STAT_ERROR_BLOCK_COUNT <DTV-STAT-ERROR-BLOCK-COUNT>` на
:ref:`DTV-STAT-TOTAL-BLOCK-COUNT`.

Возможные шкалы для этой метрики:

-  ``FE_SCALE_NOT_AVAILABLE`` — измерить не удалось или
   измерение ещё не завершено.

-  ``FE_SCALE_COUNTER`` — число блоков, подсчитанных при измерении
   :ref:`DTV_STAT_ERROR_BLOCK_COUNT <DTV-STAT-ERROR-BLOCK-COUNT>`.
