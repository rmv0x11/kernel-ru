*****************
ioctl DMX_REQBUFS
*****************

Имя
===

DMX_REQBUFS - Инициирует отображение памяти и/или буферный ввод-вывод DMA

.. warning:: этот API всё ещё является экспериментальным

Синопсис
========

.. c:macro:: DMX_REQBUFS

``int ioctl(int fd, DMX_REQBUFS, struct dmx_requestbuffers *argp)``

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый :c:func:`open()`.

``argp``
    Указатель на struct :c:type:`dmx_requestbuffers`.

Описание
========

Этот ioctl используется для инициирования ввода-вывода демультиплексирования
(demux) на основе отображения памяти или DMABUF.

Буферы с отображением памяти располагаются в памяти устройства и должны быть
выделены с помощью этого ioctl, прежде чем они могут быть отображены в адресное
пространство приложения. Буферы пользователя выделяются самими приложениями, и
этот ioctl используется лишь для переключения драйвера в режим ввода-вывода с
указателем пользователя и для настройки некоторых внутренних структур. Аналогично,
буферы DMABUF выделяются приложениями через драйвер устройства, и этот ioctl лишь
конфигурирует драйвер в режим ввода-вывода DMABUF, не выполняя какого-либо прямого
выделения.

Чтобы выделить буферы устройства, приложения инициализируют все поля структуры
struct :c:type:`dmx_requestbuffers`. Они устанавливают поле ``count`` в желаемое
число буферов, а ``size`` — в размер каждого буфера.

Когда этот ioctl вызывается с указателем на данную структуру, драйвер пытается
выделить запрошенное число буферов и сохраняет фактически выделенное число в поле
``count``. Значение ``count`` может быть меньше запрошенного числа, вплоть до нуля,
когда у драйвера заканчивается свободная память. Большее число также возможно,
когда драйверу требуется больше буферов для корректной работы. Фактический размер
выделенного буфера возвращается в ``size`` и может быть меньше запрошенного.

Когда этот метод ввода-вывода не поддерживается, ioctl возвращает код ошибки
``EOPNOTSUPP``.

Приложения могут снова вызвать :ref:`DMX_REQBUFS`, чтобы изменить число буферов,
однако это не может завершиться успешно, пока хоть какие-то буферы всё ещё
отображены. Значение ``count``, равное нулю, освобождает все буферы после прерывания
или завершения любого выполняемого DMA.

Возвращаемое значение
=====================

В случае успеха возвращается 0, в случае ошибки — -1, а переменная ``errno``
устанавливается соответствующим образом. Общие коды ошибок описаны в главе
:ref:`Общие коды ошибок <gen-errors>`.

EOPNOTSUPP
    Запрошенный метод ввода-вывода не поддерживается.
