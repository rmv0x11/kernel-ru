*****************
Digital TV mmap()
*****************

Имя
===

dmx-mmap - Отображение памяти устройства в адресное пространство приложения

.. warning:: этот API всё ещё является экспериментальным

Синопсис
========

.. code-block:: c

    #include <unistd.h>
    #include <sys/mman.h>

.. c:function:: void *mmap( void *start, size_t length, int prot, int flags, int fd, off_t offset )

Аргументы
=========

``start``
    Отобразить буфер по этому адресу в адресном пространстве приложения.
    Если указан флаг ``MAP_FIXED``, ``start`` должен быть кратен размеру
    страницы, и mmap завершится с ошибкой, если указанный адрес не может
    быть использован. Использование этой опции не рекомендуется;
    приложениям следует просто указывать здесь указатель ``NULL``.

``length``
    Длина отображаемой области памяти. Она должна быть кратна длине
    DVB-пакета (188 в большинстве драйверов).

``prot``
    Аргумент ``prot`` описывает желаемую защиту памяти.
    Независимо от типа устройства и направления обмена данными его
    следует устанавливать в ``PROT_READ`` | ``PROT_WRITE``, разрешая чтение
    и запись в буферы изображений. Драйверы должны поддерживать как минимум
    эту комбинацию флагов.

``flags``
    Параметр ``flags`` задаёт тип отображаемого объекта,
    параметры отображения, а также то, являются ли изменения, внесённые в
    отображённую копию страницы, приватными для процесса или должны
    разделяться с другими ссылками.

    ``MAP_FIXED`` требует, чтобы драйвер не выбирал никакой другой адрес,
    кроме указанного. Если указанный адрес не может быть использован,
    :c:func:`mmap()` завершится с ошибкой. Если указан ``MAP_FIXED``,
    ``start`` должен быть кратен размеру страницы. Использование этой опции
    не рекомендуется.

    Должен быть установлен один из флагов ``MAP_SHARED`` или ``MAP_PRIVATE``.
    ``MAP_SHARED`` позволяет приложениям разделять отображённую память с
    другими (например, дочерними) процессами.

    .. note::

       Приложения Linux Digital TV не должны устанавливать флаги
       ``MAP_PRIVATE``, ``MAP_DENYWRITE``, ``MAP_EXECUTABLE`` или ``MAP_ANON``.

``fd``
    Файловый дескриптор, возвращённый :c:func:`open()`.

``offset``
    Смещение буфера в памяти устройства, возвращённое ioctl
    :ref:`DMX_QUERYBUF`.

Описание
========

Функция :c:func:`mmap()` запрашивает отображение ``length`` байт, начиная со
смещения ``offset``, в памяти устройства, заданного через ``fd``, в адресное
пространство приложения, предпочтительно по адресу ``start``. Этот последний
адрес является лишь подсказкой и обычно указывается равным 0.

Подходящие параметры length и offset запрашиваются с помощью ioctl
:ref:`DMX_QUERYBUF`. Прежде чем буферы можно будет запросить, они должны быть
выделены с помощью ioctl :ref:`DMX_REQBUFS`.

Для отмены отображения буферов используется функция :c:func:`munmap()`.

Возвращаемое значение
=====================

В случае успеха :c:func:`mmap()` возвращает указатель на отображённый буфер. В
случае ошибки возвращается ``MAP_FAILED`` (-1), и переменная ``errno``
устанавливается соответствующим образом. Возможные коды ошибок:

EBADF
    ``fd`` не является корректным файловым дескриптором.

EACCES
    ``fd`` не открыт для чтения и записи.

EINVAL
    Значения ``start``, ``length`` или ``offset`` не подходят. (Например,
    они слишком велики или не выровнены по границе ``PAGESIZE``.)

    Значение ``flags`` или ``prot`` не поддерживается.

    Ни один буфер не был выделен с помощью ioctl
    :ref:`DMX_REQBUFS`.

ENOMEM
    Недостаточно физической или виртуальной памяти для выполнения
    запроса.
