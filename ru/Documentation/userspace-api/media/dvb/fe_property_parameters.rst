.. _fe_property_parameters:

****************************************
Параметры свойств цифрового ТВ
****************************************

Существует несколько различных параметров цифрового ТВ, которые могут
использоваться вызовами
:ref:`ioctl FE_SET_PROPERTY и FE_GET_PROPERTY<FE_GET_PROPERTY>`.
В этом разделе описывается каждый из них. Учтите, однако, что для настройки
фронтенда требуется лишь часть из них.


.. _DTV-UNDEFINED:

DTV_UNDEFINED
=============

Используется внутренне. Операция GET/SET для него ничего не меняет и ничего
не возвращает.


.. _DTV-TUNE:

DTV_TUNE
========

Интерпретирует кэш данных, формирует традиционный запрос настройки фронтенда,
чтобы можно было пройти проверку в ioctl ``FE_SET_FRONTEND``.


.. _DTV-CLEAR:

DTV_CLEAR
=========

Сбрасывает здесь кэш данных, специфичных для фронтенда. Это не затрагивает
аппаратуру.


.. _DTV-FREQUENCY:

DTV_FREQUENCY
=============

Частота транспондера/канала цифрового ТВ.

.. note::

  #. Для спутниковых систем доставки частота указывается в кГц.

  #. Для кабельных и наземных систем доставки частота указывается в
     Гц.

  #. В большинстве систем доставки частота — это центральная частота
     транспондера/канала. Исключение составляет ISDB-T, где
     основная несущая имеет смещение 1/7 относительно центра.

  #. Для ISDB-T каналы обычно передаются со смещением около 143 кГц.
     Например, допустимой частотой может быть 474,143 кГц. Шаг
     привязан к полосе пропускания канала, которая
     обычно составляет 6 МГц.

  #. В ISDB-Tsb канал состоит только из одного или трёх сегментов, шаг
     по частоте составляет 429 кГц, 3*429 соответственно.


.. _DTV-MODULATION:

DTV_MODULATION
==============

Задаёт тип модуляции фронтенда для систем доставки, которые
поддерживают несколько модуляций.

Модуляция может быть одним из типов, определённых перечислением :c:type:`fe_modulation`.

Большинство стандартов цифрового ТВ предлагают более одного возможного
типа модуляции.

В приведённой ниже таблице представлена сводка типов модуляции,
поддерживаемых каждой системой доставки, как они определены спецификациями
на данный момент.

======================= =======================================================
Стандарт		Типы модуляции
======================= =======================================================
ATSC (version 1)	8-VSB и 16-VSB.
DMTB			4-QAM, 16-QAM, 32-QAM, 64-QAM и 4-QAM-NR.
DVB-C Annex A/C		16-QAM, 32-QAM, 64-QAM и 256-QAM.
DVB-C Annex B		64-QAM.
DVB-C2			QPSK, 16-QAM, 64-QAM, 256-QAM, 1024-QAM и 4096-QAM.
DVB-T			QPSK, 16-QAM и 64-QAM.
DVB-T2			QPSK, 16-QAM, 64-QAM и 256-QAM.
DVB-S			Задавать не нужно. Поддерживает только QPSK.
DVB-S2			QPSK, 8-PSK, 16-APSK и 32-APSK.
DVB-S2X			8-APSK-L, 16-APSK-L, 32-APSK-L, 64-APSK и 64-APSK-L.
ISDB-T			QPSK, DQPSK, 16-QAM и 64-QAM.
ISDB-S			8-PSK, QPSK и BPSK.
======================= =======================================================

.. note::

   Поскольку DVB-S2X задаёт расширения к стандарту DVB-S2, используется то же
   значение перечисления системы доставки (SYS_DVBS2).

   Учтите, что некоторые из перечисленных выше типов модуляции в настоящее
   время могут быть не определены в ядре. Причина проста: ни одному драйверу
   пока не требовалось такое определение.


.. _DTV-BANDWIDTH-HZ:

DTV_BANDWIDTH_HZ
================

Полоса пропускания канала, в Гц.

Следует задавать только для наземных систем доставки.

Возможные значения: ``1712000``, ``5000000``, ``6000000``, ``7000000``,
``8000000``, ``10000000``.

======================= =======================================================
Наземный стандарт	Возможные значения полосы пропускания
======================= =======================================================
ATSC (version 1)	Задавать не нужно. Всегда 6MHz.
DMTB			Задавать не нужно. Всегда 8MHz.
DVB-T			6MHz, 7MHz и 8MHz.
DVB-T2			1.172 MHz, 5MHz, 6MHz, 7MHz, 8MHz и 10MHz
ISDB-T			5MHz, 6MHz, 7MHz и 8MHz, хотя в большинстве мест
			используется 6MHz.
======================= =======================================================


.. note::


  #. Для ISDB-Tsb полоса пропускания может изменяться в зависимости от
     числа подключённых сегментов.

     Её легко вывести из других параметров
     (DTV_ISDBT_SB_SEGMENT_IDX, DTV_ISDBT_SB_SEGMENT_COUNT).

  #. В спутниковых и кабельных системах доставки полоса пропускания зависит
     от символьной скорости. Ядро молча проигнорирует любую установку
     :ref:`DTV-BANDWIDTH-HZ` и перезапишет её оценкой полосы пропускания.

     Такая оценка полосы пропускания учитывает символьную скорость, заданную
     с помощью :ref:`DTV-SYMBOL-RATE`, и коэффициент скругления (rolloff),
     который фиксирован для DVB-C и DVB-S.

     Для DVB-S2 коэффициент скругления также следует задавать через :ref:`DTV-ROLLOFF`.

DTV_VOLTAGE
===========

Используется в спутниковых системах доставки.

Напряжение обычно используется с LNB, не поддерживающими DiSEqC, для
переключения поляризации (горизонтальная/вертикальная). При использовании
оборудования DiSEqC это напряжение должно переключаться согласованно с
командами DiSEqC, как описано в спецификации DiSEqC.

Допустимые значения определены типом :c:type:`fe_sec_voltage`.

.. _DTV-ISDBT-PARTIAL-RECEPTION:

DTV_ISDBT_PARTIAL_RECEPTION
===========================

Используется только в ISDB.

Если ``DTV_ISDBT_SOUND_BROADCASTING`` равно «0», это битовое поле
показывает, находится ли канал в режиме частичного приёма или нет.

Если «1», значения ``DTV_ISDBT_LAYERA_*`` присваиваются центральному сегменту,
а ``DTV_ISDBT_LAYERA_SEGMENT_COUNT`` должно быть равно «1».

Если вдобавок ``DTV_ISDBT_SOUND_BROADCASTING`` равно «1»,
``DTV_ISDBT_PARTIAL_RECEPTION`` показывает, состоит ли этот канал ISDB-Tsb
из одного сегмента и слоя или из трёх сегментов и двух слоёв.

Возможные значения: 0, 1, -1 (AUTO)


.. _DTV-ISDBT-SOUND-BROADCASTING:

DTV_ISDBT_SOUND_BROADCASTING
============================

Используется только в ISDB.

Это поле показывает, относятся ли прочие параметры DTV_ISDBT_*
к каналу ISDB-T или к каналу ISDB-Tsb. (См. также
``DTV_ISDBT_PARTIAL_RECEPTION``).

Возможные значения: 0, 1, -1 (AUTO)


.. _DTV-ISDBT-SB-SUBCHANNEL-ID:

DTV_ISDBT_SB_SUBCHANNEL_ID
==========================

Используется только в ISDB.

Это поле применяется, только если ``DTV_ISDBT_SOUND_BROADCASTING`` равно «1».

(Примечание автора: возможно, это не вполне корректное описание
``SUBCHANNEL-ID`` во всех деталях, но это моё понимание
технической подоплёки, необходимой для программирования устройства)

Канал ISDB-Tsb (1 или 3 сегмента) может транслироваться как отдельно, так и в
наборе связанных каналов ISDB-Tsb. В таком наборе каналов каждый
канал может приниматься независимо. Число связанных сегментов ISDB-Tsb
может изменяться, например в зависимости от доступной полосы частотного
спектра.

Пример: предположим, транслируются 8 связанных сегментов ISDB-Tsb.
У вещателя есть несколько возможностей разместить эти каналы в эфире:
предполагая обычный 13-сегментный спектр ISDB-T, он может выровнять 8 сегментов
с позиции 1-8 до 5-13 или любым промежуточным образом.

Нижележащий слой сегментов — это субканалы: каждый сегмент
состоит из нескольких субканалов с предопределёнными ID. Субканал
используется для помощи демодулятору в синхронизации по каналу.

Канал ISDB-T всегда центрирован по всем субканалам. Как видно из
приведённого выше примера, в ISDB-Tsb всё уже не так просто.

Параметр ``The DTV_ISDBT_SB_SUBCHANNEL_ID`` используется для задания
ID субканала сегмента, который должен быть демодулирован.

Возможные значения: 0 .. 41, -1 (AUTO)


.. _DTV-ISDBT-SB-SEGMENT-IDX:

DTV_ISDBT_SB_SEGMENT_IDX
========================

Используется только в ISDB.

Это поле применяется, только если ``DTV_ISDBT_SOUND_BROADCASTING`` равно «1».

``DTV_ISDBT_SB_SEGMENT_IDX`` задаёт индекс сегмента, который должен быть
демодулирован, для канала ISDB-Tsb, в котором несколько из них
передаются связанным образом.

Возможные значения: 0 .. ``DTV_ISDBT_SB_SEGMENT_COUNT`` - 1

Примечание: это значение не может быть определено автоматическим поиском каналов.


.. _DTV-ISDBT-SB-SEGMENT-COUNT:

DTV_ISDBT_SB_SEGMENT_COUNT
==========================

Используется только в ISDB.

Это поле применяется, только если ``DTV_ISDBT_SOUND_BROADCASTING`` равно «1».

``DTV_ISDBT_SB_SEGMENT_COUNT`` задаёт общее число связанных
каналов ISDB-Tsb.

Возможные значения: 1 .. 13

Примечание: это значение не может быть определено автоматическим поиском каналов.


.. _isdb-hierq-layers:

Параметры DTV-ISDBT-LAYER[A-C]
==============================

Используется только в ISDB.

Каналы ISDB-T могут кодироваться иерархически. В отличие от DVB-T, в
ISDB-T иерархические слои могут декодироваться одновременно. По этой
причине демодулятор ISDB-T имеет 3 декодера Витерби и 3 декодера Рида-Соломона.

ISDB-T имеет 3 иерархических слоя, каждый из которых может использовать часть
доступных сегментов. Общее число сегментов по всем слоям должно
составлять 13 в ISDB-T.

Существует 3 набора параметров — для слоёв A, B и C.


.. _DTV-ISDBT-LAYER-ENABLED:

DTV_ISDBT_LAYER_ENABLED
-----------------------

Используется только в ISDB.

Иерархический приём в ISDB-T достигается включением или отключением
слоёв в процессе декодирования. Установка всех битов
``DTV_ISDBT_LAYER_ENABLED`` в «1» заставляет все слои (если применимо)
быть демодулированными. Это значение по умолчанию.

Если канал находится в режиме частичного приёма
(``DTV_ISDBT_PARTIAL_RECEPTION`` = 1), центральный сегмент может декодироваться
независимо от остальных 12 сегментов. В этом режиме слой A должен иметь
``SEGMENT_COUNT``, равный 1.

В ISDB-Tsb используется только слой A, в ISDB-Tsb он может быть 1 или 3 согласно
``DTV_ISDBT_PARTIAL_RECEPTION``. ``SEGMENT_COUNT`` должен быть заполнен
соответствующим образом.

Используются значения только первых 3 битов. Остальные биты будут молча проигнорированы:

``DTV_ISDBT_LAYER_ENABLED`` бит 0: слой A включён

``DTV_ISDBT_LAYER_ENABLED`` бит 1: слой B включён

``DTV_ISDBT_LAYER_ENABLED`` бит 2: слой C включён

``DTV_ISDBT_LAYER_ENABLED`` биты 3-31: не используются


.. _DTV-ISDBT-LAYER-FEC:

DTV_ISDBT_LAYER[A-C]_FEC
------------------------

Используется только в ISDB.

Механизм прямой коррекции ошибок (Forward Error Correction),
используемый данным слоем ISDB, как определено типом :c:type:`fe_code_rate`.


Возможные значения: ``FEC_AUTO``, ``FEC_1_2``, ``FEC_2_3``, ``FEC_3_4``,
``FEC_5_6``, ``FEC_7_8``


.. _DTV-ISDBT-LAYER-MODULATION:

DTV_ISDBT_LAYER[A-C]_MODULATION
-------------------------------

Используется только в ISDB.

Модуляция, используемая данным слоем ISDB, как определено типом
:c:type:`fe_modulation`.

Возможные значения: ``QAM_AUTO``, ``QPSK``, ``QAM_16``, ``QAM_64``, ``DQPSK``

.. note::

   #. Если слой C — ``DQPSK``, то слой B должен быть ``DQPSK``.

   #. Если слой B — ``DQPSK`` и ``DTV_ISDBT_PARTIAL_RECEPTION``\ = 0,
      то слой должен быть ``DQPSK``.

.. _DTV-TRANSMISSION-MODE:

DTV_TRANSMISSION_MODE
=====================


Используется только в стандартах на основе OFTM, например DVB-T/T2, ISDB-T, DTMB.

Задаёт размер FFT (который соответствует приблизительному числу
несущих), используемый стандартом.

Допустимые значения определены типом :c:type:`fe_transmit_mode`.

.. note::

   #. ISDB-T поддерживает три размера несущей/символа: 8K, 4K, 2K. В этом
      стандарте он называется **mode** (режим) и нумеруется от 1 до 3:

      =====  ==========  ========================
      Режим  Размер FFT  Режим передачи
      =====  ==========  ========================
      1      2K          ``TRANSMISSION_MODE_2K``
      2      4K          ``TRANSMISSION_MODE_4K``
      3      8K          ``TRANSMISSION_MODE_8K``
      =====  ==========  ========================

   #. Если ``DTV_TRANSMISSION_MODE`` установлен в ``TRANSMISSION_MODE_AUTO``,
      аппаратура попытается найти правильный размер FFT (если способна) и
      использует TMCC для заполнения недостающих параметров.

   #. DVB-T задаёт 2K и 8K как допустимые размеры.

   #. DVB-T2 задаёт 1K, 2K, 4K, 8K, 16K и 32K.

   #. DTMB задаёт C1 и C3780.

.. _DTV-STREAM-ID:

DTV_STREAM_ID
=============

Используется в DVB-C2, DVB-S2, DVB-T2 и ISDB-S.

DVB-C2, DVB-S2, DVB-T2 и ISDB-S поддерживают передачу нескольких
потоков в одном транспортном потоке. Это свойство позволяет драйверу
цифрового ТВ обрабатывать фильтрацию подпотоков, когда это поддерживается аппаратурой.
По умолчанию фильтрация подпотоков отключена.

Для DVB-C2, DVB-S2 и DVB-T2 допустимый диапазон id подпотока — от 0 до
255.

Для ISDB допустимый диапазон id подпотока — от 1 до 65535.

Чтобы отключить её, следует использовать специальный макрос NO_STREAM_ID_FILTER.

Примечание: любое значение вне диапазона id также отключает фильтрацию.

.. _DTV-ENUM-DELSYS:

DTV_ENUM_DELSYS
===============

Многостандартный фронтенд должен сообщать о предоставляемых системах
доставки. Приложениям необходимо перечислить предоставляемые системы доставки
до использования любой другой операции с фронтендом. До его
появления для определения типа фронтенда использовался FE_GET_INFO. Для
фронтенда, который предоставляет более одной системы доставки,
FE_GET_INFO мало чем помогает. Приложения, которые намереваются использовать
многостандартный фронтенд, должны перечислять связанные с ним системы доставки,
а не пытаться использовать FE_GET_INFO. В случае
устаревшего фронтенда результат точно такой же, как у FE_GET_INFO, но
в более структурированном формате.

Допустимые значения определены типом :c:type:`fe_delivery_system`.

.. _DTV-LNA:

DTV_LNA
=======

Малошумящий усилитель (low-noise amplifier).

Аппаратура может предоставлять управляемый LNA, который можно задавать вручную с
помощью этого параметра. Обычно LNA можно найти только в наземных устройствах,
если он вообще есть.

Возможные значения: 0, 1, LNA_AUTO

0, LNA выключен

1, LNA включён

используйте специальный макрос LNA_AUTO, чтобы задать автоматический режим LNA


.. _DTV-SCRAMBLING-SEQUENCE-INDEX:

DTV_SCRAMBLING_SEQUENCE_INDEX
=============================

Используется в DVB-S2.

Это 18-битное поле, когда присутствует, несёт индекс последовательности
скремблирования физического уровня DVB-S2, как определено в пункте 5.5.4 EN 302 307.
Явного метода сигнализации для передачи индекса последовательности скремблирования
приёмнику не существует. Если доступен дескриптор системы спутниковой доставки S2,
его можно использовать для чтения индекса последовательности скремблирования (EN 300 468, таблица 41).

По умолчанию используется gold-последовательность скремблирования с индексом 0.

Допустимый диапазон индекса последовательности скремблирования — от 0 до 262142.
