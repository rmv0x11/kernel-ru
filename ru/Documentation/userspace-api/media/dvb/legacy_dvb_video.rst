============================
DVB-устройство видео (Video)
============================

.. attention:: **Не** используйте в новых драйверах!
             См.: :ref:`legacy_dvb_decoder_notes`

DVB-устройство видео управляет декодером видео MPEG2 в составе DVB-оборудования.
Доступ к нему осуществляется через ``/dev/dvb/adapter0/video0``. Типы данных
и определения ioctl доступны при включении в ваше приложение заголовочного файла
``linux/dvb/video.h``.

Обратите внимание, что DVB-устройство видео управляет только декодированием
видеопотока MPEG, но не его выводом на экран телевизора или компьютера. На ПК это
обычно обрабатывается связанным устройством video4linux, например ``/dev/video``,
которое позволяет масштабировать изображение и задавать окна вывода.

Большинство DVB-карт не имеют собственного декодера MPEG, что приводит к отсутствию
как устройств аудио и видео, так и устройства video4linux.

Эти ioctl также использовались в V4L2 для управления декодерами MPEG, реализованными
в V4L2. Использование этих ioctl для данной цели объявлено устаревшим, и для замены
этой функциональности были созданы соответствующие ioctl и элементы управления (control)
V4L2. Для новых драйверов используйте :ref:`ioctl V4L2<video>`!

=================



video_format_t
--------------

Краткое описание
~~~~~~~~~~~~~~~~

.. code-block:: c

    typedef enum {
	VIDEO_FORMAT_4_3,
	VIDEO_FORMAT_16_9,
	VIDEO_FORMAT_221_1
    } video_format_t;

Константы
~~~~~~~~~

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    -  ..

       -  ``VIDEO_FORMAT_4_3``

       -  Выбрать формат 4:3.

    -  ..

       -  ``VIDEO_FORMAT_16_9``

       -  Выбрать формат 16:9.

    -  ..

       -  ``VIDEO_FORMAT_221_1``

       -  Выбрать формат 2.21:1.

Описание
~~~~~~~~

Тип данных ``video_format_t``
используется в функции `VIDEO_SET_FORMAT`_, чтобы сообщить драйверу, какое
соотношение сторон имеет оборудование вывода (например, телевизор). Он также
используется в структурах данных `video_status`_, возвращаемой `VIDEO_GET_STATUS`_,
и `video_event`_, возвращаемой `VIDEO_GET_EVENT`_, которые сообщают
о формате отображения текущего видеопотока.

Константы
~~~~~~~~~

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    -  ..

       -  ``VIDEO_SOURCE_DEMUX``

       -  :cspan:`1` Выбрать демультиплексор (demux) в качестве основного источника.

    -  ..

       -  Если выбран этот источник, поток

Описание
~~~~~~~~

Источник видеопотока задаётся вызовом `VIDEO_SELECT_SOURCE`_
и может принимать следующие значения в зависимости от того, воспроизводим ли мы
из внутреннего (демультиплексор) или внешнего (запись пользователем) источника.
VIDEO_SOURCE_DEMUX выбирает в качестве источника видеопотока демультиплексор
(на вход которого подаются данные либо от
frontend, либо от устройства DVR). Если выбран
VIDEO_SOURCE_MEMORY, поток поступает от приложения
через системный вызов `write()`_.

    struct video_command {
	__u32 cmd;
	__u32 flags;
	union {
	    struct {
		__u64 pts;
	    } stop;

	    struct {
		__s32 speed;
		__u32 format;
	    } play;

	    struct {
		__u32 data[16];
	    } raw;
	};
    };


Переменные
~~~~~~~~~~

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    -  ..

       -  ``__u32 cmd``

       -  `Decoder command`_

    -  ..

       -  ``__u32 flags``

       -  Флаги для `Decoder command`_.

    -  ..

       -  ``struct stop``

       -  ``__u64 pts``

       -  MPEG PTS

    -  ..

       -   -1: задаёт покадровое продвижение назад,

       -  `Play input formats`_

    -  ..

       -  ``__u32 data[16]``

       -  Зарезервировано

~~~~~~~~

что в будущем её можно будет безопасно расширять.

video_status
------------

Краткое описание
~~~~~~~~~~~~~~~~

Вызов `VIDEO_GET_STATUS`_ возвращает следующую структуру, информирующую
о различных состояниях операции воспроизведения.

.. code-block:: c

    struct video_status {
	int                    video_blank;
	video_play_state_t     play_state;
	video_stream_source_t  stream_source;
	video_format_t         video_format;
	video_displayformat_t  display_format;
    };

Переменные
~~~~~~~~~~

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    -  ..

       -  :rspan:`2` ``int video_blank``

       -  :cspan:`1` Показывать пустое видео при заморозке?

    -  ..

       -  TRUE  ( != 0 )

       -  Затемнить экран при заморозке.

    -  ..

       -  FALSE ( == 0 )

       -  Показать последний декодированный кадр.

    -  ..

       -  `video_play_state_t`_ ``play_state``

       -  Текущее состояние воспроизведения.

    -  ..

       -  `video_stream_source_t`_ ``stream_source``

       -  Текущий источник (demux/память).

    -  ..

       -  `video_format_t`_ ``video_format``

       -  Текущее соотношение сторон потока.

    -  ..

       -  `video_displayformat_t`_ ``display_format``

       -  Применённый режим обрезки.

Описание
~~~~~~~~

Если ``video_blank`` установлено в ``TRUE``, видео будет затемнено при смене
канала или при остановке воспроизведения. В противном случае будет отображаться
последнее изображение. ``play_state`` указывает, заморожено ли видео в данный
момент, остановлено или воспроизводится. ``stream_source`` соответствует
выбранному источнику видеопотока. Он может поступать либо от
демультиплексора, либо из памяти. ``video_format`` указывает соотношение
сторон (одно из 4:3 или 16:9) воспроизводимого в данный момент видеопотока.
Наконец, ``display_format`` соответствует применённому режиму обрезки в
случае, если исходный видеоформат не совпадает с форматом устройства
вывода.

VIDEO_STOP
----------

Краткое описание
~~~~~~~~~~~~~~~~

.. c:macro:: VIDEO_STOP

.. code-block:: c

	int ioctl(fd, VIDEO_STOP, int mode)

Аргументы
~~~~~~~~~

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    -  ..

       -  ``int fd``

       -  :cspan:`1` Файловый дескриптор, возвращённый предыдущим вызовом
          `open()`_.

    -  ..

       -  ``int request``

       -  :cspan:`1` Равен ``VIDEO_STOP`` для этой команды.

    -  ..

       -  :rspan:`2` ``int mode``

       -  :cspan:`1` Указывает, как следует обработать экран.

    -  ..

       -  TRUE  ( != 0 )

       -  Затемнить экран при остановке.

    -  ..

       -  FALSE ( == 0 )

       -  Показать последний декодированный кадр.

Описание
~~~~~~~~

.. attention:: **Не** используйте в новых драйверах!
             См.: :ref:`legacy_dvb_decoder_notes`

Этот ioctl предназначен только для устройств цифрового ТВ. Для управления
декодером V4L2 используйте вместо него V4L2 :ref:`VIDIOC_DECODER_CMD`.

Этот вызов ioctl просит устройство видео прекратить воспроизведение текущего
потока. В зависимости от входного параметра экран может быть затемнён
или отображать последний декодированный кадр.

Возвращаемое значение
~~~~~~~~~~~~~~~~~~~~~~

При успехе возвращается 0, при ошибке -1, и переменная ``errno`` устанавливается
соответствующим образом. Общие коды ошибок описаны в главе
:ref:`Общие коды ошибок <gen-errors>`.

    -  ..

       -  Указывает на место, где должно быть сохранено событие, если оно есть.

Описание
~~~~~~~~

.. attention:: **Не** используйте в новых драйверах!
             См.: :ref:`legacy_dvb_decoder_notes`

Этот ioctl предназначен только для DVB-устройств. Для получения событий от
декодера V4L2 используйте вместо него V4L2 ioctl :ref:`VIDIOC_DQEVENT`.

Этот вызов ioctl возвращает событие типа `video_event`_, если оно доступно.
Определённое число последних событий будет помещено в очередь и возвращено в порядке
их возникновения. Более старые события могут быть отброшены, если не извлечены
вовремя. Если
событие недоступно, поведение зависит от того, находится ли устройство
в блокирующем или неблокирующем режиме. В последнем случае вызов завершается
неудачей немедленно с установкой errno в значение ``EWOULDBLOCK``. В первом случае
вызов блокируется до тех пор, пока событие не станет доступным. Стандартные системные вызовы Linux poll()
и/или select() можно использовать с файловым дескриптором устройства,
чтобы отслеживать новые события. Для select() файловый дескриптор должен быть
включён в аргумент exceptfds, а для poll() в качестве условия пробуждения должен быть
указан POLLPRI. Для этого вызова ioctl достаточно прав
только на чтение.

open()
------

Краткое описание
~~~~~~~~~~~~~~~~

.. code-block:: c

    #include <fcntl.h>

.. c:function:: 	int open(const char *deviceName, int flags)

Аргументы
~~~~~~~~~

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    -  ..

       -  ``const char *deviceName``

       -  Имя конкретного устройства видео.

    -  ..

       -  :rspan:`3` ``int flags``

       -  :cspan:`1` Побитовое ИЛИ следующих флагов:

    -  ..

       -  ``O_RDONLY``

       -  доступ только на чтение

    -  ..

       -  ``O_RDWR``

       -  доступ на чтение/запись

    -  ..

       -  ``O_NONBLOCK``
       -  | Открыть в неблокирующем режиме
          | (блокирующий режим используется по умолчанию)

Описание
~~~~~~~~

Этот системный вызов открывает именованное устройство видео (например,
/dev/dvb/adapter?/video?) для последующего использования.

Когда вызов open() завершается успешно, устройство будет готово к использованию.
Значимость блокирующего или неблокирующего режима описана в
документации к функциям, где есть различие. Это не
влияет на семантику самого вызова open(). Устройство, открытое в
блокирующем режиме, позже может быть переведено в неблокирующий режим (и наоборот)
с помощью команды F_SETFL системного вызова fcntl. Это стандартный
системный вызов, описанный в руководстве Linux для fcntl. Только один
пользователь может открыть устройство видео в режиме O_RDWR. Все остальные попытки
открыть устройство в этом режиме завершатся неудачей, и будет возвращён
код ошибки. Если устройство видео открыто в режиме O_RDONLY, единственным
вызовом ioctl, который можно использовать, является `VIDEO_GET_STATUS`_. Все остальные вызовы
вернут код ошибки.

Возвращаемое значение
~~~~~~~~~~~~~~~~~~~~~~

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    -  ..

       -  ``ENODEV``

       -  :cspan:`1` Драйвер устройства не загружен/недоступен.

    -  ..

       -  ``EINTERNAL``

       -  Внутренняя ошибка.

    -  ..

       -  ``EBUSY``

       -  Устройство или ресурс заняты.

    -  ..

       -  ``EINVAL``

       -  Недопустимый аргумент.

    -  ..

       -  ``void *buf``

       -  Указатель на буфер, содержащий данные PES.

    -  ..

       -  ``size_t count``

       -  Размер buf.

Описание
~~~~~~~~

Этот системный вызов можно использовать, только если в вызове ioctl
`VIDEO_SELECT_SOURCE`_ выбран VIDEO_SOURCE_MEMORY. Предоставляемые данные должны быть в
формате PES, если только возможности устройства не допускают других форматов. TS является
наиболее распространённым форматом для хранения DVB-данных, он обычно также поддерживается.
Если O_NONBLOCK не указан, функция будет блокироваться до тех пор, пока не появится место
в буфере. Объём передаваемых данных задаётся параметром count.
