.. _requisites:

Что нужно знать
===============

От читателя этого документа требуется некоторое знание области цифрового
телевизионного вещания (Digital TV); читатель должен быть знаком с частью I
спецификации MPEG2 ISO/IEC 13818 (она же ITU-T H.222), то есть вы должны знать,
что такое программный/транспортный поток (program/transport stream, PS/TS) и что
понимается под пакетированным элементарным потоком (packetized elementary stream,
PES) или I-кадром (I-frame).

Различные документы стандартов цифрового телевидения доступны для скачивания по адресам:

- Европейские стандарты (DVB): http://www.dvb.org и/или http://www.etsi.org.
- Американские стандарты (ATSC): https://www.atsc.org/standards/
- Японские стандарты (ISDB): http://www.dibeg.org/

Также необходимо знать, как обращаться к устройствам Linux и как использовать
вызовы ioctl. Сюда же относится знание языка C или C++.


.. _history:

История
=======

Первый API для карт цифрового телевидения, который мы использовали в Convergence
в конце 1999 года, был расширением API Video4Linux, изначально разработанного для
карт захвата кадров (frame grabber). В силу этого он был не очень хорошо приспособлен
для использования с картами цифрового телевидения и их новыми возможностями, такими
как запись потоков MPEG и фильтрация нескольких потоков секций и данных PES одновременно.

В начале 2000 года к Convergence обратилась компания Nokia с предложением о новом
стандартном API цифрового телевидения для Linux. В рамках обязательства по разработке
терминалов на основе открытых стандартов, Nokia и Convergence сделали его доступным
всем разработчикам Linux и опубликовали его на https://linuxtv.org в сентябре
2000 года. Вместе с драйвером Linux для карты Siemens/Hauppauge DVB PCI компания
Convergence предоставила первую реализацию API цифрового телевидения Linux.
В те ранние дни Convergence была сопровождающим (maintainer) API цифрового
телевидения Linux.

Теперь API сопровождается сообществом LinuxTV (то есть вами, читателем этого
документа). API цифрового телевидения Linux постоянно пересматривается и
улучшается вместе с улучшениями ядра подсистемы в Kernel.


.. _overview:

Обзор
=====


.. _stb_components:

.. kernel-figure:: dvbstb.svg
    :alt:   dvbstb.svg
    :align: center

    Компоненты карты цифрового телевидения / STB

Карта цифрового телевидения или приставка (set-top-box, STB) обычно состоит из
следующих основных аппаратных компонентов:

Frontend, состоящий из тюнера и цифрового ТВ-демодулятора
   Здесь необработанный сигнал попадает в аппаратуру цифрового телевидения со
   спутниковой тарелки, антенны или напрямую из кабеля. Frontend понижает частоту
   и демодулирует этот сигнал в транспортный поток MPEG (transport stream, TS).
   В случае спутникового frontend сюда входит средство управления спутниковым
   оборудованием (satellite equipment control, SEC), которое позволяет управлять
   поляризацией LNB, переключателями нескольких источников (multi feed switches)
   или поворотными механизмами тарелки.

Аппаратура условного доступа (Conditional Access, CA), такая как адаптеры CI и слоты для смарт-карт
   Полный TS пропускается через аппаратуру CA. Программы, к которым у пользователя
   есть доступ (контролируется смарт-картой), декодируются в реальном времени и
   повторно вставляются в TS.

   .. note::

      Не всякая аппаратура цифрового телевидения предоставляет аппаратуру условного доступа.

Демультиплексор, фильтрующий входящий поток Digital TV MPEG-TS
   Демультиплексор разбивает TS на его компоненты, такие как аудио- и
   видеопотоки. Помимо обычно нескольких таких аудио- и видеопотоков, он также
   содержит потоки данных с информацией о программах, предлагаемых в этом или
   других потоках того же провайдера.

Аудио- и видеодекодер
   Основными адресатами демультиплексора являются аудио- и видеодекодеры.
   После декодирования они передают несжатые аудио и видео на экран компьютера
   или на телевизор.

   .. note::

      Современная аппаратура обычно не имеет отдельной аппаратуры декодера, так как
      подобная функциональность может предоставляться основным CPU, графическим
      адаптером системы или аппаратурой обработки сигналов, встроенной в интегральную
      микросхему Systems on a Chip (SoC).

      Она также может не требоваться для определённых применений (например, для
      применений, использующих только данные, таких как «интернет через спутник»).

:ref:`stb_components` показывает грубую схему управляющего потока и потока данных
между этими компонентами.



.. _dvb_devices:

Устройства цифрового телевидения Linux
======================================

API цифрового телевидения Linux позволяет вам управлять этими аппаратными
компонентами через имеющиеся в настоящее время шесть символьных устройств в
стиле Unix: для видео, аудио, frontend, демультиплексора (demux), CA и сетевого
взаимодействия IP-over-DVB. Устройства видео и аудио управляют аппаратурой
декодера MPEG2, устройство frontend — тюнером и цифровым ТВ-демодулятором.
Устройство demux даёт вам контроль над фильтрами PES и секций аппаратуры. Если
аппаратура не поддерживает фильтрацию, эти фильтры могут быть реализованы
программно. Наконец, устройство CA управляет всеми возможностями условного
доступа аппаратуры. От индивидуальных требований к безопасности платформы может
зависеть, предоставляются ли приложению функции CA через это устройство и сколько
их предоставляется.

Все устройства можно найти в дереве ``/dev`` под ``/dev/dvb``. Отдельные
устройства называются:

-  ``/dev/dvb/adapterN/audioM``,

-  ``/dev/dvb/adapterN/videoM``,

-  ``/dev/dvb/adapterN/frontendM``,

-  ``/dev/dvb/adapterN/netM``,

-  ``/dev/dvb/adapterN/demuxM``,

-  ``/dev/dvb/adapterN/dvrM``,

-  ``/dev/dvb/adapterN/caM``,

где ``N`` нумерует карты цифрового телевидения в системе начиная с 0, а
``M`` нумерует устройства каждого типа в пределах каждого адаптера, тоже начиная
с 0. В дальнейшем обсуждении этих устройств мы будем опускать «``/dev/dvb/adapterN/``\ ».

Более подробно структуры данных и вызовы функций всех устройств описаны в
следующих главах.


.. _include_files:

Включаемые файлы API
====================

Для каждого из устройств цифрового телевидения существует соответствующий
включаемый файл. Включаемые файлы API цифрового телевидения следует подключать в
исходных текстах приложения с использованием частичного пути, например:


.. code-block:: c

	#include <linux/dvb/ca.h>

	#include <linux/dvb/dmx.h>

	#include <linux/dvb/frontend.h>

	#include <linux/dvb/net.h>


Чтобы приложения могли поддерживать различные версии API, существует
дополнительный включаемый файл ``linux/dvb/version.h``, который определяет
константу ``DVB_API_VERSION``. Этот документ описывает ``DVB_API_VERSION 5.10``.
