==========================
Аудиоустройство DVB
==========================

.. attention:: **Не** используйте в новых драйверах!
             См.: :ref:`legacy_dvb_decoder_notes`

Аудиоустройство DVB управляет декодером звука MPEG2 в составе аппаратуры
DVB. Доступ к нему осуществляется через ``/dev/dvb/adapter?/audio?``.
Типы данных и определения ioctl становятся доступны при включении
``linux/dvb/audio.h`` в ваше приложение.

Обратите внимание, что большинство DVB-карт не имеют собственного
MPEG-декодера, что приводит к отсутствию аудио- и видеоустройства.

Эти ioctl также использовались в V4L2 для управления MPEG-декодерами,
реализованными в V4L2. Применение этих ioctl для данной цели объявлено
устаревшим, и для замены этой функциональности были созданы
соответствующие ioctl или элементы управления V4L2. Для новых драйверов
используйте :ref:`ioctl V4L2<audio>`!


Типы данных аудио
=================

В этом разделе описаны структуры, типы данных и определения, используемые
при взаимодействии с аудиоустройством.

audio_stream_source_t
---------------------

Синопсис
~~~~~~~~

.. c:enum:: audio_stream_source_t

.. code-block:: c

    typedef enum {
    AUDIO_SOURCE_DEMUX,
    AUDIO_SOURCE_MEMORY
    } audio_stream_source_t;

Константы
~~~~~~~~~

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    -  ..

       -  ``AUDIO_SOURCE_DEMUX``

       -  :cspan:`1` Выбирает демультиплексор (получающий данные либо от
          фронтенда, либо от устройства DVR) в качестве источника
          видеопотока.

    -  ..

       -  ``AUDIO_SOURCE_MEMORY``

       -  Выбирает поток от приложения, поступающий через системный вызов
          `write()`_.

Описание
~~~~~~~~

Источник аудиопотока задаётся вызовом `AUDIO_SELECT_SOURCE`_ и может
принимать следующие значения в зависимости от того, воспроизводим ли мы
из внутреннего (demux) или внешнего (запись пользователя) источника.

Данные, передаваемые в декодер, также управляются PID-фильтром.
Выбор вывода: :c:type:`dmx_output` ``DMX_OUT_DECODER``.

~~~~~~~~~

.. flat-table::
    :header-rows:  0

    -  ..

       -  :cspan:`1` Файловый дескриптор, возвращённый предыдущим вызовом

    -  ..

       -  ``int request``

       -  Равен ``AUDIO_SELECT_SOURCE`` для этой команды.

    -  ..

       -  Указывает источник, который должен использоваться для

Описание
~~~~~~~~

.. attention:: **Не** используйте в новых драйверах!
             См.: :ref:`legacy_dvb_decoder_notes`

Этот вызов ioctl сообщает аудиоустройству, какой источник должен
использоваться для входных данных. Возможные источники — demux или
память. Если выбран ``AUDIO_SOURCE_MEMORY``, данные передаются в
аудиоустройство через команду записи. Если выбран ``AUDIO_SOURCE_DEMUX``,
данные передаются напрямую от встроенного demux-устройства к декодеру.
Примечание: пока это поддерживает только DVB-устройства с одним demux и
одним декодером.

устанавливается соответствующим образом. Общие коды ошибок описаны в
главе :ref:`Общие коды ошибок <gen-errors>`.

open()
------

Синопсис
~~~~~~~~

.. code-block:: c

    #include <fcntl.h>

.. c:function:: int  open(const char *deviceName, int flags)

Аргументы
~~~~~~~~~

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    -  ..

       -  ``const char *deviceName``

       -  Имя конкретного аудиоустройства.

    -  ..

       -  :rspan:`3` ``int flags``

       -  :cspan:`1` Побитовое ИЛИ следующих флагов:

    -  ..

       -  ``O_RDONLY``

       -  доступ только для чтения

    -  ..

       -  ``O_RDWR``

       -  доступ на чтение/запись

    -  ..

       -  ``O_NONBLOCK``
       -  | Открытие в неблокирующем режиме
          | (по умолчанию используется блокирующий режим)

Описание
~~~~~~~~

Этот системный вызов открывает именованное аудиоустройство (например,
``/dev/dvb/adapter0/audio0``) для последующего использования. После
успешного выполнения вызова open() устройство будет готово к
использованию. Значение блокирующего или неблокирующего режима описано в
документации к функциям, где есть различие. На семантику самого вызова
open() это не влияет. Устройство, открытое в блокирующем режиме, может
быть позже переведено в неблокирующий режим (и наоборот) с помощью
команды F_SETFL системного вызова fcntl. Это стандартный системный
вызов, описанный в man-странице Linux для fcntl. Только один пользователь
может открыть аудиоустройство в режиме O_RDWR. Все остальные попытки
открыть устройство в этом режиме завершатся неудачей, и будет возвращён
код ошибки. Если аудиоустройство открыто в режиме O_RDONLY, единственным
вызовом ioctl, который можно использовать, является `AUDIO_GET_STATUS`_.
Все остальные вызовы вернут код ошибки.

Возвращаемое значение
~~~~~~~~~~~~~~~~~~~~~~

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    -  ..

       -  ``ENODEV``

       -  Драйвер устройства не загружен/недоступен.

    -  ..

       -  ``EBUSY``

       -  Устройство или ресурс заняты.

    -  ..

       -  ``EINVAL``

       -  Недопустимый аргумент.

в буфере. Объём передаваемых данных задаётся параметром count.
