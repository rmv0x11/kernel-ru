*******************************
ioctl FE_SET_FRONTEND_TUNE_MODE
*******************************

Имя
===

FE_SET_FRONTEND_TUNE_MODE - Позволяет задать флаги режима тюнера для frontend.

Обзор
=====

.. c:macro:: FE_SET_FRONTEND_TUNE_MODE

``int ioctl(int fd, FE_SET_FRONTEND_TUNE_MODE, unsigned int flags)``

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый :c:func:`open()`.

``flags``
    Допустимые флаги:

    -  0 - обычный режим настройки

    -  ``FE_TUNE_MODE_ONESHOT`` - Когда установлен, этот флаг отключает любое
       зигзагообразное (zigzagging) или иное «обычное» поведение настройки. Кроме того,
       не будет автоматического отслеживания статуса захвата (lock), и,
       следовательно, не будут генерироваться события frontend. Если устройство frontend
       закрыто, этот флаг будет автоматически выключен при повторном открытии
       устройства на чтение-запись.

Описание
========

Позволяет задать флаги режима тюнера для frontend, между 0 (обычный) и режимом
``FE_TUNE_MODE_ONESHOT``

Возвращаемое значение
=====================

При успехе возвращается 0.

При ошибке возвращается -1, и переменной ``errno`` присваивается
соответствующее значение.

Общие коды ошибок описаны в главе
:ref:`Общие коды ошибок <gen-errors>`.
