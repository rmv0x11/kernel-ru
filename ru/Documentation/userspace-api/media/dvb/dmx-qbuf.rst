*************************
ioctl DMX_QBUF, DMX_DQBUF
*************************

Имя
===

DMX_QBUF - DMX_DQBUF - Обмен буфером с драйвером

.. warning:: этот API всё ещё является экспериментальным

Синопсис
========

.. c:macro:: DMX_QBUF

``int ioctl(int fd, DMX_QBUF, struct dmx_buffer *argp)``

.. c:macro:: DMX_DQBUF

``int ioctl(int fd, DMX_DQBUF, struct dmx_buffer *argp)``

Аргументы
=========

``fd``
    Файловый дескриптор, возвращаемый :c:func:`open()`.

``argp``
    Указатель на struct :c:type:`dmx_buffer`.

Описание
========

Приложения вызывают ioctl ``DMX_QBUF``, чтобы поставить в очередь
входящих буферов драйвера пустой (захват) или заполненный (вывод) буфер.
Семантика зависит от выбранного метода ввода-вывода.

Чтобы поставить буфер в очередь, приложения устанавливают поле ``index``.
Допустимые значения индекса находятся в диапазоне от нуля до числа буферов,
выделенных с помощью :ref:`DMX_REQBUFS` (struct :c:type:`dmx_requestbuffers`
``count``) минус один. Также подойдёт содержимое struct :c:type:`dmx_buffer`,
возвращённое вызовом ioctl :ref:`DMX_QUERYBUF`.

Когда ``DMX_QBUF`` вызывается с указателем на эту структуру, он блокирует
страницы памяти буфера в физической памяти, так что они не могут быть
выгружены на диск. Буферы остаются заблокированными до извлечения из очереди
либо до закрытия устройства.

Приложения вызывают ioctl ``DMX_DQBUF``, чтобы извлечь заполненный буфер
(захват) из исходящей очереди драйвера. Они просто устанавливают поле
``index`` идентификатором буфера, который нужно поставить в очередь.
Когда ``DMX_DQBUF`` вызывается с указателем на struct :c:type:`dmx_buffer`,
драйвер заполняет оставшиеся поля или возвращает код ошибки.

По умолчанию ``DMX_DQBUF`` блокируется, когда в исходящей очереди нет ни
одного буфера. Если функции :c:func:`open()` был передан флаг
``O_NONBLOCK``, ``DMX_DQBUF`` немедленно возвращается с кодом ошибки
``EAGAIN``, когда нет доступного буфера.

Структура struct :c:type:`dmx_buffer` описана в :ref:`buffer`.

Возвращаемое значение
=====================

В случае успеха возвращается 0, в случае ошибки -1, а переменной ``errno``
присваивается соответствующее значение. Общие коды ошибок описаны в главе
:ref:`Общие коды ошибок <gen-errors>`.

EAGAIN
    Был выбран неблокирующий ввод-вывод с помощью ``O_NONBLOCK``, и в
    исходящей очереди не было ни одного буфера.

EINVAL
    Значение ``index`` находится вне допустимого диапазона, либо буферы ещё
    не были выделены.

EIO
    ``DMX_DQBUF`` завершился неудачей из-за внутренней ошибки. Также может
    указывать на временные проблемы, такие как потеря сигнала или ошибки CRC.
