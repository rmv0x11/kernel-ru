.. _DMX_QUERYBUF:

******************
ioctl DMX_QUERYBUF
******************

Имя
===

DMX_QUERYBUF - Запрос состояния буфера

.. warning:: этот API всё ещё является экспериментальным

Краткое описание
================

.. c:macro:: DMX_QUERYBUF

``int ioctl(int fd, DMX_QUERYBUF, struct dvb_buffer *argp)``

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый :c:func:`open()`.

``argp``
    Указатель на struct :c:type:`dvb_buffer`.

Описание
========

Этот ioctl является частью метода потокового ввода-вывода mmap. Он может
использоваться для запроса состояния буфера в любой момент после того, как
буферы были выделены с помощью ioctl :ref:`DMX_REQBUFS`.

Приложения устанавливают поле ``index``. Допустимые номера индексов находятся
в диапазоне от нуля до числа буферов, выделенных с помощью :ref:`DMX_REQBUFS`
(struct :c:type:`dvb_requestbuffers` ``count``), минус один.

После вызова :ref:`DMX_QUERYBUF` с указателем на эту структуру драйверы
возвращают код ошибки или заполняют остальную часть структуры.

В случае успеха поле ``offset`` будет содержать смещение буфера от начала
памяти устройства, поле ``length`` — его размер, а поле ``bytesused`` — число
байт, занятых данными в буфере (полезной нагрузкой).

Возвращаемое значение
=====================

В случае успеха возвращается 0, поле ``offset`` будет содержать смещение
буфера от начала памяти устройства, поле ``length`` — его размер, а поле
``bytesused`` — число байт, занятых данными в буфере (полезной нагрузкой).

В случае ошибки возвращается -1, а переменная ``errno`` устанавливается
соответствующим образом. Общие коды ошибок описаны в главе
:ref:`Generic Error Codes <gen-errors>`.

EINVAL
    Значение ``index`` выходит за пределы допустимого диапазона.
