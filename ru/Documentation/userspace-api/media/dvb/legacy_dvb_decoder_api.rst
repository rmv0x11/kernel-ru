.. _legacy_dvb_decoder_api:

==========================================
Устаревшие API декодера DVB MPEG
==========================================

.. _legacy_dvb_decoder_notes:

Общие замечания
===============

Этот API изначально был разработан только для DVB и поэтому ограничен
:ref:`legacy_dvb_decoder_formats`, используемыми в подобных системах цифрового
телевещания.

Чтобы обойти эти ограничения, был разработан более универсальный API
:ref:`V4L2 <v4l2spec>`, который заменяет эту часть API DVB.

Тем не менее существуют проекты, построенные вокруг этого API.
Для обеспечения совместимости данный API сохраняется в неизменном виде.

.. attention:: **Не** используйте этот API в новых драйверах!

    Для аудио и видео используйте API :ref:`V4L2 <v4l2spec>` и ALSA.

    Конвейеры (pipelines) следует настраивать с помощью :ref:`Media Controller  API<media_controller>`.

На практике с декодерами, по-видимому, работают по-разному. Приложение обычно
знает, какой декодер используется, или специально написано под один тип декодера.
Запрос возможностей применяется редко, поскольку они уже известны.


.. _legacy_dvb_decoder_formats:

Форматы данных
==============

Этот API разработан для DVB и совместимых систем вещания.
В силу этого единственными поддерживаемыми форматами данных являются
MPEG-потоки, совместимые с ISO/IEC 13818-1. Поддерживаемые полезные нагрузки
могут различаться в зависимости от используемого декодера.

Временные метки всегда являются MPEG PTS, как определено в ITU T-REC-H.222.0 /
ISO/IEC 13818-1, если не указано иное.

Для хранения записей обычно используются TS-потоки, в меньшей степени — PES.
Оба варианта повсеместно принимаются для воспроизведения, но это может зависеть от драйвера.
