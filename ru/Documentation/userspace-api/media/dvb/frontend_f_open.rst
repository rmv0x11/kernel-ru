************************************
Цифровое ТВ: open() для frontend
************************************

Имя
===

fe-open - Открыть устройство frontend

Синопсис
========

.. code-block:: c

    #include <fcntl.h>

.. c:function:: int open( const char *device_name, int flags )

Аргументы
=========

``device_name``
    Открываемое устройство.

``flags``
    Флаги открытия. Доступ может быть либо ``O_RDWR``, либо ``O_RDONLY``.

    С ``O_RDONLY`` допускается несколько открытий. В этом режиме разрешены
    только ioctl-вызовы для запроса и чтения.

    С ``O_RDWR`` допускается только одно открытие. В этом режиме разрешены
    все ioctl-вызовы.

    Если указан флаг ``O_NONBLOCK``, системные вызовы могут возвращать
    код ошибки ``EAGAIN``, когда данные недоступны или когда драйвер
    устройства временно занят.

    Прочие флаги не оказывают влияния.

Описание
========

Этот системный вызов открывает именованное устройство frontend
(``/dev/dvb/adapter?/frontend?``) для последующего использования. Обычно
первое, что следует сделать после успешного открытия, — это определить тип
frontend с помощью :ref:`FE_GET_INFO`.

Устройство может быть открыто в режиме «только для чтения», который позволяет
лишь отслеживать состояние и статистику устройства, либо в режиме
«чтение/запись», который позволяет использовать устройство любым образом
(например, выполнять операции настройки).

В системе с несколькими frontend обычно невозможно одновременно открыть
несколько устройств в режиме «чтение/запись». Пока устройство frontend
открыто в режиме «чтение/запись», другие вызовы open() в режиме
«чтение/запись» будут либо завершаться неудачей, либо блокироваться — в
зависимости от того, был ли указан неблокирующий или блокирующий режим.
Устройство frontend, открытое в блокирующем режиме, может быть впоследствии
переведено в неблокирующий режим (и наоборот) с помощью команды F_SETFL
системного вызова fcntl. Это стандартный системный вызов, описанный в
man-странице Linux для fcntl. После успешного вызова open() устройство будет
готово к использованию в указанном режиме. Это подразумевает, что
соответствующее оборудование включено, и что другие frontend, возможно,
были выключены, чтобы это стало возможным.

Возвращаемое значение
=====================

При успехе :c:func:`open()` возвращает новый файловый дескриптор.
При ошибке возвращается -1, а переменной ``errno`` присваивается
соответствующее значение.

Возможные коды ошибок:

При успехе возвращается 0, и заполняется :c:type:`ca_slot_info`.

При ошибке возвращается -1, а переменной ``errno`` присваивается
соответствующее значение.

.. tabularcolumns:: |p{2.5cm}|p{15.0cm}|

.. flat-table::
    :header-rows:  0
    :stub-columns: 0
    :widths: 1 16

    -  - ``EPERM``
       -  Вызывающая сторона не имеет прав на доступ к устройству.

    -  - ``EBUSY``
       -  Драйвер устройства уже используется.

    -  - ``EMFILE``
       -  Процесс уже открыл максимально допустимое число файлов.

    -  - ``ENFILE``
       -  Достигнут предел общего числа файлов, открытых в системе.
