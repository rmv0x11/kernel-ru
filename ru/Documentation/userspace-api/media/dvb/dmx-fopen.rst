========================================
Цифровое ТВ: open() для демультиплексора
========================================

Имя
---

Цифровое ТВ: open() для демультиплексора

Синопсис
--------

.. c:function:: int open(const char *deviceName, int flags)

Аргументы
---------

``name``
  Имя конкретного устройства демультиплексора цифрового ТВ.

``flags``
  Побитовое ИЛИ (OR) следующих флагов:

.. tabularcolumns:: |p{2.5cm}|p{15.0cm}|

.. flat-table::
    :header-rows:  0
    :stub-columns: 0
    :widths: 1 16

    -
       - ``O_RDONLY``
       - доступ только на чтение

    -
       - ``O_RDWR``
       - доступ на чтение/запись

    -
       - ``O_NONBLOCK``
       - открытие в неблокирующем режиме
         (по умолчанию используется блокирующий режим)

Описание
--------

Этот системный вызов, используемый с именем устройства
``/dev/dvb/adapter?/demux?``, выделяет новый фильтр и возвращает
дескриптор, который может использоваться для последующего управления
этим фильтром. Этот вызов должен выполняться для каждого используемого
фильтра, то есть каждый возвращаемый файловый дескриптор является ссылкой
на один фильтр. ``/dev/dvb/adapter?/dvr?`` — это логическое устройство,
используемое для извлечения транспортных потоков (Transport Stream) для
цифровой видеозаписи. При чтении из этого устройства возвращается
транспортный поток, содержащий пакеты от всех PES-фильтров, установленных
в соответствующем устройстве демультиплексора
(``/dev/dvb/adapter?/demux?``), у которых вывод установлен в
``DMX_OUT_TS_TAP``. Записанный транспортный поток воспроизводится путём
записи в это устройство.

Значимость блокирующего или неблокирующего режима описана в документации
по тем функциям, где есть различие. Это не влияет на семантику самого
вызова ``open()``. Устройство, открытое в блокирующем режиме, может быть
позднее переведено в неблокирующий режим (и наоборот) с помощью команды
``F_SETFL`` системного вызова fcntl.

Возвращаемое значение
---------------------

В случае успеха возвращается 0.

В случае ошибки возвращается -1, и переменной ``errno`` присваивается
соответствующее значение.
