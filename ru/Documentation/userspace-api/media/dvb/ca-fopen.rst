====================
Digital TV CA open()
====================

Имя
---

Digital TV CA open()

Синопсис
--------

.. c:function:: int open(const char *name, int flags)

Аргументы
---------

``name``
  Имя конкретного устройства Digital TV CA.

``flags``
  Побитовое ИЛИ следующих флагов:

.. tabularcolumns:: |p{2.5cm}|p{15.0cm}|

.. flat-table::
    :header-rows:  0
    :stub-columns: 0
    :widths: 1 16

    -  - ``O_RDONLY``
       - доступ только для чтения

    -  - ``O_RDWR``
       - доступ для чтения/записи

    -  - ``O_NONBLOCK``
       - открытие в неблокирующем режиме
         (по умолчанию используется блокирующий режим)

Описание
--------

Этот системный вызов открывает именованное устройство ca (например,
``/dev/dvb/adapter?/ca?``) для последующего использования.

После успешного выполнения вызова ``open()`` устройство будет готово к
использованию. Значение блокирующего и неблокирующего режимов описано в
документации по функциям, для которых есть различие. Оно не влияет на семантику
самого вызова ``open()``. Устройство, открытое в блокирующем режиме, может быть
впоследствии переведено в неблокирующий режим (и наоборот) с помощью команды
``F_SETFL`` системного вызова ``fcntl``. Это стандартный системный вызов,
описанный в man-странице Linux для fcntl. Только один пользователь может открыть
устройство CA в режиме ``O_RDWR``. Все остальные попытки открыть устройство в
этом режиме завершатся неудачей, и будет возвращён код ошибки.

Возвращаемое значение
---------------------

При успехе возвращается 0.

При ошибке возвращается -1, а переменной ``errno`` присваивается
соответствующее значение.

Общие коды ошибок описаны в главе
:ref:`Общие коды ошибок <gen-errors>`.
