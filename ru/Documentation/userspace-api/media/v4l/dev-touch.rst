***********************
Сенсорные устройства
***********************

Доступ к сенсорным устройствам осуществляется через специальные символьные
файлы устройств с именами от ``/dev/v4l-touch0`` до ``/dev/v4l-touch255`` со
старшим номером (major) 81 и динамически выделяемыми младшими номерами (minor)
от 0 до 255.

Обзор
=====

Датчики могут быть оптическими или проективно-ёмкостными сенсорными (PCT).

Для анализа сырых данных и формирования событий ввода требуется обработка.
В некоторых системах она может выполняться на ASIC, и сырые данные служат
исключительно дополнительным каналом для диагностики или настройки. В других
системах ASIC представляет собой простое аналоговое входное устройство,
поставляющее сенсорные данные с высокой частотой, и вся обработка касаний
должна выполняться на хосте.

При ёмкостном измерении касания сенсорный экран состоит из массива
горизонтальных и вертикальных проводников (которые также называют
строками/столбцами, линиями X/Y или tx/rx). Взаимная ёмкость (Mutual
Capacitance) измеряется в узлах, где проводники пересекаются. В качестве
альтернативы собственная ёмкость (Self Capacitance) измеряет сигнал от каждого
столбца и строки по отдельности.

Касание может быть определено путём сравнения сырого измерения ёмкости с
эталонным измерением без касания (или «базовой линией», baseline):

Delta = Raw - Reference

Эталонное измерение учитывает изменения ёмкости по всей матрице сенсорного
датчика, например производственные неравномерности, влияние окружающей среды
или краевые эффекты.

Запрос возможностей
===================

Устройства, поддерживающие сенсорный интерфейс, устанавливают флаг
``V4L2_CAP_VIDEO_CAPTURE`` и флаг ``V4L2_CAP_TOUCH`` в поле ``capabilities``
структуры :c:type:`v4l2_capability`, возвращаемой ioctl
:ref:`VIDIOC_QUERYCAP`.

Должен поддерживаться по крайней мере один из методов ввода-вывода: read/write
или потоковый (streaming).

Форматы, поддерживаемые сенсорными устройствами, описаны в разделе
:ref:`Touch Formats <tch-formats>`.

Согласование формата данных
===========================

Сенсорное устройство может поддерживать любой метод ввода-вывода.
