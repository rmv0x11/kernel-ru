.. _fm-rx-controls:

***********************************************
Справочник по элементам управления FM-приёмника
***********************************************

Класс FM-приёмника (FM_RX) включает элементы управления для общих
возможностей устройств, способных принимать FM.


.. _fm-rx-control-id:

Идентификаторы элементов управления FM_RX
=========================================

``V4L2_CID_FM_RX_CLASS (class)``
    Дескриптор класса FM_RX. Вызов
    :ref:`VIDIOC_QUERYCTRL` для этого элемента управления
    вернёт описание данного класса элементов управления.

``V4L2_CID_RDS_RECEPTION (boolean)``
    Включает/отключает приём RDS радиотюнером.

``V4L2_CID_RDS_RX_PTY (integer)``
    Получает поле типа программы RDS (Programme Type). В нём кодируется до 31
    предопределённого типа программы.

``V4L2_CID_RDS_RX_PS_NAME (string)``
    Получает имя службы программ (PS_NAME). Оно предназначено для
    статического отображения на приёмнике. Это основное вспомогательное
    средство для слушателей при идентификации и выборе службы программ.
    В приложении E к
    :ref:`iec62106`, спецификации RDS, приведено полное
    описание правильной кодировки символов для строк имени службы
    программ. Также, согласно спецификации RDS, PS обычно представляет собой
    единый текст из восьми символов. Однако можно встретить и приёмники,
    которые умеют прокручивать строки размером 8 x N символов. Поэтому данный
    элемент управления должен настраиваться с шагом в 8 символов. В результате
    он всегда должен содержать строку, размер которой кратен 8.

``V4L2_CID_RDS_RX_RADIO_TEXT (string)``
    Получает информацию Radio Text. Это текстовое описание того, что
    транслируется в эфире. RDS Radio Text может применяться, когда вещатель
    желает передать более длинные имена PS, сведения, связанные с программой,
    или любой иной текст. В этих случаях RadioText может использоваться
    дополнительно к
    ``V4L2_CID_RDS_RX_PS_NAME``. Кодировка строк Radio Text также
    полностью описана в приложении E к :ref:`iec62106`. Длина строк
    Radio Text зависит от того, какой блок RDS используется для
    их передачи, — либо 32 (блок 2A), либо 64 (блок 2B). Однако можно
    встретить и приёмники, которые умеют прокручивать строки размером 32
    x N или 64 x N символов. Поэтому данный элемент управления должен
    настраиваться с шагом в 32 или 64 символа. В результате он всегда должен
    содержать строку, размер которой кратен 32 или 64.

``V4L2_CID_RDS_RX_TRAFFIC_ANNOUNCEMENT (boolean)``
    Если установлено, то в данный момент передаётся сообщение о дорожной
    обстановке.

``V4L2_CID_RDS_RX_TRAFFIC_PROGRAM (boolean)``
    Если установлено, то настроенная программа несёт сообщения о дорожной
    обстановке.

``V4L2_CID_RDS_RX_MUSIC_SPEECH (boolean)``
    Если установлено, то этот канал транслирует музыку. Если сброшено, то он
    транслирует речь. Если передатчик не делает такого различия,
    то значение будет установлено.

``V4L2_CID_TUNE_DEEMPHASIS (enum)``
    Настраивает значение деэмфазиса для приёма. Фильтр деэмфазиса
    применяется к эфиру, чтобы подчеркнуть высокие звуковые
    частоты. В зависимости от региона используется постоянная времени либо 50,
    либо 75 микросекунд. Перечисление v4l2_deemphasis определяет возможные
    значения деэмфазиса. Это:

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    * - ``V4L2_DEEMPHASIS_DISABLED``
      - Деэмфазис не применяется.
    * - ``V4L2_DEEMPHASIS_50_uS``
      - Используется деэмфазис 50 мкс.
    * - ``V4L2_DEEMPHASIS_75_uS``
      - Используется деэмфазис 75 мкс.
