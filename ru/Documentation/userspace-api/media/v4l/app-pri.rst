************************
Приоритет приложения
************************

Когда несколько приложений совместно используют одно устройство, может быть
желательно назначить им разные приоритеты. Вопреки традиционному подходу в духе
«rm -rf /», приложение для записи видео могло бы, например, блокировать другим
приложениям изменение видеоэлементов управления или переключение текущего
телеканала. Другая цель — позволить низкоприоритетным приложениям работать в
фоновом режиме; их могут вытеснять управляемые пользователем приложения, а позже
они автоматически вновь получают контроль над устройством.

Поскольку эти возможности невозможно реализовать целиком в пространстве
пользователя, V4L2 определяет ioctl'ы :ref:`VIDIOC_G_PRIORITY <VIDIOC_G_PRIORITY>` и
:ref:`VIDIOC_S_PRIORITY <VIDIOC_G_PRIORITY>` для запроса и
получения приоритета доступа, связанного с файловым дескриптором. Открытие
устройства назначает средний приоритет, совместимый с более ранними версиями
V4L2 и драйверами, не поддерживающими эти ioctl'ы. Приложения, которым требуется
другой приоритет, обычно вызывают :ref:`VIDIOC_S_PRIORITY
<VIDIOC_G_PRIORITY>` после проверки устройства с помощью ioctl
:ref:`VIDIOC_QUERYCAP`.

Ioctl'ы, изменяющие свойства драйвера, такие как
:ref:`VIDIOC_S_INPUT <VIDIOC_G_INPUT>`, возвращают код ошибки ``EBUSY``
после того, как другое приложение получило более высокий приоритет.
