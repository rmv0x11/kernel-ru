************************
Интерфейс метаданных
************************

Метаданные — это любые неизобразительные данные, дополняющие видеокадры
дополнительной информацией. Это может включать статистику, вычисленную по
изображению, параметры захвата кадра, предоставляемые источником изображения,
или специфичные для устройства параметры, задающие то, как устройство
обрабатывает изображения. Этот интерфейс предназначен для передачи метаданных
между пространством пользователя (userspace) и оборудованием, а также для
управления этой операцией.

Интерфейс метаданных реализуется на узлах видеоустройства. Устройство может быть
выделено под метаданные либо поддерживать как видео, так и метаданные, что
указывается в сообщаемых им возможностях.

Запрос возможностей
===================

Узлы устройства, поддерживающие интерфейс захвата метаданных, устанавливают флаг
``V4L2_CAP_META_CAPTURE`` в поле ``device_caps`` структуры
:c:type:`v4l2_capability`, возвращаемой ioctl-вызовом :c:func:`VIDIOC_QUERYCAP`.
Этот флаг означает, что устройство может захватывать метаданные в память.
Аналогично, узлы устройства, поддерживающие интерфейс вывода метаданных,
устанавливают флаг ``V4L2_CAP_META_OUTPUT`` в поле ``device_caps`` структуры
:c:type:`v4l2_capability`. Этот флаг означает, что устройство может читать
метаданные из памяти.

Должен поддерживаться хотя бы один из методов ввода-вывода: read/write или
потоковый (streaming).


Согласование формата данных
===========================

Устройство метаданных использует ioctl-вызовы :ref:`format` для выбора формата
захвата. Формат содержимого буфера метаданных привязан к этому выбранному
формату. Помимо базовых ioctl-вызовов :ref:`format`, должен поддерживаться также
ioctl-вызов :c:func:`VIDIOC_ENUM_FMT`.

Чтобы использовать ioctl-вызовы :ref:`format`, приложения устанавливают поле
``type`` структуры :c:type:`v4l2_format` в ``V4L2_BUF_TYPE_META_CAPTURE`` или в
``V4L2_BUF_TYPE_META_OUTPUT`` и используют член ``meta`` типа
:c:type:`v4l2_meta_format` объединения ``fmt`` по мере необходимости в
соответствии с желаемой операцией. И драйверы, и приложения должны установить
остальную часть структуры :c:type:`v4l2_format` в 0.

Устройства, захватывающие метаданные построчно, имеют установленный флаг
``V4L2_FMT_FLAG_META_LINE_BASED`` структуры v4l2_fmtdesc для
:c:func:`VIDIOC_ENUM_FMT`. Такие устройства, как правило, также могут
:ref:`захватывать данные изображения <capture>`. В основном это касается
устройств, получающих данные от других устройств, таких как сенсор камеры.

.. flat-table:: struct v4l2_meta_format
    :header-rows:  0
    :stub-columns: 0
    :widths:       1 1 2

    * - __u32
      - ``dataformat``
      - Формат данных, устанавливаемый приложением. Это четырёхсимвольный код
        (four character code) в формате little endian
        :ref:`four character code <v4l2-fourcc>`. V4L2 определяет форматы
        метаданных в :ref:`meta-formats`.
    * - __u32
      - ``buffersize``
      - Максимальный размер буфера в байтах, требуемый для данных. Значение
        устанавливается драйвером.
    * - __u32
      - ``width``
      - Ширина строки метаданных в единицах данных (Data Units). Действительно,
	когда установлен флаг ``V4L2_FMT_FLAG_META_LINE_BASED`` структуры
	:c:type`v4l2_fmtdesc`, иначе ноль. См. :c:func:`VIDIOC_ENUM_FMT`.
    * - __u32
      - ``height``
      - Число строк метаданных. Действительно, когда установлен флаг
	``V4L2_FMT_FLAG_META_LINE_BASED`` структуры :c:type`v4l2_fmtdesc`,
	иначе ноль. См. :c:func:`VIDIOC_ENUM_FMT`.
    * - __u32
      - ``bytesperline``
      - Смещение в байтах между началами двух последовательных строк.
	Действительно, когда установлен флаг ``V4L2_FMT_FLAG_META_LINE_BASED``
	структуры :c:type`v4l2_fmtdesc`, иначе ноль. См.
	:c:func:`VIDIOC_ENUM_FMT`.
