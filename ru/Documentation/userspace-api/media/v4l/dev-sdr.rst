**********************************************************
Интерфейс программно-определяемого радио (SDR)
**********************************************************

SDR — это сокращение от Software Defined Radio (программно-определяемое
радио), радиоустройства, которое использует прикладное программное
обеспечение для модуляции или демодуляции. Этот интерфейс предназначен
для управления такими устройствами и потоковой передачи данных.

Доступ к устройствам SDR осуществляется через специальные символьные
файлы устройств с именами от ``/dev/swradio0`` до ``/dev/swradio255``
со старшим номером 81 и динамически выделяемыми младшими номерами от 0
до 255.


Запрос возможностей
===================

Устройства, поддерживающие интерфейс SDR-приёмника, устанавливают флаги
``V4L2_CAP_SDR_CAPTURE`` и ``V4L2_CAP_TUNER`` в поле ``capabilities``
структуры :c:type:`v4l2_capability`, возвращаемой ioctl
:ref:`VIDIOC_QUERYCAP`. Этот флаг означает, что
устройство имеет аналого-цифровой преобразователь (ADC), который
является обязательным элементом для SDR-приёмника.

Устройства, поддерживающие интерфейс SDR-передатчика, устанавливают флаги
``V4L2_CAP_SDR_OUTPUT`` и ``V4L2_CAP_MODULATOR`` в поле ``capabilities``
структуры :c:type:`v4l2_capability`, возвращаемой ioctl
:ref:`VIDIOC_QUERYCAP`. Этот флаг означает, что
устройство имеет цифро-аналоговый преобразователь (DAC), который
является обязательным элементом для SDR-передатчика.

Должен поддерживаться хотя бы один из методов ввода-вывода: read/write
или потоковый.


Дополнительные функции
======================

Устройства SDR могут поддерживать :ref:`элементы управления <control>`
и должны поддерживать ioctl :ref:`tuner`. Ioctl тюнера используются для
задания частоты дискретизации ADC/DAC (частоты выборки) и возможной
радиочастоты (RF).

Тип тюнера ``V4L2_TUNER_SDR`` используется для задания частоты ADC/DAC
устройства SDR, а тип тюнера ``V4L2_TUNER_RF`` используется для задания
радиочастоты. Индекс тюнера RF (если он есть) всегда должен следовать за
индексом тюнера SDR. Обычно тюнер SDR имеет номер #0, а тюнер RF — #1.

Ioctl :ref:`VIDIOC_S_HW_FREQ_SEEK` не
поддерживается.


Согласование формата данных
===========================

Устройство SDR использует ioctl :ref:`format` для выбора формата захвата
и вывода. И разрешение дискретизации, и формат потоковой передачи данных
привязаны к этому выбираемому формату. В дополнение к базовым ioctl
:ref:`format` также должен поддерживаться ioctl
:ref:`VIDIOC_ENUM_FMT`.

Для использования ioctl :ref:`format` приложения устанавливают поле
``type`` структуры :c:type:`v4l2_format` в значение
``V4L2_BUF_TYPE_SDR_CAPTURE`` или ``V4L2_BUF_TYPE_SDR_OUTPUT`` и
используют член ``sdr`` структуры :c:type:`v4l2_sdr_format`
объединения ``fmt`` по необходимости в соответствии с желаемой операцией.
В настоящее время используются два поля структуры
:c:type:`v4l2_sdr_format`, ``pixelformat`` и ``buffersize``.
Содержимое ``pixelformat`` — это код V4L2 FourCC формата данных.
Поле ``buffersize`` — это максимальный размер буфера в байтах,
требуемый для передачи данных, задаваемый драйвером с целью
информирования приложения.


.. c:type:: v4l2_sdr_format

.. tabularcolumns:: |p{4.4cm}|p{4.4cm}|p{8.5cm}|

.. flat-table:: struct v4l2_sdr_format
    :header-rows:  0
    :stub-columns: 0
    :widths:       1 1 2

    * - __u32
      - ``pixelformat``
      - Формат данных или тип сжатия, задаваемый приложением.
	Это код из четырёх символов в порядке little endian,
	:ref:`four character code <v4l2-fourcc>`. V4L2 определяет
	форматы SDR в :ref:`sdr-formats`.
    * - __u32
      - ``buffersize``
      - Максимальный размер в байтах, требуемый для данных. Значение
	задаётся драйвером.
    * - __u8
      - ``reserved[24]``
      - Этот массив зарезервирован для будущих расширений. Драйверы и
	приложения должны устанавливать его в ноль.


Устройство SDR может поддерживать ввод-вывод :ref:`read/write <rw>` и/или
потоковый (:ref:`отображение в память <mmap>` или
:ref:`пользовательский указатель <userp>`).
