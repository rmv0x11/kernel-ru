*****************************************************
Интерфейс video memory-to-memory (память-в-память)
*****************************************************

Устройство V4L2 memory-to-memory может сжимать, распаковывать,
преобразовывать или иным образом конвертировать видеоданные из одного
формата в другой прямо в памяти. Такие устройства memory-to-memory
устанавливают возможность ``V4L2_CAP_VIDEO_M2M`` или
``V4L2_CAP_VIDEO_M2M_MPLANE``. Примерами устройств memory-to-memory
являются кодеки, масштабаторы, деинтерлейсеры или преобразователи
форматов (например, преобразование из YUV в RGB).

Видеоузел memory-to-memory ведёт себя так же, как обычный видеоузел, но
поддерживает потоковый ввод-вывод (stream I/O) как для вывода (отправка
кадров из памяти в оборудование), так и для захвата (приём обработанных
кадров из оборудования в память). Приложению придётся настроить
потоковый ввод-вывод для обеих сторон и в итоге вызвать
:ref:`VIDIOC_STREAMON <VIDIOC_STREAMON>` как для захвата, так и для
вывода, чтобы запустить оборудование.

Устройства memory-to-memory работают как разделяемый ресурс: вы можете
открыть видеоузел несколько раз, при этом каждое приложение настраивает
собственные свойства, локальные для своего файлового дескриптора, и
каждое может использовать узел независимо от других. Драйвер
арбитрирует доступ к оборудованию и перепрограммирует его всякий раз,
когда доступ получает другой файловый дескриптор. Это отличается от
обычного поведения видеоузла, где свойства видео являются глобальными
для устройства (то есть изменение чего-либо через один файловый
дескриптор видно через другой файловый дескриптор).

Одним из наиболее распространённых устройств memory-to-memory является
кодек. Кодеки сложнее большинства устройств и требуют дополнительной
настройки своих параметров. Это делается с помощью элементов управления
кодеком (codec controls). См. :ref:`codec-controls`. Более подробные
сведения о том, как использовать устройства memory-to-memory типа
кодек, приведены в следующих разделах.
