***********************************************
Определение цветовых пространств в V4L2
***********************************************

В V4L2 цветовые пространства задаются четырьмя значениями. Первое — это
идентификатор цветового пространства (enum :c:type:`v4l2_colorspace`),
который определяет цветности, передаточную функцию по умолчанию,
кодирование Y'CbCr по умолчанию и метод квантования по умолчанию. Второе —
это идентификатор передаточной функции (enum
:c:type:`v4l2_xfer_func`) для указания нестандартных
передаточных функций. Третье — это идентификатор кодирования Y'CbCr (enum
:c:type:`v4l2_ycbcr_encoding`) для указания
нестандартных кодирований Y'CbCr, а четвёртое — это идентификатор
квантования (enum :c:type:`v4l2_quantization`) для
указания нестандартных методов квантования. В большинстве случаев требуется
заполнить только поле colorspace структуры :c:type:`v4l2_pix_format`
или структуры :c:type:`v4l2_pix_format_mplane`.

.. _hsv-colorspace:

В :ref:`форматах HSV <hsv-formats>` *Hue* (тон) определяется как угол в
цилиндрическом представлении цвета. Обычно этот угол измеряется в
градусах, т. е. 0–360. При отображении значения этого угла в 8 бит есть
два основных способа: разделить угловое значение на 2 (0–179) или
использовать весь диапазон, 0–255, разделив угловое значение на 1.41. Enum
:c:type:`v4l2_hsv_encoding` указывает, какое кодирование используется.

.. note:: Квантование R'G'B' по умолчанию — полнодиапазонное (full range) для всех
   цветовых пространств. Форматы HSV всегда полнодиапазонные.

.. c:type:: v4l2_hsv_encoding

.. tabularcolumns:: |p{6.5cm}|p{11.0cm}|

.. flat-table:: Кодирования HSV в V4L2
    :header-rows:  1
    :stub-columns: 0

    * - Идентификатор
      - Подробности
    * - ``V4L2_HSV_ENC_180``
      - Для тона (Hue) каждый младший значащий разряд (LSB) равен двум градусам.
    * - ``V4L2_HSV_ENC_256``
      - Для тона (Hue) 360 градусов отображаются в 8 бит, т. е. каждый
	младший значащий разряд (LSB) равен примерно 1.41 градуса.



.. c:type:: v4l2_quantization

.. tabularcolumns:: |p{6.5cm}|p{11.0cm}|

.. flat-table:: Методы квантования V4L2
    :header-rows:  1
    :stub-columns: 0

    * - Идентификатор
      - Подробности
    * - ``V4L2_QUANTIZATION_DEFAULT``
      - Использовать кодирование квантования по умолчанию, заданное
	цветовым пространством. Для R'G'B' и HSV оно всегда полнодиапазонное (full range).
	Для Y'CbCr оно обычно ограниченного диапазона (limited range).
    * - ``V4L2_QUANTIZATION_FULL_RANGE``
      - Использовать полнодиапазонное (full range) кодирование квантования. Т. е. диапазон [0…1]
	отображается в [0…255] (с возможным ограничением до [1…254], чтобы избежать
	значений 0x00 и 0xff). Cb и Cr отображаются из [-0.5…0.5] в
	[0…255] (с возможным ограничением до [1…254], чтобы избежать значений 0x00 и
	0xff).
    * - ``V4L2_QUANTIZATION_LIM_RANGE``
      - Использовать кодирование квантования ограниченного диапазона (limited range). Т. е. диапазон [0…1]
	отображается в [16…235]. Cb и Cr отображаются из [-0.5…0.5] в
	[16…240]. Ограниченный диапазон не может использоваться с HSV.
