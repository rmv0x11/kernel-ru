.. _func-munmap:

*************
V4L2 munmap()
*************

Имя
===

v4l2-munmap - Отображение памяти устройства снимается

Синопсис
========

.. code-block:: c

    #include <unistd.h>
    #include <sys/mman.h>

.. c:function:: int munmap( void *start, size_t length )

Аргументы
=========

``start``
    Адрес отображённого буфера, возвращённый функцией
    :c:func:`mmap()`.

``length``
    Длина отображённого буфера. Это значение должно совпадать с тем, что было
    передано функции :c:func:`mmap()` и возвращено драйвером в поле
    ``length`` структуры :c:type:`v4l2_buffer` для
    одноплоскостного (single-planar) API и в поле
    ``length`` структуры :c:type:`v4l2_plane` для
    многоплоскостного (multi-planar) API.

Описание
========

Снимает отображение буфера, ранее отображённого функцией :c:func:`mmap()`,
и освобождает его, если это возможно.

Возвращаемое значение
=====================

При успехе :c:func:`munmap()` возвращает 0, при ошибке — -1, и
переменной ``errno`` присваивается соответствующее значение:

EINVAL
    Некорректное значение ``start`` или ``length``, либо ещё не было
    отображено ни одного буфера.
