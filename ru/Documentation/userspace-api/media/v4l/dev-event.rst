*******************
Интерфейс событий
*******************

Интерфейс событий V4L2 предоставляет пользователю средство для немедленного
уведомления о возникновении определённых условий на устройстве. Это может
включать, например, события начала кадра или потери сигнала. Изменения
значения или состояния элемента управления V4L2 также могут передаваться
через события.

Чтобы получать события, пользователь сначала должен подписаться на
интересующие его события с помощью ioctl
:ref:`VIDIOC_SUBSCRIBE_EVENT`. После того как
подписка на событие оформлена, события подписанных типов можно извлекать из
очереди с помощью ioctl :ref:`VIDIOC_DQEVENT`. Отписаться от событий можно с
помощью ioctl VIDIOC_UNSUBSCRIBE_EVENT. Специальный тип события
V4L2_EVENT_ALL может использоваться для отписки от всех событий, которые
поддерживает драйвер.

Подписки на события и очереди событий специфичны для дескрипторов файлов.
Подписка на событие на одном дескрипторе файла не влияет на другие
дескрипторы файлов.

Информация о событиях, которые можно извлечь из очереди, получается с помощью
системных вызовов select или poll на видеоустройствах. События V4L2
используют события POLLPRI в системном вызове poll и исключения в системном
вызове select.

Начиная с ядра 3.1 в отношении событий могут быть даны определённые гарантии:

1. Каждое подписанное событие имеет собственную внутреннюю выделенную очередь
   событий. Это означает, что переполнение одного типа событий не будет мешать
   другим типам событий.

2. Если внутренняя очередь событий для конкретного подписанного события
   становится заполненной, то самое старое событие в этой очереди будет
   отброшено.

3. Там, где это применимо, определённые типы событий могут гарантировать, что
   полезная нагрузка самого старого события, которое вот-вот будет отброшено,
   будет объединена с полезной нагрузкой следующего по старшинству события.
   Тем самым гарантируется, что никакая информация не теряется, а пропускается
   лишь промежуточный шаг, ведущий к этой информации. См. документацию по
   событию, на которое вы хотите подписаться, чтобы узнать, применимо это к
   данному событию или нет.
