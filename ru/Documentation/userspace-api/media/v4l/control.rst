************************************
Пользовательские элементы управления
************************************

Устройства обычно имеют ряд настраиваемых пользователем элементов управления,
таких как яркость, насыщенность и так далее, которые отображаются пользователю
в графическом интерфейсе. Однако разные устройства имеют разный набор доступных
элементов управления, и, более того, диапазон возможных значений и значение по
умолчанию варьируются от устройства к устройству. Ioctl'ы для элементов
управления предоставляют эту информацию и механизм для создания удобного
пользовательского интерфейса для этих элементов управления, который будет
корректно работать с любым устройством.

Доступ ко всем элементам управления осуществляется с помощью значения ID. V4L2
определяет несколько ID для конкретных целей. Драйверы также могут реализовывать
собственные пользовательские элементы управления, используя
``V4L2_CID_PRIVATE_BASE``  [#f1]_ и более высокие значения. Предопределённые ID
элементов управления имеют префикс ``V4L2_CID_`` и перечислены в
:ref:`control-id`. ID используется при запросе атрибутов элемента управления, а
также при получении или установке его текущего значения.

Как правило, приложения должны представлять элементы управления пользователю без
предположений об их назначении. Каждый элемент управления сопровождается строкой
с именем, которую пользователь должен понимать. Когда назначение не интуитивно
понятно, автор драйвера должен предоставить руководство пользователя, плагин
пользовательского интерфейса или специфичное для драйвера приложение-панель.
Предопределённые ID были введены для программного изменения нескольких элементов
управления, например для отключения звука устройства во время переключения
канала.

Драйверы могут перечислять разные элементы управления после переключения
текущего видеовхода или видеовыхода, тюнера или модулятора, аудиовхода или
аудиовыхода. Разные в смысле других границ, другого значения по умолчанию и
текущего значения, размера шага или других пунктов меню. Элемент управления с
определённым *пользовательским* ID также может менять имя и тип.

Если элемент управления неприменим к текущей конфигурации устройства (например,
он не относится к текущему видеовходу), драйверы устанавливают флаг
``V4L2_CTRL_FLAG_INACTIVE``.

Значения элементов управления хранятся глобально, они не меняются при
переключении, кроме как для того, чтобы оставаться в пределах сообщённых границ.
Они также не меняются, например, при открытии или закрытии устройства, при
изменении радиочастоты тюнера и вообще никогда без запроса приложения.

V4L2 задаёт механизм событий для уведомления приложений об изменении значения
элементов управления (см.
:ref:`VIDIOC_SUBSCRIBE_EVENT`, событие
``V4L2_EVENT_CTRL``); приложения-панели могут захотеть воспользоваться им, чтобы
всегда отражать корректное значение элемента управления.

Все элементы управления используют машинный порядок байтов.


.. _control-id:

ID элементов управления
=======================

``V4L2_CID_BASE``
    Первый предопределённый ID, равный ``V4L2_CID_BRIGHTNESS``.

``V4L2_CID_USER_BASE``
    Синоним ``V4L2_CID_BASE``.

``V4L2_CID_BRIGHTNESS`` ``(integer)``
    Яркость изображения, или, точнее, уровень чёрного.

``V4L2_CID_CONTRAST`` ``(integer)``
    Контрастность изображения или усиление яркостной составляющей (luma gain).

``V4L2_CID_SATURATION`` ``(integer)``
    Цветовая насыщенность изображения или усиление цветности (chroma gain).

``V4L2_CID_HUE`` ``(integer)``
    Оттенок или цветовой баланс.

``V4L2_CID_AUDIO_VOLUME`` ``(integer)``
    Общая громкость звука. Обратите внимание, что некоторые драйверы также
    предоставляют интерфейс микшера OSS или ALSA.

``V4L2_CID_AUDIO_BALANCE`` ``(integer)``
    Баланс стереозвука. Минимум соответствует полному смещению влево, максимум —
    вправо.

``V4L2_CID_AUDIO_BASS`` ``(integer)``
    Регулировка нижних частот звука.

``V4L2_CID_AUDIO_TREBLE`` ``(integer)``
    Регулировка верхних частот звука.

``V4L2_CID_AUDIO_MUTE`` ``(boolean)``
    Отключение звука, т. е. установка громкости в ноль, однако без воздействия на
    ``V4L2_CID_AUDIO_VOLUME``. Подобно драйверам ALSA, драйверы V4L2 должны
    отключать звук во время загрузки, чтобы избежать чрезмерного шума. На самом
    деле всё устройство должно быть переведено в состояние низкого
    энергопотребления.

``V4L2_CID_AUDIO_LOUDNESS`` ``(boolean)``
    Режим тонкомпенсации (loudness, усиление низких частот).

``V4L2_CID_BLACK_LEVEL`` ``(integer)``
    Другое название для яркости (не синоним
    ``V4L2_CID_BRIGHTNESS``). Этот элемент управления устарел и не должен
    использоваться в новых драйверах и приложениях.

``V4L2_CID_AUTO_WHITE_BALANCE`` ``(boolean)``
    Автоматический баланс белого (камеры).

``V4L2_CID_DO_WHITE_BALANCE`` ``(button)``
    Это элемент управления действием. При установке (значение игнорируется)
    устройство выполнит баланс белого и затем удержит текущую настройку.
    Сравните это с булевым ``V4L2_CID_AUTO_WHITE_BALANCE``,
    который при активации продолжает регулировать баланс белого.

``V4L2_CID_RED_BALANCE`` ``(integer)``
    Баланс красной цветности.

``V4L2_CID_BLUE_BALANCE`` ``(integer)``
    Баланс синей цветности.

``V4L2_CID_GAMMA`` ``(integer)``
    Регулировка гаммы.

``V4L2_CID_WHITENESS`` ``(integer)``
    Белизна для устройств с оттенками серого. Это синоним
    ``V4L2_CID_GAMMA``. Этот элемент управления устарел и не должен
    использоваться в новых драйверах и приложениях.

``V4L2_CID_EXPOSURE`` ``(integer)``
    Экспозиция (камеры). [Единица измерения?]

``V4L2_CID_AUTOGAIN`` ``(boolean)``
    Автоматическое управление усилением/экспозицией.

``V4L2_CID_GAIN`` ``(integer)``
    Управление усилением.

    Используется в первую очередь для управления усилением, например, на
    ТВ-тюнерах, но также и на веб-камерах. Большинство устройств управляют этим
    элементом управления только цифровым усилением, но на некоторых сюда может
    также входить аналоговое усиление. Устройства, которые различают цифровое и
    аналоговое усиление, используют элементы управления
    ``V4L2_CID_DIGITAL_GAIN`` и ``V4L2_CID_ANALOGUE_GAIN``.

.. _v4l2-cid-hflip:

``V4L2_CID_HFLIP`` ``(boolean)``
    Зеркальное отражение изображения по горизонтали.

.. _v4l2-cid-vflip:

``V4L2_CID_VFLIP`` ``(boolean)``
    Зеркальное отражение изображения по вертикали.

.. _v4l2-power-line-frequency:

``V4L2_CID_POWER_LINE_FREQUENCY`` ``(enum)``
    Включает фильтр частоты сети электропитания для устранения мерцания.
    Возможные значения для ``enum v4l2_power_line_frequency``:

    ==========================================  ==
    ``V4L2_CID_POWER_LINE_FREQUENCY_DISABLED``	 0
    ``V4L2_CID_POWER_LINE_FREQUENCY_50HZ``	 1
    ``V4L2_CID_POWER_LINE_FREQUENCY_60HZ``	 2
    ``V4L2_CID_POWER_LINE_FREQUENCY_AUTO``	 3
    ==========================================  ==

``V4L2_CID_HUE_AUTO`` ``(boolean)``
    Включает автоматическое управление оттенком со стороны устройства. Эффект
    установки ``V4L2_CID_HUE`` при включённом автоматическом управлении оттенком
    не определён, драйверы должны игнорировать такой запрос.

``V4L2_CID_WHITE_BALANCE_TEMPERATURE`` ``(integer)``
    Этот элемент управления задаёт настройки баланса белого как цветовую
    температуру в кельвинах. Драйвер должен иметь минимум от 2800
    (лампа накаливания) до 6500 (дневной свет). Для дополнительной информации о
    цветовой температуре см.
    `Wikipedia <http://en.wikipedia.org/wiki/Color_temperature>`__.

``V4L2_CID_SHARPNESS`` ``(integer)``
    Регулирует фильтры резкости в камере. Минимальное значение
    отключает фильтры, более высокие значения дают более резкое изображение.

``V4L2_CID_BACKLIGHT_COMPENSATION`` ``(integer)``
    Регулирует компенсацию задней засветки в камере. Минимальное значение
    отключает компенсацию задней засветки.

``V4L2_CID_CHROMA_AGC`` ``(boolean)``
    Автоматическое управление усилением цветности.

``V4L2_CID_CHROMA_GAIN`` ``(integer)``
    Регулирует управление усилением цветности (для использования, когда AGC
    цветности отключён).

``V4L2_CID_COLOR_KILLER`` ``(boolean)``
    Включает гаситель цвета (color killer) (т. е. принудительно делает
    чёрно-белое изображение в случае слабого видеосигнала).

.. _v4l2-colorfx:

``V4L2_CID_COLORFX`` ``(enum)``
    Выбирает цветовой эффект. Определены следующие значения:



.. tabularcolumns:: |p{5.7cm}|p{11.8cm}|

.. flat-table::
    :header-rows:  0
    :stub-columns: 0
    :widths: 11 24

    * - ``V4L2_COLORFX_NONE``
      - Цветовой эффект отключён.
    * - ``V4L2_COLORFX_ANTIQUE``
      - Эффект старения (старое фото).
    * - ``V4L2_COLORFX_ART_FREEZE``
      - Эффект морозного цвета.
    * - ``V4L2_COLORFX_AQUA``
      - Цвет воды, холодный тон.
    * - ``V4L2_COLORFX_BW``
      - Чёрно-белое.
    * - ``V4L2_COLORFX_EMBOSS``
      - Тиснение: блики и тени заменяют границы светлого/тёмного,
	а области низкой контрастности заполняются серым фоном.
    * - ``V4L2_COLORFX_GRASS_GREEN``
      - Травянисто-зелёный.
    * - ``V4L2_COLORFX_NEGATIVE``
      - Негатив.
    * - ``V4L2_COLORFX_SEPIA``
      - Тон сепии.
    * - ``V4L2_COLORFX_SKETCH``
      - Эскиз.
    * - ``V4L2_COLORFX_SKIN_WHITEN``
      - Отбеливание кожи.
    * - ``V4L2_COLORFX_SKY_BLUE``
      - Небесно-голубой.
    * - ``V4L2_COLORFX_SOLARIZATION``
      - Соляризация: изображение частично инвертируется по тону, инвертируются
	только цветовые значения выше или ниже определённого порога.
    * - ``V4L2_COLORFX_SILHOUETTE``
      - Силуэт (контур).
    * - ``V4L2_COLORFX_VIVID``
      - Яркие цвета.
    * - ``V4L2_COLORFX_SET_CBCR``
      - Цветностные компоненты Cb и Cr заменяются фиксированными коэффициентами,
	определяемыми элементом управления ``V4L2_CID_COLORFX_CBCR``.
    * - ``V4L2_COLORFX_SET_RGB``
      - Компоненты RGB заменяются фиксированными компонентами RGB, определяемыми
        элементом управления ``V4L2_CID_COLORFX_RGB``.


``V4L2_CID_COLORFX_RGB`` ``(integer)``
    Определяет коэффициенты красного, зелёного и синего для
    цветового эффекта ``V4L2_COLORFX_SET_RGB``.
    Биты [7:0] предоставленного 32-битного значения интерпретируются как
    компонента синего,
    биты [15:8] как компонента зелёного, биты [23:16] как компонента красного, а
    биты [31:24] должны быть нулевыми.

``V4L2_CID_COLORFX_CBCR`` ``(integer)``
    Определяет коэффициенты Cb и Cr для цветового эффекта
    ``V4L2_COLORFX_SET_CBCR``. Биты [7:0] предоставленного 32-битного значения
    интерпретируются как компонента Cr, биты [15:8] как компонента Cb, а биты
    [31:16] должны быть нулевыми.

``V4L2_CID_AUTOBRIGHTNESS`` ``(boolean)``
    Включает автоматическую яркость.

``V4L2_CID_ROTATE`` ``(integer)``
    Поворачивает изображение на заданный угол. Распространённые углы — 90, 270 и
    180. Поворот изображения на 90 и 270 поменяет местами высоту и
    ширину окна отображения. Необходимо задать новую высоту
    и ширину изображения с помощью ioctl
    :ref:`VIDIOC_S_FMT <VIDIOC_G_FMT>` в соответствии с выбранным
    углом поворота.

``V4L2_CID_BG_COLOR`` ``(integer)``
    Устанавливает цвет фона на текущем устройстве вывода. Цвет фона
    должен быть задан в формате RGB24. Предоставленное 32-битное
    значение интерпретируется так: биты 0-7 — информация о красном цвете, биты 8-15
    — информация о зелёном цвете, биты 16-23 — информация о синем цвете, а биты
    24-31 должны быть нулевыми.

``V4L2_CID_ILLUMINATORS_1 V4L2_CID_ILLUMINATORS_2`` ``(boolean)``
    Включают или выключают осветитель 1 или 2 устройства (обычно
    микроскопа).

``V4L2_CID_MIN_BUFFERS_FOR_CAPTURE`` ``(integer)``
    Это элемент управления только для чтения, который может быть прочитан
    приложением и использован как подсказка для определения числа буферов CAPTURE,
    передаваемых в REQBUFS. Значение — это минимальное число буферов CAPTURE,
    необходимое для работы оборудования. Этот элемент управления обязателен для
    декодеров с состоянием (stateful).

``V4L2_CID_MIN_BUFFERS_FOR_OUTPUT`` ``(integer)``
    Это элемент управления только для чтения, который может быть прочитан
    приложением и использован как подсказка для определения числа буферов OUTPUT,
    передаваемых в REQBUFS. Значение — это минимальное число буферов OUTPUT,
    необходимое для работы оборудования. Этот элемент управления обязателен для
    кодировщиков с состоянием (stateful).

.. _v4l2-alpha-component:

``V4L2_CID_ALPHA_COMPONENT`` ``(integer)``
    Устанавливает альфа-компоненту цвета. Когда устройство захвата (или очередь
    захвата устройства mem-to-mem) производит формат кадра, включающий
    альфа-компоненту (например,
    :ref:`packed RGB image formats <pixfmt-rgb>`), а значение альфа
    не определено устройством или входными данными mem-to-mem, этот
    элемент управления позволяет выбрать значение альфа-компоненты всех пикселей.
    Когда устройство вывода (или очередь вывода устройства mem-to-mem)
    потребляет формат кадра, не включающий альфа-компоненту, и
    устройство поддерживает обработку альфа-канала, этот элемент управления
    позволяет задать значение альфа-компоненты всех пикселей для дальнейшей
    обработки в устройстве.

``V4L2_CID_LASTP1``
    Конец предопределённых ID элементов управления (в настоящее время
    ``V4L2_CID_ALPHA_COMPONENT`` + 1).

``V4L2_CID_PRIVATE_BASE``
    ID первого пользовательского (специфичного для драйвера) элемента управления.
    Приложения, зависящие от конкретных пользовательских элементов управления,
    должны проверять имя и версию драйвера, см. :ref:`querycap`.

Приложения могут перечислять доступные элементы управления с помощью ioctl'ов
:ref:`VIDIOC_QUERYCTRL` и
:ref:`VIDIOC_QUERYMENU <VIDIOC_QUERYCTRL>`, получать и устанавливать значение
элемента управления с помощью ioctl'ов :ref:`VIDIOC_G_CTRL <VIDIOC_G_CTRL>` и
:ref:`VIDIOC_S_CTRL <VIDIOC_G_CTRL>`. Драйверы должны реализовывать
``VIDIOC_QUERYCTRL``, ``VIDIOC_G_CTRL`` и ``VIDIOC_S_CTRL``, когда
устройство имеет один или несколько элементов управления, ``VIDIOC_QUERYMENU``,
когда оно имеет один или несколько элементов управления типа меню.


.. _enum_all_controls:

Пример: Перечисление всех элементов управления
==============================================

.. code-block:: c

    struct v4l2_queryctrl queryctrl;
    struct v4l2_querymenu querymenu;

    static void enumerate_menu(__u32 id)
    {
	printf("  Menu items:\\n");

	memset(&querymenu, 0, sizeof(querymenu));
	querymenu.id = id;

	for (querymenu.index = queryctrl.minimum;
	     querymenu.index <= queryctrl.maximum;
	     querymenu.index++) {
	    if (0 == ioctl(fd, VIDIOC_QUERYMENU, &querymenu)) {
		printf("  %s\\n", querymenu.name);
	    }
	}
    }

    memset(&queryctrl, 0, sizeof(queryctrl));

    queryctrl.id = V4L2_CTRL_FLAG_NEXT_CTRL;
    while (0 == ioctl(fd, VIDIOC_QUERYCTRL, &queryctrl)) {
	if (!(queryctrl.flags & V4L2_CTRL_FLAG_DISABLED)) {
	    printf("Control %s\\n", queryctrl.name);

	    if (queryctrl.type == V4L2_CTRL_TYPE_MENU)
	        enumerate_menu(queryctrl.id);
        }

	queryctrl.id |= V4L2_CTRL_FLAG_NEXT_CTRL;
    }
    if (errno != EINVAL) {
	perror("VIDIOC_QUERYCTRL");
	exit(EXIT_FAILURE);
    }

Пример: Перечисление всех элементов управления, включая составные
=================================================================

.. code-block:: c

    struct v4l2_query_ext_ctrl query_ext_ctrl;

    memset(&query_ext_ctrl, 0, sizeof(query_ext_ctrl));

    query_ext_ctrl.id = V4L2_CTRL_FLAG_NEXT_CTRL | V4L2_CTRL_FLAG_NEXT_COMPOUND;
    while (0 == ioctl(fd, VIDIOC_QUERY_EXT_CTRL, &query_ext_ctrl)) {
	if (!(query_ext_ctrl.flags & V4L2_CTRL_FLAG_DISABLED)) {
	    printf("Control %s\\n", query_ext_ctrl.name);

	    if (query_ext_ctrl.type == V4L2_CTRL_TYPE_MENU)
	        enumerate_menu(query_ext_ctrl.id);
        }

	query_ext_ctrl.id |= V4L2_CTRL_FLAG_NEXT_CTRL | V4L2_CTRL_FLAG_NEXT_COMPOUND;
    }
    if (errno != EINVAL) {
	perror("VIDIOC_QUERY_EXT_CTRL");
	exit(EXIT_FAILURE);
    }

    memset(&queryctrl, 0, sizeof(queryctrl));

    for (queryctrl.id = V4L2_CID_BASE;
	 queryctrl.id < V4L2_CID_LASTP1;
	 queryctrl.id++) {
	if (0 == ioctl(fd, VIDIOC_QUERYCTRL, &queryctrl)) {
	    if (queryctrl.flags & V4L2_CTRL_FLAG_DISABLED)
		continue;

	    printf("Control %s\\n", queryctrl.name);

	    if (queryctrl.type == V4L2_CTRL_TYPE_MENU)
		enumerate_menu(queryctrl.id);
	} else {
	    if (errno == EINVAL)
		continue;

	    perror("VIDIOC_QUERYCTRL");
	    exit(EXIT_FAILURE);
	}
    }

    for (queryctrl.id = V4L2_CID_PRIVATE_BASE;;
	 queryctrl.id++) {
	if (0 == ioctl(fd, VIDIOC_QUERYCTRL, &queryctrl)) {
	    if (queryctrl.flags & V4L2_CTRL_FLAG_DISABLED)
		continue;

	    printf("Control %s\\n", queryctrl.name);

	    if (queryctrl.type == V4L2_CTRL_TYPE_MENU)
		enumerate_menu(queryctrl.id);
	} else {
	    if (errno == EINVAL)
		break;

	    perror("VIDIOC_QUERYCTRL");
	    exit(EXIT_FAILURE);
	}
    }


Пример: Изменение элементов управления
======================================

.. code-block:: c

    struct v4l2_queryctrl queryctrl;
    struct v4l2_control control;

    memset(&queryctrl, 0, sizeof(queryctrl));
    queryctrl.id = V4L2_CID_BRIGHTNESS;

    if (-1 == ioctl(fd, VIDIOC_QUERYCTRL, &queryctrl)) {
	if (errno != EINVAL) {
	    perror("VIDIOC_QUERYCTRL");
	    exit(EXIT_FAILURE);
	} else {
	    printf("V4L2_CID_BRIGHTNESS is not supported\n");
	}
    } else if (queryctrl.flags & V4L2_CTRL_FLAG_DISABLED) {
	printf("V4L2_CID_BRIGHTNESS is not supported\n");
    } else {
	memset(&control, 0, sizeof (control));
	control.id = V4L2_CID_BRIGHTNESS;
	control.value = queryctrl.default_value;

	if (-1 == ioctl(fd, VIDIOC_S_CTRL, &control)) {
	    perror("VIDIOC_S_CTRL");
	    exit(EXIT_FAILURE);
	}
    }

    memset(&control, 0, sizeof(control));
    control.id = V4L2_CID_CONTRAST;

    if (0 == ioctl(fd, VIDIOC_G_CTRL, &control)) {
	control.value += 1;

	/* The driver may clamp the value or return ERANGE, ignored here */

	if (-1 == ioctl(fd, VIDIOC_S_CTRL, &control)
	    && errno != ERANGE) {
	    perror("VIDIOC_S_CTRL");
	    exit(EXIT_FAILURE);
	}
    /* Ignore if V4L2_CID_CONTRAST is unsupported */
    } else if (errno != EINVAL) {
	perror("VIDIOC_G_CTRL");
	exit(EXIT_FAILURE);
    }

    control.id = V4L2_CID_AUDIO_MUTE;
    control.value = 1; /* silence */

    /* Errors ignored */
    ioctl(fd, VIDIOC_S_CTRL, &control);

.. [#f1]
   Использование ``V4L2_CID_PRIVATE_BASE`` проблематично, поскольку разные
   драйверы могут использовать один и тот же ID ``V4L2_CID_PRIVATE_BASE`` для
   разных элементов управления. Это затрудняет программную установку таких
   элементов управления, поскольку смысл элемента управления с этим ID зависит от
   драйвера. Чтобы решить это, драйверы используют уникальные ID, а ID
   ``V4L2_CID_PRIVATE_BASE`` отображаются на эти уникальные ID ядром. Считайте эти
   ID ``V4L2_CID_PRIVATE_BASE`` псевдонимами реальных ID.

   Многие приложения сегодня всё ещё используют ID ``V4L2_CID_PRIVATE_BASE``
   вместо использования :ref:`VIDIOC_QUERYCTRL` с
   флагом ``V4L2_CTRL_FLAG_NEXT_CTRL`` для перечисления всех ID, поэтому
   поддержка ``V4L2_CID_PRIVATE_BASE`` всё ещё присутствует.
