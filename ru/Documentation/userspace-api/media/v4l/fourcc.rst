Рекомендации по 4CC-кодам пиксельных форматов Video4Linux
=========================================================

В этом документе изложены рекомендации по 4CC-кодам Video4Linux, задаваемым
с помощью v4l2_fourcc(). Первый из символов определяет природу пиксельного
формата, сжатие и цветовое пространство. Интерпретация остальных трёх символов
зависит от первого.

Существующие 4CC-коды могут не следовать этим рекомендациям.

Raw bayer
---------

Сырыми форматами bayer используются следующие первые символы:

- B: сырой bayer, без сжатия
- b: сырой bayer, со сжатием DPCM
- a: сжатие по A-закону (A-law)
- u: сжатие по u-закону (u-law)

2-й символ: порядок пикселей

- B: BGGR
- G: GBRG
- g: GRBG
- R: RGGB

3-й символ: число бит на пиксель без сжатия 0--9, A--

4-й символ: число бит на пиксель со сжатием 0--9, A--
