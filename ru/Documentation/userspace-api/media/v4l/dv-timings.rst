*******************************
Тайминги цифрового видео (DV)
*******************************

Видеостандарты, рассмотренные до сих пор, имели дело с аналоговым ТВ
и соответствующими видеотаймингами. Сегодня существует гораздо больше различных
аппаратных интерфейсов, таких как интерфейсы телевидения высокой чёткости (HDMI), VGA,
разъёмы DVI и т.д., которые передают видеосигналы, и возникает необходимость
расширить API для выбора видеотаймингов для этих интерфейсов. Поскольку
расширить :ref:`v4l2_std_id <v4l2-std-id>` невозможно
из-за ограниченного числа доступных битов, был добавлен новый набор ioctl для
установки/получения видеотаймингов на входе и выходе.

Эти ioctl имеют дело с детальными таймингами цифрового видео, которые определяют
каждый видеоформат. Сюда входят такие параметры, как ширина и высота активного
видео, полярности сигналов, frontporch, backporch, ширины синхросигналов
и т.д. Заголовочный файл ``linux/v4l2-dv-timings.h`` можно использовать для получения
таймингов форматов стандартов :ref:`cea861` и :ref:`vesadmt`.

Для перечисления и запроса атрибутов DV-таймингов, поддерживаемых
устройством, приложения используют ioctl
:ref:`VIDIOC_ENUM_DV_TIMINGS` и
:ref:`VIDIOC_DV_TIMINGS_CAP`. Для установки
DV-таймингов устройства приложения используют ioctl
:ref:`VIDIOC_S_DV_TIMINGS <VIDIOC_G_DV_TIMINGS>`, а для получения
текущих DV-таймингов используют ioctl
:ref:`VIDIOC_G_DV_TIMINGS <VIDIOC_G_DV_TIMINGS>`. Для определения
DV-таймингов так, как их видит видеоприёмник, приложения используют ioctl
:ref:`VIDIOC_QUERY_DV_TIMINGS`.

Когда аппаратура обнаруживает изменение источника видео (например, видеосигнал
появляется или исчезает или меняется разрешение видео),
она выдаёт событие `V4L2_EVENT_SOURCE_CHANGE`. Используйте
:ref:`ioctl VIDIOC_SUBSCRIBE_EVENT <VIDIOC_SUBSCRIBE_EVENT>` и
:ref:`VIDIOC_DQEVENT`, чтобы проверить, было ли сообщено об этом событии.

Если видеосигнал изменился, то приложение должно остановить
потоковую передачу, освободить все буферы и вызвать :ref:`VIDIOC_QUERY_DV_TIMINGS`
для получения новых видеотаймингов, и, если они корректны, оно может установить
их, вызвав :ref:`ioctl VIDIOC_S_DV_TIMINGS <VIDIOC_G_DV_TIMINGS>`.
Это также обновит формат, поэтому используйте :ref:`ioctl VIDIOC_G_FMT <VIDIOC_G_FMT>`
для получения нового формата. Теперь приложение может выделить новые буферы
и снова начать потоковую передачу.

:ref:`VIDIOC_QUERY_DV_TIMINGS` лишь сообщит то, что
обнаружила аппаратура, но никогда не изменит конфигурацию. Если
текущие установленные тайминги и фактически обнаруженные тайминги различаются, то
обычно это означает, что вы не сможете захватить никакого
видео. Правильный подход — полагаться на событие `V4L2_EVENT_SOURCE_CHANGE`,
чтобы знать, когда что-то изменилось.

Приложения могут использовать флаги :ref:`input-capabilities` и
:ref:`output-capabilities`, чтобы определить, можно ли использовать ioctl
цифрового видео с данным входом или выходом.
