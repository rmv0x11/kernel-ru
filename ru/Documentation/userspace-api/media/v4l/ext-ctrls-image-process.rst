.. _image-process-controls:

*********************************************************
Справочник по управляющим элементам обработки изображения
*********************************************************

Класс управляющих элементов обработки изображения (Image Process) предназначен
для низкоуровневого управления функциями обработки изображения. В отличие от
``V4L2_CID_IMAGE_SOURCE_CLASS``, управляющие элементы этого класса влияют на
обработку изображения и не управляют его захватом.


.. _image-process-control-id:

Идентификаторы управляющих элементов обработки изображения
==========================================================

``V4L2_CID_IMAGE_PROC_CLASS (class)``
    Дескриптор класса IMAGE_PROC.

.. _v4l2-cid-link-freq:

``V4L2_CID_LINK_FREQ (integer menu)``
    Частота шины данных (например, параллельной или CSI-2).

.. _v4l2-cid-pixel-rate:

``V4L2_CID_PIXEL_RATE (64-bit integer)``
    Частота выборки пикселей в массиве пикселей устройства. Этот управляющий
    элемент доступен только для чтения, его единица измерения — пиксели в секунду.

    Некоторые устройства используют горизонтальное и вертикальное гашение для
    настройки частоты кадров. Частоту кадров можно вычислить из частоты пикселей,
    прямоугольника аналоговой обрезки, а также горизонтального и вертикального
    гашения. Управляющий элемент частоты пикселей может находиться в другом
    субустройстве, чем управляющие элементы гашения и конфигурация прямоугольника
    аналоговой обрезки.

    Настройка частоты кадров выполняется путём выбора желаемого горизонтального и
    вертикального гашения. Единица измерения этого управляющего элемента — Гц.

``V4L2_CID_TEST_PATTERN (menu)``
    Некоторые устройства захвата/отображения/датчики обладают способностью
    генерировать изображения тестовых таблиц. Эти аппаратно-специфичные тестовые
    таблицы можно использовать для проверки правильности работы устройства.

``V4L2_CID_DEINTERLACING_MODE (menu)``
    Режим деинтерлейсинга видео (например, Bob, Weave, ...). Пункты меню зависят
    от драйвера и документированы в :ref:`uapi-v4l-drivers`.

``V4L2_CID_DIGITAL_GAIN (integer)``
    Цифровое усиление — это значение, на которое умножаются все компоненты цвета.
    Обычно применяемое цифровое усиление равно значению управляющего элемента,
    делённому, например, на 0x100, то есть для отсутствия цифрового усиления
    значение управляющего элемента должно быть равно 0x100. Конфигурация без
    усиления также обычно является значением по умолчанию.
