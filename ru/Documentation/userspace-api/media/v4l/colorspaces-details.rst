.. _col-smpte-170m:

Цветовое пространство SMPTE 170M (V4L2_COLORSPACE_SMPTE170M)
============================================================

Стандарт :ref:`smpte170m` определяет цветовое пространство, используемое в
NTSC и PAL, а также в SDTV в целом. Передаточной функцией по умолчанию
является ``V4L2_XFER_FUNC_709``. Кодированием Y'CbCr по умолчанию является
``V4L2_YCBCR_ENC_601``. Квантованием Y'CbCr по умолчанию является ограниченный
диапазон (limited range). Координаты цветности основных цветов и опорного
белого таковы:

.. flat-table:: Координаты цветности SMPTE 170M
    :header-rows:  1
    :stub-columns: 0
    :widths:       1 1 2

    * - Цвет
      - x
      - y
    * - Красный
      - 0.630
      - 0.340
    * - Зелёный
      - 0.310
      - 0.595
    * - Синий
      - 0.155
      - 0.070
    * - Опорный белый (D65)
      - 0.3127
      - 0.3290


Координаты цветности красного, зелёного и синего также часто называют набором
SMPTE C, поэтому это цветовое пространство иногда также называют SMPTE C.

Передаточная функция, определённая для SMPTE 170M, совпадает с функцией,
определённой в Rec. 709.

.. math::

    L' = -1.099(-L)^{0.45} + 0.099 \text{, for } L \le-0.018

    L' = 4.5L \text{, for } -0.018 < L < 0.018

    L' = 1.099L^{0.45} - 0.099 \text{, for } L \ge 0.018

Обратная передаточная функция:

.. math::

    L = -\left( \frac{L' - 0.099}{-1.099} \right) ^{\frac{1}{0.45}} \text{, for } L' \le -0.081

    L = \frac{L'}{4.5} \text{, for } -0.081 < L' < 0.081

    L = \left(\frac{L' + 0.099}{1.099}\right)^{\frac{1}{0.45} } \text{, for } L' \ge 0.081

Яркость (Y') и цветоразностные сигналы (Cb и Cr) получаются с помощью
следующего кодирования ``V4L2_YCBCR_ENC_601``:

.. math::

    Y' = 0.2990R' + 0.5870G' + 0.1140B'

    Cb = -0.1687R' - 0.3313G' + 0.5B'

    Cr = 0.5R' - 0.4187G' - 0.0813B'

Y' ограничивается диапазоном [0…1], а Cb и Cr ограничиваются диапазоном
[-0.5…0.5]. Это преобразование в Y'CbCr идентично преобразованию, определённому
в стандарте :ref:`itu601`, и это цветовое пространство иногда также называют
BT.601, хотя BT.601 не упоминает никаких основных цветов.

Квантованием по умолчанию является ограниченный диапазон, но возможен и полный
диапазон, хотя он встречается редко.


.. _col-rec709:

Цветовое пространство Rec. 709 (V4L2_COLORSPACE_REC709)
=======================================================

Стандарт :ref:`itu709` определяет цветовое пространство, используемое в HDTV
в целом. Передаточной функцией по умолчанию является ``V4L2_XFER_FUNC_709``.
Кодированием Y'CbCr по умолчанию является ``V4L2_YCBCR_ENC_709``. Квантованием
Y'CbCr по умолчанию является ограниченный диапазон. Координаты цветности
основных цветов и опорного белого таковы:

.. flat-table:: Координаты цветности Rec. 709
    :header-rows:  1
    :stub-columns: 0
    :widths:       1 1 2

    * - Цвет
      - x
      - y
    * - Красный
      - 0.640
      - 0.330
    * - Зелёный
      - 0.300
      - 0.600
    * - Синий
      - 0.150
      - 0.060
    * - Опорный белый (D65)
      - 0.3127
      - 0.3290


Полное название этого стандарта — Rec. ITU-R BT.709-5.

Передаточная функция. Обычно L находится в диапазоне [0…1], но для расширенной
гаммы кодирования xvYCC допускаются значения вне этого диапазона.

.. math::

    L' = -1.099(-L)^{0.45} + 0.099 \text{, for } L \le -0.018

    L' = 4.5L \text{, for } -0.018 < L < 0.018

    L' = 1.099L^{0.45} - 0.099 \text{, for } L \ge 0.018

Обратная передаточная функция:

.. math::

    L = -\left( \frac{L' - 0.099}{-1.099} \right)^\frac{1}{0.45} \text{, for } L' \le -0.081

    L = \frac{L'}{4.5}\text{, for } -0.081 < L' < 0.081

    L = \left(\frac{L' + 0.099}{1.099}\right)^{\frac{1}{0.45} } \text{, for } L' \ge 0.081

Яркость (Y') и цветоразностные сигналы (Cb и Cr) получаются с помощью
следующего кодирования ``V4L2_YCBCR_ENC_709``:

.. math::

    Y' = 0.2126R' + 0.7152G' + 0.0722B'

    Cb = -0.1146R' - 0.3854G' + 0.5B'

    Cr = 0.5R' - 0.4542G' - 0.0458B'

Y' ограничивается диапазоном [0…1], а Cb и Cr ограничиваются диапазоном
[-0.5…0.5].

Квантованием по умолчанию является ограниченный диапазон, но возможен и полный
диапазон, хотя он встречается редко.

Описанное выше кодирование ``V4L2_YCBCR_ENC_709`` является используемым по
умолчанию для этого цветового пространства, но оно может быть переопределено на
``V4L2_YCBCR_ENC_601``, и в этом случае используется кодирование Y'CbCr BT.601.

С этим цветовым пространством также возможны два дополнительных кодирования
Y'CbCr с расширенной гаммой:

Кодирование xvYCC 709 (``V4L2_YCBCR_ENC_XV709``, :ref:`xvycc`) похоже на
кодирование Rec. 709, но оно допускает значения R', G' и B', которые находятся
вне диапазона [0…1]. Получаемые значения Y', Cb и Cr масштабируются и смещаются
в соответствии с формулой ограниченного диапазона:

.. math::

    Y' = \frac{219}{256} * (0.2126R' + 0.7152G' + 0.0722B') + \frac{16}{256}

    Cb = \frac{224}{256} * (-0.1146R' - 0.3854G' + 0.5B')

    Cr = \frac{224}{256} * (0.5R' - 0.4542G' - 0.0458B')

Кодирование xvYCC 601 (``V4L2_YCBCR_ENC_XV601``, :ref:`xvycc`) похоже на
кодирование BT.601, но оно допускает значения R', G' и B', которые находятся
вне диапазона [0…1]. Получаемые значения Y', Cb и Cr масштабируются и смещаются
в соответствии с формулой ограниченного диапазона:

.. math::

    Y' = \frac{219}{256} * (0.2990R' + 0.5870G' + 0.1140B') + \frac{16}{256}

    Cb = \frac{224}{256} * (-0.1687R' - 0.3313G' + 0.5B')

    Cr = \frac{224}{256} * (0.5R' - 0.4187G' - 0.0813B')

Y' ограничивается диапазоном [0…1], а Cb и Cr ограничиваются диапазоном
[-0.5…0.5] и квантуются без дальнейшего масштабирования или смещения.
Нестандартные кодирования xvYCC 709 или xvYCC 601 могут быть выбраны с помощью
``V4L2_YCBCR_ENC_XV709`` или ``V4L2_YCBCR_ENC_XV601``.
Как видно из формул xvYCC, эти кодирования всегда используют квантование
ограниченного диапазона, варианта с полным диапазоном нет. Весь смысл этих
кодирований с расширенной гаммой состоит в том, что значения вне ограниченного
диапазона по-прежнему действительны, хотя они отображаются на значения R', G' и
B' вне диапазона [0…1] и поэтому находятся вне гаммы цветового пространства
Rec. 709.


.. _col-srgb:

Цветовое пространство sRGB (V4L2_COLORSPACE_SRGB)
=================================================

Стандарт :ref:`srgb` определяет цветовое пространство, используемое
большинством веб-камер и компьютерной графикой. Передаточной функцией по
умолчанию является ``V4L2_XFER_FUNC_SRGB``. Кодированием Y'CbCr по умолчанию
является ``V4L2_YCBCR_ENC_601``. Квантованием Y'CbCr по умолчанию является
ограниченный диапазон.

Обратите внимание, что стандарт :ref:`sycc` задаёт квантование полного
диапазона, однако всё текущее оборудование захвата, поддерживаемое ядром,
преобразует R'G'B' в Y'CbCr ограниченного диапазона. Поэтому выбор полного
диапазона по умолчанию нарушил бы то, как приложения интерпретируют диапазон
квантования.

Координаты цветности основных цветов и опорного белого таковы:

.. flat-table:: Координаты цветности sRGB
    :header-rows:  1
    :stub-columns: 0
    :widths:       1 1 2

    * - Цвет
      - x
      - y
    * - Красный
      - 0.640
      - 0.330
    * - Зелёный
      - 0.300
      - 0.600
    * - Синий
      - 0.150
      - 0.060
    * - Опорный белый (D65)
      - 0.3127
      - 0.3290


Эти координаты цветности идентичны цветовому пространству Rec. 709.

Передаточная функция. Обратите внимание, что отрицательные значения L
используются только при преобразовании Y'CbCr.

.. math::

    L' = -1.055(-L)^{\frac{1}{2.4} } + 0.055\text{, for }L < -0.0031308

    L' = 12.92L\text{, for }-0.0031308 \le L \le 0.0031308

    L' = 1.055L ^{\frac{1}{2.4} } - 0.055\text{, for }0.0031308 < L \le 1

Обратная передаточная функция:

.. math::

    L = -((-L' + 0.055) / 1.055) ^{2.4}\text{, for }L' < -0.04045

    L = L' / 12.92\text{, for }-0.04045 \le L' \le 0.04045

    L = ((L' + 0.055) / 1.055) ^{2.4}\text{, for }L' > 0.04045

Яркость (Y') и цветоразностные сигналы (Cb и Cr) получаются с помощью
следующего кодирования ``V4L2_YCBCR_ENC_601``, как определено в :ref:`sycc`:

.. math::

    Y' = 0.2990R' + 0.5870G' + 0.1140B'

    Cb = -0.1687R' - 0.3313G' + 0.5B'

    Cr = 0.5R' - 0.4187G' - 0.0813B'

Y' ограничивается диапазоном [0…1], а Cb и Cr ограничиваются диапазоном
[-0.5…0.5]. Это преобразование идентично преобразованию, определённому в SMPTE
170M/BT.601. Квантованием Y'CbCr является ограниченный диапазон.

Цветовое пространство BT.2020 (V4L2_COLORSPACE_BT2020)

Стандарт :ref:`itu2020` определяет цветовое пространство, используемое
телевидением сверхвысокой чёткости (UHDTV). Передаточной функцией по умолчанию
является ``V4L2_XFER_FUNC_709``. Кодированием Y'CbCr по умолчанию является
``V4L2_YCBCR_ENC_BT2020``. Квантованием Y'CbCr по умолчанию является
ограниченный диапазон.
Координаты цветности основных цветов и опорного белого таковы:

.. flat-table:: Координаты цветности BT.2020
    :header-rows:  1
    :stub-columns: 0
    :widths:       1 1 2

    * - Цвет
      - x
      - y
    * - Красный
      - 0.708
      - 0.292
    * - Зелёный
      - 0.170
      - 0.797
    * - Синий
      - 0.131
      - 0.046
    * - Опорный белый (D65)
      - 0.3127
      - 0.3290



Передаточная функция (та же, что и в Rec. 709):

.. math::

    L' = 4.5L\text{, for }0 \le L < 0.018

    L' = 1.099L ^{0.45} - 0.099\text{, for } 0.018 \le L \le 1

Обратная передаточная функция:

.. math::

    L = L' / 4.5\text{, for } L' < 0.081

    L = \left( \frac{L' + 0.099}{1.099}\right) ^{\frac{1}{0.45} }\text{, for } L' \ge 0.081

Обратите внимание, что хотя Rec. 709 определена стандартом :ref:`itu2020` как
передаточная функция по умолчанию, на практике это цветовое пространство часто
используется с :ref:`xf-smpte-2084`. В частности, эту комбинацию используют
диски Ultra HD Blu-ray.

Яркость (Y') и цветоразностные сигналы (Cb и Cr) получаются с помощью
следующего кодирования ``V4L2_YCBCR_ENC_BT2020``:

.. math::

    Y' = 0.2627R' + 0.6780G' + 0.0593B'

    Cb = -0.1396R' - 0.3604G' + 0.5B'

    Cr = 0.5R' - 0.4598G' - 0.0402B'

Y' ограничивается диапазоном [0…1], а Cb и Cr ограничиваются диапазоном
[-0.5…0.5]. Квантованием Y'CbCr является ограниченный диапазон.

Существует также альтернативное кодирование с постоянной яркостью R'G'B' в
Yc'CbcCrc (``V4L2_YCBCR_ENC_BT2020_CONST_LUM``):

Luma:

.. math::
    :nowrap:

    \begin{align*}
    Yc' = (0.2627R + 0.6780G + 0.0593B)'& \\
    B' - Yc' \le 0:& \\
        &Cbc = (B' - Yc') / 1.9404 \\
    B' - Yc' > 0: & \\
        &Cbc = (B' - Yc') / 1.5816 \\
    R' - Yc' \le 0:& \\
        &Crc = (R' - Y') / 1.7184 \\
    R' - Yc' > 0:& \\
        &Crc = (R' - Y') / 0.9936
    \end{align*}

Yc' ограничивается диапазоном [0…1], а Cbc и Crc ограничиваются диапазоном
[-0.5…0.5]. Квантованием Yc'CbcCrc является ограниченный диапазон.


.. _col-dcip3:

Цветовое пространство DCI-P3 (V4L2_COLORSPACE_DCI_P3)
=====================================================

Стандарт :ref:`smpte431` определяет цветовое пространство, используемое
кинопроекторами, которые используют цветовое пространство DCI-P3. Передаточной
функцией по умолчанию является ``V4L2_XFER_FUNC_DCI_P3``. Кодированием Y'CbCr по
умолчанию является ``V4L2_YCBCR_ENC_709``. Квантованием Y'CbCr по умолчанию

.. note::

   Обратите внимание, что этот стандарт цветового пространства не задаёт
   кодирование Y'CbCr, поскольку он не предназначен для кодирования в Y'CbCr.
   Поэтому это кодирование Y'CbCr по умолчанию было выбрано потому, что оно
   является кодированием HDTV.

следующего кодирования ``V4L2_YCBCR_ENC_SMPTE240M``:

.. math::

    Y' = 0.2122R' + 0.7013G' + 0.0865B'

    Cb = -0.1161R' - 0.3839G' + 0.5B'

    Cr = 0.5R' - 0.4451G' - 0.0549B'

Y' ограничивается диапазоном [0…1], а Cb и Cr ограничиваются диапазоном
[-0.5…0.5]. Квантованием Y'CbCr является ограниченный диапазон.


.. _col-sysm:

Цветовое пространство NTSC 1953 (V4L2_COLORSPACE_470_SYSTEM_M)
==============================================================

Этот стандарт определяет цветовое пространство, использовавшееся в NTSC в 1953
году. На практике это цветовое пространство устарело, и вместо него следует
использовать SMPTE 170M. Передаточной функцией по умолчанию является
``V4L2_XFER_FUNC_709``. Кодированием Y'CbCr по умолчанию является
``V4L2_YCBCR_ENC_601``. Квантованием Y'CbCr по умолчанию является ограниченный
диапазон. Координаты цветности основных цветов и опорного белого таковы:


.. flat-table:: Координаты цветности NTSC 1953
    :header-rows:  1
    :stub-columns: 0
    :widths:       1 1 2

    * - Цвет
      - x
      - y
    * - Красный
      - 0.67
      - 0.33
    * - Зелёный
      - 0.21
      - 0.71
    * - Синий
      - 0.14
      - 0.08
    * - Опорный белый (C)
      - 0.310
      - 0.316


.. note::

   Это цветовое пространство использует осветитель C вместо D65 в качестве
   опорного белого. Чтобы корректно преобразовать изображение в этом цветовом
   пространстве в другое, использующее D65, необходимо применить алгоритм
   хроматической адаптации, такой как метод Брэдфорда.

Передаточная функция никогда не была должным образом определена для NTSC 1953.
В литературе рекомендуется передаточная функция Rec. 709:

.. math::

    L' = 4.5L\text{, for } 0 \le L < 0.018

    L' = 1.099L ^{0.45} - 0.099\text{, for } 0.018 \le L \le 1

Обратная передаточная функция:

.. math::

    L = \frac{L'}{4.5} \text{, for } L' < 0.081

    L = \left( \frac{L' + 0.099}{1.099}\right) ^{\frac{1}{0.45} }\text{, for } L' \ge 0.081

Яркость (Y') и цветоразностные сигналы (Cb и Cr) получаются с помощью
следующего кодирования ``V4L2_YCBCR_ENC_601``:

.. math::

    Y' = 0.2990R' + 0.5870G' + 0.1140B'

    Cb = -0.1687R' - 0.3313G' + 0.5B'

    Cr = 0.5R' - 0.4187G' - 0.0813B'

Y' ограничивается диапазоном [0…1], а Cb и Cr ограничиваются диапазоном
[-0.5…0.5]. Квантованием Y'CbCr является ограниченный диапазон. Это
преобразование идентично преобразованию, определённому в SMPTE 170M/BT.601.

.. _col-jpeg:

Цветовое пространство JPEG (V4L2_COLORSPACE_JPEG)
=================================================

Это цветовое пространство определяет цветовое пространство, используемое
большинством форматов (Motion-)JPEG. Координаты цветности основных цветов и
опорного белого идентичны sRGB. Используемой передаточной функцией является
``V4L2_XFER_FUNC_SRGB``. Кодированием Y'CbCr является ``V4L2_YCBCR_ENC_601``
с квантованием полного диапазона, где Y' масштабируется до [0…255], а Cb/Cr
масштабируются до [-128…128] и затем обрезаются до [-128…127].

.. note::

   Стандарт JPEG фактически не хранит информацию о цветовом пространстве.
   Поэтому если используется что-то отличное от sRGB, то драйверу придётся
   задать эту информацию явно. По сути, ``V4L2_COLORSPACE_JPEG`` можно считать
   сокращением для ``V4L2_COLORSPACE_SRGB``, ``V4L2_XFER_FUNC_SRGB``,
   ``V4L2_YCBCR_ENC_601`` и ``V4L2_QUANTIZATION_FULL_RANGE``.

.. _xf-smpte-2084:

Передаточная функция SMPTE 2084 (V4L2_XFER_FUNC_SMPTE2084)
==========================================================

Стандарт :ref:`smpte2084` определяет передаточную функцию, используемую для
контента с высоким динамическим диапазоном (High Dynamic Range).

Константы:
    m1 = (2610 / 4096) / 4

    m2 = (2523 / 4096) * 128

    c1 = 3424 / 4096

    c2 = (2413 / 4096) * 32

    c3 = (2392 / 4096) * 32

Передаточная функция:
    L' = ((c1 + c2 * L\ :sup:`m1`) / (1 + c3 * L\ :sup:`m1`))\ :sup:`m2`

Обратная передаточная функция:
    L = (max(L':sup:`1/m2` - c1, 0) / (c2 - c3 *
    L'\ :sup:`1/m2`))\ :sup:`1/m1`

Будьте внимательны при преобразовании между этой передаточной функцией и
передаточными функциями, не относящимися к HDR: линейные значения RGB [0…1]
контента HDR отображаются на диапазон яркости от 0 до 10000 кд/м\ :sup:`2`,
тогда как линейные значения RGB не-HDR (известного также как стандартный
динамический диапазон, Standard Dynamic Range, или SDR) отображаются на диапазон
яркости от 0 до 100 кд/м\ :sup:`2`.

Чтобы перейти от SDR к HDR, нужно сначала разделить L на 100. Чтобы перейти в
обратном направлении, нужно умножить L на 100. Разумеется, при этом все значения
яркости свыше 100 кд/м\ :sup:`2` ограничиваются до 100 кд/м\ :sup:`2`.

Существуют более эффективные методы; см., например, :ref:`colimg`, где
содержится более подробная информация об этом.
