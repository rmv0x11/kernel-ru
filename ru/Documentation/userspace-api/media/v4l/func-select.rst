*************
V4L2 select()
*************

Имя
===

v4l2-select - Синхронное мультиплексирование ввода-вывода

Синтаксис
=========

.. code-block:: c

    #include <sys/time.h>
    #include <sys/types.h>
    #include <unistd.h>

.. c:function:: int select( int nfds, fd_set *readfds, fd_set *writefds, fd_set *exceptfds, struct timeval *timeout )

Аргументы
=========

``nfds``
  Наибольший по номеру файловый дескриптор в любом из трёх наборов плюс 1.

``readfds``
  Файловые дескрипторы, за которыми ведётся наблюдение на случай, если вызов read() не будет блокироваться.

``writefds``
  Файловые дескрипторы, за которыми ведётся наблюдение на случай, если вызов write() не будет блокироваться.

``exceptfds``
  Файловые дескрипторы, за которыми ведётся наблюдение в ожидании событий V4L2.

``timeout``
  Максимальное время ожидания.

Описание
========

С помощью функции :c:func:`select()` приложения могут приостановить
выполнение до тех пор, пока драйвер не захватит данные или не будет готов
принять данные для вывода.

Когда согласован потоковый ввод-вывод, эта функция ожидает, пока буфер не
будет заполнен или отображён и не сможет быть извлечён из очереди с помощью
ioctl :ref:`VIDIOC_DQBUF <VIDIOC_QBUF>`. Когда буферы уже находятся в
исходящей очереди драйвера, функция возвращается немедленно.

В случае успеха :c:func:`select()` возвращает общее количество установленных
битов в ``fd_set``. Если истекло время ожидания функции, она возвращает
значение ноль. При неудаче она возвращает -1, и переменной ``errno``
присваивается соответствующее значение. Если приложение ещё не вызвало
:ref:`VIDIOC_QBUF` или
:ref:`VIDIOC_STREAMON`, функция :c:func:`select()`
завершается успешно, устанавливая бит файлового дескриптора в ``readfds``
или ``writefds``, но последующие вызовы :ref:`VIDIOC_DQBUF <VIDIOC_QBUF>`
завершатся неудачей. [#f1]_

Когда согласовано использование функции :c:func:`read()`, а драйвер ещё не
ведёт захват, функция :c:func:`select()` начинает захват. Если он не удаётся,
:c:func:`select()` возвращается успешно, а последующий вызов :c:func:`read()`,
который также пытается начать захват, вернёт соответствующий код ошибки. Когда
драйвер ведёт захват непрерывно (в отличие, например, от неподвижных
изображений) и данные уже доступны, функция :c:func:`select()` возвращается
немедленно.

Когда согласовано использование функции :c:func:`write()`, функция
:c:func:`select()` просто ожидает, пока драйвер не будет готов к
неблокирующему вызову :c:func:`write()`.

Все драйверы, реализующие функцию :c:func:`read()` или :c:func:`write()`
либо потоковый ввод-вывод, должны также поддерживать функцию :c:func:`select()`.

Дополнительные сведения см. на странице руководства :c:func:`select()`.

Возвращаемое значение
=====================

В случае успеха :c:func:`select()` возвращает количество дескрипторов,
содержащихся в трёх возвращаемых наборах дескрипторов, которое будет равно
нулю, если истекло время ожидания. В случае ошибки возвращается -1, и
переменной ``errno`` присваивается соответствующее значение; наборы и
``timeout`` не определены. Возможные коды ошибок:

EBADF
    Один или несколько наборов файловых дескрипторов содержали файловый
    дескриптор, который не открыт.

EBUSY
    Драйвер не поддерживает несколько потоков чтения или записи, а
    устройство уже используется.

EFAULT
    Указатель ``readfds``, ``writefds``, ``exceptfds`` или ``timeout``
    ссылается на недоступную область памяти.

EINTR
    Вызов был прерван сигналом.

EINVAL
    Аргумент ``nfds`` меньше нуля или больше
    ``FD_SETSIZE``.

.. [#f1]
   Ядро Linux реализует :c:func:`select()` подобно функции
   :c:func:`poll()`, но :c:func:`select()` не может
   вернуть ``POLLERR``.
