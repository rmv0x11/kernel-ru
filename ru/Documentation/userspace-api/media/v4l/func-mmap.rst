***********
V4L2 mmap()
***********

Имя
===

v4l2-mmap - Отображение памяти устройства в адресное пространство приложения

Синтаксис
=========

.. code-block:: c

    #include <unistd.h>
    #include <sys/mman.h>

.. c:function:: void *mmap( void *start, size_t length, int prot, int flags, int fd, off_t offset )

Аргументы
=========

``start``
    Отобразить буфер по этому адресу в адресном пространстве приложения.
    Если указан флаг ``MAP_FIXED``, значение ``start`` должно быть
    кратно размеру страницы, и mmap завершится с ошибкой, если указанный
    адрес не может быть использован. Использование этого параметра не
    рекомендуется; приложениям следует просто указывать здесь нулевой
    указатель ``NULL``.

``length``
    Длина области памяти для отображения. Это значение должно совпадать
    со значением, возвращаемым драйвером в поле ``length`` структуры
    :c:type:`v4l2_buffer` для однопланового (single-planar) API, и со
    значением, возвращаемым драйвером в поле ``length`` структуры
    :c:type:`v4l2_plane` для многопланового (multi-planar) API.

``prot``
    Аргумент ``prot`` описывает желаемую защиту памяти. Независимо от
    типа устройства и направления обмена данными он должен быть установлен
    в ``PROT_READ`` | ``PROT_WRITE``, разрешая доступ на чтение и запись
    к буферам изображений. Драйверы должны поддерживать как минимум эту
    комбинацию флагов.

    .. note::

      #. Модуль ядра Linux ``videobuf``, используемый некоторыми
	 драйверами, поддерживает только ``PROT_READ`` | ``PROT_WRITE``. Если
	 драйвер не поддерживает желаемую защиту, функция
	 :c:func:`mmap()` завершается с ошибкой.

      #. Обращения к памяти устройства (например, к памяти на графической
	 карте с аппаратурой видеозахвата) могут приводить к снижению
	 производительности по сравнению с обращениями к основной памяти,
	 либо чтение может быть существенно медленнее записи или наоборот.
	 В таких случаях другие методы ввода-вывода могут быть более
	 эффективными.

``flags``
    Параметр ``flags`` задаёт тип отображаемого объекта, параметры
    отображения, а также то, являются ли изменения, внесённые в
    отображённую копию страницы, приватными для процесса или должны быть
    разделяемыми с другими ссылками.

    Флаг ``MAP_FIXED`` требует, чтобы драйвер не выбирал никакой другой
    адрес, кроме указанного. Если указанный адрес не может быть
    использован, :c:func:`mmap()` завершится с ошибкой. Если указан
    ``MAP_FIXED``, значение ``start`` должно быть кратно размеру страницы.
    Использование этого параметра не рекомендуется.

    Должен быть установлен один из флагов ``MAP_SHARED`` или ``MAP_PRIVATE``.
    Флаг ``MAP_SHARED`` позволяет приложениям разделять отображённую память
    с другими (например, дочерними) процессами.

    .. note::

       Модуль Linux ``videobuf``, используемый некоторыми драйверами,
       поддерживает только ``MAP_SHARED``. Флаг ``MAP_PRIVATE`` запрашивает
       семантику копирования при записи (copy-on-write). Приложениям V4L2
       не следует устанавливать флаги ``MAP_PRIVATE``, ``MAP_DENYWRITE``,
       ``MAP_EXECUTABLE`` или ``MAP_ANON``.

``fd``
    Файловый дескриптор, возвращаемый функцией :c:func:`open()`.

``offset``
    Смещение буфера в памяти устройства. Это значение должно совпадать
    со значением, возвращаемым драйвером в поле ``offset`` объединения
    ``m`` структуры :c:type:`v4l2_buffer` для однопланового
    (single-planar) API, и со значением, возвращаемым драйвером в поле
    ``mem_offset`` объединения ``m`` структуры :c:type:`v4l2_plane`
    для многопланового (multi-planar) API.

Описание
========

Функция :c:func:`mmap()` запрашивает отображение ``length`` байт,
начиная со смещения ``offset`` в памяти устройства, заданного через
``fd``, в адресное пространство приложения, предпочтительно по адресу
``start``. Этот последний адрес является лишь подсказкой и обычно
задаётся равным 0.

Подходящие значения параметров length и offset запрашиваются с помощью
ioctl :ref:`VIDIOC_QUERYBUF`. Буферы должны быть выделены с помощью
ioctl :ref:`VIDIOC_REQBUFS` до того, как их можно будет запросить.

Для отмены отображения буферов используется функция :c:func:`munmap()`.

Возвращаемое значение
=====================

В случае успеха :c:func:`mmap()` возвращает указатель на отображённый
буфер. В случае ошибки возвращается ``MAP_FAILED`` (-1), а переменная
``errno`` устанавливается соответствующим образом. Возможные коды
ошибок:

EBADF
    ``fd`` не является корректным файловым дескриптором.

EACCES
    ``fd`` не открыт для чтения и записи.

EINVAL
    Значения ``start``, ``length`` или ``offset`` не подходят.
    (Например, они слишком велики или не выровнены по границе
    ``PAGESIZE``.)

    Значение ``flags`` или ``prot`` не поддерживается.

    Не было выделено ни одного буфера с помощью ioctl
    :ref:`VIDIOC_REQBUFS`.

ENOMEM
    Для выполнения запроса было недостаточно физической или виртуальной
    памяти.
