*********************************************************
Справочник по элементам управления источником изображения
*********************************************************

Класс элементов управления источником изображения (Image Source) предназначен
для низкоуровневого управления устройствами — источниками изображения, такими
как датчики изображения. Эти устройства оснащены аналого-цифровым
преобразователем и шинным передатчиком для передачи данных изображения за
пределы устройства.


.. _image-source-control-id:

Идентификаторы элементов управления источником изображения
===========================================================

``V4L2_CID_IMAGE_SOURCE_CLASS (class)``
    Дескриптор класса IMAGE_SOURCE.

``V4L2_CID_VBLANK (integer)``
    Вертикальное гашение. Период простоя после каждого кадра, в течение которого
    данные изображения не формируются. Единицей измерения вертикального гашения
    является строка. Каждая строка имеет длину, равную ширине изображения плюс
    горизонтальное гашение, при частоте пикселей, заданной элементом управления
    ``V4L2_CID_PIXEL_RATE`` в том же sub-device.

``V4L2_CID_HBLANK (integer)``
    Горизонтальное гашение. Период простоя после каждой строки данных
    изображения, в течение которого данные изображения не формируются. Единицей
    измерения горизонтального гашения являются пиксели.

``V4L2_CID_ANALOGUE_GAIN (integer)``
    Аналоговое усиление — это усиление, влияющее на все цветовые компоненты в
    матрице пикселей. Операция усиления выполняется в аналоговой области до
    A/D-преобразования.

``V4L2_CID_TEST_PATTERN_RED (integer)``
    Красная цветовая компонента тестового шаблона.

``V4L2_CID_TEST_PATTERN_GREENR (integer)``
    Зелёная (рядом с красной) цветовая компонента тестового шаблона.

``V4L2_CID_TEST_PATTERN_BLUE (integer)``
    Синяя цветовая компонента тестового шаблона.

``V4L2_CID_TEST_PATTERN_GREENB (integer)``
    Зелёная (рядом с синей) цветовая компонента тестового шаблона.

``V4L2_CID_UNIT_CELL_SIZE (struct)``
    Этот элемент управления возвращает размер элементарной ячейки в нанометрах.
    Структура :c:type:`v4l2_area` предоставляет ширину и высоту в отдельных
    полях, чтобы учитывать асимметричные пиксели.
    Этот элемент управления не учитывает какое-либо возможное аппаратное
    объединение (binning).
    Элементарная ячейка состоит из всей площади пикселя — как чувствительной,
    так и нечувствительной.
    Этот элемент управления необходим для автоматической калибровки
    датчиков/камер.

.. c:type:: v4l2_area

.. flat-table:: struct v4l2_area
    :header-rows:  0
    :stub-columns: 0
    :widths:       1 1 2

    * - __u32
      - ``width``
      - Ширина области.
    * - __u32
      - ``height``
      - Высота области.

``V4L2_CID_NOTIFY_GAINS (integer array)``
    Датчик уведомляется о том, какие коэффициенты усиления будут применены к
    различным цветовым каналам при последующей обработке (например, со стороны
    ISP). Датчик лишь информируется об этих значениях на случай, если он
    выполняет обработку, которая их требует, но сам он не применяет их к
    выходным пикселям.

    В настоящее время это определено только для датчиков Bayer и представляет
    собой элемент управления в виде массива, принимающий 4 значения усиления —
    коэффициенты усиления для каждого из каналов Bayer. Коэффициенты усиления
    всегда указываются в порядке B, Gb, Gr и R, независимо от точного порядка
    Bayer самого датчика.

    Использование массива позволяет расширить этот элемент управления на
    датчики, например, с не-Bayer CFA (colour filter arrays, цветовые
    фильтрующие массивы).

    Единицы измерения значений усиления являются линейными, при этом значение по
    умолчанию соответствует усилению ровно 1.0. Например, если это значение по
    умолчанию указано как (скажем) 128, то значение 192 будет соответствовать
    усилению ровно 1.5.
