***********
V4L2 poll()
***********

Имя
===

v4l2-poll - Ожидание некоторого события на файловом дескрипторе

Синопсис
========

.. code-block:: c

    #include <sys/poll.h>

.. c:function:: int poll( struct pollfd *ufds, unsigned int nfds, int timeout )

Аргументы
=========


Описание
========

С помощью функции :c:func:`poll()` приложения могут приостановить
выполнение до тех пор, пока драйвер не захватит данные или не будет
готов принять данные для вывода.

Если был согласован потоковый ввод-вывод, эта функция ждёт, пока буфер
не будет заполнен устройством захвата и не сможет быть извлечён из
очереди с помощью ioctl :ref:`VIDIOC_DQBUF <VIDIOC_QBUF>`. Для устройств
вывода эта функция ждёт, пока устройство не будет готово принять новый
буфер для постановки в очередь с помощью ioctl :ref:`VIDIOC_QBUF <VIDIOC_QBUF>`
для отображения. Если буферы уже находятся в исходящей очереди драйвера
(захват) или входящая очередь не заполнена (отображение), функция
возвращается немедленно.

В случае успеха :c:func:`poll()` возвращает количество файловых
дескрипторов, которые были выбраны (то есть файловых дескрипторов, для
которых поле ``revents`` соответствующей структуры ``struct pollfd``
ненулевое). Устройства захвата устанавливают флаги ``POLLIN`` и
``POLLRDNORM`` в поле ``revents``, устройства вывода — флаги ``POLLOUT``
и ``POLLWRNORM``. Если функция завершилась по тайм-ауту, она возвращает
значение ноль; при ошибке она возвращает -1, а переменной ``errno``
присваивается соответствующее значение. Если приложение не вызвало
:ref:`VIDIOC_STREAMON <VIDIOC_STREAMON>`, функция :c:func:`poll()`
завершается успешно, но устанавливает флаг ``POLLERR`` в поле
``revents``. Если приложение вызвало
:ref:`VIDIOC_STREAMON <VIDIOC_STREAMON>` для устройства захвата, но ещё
не вызвало :ref:`VIDIOC_QBUF <VIDIOC_QBUF>`, функция
:c:func:`poll()` завершается успешно и устанавливает флаг ``POLLERR`` в
поле ``revents``. Для устройств вывода эта же ситуация приведёт к тому,
что :c:func:`poll()` также завершится успешно, но при этом установит
флаги ``POLLOUT`` и ``POLLWRNORM`` в поле ``revents``.

Если произошло событие (см. :ref:`VIDIOC_DQEVENT`), то в поле
``revents`` будет установлен флаг ``POLLPRI`` и :c:func:`poll()`
вернётся.

Если было согласовано использование функции :c:func:`read()`, а драйвер
ещё не захватывает данные, функция :c:func:`poll()` начинает захват.
Если это не удаётся, она возвращает ``POLLERR``, как описано выше. В
противном случае она ждёт, пока данные не будут захвачены и их можно
будет прочитать. Если драйвер захватывает непрерывно (в отличие,
например, от неподвижных изображений), функция может вернуться
немедленно.

Если было согласовано использование функции :c:func:`write()`, а драйвер
ещё не выполняет потоковую передачу, функция :c:func:`poll()` начинает
потоковую передачу. Если это не удаётся, она возвращает ``POLLERR``, как
описано выше. В противном случае она ждёт, пока драйвер не будет готов к
неблокирующему вызову :c:func:`write()`.

Если вызывающую сторону интересуют только события (в поле ``events``
установлен лишь ``POLLPRI``), то :c:func:`poll()` *не* начнёт потоковую
передачу, если драйвер ещё не выполняет её. Это позволяет опрашивать
только события, а не буферы.

Все драйверы, реализующие функцию :c:func:`read()` или :c:func:`write()`
либо потоковый ввод-вывод, должны также поддерживать функцию
:c:func:`poll()`.

Более подробную информацию см. на странице руководства :c:func:`poll()`.

Возвращаемое значение
=====================

В случае успеха :c:func:`poll()` возвращает количество структур, у
которых поля ``revents`` ненулевые, или ноль, если вызов завершился по
тайм-ауту. При ошибке возвращается -1, а переменной ``errno``
присваивается соответствующее значение:

EBADF
    Один или несколько элементов ``ufds`` указывают недопустимый
    файловый дескриптор.

EBUSY
    Драйвер не поддерживает несколько потоков чтения или записи, а
    устройство уже используется.

EFAULT
    ``ufds`` ссылается на недоступную область памяти.

EINTR
    Вызов был прерван сигналом.

EINVAL
    Значение ``nfds`` превышает значение ``RLIMIT_NOFILE``. Используйте
    ``getrlimit()`` для получения этого значения.
