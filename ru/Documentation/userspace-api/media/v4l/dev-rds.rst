***************
Интерфейс RDS
***************

Radio Data System (система передачи радиоданных) передаёт дополнительную
информацию в двоичном формате, например название станции или информацию о
дорожном движении, на неслышимой звуковой поднесущей радиопрограммы. Данный
интерфейс предназначен для устройств, способных принимать и/или передавать
информацию RDS.

Дополнительные сведения см. в основном стандарте RDS :ref:`iec62106` и
стандарте RBDS :ref:`nrsc4`.

.. note::

   Обратите внимание, что стандарт RBDS, используемый в США, практически
   идентичен стандарту RDS. Любой декодер/кодер RDS также способен работать с
   RBDS. Лишь некоторые поля имеют немного различающиеся значения. Подробнее
   см. в стандарте RBDS.

Стандарт RBDS также определяет поддержку MMBS (Modified Mobile
Search). Это проприетарный формат, который, по-видимому, более не
используется. Интерфейс RDS не поддерживает этот формат. Если потребуется
поддержка MMBS (или так называемых «E-блоков» в целом), обратитесь в список
рассылки linux-media:
`https://linuxtv.org/lists.php <https://linuxtv.org/lists.php>`__.

Опрос возможностей
==================

Устройства, поддерживающие API захвата RDS, устанавливают флаг
``V4L2_CAP_RDS_CAPTURE`` в поле ``capabilities`` структуры
:c:type:`v4l2_capability`, возвращаемой ioctl
:ref:`VIDIOC_QUERYCAP`. Любой тюнер, поддерживающий
RDS, устанавливает флаг ``V4L2_TUNER_CAP_RDS`` в поле
``capability`` структуры :c:type:`v4l2_tuner`. Если
драйвер только передаёт блоки RDS, не интерпретируя данные, должен быть
установлен флаг ``V4L2_TUNER_CAP_RDS_BLOCK_IO``, см.
:ref:`Чтение данных RDS <reading-rds-data>`. Для будущего использования также
определён флаг ``V4L2_TUNER_CAP_RDS_CONTROLS``. Однако драйвер
для радиотюнера с такой возможностью пока не существует, поэтому, если вы
планируете написать подобный драйвер, вам следует обсудить это в
списке рассылки linux-media:
`https://linuxtv.org/lists.php <https://linuxtv.org/lists.php>`__.

Наличие сигнала RDS можно определить, проверив
поле ``rxsubchans`` структуры :c:type:`v4l2_tuner`:
``V4L2_TUNER_SUB_RDS`` будет установлен, если обнаружены данные RDS.

Устройства, поддерживающие API вывода RDS, устанавливают флаг ``V4L2_CAP_RDS_OUTPUT``
в поле ``capabilities`` структуры
:c:type:`v4l2_capability`, возвращаемой ioctl
:ref:`VIDIOC_QUERYCAP`. Любой модулятор,
поддерживающий RDS, устанавливает флаг ``V4L2_TUNER_CAP_RDS`` в поле
``capability`` структуры
:c:type:`v4l2_modulator`. Чтобы включить передачу
RDS, необходимо установить бит ``V4L2_TUNER_SUB_RDS`` в
поле ``txsubchans`` структуры
:c:type:`v4l2_modulator`. Если драйвер только передаёт блоки RDS
без интерпретации данных, должен быть установлен флаг ``V4L2_TUNER_CAP_RDS_BLOCK_IO``.
Если тюнер способен обрабатывать сущности RDS,
такие как коды идентификации программы и радиотекст, должен быть установлен флаг
``V4L2_TUNER_CAP_RDS_CONTROLS``, см.
:ref:`Запись данных RDS <writing-rds-data>` и
:ref:`Справочник по управлению FM-передатчиком <fm-tx-controls>`.

.. _reading-rds-data:

Чтение данных RDS
=================

Данные RDS можно прочитать с радиоустройства с помощью
функции :c:func:`read()`. Данные упакованы в группы по
три байта.

.. _writing-rds-data:

Запись данных RDS
=================

Данные RDS можно записать на радиоустройство с помощью
функции :c:func:`write()`. Данные упакованы в группы по
три байта следующим образом:

Структуры данных RDS
====================

.. c:type:: v4l2_rds_data

.. flat-table:: struct v4l2_rds_data
    :header-rows:  0
    :stub-columns: 0
    :widths:       1 1 5

    * - __u8
      - ``lsb``
      - Младший значащий байт блока RDS
    * - __u8
      - ``msb``
      - Старший значащий байт блока RDS
    * - __u8
      - ``block``
      - Описание блока


.. _v4l2-rds-block:

.. tabularcolumns:: |p{2.9cm}|p{14.6cm}|

.. flat-table:: Описание блока
    :header-rows:  0
    :stub-columns: 0
    :widths:       1 5

    * - Биты 0-2
      - Блок (он же смещение) принятых данных.
    * - Биты 3-5
      - Устаревшее. В настоящее время идентично битам 0-2. Не используйте эти
	биты.
    * - Бит 6
      - Бит исправления. Указывает, что в этом блоке данных была исправлена
	ошибка.
    * - Бит 7
      - Бит ошибки. Указывает, что при приёме этого блока произошла
	неисправимая ошибка.
