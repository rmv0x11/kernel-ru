*******************************************
Справочник по управлению цифровым видео
*******************************************

Класс управления цифровым видео (Digital Video) предназначен для управления
приёмниками и передатчиками для `VGA <http://en.wikipedia.org/wiki/Vga>`__,
`DVI <http://en.wikipedia.org/wiki/Digital_Visual_Interface>`__
(Digital Visual Interface), HDMI (:ref:`hdmi`) и DisplayPort
(:ref:`dp`). Как правило, ожидается, что эти элементы управления являются частными
для приёмного или передающего субустройства, которое их реализует, поэтому они
доступны только на узле устройства ``/dev/v4l-subdev*``.

.. note::

   Обратите внимание, что эти устройства могут иметь несколько входных или выходных
   контактных площадок (pad), которые подключаются, например, к разъёмам HDMI. Даже
   несмотря на то что субустройство будет принимать или передавать видео от/к только
   одной из этих площадок, остальные площадки всё ещё могут быть активны в том, что
   касается обработки EDID (Extended Display Identification Data, :ref:`vesaedid`) и
   HDCP (High-bandwidth Digital Content Protection System, :ref:`hdcp`), позволяя
   устройству заранее выполнять довольно медленную обработку EDID/HDCP. Это
   обеспечивает быстрое переключение между разъёмами.

Эти площадки фигурируют в нескольких элементах управления данного раздела в виде
битовых масок, по одному биту на каждую площадку. Бит 0 соответствует площадке 0,
бит 1 — площадке 1 и т. д. Максимальное значение элемента управления — это набор
допустимых площадок.


.. _dv-control-id:

Идентификаторы управления цифровым видео
========================================

``V4L2_CID_DV_CLASS (class)``
    Дескриптор класса цифрового видео (Digital Video).

``V4L2_CID_DV_TX_HOTPLUG (bitmask)``
    Многие разъёмы имеют контакт горячего подключения (hotplug), на котором высокий
    уровень означает, что от источника доступна информация EDID. Этот элемент
    управления показывает состояние контакта горячего подключения, как его видит
    передатчик. Каждый бит соответствует выходной площадке передатчика. Если выходная
    площадка не имеет связанного с ней контакта горячего подключения, то бит для этой
    площадки будет равен 0. Этот элемент управления только для чтения применим к
    разъёмам DVI-D, HDMI и DisplayPort.

``V4L2_CID_DV_TX_RXSENSE (bitmask)``
    Rx Sense — это обнаружение подтягивающих резисторов (pull-up) на линиях тактового
    сигнала TMDS. Обычно это означает, что приёмник (sink) вышел из режима ожидания
    или вошёл в него (т. е. передатчик может определить, что приёмник готов принимать
    видео). Каждый бит соответствует выходной площадке передатчика. Если выходная
    площадка не имеет связанного с ней Rx Sense, то бит для этой площадки будет равен
    0. Этот элемент управления только для чтения применим к устройствам DVI-D и HDMI.

``V4L2_CID_DV_TX_EDID_PRESENT (bitmask)``
    Когда передатчик видит сигнал горячего подключения от приёмника, он пытается
    прочитать EDID. Если бит установлен, то передатчик прочитал по меньшей мере
    первый блок (= 128 байт). Каждый бит соответствует выходной площадке передатчика.
    Если выходная площадка не поддерживает EDID, то бит для этой площадки будет равен
    0. Этот элемент управления только для чтения применим к разъёмам VGA, DVI-A/D,
    HDMI и DisplayPort.

``V4L2_CID_DV_TX_MODE``
    (enum)

enum v4l2_dv_tx_mode -
    HDMI-передатчики могут передавать в режиме DVI-D (только видео) или в режиме HDMI
    (видео + аудио + вспомогательные данные). Этот элемент управления выбирает, какой
    режим использовать: V4L2_DV_TX_MODE_DVI_D или V4L2_DV_TX_MODE_HDMI.
    Этот элемент управления применим к разъёмам HDMI.

``V4L2_CID_DV_TX_RGB_RANGE``
    (enum)

enum v4l2_dv_rgb_range -
    Выбор диапазона квантования для вывода RGB. V4L2_DV_RANGE_AUTO следует диапазону
    квантования RGB, указанному в стандарте для данного видеоинтерфейса (т. е.
    :ref:`cea861` для HDMI).
    V4L2_DV_RANGE_LIMITED и V4L2_DV_RANGE_FULL переопределяют стандарт ради
    совместимости с приёмниками, которые реализовали стандарт некорректно (к
    сожалению, это довольно распространено для HDMI и DVI-D). Полный диапазон (full
    range) позволяет использовать все возможные значения, тогда как ограниченный
    диапазон (limited range) задаёт диапазон (16 << (N-8)) - (235 << (N-8)), где N —
    число бит на компоненту. Этот элемент управления применим к разъёмам VGA,
    DVI-A/D, HDMI и DisplayPort.

``V4L2_CID_DV_TX_IT_CONTENT_TYPE``
    (enum)

enum v4l2_dv_it_content_type -
    Настраивает тип контента IT (IT Content Type) передаваемого видео. Эта информация
    передаётся через разъёмы HDMI и DisplayPort в составе AVI InfoFrame. Термин
    «IT Content» используется для контента, который происходит из компьютера, в
    отличие от контента из телевещания или из аналогового источника. Перечисление
    enum v4l2_dv_it_content_type определяет возможные типы контента:

.. tabularcolumns:: |p{7.3cm}|p{10.2cm}|

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    * - ``V4L2_DV_IT_CONTENT_TYPE_GRAPHICS``
      - Графический контент. Пиксельные данные должны передаваться без фильтрации и
	без аналоговой реконструкции.
    * - ``V4L2_DV_IT_CONTENT_TYPE_PHOTO``
      - Фотоконтент. Контент получен из цифровых неподвижных изображений.
	Контент должен передаваться с минимальным масштабированием и
	улучшениями изображения.
    * - ``V4L2_DV_IT_CONTENT_TYPE_CINEMA``
      - Кинематографический контент.
    * - ``V4L2_DV_IT_CONTENT_TYPE_GAME``
      - Игровой контент. Задержка (latency) аудио и видео должна быть минимизирована.
    * - ``V4L2_DV_IT_CONTENT_TYPE_NO_ITC``
      - Информация о типе контента IT недоступна, и бит ITC в AVI
	InfoFrame установлен в 0.



``V4L2_CID_DV_RX_POWER_PRESENT (bitmask)``
    Определяет, получает ли приёмник питание от источника (например, HDMI несёт 5 В
    на одном из контактов). Это часто используется для питания микросхемы eeprom,
    которая содержит информацию EDID, так чтобы источник мог прочитать EDID, даже
    если приёмник (sink) находится в режиме ожидания или выключен. Каждый бит
    соответствует входной площадке приёмника. Если входная площадка не может
    определить, присутствует ли питание, то бит для этой площадки будет равен 0. Этот
    элемент управления только для чтения применим к разъёмам DVI-D, HDMI и
    DisplayPort.

``V4L2_CID_DV_RX_RGB_RANGE``
    (enum)

enum v4l2_dv_rgb_range -
    Выбор диапазона квантования для ввода RGB. V4L2_DV_RANGE_AUTO следует диапазону
    квантования RGB, указанному в стандарте для данного видеоинтерфейса (т. е.
    :ref:`cea861` для HDMI).
    V4L2_DV_RANGE_LIMITED и V4L2_DV_RANGE_FULL переопределяют стандарт ради
    совместимости с источниками, которые реализовали стандарт некорректно (к
    сожалению, это довольно распространено для HDMI и DVI-D). Полный диапазон (full
    range) позволяет использовать все возможные значения, тогда как ограниченный
    диапазон (limited range) задаёт диапазон (16 << (N-8)) - (235 << (N-8)), где N —
    число бит на компоненту. Этот элемент управления применим к разъёмам VGA,
    DVI-A/D, HDMI и DisplayPort.

``V4L2_CID_DV_RX_IT_CONTENT_TYPE``
    (enum)

enum v4l2_dv_it_content_type -
    Читает тип контента IT (IT Content Type) принимаемого видео. Эта информация
    передаётся через разъёмы HDMI и DisplayPort в составе AVI InfoFrame. Термин
    «IT Content» используется для контента, который происходит из компьютера, в
    отличие от контента из телевещания или из аналогового источника. Доступные типы
    контента см. в ``V4L2_CID_DV_TX_IT_CONTENT_TYPE``.
