************
V4L2 write()
************

Имя
===

v4l2-write - Запись в устройство V4L2

Синопсис
========

.. code-block:: c

    #include <unistd.h>

.. c:function:: ssize_t write( int fd, void *buf, size_t count )

Аргументы
=========

``fd``
    Файловый дескриптор, возвращённый :c:func:`open()`.

``buf``
     Буфер с данными для записи

``count``
    Количество байт в буфере

Описание
========

:c:func:`write()` записывает до ``count`` байт в устройство,
на которое ссылается файловый дескриптор ``fd``, из буфера, начинающегося с
``buf``. Если аппаратные выходы ещё не активны, эта функция
включает их. Если ``count`` равно нулю, :c:func:`write()` возвращает 0
без какого-либо иного эффекта.

Если приложение не предоставляет новые данные вовремя, предыдущий
видеокадр, необработанное изображение VBI, нарезанные данные VPS или WSS отображаются снова.
Нарезанные данные телетекста или Closed Caption не повторяются, вместо них драйвер
вставляет пустую строку.

Возвращаемое значение
=====================

В случае успеха возвращается количество записанных байт. Ноль указывает на то,
что ничего не было записано. В случае ошибки возвращается -1, и переменной ``errno``
присваивается соответствующее значение. В этом случае следующая запись начнётся с
начала нового кадра. Возможные коды ошибок:

EAGAIN
    С помощью флага :ref:`O_NONBLOCK <func-open>` был выбран
    неблокирующий ввод-вывод, и не было доступного пространства в буфере
    для немедленной записи данных.

EBADF
    ``fd`` не является допустимым файловым дескриптором или не открыт для записи.

EBUSY
    Драйвер не поддерживает несколько потоков записи, и устройство
    уже используется.

EFAULT
    ``buf`` ссылается на недоступную область памяти.

EINTR
    Вызов был прерван сигналом до того, как были записаны какие-либо данные.

EIO
    Ошибка ввода-вывода. Это указывает на некоторую аппаратную проблему.

EINVAL
    Функция :c:func:`write()` не поддерживается этим драйвером,
    не на этом устройстве или в целом не на устройствах этого типа.
