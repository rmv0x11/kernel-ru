***********
V4L2 open()
***********

Имя
===

v4l2-open - Открытие устройства V4L2

Синопсис
========

.. code-block:: c

    #include <fcntl.h>

.. c:function:: int open( const char *device_name, int flags )

Аргументы
=========

``device_name``
    Устройство, которое нужно открыть.

``flags``
    Флаги открытия. Режим доступа должен быть ``O_RDWR``. Это всего лишь
    формальность: устройства ввода по-прежнему поддерживают только чтение, а
    устройства вывода — только запись.

    Когда задан флаг ``O_NONBLOCK``, функция :c:func:`read()`
    и ioctl :ref:`VIDIOC_DQBUF <VIDIOC_QBUF>` будут
    возвращать код ошибки ``EAGAIN``, если данных нет или в исходящей
    очереди драйвера нет ни одного буфера; в противном случае эти функции
    блокируются до тех пор, пока данные не станут доступны. Все драйверы V4L2,
    обменивающиеся данными с приложениями, должны поддерживать флаг ``O_NONBLOCK``.

    Остальные флаги не оказывают никакого эффекта.

Описание
========

Чтобы открыть устройство V4L2, приложения вызывают :c:func:`open()` с
желаемым именем устройства. Эта функция не имеет побочных эффектов; все параметры
формата данных, текущий ввод или вывод, значения элементов управления и другие свойства
остаются неизменными. При первом вызове :c:func:`open()` после загрузки
драйвера они будут сброшены к значениям по умолчанию; драйверы никогда не находятся в
неопределённом состоянии.

Возвращаемое значение
=====================

В случае успеха :c:func:`open()` возвращает новый файловый дескриптор. В случае ошибки
возвращается -1, а переменной ``errno`` присваивается соответствующее значение.
Возможные коды ошибок:

EACCES
    У вызывающей стороны нет прав доступа к устройству.

EBUSY
    Драйвер не поддерживает множественные открытия, а устройство уже
    используется.

ENODEV
    Устройство не найдено или было удалено.

ENOMEM
    Недостаточно памяти ядра для выполнения запроса.

EMFILE
    У процесса уже открыто максимальное количество файлов.

ENFILE
    Достигнут предел общего числа файлов, открытых в системе.
