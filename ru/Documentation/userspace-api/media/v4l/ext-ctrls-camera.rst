.. _camera-controls:

**************************************
Справочник элементов управления камеры
**************************************

Класс Camera включает элементы управления механическими (или
эквивалентными цифровыми) функциями устройства, такими как управляемые
объективы или датчики.


.. _camera-control-id:

Идентификаторы элементов управления камеры
==========================================

``V4L2_CID_CAMERA_CLASS (class)``
    Дескриптор класса Camera. Вызов
    :ref:`VIDIOC_QUERYCTRL` для этого элемента управления вернёт
    описание данного класса элементов управления.

.. _v4l2-exposure-auto-type:

``V4L2_CID_EXPOSURE_AUTO``
    (enum)

enum v4l2_exposure_auto_type -
    Включает автоматическую регулировку времени экспозиции и/или
    отверстия диафрагмы (ирисовой диафрагмы). Эффект от ручного изменения
    времени экспозиции или отверстия диафрагмы при включённых этих
    функциях не определён, драйверы должны игнорировать такие запросы.
    Возможные значения:


.. tabularcolumns:: |p{7.1cm}|p{10.4cm}|

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    * - ``V4L2_EXPOSURE_AUTO``
      - Автоматическое время экспозиции, автоматическое отверстие диафрагмы.
    * - ``V4L2_EXPOSURE_MANUAL``
      - Ручное время экспозиции, ручная диафрагма.
    * - ``V4L2_EXPOSURE_SHUTTER_PRIORITY``
      - Ручное время экспозиции, автоматическая диафрагма.
    * - ``V4L2_EXPOSURE_APERTURE_PRIORITY``
      - Автоматическое время экспозиции, ручная диафрагма.



``V4L2_CID_EXPOSURE_ABSOLUTE (integer)``
    Определяет время экспозиции датчика камеры. Время экспозиции
    ограничено интервалом между кадрами. Драйверы должны интерпретировать
    значения как единицы по 100 мкс, где значение 1 означает 1/10000 долю
    секунды, 10000 — 1 секунду, а 100000 — 10 секунд.

``V4L2_CID_EXPOSURE_AUTO_PRIORITY (boolean)``
    Когда ``V4L2_CID_EXPOSURE_AUTO`` установлен в ``AUTO`` или
    ``APERTURE_PRIORITY``, этот элемент управления определяет, может ли
    устройство динамически изменять частоту кадров. По умолчанию эта
    функция отключена (0), и частота кадров должна оставаться постоянной.

``V4L2_CID_AUTO_EXPOSURE_BIAS (integer menu)``
    Определяет автоматическую компенсацию экспозиции; действует только
    тогда, когда элемент управления ``V4L2_CID_EXPOSURE_AUTO`` установлен
    в ``AUTO``, ``SHUTTER_PRIORITY`` или ``APERTURE_PRIORITY``. Выражается
    в единицах EV, драйверы должны интерпретировать значения как единицы
    по 0,001 EV, где значение 1000 означает +1 EV.

    Увеличение значения компенсации экспозиции эквивалентно уменьшению
    значения экспозиции (EV) и увеличивает количество света на датчике
    изображения. Камера выполняет компенсацию экспозиции путём
    регулировки абсолютного времени экспозиции и/или диафрагмы.

.. _v4l2-exposure-metering:

``V4L2_CID_EXPOSURE_METERING``
    (enum)

enum v4l2_exposure_metering -
    Определяет, как камера измеряет количество света, доступного для
    экспонирования кадра. Возможные значения:

.. tabularcolumns:: |p{8.7cm}|p{8.7cm}|

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    * - ``V4L2_EXPOSURE_METERING_AVERAGE``
      - Использует информацию о свете, поступающую со всего кадра, и
	усредняет её, не придавая веса какой-либо отдельной части измеряемой
	области.
    * - ``V4L2_EXPOSURE_METERING_CENTER_WEIGHTED``
      - Усредняет информацию о свете, поступающую со всего кадра, отдавая
	приоритет центру измеряемой области.
    * - ``V4L2_EXPOSURE_METERING_SPOT``
      - Измеряет только очень небольшую область в центре кадра.
    * - ``V4L2_EXPOSURE_METERING_MATRIX``
      - Многозонное измерение. Интенсивность света измеряется в нескольких
	точках кадра, и результаты комбинируются. Алгоритм выбора зон и их
	значимость при вычислении итогового значения зависят от устройства.



``V4L2_CID_PAN_RELATIVE (integer)``
    Этот элемент управления поворачивает камеру по горизонтали на
    указанную величину. Единица измерения не определена. Положительное
    значение перемещает камеру вправо (по часовой стрелке при взгляде
    сверху), отрицательное — влево. Нулевое значение не вызывает движения.
    Это элемент управления только для записи.

``V4L2_CID_TILT_RELATIVE (integer)``
    Этот элемент управления поворачивает камеру по вертикали на указанную
    величину. Единица измерения не определена. Положительное значение
    перемещает камеру вверх, отрицательное — вниз. Нулевое значение не
    вызывает движения. Это элемент управления только для записи.

``V4L2_CID_PAN_RESET (button)``
    При установке этого элемента управления камера перемещается по
    горизонтали в положение по умолчанию.

``V4L2_CID_TILT_RESET (button)``
    При установке этого элемента управления камера перемещается по
    вертикали в положение по умолчанию.

``V4L2_CID_PAN_ABSOLUTE (integer)``
    Этот элемент управления поворачивает камеру по горизонтали в указанное
    положение. Положительные значения перемещают камеру вправо (по часовой
    стрелке при взгляде сверху), отрицательные — влево. Драйверы должны
    интерпретировать значения как угловые секунды, при этом допустимые
    значения находятся в диапазоне от -180 * 3600 до +180 * 3600
    включительно.

``V4L2_CID_TILT_ABSOLUTE (integer)``
    Этот элемент управления поворачивает камеру по вертикали в указанное
    положение. Положительные значения перемещают камеру вверх,
    отрицательные — вниз. Драйверы должны интерпретировать значения как
    угловые секунды, при этом допустимые значения находятся в диапазоне
    от -180 * 3600 до +180 * 3600 включительно.

``V4L2_CID_FOCUS_ABSOLUTE (integer)``
    Этот элемент управления устанавливает точку фокусировки камеры в
    указанное положение. Единица измерения не определена. Положительные
    значения устанавливают фокус ближе к камере, отрицательные — в сторону
    бесконечности.

``V4L2_CID_FOCUS_RELATIVE (integer)``
    Этот элемент управления перемещает точку фокусировки камеры на
    указанную величину. Единица измерения не определена. Положительные
    значения перемещают фокус ближе к камере, отрицательные — в сторону
    бесконечности. Это элемент управления только для записи.

``V4L2_CID_FOCUS_AUTO (boolean)``
    Включает непрерывную автоматическую регулировку фокуса. Эффект от
    ручной регулировки фокуса при включённой этой функции не определён,
    драйверы должны игнорировать такие запросы.

``V4L2_CID_AUTO_FOCUS_START (button)``
    Запускает однократный процесс автофокусировки. Эффект от установки
    этого элемента управления, когда ``V4L2_CID_FOCUS_AUTO`` установлен в
    ``TRUE`` (1), не определён, драйверы должны игнорировать такие запросы.

``V4L2_CID_AUTO_FOCUS_STOP (button)``
    Прерывает автоматическую фокусировку, запущенную элементом управления
    ``V4L2_CID_AUTO_FOCUS_START``. Действует только тогда, когда
    непрерывная автофокусировка отключена, то есть когда элемент
    управления ``V4L2_CID_FOCUS_AUTO`` установлен в ``FALSE`` (0).

.. _v4l2-auto-focus-status:

``V4L2_CID_AUTO_FOCUS_STATUS (bitmask)``
    Статус автоматической фокусировки. Это элемент управления только для
    чтения.

    Установка бита блокировки ``V4L2_LOCK_FOCUS`` элемента управления
    ``V4L2_CID_3A_LOCK`` может остановить обновление значения элемента
    управления ``V4L2_CID_AUTO_FOCUS_STATUS``.

.. tabularcolumns:: |p{6.8cm}|p{10.7cm}|

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    * - ``V4L2_AUTO_FOCUS_STATUS_IDLE``
      - Автоматическая фокусировка не активна.
    * - ``V4L2_AUTO_FOCUS_STATUS_BUSY``
      - Автоматическая фокусировка выполняется.
    * - ``V4L2_AUTO_FOCUS_STATUS_REACHED``
      - Фокус достигнут.
    * - ``V4L2_AUTO_FOCUS_STATUS_FAILED``
      - Автоматическая фокусировка не удалась, драйвер не выйдет из этого
	состояния, пока приложение не выполнит другое действие.



.. _v4l2-auto-focus-range:

``V4L2_CID_AUTO_FOCUS_RANGE``
    (enum)

enum v4l2_auto_focus_range -
    Определяет диапазон расстояний автоматической фокусировки, в пределах
    которого может регулироваться объектив.

.. tabularcolumns:: |p{6.9cm}|p{10.6cm}|

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    * - ``V4L2_AUTO_FOCUS_RANGE_AUTO``
      - Камера автоматически выбирает диапазон фокусировки.
    * - ``V4L2_AUTO_FOCUS_RANGE_NORMAL``
      - Нормальный диапазон расстояний, ограниченный для наилучшей
	производительности автоматической фокусировки.
    * - ``V4L2_AUTO_FOCUS_RANGE_MACRO``
      - Макро (крупный план) автофокусировка. Камера будет использовать
	минимально возможное расстояние для автофокусировки.
    * - ``V4L2_AUTO_FOCUS_RANGE_INFINITY``
      - Объектив настроен на фокусировку на объекте, расположенном на
	бесконечном расстоянии.



``V4L2_CID_ZOOM_ABSOLUTE (integer)``
    Задаёт фокусное расстояние объектива как абсолютное значение. Единица
    масштабирования зависит от драйвера, и её значение должно быть
    положительным целым числом.

``V4L2_CID_ZOOM_RELATIVE (integer)``
    Задаёт фокусное расстояние объектива относительно текущего значения.
    Положительные значения перемещают группу линз трансфокатора в сторону
    телефото, отрицательные — в сторону широкого угла. Единица
    масштабирования зависит от драйвера. Это элемент управления только для
    записи.

``V4L2_CID_ZOOM_CONTINUOUS (integer)``
    Перемещает группу линз объектива с указанной скоростью, пока она не
    достигнет физических пределов устройства или пока не поступит явный
    запрос на остановку движения. Положительное значение перемещает группу
    линз трансфокатора в сторону телефото. Нулевое значение останавливает
    движение группы линз трансфокатора. Отрицательное значение перемещает
    группу линз трансфокатора в сторону широкого угла. Единица скорости
    масштабирования зависит от драйвера.

``V4L2_CID_IRIS_ABSOLUTE (integer)``
    Этот элемент управления устанавливает диафрагму камеры в указанное
    значение. Единица измерения не определена. Большие значения шире
    открывают диафрагму, меньшие — закрывают её.

``V4L2_CID_IRIS_RELATIVE (integer)``
    Этот элемент управления изменяет диафрагму камеры на указанную
    величину. Единица измерения не определена. Положительные значения
    открывают диафрагму на один шаг шире, отрицательные — закрывают её на
    один шаг. Это элемент управления только для записи.

``V4L2_CID_PRIVACY (boolean)``
    Предотвращает захват видео камерой. Когда этот элемент управления
    установлен в ``TRUE`` (1), камера не может захватывать изображение.
    Распространёнными способами обеспечения приватности являются
    механическое закрытие датчика и обработка изображения прошивкой
    (firmware), но устройство не ограничено этими методами. Устройства,
    реализующие элемент управления приватностью, должны поддерживать
    доступ на чтение и могут поддерживать доступ на запись.

``V4L2_CID_BAND_STOP_FILTER (integer)``
    Включает или выключает полосозаграждающий фильтр датчика камеры либо
    задаёт его силу. Такие полосозаграждающие фильтры могут
    использоваться, например, для отфильтровывания составляющей
    флуоресцентного освещения.

.. _v4l2-auto-n-preset-white-balance:

``V4L2_CID_AUTO_N_PRESET_WHITE_BALANCE``
    (enum)

enum v4l2_auto_n_preset_white_balance -
    Устанавливает баланс белого в автоматический, ручной режим или
    предустановку. Предустановки определяют цветовую температуру света как
    подсказку камере для регулировки баланса белого, обеспечивающей
    наиболее точную передачу цвета. Следующие предустановки баланса белого
    перечислены в порядке возрастания цветовой температуры.

.. tabularcolumns:: |p{7.4cm}|p{10.1cm}|

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    * - ``V4L2_WHITE_BALANCE_MANUAL``
      - Ручной баланс белого.
    * - ``V4L2_WHITE_BALANCE_AUTO``
      - Автоматическая регулировка баланса белого.
    * - ``V4L2_WHITE_BALANCE_INCANDESCENT``
      - Настройка баланса белого для освещения лампами накаливания
	(вольфрамовыми). Она обычно охлаждает цвета и соответствует
	приблизительно диапазону цветовой температуры 2500...3500 K.
    * - ``V4L2_WHITE_BALANCE_FLUORESCENT``
      - Предустановка баланса белого для флуоресцентного освещения. Она
	соответствует приблизительно цветовой температуре 4000...5000 K.
    * - ``V4L2_WHITE_BALANCE_FLUORESCENT_H``
      - При этой настройке камера будет компенсировать флуоресцентное
	освещение типа H.
    * - ``V4L2_WHITE_BALANCE_HORIZON``
      - Настройка баланса белого для дневного света у горизонта. Она
	соответствует приблизительно цветовой температуре 5000 K.
    * - ``V4L2_WHITE_BALANCE_DAYLIGHT``
      - Предустановка баланса белого для дневного света (при ясном небе).
	Она соответствует приблизительно цветовой температуре 5000...6500 K.
    * - ``V4L2_WHITE_BALANCE_FLASH``
      - При этой настройке камера будет компенсировать свет вспышки. Она
	слегка прогревает цвета и приблизительно соответствует цветовой
	температуре 5000...5500 K.
    * - ``V4L2_WHITE_BALANCE_CLOUDY``
      - Предустановка баланса белого для умеренно облачного неба. Этот
	вариант соответствует приблизительно диапазону цветовой температуры
	6500...8000 K.
    * - ``V4L2_WHITE_BALANCE_SHADE``
      - Предустановка баланса белого для тени или сильно облачного неба.
	Она соответствует приблизительно цветовой температуре 9000...10000 K.



.. _v4l2-wide-dynamic-range:

``V4L2_CID_WIDE_DYNAMIC_RANGE (boolean)``
    Включает или выключает функцию широкого динамического диапазона
    камеры. Эта функция позволяет получать чёткие изображения в
    ситуациях, когда интенсивность освещения значительно меняется по всей
    сцене, т. е. одновременно присутствуют очень тёмные и очень яркие
    области. Чаще всего в камерах она реализуется путём комбинирования
    двух последовательных кадров с разным временем экспозиции.  [#f1]_

.. _v4l2-image-stabilization:

``V4L2_CID_IMAGE_STABILIZATION (boolean)``
    Включает или выключает стабилизацию изображения.

``V4L2_CID_ISO_SENSITIVITY (integer menu)``
    Определяет эквивалент ISO датчика изображения, указывающий
    чувствительность датчика к свету. Числа выражаются в арифметической
    шкале согласно стандарту :ref:`iso12232`, где удвоение
    чувствительности датчика представляется удвоением числового значения
    ISO. Приложения должны интерпретировать значения как стандартные
    значения ISO, умноженные на 1000, например значение элемента
    управления 800 означает ISO 0.8. Драйверы обычно поддерживают только
    подмножество стандартных значений ISO. Эффект от установки этого
    элемента управления, когда элемент управления
    ``V4L2_CID_ISO_SENSITIVITY_AUTO`` установлен в значение, отличное от
    ``V4L2_CID_ISO_SENSITIVITY_MANUAL``, не определён, драйверы должны
    игнорировать такие запросы.

    (enum)

enum v4l2_iso_sensitivity_type -
    Включает или выключает автоматическую регулировку чувствительности ISO.



.. flat-table::

    :stub-columns: 0

    * - ``V4L2_CID_ISO_SENSITIVITY_MANUAL``
      - Ручная чувствительность ISO.
    * - ``V4L2_CID_ISO_SENSITIVITY_AUTO``
      - Автоматическая регулировка чувствительности ISO.



.. _v4l2-scene-mode:

``V4L2_CID_SCENE_MODE``
    (enum)

enum v4l2_scene_mode -
    Этот элемент управления позволяет выбирать программы сцен как
    автоматические режимы камеры, оптимизированные под распространённые
    сюжеты съёмки. В этих режимах камера определяет наилучшую экспозицию,
    диафрагму, фокусировку, измерение света, баланс белого и эквивалентную
    чувствительность. Элементы управления этими параметрами зависят от
    элемента управления режимом сцены. Точное поведение в каждом режиме
    определяется спецификацией камеры.

    Когда функция режима сцены не используется, этот элемент управления
    следует установить в ``V4L2_SCENE_MODE_NONE``, чтобы гарантировать
    доступность других, возможно связанных, элементов управления.
    Определены следующие программы сцен:

    * - ``V4L2_SCENE_MODE_NONE``
      - Функция режима сцены отключена.
    * - ``V4L2_SCENE_MODE_BACKLIGHT``
      - Контровой свет. Компенсирует тёмные тени, когда свет идёт из-за
	объекта съёмки, в том числе путём автоматического включения вспышки.
    * - ``V4L2_SCENE_MODE_BEACH_SNOW``
      - Пляж и снег. Этот режим компенсирует полностью белые или яркие
	сцены, которые обычно выглядят серыми и малоконтрастными, когда
	автоматическая экспозиция камеры основана на средней яркости сцены.
	Для компенсации этот режим автоматически слегка переэкспонирует
	кадры. Баланс белого также может корректироваться для компенсации
	того, что отражённый снег выглядит голубоватым, а не белым.
    * - ``V4L2_SCENE_MODE_CANDLELIGHT``
      - Свет свечей. Камера обычно повышает чувствительность ISO и снижает
	скорость затвора. Этот режим компенсирует относительно близкий
	объект в сцене. Вспышка отключается, чтобы сохранить атмосферу
	освещения.
    * - ``V4L2_SCENE_MODE_DAWN_DUSK``
      - Рассвет и сумерки. Сохраняет цвета, видимые при слабом
	естественном свете перед сумерками и после рассвета. Камера может
	отключить вспышку и автоматически сфокусироваться на бесконечности.
	Обычно она усиливает насыщенность и снижает скорость затвора.
    * - ``V4L2_SCENE_MODE_FALL_COLORS``
      - Осенние цвета. Увеличивает насыщенность и корректирует баланс
	белого для усиления цвета. На снимках осенней листвы получаются
	насыщенные красные и жёлтые тона.
    * - ``V4L2_SCENE_MODE_FIREWORKS``
      - Фейерверк. Используется длительное время экспозиции для захвата
	расширяющейся вспышки света от фейерверка. Камера может задействовать
	стабилизацию изображения.
    * - ``V4L2_SCENE_MODE_LANDSCAPE``
      - Пейзаж. Камера может выбрать малое отверстие диафрагмы для
	обеспечения большой глубины резкости и длительное время экспозиции,
	чтобы помочь захватить детали в условиях слабого освещения. Фокус
	фиксируется на бесконечности. Подходит для удалённых и широких видов.
    * - ``V4L2_SCENE_MODE_NIGHT``
      - Ночь, также известна как Ночной пейзаж. Предназначен для условий
	слабого освещения, сохраняет детали в тёмных областях, не
	засвечивая яркие объекты. Камера обычно устанавливает
	средне-высокую чувствительность ISO, относительно длительное время
	экспозиции и отключает вспышку. В связи с этим возрастает шум
	изображения и появляется вероятность смазанного изображения.
    * - ``V4L2_SCENE_MODE_PARTY_INDOOR``
      - Вечеринка и помещение. Предназначен для съёмки сцен в помещении,
	освещённых как фоновым внутренним освещением, так и вспышкой. Камера
	обычно повышает чувствительность ISO и корректирует экспозицию под
	условия слабого освещения.
    * - ``V4L2_SCENE_MODE_PORTRAIT``
      - Портрет. Камера корректирует диафрагму так, чтобы глубина резкости
	была уменьшена, что помогает выделить объект на фоне сглаженного
	заднего плана. Большинство камер распознают наличие лиц в сцене и
	фокусируются на них. Цветовой тон корректируется для усиления
	телесных оттенков. Интенсивность вспышки часто уменьшается.
    * - ``V4L2_SCENE_MODE_SPORTS``
      - Спорт. Значительно повышает ISO и использует высокую скорость
	затвора для замораживания движения быстро движущихся объектов. В
	этом режиме может наблюдаться повышенный шум изображения.
    * - ``V4L2_SCENE_MODE_SUNSET``
      - Закат. Сохраняет глубокие оттенки, видимые при закатах и восходах.
	Усиливает насыщенность.
    * - ``V4L2_SCENE_MODE_TEXT``
      - Текст. Применяет дополнительный контраст и резкость; обычно это
	чёрно-белый режим, оптимизированный для удобочитаемости.
	Автоматическая фокусировка может быть переключена в режим крупного
	плана, и эта настройка может также включать некоторую коррекцию
	искажений объектива.

``V4L2_CID_PAN_SPEED (integer)``
    Этот элемент управления поворачивает камеру по горизонтали с указанной
    скоростью. Единица измерения не определена. Положительное значение
    перемещает камеру вправо (по часовой стрелке при взгляде сверху),
    отрицательное — влево. Нулевое значение останавливает движение, если
    оно выполняется, и не оказывает иного эффекта в противном случае.

``V4L2_CID_TILT_SPEED (integer)``
    Этот элемент управления поворачивает камеру по вертикали с указанной
    скоростью. Единица измерения не определена. Положительное значение
    перемещает камеру вверх, отрицательное — вниз. Нулевое значение
    останавливает движение, если оно выполняется, и не оказывает иного
    эффекта в противном случае.

.. _v4l2-camera-sensor-orientation:

``V4L2_CID_CAMERA_ORIENTATION (menu)``
    Этот элемент управления только для чтения описывает ориентацию камеры,
    сообщая её положение монтажа на устройстве, где установлена камера.
    Значение элемента управления постоянно и не изменяется программно.
    Этот элемент управления особенно значим для устройств с чётко
    определённой ориентацией, таких как телефоны, ноутбуки и портативные
    устройства, поскольку он выражается как положение относительно
    предполагаемой ориентации использования устройства. Например, о
    камере, установленной на обращённой к пользователю стороне телефона,
    планшета или ноутбука, говорят, что она имеет ориентацию
    ``V4L2_CAMERA_ORIENTATION_FRONT``, а о камере, установленной на
    противоположной фронтальной стороне, говорят, что она имеет ориентацию
    ``V4L2_CAMERA_ORIENTATION_BACK``. О датчиках камеры, не прикреплённых
    непосредственно к устройству или прикреплённых так, что они могут
    свободно перемещаться, таких как веб-камеры и цифровые фотоаппараты,
    говорят, что они имеют ориентацию ``V4L2_CAMERA_ORIENTATION_EXTERNAL``.


.. tabularcolumns:: |p{7.7cm}|p{9.8cm}|

.. flat-table::
    :header-rows:  0
    :stub-columns: 0

    * - ``V4L2_CAMERA_ORIENTATION_FRONT``
      - Камера ориентирована в сторону обращённой к пользователю стороны устройства.
    * - ``V4L2_CAMERA_ORIENTATION_BACK``
      - Камера ориентирована в сторону задней стороны устройства.
    * - ``V4L2_CAMERA_ORIENTATION_EXTERNAL``
      - Камера не прикреплена непосредственно к устройству и свободно перемещается.

``V4L2_CID_CAMERA_SENSOR_ROTATION (integer)``
    Этот элемент управления только для чтения описывает коррекцию поворота
    в градусах против часовой стрелки, которую необходимо применить к
    захваченным изображениям после их захвата в память для компенсации
    поворота при монтаже датчика камеры.

    Точное определение поворота при монтаже датчика см. в подробном
    описании свойств 'rotation' в файле привязок (binding) устройств
    девицетри 'video-interfaces.txt'.

    Ниже приведено несколько примеров, в которых в качестве примера
    захватываемой сцены используется акула, плывущая слева направо перед
    пользователем. ::

                 0               X-axis
               0 +------------------------------------->
                 !
                 !
                 !
                 !           |\____)\___
                 !           ) _____  __`<
                 !           |/     )/
                 !
                 !
                 !
                 V
               Y-axis

    Пример первый — Веб-камера

    Предполагая, что вы можете взять с собой ноутбук во время плавания с
    акулами, модуль камеры ноутбука установлен на обращённой к
    пользователю части корпуса экрана ноутбука и обычно используется для
    видеозвонков. Захваченные изображения предназначены для отображения в
    ландшафтном режиме (ширина > высоты) на экране ноутбука.

    Камера обычно монтируется вверх ногами для компенсации эффекта
    оптической инверсии объектива. В этом случае значение элемента
    управления V4L2_CID_CAMERA_SENSOR_ROTATION равно 0, поворот для
    корректного отображения изображений пользователю не требуется.

    Если датчик камеры смонтирован не вверх ногами, требуется
    скомпенсировать эффект оптической инверсии объектива, и значение
    элемента управления V4L2_CID_CAMERA_SENSOR_ROTATION составляет 180
    градусов, так как изображения при захвате в память окажутся
    повёрнутыми. ::

                 +--------------------------------------+
                 !                                      !
                 !                                      !
                 !                                      !
                 !              __/(_____/|             !
                 !            >.___  ____ (             !
                 !                 \(    \|             !
                 !                                      !
                 !                                      !
                 !                                      !
                 +--------------------------------------+

    Для корректного отображения изображения на экране пользователя
    необходимо применить программную коррекцию поворота на 180 градусов. ::

                 +--------------------------------------+
                 !                                      !
                 !                                      !
                 !                                      !
                 !             |\____)\___              !
                 !             ) _____  __`<            !
                 !             |/     )/                !
                 !                                      !
                 !                                      !
                 !                                      !
                 +--------------------------------------+

    Пример второй — Камера телефона

    Удобнее отправиться плавать с акулами, имея при себе только мобильный
    телефон, и снимать камерой, которая установлена на задней стороне
    устройства, обращённой от пользователя. Захваченные изображения
    предназначены для отображения в портретном режиме (высота > ширины),
    чтобы соответствовать ориентации экрана устройства и ориентации
    использования устройства, применявшейся при съёмке.

    Датчик камеры обычно монтируется так, что более длинная сторона его
    массива пикселей выровнена с более длинной стороной устройства,
    смонтирован вверх ногами для компенсации эффекта оптической инверсии
    объектива.

    Изображения после захвата в память будут повёрнуты, и значение
    V4L2_CID_CAMERA_SENSOR_ROTATION сообщит о повороте на 90 градусов. ::


                 +-------------------------------------+
                 |                 _ _                 |
                 |                \   /                |
                 |                 | |                 |
                 |                 | |                 |
                 |                 |  >                |
                 |                <  |                 |
                 |                 | |                 |
                 |                   .                 |
                 |                  V                  |
                 +-------------------------------------+

    Для корректного отображения изображения в портретном режиме на экране
    устройства необходимо применить коррекцию на 90 градусов против
    часовой стрелки. ::

                          +--------------------+
                          |                    |
                          |                    |
                          |                    |
                          |                    |
                          |                    |
                          |                    |
                          |   |\____)\___      |
                          |   ) _____  __`<    |
                          |   |/     )/        |
                          |                    |
                          |                    |
                          |                    |
                          |                    |
                          |                    |
                          +--------------------+

``V4L2_CID_HDR_SENSOR_MODE (menu)``
    Изменяет режим HDR датчика. HDR-изображение получается путём слияния
    двух захватов одной и той же сцены с использованием двух разных
    периодов экспозиции. Режим HDR описывает способ слияния этих двух
    захватов в датчике.

    Поскольку режимы различаются для каждого датчика, пункты меню не
    стандартизированы этим элементом управления и оставлены на усмотрение
    программиста.
