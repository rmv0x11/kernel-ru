=================================================
Расшифровка магического числа IOCTL
=================================================

Чтобы расшифровать шестнадцатеричный код IOCTL:

Большинство архитектур использует этот общий формат, но проверяйте
include/ARCH/ioctl.h на предмет особенностей; например, powerpc
использует 3 бита для кодирования чтения/записи и 13 бит для размера.

 ====== ==================================
 биты   значение
 ====== ==================================
 31-30	00 - нет параметров: использует макрос _IO
	10 - чтение: _IOR
	01 - запись: _IOW
	11 - чтение/запись: _IOWR

 29-16	размер аргументов

 15-8	ascii-символ, предположительно
	уникальный для каждого драйвера

 7-0	номер функции #
 ====== ==================================


Так, например, 0x82187201 — это чтение с длиной аргумента 0x218,
символ 'r', функция 1. Поиск по исходному коду показывает, что это::

	#define VFAT_IOCTL_READDIR_BOTH         _IOR('r', 1, struct dirent [2])
