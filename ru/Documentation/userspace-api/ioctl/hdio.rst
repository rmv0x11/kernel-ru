=================================
Сводка вызовов ioctl `HDIO_`
=================================

- Edward A. Falk <efalk@google.com>

Ноябрь 2004 г.

Этот документ призван описать вызовы ioctl(2), поддерживаемые
слоем HD/IDE.  По большей части они реализованы (по состоянию на Linux 5.11)
в drivers/ata/libata-scsi.c.

Значения ioctl перечислены в <linux/hdreg.h>.  На момент написания
они таковы:

    ioctl, передающие указатели на аргументы в пространство пользователя:

	=======================	=======================================
	HDIO_GETGEO		получить геометрию устройства
	HDIO_GET_32BIT		получить текущую настройку io_32bit
	HDIO_GET_IDENTITY	получить идентификационную информацию IDE
	HDIO_DRIVE_TASKFILE	выполнить «сырой» taskfile
	HDIO_DRIVE_TASK		выполнить task и специальную команду накопителя
	HDIO_DRIVE_CMD		выполнить специальную команду накопителя
	=======================	=======================================

    ioctl, передающие значения, не являющиеся указателями:

	=======================	=======================================
	HDIO_SET_32BIT		изменить флаги io_32bit
	=======================	=======================================


Приведённая ниже информация была получена при чтении исходного кода
ядра.  Вероятно, со временем будут внесены некоторые исправления.

------------------------------------------------------------------------------

Общие сведения:

	Если не указано иное, все вызовы ioctl возвращают 0 при успехе
	и -1 с установкой errno в соответствующее значение при ошибке.

	Если не указано иное, все вызовы ioctl возвращают -1 и устанавливают
	errno в EFAULT при неудачной попытке скопировать данные в адресное
	пространство пользователя или из него.

	Если не указано иное, все структуры данных и константы
	определены в <linux/hdreg.h>

------------------------------------------------------------------------------

HDIO_GETGEO
	получить геометрию устройства


	использование::

	  struct hd_geometry geom;

	  ioctl(fd, HDIO_GETGEO, &geom);


	входные данные:
		нет



	выходные данные:
		структура hd_geometry, содержащая:


	    =========	==================================
	    heads	число головок
	    sectors	число секторов на дорожку
	    cylinders	число цилиндров, по модулю 65536
	    start	начальный сектор этого раздела.
	    =========	==================================


	возвращаемые ошибки:
	  - EINVAL

			если устройство не является дисковым накопителем или
			дисководом гибких дисков, либо если пользователь
			передаёт нулевой указатель


	примечания:
		Не особенно полезен для современных дисковых накопителей,
		чья геометрия в любом случае является вежливой фикцией.
		Современные накопители в наши дни адресуются исключительно
		по номеру сектора (адресация lba), а геометрия накопителя
		является абстракцией, которая на деле может изменяться.
		В настоящее время (по состоянию на ноябрь 2004 г.) значения
		геометрии — это «биосовские» значения, предположительно те
		значения, которые накопитель имел при первой загрузке Linux.

		Кроме того, поле cylinders структуры hd_geometry имеет тип
		unsigned short, что означает, что на большинстве архитектур
		этот ioctl не вернёт осмысленного значения для накопителей,
		имеющих более 65535 дорожек.

		Поле start имеет тип unsigned long, что означает, что оно не
		будет содержать осмысленного значения для дисков размером
		более 219 Гб.



HDIO_GET_IDENTITY
	получить идентификационную информацию IDE


	использование::

	  unsigned char identity[512];

	  ioctl(fd, HDIO_GET_IDENTITY, identity);

	входные данные:
		нет



	выходные данные:
		идентификационная информация накопителя ATA.  Полное описание
		см. в командах IDENTIFY DEVICE и IDENTIFY PACKET DEVICE
		в спецификации ATA.

	возвращаемые ошибки:
	  - EINVAL	Вызван для раздела, а не для устройства диска целиком
	  - ENOMSG	Информация IDENTIFY DEVICE недоступна

	примечания:
		Возвращает информацию, которая была получена при опросе
		накопителя.  Часть этой информации может изменяться, а данный
		ioctl не выполняет повторный опрос накопителя для обновления
		информации.

		Эта информация также доступна из /proc/ide/hdX/identify



HDIO_GET_32BIT
	получить текущую настройку io_32bit


	использование::

	  long val;

	  ioctl(fd, HDIO_GET_32BIT, &val);

	входные данные:
		нет



	выходные данные:
		Значение текущей настройки io_32bit



	примечания:
		0=16-бит, 1=32-бит, 2,3 = 32-бит+sync



HDIO_DRIVE_TASKFILE
	выполнить «сырой» taskfile


	Примечание:
		Если у вас под рукой нет копии спецификации ANSI ATA,
		вам, вероятно, следует проигнорировать этот ioctl.

	- Выполнить команду диска ATA напрямую путём записи в регистры
	  «taskfile» накопителя.  Требует привилегий доступа ADMIN и RAWIO.

	использование::

	  struct {

	    ide_task_request_t req_task;
	    u8 outbuf[OUTPUT_SIZE];
	    u8 inbuf[INPUT_SIZE];
	  } task;
	  memset(&task.req_task, 0, sizeof(task.req_task));
	  task.req_task.out_size = sizeof(task.outbuf);
	  task.req_task.in_size = sizeof(task.inbuf);
	  ...
	  ioctl(fd, HDIO_DRIVE_TASKFILE, &task);
	  ...

	входные данные:

	  (Подробности об области памяти, передаваемой в ioctl, см. ниже.)

	  ============	===================================================
	  io_ports[8]	значения для записи в регистры taskfile
	  hob_ports[8]	старшие байты, для расширенных команд.
	  out_flags	флаги, указывающие, какие регистры действительны
	  in_flags	флаги, указывающие, какие регистры следует вернуть
	  data_phase	см. ниже
	  req_cmd	тип выполняемой команды
	  out_size	размер выходного буфера
	  outbuf	буфер данных, передаваемых на диск
	  inbuf		буфер данных, принимаемых с диска (см. [1])
	  ============	===================================================

	выходные данные:

	  ===========	====================================================
	  io_ports[]	значения, возвращённые в регистрах taskfile
	  hob_ports[]	старшие байты, для расширенных команд.
	  out_flags	флаги, указывающие, какие регистры действительны (см. [2])
	  in_flags	флаги, указывающие, какие регистры следует вернуть
	  outbuf	буфер данных, передаваемых на диск (см. [1])
	  inbuf		буфер данных, принимаемых с диска
	  ===========	====================================================

	возвращаемые ошибки:
	  - EACCES	Привилегия CAP_SYS_ADMIN или CAP_SYS_RAWIO не установлена.
	  - ENOMSG	Устройство не является дисковым накопителем.
	  - ENOMEM	Не удалось выделить память для task
	  - EFAULT	req_cmd == TASKFILE_IN_OUT (не реализовано по состоянию на 2.6.8)
	  - EPERM

			req_cmd == TASKFILE_MULTI_OUT, а multi-count
			накопителя ещё не установлен.
	  - EIO		Накопитель не смог выполнить команду.

	примечания:

	  [1] ПРОЧИТАЙТЕ СЛЕДУЮЩИЕ ПРИМЕЧАНИЯ *ВНИМАТЕЛЬНО*.  ЭТОТ IOCTL
	  ПОЛОН ПОДВОДНЫХ КАМНЕЙ.  При использовании этого ioctl следует
	  соблюдать крайнюю осторожность.  Ошибка может легко повредить
	  данные или подвесить систему.

	  [2] Как входной, так и выходной буферы копируются от
	  пользователя и записываются обратно пользователю, даже когда
	  не используются.

	  [3] Если в out_flags установлен один или несколько битов, а
	  in_flags равно нулю, для in_flags.all используются следующие
	  значения, которые по завершении записываются обратно в in_flags.

	   * IDE_TASKFILE_STD_IN_FLAGS | (IDE_HOB_STD_IN_FLAGS << 8)
	     если для накопителя включена адресация LBA48
	   * IDE_TASKFILE_STD_IN_FLAGS
	     если CHS/LBA28

	  Соответствие между in_flags.all и каждым битовым полем разрешения
	  меняется в зависимости от порядка байтов; к счастью, TASKFILE
	  использует только бит inflags.b.data и игнорирует все остальные
	  биты.  В итоге на машинах с любым порядком байтов это не имеет
	  иного эффекта, кроме изменения in_flags по завершении.

	  [4] Значение SELECT по умолчанию равно (0xa0|DEV_bit|LBA_bit),
	  за исключением чипсетов с четырьмя накопителями на порт.  Для
	  чипсетов с четырьмя накопителями на порт оно равно
	  (0xa0|DEV_bit|LBA_bit) для первой пары и (0x80|DEV_bit|LBA_bit)
	  для второй пары.

	  [5] Аргумент ioctl — это указатель на область памяти,
	  содержащую структуру ide_task_request_t, за которой следует
	  необязательный буфер данных, передаваемых на накопитель, за
	  которым следует необязательный буфер для приёма данных с
	  накопителя.

	  Команда передаётся дисковому накопителю через структуру
	  ide_task_request_t, которая содержит следующие поля:

	    ============	===============================================
	    io_ports[8]		значения для регистров taskfile
	    hob_ports[8]	старшие байты, для расширенных команд
	    out_flags		флаги, указывающие, какие элементы массивов
				io_ports[] и hob_ports[] содержат
				действительные значения.  Тип ide_reg_valid_t.
	    in_flags		флаги, указывающие, какие элементы массивов
				io_ports[] и hob_ports[] должны содержать
				действительные значения при возврате.
	    data_phase		См. ниже
	    req_cmd		Тип команды, см. ниже
	    out_size		размер выходного буфера (пользователь->накопитель), байты
	    in_size		размер входного буфера (накопитель->пользователь), байты
	    ============	===============================================

	  Когда out_flags равно нулю, загружаются следующие регистры.

	    ============	===============================================
	    HOB_FEATURE		Если накопитель поддерживает LBA48
	    HOB_NSECTOR		Если накопитель поддерживает LBA48
	    HOB_SECTOR		Если накопитель поддерживает LBA48
	    HOB_LCYL		Если накопитель поддерживает LBA48
	    HOB_HCYL		Если накопитель поддерживает LBA48
	    FEATURE
	    NSECTOR
	    SECTOR
	    LCYL
	    HCYL
	    SELECT		Сначала маскируется значением 0xE0, если LBA48,
				иначе 0xEF; затем применяется операция «или» со
				значением SELECT по умолчанию.
	    ============	===============================================

	  Если установлен какой-либо бит в out_flags, загружаются следующие регистры.

	    ============	===============================================
	    HOB_DATA		Если установлен out_flags.b.data.  HOB_DATA будет
				передаваться по DD8-DD15 на машинах с прямым
				порядком байтов и по DD0-DD7 на машинах с
				обратным порядком байтов.
	    DATA		Если установлен out_flags.b.data.  DATA будет
				передаваться по DD0-DD7 на машинах с прямым
				порядком байтов и по DD8-DD15 на машинах с
				обратным порядком байтов.
	    HOB_NSECTOR		Если установлен out_flags.b.nsector_hob
	    HOB_SECTOR		Если установлен out_flags.b.sector_hob
	    HOB_LCYL		Если установлен out_flags.b.lcyl_hob
	    HOB_HCYL		Если установлен out_flags.b.hcyl_hob
	    FEATURE		Если установлен out_flags.b.feature
	    NSECTOR		Если установлен out_flags.b.nsector
	    SECTOR		Если установлен out_flags.b.sector
	    LCYL		Если установлен out_flags.b.lcyl
	    HCYL		Если установлен out_flags.b.hcyl
	    SELECT		Применяется операция «или» со значением SELECT
				по умолчанию и загружается независимо от
				out_flags.b.select.
	    ============	===============================================

	  Регистры taskfile считываются обратно из накопителя в
	  {io|hob}_ports[] после завершения команды тогда и только тогда,
	  когда выполнено одно из следующих условий; в противном случае
	  обратно записываются исходные значения без изменений.

	    1. Накопитель не смог выполнить команду (EIO).
	    2. В out_flags установлен один или несколько битов.
	    3. Запрошенная data_phase равна TASKFILE_NO_DATA.

	    ============	===============================================
	    HOB_DATA		Если установлен in_flags.b.data.  Будет содержать
				DD8-DD15 на машинах с прямым порядком байтов и
				DD0-DD7 на машинах с обратным порядком байтов.
	    DATA		Если установлен in_flags.b.data.  Будет содержать
				DD0-DD7 на машинах с прямым порядком байтов и
				DD8-DD15 на машинах с обратным порядком байтов.
	    HOB_FEATURE		Если накопитель поддерживает LBA48
	    HOB_NSECTOR		Если накопитель поддерживает LBA48
	    HOB_SECTOR		Если накопитель поддерживает LBA48
	    HOB_LCYL		Если накопитель поддерживает LBA48
	    HOB_HCYL		Если накопитель поддерживает LBA48
	    NSECTOR
	    SECTOR
	    LCYL
	    HCYL
	    ============	===============================================

	  Поле data_phase описывает выполняемую передачу данных.
	  Значение является одним из:

	    ===================        ========================================
	    TASKFILE_IN
	    TASKFILE_MULTI_IN
	    TASKFILE_OUT
	    TASKFILE_MULTI_OUT
	    TASKFILE_IN_OUT
	    TASKFILE_IN_DMA
	    TASKFILE_IN_DMAQ		== IN_DMA (организация очередей не поддерживается)
	    TASKFILE_OUT_DMA
	    TASKFILE_OUT_DMAQ		== OUT_DMA (организация очередей не поддерживается)
	    TASKFILE_P_IN		не реализовано
	    TASKFILE_P_IN_DMA		не реализовано
	    TASKFILE_P_IN_DMAQ		не реализовано
	    TASKFILE_P_OUT		не реализовано
	    TASKFILE_P_OUT_DMA		не реализовано
	    TASKFILE_P_OUT_DMAQ		не реализовано
	    ===================        ========================================

	  Поле req_cmd классифицирует тип команды.  Может быть
	  одним из:

	    ========================    =======================================
	    IDE_DRIVE_TASK_NO_DATA
	    IDE_DRIVE_TASK_SET_XFER	не реализовано
	    IDE_DRIVE_TASK_IN
	    IDE_DRIVE_TASK_OUT		не реализовано
	    IDE_DRIVE_TASK_RAW_WRITE
	    ========================    =======================================

	  [6] Не обращайтесь к {in|out}_flags->all, кроме как для сброса
	  всех битов.  Всегда обращайтесь к отдельным битовым полям.  Значение
	  ->all будет меняться в зависимости от порядка байтов.  По той же
	  причине не используйте константы IDE_{TASKFILE|HOB}_STD_{OUT|IN}_FLAGS,
	  определённые в hdreg.h.



HDIO_DRIVE_CMD
	выполнить специальную команду накопителя


	Примечание:  Если у вас под рукой нет копии спецификации ANSI ATA,
	вам, вероятно, следует проигнорировать этот ioctl.

	использование::

	  u8 args[4+XFER_SIZE];

	  ...
	  ioctl(fd, HDIO_DRIVE_CMD, args);

	входные данные:
	    Команды, отличные от WIN_SMART:

	    =======     =======
	    args[0]	COMMAND
	    args[1]	NSECTOR
	    args[2]	FEATURE
	    args[3]	NSECTOR
	    =======     =======

	    WIN_SMART:

	    =======     =======
	    args[0]	COMMAND
	    args[1]	SECTOR
	    args[2]	FEATURE
	    args[3]	NSECTOR
	    =======     =======

	выходные данные:
		буфер args[] заполняется значениями регистров, за которыми
		следуют любые


	  данные, возвращённые диском.

	    ========	====================================================
	    args[0]	status
	    args[1]	error
	    args[2]	NSECTOR
	    args[3]	не определено
	    args[4+]	NSECTOR * 512 байт данных, возвращённых командой.
	    ========	====================================================

	возвращаемые ошибки:
	  - EACCES	Доступ запрещён:  требуется CAP_SYS_RAWIO
	  - ENOMEM	Не удалось выделить память для task
	  - EIO		Накопитель сообщает об ошибке

	примечания:

	  [1] Для команд, отличных от WIN_SMART, args[1] должно быть равно
	  args[3].  SECTOR, LCYL и HCYL не определены.  Для
	  WIN_SMART в LCYL и HCYL загружаются 0x4f и 0xc2
	  соответственно.  В обоих случаях SELECT будет содержать значение
	  по умолчанию для накопителя.  Значение SELECT по умолчанию см. в
	  примечаниях к HDIO_DRIVE_TASKFILE.

	  [2] Если значение NSECTOR больше нуля и накопитель устанавливает
	  DRQ при прерывании по команде, то NSECTOR * 512 байт
	  считываются из устройства в область, следующую за NSECTOR.
	  В приведённом выше примере этой областью была бы
	  args[4..4+XFER_SIZE].  Используется 16-битный PIO независимо от
	  настройки HDIO_SET_32BIT.

	  [3] Если COMMAND == WIN_SETFEATURES && FEATURE == SETFEATURES_XFER
	  && NSECTOR >= XFER_SW_DMA_0 && накопитель поддерживает какой-либо
	  режим DMA, драйвер IDE попытается настроить режим передачи
	  накопителя соответствующим образом.



HDIO_DRIVE_TASK
	выполнить task и специальную команду накопителя


	Примечание:  Если у вас под рукой нет копии спецификации ANSI ATA,
	вам, вероятно, следует проигнорировать этот ioctl.

	использование::

	  u8 args[7];

	  ...
	  ioctl(fd, HDIO_DRIVE_TASK, args);

	входные данные:
	    Значения регистров taskfile:

	    =======	=======
	    args[0]	COMMAND
	    args[1]	FEATURE
	    args[2]	NSECTOR
	    args[3]	SECTOR
	    args[4]	LCYL
	    args[5]	HCYL
	    args[6]	SELECT
	    =======	=======

	выходные данные:
	    Значения регистров taskfile:


	    =======	=======
	    args[0]	status
	    args[1]	error
	    args[2]	NSECTOR
	    args[3]	SECTOR
	    args[4]	LCYL
	    args[5]	HCYL
	    args[6]	SELECT
	    =======	=======

	возвращаемые ошибки:
	  - EACCES	Доступ запрещён:  требуется CAP_SYS_RAWIO
	  - ENOMEM	Не удалось выделить память для task
	  - ENOMSG	Устройство не является дисковым накопителем.
	  - EIO		Накопитель не смог выполнить команду.

	примечания:

	  [1] Бит DEV (0x10) регистра SELECT игнорируется, и используется
	  соответствующее значение для накопителя.  Все остальные биты
	  используются без изменений.



HDIO_SET_32BIT
	изменить флаги io_32bit


	использование::

	  int val;

	  ioctl(fd, HDIO_SET_32BIT, val);
