Проверка необходимости обновления переводов
============================================

Этот скрипт помогает отслеживать состояние перевода документации на
различных локалях, то есть актуальность перевода относительно его
англоязычного аналога.

Как это работает
----------------

Скрипт использует команду ``git log``, чтобы отследить последний английский
коммит начиная с коммита перевода (упорядочивая по дате автора) и последние
английские коммиты от HEAD. При наличии каких-либо различий файл считается
устаревшим, после чего коммиты, которые требуется учесть, собираются и
выводятся в отчёте.

Реализованная функциональность

-  проверка всех файлов в определённой локали
-  проверка отдельного файла или набора файлов
-  предоставление опций для изменения формата вывода
-  отслеживание состояния перевода для файлов, не имеющих перевода

Использование
-------------

::

   tools/docs/checktransupdate.py --help

Подробности об использовании см. в выводе парсера аргументов.

Примеры

-  ``tools/docs/checktransupdate.py -l zh_CN``
   Это выведет все файлы, которые требуют обновления в локали zh_CN.
-  ``tools/docs/checktransupdate.py Documentation/translations/zh_CN/dev-tools/testing-overview.rst``
   Это выведет только состояние указанного файла.

После чего вывод будет примерно следующим:

::

    Documentation/dev-tools/kfence.rst
    No translation in the locale of zh_CN

    Documentation/translations/zh_CN/dev-tools/testing-overview.rst
    commit 42fb9cfd5b18 ("Documentation: dev-tools: Add link to RV docs")
    1 commits needs resolving in total

Функциональность, которую предстоит реализовать

- в качестве входных данных может выступать каталог, а не только файл
