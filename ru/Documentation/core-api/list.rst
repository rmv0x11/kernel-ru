==============================
Связные списки в Linux
==============================

:Author: Nicolas Frattaroli <nicolas.frattaroli@collabora.com>

.. contents::

Введение
========

Связные списки — одна из самых базовых структур данных, используемых во множестве
программ. Ядро Linux реализует несколько различных разновидностей связных списков.
Цель этого документа — не объяснить связные списки в общем виде, а показать новым
разработчикам ядра, как пользоваться реализациями связных списков из ядра Linux.

Обратите внимание, что хотя связные списки, безусловно, встречаются повсеместно, они
редко являются наилучшей структурой данных для случаев, когда простого массива ещё
недостаточно. В частности, из-за плохой локальности данных связные списки — плохой
выбор в ситуациях, где может иметь значение производительность. Настоятельно
рекомендуется ознакомиться с другими универсальными структурами данных, имеющимися в
ядре, особенно для конкурентного доступа.

Реализация двусвязных списков в Linux
=====================================

Реализации связных списков из Linux можно использовать, включив заголовочный файл
``<linux/list.h>``.

Двусвязный список, вероятно, окажется наиболее знакомым многим читателям. Это список,
по которому можно эффективно проходить как вперёд, так и назад.

Двусвязный список в ядре Linux по своей природе кольцевой. Это означает, что, чтобы
попасть от головного узла к хвостовому, мы можем просто пройти одно ребро назад. Точно
так же, чтобы попасть от хвостового узла к головному, мы можем просто пройти вперёд
«за» хвост и вернуться обратно к голове.

Объявление узла
---------------

Узел двусвязного списка объявляется добавлением члена struct list_head к структуре
данных, которую вы хотите поместить в список:

.. code-block:: c

  struct clown {
          unsigned long long shoe_size;
          const char *name;
          struct list_head node;  /* the aforementioned member */
  };

Для некоторых такой подход может оказаться непривычным, так как классическое объяснение
связного списка — это структура данных узла списка с указателями на предыдущий и
следующий узлы списка, а также полезные данные. Linux выбирает этот подход, потому что
он позволяет писать универсальный код изменения списка независимо от того, какая
структура данных содержится в списке. Поскольку член struct list_head является не
указателем, а частью самой структуры данных, реализация списка может использовать
шаблон container_of() для доступа к полезным данным независимо от их типа, оставаясь в
неведении относительно того, каков этот тип на самом деле.

Объявление и инициализация списка
---------------------------------

Двусвязный список можно затем объявить как ещё один struct list_head и
проинициализировать макросом LIST_HEAD_INIT() при первоначальном присваивании или
функцией INIT_LIST_HEAD() позже:

.. code-block:: c

  struct clown_car {
          int tyre_pressure[4];
          struct list_head clowns;        /* Looks like a node! */
  };

  /* ... Somewhere later in our driver ... */

  static int circus_init(struct circus_priv *circus)
  {
          struct clown_car other_car = {
                .tyre_pressure = {10, 12, 11, 9},
                .clowns = LIST_HEAD_INIT(other_car.clowns)
          };

          INIT_LIST_HEAD(&circus->car.clowns);

          return 0;
  }

Ещё одним источником путаницы для некоторых может быть то, что у самого списка на самом
деле нет собственного типа. Понятие всего связного списка и член struct list_head,
который указывает на другие элементы списка, — это одно и то же.

Добавление узлов в список
-------------------------

Добавление узла в связный список выполняется посредством макроса list_add().

Мы вернёмся к нашему примеру с автомобилем клоунов, чтобы проиллюстрировать, как узлы
добавляются в список:

.. code-block:: c

  static int circus_fill_car(struct circus_priv *circus)
  {
          struct clown_car *car = &circus->car;
          struct clown *grock;
          struct clown *dimitri;

          /* State 1 */

          grock = kzalloc(sizeof(*grock), GFP_KERNEL);
          if (!grock)
                  return -ENOMEM;
          grock->name = "Grock";
          grock->shoe_size = 1000;

          /* Note that we're adding the "node" member */
          list_add(&grock->node, &car->clowns);

          /* State 2 */

          dimitri = kzalloc(sizeof(*dimitri), GFP_KERNEL);
          if (!dimitri)
                  return -ENOMEM;
          dimitri->name = "Dimitri";
          dimitri->shoe_size = 50;

          list_add(&dimitri->node, &car->clowns);

          /* State 3 */

          return 0;
  }

В состоянии State 1 наш список клоунов всё ещё пуст::

         .------.
         v      |
    .--------.  |
    | clowns |--'
    '--------'

На этой диаграмме показан единственный узел «clowns», указывающий сам на себя. На этой
диаграмме и на всех последующих диаграммах для ясности показаны только ребра,
направленные вперёд.

В состоянии State 2 мы добавили Grock после головы списка::

         .--------------------.
         v                    |
    .--------.     .-------.  |
    | clowns |---->| Grock |--'
    '--------'     '-------'

На этой диаграмме показан узел «clowns», указывающий на новый узел с меткой «Grock».
Узел Grock указывает обратно на узел «clowns».

В состоянии State 3 мы добавили Dimitri после головы списка, что приводит к следующему::

         .------------------------------------.
         v                                    |
    .--------.     .---------.     .-------.  |
    | clowns |---->| Dimitri |---->| Grock |--'
    '--------'     '---------'     '-------'

На этой диаграмме показан узел «clowns», указывающий на новый узел с меткой «Dimitri»,
который затем указывает на узел с меткой «Grock». Узел «Grock» по-прежнему указывает
обратно на узел «clowns».

Если бы мы хотели вместо этого вставить Dimitri в конец списка, мы бы использовали
list_add_tail(). Тогда наш код выглядел бы так:

.. code-block:: c

  static int circus_fill_car(struct circus_priv *circus)
  {
          /* ... */

          list_add_tail(&dimitri->node, &car->clowns);

          /* State 3b */

          return 0;
  }

Это приводит к следующему списку::

         .------------------------------------.
         v                                    |
    .--------.     .-------.     .---------.  |
    | clowns |---->| Grock |---->| Dimitri |--'
    '--------'     '-------'     '---------'

На этой диаграмме показан узел «clowns», указывающий на узел с меткой «Grock», который
указывает на новый узел с меткой «Dimitri». Узел с меткой «Dimitri» указывает обратно
на узел «clowns».

Проход по списку
----------------

Чтобы итерировать список, мы можем пройти в цикле по всем узлам внутри списка с помощью
list_for_each().

В нашем примере с клоунами это приводит к следующему несколько неуклюжему коду:

.. code-block:: c

  static unsigned long long circus_get_max_shoe_size(struct circus_priv *circus)
  {
          unsigned long long res = 0;
          struct clown *e;
          struct list_head *cur;

          list_for_each(cur, &circus->car.clowns) {
                  e = list_entry(cur, struct clown, node);
                  if (e->shoe_size > res)
                          res = e->shoe_size;
          }

          return res;
  }

Макрос list_entry() внутри использует упомянутый ранее container_of(), чтобы получить
экземпляр структуры данных, членом которой является ``node``.

Обратите внимание, насколько неуклюжим здесь является дополнительный вызов
list_entry(). Он присутствует только потому, что мы итерируем по членам ``node``, а в
действительности мы хотим итерировать по полезным данным, то есть по ``struct clown``,
содержащему struct list_head каждого узла. По этой причине существует второй макрос:
list_for_each_entry()

Его использование изменило бы наш код примерно на такой:

.. code-block:: c

  static unsigned long long circus_get_max_shoe_size(struct circus_priv *circus)
  {
          unsigned long long res = 0;
          struct clown *e;

          list_for_each_entry(e, &circus->car.clowns, node) {
                  if (e->shoe_size > res)
                          res = e->shoe_size;
          }

          return res;
  }

Это устраняет необходимость в шаге list_entry(), и наш курсор цикла теперь имеет тип
наших полезных данных. Макросу передаётся имя члена, соответствующего struct list_head
списка внутри структуры данных clown, чтобы он всё ещё мог обходить список.

Удаление узлов из списка
------------------------

Функцию list_del() можно использовать для удаления элементов из списка. Она не только
удаляет данный элемент из списка, но и «отравляет» указатели ``prev`` и ``next``
элемента, чтобы непреднамеренное использование элемента после удаления не осталось
незамеченным.

Мы можем расширить наш предыдущий пример, чтобы удалить один из элементов:

.. code-block:: c

  static int circus_fill_car(struct circus_priv *circus)
  {
          /* ... */

          list_add(&dimitri->node, &car->clowns);

          /* State 3 */

          list_del(&dimitri->node);

          /* State 4 */

          return 0;
  }

Результатом этого будет следующее::

         .--------------------.
         v                    |
    .--------.     .-------.  |      .---------.
    | clowns |---->| Grock |--'      | Dimitri |
    '--------'     '-------'         '---------'

На этой диаграмме показан узел «clowns», указывающий на узел с меткой «Grock», который
указывает обратно на узел «clowns». В стороне находится одиночный узел с меткой
«Dimitri», у которого нет стрелок, указывающих куда-либо.

Обратите внимание, что узел Dimitri не указывает сам на себя; его указатели намеренно
установлены в «отравленное» значение, по которому код списка отказывается проходить.

Если бы мы хотели вместо этого переинициализировать удалённый узел, чтобы он снова
указывал сам на себя как пустая голова списка, мы можем вместо этого использовать
list_del_init():

.. code-block:: c

  static int circus_fill_car(struct circus_priv *circus)
  {
          /* ... */

          list_add(&dimitri->node, &car->clowns);

          /* State 3 */

          list_del_init(&dimitri->node);

          /* State 4b */

          return 0;
  }

Это приводит к тому, что удалённый узел снова указывает сам на себя::

         .--------------------.           .-------.
         v                    |           v       |
    .--------.     .-------.  |      .---------.  |
    | clowns |---->| Grock |--'      | Dimitri |--'
    '--------'     '-------'         '---------'

На этой диаграмме показан узел «clowns», указывающий на узел с меткой «Grock», который
указывает обратно на узел «clowns». В стороне находится одиночный узел с меткой
«Dimitri», который указывает сам на себя.

Проход с одновременным удалением узлов
--------------------------------------

Удаление элементов во время прохода по списку вызовет проблемы, если мы используем
list_for_each() и list_for_each_entry(), так как удаление текущего элемента изменило бы
его указатель ``next``, а это означает, что проход не может корректно продвинуться к
следующему элементу списка.

Однако для этого есть решение: list_for_each_safe() и list_for_each_entry_safe(). Они
принимают дополнительный параметр — указатель на struct list_head, используемый как
временное хранилище для следующего элемента во время итерации, что решает проблему.

Пример того, как это использовать:

.. code-block:: c

  static void circus_eject_insufficient_clowns(struct circus_priv *circus)
  {
          struct clown *e;
          struct clown *n;      /* temporary storage for safe iteration */

          list_for_each_entry_safe(e, n, &circus->car.clowns, node) {
                if (e->shoe_size < 500)
                        list_del(&e->node);
          }
  }

Правильное управление памятью (то есть освобождение удалённого узла с одновременной
проверкой того, что на него больше ничего не ссылается) в этом случае оставлено в
качестве упражнения для читателя.

Разрезание списка
-----------------

Существуют две вспомогательные функции для разрезания списков. Обе берут элементы из
списка ``head`` и заменяют содержимое списка ``list``.

Первая такая функция — list_cut_position(). Она удаляет все элементы списка из ``head``
вплоть до ``entry`` включительно, помещая их вместо этого в ``list``.

В этом примере предполагается, что мы начинаем со следующего списка::

         .----------------------------------------------------------------.
         v                                                                |
    .--------.     .-------.     .---------.     .-----.     .---------.  |
    | clowns |---->| Grock |---->| Dimitri |---->| Pic |---->| Alfredo |--'
    '--------'     '-------'     '---------'     '-----'     '---------'

С помощью следующего кода каждый клоун вплоть до «Pic» включительно перемещается из
головы списка «clowns» в отдельный struct list_head, инициализированный в локальной
переменной стека ``retirement``:

.. code-block:: c

  static void circus_retire_clowns(struct circus_priv *circus)
  {
          struct list_head retirement = LIST_HEAD_INIT(retirement);
          struct clown *grock, *dimitri, *pic, *alfredo;
          struct clown_car *car = &circus->car;

          /* ... clown initialization, list adding ... */

          list_cut_position(&retirement, &car->clowns, &pic->node);

          /* State 1 */
  }

Получившийся список ``car->clowns`` будет таким::

         .----------------------.
         v                      |
    .--------.     .---------.  |
    | clowns |---->| Alfredo |--'
    '--------'     '---------'

Тем временем список ``retirement`` преобразуется в следующий::

           .--------------------------------------------------.
           v                                                  |
    .------------.     .-------.     .---------.     .-----.  |
    | retirement |---->| Grock |---->| Dimitri |---->| Pic |--'
    '------------'     '-------'     '---------'     '-----'

Вторая функция, list_cut_before(), во многом такая же, за исключением того, что она
разрезает перед узлом ``entry``, то есть удаляет все элементы списка из ``head`` вплоть
до ``entry``, но не включая его, помещая их вместо этого в ``list``. В этом примере
предполагается тот же начальный список, что и в предыдущем примере:

.. code-block:: c

  static void circus_retire_clowns(struct circus_priv *circus)
  {
          struct list_head retirement = LIST_HEAD_INIT(retirement);
          struct clown *grock, *dimitri, *pic, *alfredo;
          struct clown_car *car = &circus->car;

          /* ... clown initialization, list adding ... */

          list_cut_before(&retirement, &car->clowns, &pic->node);

          /* State 1b */
  }

Получившийся список ``car->clowns`` будет таким::

         .----------------------------------.
         v                                  |
    .--------.     .-----.     .---------.  |
    | clowns |---->| Pic |---->| Alfredo |--'
    '--------'     '-----'     '---------'

Тем временем список ``retirement`` преобразуется в следующий::

           .--------------------------------------.
           v                                      |
    .------------.     .-------.     .---------.  |
    | retirement |---->| Grock |---->| Dimitri |--'
    '------------'     '-------'     '---------'

Следует отметить, что обе функции уничтожат связи с любыми существующими узлами в
списке-приёмнике ``struct list_head *list``.

Перемещение элементов и частичных списков
-----------------------------------------

Функции list_move() и list_move_tail() можно использовать для перемещения элемента из
одного списка в другой — к началу или к концу соответственно.

В следующем примере мы будем предполагать, что начинаем с двух списков («clowns» и
«sidewalk») в следующем начальном состоянии «State 0»::

         .----------------------------------------------------------------.
         v                                                                |
    .--------.     .-------.     .---------.     .-----.     .---------.  |
    | clowns |---->| Grock |---->| Dimitri |---->| Pic |---->| Alfredo |--'
    '--------'     '-------'     '---------'     '-----'     '---------'

          .-------------------.
          v                   |
    .----------.     .-----.  |
    | sidewalk |---->| Pio |--'
    '----------'     '-----'

Мы применяем к этим двум спискам следующий пример кода:

.. code-block:: c

  static void circus_clowns_exit_car(struct circus_priv *circus)
  {
          struct list_head sidewalk = LIST_HEAD_INIT(sidewalk);
          struct clown *grock, *dimitri, *pic, *alfredo, *pio;
          struct clown_car *car = &circus->car;

          /* ... clown initialization, list adding ... */

          /* State 0 */

          list_move(&pic->node, &sidewalk);

          /* State 1 */

          list_move_tail(&dimitri->node, &sidewalk);

          /* State 2 */
  }

В состоянии State 1 мы приходим к следующей ситуации::

        .-----------------------------------------------------.
        |                                                     |
        v                                                     |
    .--------.     .-------.     .---------.     .---------.  |
    | clowns |---->| Grock |---->| Dimitri |---->| Alfredo |--'
    '--------'     '-------'     '---------'     '---------'

          .-------------------------------.
          v                               |
    .----------.     .-----.     .-----.  |
    | sidewalk |---->| Pic |---->| Pio |--'
    '----------'     '-----'     '-----'

В состоянии State 2, после того как мы переместили Dimitri в хвост списка sidewalk,
ситуация меняется следующим образом::

        .-------------------------------------.
        |                                     |
        v                                     |
    .--------.     .-------.     .---------.  |
    | clowns |---->| Grock |---->| Alfredo |--'
    '--------'     '-------'     '---------'

          .-----------------------------------------------.
          v                                               |
    .----------.     .-----.     .-----.     .---------.  |
    | sidewalk |---->| Pic |---->| Pio |---->| Dimitri |--'
    '----------'     '-----'     '-----'     '---------'

Пока голова исходного и целевого списка являются частью одного и того же списка, мы
можем также эффективно массово переместить сегмент списка в хвостовой конец списка. Мы
продолжаем предыдущий пример, добавляя list_bulk_move_tail() после состояния State 2,
перемещая Pic и Pio в хвостовой конец списка sidewalk.

.. code-block:: c

  static void circus_clowns_exit_car(struct circus_priv *circus)
  {
          struct list_head sidewalk = LIST_HEAD_INIT(sidewalk);
          struct clown *grock, *dimitri, *pic, *alfredo, *pio;
          struct clown_car *car = &circus->car;

          /* ... clown initialization, list adding ... */

          /* State 0 */

          list_move(&pic->node, &sidewalk);

          /* State 1 */

          list_move_tail(&dimitri->node, &sidewalk);

          /* State 2 */

          list_bulk_move_tail(&sidewalk, &pic->node, &pio->node);

          /* State 3 */
  }

Ради краткости на следующей диаграмме изображён только изменённый список «sidewalk» в
состоянии State 3::

          .-----------------------------------------------.
          v                                               |
    .----------.     .---------.     .-----.     .-----.  |
    | sidewalk |---->| Dimitri |---->| Pic |---->| Pio |--'
    '----------'     '---------'     '-----'     '-----'

Обратите внимание, что list_bulk_move_tail() не выполняет никакой проверки того,
действительно ли все три переданных параметра ``struct list_head *`` принадлежат одному
и тому же списку. Если вы используете её за пределами ограничений, заданных
документацией, то результат — это дело между вами и реализацией.

Циклический сдвиг элементов
---------------------------

Распространённая операция записи над списками, особенно когда их используют как
очереди, — это циклический сдвиг. Циклический сдвиг списка означает, что элементы из
начала отправляются в конец.

Для циклического сдвига Linux предоставляет нам две функции: list_rotate_left() и
list_rotate_to_front(). Первую можно представить как велосипедную цепь: она берёт
элемент после переданного ``struct list_head *`` и перемещает его в хвост, что по сути
означает, что весь список, в силу своей кольцевой природы, сдвигается на одну позицию.

Вторая, list_rotate_to_front(), развивает ту же идею на шаг дальше: вместо продвижения
списка на один элемент она продвигает его *до тех пор, пока* указанный элемент не
станет новым началом.

В следующем примере наше начальное состояние State 0 следующее::

         .-----------------------------------------------------------------.
         v                                                                 |
    .--------.   .-------.   .---------.   .-----.   .---------.   .-----. |
    | clowns |-->| Grock |-->| Dimitri |-->| Pic |-->| Alfredo |-->| Pio |-'
    '--------'   '-------'   '---------'   '-----'   '---------'   '-----'

Пример кода, используемый для демонстрации циклических сдвигов списка, следующий:

.. code-block:: c

  static void circus_clowns_rotate(struct circus_priv *circus)
  {
          struct clown *grock, *dimitri, *pic, *alfredo, *pio;
          struct clown_car *car = &circus->car;

          /* ... clown initialization, list adding ... */

          /* State 0 */

          list_rotate_left(&car->clowns);

          /* State 1 */

          list_rotate_to_front(&alfredo->node, &car->clowns);

          /* State 2 */

  }

В состоянии State 1 мы приходим к следующей ситуации::

         .-----------------------------------------------------------------.
         v                                                                 |
    .--------.   .---------.   .-----.   .---------.   .-----.   .-------. |
    | clowns |-->| Dimitri |-->| Pic |-->| Alfredo |-->| Pio |-->| Grock |-'
    '--------'   '---------'   '-----'   '---------'   '-----'   '-------'

Далее, после вызова list_rotate_to_front(), мы приходим к следующему состоянию
State 2::

         .-----------------------------------------------------------------.
         v                                                                 |
    .--------.   .---------.   .-----.   .-------.   .---------.   .-----. |
    | clowns |-->| Alfredo |-->| Pio |-->| Grock |-->| Dimitri |-->| Pic |-'
    '--------'   '---------'   '-----'   '-------'   '---------'   '-----'

Как, надеемся, видно из диаграмм, элементы, стоявшие перед «Alfredo», были перемещены
в хвостовой конец списка.

Обмен элементов местами
-----------------------

Ещё одна распространённая операция — необходимость поменять два элемента местами.

Для этого Linux предоставляет нам list_swap().

В следующем примере у нас есть список с тремя элементами, и мы меняем местами два из
них. Это наше начальное состояние в «State 0»::

         .-----------------------------------------.
         v                                         |
    .--------.   .-------.   .---------.   .-----. |
    | clowns |-->| Grock |-->| Dimitri |-->| Pic |-'
    '--------'   '-------'   '---------'   '-----'

.. code-block:: c

  static void circus_clowns_swap(struct circus_priv *circus)
  {
          struct clown *grock, *dimitri, *pic;
          struct clown_car *car = &circus->car;

          /* ... clown initialization, list adding ... */

          /* State 0 */

          list_swap(&dimitri->node, &pic->node);

          /* State 1 */
  }

Получившийся список в состоянии State 1 следующий::

         .-----------------------------------------.
         v                                         |
    .--------.   .-------.   .-----.   .---------. |
    | clowns |-->| Grock |-->| Pic |-->| Dimitri |-'
    '--------'   '-------'   '-----'   '---------'

Как видно из сравнения диаграмм, узлы «Pic» и «Dimitri» поменялись местами.

Сращивание двух списков
-----------------------

Допустим, у нас есть два списка, в следующем примере один представлен головой списка,
которую мы называем «knie», и один — «stey». В гипотетическом приобретении цирка два
списка клоунов должны быть срощены вместе. Вот наша ситуация в «State 0»::

        .-----------------------------------------.
        |                                         |
        v                                         |
    .------.   .-------.   .---------.   .-----.  |
    | knie |-->| Grock |-->| Dimitri |-->| Pic |--'
    '------'   '-------'   '---------'   '-----'

        .-----------------------------.
        v                             |
    .------.   .---------.   .-----.  |
    | stey |-->| Alfredo |-->| Pio |--'
    '------'   '---------'   '-----'

Функция для сращивания этих двух списков вместе — list_splice(). Наш пример кода
следующий:

.. code-block:: c

  static void circus_clowns_splice(void)
  {
          struct clown *grock, *dimitri, *pic, *alfredo, *pio;
          struct list_head knie = LIST_HEAD_INIT(knie);
          struct list_head stey = LIST_HEAD_INIT(stey);

          /* ... Clown allocation and initialization here ... */

          list_add_tail(&grock->node, &knie);
          list_add_tail(&dimitri->node, &knie);
          list_add_tail(&pic->node, &knie);
          list_add_tail(&alfredo->node, &stey);
          list_add_tail(&pio->node, &stey);

          /* State 0 */

          list_splice(&stey, &dimitri->node);

          /* State 1 */
  }

Вызов list_splice() здесь добавляет все элементы из ``stey`` в список, в котором
находится list_head поля ``node`` элемента ``dimitri``, после ``node`` элемента
``dimitri``. Несколько неожиданная диаграмма получившегося «State 1» следует ниже::

        .-----------------------------------------------------------------.
        |                                                                 |
        v                                                                 |
    .------.   .-------.   .---------.   .---------.   .-----.   .-----.  |
    | knie |-->| Grock |-->| Dimitri |-->| Alfredo |-->| Pio |-->| Pic |--'
    '------'   '-------'   '---------'   '---------'   '-----'   '-----'
                                              ^
              .-------------------------------'
              |
    .------.  |
    | stey |--'
    '------'

Проход по списку ``stey`` больше не приводит к корректному поведению. Вызов
list_for_each() на ``stey`` приводит к бесконечному циклу, так как он никогда не
возвращается обратно к голове списка ``stey``.

Это происходит потому, что list_splice() не переинициализировала list_head, из которого
она взяла элементы, оставив его указатель указывающим в то, что теперь является уже
другим списком.

Если мы хотим избежать этой ситуации, можно использовать list_splice_init(). Она делает
то же самое, что и list_splice(), за исключением того, что переинициализирует list_head
донора после пересадки.

Соображения о конкурентности
----------------------------

Конкурентный доступ к списку и его изменение в большинстве случаев должны быть защищены
блокировкой. В качестве альтернативы и предпочтительно можно использовать примитивы RCU
для списков в случаях с преобладанием чтения, где доступ к списку на чтение
распространён, а изменения списка — менее. Подробнее см.
Documentation/RCU/listRCU.rst.
