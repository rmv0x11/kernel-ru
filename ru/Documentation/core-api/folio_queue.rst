Обзор
=====

Структура folio_queue образует один сегмент в сегментированном списке folio,
который может использоваться для формирования буфера ввода-вывода. Соответственно,
этот список можно обходить с помощью типа iov_iter ITER_FOLIOQ.

Публично доступные члены структуры::

	struct folio_queue {
		struct folio_queue *next;
		struct folio_queue *prev;
		...
	};

Предоставляется пара указателей, ``next`` и ``prev``, которые указывают на
сегменты по обе стороны от сегмента, к которому осуществляется доступ. Хотя это
двусвязный список, он намеренно не является циклическим; внешние указатели на
соседей в концевых сегментах должны быть NULL.

Каждый сегмент в списке также хранит:

 * упорядоченную последовательность указателей на folio,
 * размер каждого folio и
 * три однобитовые метки на каждый folio,

но к ним не следует обращаться напрямую, поскольку внутренняя структура данных
может измениться; вместо этого следует использовать функции доступа, описанные
ниже.

Доступ к этому средству можно получить через::

	#include <linux/folio_queue.h>

а для использования итератора::

	#include <linux/uio.h>


Инициализация
=============

Сегмент следует инициализировать вызовом::

	void folioq_init(struct folio_queue *folioq);

с указателем на инициализируемый сегмент. Обратите внимание, что это не
обязательно инициализирует все указатели на folio, поэтому нужно с осторожностью
проверять число добавленных folio.


Добавление и удаление folio
===========================

Folio можно поместить в следующий неиспользованный слот структуры сегмента,
вызвав одну из::

	unsigned int folioq_append(struct folio_queue *folioq,
				   struct folio *folio);

	unsigned int folioq_append_mark(struct folio_queue *folioq,
					struct folio *folio);

Обе функции обновляют хранимый счётчик folio, сохраняют folio и фиксируют его
размер. Вторая функция также устанавливает первую метку для добавленного folio.
Обе функции возвращают номер использованного слота. [!] Обратите внимание, что
не предпринимается никаких попыток проверить, не превышена ли ёмкость, и список
не будет расширен автоматически.

Folio можно изъять, вызвав::

	void folioq_clear(struct folio_queue *folioq, unsigned int slot);

Это очищает слот в массиве, а также очищает все метки для этого folio, но не
изменяет счётчик folio — поэтому при будущих обращениях к этому слоту необходимо
проверять, занят ли слот.


Запрос информации о folio
=========================

Информацию о folio в конкретном слоте можно запросить с помощью следующей
функции::

	struct folio *folioq_folio(const struct folio_queue *folioq,
				   unsigned int slot);

Если folio ещё не был установлен в этом слоте, это может вернуть неопределённый
указатель. Размер folio в слоте можно запросить с помощью любой из::

	unsigned int folioq_folio_order(const struct folio_queue *folioq,
					unsigned int slot);

	size_t folioq_folio_size(const struct folio_queue *folioq,
				 unsigned int slot);

Первая функция возвращает размер в виде порядка (order), а вторая — в виде числа
байтов.


Запрос информации о folio_queue
===============================

Информацию о конкретном сегменте можно получить с помощью следующих функций::

	unsigned int folioq_nr_slots(const struct folio_queue *folioq);

	unsigned int folioq_count(struct folio_queue *folioq);

	bool folioq_full(struct folio_queue *folioq);

Первая функция возвращает максимальную ёмкость сегмента. Не следует полагать,
что она не будет меняться от сегмента к сегменту. Вторая возвращает число folio,
добавленных в сегмент, а третья является сокращением для индикации того,
заполнен ли сегмент до своей ёмкости.

Обратите внимание, что счётчик и заполненность не изменяются при очистке folio
из сегмента. Они в большей степени служат для индикации того, сколько слотов в
массиве было инициализировано, и предполагается, что слоты не будут повторно
использоваться, а сегмент будет отброшен по мере потребления очереди.


Метки folio
===========

Folio в очереди также могут иметь назначенные им метки. Эти метки можно
использовать для фиксации информации, такой как необходимость вызова
folio_put() для folio. Для каждого folio доступно три метки для установки.

Метки можно установить с помощью::

	void folioq_mark(struct folio_queue *folioq, unsigned int slot);
	void folioq_mark2(struct folio_queue *folioq, unsigned int slot);

Очистить с помощью::

	void folioq_unmark(struct folio_queue *folioq, unsigned int slot);
	void folioq_unmark2(struct folio_queue *folioq, unsigned int slot);

А запросить метки можно с помощью::

	bool folioq_is_marked(const struct folio_queue *folioq, unsigned int slot);
	bool folioq_is_marked2(const struct folio_queue *folioq, unsigned int slot);

Метки можно использовать для любых целей, и этот API их не интерпретирует.


Обход очереди folio
===================

Список сегментов можно обходить с помощью средства итератора ввода-вывода,
используя итератор ``iov_iter`` типа ``ITER_FOLIOQ``. Итератор можно
инициализировать с помощью::

	void iov_iter_folio_queue(struct iov_iter *i, unsigned int direction,
				  const struct folio_queue *folioq,
				  unsigned int first_slot, unsigned int offset,
				  size_t count);

Ему можно указать начинать с конкретного сегмента, слота и смещения в пределах
очереди. Функции итератора iov будут следовать указателям next при продвижении
вперёд и указателям prev при откате назад, когда это необходимо.


Проблемы одновременного производства/потребления без блокировок
===============================================================

При надлежащем управлении список может одновременно расширяться производителем
со стороны головы и укорачиваться потребителем со стороны хвоста без
необходимости брать блокировки. Итератор ITER_FOLIOQ вставляет
соответствующие барьеры, чтобы помочь в этом.

Необходимо проявлять осторожность при одновременном производстве и потреблении
списка. Если достигнут последний сегмент и folio, на которые он ссылается,
полностью потреблены итераторами IOV, структура iov_iter останется указывающей
на последний сегмент с номером слота, равным ёмкости этого сегмента. Итератор
попытается продолжить с этого места, если при его повторном использовании
доступен другой сегмент, но необходимо проявлять осторожность, чтобы сегмент не
был удалён и освобождён потребителем до того, как итератор был продвинут.

Рекомендуется, чтобы очередь всегда содержала хотя бы один сегмент, даже если
этот сегмент никогда не заполнялся или полностью исчерпан. Это предотвращает
схлопывание указателей головы и хвоста.
