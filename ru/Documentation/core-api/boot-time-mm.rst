==========================================
Управление памятью на этапе загрузки
==========================================

Ранняя инициализация системы не может использовать «обычное» управление
памятью просто потому, что оно ещё не настроено. Но при этом всё равно
необходимо выделять память для различных структур данных, например для
физического страничного аллокатора.

Управлением памятью на этапе загрузки занимается специализированный
аллокатор под названием ``memblock``. Архитектурно-зависимая инициализация
должна настроить его в функции :c:func:`setup_arch` и освободить его в
функции :c:func:`mem_init`.

Как только раннее управление памятью становится доступным, оно предоставляет
множество функций и макросов для выделения памяти. Запрос на выделение может
быть направлен к первому (и, вероятно, единственному) узлу или к конкретному
узлу в NUMA-системе. Существуют варианты API, которые вызывают панику при
неудачном выделении, и те, которые этого не делают.

Memblock также предоставляет множество API, управляющих его собственным
поведением.
