================================
API точек трассировки ядра Linux
================================

:Author: Jason Baron
:Author: William Cohen

Введение
========

Точки трассировки (tracepoints) — это статические точки зондирования,
расположенные в стратегических местах по всему ядру. «Зонды» (probes)
регистрируются и снимаются с регистрации в точках трассировки через
механизм обратных вызовов. «Зонды» — это строго типизированные функции,
которым передаётся уникальный набор параметров, определённый каждой точкой
трассировки.

Опираясь на этот простой механизм обратных вызовов, «зонды» можно
использовать для профилирования, отладки и понимания поведения ядра.
Существует ряд инструментов, предоставляющих фреймворк для использования
«зондов». В число этих инструментов входят Systemtap, ftrace и LTTng.

Точки трассировки определяются в ряде заголовочных файлов с помощью
различных макросов. Поэтому цель данного документа — дать ясный учёт
доступных точек трассировки. Замысел состоит в том, чтобы понять не только
то, какие точки трассировки доступны, но и то, где могут быть добавлены
будущие точки трассировки.

Представленный API содержит функции вида:
``trace_tracepointname(function parameters)``. Это обратные вызовы точек
трассировки, которые встречаются по всему коду. Регистрация и снятие
регистрации зондов в этих местах обратных вызовов рассматривается в
каталоге ``Documentation/trace/*``.
