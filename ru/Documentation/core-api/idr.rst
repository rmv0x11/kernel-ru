================================
Распределение ID (ID Allocation)
================================

:Author: Matthew Wilcox

Обзор
=====

Распространённая задача — распределение идентификаторов (ID); как правило, это
небольшие числа, идентифицирующие тот или иной объект. Примеры включают файловые
дескрипторы, идентификаторы процессов (process IDs), идентификаторы пакетов в
сетевых протоколах, теги SCSI и номера экземпляров устройств. IDR и IDA
предоставляют разумное решение этой задачи, чтобы каждому не приходилось изобретать
собственное. IDR даёт возможность отображать ID на указатель, тогда как IDA
обеспечивает только распределение ID и в результате гораздо более эффективен по
памяти.

Интерфейс IDR устарел; пожалуйста, используйте вместо него :doc:`XArray <xarray>`.

Использование IDR
=================

Начните с инициализации IDR — либо с помощью DEFINE_IDR()
для статически распределённых IDR, либо с помощью idr_init() для динамически
распределённых IDR.

Вы можете вызвать idr_alloc(), чтобы распределить неиспользуемый ID. Найдите
указатель, который вы связали с этим ID, вызвав idr_find(),
и освободите ID, вызвав idr_remove().

Если вам нужно изменить указатель, связанный с ID, вы можете вызвать
idr_replace(). Одна из распространённых причин сделать это — зарезервировать
ID, передав указатель ``NULL`` в функцию распределения; инициализируйте
объект с зарезервированным ID и, наконец, вставьте инициализированный объект
в IDR.

Некоторым пользователям необходимо распределять ID, превышающие ``INT_MAX``. До сих пор все
эти пользователи довольствовались пределом ``UINT_MAX`` и используют
idr_alloc_u32(). Если вам нужны ID, которые не помещаются в u32,
мы поработаем с вами над решением ваших задач.

Если вам нужно распределять ID последовательно, вы можете использовать
idr_alloc_cyclic(). IDR становится менее эффективным при работе
с бо́льшими ID, поэтому использование этой функции обходится в небольшую цену.

Чтобы выполнить действие над всеми указателями, используемыми IDR, вы можете
использовать либо основанный на колбэке idr_for_each(), либо
итераторный idr_for_each_entry(). Вам может понадобиться использовать
idr_for_each_entry_continue() для продолжения итерации. Вы можете
также использовать idr_get_next(), если итератор не подходит под ваши нужды.

Когда вы закончили использовать IDR, вы можете вызвать idr_destroy(),
чтобы освободить память, используемую IDR. При этом объекты, на которые
указывает IDR, освобождены не будут; если вы хотите сделать это, воспользуйтесь
одним из итераторов.

Вы можете использовать idr_is_empty(), чтобы узнать, есть ли какие-либо
ID, распределённые в данный момент.

Если вам нужно взять блокировку во время распределения нового ID из IDR,
вам может потребоваться передать ограничительный набор флагов GFP, что может привести
к тому, что IDR окажется не в состоянии распределить память. Чтобы обойти это,
вы можете вызвать idr_preload() перед взятием блокировки, а затем
idr_preload_end() после распределения.
