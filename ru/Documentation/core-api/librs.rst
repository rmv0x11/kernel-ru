Программный интерфейс библиотеки Рида — Соломона (Reed-Solomon)
================================================================

:Author: Thomas Gleixner

Введение
========

Универсальная библиотека Рида — Соломона предоставляет функции кодирования,
декодирования и исправления ошибок.

Коды Рида — Соломона используются в коммуникационных и накопительных
приложениях для обеспечения целостности данных.

Эта документация предназначена для разработчиков, которые хотят использовать
функции, предоставляемые библиотекой.

Известные ошибки и допущения
============================

Нет.

Использование
=============

Эта глава содержит примеры использования библиотеки.

Инициализация
-------------

Функция инициализации init_rs возвращает указатель на структуру декодера rs,
которая хранит информацию, необходимую для кодирования, декодирования и
исправления ошибок с заданным многочленом. Она либо использует уже существующий
подходящий декодер, либо создаёт новый. При создании формируются все таблицы
поиска для быстрого кодирования/декодирования. Функция может выполняться
некоторое время, поэтому не следует вызывать её в критичных по времени участках
кода.

::

    /* the Reed Solomon control structure */
    static struct rs_control *rs_decoder;

    /* Symbolsize is 10 (bits)
     * Primitive polynomial is x^10+x^3+1
     * first consecutive root is 0
     * primitive element to generate roots = 1
     * generator polynomial degree (number of roots) = 6
     */
    rs_decoder = init_rs (10, 0x409, 0, 1, 6);


Кодирование
-----------

Кодировщик вычисляет код Рида — Соломона по данным заданной длины и сохраняет
результат в буфере чётности. Обратите внимание, что буфер чётности должен быть
инициализирован перед вызовом кодировщика.

Расширенные данные могут инвертироваться на лету путём передачи ненулевой маски
инверсии. Расширенные данные подвергаются операции XOR с маской. Это используется,
например, для FLASH ECC, где все 0xFF инвертируются во все 0x00. Код Рида —
Соломона для всех 0x00 — это все 0x00. Код инвертируется перед сохранением во
FLASH, поэтому он также становится 0xFF. Это предотвращает то, что чтение из
стёртой FLASH приводит к ошибкам ECC.

Байты данных расширяются до заданного размера символа на лету. В настоящий момент
поддержки кодирования непрерывных битовых потоков с размером символа != 8 нет.
Если это потребуется, реализовать такую функциональность не составит большого
труда.

::

    /* Parity buffer. Size = number of roots */
    uint16_t par[6];
    /* Initialize the parity buffer */
    memset(par, 0, sizeof(par));
    /* Encode 512 byte in data8. Store parity in buffer par */
    encode_rs8 (rs_decoder, data8, 512, par, 0);


Декодирование
-------------

Декодер вычисляет синдром по данным заданной длины и полученным символам
чётности и исправляет ошибки в данных.

Если синдром доступен от аппаратного декодера, то вычисление синдрома
пропускается.

Исправление буфера данных можно подавить, передав декодеру буфер шаблона
исправления и буфер расположения ошибок. Декодер сохраняет вычисленное
расположение ошибки и битовую маску исправления в заданных буферах. Это полезно
для аппаратных декодеров, которые используют странную схему упорядочивания бит.

Байты данных расширяются до заданного размера символа на лету. В настоящий момент
поддержки декодирования непрерывных битовых потоков с размером символа != 8 нет.
Если это потребуется, реализовать такую функциональность не составит большого
труда.

Декодирование с вычислением синдрома, прямое исправление данных
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

::

    /* Parity buffer. Size = number of roots */
    uint16_t par[6];
    uint8_t  data[512];
    int numerr;
    /* Receive data */
    .....
    /* Receive parity */
    .....
    /* Decode 512 byte in data8.*/
    numerr = decode_rs8 (rs_decoder, data8, par, 512, NULL, 0, NULL, 0, NULL);


Декодирование с синдромом, заданным аппаратным декодером, прямое исправление данных
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

::

    /* Parity buffer. Size = number of roots */
    uint16_t par[6], syn[6];
    uint8_t  data[512];
    int numerr;
    /* Receive data */
    .....
    /* Receive parity */
    .....
    /* Get syndrome from hardware decoder */
    .....
    /* Decode 512 byte in data8.*/
    numerr = decode_rs8 (rs_decoder, data8, par, 512, syn, 0, NULL, 0, NULL);


Декодирование с синдромом, заданным аппаратным декодером, без прямого исправления данных.
~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

Примечание: передавать данные и полученную чётность декодеру не обязательно.

::

    /* Parity buffer. Size = number of roots */
    uint16_t par[6], syn[6], corr[8];
    uint8_t  data[512];
    int numerr, errpos[8];
    /* Receive data */
    .....
    /* Receive parity */
    .....
    /* Get syndrome from hardware decoder */
    .....
    /* Decode 512 byte in data8.*/
    numerr = decode_rs8 (rs_decoder, NULL, NULL, 512, syn, 0, errpos, 0, corr);
    for (i = 0; i < numerr; i++) {
        do_error_correction_in_your_buffer(errpos[i], corr[i]);
    }


Очистка
-------

Функция free_rs освобождает выделенные ресурсы, если вызывающая сторона является
последним пользователем декодера.

::

    /* Release resources */
    free_rs(rs_decoder);
