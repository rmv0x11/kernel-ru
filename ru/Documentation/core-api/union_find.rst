====================
Union-Find в Linux
====================


:Date: June 21, 2024
:Author: Xavier <xavier_qy@163.com>

Что такое union-find и для чего он используется?
------------------------------------------------

Union-find — это структура данных, используемая для обработки объединения и
запросов к непересекающимся множествам. Основные операции, поддерживаемые
union-find:

	Инициализация (Initialization): сброс каждого элемента в отдельное
		множество, при котором начальный родительский узел каждого
		множества указывает на самого себя.

	Поиск (Find): определение того, какому множеству принадлежит конкретный
		элемент, обычно посредством возврата «представительного элемента»
		этого множества. Эта операция используется для проверки того,
		находятся ли два элемента в одном и том же множестве.

	Объединение (Union): слияние двух множеств в одно.

Будучи структурой данных, используемой для поддержки множеств (групп),
union-find обычно применяется для решения задач, связанных с офлайн-запросами,
динамической связностью и теорией графов. Он также является ключевым
компонентом алгоритма Краскала для вычисления минимального остовного дерева,
что имеет решающее значение в таких сценариях, как сетевая маршрутизация.
Следовательно, на union-find широко ссылаются. Кроме того, union-find находит
применение в символьных вычислениях, распределении регистров и многом другом.

Пространственная сложность: O(n), где n — число узлов.

Временная сложность: использование сжатия путей позволяет снизить временную
сложность операции поиска, а использование объединения по рангу позволяет
снизить временную сложность операции объединения. Эти оптимизации снижают
среднюю временную сложность каждой операции поиска и объединения до O(α(n)),
где α(n) — обратная функция Аккермана. На практике её можно приблизительно
считать константной временной сложностью.

Этот документ описывает использование реализации union-find в Linux. Подробнее
о природе и реализации union-find см.:

  Статья в Википедии о union-find
    https://en.wikipedia.org/wiki/Disjoint-set_data_structure

Реализация union-find в Linux
-----------------------------

Реализация union-find в Linux находится в файле "lib/union_find.c".
Чтобы использовать её, выполните "#include <linux/union_find.h>".

Структура данных union-find определена следующим образом::

	struct uf_node {
		struct uf_node *parent;
		unsigned int rank;
	};

В этой структуре parent указывает на родительский узел текущего узла. Поле
rank представляет высоту текущего дерева. Во время операции объединения дерево
с меньшим рангом присоединяется под деревом с большим рангом для поддержания
сбалансированности.

Инициализация union-find
-------------------------

Инициализацию можно выполнить, используя либо статический интерфейс, либо
интерфейс инициализации. Инициализируйте указатель parent так, чтобы он
указывал на самого себя, и установите rank в 0.
Пример::

	struct uf_node my_node = UF_INIT_NODE(my_node);

или

	uf_node_init(&my_node);

Поиск корневого узла union-find
-------------------------------

Эта операция в основном используется для определения того, принадлежат ли два
узла одному и тому же множеству в union-find. Если у них одинаковый корень, они
находятся в одном множестве. Во время операции поиска выполняется сжатие путей
для повышения эффективности последующих операций поиска.
Пример::

	int connected;
	struct uf_node *root1 = uf_find(&node_1);
	struct uf_node *root2 = uf_find(&node_2);
	if (root1 == root2)
		connected = 1;
	else
		connected = 0;

Объединение двух множеств в union-find
--------------------------------------

Чтобы объединить два множества в union-find, необходимо сначала найти их
соответствующие корневые узлы, а затем связать меньший узел с большим узлом на
основе ранга корневых узлов.
Пример::

	uf_union(&node_1, &node_2);
