==========
Maple Tree
==========

:Author: Liam R. Howlett

Обзор
=====

Maple Tree — это структура данных типа B-Tree, оптимизированная для хранения
непересекающихся диапазонов, в том числе диапазонов размером 1. Дерево
проектировалось так, чтобы быть простым в использовании и не требовать от
пользователя написания собственного метода поиска. Оно поддерживает итерацию по
диапазону записей и переход к предыдущей или следующей записи эффективным по
отношению к кэшу образом. Дерево также может быть переведено в RCU-безопасный
режим работы, который позволяет одновременно читать и писать. Писатели должны
синхронизироваться через блокировку, которой может быть спинлок (spinlock) по
умолчанию, либо пользователь может задать в качестве блокировки внешнюю
блокировку другого типа.

Maple Tree расходует немного памяти и спроектирован для эффективного
использования кэша современных процессоров. Большинство пользователей смогут
использовать обычный API. Для более сложных сценариев существует
:ref:`maple-tree-advanced-api`. Самое важное применение Maple Tree — отслеживание
областей виртуальной памяти.

Maple Tree может хранить значения от ``0`` до ``ULONG_MAX``. Maple Tree
резервирует для внутреннего использования значения, у которых два младших бита
установлены в '10' и которые меньше 4096 (то есть 2, 6, 10 .. 4094). Если записи
могут использовать зарезервированные значения, то пользователи могут
преобразовывать записи с помощью xa_mk_value() и преобразовывать их обратно,
вызывая xa_to_value(). Если пользователю необходимо использовать
зарезервированное значение, то он может преобразовать это значение при
использовании :ref:`maple-tree-advanced-api`, но обычный API это блокирует.

Maple Tree также может быть настроен для поддержки поиска промежутка заданного
размера (или больше).

Предварительное выделение узлов также поддерживается с помощью
:ref:`maple-tree-advanced-api`. Это полезно для пользователей, которым необходимо
гарантировать успешное выполнение операции сохранения в пределах заданного
сегмента кода, когда выделение памяти невозможно. Узлы выделяются относительно
небольшими — порядка 256 байт.

Обычный API
===========

Начните с инициализации maple tree: либо с помощью DEFINE_MTREE() для статически
выделяемых maple tree, либо с помощью mt_init() для динамически выделяемых.
Только что инициализированный maple tree содержит указатель ``NULL`` для диапазона
``0`` - ``ULONG_MAX``. В настоящее время поддерживаются два типа maple tree:
дерево выделения (allocation tree) и обычное дерево (regular tree). Обычное дерево
имеет более высокий коэффициент ветвления для внутренних узлов. Дерево выделения
имеет более низкий коэффициент ветвления, но позволяет пользователю искать
промежуток заданного размера или больше либо от ``0`` вверх, либо от
``ULONG_MAX`` вниз. Дерево выделения можно использовать, передав флаг
``MT_FLAGS_ALLOC_RANGE`` при инициализации дерева.

Затем вы можете задавать записи с помощью mtree_store() или mtree_store_range().
mtree_store() перезаписывает любую запись новой записью и возвращает 0 при успехе
или код ошибки в противном случае. mtree_store_range() работает таким же образом,
но принимает диапазон. mtree_load() используется для получения записи, хранящейся
по заданному индексу. Вы можете использовать mtree_erase(), чтобы стереть целый
диапазон, зная всего одно значение в пределах этого диапазона, либо вызов
mtree_store() с записью NULL может использоваться для частичного стирания
диапазона или сразу нескольких диапазонов.

Если вы хотите сохранить новую запись в диапазоне (или по индексу) только в том
случае, если этот диапазон в данный момент равен ``NULL``, вы можете использовать
mtree_insert_range() или mtree_insert(), которые возвращают -EEXIST, если
диапазон не пуст.

Вы можете искать запись начиная с заданного индекса и вверх с помощью mt_find().

Вы можете обойти каждую запись в пределах диапазона, вызвав mt_for_each(). Вы
должны предоставить временную переменную для хранения курсора. Если вы хотите
обойти каждый элемент дерева, то в качестве диапазона можно использовать ``0`` и
``ULONG_MAX``. Если вызывающая сторона собирается удерживать блокировку на всё
время обхода, то стоит обратить внимание на API mas_for_each() в разделе
:ref:`maple-tree-advanced-api`.

Иногда необходимо обеспечить, чтобы следующий вызов сохранения в maple tree не
выделял память; пожалуйста, см. :ref:`maple-tree-advanced-api` для этого варианта
использования.

Вы можете использовать mtree_dup() для дублирования всего maple tree. Это более
эффективный способ, чем вставка всех элементов один за другим в новое дерево.

Наконец, вы можете удалить все записи из maple tree, вызвав mtree_destroy(). Если
записи maple tree являются указателями, то, возможно, вы захотите сначала
освободить эти записи.

Выделение узлов
---------------

Выделения обрабатываются внутренним кодом дерева. См.
:ref:`maple-tree-advanced-alloc` для других вариантов.

Блокировка
----------

Вам не нужно беспокоиться о блокировке. См. :ref:`maple-tree-advanced-locks`
для других вариантов.

Maple Tree использует RCU и внутренний спинлок (spinlock) для синхронизации
доступа:

Берёт RCU read lock:
 * mtree_load()
 * mt_find()
 * mt_for_each()
 * mt_next()
 * mt_prev()

Берёт ma_lock внутренне:
 * mtree_store()
 * mtree_store_range()
 * mtree_insert()
 * mtree_insert_range()
 * mtree_erase()
 * mtree_dup()
 * mtree_destroy()
 * mt_set_in_rcu()
 * mt_clear_in_rcu()

Если вы хотите воспользоваться внутренней блокировкой для защиты структур данных,
которые вы храните в Maple Tree, вы можете вызвать mtree_lock() перед вызовом
mtree_load(), затем взять счётчик ссылок на найденный объект перед вызовом
mtree_unlock(). Это предотвратит удаление объекта из дерева операциями сохранения
в промежутке между поиском объекта и инкрементом счётчика ссылок. Вы также можете
использовать RCU, чтобы избежать разыменования освобождённой памяти, но
объяснение этого выходит за рамки данного документа.

Расширенный API
===============

Расширенный API предлагает большую гибкость и лучшую производительность ценой
интерфейса, который может быть сложнее в использовании и имеет меньше
предохранителей. При использовании расширенного API вы должны самостоятельно
заботиться о собственной блокировке. Для защиты вы можете использовать ma_lock,
RCU или внешнюю блокировку. Вы можете смешивать расширенные и обычные операции на
одном и том же массиве, при условии что блокировка совместима.
:ref:`maple-tree-normal-api` реализован в терминах расширенного API.

Расширенный API строится вокруг ma_state — отсюда и происходит префикс 'mas'.
Структура ma_state отслеживает операции с деревом, чтобы упростить жизнь как
внутренним, так и внешним пользователям дерева.

Инициализация maple tree выполняется так же, как в :ref:`maple-tree-normal-api`.
Пожалуйста, см. выше.

Maple state отслеживает начало и конец диапазона в mas->index и mas->last
соответственно.

mas_walk() обходит дерево до местоположения mas->index и устанавливает mas->index
и mas->last в соответствии с диапазоном для данной записи.

Вы можете задавать записи с помощью mas_store(). mas_store() перезаписывает любую
запись новой записью и возвращает первую существующую запись, которая была
перезаписана. Диапазон передаётся как члены maple state: index и last.

Вы можете использовать mas_erase(), чтобы стереть целый диапазон, установив index
и last maple state в желаемый для стирания диапазон. Это сотрёт первый диапазон,
найденный в этом диапазоне, установит index и last maple state равными стёртому
диапазону и вернёт запись, которая существовала в этом местоположении.

Вы можете обойти каждую запись в пределах диапазона с помощью mas_for_each().
Если вы хотите обойти каждый элемент дерева, то в качестве диапазона можно
использовать ``0`` и ``ULONG_MAX``. Если блокировку необходимо периодически
отпускать, см. раздел о блокировке mas_pause().

Использование maple state позволяет mas_next() и mas_prev() работать так, как
если бы дерево было связным списком. При столь высоком коэффициенте ветвления
амортизированный штраф к производительности перекрывается оптимизацией кэша.
mas_next() возвращает следующую запись, которая идёт после записи по индексу.
mas_prev() возвращает предыдущую запись, которая идёт перед записью по индексу.

mas_find() при первом вызове находит первую запись, которая существует по индексу
или выше него, а при каждом последующем вызове — следующую запись.

mas_find_rev() при первом вызове находит первую запись, которая существует по
last или ниже него, а при каждом последующем вызове — предыдущую запись.

Если пользователю необходимо уступить блокировку во время операции, то maple
state должен быть приостановлен с помощью mas_pause().

При использовании дерева выделения предоставляется несколько дополнительных
интерфейсов. Если вы хотите искать промежуток в пределах диапазона, то можно
использовать mas_empty_area() или mas_empty_area_rev(). mas_empty_area() ищет
промежуток, начиная с заданного наименьшего индекса и до максимума диапазона.
mas_empty_area_rev() ищет промежуток, начиная с заданного наибольшего индекса, и
продолжает движение вниз до нижней границы диапазона.

Расширенное выделение узлов
---------------------------

Выделения обычно обрабатываются внутри дерева, однако если выделения должны
произойти до выполнения записи, то вызов mas_expected_entries() выделит
наихудшее число необходимых узлов для вставки указанного количества диапазонов.
Это также переводит дерево в режим массовой вставки. После завершения вставок
вызов mas_destroy() для maple state освободит неиспользованные выделения.

.. _maple-tree-advanced-locks:

Расширенная блокировка
----------------------

По умолчанию maple tree использует спинлок (spinlock), но для обновлений дерева
также могут использоваться внешние блокировки. Чтобы использовать внешнюю
блокировку, дерево должно быть инициализировано с флагом
``MT_FLAGS_LOCK_EXTERN flag``; обычно это делается с помощью #define
MTREE_INIT_EXT(), который принимает внешнюю блокировку в качестве аргумента.
