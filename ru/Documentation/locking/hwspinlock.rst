===============================================
Каркас аппаратных спинлоков (Hardware Spinlock)
===============================================

Введение
========

Модули аппаратных спинлоков предоставляют аппаратную поддержку синхронизации
и взаимного исключения между разнородными процессорами и теми, что работают
не под управлением единой общей операционной системы.

Например, в OMAP4 имеются два процессора Cortex-A9, два Cortex-M3 и DSP C64x+,
каждый из которых выполняет собственную операционную систему (ведущий процессор,
A9, обычно работает под управлением Linux, а ведомые процессоры, M3 и DSP,
выполняют ту или иную разновидность RTOS).

Обобщённый каркас hwspinlock позволяет платформенно-независимым драйверам
использовать устройство hwspinlock для доступа к структурам данных, которые
совместно используются удалёнными процессорами и которые в противном случае
не имеют альтернативного механизма для выполнения операций синхронизации и
взаимного исключения.

Это необходимо, например, для межпроцессорного взаимодействия:
в OMAP4 ресурсоёмкие мультимедийные задачи выгружаются хостом на
удалённые ведомые процессоры M3 и/или C64x+ (посредством подсистемы IPC под
названием Syslink).

Чтобы обеспечить быстрое взаимодействие на основе сообщений, требуется
минимальная поддержка со стороны ядра для доставки сообщений, приходящих от
удалённого процессора, соответствующему пользовательскому процессу.

Это взаимодействие основано на простых структурах данных, которые совместно
используются удалёнными процессорами, и доступ к ним синхронизируется с
помощью модуля hwspinlock (удалённый процессор напрямую помещает новые
сообщения в эту общую структуру данных).

Общий интерфейс hwspinlock позволяет иметь обобщённые, платформенно-
независимые драйверы.

Пользовательский API
====================

::

  struct hwspinlock *hwspin_lock_request_specific(unsigned int id);

Назначить конкретный идентификатор hwspinlock и вернуть его адрес или NULL,
если данный hwspinlock уже используется. Обычно эту функцию вызывает код
платы, чтобы зарезервировать конкретные идентификаторы hwspinlock для
заранее определённых целей.

Должна вызываться из контекста процесса (может уснуть).

::

  int of_hwspin_lock_get_id(struct device_node *np, int index);

Получить глобальный идентификатор блокировки для конкретной блокировки,
заданной через OF phandle. Эта функция предоставляет DT-пользователям модуля
hwspinlock способ получить глобальный идентификатор конкретного hwspinlock,
чтобы его можно было запросить с помощью обычного API
hwspin_lock_request_specific().

При успехе функция возвращает номер идентификатора блокировки, -EPROBE_DEFER,
если устройство hwspinlock ещё не зарегистрировано в ядре, или другие
значения ошибок.

Должна вызываться из контекста процесса (может уснуть).

::

  int hwspin_lock_free(struct hwspinlock *hwlock);

Освободить ранее назначенный hwspinlock; возвращает 0 при успехе или
соответствующий код ошибки при неудаче (например, -EINVAL, если hwspinlock
уже свободен).

Должна вызываться из контекста процесса (может уснуть).

::

  int hwspin_lock_bust(struct hwspinlock *hwlock, unsigned int id);

После проверки владельца hwspinlock освободить ранее захваченный
hwspinlock; возвращает 0 при успехе или соответствующий код ошибки при
неудаче (например, -EOPNOTSUPP, если операция bust не определена для данного
hwspinlock).

Должна вызываться из контекста процесса (может уснуть).

::

  int hwspin_lock_timeout(struct hwspinlock *hwlock, unsigned int timeout);

Заблокировать ранее назначенный hwspinlock с ограничением по времени ожидания
(задаётся в миллисекундах). Если hwspinlock уже захвачен, функция будет
выполнять активное ожидание (busy loop) его освобождения, но прекратит, когда
истечёт время ожидания. После успешного возврата из этой функции вытеснение
(preemption) отключено, поэтому вызывающая сторона не должна уходить в сон и
рекомендуется освободить hwspinlock как можно скорее, чтобы минимизировать
опрос удалёнными ядрами аппаратного межсоединения.

Возвращает 0 при успехе и соответствующий код ошибки в противном случае (в
первую очередь -ETIMEDOUT, если hwspinlock всё ещё занят по истечении timeout
миллисекунд). Функция никогда не уходит в сон.

::

  int hwspin_lock_timeout_irq(struct hwspinlock *hwlock, unsigned int timeout);

Заблокировать ранее назначенный hwspinlock с ограничением по времени ожидания
(задаётся в миллисекундах). Если hwspinlock уже захвачен, функция будет
выполнять активное ожидание (busy loop) его освобождения, но прекратит, когда
истечёт время ожидания. После успешного возврата из этой функции вытеснение
и локальные прерывания отключены, поэтому вызывающая сторона не должна уходить
в сон и рекомендуется освободить hwspinlock как можно скорее.

Возвращает 0 при успехе и соответствующий код ошибки в противном случае (в
первую очередь -ETIMEDOUT, если hwspinlock всё ещё занят по истечении timeout
миллисекунд). Функция никогда не уходит в сон.

::

  int hwspin_lock_timeout_irqsave(struct hwspinlock *hwlock, unsigned int to,
				  unsigned long *flags);

Заблокировать ранее назначенный hwspinlock с ограничением по времени ожидания
(задаётся в миллисекундах). Если hwspinlock уже захвачен, функция будет
выполнять активное ожидание (busy loop) его освобождения, но прекратит, когда
истечёт время ожидания. После успешного возврата из этой функции вытеснение
отключено, локальные прерывания отключены, а их предыдущее состояние сохранено

сон и рекомендуется освободить hwspinlock как можно скорее.

Возвращает 0 при успехе и соответствующий код ошибки в противном случае (в
первую очередь -ETIMEDOUT, если hwspinlock всё ещё занят по истечении timeout
миллисекунд).

Функция никогда не уходит в сон.

::

  int hwspin_lock_timeout_raw(struct hwspinlock *hwlock, unsigned int timeout);

Заблокировать ранее назначенный hwspinlock с ограничением по времени ожидания
(задаётся в миллисекундах). Если hwspinlock уже захвачен, функция будет
выполнять активное ожидание (busy loop) его освобождения, но прекратит, когда
истечёт время ожидания.

Внимание: пользователь должен защищать процедуру получения аппаратной
блокировки с помощью mutex или spinlock во избежание взаимоблокировки
(dead-lock), что позволит пользователю выполнять некоторые длительные или
способные уйти в сон операции под аппаратной блокировкой.

Возвращает 0 при успехе и соответствующий код ошибки в противном случае (в
первую очередь -ETIMEDOUT, если hwspinlock всё ещё занят по истечении timeout

  int hwspin_lock_timeout_in_atomic(struct hwspinlock *hwlock, unsigned int to);

Возвращает 0 при успехе и соответствующий код ошибки в противном случае (в
первую очередь -ETIMEDOUT, если hwspinlock всё ещё занят по истечении timeout
миллисекунд).

Функция никогда не уходит в сон.

первую очередь -EBUSY, если hwspinlock уже был захвачен).

Типичное использование
=======================

::

	#include <linux/hwspinlock.h>
	#include <linux/err.h>

	int hwspinlock_example(void)
	{
		struct hwspinlock *hwlock;
		int ret;

		/*
		* assign a specific hwspinlock id - this should be called early
		* by board init code.
		*/
		hwlock = hwspin_lock_request_specific(PREDEFINED_LOCK_ID);
		if (!hwlock)
			...

		/* try to take it, but don't spin on it */
		ret = hwspin_trylock(hwlock);
		if (!ret) {
			pr_info("lock is already taken\n");
			return -EBUSY;
		}

		/*
		* we took the lock, do our thing now, but do NOT sleep
		*/

		/* release the lock */
		hwspin_unlock(hwlock);

		/* free the lock */
		ret = hwspin_lock_free(hwlock);
		if (ret)
			...

		return ret;
	}


API для разработчиков реализаций
================================

::

  int hwspin_lock_register(struct hwspinlock_device *bank, struct device *dev,
		const struct hwspinlock_ops *ops, int base_id, int num_locks);

Вызывается из нижележащей платформенно-специфичной реализации для регистрации
нового устройства hwspinlock (которое обычно представляет собой банк
многочисленных блокировок). Должна вызываться из контекста процесса (эта
функция может уснуть).

Возвращает 0 при успехе или соответствующий код ошибки при неудаче.

::

  int hwspin_lock_unregister(struct hwspinlock_device *bank);

Вызывается из нижележащей реализации, специфичной для производителя, для
отмены регистрации устройства hwspinlock (которое обычно представляет собой
банк многочисленных блокировок).

Должна вызываться из контекста процесса (эта функция может уснуть).

Возвращает адрес hwspinlock при успехе или NULL при ошибке (например,
если hwspinlock всё ещё используется).

Важные структуры
================

struct hwspinlock_device — это устройство, которое обычно содержит банк
аппаратных блокировок. Оно регистрируется нижележащей реализацией hwspinlock
с помощью API hwspin_lock_register().

::

	/**
	* struct hwspinlock_device - a device which usually spans numerous hwspinlocks
	* @dev: underlying device, will be used to invoke runtime PM api
	* @ops: platform-specific hwspinlock handlers
	* @base_id: id index of the first lock in this device
	* @num_locks: number of locks in this device
	* @lock: dynamically allocated array of 'struct hwspinlock'
	*/
	struct hwspinlock_device {
		struct device *dev;
		const struct hwspinlock_ops *ops;
		int base_id;
		int num_locks;
		struct hwspinlock lock[0];
	};

struct hwspinlock_device содержит массив структур hwspinlock, каждая
из которых представляет одну аппаратную блокировку::

	/**
	* struct hwspinlock - this struct represents a single hwspinlock instance
	* @bank: the hwspinlock_device structure which owns this lock
	* @lock: initialized and used by hwspinlock core
	* @priv: private data, owned by the underlying platform-specific hwspinlock drv
	*/
	struct hwspinlock {
		struct hwspinlock_device *bank;
		spinlock_t lock;
		void *priv;
	};

При регистрации банка блокировок драйверу hwspinlock нужно установить только
члены priv этих блокировок. Остальные члены устанавливаются и
инициализируются самим ядром hwspinlock.

Колбэки реализации
==================

В 'struct hwspinlock_ops' определены три возможных колбэка::

	struct hwspinlock_ops {
		int (*trylock)(struct hwspinlock *lock);
		void (*unlock)(struct hwspinlock *lock);
		void (*relax)(struct hwspinlock *lock);
	};

Первые два колбэка являются обязательными:

Колбэк ->trylock() должен делать одну попытку захватить блокировку и
возвращать 0 при неудаче и 1 при успехе. Этот колбэк **не** имеет права уходить в сон.

Колбэк ->unlock() освобождает блокировку. Он всегда завершается успешно и
также **не** имеет права уходить в сон.

Колбэк ->relax() является необязательным. Он вызывается ядром hwspinlock во
время активного ожидания (spinning) на блокировке и может использоваться
нижележащей реализацией для принудительной задержки между двумя
последовательными вызовами ->trylock(). Он **не** имеет права уходить в сон.
