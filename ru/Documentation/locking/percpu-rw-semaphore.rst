=========================
Percpu rw-семафоры
=========================

Percpu rw-семафоры (percpu read-write semaphores) — это новая конструкция
read-write семафора, оптимизированная для блокировки на чтение.

Проблема традиционных read-write семафоров заключается в том, что когда
несколько ядер берут блокировку на чтение, строка кэша, содержащая семафор,
«перебрасывается» между кэшами L1 этих ядер, вызывая снижение
производительности.

Блокировка на чтение выполняется очень быстро, она использует RCU и избегает
любых атомарных инструкций в путях блокировки и разблокировки. С другой стороны,
блокировка на запись очень затратна, она вызывает synchronize_rcu(), что может
занять сотни миллисекунд.

Блокировка объявляется с типом "struct percpu_rw_semaphore".
Блокировка инициализируется с помощью percpu_init_rwsem, которая возвращает 0
при успехе и -ENOMEM при ошибке выделения памяти.
Блокировка должна быть освобождена с помощью percpu_free_rwsem во избежание
утечки памяти.

Блокировка на чтение выполняется с помощью percpu_down_read, percpu_up_read,
а на запись — с помощью percpu_down_write, percpu_up_write.
