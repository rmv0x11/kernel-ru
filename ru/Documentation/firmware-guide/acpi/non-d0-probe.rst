=====================================================
Зондирование устройств в состояниях D, отличных от D0
=====================================================

Введение
========

В некоторых случаях бывает предпочтительно оставить определённые устройства
выключенными на протяжении всей загрузки системы, если включение этих устройств
влечёт за собой нежелательные побочные эффекты, выходящие за рамки простого
включения данного устройства.

Как это работает
================

Объект _DSC (Device State for Configuration), вычисляемый в целое число, может
использоваться, чтобы сообщить Linux наивысшее допустимое состояние D для
устройства во время зондирования. Поддержка _DSC требует поддержки со стороны
типа шины ядра, если драйвер шины обычно переводит устройство в состояние D0 для
зондирования.

Недостаток использования _DSC в том, что, поскольку устройство не включено, даже
при наличии проблемы с устройством драйвер, скорее всего, успешно завершит
зондирование, но первый же пользователь обнаружит, что устройство не работает,
вместо того чтобы получить сбой на этапе зондирования. Поэтому эту возможность
следует использовать с осторожностью.

I²C
---

Если драйвер I²C сообщает о поддержке этого, устанавливая флаг
I2C_DRV_ACPI_WAIVE_D0_PROBE в поле struct i2c_driver.flags, и объект _DSC
вычисляется в целое число, большее, чем состояние D устройства, то устройство не
будет включено (переведено в состояние D0) для зондирования.

Состояния D
-----------

Состояния D, а значит и допустимые значения для _DSC, перечислены ниже.
Дополнительную информацию о состояниях питания устройств см. в [1].

.. code-block:: text

	Number	State	Description
	0	D0	Device fully powered on
	1	D1
	2	D2
	3	D3hot
	4	D3cold	Off

Ссылки
======

[1] https://uefi.org/specifications/ACPI/6.4/02_Definition_of_Terms/Definition_of_Terms.html#device-power-state-definitions

Пример
======

Пример на ASL, описывающий ACPI-устройство, которое с помощью объекта _DSC
сообщает операционной системе, что устройство должно оставаться выключенным во
время зондирования, выглядит следующим образом. Некоторые объекты, не имеющие
значения с точки зрения данного примера, опущены.

.. code-block:: text

	Device (CAM0)
	{
		Name (_HID, "SONY319A")
		Name (_UID, Zero)
		Name (_CRS, ResourceTemplate ()
		{
			I2cSerialBus(0x0020, ControllerInitiated, 0x00061A80,
				     AddressingMode7Bit, "\\_SB.PCI0.I2C0",
				     0x00, ResourceConsumer)
		})
		Method (_DSC, 0, NotSerialized)
		{
			Return (0x4)
		}
	}
