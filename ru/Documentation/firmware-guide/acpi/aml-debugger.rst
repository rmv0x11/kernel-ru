==============
Отладчик AML
==============

:Copyright: |copy| 2016, Intel Corporation
:Author: Lv Zheng <lv.zheng@intel.com>


Этот документ описывает использование отладчика AML, встроенного в ядро Linux.

1. Сборка отладчика
===================

Для включения интерфейса отладчика AML из ядра Linux требуются следующие
параметры конфигурации ядра::

   CONFIG_ACPI_DEBUGGER=y
   CONFIG_ACPI_DEBUGGER_USER=m

Утилиты пространства пользователя можно собрать из дерева исходного кода ядра
с помощью следующих команд::

   $ cd tools
   $ make acpi

Полученный двоичный файл инструмента пространства пользователя располагается
по пути::

   tools/power/acpi/acpidbg

Его можно установить в системные каталоги, выполнив "make install" (от имени
пользователя с достаточными привилегиями).

2. Запуск интерфейса отладчика в пространстве пользователя
==========================================================

После загрузки ядра со встроенным отладчиком отладчик можно запустить с помощью
следующих команд::

   # mount -t debugfs none /sys/kernel/debug
   # modprobe acpi_dbg
   # tools/power/acpi/acpidbg

Это порождает интерактивную среду отладчика AML, в которой можно выполнять
команды отладчика.

Команды описаны в "ACPICA Overview and Programmer Reference", которую можно
загрузить с

https://acpica.org/documentation

Подробный справочник по командам отладчика находится в главе 12 "ACPICA
Debugger Reference".  Для быстрой справки можно использовать команду "help".

3. Остановка интерфейса отладчика в пространстве пользователя
=============================================================

Интерактивный интерфейс отладчика можно закрыть, нажав Ctrl+C или используя
команды "quit" или "exit".  По завершении выгрузите модуль с помощью::

   # rmmod acpi_dbg

Выгрузка модуля может завершиться неудачей, если запущен экземпляр acpidbg.

4. Запуск отладчика в скрипте
=============================

Может быть полезно запускать отладчик AML в тестовом скрипте. "acpidbg"
поддерживает это в специальном "пакетном" (batch) режиме.  Например, следующая
команда выводит всё пространство имён ACPI::

   # acpidbg -b "namespace"
