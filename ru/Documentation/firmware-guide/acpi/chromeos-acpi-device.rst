=========================
ACPI-устройство Chrome OS
=========================

Аппаратная функциональность, специфичная для Chrome OS, предоставляется через ACPI-устройство Chrome OS.
Plug and play ID ACPI-устройства Chrome OS — GGL0001, а hardware ID —
GOOG0016.  Поддерживаются следующие ACPI-объекты:

.. flat-table:: Поддерживаемые ACPI-объекты
   :widths: 1 2
   :header-rows: 1

   * - Объект
     - Описание

   * - CHSW
     - Положения переключателей Chrome OS

   * - HWID
     - Hardware ID Chrome OS

   * - FWID
     - Версия прошивки Chrome OS

   * - FRID
     - Версия прошивки Chrome OS, доступной только для чтения

   * - BINF
     - Информация о загрузке Chrome OS

   * - GPIO
     - Назначения GPIO Chrome OS

   * - VBNV
     - Расположения NVRAM Chrome OS

   * - VDTA
     - Данные проверенной загрузки Chrome OS

   * - FMAP
     - Базовый адрес flashmap Chrome OS

   * - MLST
     - Список методов Chrome OS

CHSW (положения переключателей Chrome OS)
=========================================
Этот управляющий метод возвращает положения переключателей для специфичных для Chrome OS аппаратных переключателей.

Аргументы:
----------
Отсутствуют

Код результата:
---------------
Целое число, содержащее положения переключателей в виде битовых полей:

.. flat-table::
   :widths: 1 2

   * - 0x00000002
     - Кнопка восстановления была нажата при загрузке прошивки x86.

   * - 0x00000004
     - Кнопка восстановления была нажата при загрузке прошивки EC. (обязательно, если EEPROM EC
       перезаписываемый; в противном случае необязательно)

   * - 0x00000020
     - Переключатель режима разработчика был включён при загрузке прошивки x86.

   * - 0x00000200
     - Защита прошивки от записи была отключена при загрузке прошивки x86. (обязательно, если
       защита прошивки от записи управляется через BIOS x86; в противном случае необязательно)

Все остальные биты зарезервированы и должны быть установлены в 0.

HWID (hardware ID Chrome OS)
============================
Этот управляющий метод возвращает hardware ID для Chromebook.

Аргументы:
----------
Отсутствуют

Код результата:
---------------
Завершающаяся нулём ASCII-строка, содержащая hardware ID из области Model-Specific Data
EEPROM.

Обратите внимание, что hardware ID может иметь длину до 256 символов, включая завершающий нуль.

FWID (версия прошивки Chrome OS)
================================
Этот управляющий метод возвращает версию прошивки для перезаписываемой части прошивки
основного процессора.

Аргументы:
----------
Отсутствуют

Код результата:
---------------
Завершающаяся нулём ASCII-строка, содержащая полную версию прошивки для перезаписываемой
части прошивки основного процессора.

FRID (версия прошивки Chrome OS, доступной только для чтения)
=============================================================
Этот управляющий метод возвращает версию прошивки для доступной только для чтения части прошивки
основного процессора.

Аргументы:
----------
Отсутствуют

Код результата:
---------------
Завершающаяся нулём ASCII-строка, содержащая полную версию прошивки для доступной только для чтения
(bootstrap + recovery) части прошивки основного процессора.

BINF (информация о загрузке Chrome OS)
======================================
Этот управляющий метод возвращает информацию о текущей загрузке.

Аргументы:
----------
Отсутствуют

Код результата:
---------------

.. code-block::

   Package {
           Reserved1
           Reserved2
           Active EC Firmware
           Active Main Firmware Type
           Reserved5
   }

.. flat-table::
   :widths: 1 1 2
   :header-rows: 1

   * - Поле
     - Формат
     - Описание

   * - Reserved1
     - DWORD
     - Установлено в 256 (0x100). Это указывает, что данное поле больше не используется.

   * - Reserved2
     - DWORD
     - Установлено в 256 (0x100). Это указывает, что данное поле больше не используется.

   * - Active EC firmware
     - DWORD
     - Прошивка EC, которая использовалась во время загрузки.

       - 0 - Прошивка, доступная только для чтения (recovery)
       - 1 - Перезаписываемая прошивка.

       Установлено в 0, если прошивка EC всегда доступна только для чтения.

   * - Active Main Firmware Type
     - DWORD
     - Тип основной прошивки, которая использовалась во время загрузки.

       - 0 - Recovery
       - 1 - Normal
       - 2 - Developer
       - 3 - netboot (только заводская установка)

       Остальные значения зарезервированы.

   * - Reserved5
     - DWORD
     - Установлено в 256 (0x100). Это указывает, что данное поле больше не используется.

GPIO (назначения GPIO Chrome OS)
================================
Этот управляющий метод возвращает информацию о специфичных для Chrome OS назначениях GPIO для
аппаратного обеспечения Chrome OS, чтобы ядро могло напрямую управлять этим оборудованием.

Аргументы:
----------
Отсутствуют

Код результата:
---------------
.. code-block::

        Package {
                Package {
                        // First GPIO assignment
                        Signal Type        //DWORD
                        Attributes         //DWORD
                        Controller Offset  //DWORD
                        Controller Name    //ASCIIZ
                },
                ...
                Package {
                        // Last GPIO assignment
                        Signal Type        //DWORD
                        Attributes         //DWORD
                        Controller Offset  //DWORD
                        Controller Name    //ASCIIZ
                }
        }

Где ASCIIZ означает завершающуюся нулём ASCII-строку.

.. flat-table::
   :widths: 1 1 2
   :header-rows: 1

   * - Поле
     - Формат
     - Описание

   * - Signal Type
     - DWORD
     - Тип сигнала GPIO

       - 0x00000001 - Кнопка восстановления
       - 0x00000002 - Переключатель режима разработчика
       - 0x00000003 - Переключатель защиты прошивки от записи
       - 0x00000100 - Отладочный разъём GPIO 0
       - ...
       - 0x000001FF - Отладочный разъём GPIO 255

       Остальные значения зарезервированы.

   * - Attributes
     - DWORD
     - Атрибуты сигнала в виде битовых полей:

       - 0x00000001 - Сигнал активен по высокому уровню (для кнопки значение GPIO,
         равное 1, означает, что кнопка нажата; для переключателей значение GPIO,
         равное 1, означает, что переключатель включён). Если этот бит равен 0, сигнал
         активен по низкому уровню. Установлено в 0 для GPIO отладочного разъёма.

   * - Controller Offset
     - DWORD
     - Номер GPIO на указанном контроллере.

   * - Controller Name
     - ASCIIZ
     - Имя контроллера для данного GPIO.
       В настоящее время поддерживаемые имена:
       "NM10" - чип Intel NM10

VBNV (расположения NVRAM Chrome OS)
===================================
Этот управляющий метод возвращает информацию о расположениях NVRAM (CMOS), используемых для
взаимодействия с BIOS.

Аргументы:
----------
Отсутствуют

Код результата:
---------------
.. code-block::

        Package {
                NV Storage Block Offset  //DWORD
                NV Storage Block Size    //DWORD
        }

.. flat-table::
   :widths: 1 1 2
   :header-rows: 1

   * - Поле
     - Формат
     - Описание

   * - NV Storage Block Offset
     - DWORD
     - Смещение в банке 0 CMOS блока энергонезависимого хранилища проверенной загрузки, отсчитываемое
       от первого доступного для записи байта CMOS (то есть offset=0 — это байт, следующий за 14 байтами
       данных часов).

   * - NV Storage Block Size
     - DWORD
     - Размер в байтах блока энергонезависимого хранилища проверенной загрузки.

FMAP (адрес flashmap Chrome OS)
===============================
Этот управляющий метод возвращает физический адрес памяти начала flashmap прошивки основного
процессора.

Аргументы:
----------
Отсутствуют

NoneResult code:
----------------
DWORD, содержащий физический адрес памяти начала flashmap прошивки основного
процессора.

VDTA (данные проверенной загрузки Chrome OS)
============================================
Этот управляющий метод возвращает блок данных проверенной загрузки, общий для этапа
верификации прошивки и этапа верификации ядра.

Аргументы:
----------
Отсутствуют

Код результата:
---------------
Буфер, содержащий блок данных проверенной загрузки.

MECK (контрольная сумма Management Engine)
==========================================
Этот управляющий метод возвращает хеш SHA-1 или SHA-256, считываемый из расширенных регистров
Management Engine во время загрузки. Хеш экспортируется через ACPI, чтобы ОС могла проверить, что
прошивка ME не изменилась. Если Management Engine отсутствует или если прошивка не смогла
прочитать расширенные регистры, этот буфер может быть нулевым.

Аргументы:
----------
Отсутствуют

Код результата:
---------------
Буфер, содержащий хеш ME.

MLST (список методов Chrome OS)
===============================
Этот управляющий метод возвращает список других управляющих методов, поддерживаемых аппаратным
устройством Chrome OS.

Аргументы:
----------
Отсутствуют

Код результата:
---------------
Package, содержащий список завершающихся нулём ASCII-строк, по одной для каждого управляющего метода,
поддерживаемого аппаратным устройством Chrome OS, не включая сам метод MLST.
Для этой версии спецификации результат таков:

.. code-block::

        Package {
                "CHSW",
                "FWID",
                "HWID",
                "FRID",
                "BINF",
                "GPIO",
                "VBNV",
                "FMAP",
                "VDTA",
                "MECK"
        }
