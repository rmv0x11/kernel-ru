_DSD
====

_DSD (Device Specific Data) [dsd-guide] — это предопределённый объект
конфигурации ACPI-устройства, который может использоваться для передачи
информации об аппаратных возможностях, специально не охваченных
спецификацией ACPI [acpi]. Существуют два расширения _DSD, релевантных
для графов: расширение свойств (property) [dsd-guide] и расширение
иерархических данных. Расширение свойств предоставляет обобщённые пары
«ключ-значение», тогда как расширение иерархических данных поддерживает
узлы со ссылками на другие узлы, образуя дерево. Узлы дерева могут
содержать свойства, определяемые расширением свойств. Вместе эти два
расширения дают древовидную структуру с нулём или более свойствами
(парами «ключ-значение») в каждом узле дерева.

Доступ к структуре данных может выполняться во время работы с помощью
функций device_* и fwnode_*, определённых в include/linux/fwnode.h .

Fwnode представляет обобщённый объект узла прошивки. Он не зависит от типа
прошивки. В ACPI fwnode-ы — это объекты расширений иерархических данных
_DSD. Объект _DSD устройства представлен fwnode-ом.

На структуру данных можно сослаться из других мест таблиц ACPI, используя
жёсткую ссылку на само устройство и индекс в массиве расширений
иерархических данных на каждом уровне глубины.


Порты и точки подключения
=========================

Концепции порта (port) и точки подключения (endpoint) очень близки к
аналогичным концепциям в Devicetree [devicetree, graph-bindings]. Порт
представляет интерфейс устройства, а точка подключения представляет
соединение с этим интерфейсом. См. также [data-node-ref] для обобщённых
ссылок на узлы данных.

Все узлы портов располагаются под узлом "_DSD" устройства в дереве
расширения иерархических данных. Расширение данных, относящееся к каждому
узлу порта, должно начинаться с "port", за которым должны следовать символ
"@" и номер порта в качестве его ключа. Целевой объект, на который оно
ссылается, должен называться "PRTX", где "X" — номер порта. Пример такого
package выглядел бы так::

    Package() { "port@4", "PRT4" }

Далее, точки подключения располагаются под узлами портов. Ключ расширения
иерархических данных узлов точек подключения должен начинаться с
"endpoint", за которым должны следовать символ "@" и номер точки
подключения. Объект, на который оно ссылается, должен называться "EPXY",
где "X" — номер порта, а "Y" — номер точки подключения. Пример такого
package выглядел бы так::

    Package() { "endpoint@0", "EP40" }

Каждый узел порта содержит ключ расширения свойств "port", значение
которого — номер порта. Каждая точка подключения аналогичным образом
нумеруется ключом расширения свойств "reg", значение которого — номер
точки подключения. Номера портов должны быть уникальными в пределах
устройства, а номера точек подключения должны быть уникальными в пределах
порта. Если объект устройства может иметь только один порт, то номер этого
порта должен быть равен нулю. Аналогично, если порт может иметь только одну
точку подключения, номер этой точки подключения должен быть равен нулю.

Ссылка на точку подключения использует расширение свойств с именем свойства
"remote-endpoint", за которым следует строковая ссылка в том же package.
[data-node-ref]::

    "device.datanode"

В приведённом выше примере "X" — это номер порта, а "Y" — номер точки
подключения.

Ссылки на точки подключения всегда должны устанавливаться в обе стороны: на
удалённую точку подключения и обратно от узла удалённой точки подключения,
на которую ссылаются.

Простой пример этого показан ниже::

    Scope (\_SB.PCI0.I2C2)
    {
	Device (CAM0)
	{
	    Name (_DSD, Package () {
		ToUUID("daffd814-6eba-4d8c-8a91-bc9bbf4aa301"),
		Package () {
		    Package () { "compatible", Package () { "nokia,smia" } },
		},
		ToUUID("dbb8e3e6-5886-4ba6-8795-1319f52a966b"),
		Package () {
		    Package () { "port@0", "PRT0" },
		}
	    })
	    Name (PRT0, Package() {
		ToUUID("daffd814-6eba-4d8c-8a91-bc9bbf4aa301"),
		Package () {
		    Package () { "reg", 0 },
		},
		ToUUID("dbb8e3e6-5886-4ba6-8795-1319f52a966b"),
		Package () {
		    Package () { "endpoint@0", "EP00" },
		}
	    })
	    Name (EP00, Package() {
		ToUUID("daffd814-6eba-4d8c-8a91-bc9bbf4aa301"),
		Package () {
		    Package () { "reg", 0 },
		    Package () { "remote-endpoint", "\\_SB.PCI0.ISP.EP40" },
		}
	    })
	}
    }

    Scope (\_SB.PCI0)
    {
	Device (ISP)
	{
	    Name (_DSD, Package () {
		ToUUID("dbb8e3e6-5886-4ba6-8795-1319f52a966b"),
		Package () {
		    Package () { "port@4", "PRT4" },
		}
	    })

	    Name (PRT4, Package() {
		ToUUID("daffd814-6eba-4d8c-8a91-bc9bbf4aa301"),
		Package () {
		    Package () { "reg", 4 }, /* CSI-2 port number */
		},
		ToUUID("dbb8e3e6-5886-4ba6-8795-1319f52a966b"),
		Package () {
		    Package () { "endpoint@0", "EP40" },
		}
	    })

	    Name (EP40, Package() {
		ToUUID("daffd814-6eba-4d8c-8a91-bc9bbf4aa301"),
		Package () {
		    Package () { "reg", 0 },
		    Package () { "remote-endpoint", "\\_SB.PCI0.I2C2.CAM0.EP00" },
		}
	    })
	}
    }

Здесь порт 0 устройства "CAM0" соединён с портом 4 устройства "ISP" и
наоборот.
