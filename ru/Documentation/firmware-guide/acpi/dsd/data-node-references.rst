=========================================================
Ссылки на иерархические узлы данных
=========================================================

:Copyright: |copy| 2018, 2021 Intel Corporation
:Author: Sakari Ailus <sakari.ailus@linux.intel.com>

ACPI в общем случае позволяет ссылаться только на объекты устройств в дереве.
На узлы иерархического расширения данных (hierarchical data extension nodes)
напрямую ссылаться нельзя, поэтому данный документ определяет схему реализации
таких ссылок.

Ссылка на иерархический узел данных _DSD — это строка, состоящая из ссылки на
объект устройства, за которой следует точка (".") и относительный путь к объекту
узла данных. Не используйте нестроковые ссылки, так как это приведёт к
созданию копии иерархического узла данных, а не ссылки на него!

Узел иерархического расширения данных, на который ссылаются, должен располагаться
непосредственно под своим родительским объектом, то есть либо под объектом
устройства, либо под другим узлом иерархического расширения данных [dsd-guide].

Ключи в иерархических узлах данных должны состоять из имени узла, символа "@" и
номера узла в шестнадцатеричной записи (без префиксов или постфиксов). Тот же
объект ACPI должен включать расширение свойств _DSD со свойством "reg", которое
должно иметь то же числовое значение, что и номер узла.

В случае, когда узел иерархического расширения данных не имеет числового значения,
свойство "reg" должно быть опущено из свойств _DSD объекта ACPI, а символ "@" и
номер должны быть опущены из ключа иерархического расширения данных.


Пример
======

В приведённом ниже фрагменте ASL свойство _DSD "reference" содержит строковую
ссылку на узел иерархического расширения данных ANOD, находящийся под DEV0 под
родителем DEV1. ANOD также является конечным целевым узлом этой ссылки.
::

	Device (DEV0)
	{
	    Name (_DSD, Package () {
		ToUUID("dbb8e3e6-5886-4ba6-8795-1319f52a966b"),
		Package () {
		    Package () { "node@0", "NOD0" },
		    Package () { "node@1", "NOD1" },
		}
	    })
	    Name (NOD0, Package() {
		ToUUID("daffd814-6eba-4d8c-8a91-bc9bbf4aa301"),
		Package () {
		    Package () { "reg", 0 },
		    Package () { "random-property", 3 },
		}
	    })
	    Name (NOD1, Package() {
		ToUUID("dbb8e3e6-5886-4ba6-8795-1319f52a966b"),
		Package () {
		    Package () { "reg", 1 },
		    Package () { "anothernode", "ANOD" },
		}
	    })
	    Name (ANOD, Package() {
		ToUUID("daffd814-6eba-4d8c-8a91-bc9bbf4aa301"),
		Package () {
		    Package () { "random-property", 0 },
		}
	    })
	}

	Device (DEV1)
	{
	    Name (_DSD, Package () {
		ToUUID("daffd814-6eba-4d8c-8a91-bc9bbf4aa301"),
		Package () {
		    Package () { "reference", "^DEV0.ANOD" }
		    },
		}
	    })
	}

См. также пример с графом в
Documentation/firmware-guide/acpi/dsd/graph.rst.
