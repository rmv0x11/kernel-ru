Sparse
======

Sparse — это семантический анализатор для программ на C; его можно
использовать для поиска ряда потенциальных проблем в коде ядра. Обзор
sparse см. на https://lwn.net/Articles/689907/; данный документ содержит
некоторую информацию о sparse, специфичную для ядра.
Дополнительные сведения о sparse, в основном о его внутреннем устройстве,
можно найти на его официальных страницах по адресу
https://sparse.docs.kernel.org.


Использование sparse для проверки типов
---------------------------------------

"__bitwise" — это атрибут типа, поэтому нужно написать что-то вроде этого::

        typedef int __bitwise pm_request_t;

        enum pm_request {
                PM_SUSPEND = (__force pm_request_t) 1,
                PM_RESUME = (__force pm_request_t) 2
        };

что делает PM_SUSPEND и PM_RESUME "побитовыми" (bitwise) целыми числами
(атрибут "__force" присутствует здесь потому, что sparse будет жаловаться
на приведение к побитовому типу и обратно, но в данном случае мы
действительно _хотим_ принудительно выполнить преобразование). А поскольку
все значения enum имеют один и тот же тип, теперь "enum pm_request" тоже
будет иметь этот тип.

А при использовании gcc все эти "__bitwise"/"__force"-конструкции
исчезают, и для gcc всё в итоге выглядит просто как целые числа.

Откровенно говоря, enum здесь не нужен. Всё вышесказанное на самом деле
сводится к одному специальному типу "int __bitwise".

Поэтому проще всего просто сделать так::

        typedef int __bitwise pm_request_t;

        #define PM_SUSPEND ((__force pm_request_t) 1)
        #define PM_RESUME ((__force pm_request_t) 2)

и теперь у вас есть вся инфраструктура, необходимая для строгой проверки
типов.

Небольшое замечание: целочисленная константа "0" является особой. Можно
использовать константный ноль как побитовый целый тип, и sparse никогда
не будет на это жаловаться. Это связано с тем, что "bitwise" (как
следует из названия) был разработан для того, чтобы гарантировать, что
побитовые типы не смешиваются между собой (little-endian против
big-endian против cpu-endian против чего угодно), и там константа "0"
действительно _является_ особой.

Получение sparse
----------------

Tarball-архивы последних выпущенных версий можно получить с:
https://www.kernel.org/pub/software/devel/sparse/dist/

Кроме того, можно получить снимки последней разрабатываемой версии sparse,
склонировав её с помощью git::

        git://git.kernel.org/pub/scm/devel/sparse/sparse.git

После того как вы её получите, просто выполните::

        make
        make install

от имени обычного пользователя, и sparse будет установлен в ваш каталог
~/bin.

Использование sparse
---------------------

Выполните сборку ядра с помощью "make C=1", чтобы запустить sparse на всех
файлах C, которые перекомпилируются, или используйте "make C=2", чтобы
запустить sparse на файлах независимо от того, нужно их перекомпилировать
или нет. Последнее — быстрый способ проверить всё дерево, если вы его уже
собрали.

Необязательную make-переменную CF можно использовать для передачи
аргументов в sparse. Система сборки автоматически передаёт -Wbitwise в
sparse.

Обратите внимание, что sparse определяет символ препроцессора __CHECKER__.
