Дополнительная информация
=========================

Существует множество источников информации по разработке ядра Linux и
смежным темам. Первым среди них всегда будет каталог Documentation,
находящийся в дистрибутиве исходного кода ядра. Начните с
:ref:`process/howto.rst <process_howto>` верхнего уровня; также прочитайте
:ref:`process/submitting-patches.rst <submittingpatches>`. Многие внутренние
API ядра документированы с помощью механизма kerneldoc; для генерации этих
документов в формате HTML или PDF можно использовать "make htmldocs" или
"make pdfdocs" (хотя версия TeX, поставляемая некоторыми дистрибутивами,
упирается во внутренние ограничения и не может корректно обработать
документы).

Различные веб-сайты обсуждают разработку ядра на всех уровнях детализации.
Автор позволит себе скромно предложить в качестве источника https://lwn.net/;
информацию по многим конкретным темам ядра можно найти через индекс ядра LWN
по адресу:

	https://lwn.net/Kernel/Index/

Кроме того, ценным ресурсом для разработчиков ядра является:

	https://kernelnewbies.org/

И, конечно, не следует забывать про https://kernel.org/ — авторитетный
источник информации о релизах ядра.

Существует ряд книг по разработке ядра:

	Linux Device Drivers, 3rd Edition (Jonathan Corbet, Alessandro
	Rubini, and Greg Kroah-Hartman). Доступна онлайн по адресу
	https://lwn.net/Kernel/LDD3/.

	Linux Kernel Development (Robert Love).

	Understanding the Linux Kernel (Daniel Bovet and Marco Cesati).

Однако все эти книги страдают общим недостатком: они склонны быть несколько
устаревшими к моменту, когда попадают на полки, а на полках они находятся уже
довольно долго. Тем не менее в них можно найти немало полезной информации.

Документацию по git можно найти по адресам:

	https://www.kernel.org/pub/software/scm/git/docs/

	https://www.kernel.org/pub/software/scm/git/docs/user-manual.html


Заключение
==========

Поздравляем всех, кто осилил этот пространный документ. Будем надеяться, что
он дал полезное понимание того, как разрабатывается ядро Linux и как вы можете
участвовать в этом процессе.

В конечном счёте важно именно участие. Любой проект программного обеспечения
с открытым исходным кодом представляет собой не более чем сумму того, что
вкладывают в него его участники. Ядро Linux развивалось столь быстро и
успешно, как оно есть, потому что ему помогала впечатляюще большая группа
разработчиков, каждый из которых работает над тем, чтобы сделать его лучше.
Ядро — выдающийся пример того, чего можно достичь, когда тысячи людей работают
вместе ради общей цели.

Тем не менее, ядро всегда выиграет от более широкой базы разработчиков. Работы
всегда хватает. Но, что не менее важно, большинство других участников
экосистемы Linux могут получить выгоду, внося вклад в ядро. Попадание кода в
mainline (основную ветку) — ключ к более высокому качеству кода, более низким
затратам на сопровождение и распространение, более высокому уровню влияния на
направление развития ядра и многому другому. Это ситуация, в которой
выигрывают все причастные. Запускайте свой редактор и присоединяйтесь к нам;
вам будут более чем рады.
