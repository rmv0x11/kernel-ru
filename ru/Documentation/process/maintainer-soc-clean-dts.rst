=========================================================
Платформы SoC с требованиями к соответствию DTS
=========================================================

Обзор
-----

Платформы SoC или субархитектуры должны следовать всем правилам из
Documentation/process/maintainer-soc.rst.  Настоящий документ, на который
ссылается MAINTAINERS, налагает дополнительные требования, перечисленные ниже.

Строгое соответствие DTS схеме DT и dtc
---------------------------------------

Никакие изменения в исходных файлах Devicetree платформы SoC (файлы DTS) не
должны вносить новые предупреждения ``make dtbs_check W=1``.  Предупреждения в
DTS новой платы, которые являются результатом проблем во включённом файле DTSI,
считаются существующими, а не новыми предупреждениями.  Для серий, разделённых
между разными деревьями (привязки DT идут через дерево подсистемы драйвера),
определяющими являются предупреждения в linux-next.  У сопровождающих платформы
имеется автоматизация, которая должна указывать на любые новые предупреждения.

Если коммит, вносящий новые предупреждения, всё же оказался принят, возникшие
проблемы должны быть исправлены в разумный срок (например, в течение одного
релиза) либо коммит должен быть откачен.
