ИИ-помощники для написания кода
+++++++++++++++++++++++++++++++

Этот документ содержит рекомендации для ИИ-инструментов и разработчиков,
использующих помощь ИИ при внесении вклада в ядро Linux.

ИИ-инструменты, помогающие в разработке ядра Linux, должны следовать
стандартному процессу разработки ядра:

* Documentation/process/development-process.rst
* Documentation/process/coding-style.rst
* Documentation/process/submitting-patches.rst

Лицензионные и юридические требования
=====================================

Все вклады должны соответствовать лицензионным требованиям ядра:

* Весь код должен быть совместим с GPL-2.0-only
* Используйте соответствующие идентификаторы лицензий SPDX
* Подробности см. в Documentation/process/license-rules.rst

Signed-off-by и Developer Certificate of Origin
===============================================

ИИ-агенты НЕ ДОЛЖНЫ добавлять теги Signed-off-by. Только люди могут на
законных основаниях удостоверять Developer Certificate of Origin (DCO).
Человек, отправляющий вклад, несёт ответственность за:

* Проверку всего кода, сгенерированного ИИ
* Обеспечение соответствия лицензионным требованиям
* Добавление собственного тега Signed-off-by для удостоверения DCO
* Принятие полной ответственности за вклад

Атрибуция
=========

Когда ИИ-инструменты вносят вклад в разработку ядра, правильная атрибуция
помогает отслеживать меняющуюся роль ИИ в процессе разработки.
Вклады должны включать тег Assisted-by в следующем формате::

  Assisted-by: AGENT_NAME:MODEL_VERSION [TOOL1] [TOOL2]

Где:

* ``AGENT_NAME`` — имя ИИ-инструмента или фреймворка
* ``MODEL_VERSION`` — конкретная использованная версия модели
* ``[TOOL1] [TOOL2]`` — необязательные специализированные инструменты анализа
  (например, coccinelle, sparse, smatch, clang-tidy)

Базовые инструменты разработки (git, gcc, make, редакторы) указывать не следует.

Пример::

  Assisted-by: Claude:claude-3-opus coccinelle sparse
