==================
Control Groupstats
==================

Control Groupstats навеяна обсуждением по адресу
https://lore.kernel.org/r/461CF883.2030308@sw.ru и реализует статистику по каждой
cgroup, как было предложено Эндрю Мортоном (Andrew Morton) в
https://lore.kernel.org/r/20070411114927.1277d7c9.akpm@linux-foundation.org.

Инфраструктура статистики по каждой cgroup повторно использует код интерфейса
taskstats. Регистрируется новый набор операций cgroup с командами и атрибутами,
специфичными для cgroup. Расширять статистику по каждой cgroup должно быть очень
легко — достаточно добавить элементы в структуру cgroupstats.

Текущая модель cgroupstats — это извлечение (pull); модель проталкивания (push)
(для публикации статистики по интересующим событиям) должно быть очень легко
добавить. В настоящее время пространство пользователя запрашивает статистику,
передавая путь к cgroup. Статистика о состоянии всех задач в cgroup возвращается
в пространство пользователя.

ПРИМЕЧАНИЕ: В настоящее время для извлечения информации о задачах, заблокированных
на операциях I/O, мы полагаемся на учёт задержек (delay accounting). Если
CONFIG_TASK_DELAY_ACCT отключён, эта информация будет недоступна.

Для извлечения статистики cgroup была разработана утилита, очень похожая на
getdelays.c; пример вывода этой утилиты приведён ниже::

  ~/balbir/cgroupstats # ./getdelays  -C "/sys/fs/cgroup/a"
  sleeping 1, blocked 0, running 1, stopped 0, uninterruptible 0
  ~/balbir/cgroupstats # ./getdelays  -C "/sys/fs/cgroup"
  sleeping 155, blocked 0, running 1, stopped 0, uninterruptible 2
