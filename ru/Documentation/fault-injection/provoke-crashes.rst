==========================================================================
Провоцирование сбоев с помощью Linux Kernel Dump Test Module (LKDTM)
==========================================================================

Модуль lkdtm предоставляет интерфейс для нарушения работы (и обычно для
аварийного завершения) ядра в заранее определённых местах кода, чтобы
оценить надёжность обработки исключений в ядре и протестировать аварийные
дампы, полученные с помощью различных решений для создания дампов. Модуль
использует KPROBE для инструментирования места срабатывания триггера, но
также может вызывать срабатывание в ядре напрямую без поддержки KPROBE
через debugfs.

Вы можете выбрать местоположение триггера («имя точки сбоя») и тип действия
(«тип точки сбоя») либо через аргументы модуля при его загрузке, либо через
интерфейс debugfs.

Использование::

	insmod lkdtm.ko [recur_count={>0}] cpoint_name=<> cpoint_type=<>
			[cpoint_count={>0}]

recur_count
	Уровень рекурсии для теста переполнения стека. По умолчанию это
	значение вычисляется динамически на основе конфигурации ядра с
	целью сделать его достаточно большим, чтобы исчерпать стек ядра.
	Значение можно увидеть в `/sys/module/lkdtm/parameters/recur_count`.

cpoint_name
	Место в ядре, где следует вызвать действие. Это может быть одно из
	INT_HARDWARE_ENTRY, INT_HW_IRQ_EN, INT_TASKLET_ENTRY,
	FS_SUBMIT_BH, MEM_SWAPOUT, TIMERADD, SCSI_QUEUE_RQ или DIRECT.

cpoint_type
	Указывает действие, которое нужно выполнить при достижении точки
	сбоя. Таких действий много, и их лучше всего запрашивать напрямую
	из debugfs. Некоторые из распространённых — PANIC, BUG, EXCEPTION,
	LOOP и OVERFLOW. Полный список см. в содержимом
	`/sys/kernel/debug/provoke-crash/DIRECT`.

cpoint_count
	Указывает количество раз, которое точка сбоя должна быть достигнута
	перед запуском действия. По умолчанию равно 10 (за исключением
	DIRECT, который всегда срабатывает немедленно).

Вы также можете вызвать сбои, смонтировав debugfs и записав тип в
<debugfs>/provoke-crash/<crashpoint>. Например::

  mount -t debugfs debugfs /sys/kernel/debug
  echo EXCEPTION > /sys/kernel/debug/provoke-crash/INT_HARDWARE_ENTRY

Специальный файл `DIRECT` вызовет действие напрямую, без инструментирования
KPROBE. Этот режим — единственный доступный, когда модуль собран для ядра
без поддержки KPROBE::

  # Instead of having a BUG kill your shell, have it kill "cat":
  cat <(echo WRITE_RO) >/sys/kernel/debug/provoke-crash/DIRECT
