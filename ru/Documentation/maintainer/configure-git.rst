Настройка Git
=============

В этой главе описывается настройка git на уровне сопровождающего (maintainer).

Помеченные тегами ветки, используемые в pull-запросах (см.
Documentation/maintainer/pull-requests.rst), должны быть подписаны
открытым GPG-ключом разработчика. Подписанные теги можно создавать, передавая
``-u <key-id>`` команде ``git tag``. Однако, поскольку *обычно* для проекта
используется один и тот же ключ, можно задать его в конфигурации и использовать
флаг ``-s``. Чтобы задать ``key-id`` по умолчанию, выполните::

	git config user.signingkey "keyname"

Либо отредактируйте файл ``.git/config`` или ``~/.gitconfig`` вручную::

	[user]
		name = Jane Developer
		email = jd@domain.org
		signingkey = jd@domain.org

Возможно, вам потребуется указать ``git`` использовать ``gpg2``::

	[gpg]
		program = /path/to/gpg2

Также может быть полезно указать ``gpg``, какой ``tty`` использовать (добавьте
в rc-файл вашей оболочки)::

	export GPG_TTY=$(tty)
