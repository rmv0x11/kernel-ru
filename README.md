# Документация ядра Linux на русском языке

Перевод официальной документации ядра Linux ([docs.kernel.org](https://docs.kernel.org/))
на русский язык с публикацией на GitHub Pages.

Перевод **точный**, без потери смысла. Технические термины (имена функций, типов,
макросов, путей, команд) **не переводятся** — см. [`glossary.md`](glossary.md).

## Как это устроено

Сайт docs.kernel.org генерируется из reStructuredText-исходников каталога
`Documentation/` дерева ядра с помощью **Sphinx**. Мы:

1. Клонируем дерево ядра на зафиксированной версии (см. [`KERNEL_PIN`](KERNEL_PIN));
2. **Накладываем** переведённые `.rst` из `ru/Documentation/` поверх оригинальных
   (непереведённые файлы остаются английскими — сайт всегда собирается и постепенно
   становится русским);
3. Собираем Sphinx с `language=ru` (русский интерфейс и полнотекстовый поиск);
4. Публикуем на GitHub Pages.

Справочник API, который Sphinx извлекает из комментариев `kernel-doc` в C-исходниках
ядра, в этой версии остаётся на английском (как и во всех апстрим-переводах ядра).

## Структура репозитория

```
ru/Documentation/      переведённые .rst (зеркалит пути каталога Documentation ядра)
glossary.md            контракт перевода: что не переводить, терминология, правила RST
KERNEL_PIN             зафиксированные тег и SHA версии ядра
scripts/setup-venv.sh  создание venv с Sphinx-тулчейном
scripts/build.sh       полная сборка сайта (клон + наложение + sphinx-build)
.github/workflows/     CI: сборка и деплой на GitHub Pages
build/                 рабочий каталог (клон ядра, вывод сборки) — не коммитится
```

## Локальная сборка

```bash
scripts/setup-venv.sh          # один раз: venv + Sphinx
scripts/build.sh               # собрать весь сайт
scripts/build.sh process mm    # собрать только указанные разделы (быстро)
# результат: build/linux/output/index.html
```

Требования: `python3` (>= 3.12), `git`, `rsync`.

## Статус перевода

Перевод ведётся по разделам и публикуется инкрементально. Прогресс отражается
в задачах проекта.
